# クロスドキュメント View Transitions の現在地 — MPA でも滑らか、ただし Firefox 未対応で非 Baseline

> トピック: cross-document View Transitions（MPA）の対応状況｜出典: MDN / CSS-Tricks｜種別: Web 標準の対応状況整理

## 一行サマリー

ページ遷移（MPA）そのものをブラウザがアニメーション化する cross-document View Transitions は、`@view-transition` ルールで同一オリジンのページ間を滑らかに繋げる。Chromium と Safari 18.2+ で使える一方、Firefox が未対応のため Baseline には達していない（＝要注意の先行機能）。同一ドキュメント版が Baseline に達したのとは状況が異なる。

## 🔰 初学者向け（何が・なぜ重要か）

Web サイトには大きく2種類あります。1つは「1枚のページ内で中身だけ差し替える」タイプ（SPA）、もう1つは「クリックで別のページへ実際に移動する」タイプ（MPA、昔ながらの多ページ構成）。後者は移動のたびに画面が一瞬白くなり、繋がりが途切れがちでした。

cross-document View Transitions は、この**ページからページへの移動**を、まるで1枚の画面の中で切り替わったかのように滑らかに見せる仕組みです。ブログの一覧から記事へ、商品一覧から詳細へ——実際にはページが変わっているのに、写真がスッと拡大するように繋がって見えます。ただし今回のポイントは注意喚起でもあります。Chrome と Safari では使えるものの、**Firefox がまだ対応していない**ため、「全ブラウザで安心」の Baseline にはなっていません。使うなら「対応ブラウザでは演出、非対応では通常遷移」という前提で組む必要があります。

## 🧠 専門的解説（技術詳細・設計・使いどころ）

### 技術的詳細（何が変わったか）

- `@view-transition { navigation: auto; }` を遷移元・遷移先の両ドキュメントに置くことで、同一オリジンのページ遷移をアニメーション化する。
- 対応: Chromium 系および Safari 18.2+。**Firefox は未対応**のため、MDN は「一部の主要ブラウザで動かないため Baseline ではない」と明記。
- 同一オリジン（same-origin）が必須。クロスオリジン遷移では発火しない。
- 要素の連続性は same-document 版と同じく `view-transition-name` で対応付け。`::view-transition-*` 疑似要素ツリーへ CSS を当てる。
- 非対応ブラウザでは単に通常のページ遷移になる（プログレッシブ・エンハンスメントとして安全に劣化）。

### 仕組み / 背景（なぜこう設計されたか）

MPA でページ遷移を滑らかにする難しさは、「遷移の瞬間に古いページは破棄され、新しいページが未ロード」という点にある。SPA なら同一ドキュメント内でスナップショットを撮れるが、MPA では**ドキュメント境界をまたいでスナップショットを引き継ぐ**必要がある。ブラウザは遷移元の描画状態を保持し、遷移先の初期描画と対応付けて合成層でアニメーションする。`@view-transition` はこの「両ドキュメントが遷移に合意する」宣言であり、両側に置かないと発火しない設計になっている。

Firefox 未対応が意味するのは、この機能が「使える／使えない」の二分ではなく**段階的採用（プログレッシブ・エンハンスメント）**を前提に設計されている、ということだ。非対応では通常遷移に静かに戻るため、演出が消えるだけでアプリは壊れない。トレードオフは「全ユーザーに一貫した体験を保証できない」点と、same-origin 制約ゆえ外部サイトへの遷移には効かない点。Baseline 化には Firefox 実装が必要で、それまでは「対応ブラウザ向けの付加価値」として扱うのが現実的だ。

### 実務での使いどころ・移行の勘所

コンテンツ主体の MPA（ブログ、EC、ドキュメントサイト）で、一覧↔詳細の連続性を出したい場合に有効。導入は低リスク——両ページに `@view-transition` を置くだけで、非対応ブラウザは自動的に通常遷移へフォールバックする。ただし「演出があること」を前提にした UX 設計（例: 遷移で位置関係を理解させる導線）は避け、あくまで**装飾的な上積み**に留める。`prefers-reduced-motion` 対応は必須。同一オリジン制約があるため、別ドメインの CDN ページやマイクロフロントエンド間遷移には使えない点に注意。Baseline 状況（Firefox 対応の有無）は継続ウォッチが必要。

## 📖 用語解説（このトピックとのつながり）

- **cross-document View Transitions**: ページ遷移（MPA）をアニメーション化する仕組み。本トピックの主役。
- **@view-transition ルール**: 両ドキュメントで遷移に合意する CSS 宣言。`navigation: auto` で有効化。
- **same-origin（同一オリジン）**: 発火の必須条件。別ドメインへの遷移には効かない。
- **プログレッシブ・エンハンスメント**: 対応ブラウザで付加、非対応では通常動作に劣化させる設計。今回の前提。
- **Baseline**: 主要ブラウザで相互運用が成立した状態。Firefox 未対応ゆえ本機能は未到達。

## 影響範囲 / 推奨アクション

- 影響: MPA を作るサイト全般。演出の付加は可能だが全ブラウザ保証は不可。
- 推奨: 装飾的上積みとして導入し、非対応時の通常遷移を前提に設計。`prefers-reduced-motion` 対応。Firefox 対応状況を継続確認。
