# NGINX CVE-2026-42530 — HTTP/3(QUIC)のUse-After-Freeで未認証DoS、条件次第でコード実行も

> トピック: 脆弱性ニュース ｜ 出典: The Hacker News / F5 公式 / SOCRadar ｜ 種別: Use-After-Free（CWE-416）

## 一行サマリー

世界中のWebサーバー/リバースプロキシで使われるNGINXのHTTP/3（QUIC）処理に、解放済みメモリを再利用してしまう欠陥があり、認証不要の遠隔攻撃でワーカープロセスが落ちる（サービス停止）。環境次第ではコード実行の可能性もあり、F5がNGINX 1.31.2で修正した。

## 🔰 初学者向け（何が・なぜ問題か）

NGINX（エンジンエックス）は、Webサイトの「玄関の受付」をする超メジャーなソフトです。訪問者（ブラウザ）からのリクエストを受け取り、裏側のサーバーへ取り次ぎます。世界中の膨大なサイトがこの受付を使っています。

今回の問題は、新しい通信方式「HTTP/3（QUIC）」を有効にしている場合に起きます。通信のやり取りの中で使う「メモ用紙（メモリ領域）」を、いったん「もう使わないから捨てた」のに、その後でまた同じメモ用紙を読み書きしようとしてしまう不具合です（これを Use-After-Free＝解放後使用 と呼びます）。捨てた紙にはすでに別の内容が書かれているかもしれず、受付係が混乱して倒れてしまう（プロセスが落ちる）か、悪意ある人が紙の内容を仕込んでおけば、受付係を操れてしまう恐れがあります。

怖いのは、攻撃に**ログインもパスワードも不要**な点です。HTTP/3で細工した通信を送るだけで受付係を落とせるため、サイトが見られなくなる「サービス停止（DoS）」につながります。状況によってはさらに踏み込んだ乗っ取り（コード実行）も否定できません。

## 🧠 専門的解説（技術詳細・メカニズム・検知対応）

### 技術的詳細（種別・前提条件）

- **CVE/CVSS**: CVE-2026-42530。CVSS v4 9.2、CVSS v3.1 8.1（High）。種別は Use-After-Free（CWE-416）。
- **影響モジュール**: `ngx_http_v3_module`（HTTP/3 / QUIC 実装）。**HTTP/3 を有効化している構成のみ**が対象（デフォルト無効）。
- **前提条件**: 認証不要・遠隔。攻撃者が特別に細工したHTTP/3セッションでQPACKエンコーダストリームを再オープンさせることでトリガ。
- **影響バージョン / 修正版**: NGINX Open Source 1.31.0〜1.31.1 が脆弱。**1.31.2**（2026-06-17 リリース）で修正。1.31.2 以降は非該当。
- **影響**: ワーカープロセスの再起動＝DoS。F5/各社の評価では「一部のシナリオ・配備条件ではコード実行の可能性」も指摘（断定はされていない＝要確認）。同時に CVE-2026-42055 も修正対象。
- **悪用状況**: 本稿時点で広範な実環境悪用の確証は要確認。ただし対象がエッジ基盤で影響範囲が広いため緊急度は高い。

### メカニズム（攻撃が成立する仕組み）

1. **QUIC/QPACKの状態管理**: HTTP/3 はヘッダ圧縮に QPACK を用い、エンコーダストリームの状態をメモリ上のオブジェクトで管理する。
2. **解放と再オープンの競合**: 細工したセッションでQPACKエンコーダストリームを「再オープン」させると、すでに解放済み（free）のオブジェクトへの参照が残ったまま再利用されてしまう。これが Use-After-Free の本体。
3. **握れるプリミティブ**: 解放後メモリへのアクセスにより、(a) クラッシュ＝ワーカー再起動（DoS）が確実に得られる。(b) 解放済み領域に攻撃者制御のデータが再割り当てされる条件が揃えば、メモリ破壊を通じた制御奪取＝コード実行に橋渡しされうる。
4. **被害への橋渡し**: NGINX はエッジ（インターネット最前線）に置かれるため、DoSは即サービス断、コード実行に至れば内部への侵入起点になる。

一般原理として、Use-After-Free は「オブジェクトのライフサイクル管理（誰がいつ解放するか）の不整合」から生じる典型的メモリ安全性欠陥で、HTTP/3 のような複雑な非同期ストリーム多重化処理はこの種のバグの温床になりやすい。武器化の具体手順・オフセット等はここでは扱わない（概念レベル）。

### 検知・暫定緩和・恒久対策

- **検知の着眼点**: NGINXワーカーの異常な再起動/クラッシュログ（`worker process ... exited on signal`）、HTTP/3 接続の急増や異常なQPACKストリーム操作、QUIC(UDP 443)宛の不審トラフィック。
- **暫定緩和**: パッチ適用までは **HTTP/3 / QUIC を無効化**（`listen ... quic` 設定を外す）。UDP 443 のアクセス制限。エッジでのレート制限。
- **恒久対策**: NGINX Open Source を **1.31.2 以降**へ更新（商用 NGINX Plus は対応リリースを確認）。CVE-2026-42055 も同時に解消されるため併せて適用。

## 📖 用語解説（このニュースとのつながり）

- **NGINX**: 世界的に普及するWebサーバー/リバースプロキシ。本件はその「最前線の受付」自体の欠陥。
- **HTTP/3 / QUIC**: UDP上で動く新世代のHTTP。低遅延が利点だが実装が複雑で、本件の不具合はこのHTTP/3処理に存在する。
- **QPACK**: HTTP/3 のヘッダ圧縮方式。本件はQPACKエンコーダストリームの再オープン処理で発生。
- **Use-After-Free（CWE-416）**: 解放したメモリを再び使う欠陥。クラッシュやコード実行の原因になる。
- **DoS（サービス妨害）**: サービスを停止・利用不能にする攻撃。本件では確実に得られる被害。

## 影響範囲 / 推奨対応

HTTP/3 を有効化した NGINX Open Source 1.31.0〜1.31.1。エッジ基盤のため影響範囲が広い。**1.31.2 以降へ更新**を最優先で実施。即時に更新できない場合は HTTP/3(QUIC) を無効化してリスクを除去。コード実行可否は配備条件依存で「要確認」だが、未認証DoSだけでも事業影響が大きいため緊急扱いが妥当。
