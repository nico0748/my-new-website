# Linux カーネル「pedit COW」CVE-2026-46331（tc act_pedit のページキャッシュ汚染でroot昇格）

> トピック: 脆弱性ニュース ｜ 出典: The Hacker News / Red Hat (RHSB-2026-008) / TuxCare ｜ 種別: 局所的権限昇格（CWE-787 境界外書き込み / 不完全なCOW）

## 一行サマリー

Linux のトラフィック制御サブシステム（tc）の `act_pedit` に境界外書き込みの欠陥（CVE-2026-46331、通称「pedit COW」）があり、ローカルの一般ユーザがページキャッシュ上の共有メモリを汚染して root に昇格できる。検証済み PoC が公開済み。v5.18〜v7.1-rc6 が影響、v7.1-rc7 で修正。RHEL 8/9・Debian 13・Ubuntu 等が影響。

## 🔰 初学者向け（何が・なぜ問題か）

Linux には、ネットワークの流れを整える「交通整理係」のような機能（tc = traffic control）があり、その中の `act_pedit` は**通っていくパケットの中身を書き換える**役割を持ちます。

問題は、この書き換え係が「どこまで書いてよいか」のチェックを正しく行わず、本来書き換えてはいけないメモリ領域まではみ出して書き込んでしまう点です。OS は、ディスク上のファイル（たとえばログイン用プログラム `/bin/su`）を高速化のためにメモリ上に「読み取り専用のコピー」として保持しています（ページキャッシュ）。攻撃者はこの欠陥を悪用し、本来は触れないはずのこの共有コピーを**書き換えて**しまいます。

`/bin/su` のような特権プログラムの中身を差し替えられると、攻撃者は自分の仕込んだコードを最高権限（root）で実行させられます。たとえるなら、図書館の「閲覧専用の原本」に、こっそり鉛筆で命令を書き加え、次に誰かがその原本を読むと攻撃者の指示通りに動いてしまう、という状況です。一般ユーザの権限さえあれば root を奪える「局所的権限昇格」で、公開 PoC もあるため放置は危険です。

## 🧠 専門的解説（技術詳細・メカニズム・検知対応）

### 技術的詳細（種別・前提条件）

- **CVE**: CVE-2026-46331（通称「pedit COW」）。Red Hat 整理は RHSB-2026-008。
- **種別**: 境界外書き込み（CWE-787）に起因する不完全な Copy-on-Write（部分COW）によるページキャッシュ汚染 → 局所的権限昇格（LPE）。
- **影響範囲**: Linux カーネル v5.18 〜 v7.1-rc6（混入コミット 899ee91156e5）。修正は v7.1-rc7。RHEL 8/9（および 10・AlmaLinux 8 等）、Debian 13(Trixie)、Ubuntu 18.04〜26.04 が影響と報告。
- **前提条件**: ローカルアカウント。`act_pedit` が到達可能で、かつ **非特権ユーザ名前空間（unprivileged user namespaces）が利用可能**な構成。攻撃者はそこから CAP_NET_ADMIN を取得する。
- **悪用状況**: 著者検証済みの公開 PoC あり（実環境での広範な悪用報告は要確認）。
- **修正/緩和**: ベンダ提供の修正カーネルへ更新＋再起動。回避策として `act_pedit` モジュールのロード禁止。

### メカニズム（攻撃が成立する仕組み）

本件は「COW（コピーオンライト）境界の計算ミス」という、ファイルバック型ページキャッシュを書き換えてしまうクラスの脆弱性（DirtyCOW 系譜の現代版）。概念レベルで分解する。

1. **不正確な COW 範囲計算**: パケット書き換えを行う `tcf_pedit_act()` は、編集前に「ここから先は自分専用にコピーしてから書く（COW）」範囲を**一度だけ**算出する。しかし、実行時に typed key が追加するランタイムオフセットを取りこぼし、計算した範囲と実際に書き込む位置がずれる。
2. **部分COW＝共有ページへの書き込み**: 結果として、本来「私的化（コピー）」されるべきページの外側に書き込みが及び、**私的化していない共有ページ（ページキャッシュ）に直接書き込んでしまう**。読み取り専用であるはずのファイルバックページが汚染される。
3. **攻撃者が握るプリミティブ**: これは「ページキャッシュ上の特定バイトを任意内容で書き換える」という強力なプリミティブを与える。攻撃者は事前に特権バイナリ（例: `/bin/su`）のページをキャッシュに載せておき、その ELF イメージのエントリポイント付近を書き換える。
4. **被害への橋渡し**: 汚染した特権バイナリが次に実行された瞬間、差し替えられた処理（概念的には setgid(0)→setuid(0)→シェル起動の流れ）が root 権限で動き、root シェルに到達する。非特権ユーザ名前空間で CAP_NET_ADMIN を得る部分が、一般ユーザが tc を操作可能にする前提を満たす。

一般原理として、(a) COW の境界・長さ計算は「実行時に変化しうるオフセット」を必ず織り込む必要があること、(b) 書き込み前に対象ページが確実に私的化されたか検証する設計が要ること、(c) 非特権ユーザ名前空間が攻撃面（CAP_NET_ADMIN 取得）を広げることを示す。武器化に必要な正確なオフセット・PoC コードは記載しない。

### 検知・暫定緩和・恒久対策

- **検知の着眼点**: 非特権プロセスによる `act_pedit`/tc 操作、unprivileged user namespace の予期せぬ生成、`/bin/su` 等特権バイナリ実行直後の不審な権限昇格、EDR による ELF ページ整合性/実行系列の異常。
- **暫定緩和（パッチ前）**: `act_pedit` モジュールのロードをブラックリスト化し自動ロードを抑止。可能なら非特権ユーザ名前空間を制限（`kernel.unprivileged_userns_clone=0` 等、AppArmor 名前空間制限）。**業務影響の事前検証は要確認**。
- **恒久対策**: ベンダ修正カーネル（RHEL 8/9/10、AlmaLinux 8、Debian 13 等）へ更新し再起動。
- **運用示唆**: マルチテナント/コンテナ基盤は非特権名前空間が有効な場合が多く優先対応。後述の DirtyClone（CVE-2026-43503）と同系統の脅威として一括で点検すると効率的。

## 📖 用語解説（このニュースとのつながり）

- **Copy-on-Write（COW）**: 共有メモリを「書き込む直前に自分専用へ複製する」省メモリ技法。本件は複製範囲の計算ミスで共有側を汚染してしまう。
- **ページキャッシュ（ファイルバックメモリ）**: ディスク上のファイルを高速化のためメモリに保持する読み取り専用コピー。これを書き換えられると `/bin/su` 等が乗っ取られる。
- **境界外書き込み（CWE-787）**: 許可された範囲を超えてメモリに書き込む欠陥。本件の根本原因で、共有ページ汚染の起点。
- **act_pedit / tc（traffic control）**: パケットを書き換えるカーネル機能。書き換え処理の COW 計算欠陥が今回の舞台。
- **非特権ユーザ名前空間**: 一般ユーザが限定的な特権（本件では CAP_NET_ADMIN）を得る仕組み。攻撃の前提を満たすため、無効化が緩和になる。

## 影響範囲 / 推奨対応

v5.18〜v7.1-rc6 を含む幅広い Linux ディストリ（RHEL 8/9、Debian 13、Ubuntu 等）でローカルユーザが root を取得可能。とくに非特権名前空間が有効なマルチテナント/コンテナ/共有サーバが高リスク。修正カーネルへ更新・再起動を最優先とし、未適用の間は `act_pedit` ロード禁止と非特権名前空間の制限で攻撃面を縮小する。
