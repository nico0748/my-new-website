# CSS contrast-color() が実用段階へ — 背景に応じて黒か白かをブラウザが自動選択

> トピック: CSS `contrast-color()`（Chrome 147 等で提供）｜出典: Chrome for Developers ブログ / MDN｜種別: CSS 機能

## 一行サマリー

`contrast-color()` は、指定した色に対して「黒か白か」十分なコントラストが取れる方をブラウザが返す CSS 関数で、Chrome 147 などで利用可能になった。ボタンやバッジの文字色を背景に応じて自動で読みやすくでき、テーマ切替時の可読性担保が JS や手計算なしで書けるようになる。

## 🔰 初学者向け（何が・なぜ重要か）

Web で「色の付いたボタン」に文字を載せるとき、背景が濃ければ白文字、薄ければ黒文字にしないと読みづらい。従来はデザイナーや開発者が背景色ごとに手で「この背景なら文字は黒」と決めたり、JavaScript で明るさを計算していた。テーマカラーをユーザーが自由に選べるアプリだと、この組み合わせが無限にあって破綻しがちだった。

`contrast-color()` は、この判断をブラウザに丸投げできる関数。「この背景色に対して、読みやすい方（黒か白）をちょうだい」と書くだけで、ブラウザが自動で選んでくれる。ダークモード・ライトモードやユーザー指定のアクセントカラーでも、文字が背景に埋もれて読めない、という事故を減らせる。アクセシビリティ（誰にとっても読みやすいこと）を「デフォルトで」担保しやすくなるのが大きい。

## 🧠 専門的解説（技術詳細・設計・使いどころ）

### 技術的詳細（何が変わったか）

`contrast-color(<color>)` は引数の色に対しコントラストが確保できる色を返す CSS 関数。Chrome for Developers の告知では Chrome 147 で `contrast-color()` が利用可能とされる（同世代では要素スコープ View Transitions なども導入）。現行仕様の第一段階では戻り値は基本的に黒または白のいずれかで、任意の2色から選ぶ／目標コントラスト比を明示する、といった拡張は仕様の将来課題として議論されている（各ブラウザの正確な対応バージョンと Baseline 状況は MDN/web-platform 情報で要確認）。

### 仕組み / 背景（なぜこう設計されたか）

文字の可読性は前景色と背景色の相対輝度比（コントラスト比）で決まり、WCAG では本文で 4.5:1 などの基準がある。従来これを満たすには、(1) デザインシステムで前景色を固定ペアとして管理、(2) ビルド時に色を計算、(3) ランタイムに JS で輝度計算、のいずれかが必要で、動的テーマとの相性が悪かった。`contrast-color()` は輝度計算をブラウザのレンダリングエンジン側に内在化し、`background: var(--brand); color: contrast-color(var(--brand));` のように宣言的に書けるようにした発想。トレードオフは、初期仕様が「黒か白の二択」に限定され、ブランド配色のニュアンス（濃紺の背景に薄い水色の文字、など）を細かく指定する用途には応えきれない点。まず安全側の実装を出し、後方互換を保ちつつ拡張する漸進的アプローチを取っている。

### 実務での使いどころ・移行の勘所

ユーザーがアクセントカラーを選べる SaaS、タグ/ラベルに任意色を許すアプリ、テーマ切替を持つデザインシステムで効果が高い。既存の「背景色ごとに前景色を手管理」しているトークンを段階的に置換できる。移行の勘所は、未対応ブラウザ向けに従来の固定 `color` をフォールバックとして残す（`color: #000; color: contrast-color(var(--brand));` のカスケード）こと、そして「黒か白の二択」で意匠上不足する箇所は従来手法を併用すること。厳密な AA/AAA 達成が要件なら、実際のレンダリング結果でコントラスト比を検証する運用は引き続き必要。

## 📖 用語解説（このトピックとのつながり）

- **コントラスト比**: 前景と背景の相対輝度の比。可読性の指標で、`contrast-color()` はこれを確保する色を返す。
- **WCAG / AA・AAA**: Web アクセシビリティのガイドライン。本文 4.5:1（AA）等の基準があり、本機能はその達成を助ける。
- **Baseline**: 主要ブラウザで安全に使える状態の目安。本機能の Baseline 到達状況は要確認。
- **相対輝度**: 色の明るさを人間の知覚に合わせて数値化した値。黒/白どちらが読みやすいかの判定に使う。

## 影響範囲 / 推奨アクション

- 動的テーマ・ユーザー指定色を扱う UI で、可読性担保のコードが大幅に簡素化される。
- 未対応環境向けにフォールバックの前景色を必ず併記する。プログレッシブエンハンスメントとして導入するのが安全。
- 二択で足りないニュアンス配色は従来のトークン管理を残す。厳密な準拠が必要なら実測での検証を継続。
