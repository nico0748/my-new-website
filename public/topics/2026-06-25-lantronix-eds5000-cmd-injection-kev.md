# Lantronix EDS5000 に CVSS 9.8 のOSコマンドインジェクション（CVE-2025-67038）— 認証失敗ログ処理でroot実行・KEV悪用中

> トピック: 脆弱性ニュース ｜ 出典: CISA KEV / The Hacker News / Security Affairs / Dataminr ｜ 種別: 不適切な中和（OSコマンドインジェクション, CWE-78）に起因する未認証root権限コード実行

## 一行サマリー

産業オートメーション向けのシリアル↔イーサネット変換装置 Lantronix EDS5000 に、認証失敗時のログ書き込み処理で**ユーザー名をそのままシェルコマンドに連結**してしまうOSコマンドインジェクション脆弱性CVE-2025-67038（CVSS 9.8）が見つかった。注入されたコマンドはroot権限で実行される。CISAは6月23日にKEV（悪用中カタログ）へ登録し、連邦機関に6月26日までの是正を命令。修正版2.2.0.0R1への更新が必須。

## 🔰 初学者向け（何が・なぜ問題か）

EDS5000は、工場やプラントにある**古いシリアル機器（RS-232などの直結ケーブルで通信する機器）を、現代のイーサネット（LAN）につなぐ「変換アダプター」**です。IoTやOT（制御技術）の現場で、レガシー機器をネットワーク経由で管理するために広く使われています。小さな箱ですが、設備とネットワークの境目に座る重要な中継点です。

今回の弱点は、ログイン失敗の記録処理にあります。誰かがログインに失敗すると、装置は「このユーザー名で失敗しました」と記録を残そうとします。ところがその際、**入力されたユーザー名を点検せずに、そのまま装置内部の命令文（シェルコマンド）に貼り付けて実行**してしまうのです。

たとえるなら、受付の記帳ノートに名前を書く欄があるのに、そこに「名前」ではなく**命令文**を書くと、警備員がその命令をそのまま実行してしまうようなものです。攻撃者はユーザー名の欄に細工した命令を仕込むだけで——**ログインに成功する必要すらなく**——装置の最高権限（root）で好きなコマンドを動かせます。CISAはすでに実際の攻撃で使われていると警告しており、ネットワークの境目にある装置が侵入拠点に変わる危険があります。

## 🧠 専門的解説（技術詳細・メカニズム・検知対応）

### 技術的詳細（種別・前提条件）

- **CVSS**: 9.8（Critical）。未認証・ネットワーク経由で悪用可能。
- **CWE分類**: CWE-78（OSコマンドインジェクション）。HTTP RPCモジュールが認証失敗時にログ書き込み用のシェルコマンドを実行する際、**username パラメータを無検証で連結**する。
- **前提条件**: 攻撃者がHTTP RPCインターフェースへネットワーク到達できること。認証成功は不要（認証失敗処理が攻撃面）。
- **影響製品・バージョン**: Lantronix EDS5000 **バージョン 2.1.0.0R3**。**修正版は 2.2.0.0R1**。
- **悪用状況**: **CISAが2026年6月23日にKEV登録**。実環境で悪用中。FCEB機関には**2026年6月26日まで**の是正が命じられている。
- **位置づけ**: EDS5000はシリアル↔イーサネット変換器で、産業オートメーション／IoT／OT機器の管理に用いられる。境界デバイスゆえ侵害時の横展開リスクが高い。

### メカニズム（攻撃が成立する仕組み）

以下は概念レベルの説明であり、武器化可能な手順・PoC等は含めない。

1. **信頼境界の所在**: 装置のHTTP RPCモジュールは、外部からの認証要求を受け付ける。ここで「ユーザー入力（ユーザー名）」と「内部のシェルコマンド」を厳格に分離できるかが信頼境界となる。
2. **危険なログ実装**: 認証が失敗すると、モジュールは失敗を記録するために**シェルコマンドを実行**する。このコマンド文字列の中に、利用者が入力したユーザー名が**サニタイズ（無害化）されずに連結**される。
3. **コマンド注入プリミティブの獲得**: 攻撃者はユーザー名フィールドにシェルのメタ文字や追加コマンドを含めることで、本来のログ用コマンドに自分のコマンドを「割り込ませる」。入力がデータではなく**コードとして解釈**される、コマンドインジェクションの本質である。
4. **root権限での実行**: 当該ログ処理が高権限（root）で動作するため、注入されたコマンドも**root権限で実行**される。攻撃者は装置上で任意コマンド実行という最大級のプリミティブを得る。
5. **未認証で完結する致命性**: 攻撃面が「認証失敗の処理」であるため、攻撃者は正規資格情報を持つ必要がない。境界に座るOT/IoT変換器がそのまま侵入拠点・横展開の足場になる。

脆弱性クラスとしては、「外部入力を検証せずシェルコマンドへ連結する古典的OSコマンドインジェクション」であり、組み込み／OT機器の管理機能で繰り返し発生する実装欠陥に該当する。ログ処理という"地味な経路"が高権限実行に直結する点が教訓的である。

### 検知・暫定緩和・恒久対策

- **検知の着眼点**: HTTP RPCエンドポイントへの異常な認証失敗要求（ユーザー名にメタ文字・コマンド片を含む）、装置からの予期しないアウトバウンド通信や子プロセス生成、OTネットワーク上の変換器からの不審なスキャン・横展開。
- **暫定緩和（パッチ前）**: 装置の管理インターフェースを**インターネットから遮断**し、運用に必要な管理セグメントのみに限定。OT網をIT網から分離。即時更新できない機器は隔離。
- **恒久対策**: **EDS5000 を 2.2.0.0R1 へアップデート**。露出していた機器は侵害前提で点検（設定改ざん・不審アカウント・常駐プロセス）。
- **運用示唆**: KEV登録案件はBOD期限（6/26）を待たず最優先対応。境界OT機器は「管理面を晒さない」を恒常原則に。

## 📖 用語解説（このニュースとのつながり）

- **EDS5000 / シリアル↔イーサネット変換器**: 古いシリアル機器をLANにつなぐ中継装置。意味＝「機器とネットの変換アダプター」。本件＝境界デバイスが乗っ取られうる。
- **OSコマンドインジェクション（CWE-78）**: 入力がシェル命令として実行される欠陥。本件＝ユーザー名が無検証で連結される。
- **HTTP RPCモジュール**: 装置の遠隔手続き呼び出し機能。本件＝認証失敗のログ処理が攻撃面。
- **サニタイズ（無害化）**: 危険文字を除去・エスケープする処理。本件＝これが欠けている。
- **KEV / BOD**: CISAの悪用中カタログと連邦機関への是正命令。本件＝6/26までの対応義務。

## 影響範囲 / 推奨対応

影響を受けるのは、Lantronix EDS5000 v2.1.0.0R3 を運用する産業オートメーション／IoT／OT環境。特に管理インターフェースをネットワークに露出している現場が高リスク。推奨対応は、(1) 2.2.0.0R1への即時更新、(2) 管理インターフェースのインターネット露出遮断とOT網分離、(3) 既に悪用観測ありのため露出機器の侵害調査、(4) 認証失敗ログの異常・装置からの不審通信の監視。
