# 東芝/Dynabook製PCの「Generic IO & Memory Access」ドライバ IOCTLアクセス制御不備 CVE-2026-56129（JVN・国内）

> トピック: 脆弱性ニュース ｜ 出典: JVN（JPCERT/CC調整）/ Vulnerability-Lookup ｜ 種別: 公開IOCTLのアクセス制御不備（CWE-782）→ 物理メモリアクセス→ 権限昇格

## 一行サマリー

東芝・Dynabook製PC（2009〜2016年提供）に搭載される「Generic IO & Memory Access」ドライバ v2.2 が、IOCTLインタフェースのアクセス制御不備により、**管理者権限を持たないログインユーザーでも物理メモリへ直接アクセス可能**。CVSSは5.5（Medium）。JPCERT/CC経由でJVN公開、国内ユーザーに広く関係。

## 🔰 初学者向け（何が・なぜ問題か）

PCには「ドライバ」という、OSとハードウェアの橋渡しをする部品があります。中には**物理メモリ（PCの作業机そのもの）に直接触れる強力なドライバ**があり、本来は管理者だけが使える“特別な工具”であるべきものです。

今回の不具合は、その特別な工具に**鍵がかかっておらず、管理者でない一般ユーザーでも使えてしまう**こと。具体的には「IOCTL」という、アプリからドライバへ命令を送る窓口に入館証チェックがなく、誰でも物理メモリを読み書きできる状態です。

物理メモリにはパスワードや暗号鍵、他人の作業内容など機微情報が一時的に置かれます。一般ユーザーがそこを覗ければ、**管理者の権限を奪う（権限昇格）・他人の秘密を盗む**踏み台になります。単体で遠隔から乗っ取られるわけではありませんが、**すでにPC内に入り込んだマルウェアや内部の悪意ある利用者が“最後のひと押し”で管理者に化ける**のに使える、という危険です（いわゆる脆弱なドライバの悪用＝BYOVD的な悪用にもつながる類型）。

## 🧠 専門的解説（技術詳細・メカニズム・検知対応）

### 技術的詳細（種別・前提条件）

- **CVE**: CVE-2026-56129 / **JVN**: JVNVU#91051826（JVNDB-2026-020742）/ **CVSS**: 5.5（Medium）/ **種別**: 公開IOCTLのアクセス制御不備（Exposed IOCTL with Insufficient Access Control、CWE-782）。
- **対象**: 東芝・Dynabook製PCに搭載の「Generic IO & Memory Access」ドライバ **バージョン2.2**（2009〜2016年に提供されたPC）。
- **前提条件**: 攻撃者が当該PCに**ローカルでログイン済み（一般権限で可）**。遠隔単独では成立しない（ローカル特権昇格型）。
- **対策**: 開発元がJPCERT/CC経由で告知。利用者は更新/該当ドライバの無効化・除去など（JVN記載の対処を**要確認**）。
- **悪用状況**: 現時点で広範な実環境悪用の報告は確認できず（**要確認**）。ただし「脆弱な署名済みドライバ」は持ち込み悪用（BYOVD）の素材になりやすく軽視は禁物。

### メカニズム（攻撃が成立する仕組み）

「非特権ユーザーモードのアプリ ↔ カーネル特権で動くドライバ」という最重要の信頼境界で、入口（IOCTL）の権限検証が欠落していることが核心。

1. **アクセス制御欠落（CWE-782）**: ドライバが公開するIOCTLハンドラに、呼び出し元の権限（管理者か否か）を確認するチェックが無い。デバイスオブジェクトのDACL/必要権限の設定不備により、一般ユーザーがハンドルを開いて命令を送れる。
2. **危険なプリミティブの露出**: このドライバは物理メモリの読み書き（IO/メモリアクセス）という極めて強力な機能を提供する。本来カーネル特権でしか許されない操作が、検証なしで非特権プロセスに開放される。
3. **物理メモリ操作→権限昇格への橋渡し**: 攻撃者は物理メモリの任意読み書きプリミティブを使い、カーネル構造体（トークン/特権ビット等）の書き換えや、機微データ（資格情報・鍵）の窃取を通じてSYSTEM/管理者へ昇格しうる。「任意物理メモリ書き込み」は事実上カーネル任意コード実行に等しい強さを持つ。
4. **連鎖の位置づけ**: 初期侵入後の特権昇格段、または正規署名ドライバを持ち込んで悪用するBYOVD型の素材として機能する。

脆弱性クラス（公開IOCTLのアクセス制御欠落による物理メモリ露出）の一般原理を示すにとどめ、**具体的なIOCTLコード・オフセット・PoCは記載しない**。

### 検知・暫定緩和・恒久対策

- **検知の着眼点**: 非特権プロセスによる当該ドライバデバイスへのハンドルオープン/IOCTL発行、未知プロセスからの物理メモリアクセス、EDRでのドライバロード/トークン改ざん検知、脆弱ドライバ持ち込み（BYOVD）痕跡。
- **暫定緩和**: 不要なら**該当ドライバ（Generic IO & Memory Access v2.2）を無効化・削除**。Windowsの脆弱ドライバブロックリスト（HVCI/WDAC）で当該ドライバの読み込みを抑止。標準ユーザー運用と最小権限の徹底。
- **恒久対策**: JVNの案内に従い更新/除去。古い機種（2009〜2016年）でサポート状況により残存しやすいため、資産棚卸しで該当機を特定。

## 📖 用語解説（このニュースとのつながり）

- **IOCTL**: アプリからドライバへ命令を送る窓口。今回はここの権限チェックが無かった。
- **アクセス制御不備（CWE-782）**: 公開IOCTLに必要な権限確認が欠けていること。本件の根本原因。
- **物理メモリアクセス**: PCのメモリを直接読み書きする強力な操作。鍵・資格情報の窃取やカーネル改ざんに直結。
- **権限昇格（LPE）**: 一般ユーザーが管理者/SYSTEMへ昇格すること。本件の最終被害。
- **BYOVD**: 正規署名された脆弱ドライバを持ち込んで悪用する手口。古い署名ドライバは格好の素材になる。

## 影響範囲 / 推奨対応

2009〜2016年提供の東芝・Dynabook製PCで当該ドライバが残るものが対象。国内に相当数が現存しうる。**該当ドライバの有無を点検し、不要なら無効化/削除、脆弱ドライバブロックリストで読み込み抑止、標準ユーザー運用を徹底**すること。遠隔単独悪用ではないが、侵入後の昇格・BYOVD素材として実戦的価値があるため放置は避ける。
