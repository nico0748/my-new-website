# Bun 1.3 — ランタイム・パッケージマネージャ・バンドラ・テストを1バイナリに、Redis/SQLite 内蔵

> トピック: Bun 1.3｜出典: Bun 公式｜種別: JS ランタイム/ツールチェーン更新

## 一行サマリー

Bun 1.3（最新は 1.3.11 系）は、ランタイム・パッケージマネージャ・バンドラ・テストランナーを単一バイナリに統合したオールインワンで、JavaScriptCore 上で低メモリ・高速コールドスタートを実現し、標準の SQLite ドライバに加え組み込み Redis クライアントを備える。

## 🔰 初学者向け（何が・なぜ重要か）

JavaScript/TypeScript でサーバーやツールを作るとき、普通はいくつもの道具を寄せ集めます。コードを動かす実行環境（Node など）、ライブラリを入れる道具（npm）、ファイルをまとめる道具（バンドラ）、テストを回す道具――と、別々のツールを組み合わせるのが当たり前でした。

**Bun（バン）** は、これらを **1つの実行ファイルに全部入り** にしたのが最大の特徴です。「実行・インストール・バンドル・テスト」を bun ひとつで完結できるので、環境構築がぐっとシンプルになります。しかも Bun は起動が速く、メモリも少なめ。サーバーレス（使うときだけ起動する方式）のように「素早い立ち上がり」が効く場面で有利です。

1.3 ではさらに、データベース（SQLite）や高速キャッシュ（Redis）に繋ぐ機能を最初から内蔵しました。追加ライブラリを入れなくても、すぐデータの読み書きが始められる――「電池付き（batteries included）」の便利さが増した、というアップデートです。

## 🧠 専門的解説（技術詳細・設計・使いどころ）

### 技術的詳細（何が変わったか）

- Bun 1.3 系の最新は **1.3.11**。ランタイム＋パッケージマネージャ＋バンドラ＋テストランナーを単一バイナリに統合。
- エンジンは V8 ではなく **JavaScriptCore（JSC）** を採用。低メモリ・高速コールドスタートが特徴。
- 標準で **SQLite ドライバ** を内蔵。1.3 で **組み込み Redis クライアント** を追加。
- Node.js 互換の向上を継続（多くの npm パッケージが動作）。具体的な API 差異・互換範囲は公式で要確認。

### 仕組み / 背景（なぜこう設計されたか）

Bun の設計思想は「JS 開発のツールチェーン断片化を1バイナリで解消する」ことにある。Node 陣営では実行=Node、依存管理=npm/pnpm/yarn、バンドル=esbuild/webpack/Rollup、テスト=Jest/Vitest と役割ごとに別プロジェクトが担う。柔軟だが、設定ファイルの乱立・ツール間のバージョン整合・起動オーバーヘッドという税がかかる。Bun はこれらを内製・統合し、しかも Zig（低レベル・高速な言語）で実装することで、各段階のオーバーヘッドを削る。

エンジンに JavaScriptCore を選んだのが Node（V8）との大きな分岐点だ。JSC は起動が速くメモリフットプリントが小さい傾向があり、これが「サーバーレスのコールドスタート」で効く。関数がリクエストごとに起動・破棄されるサーバーレスでは、起動の速さとメモリがコストと体感レイテンシに直結するからだ。SQLite/Redis の内蔵も同じ「摩擦を減らす」思想で、依存追加なしにデータ層へ即接続できる。トレードオフは、V8 前提の一部ネイティブアドオンや、V8 固有挙動に依存したコードの互換性リスク。Node は成熟と広大なエコシステム・LTS 保証で勝り、Bun は速度と統合体験で勝る、という住み分けになっている。

### 実務での使いどころ・移行の勘所

サーバーレス関数、CLI ツール、スクリプト、プロトタイプ、テスト高速化を狙う開発ワークフローで刺さる。特に「起動の速さ」「セットアップの単純さ」を重視する場面。移行では (1) 使用 npm パッケージ（特にネイティブアドオン）の Bun 互換確認、(2) Node 固有 API への依存点検、(3) CI での挙動差検証を推奨。既存の大規模本番を丸ごと乗り換えるより、新規サービスや周辺ツール（テスト・スクリプト）から Bun を採り入れて知見を貯めるのが安全。安定・長期保証が最優先の基幹系は Node LTS を維持する判断も妥当。

## 📖 用語解説（このトピックとのつながり）

- **オールインワンランタイム**: 実行/依存管理/バンドル/テストを1つに統合したツール。Bun の核。
- **JavaScriptCore（JSC）**: Safari 由来の JS エンジン。低メモリ・高速起動で Bun が採用。
- **コールドスタート**: 未起動状態からの初回起動遅延。サーバーレスで重要、Bun が得意とする領域。
- **Zig**: Bun 実装に使われる低レベル言語。オーバーヘッド削減に寄与。
- **batteries included**: DB/キャッシュ等を標準搭載する思想。1.3 の Redis/SQLite 内蔵がこれ。

## 影響範囲 / 推奨アクション

- 影響: JS/TS のサーバー・ツール開発。特にサーバーレス、CLI、テスト高速化を求めるチーム。
- 推奨: 新規サービスや周辺ツールから Bun を試験導入。npm/ネイティブアドオン互換と Node 固有 API 依存を確認。基幹系は Node LTS 継続も選択肢。
