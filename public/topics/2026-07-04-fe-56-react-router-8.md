# React Router 8 リリース ― Remix 由来の Framework Mode を統合、年1回メジャーの安定運用へ

> トピック: 開発トピック｜出典: remix.run / reactrouter.com｜種別: frontend

## 一行サマリー
React Router 8 が 2026-06 にリリースされ（Netlify は 2026-06-18 に対応表明）、Remix で培われたフルスタック機能を「Framework Mode」として本体へ統合、さらに年1回のメジャーリリースという予測可能な更新サイクルへ移行した。React のルーティング基盤の事実上の標準が、より安定した運用フェーズに入った節目。

## 🔰 初学者向け（何が・なぜ重要か）
React でページを切り替える仕組み（URL と画面の対応づけ）を担うのが React Router です。地図アプリで「今どこにいるか（URL）」と「表示する景色（画面）」を結びつける案内係のような役割です。

これまで React の世界には、シンプルな案内係の「React Router」と、その上にデータ取得やサーバー機能まで盛り込んだ豪華版の「Remix」という2系統がありました。使う側は「どっちを選べばいいの？」と迷いがちでした。React Router 8 は、この豪華版の機能を本体に取り込み、必要な人だけが「Framework Mode（フルスタック機能を有効にした状態）」として使えるようにしました。案内係とツアーガイドが一つのアプリにまとまり、旅の目的に応じて機能を足せるイメージです。加えて「毎年1回だけ大きな更新をする」と決めたので、いつ・どれくらい変わるかが読みやすくなり、長期運用の安心感が増しました。

## 🧠 専門的解説（技術詳細・設計・使いどころ）
### 技術的詳細（何が変わったか）
- React Router **8** が 2026-06 にリリース（Netlify が 2026-06-18 に対応、要確認で正確なリリース日は公式 CHANGELOG 参照）。
- Remix v0.x〜2.x で確立した API・データ規約が **Framework Mode** として React Router 本体に統合。ライブラリ的な使い方（Declarative / Data モード）とフレームワーク的な使い方（Framework モード）を一つのパッケージで提供。
- **年1回のメジャーリリース**サイクルを正式採用し、破壊的変更を予測可能な形で小刻みに導入する方針へ。
- 一方で **Remix 3（beta）** は React 依存を外し、ゼロ依存のフルスタック JS フレームワークとして別路線で進化する（React Router とは役割分担）。

### 仕組み / 背景（なぜこう設計されたか）
React Router と Remix は元々「同じチームが作る、実質同根のプロジェクト」でした。Remix はいわば React Router の“実験用フィーチャーブランチ”で、そこで検証されたローダー／アクションによるデータ取得規約やネストルーティングが成熟すると、本体へ還流されてきました。React Router 8 はこの還流を「Framework Mode」という形で正式化したものです。

なぜ統合するのか。ユーザーにとっての選択の複雑さ（どちらを使うか）を減らし、React 側の Server Components や Suspense と足並みを揃えたルーティング基盤を一本化するためです。トレードオフとして、単純な SPA しか作らない利用者にはフルスタック機能は不要なため、モード分離で「使う機能だけ有効化する」設計を採り、バンドルや学習コストの肥大化を避けています。年1回メジャーという規律も、エコシステム（Next.js 等が内部で React Router を使う場合含む）が更新に追随しやすくするための意図的な設計です。

### 実務での使いどころ・移行の勘所
- 既存の React Router v7 プロジェクトは、Declarative/Data モードのまま段階的に 8 へ更新できる想定。Framework Mode は新規のフルスタック用途で選択。
- Remix 2 からの移行者は、Remix の資産が Framework Mode として吸収されている点を確認し、パッケージ再編（`react-router` への集約）に対応。
- 「Next.js か React Router(Framework Mode) か」は、エッジ/RSC 統合の深さ・デプロイ先（Netlify 等）との相性で判断。年次メジャーの安定性を重視する運用には追い風。

## 📖 用語解説（このトピックとのつながり）
- **React Router**: React の標準的ルーティングライブラリ。本トピックの主役で、8 で Framework Mode を統合。
- **Remix**: React Router 上に構築されたフルスタック機能群。その成果が本体へ還流された。
- **Framework Mode**: React Router 8 でフルスタック機能（ローダー/アクション/SSR 等）を有効化した使い方。
- **ローダー / アクション**: ルート単位でデータ取得・更新を宣言する Remix 由来の規約。
- **メジャーリリース**: 破壊的変更を含みうる節目更新。8 は年1回サイクルを採用。

## 影響範囲 / 推奨アクション
React でルーティングを扱う全開発者に関係する。SPA 用途は影響小だが、フルスタック志向のチームは Framework Mode の採否を検討する価値がある。まずは検証ブランチで 8 に上げ、パッケージ集約とモード指定の差分、デプロイ先の対応状況（Netlify 等）を確認してから本番採用するのが安全。
