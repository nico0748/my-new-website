# Ubiquiti UniFi OS の3連鎖脆弱性で認証なしルート権限RCE（CVE-2026-34908/34909/34910）

> トピック: 脆弱性ニュース ｜ 出典: Ubiquiti SAB-064 / Bishop Fox / CISA KEV ｜ 種別: 認証回避＋パストラバーサル＋コマンドインジェクション（RCEチェーン）

## 一行サマリー

Ubiquiti の UniFi OS に3つの脆弱性（うち2つが最高評価CVSS 10.0）が見つかり、これらを鎖のようにつなぐと認証なしで機器のルート権限を奪える。CISA が実環境での悪用を確認して緊急パッチ対象（KEV）に登録済みで、UniFi OS Server 5.0.8 以降への即時更新が必要。

## 🔰 初学者向け（何が・なぜ問題か）

UniFi OS は、企業やオフィスの Wi-Fi・スイッチ・防犯カメラをまとめて管理する Ubiquiti 社の「管理者用コンソール」です。ネットワーク機器の司令塔なので、ここを乗っ取られると建物内の通信・カメラ・設定がまるごと相手の手に落ちます。

今回の問題は、この司令塔の「玄関の受付」に3つの穴が同時に見つかったことです。イメージすると次のような話です。

まず1つ目は「受付の見落とし」。玄関には「社員証を見せた人だけ通す」というルールがあるのに、受付係が名札を見るときと、実際に人を案内するときとで“別の書き方の名前”を見てしまう不備がありました（CVE-2026-34908）。攻撃者は受付には「これは通行フリーの区画あて（=認証不要のURL）」と見える書き方をしつつ、案内係には「実は奥の重要区画あて」と解釈させる、二枚舌のような細工リクエストを送ります。結果として、社員証なしで奥に入り込めてしまいます。

2つ目は「地下通路の通り抜け」。ファイルの置き場所を指定する部分で `../`（一つ上の階層へ戻る記号）を悪用され、本来触れないシステム内部のファイルにアクセスできる不備（パストラバーサル、CVE-2026-34909）。

3つ目は「厨房に自由に指示を出せる」問題。ソフトを更新する仕組みに、更新したいパッケージ名をそのまま OS のコマンドに流し込む欠陥があり、名前に紛れ込ませた“余計な指示”まで実行されてしまう（コマンドインジェクション、CVE-2026-34910）。

この3つは単独でも危険ですが、順番につなぐと「認証を素通り → 内部に潜り込む → 好きなコマンドを実行」という一本道になり、最終的に機器の最高権限（root＝管理者以上の全能権限）まで奪われます。しかも攻撃者はログイン情報を一切持っていなくてよいのが恐ろしい点です。

## 🧠 専門的解説（技術詳細・メカニズム・検知対応）

### 技術的詳細（種別・前提条件）

- **CVE-2026-34908**: 不適切なアクセス制御（認証ゲートウェイのバイパス）／CVSS 10.0 Critical。
- **CVE-2026-34909**: パストラバーサル（認証ゲートウェイ）／CVSS 10.0 Critical。
- **CVE-2026-34910**: 不適切な入力検証によるコマンドインジェクション（package-update サービス）。
- **公開**: 2026-05-21、Ubiquiti Security Advisory Bulletin 064（SAB-064）。5件の脆弱性を含み、うち上記3件が未認証攻撃者にとって最重要。
- **前提条件**: 管理インターフェイスへのネットワーク到達性のみ。認証情報は不要（未認証・リモート）。
- **影響バージョン**: UniFi OS Server で unifi-core 5.0.126（ビルド 5.0.6）以下が該当。ゲートウェイ／コンソール機（UDM、UDM-Pro、UDM-SE、UDM-Pro-Max、UDRシリーズ、Express 7、UNVRファミリ、Cloud Key、Cloud Gateway 等）も対象。
- **修正版**: UniFi OS Server 5.0.8 以降（unifi-core 5.0.153）。機器ファームウェアは 5.1.12 以降で修正。
- **悪用状況**: CISA が実環境での悪用を確認し KEV に登録。連邦機関にはパッチ適用期限が課された。Bishop Fox が解析用の検知ツールを公開（GitHub: BishopFox/CVE-2026-34908-check）。

### メカニズム（攻撃が成立する仕組み）

UniFi OS はサービス群の前段に nginx を置き、サブリクエスト方式で認証を強制している。ここに「生のURI」と「正規化後のURI」を別々に評価するという典型的な信頼境界のズレが存在した。

1. **信頼境界のズレ（parser differential）**: nginx の認証ハンドラは、リクエストの**生（percent-encoded）のURI**を見て「`/api/auth/validate-sso/` で始まるものは認証免除（公開ルート）」と判定する。ところが実際にバックエンドへ振り分けるときは**正規化後のURI**（`%2F`→`/` にデコードし、`../` を畳み込む）を用いる。攻撃者は「生では認証免除プレフィックスに見え、正規化すると認証必須の内部ルートに解決される」URIを作り、認証チェックを素通りさせる（CVE-2026-34908/34909 の本質）。これは同一入力を2つのコンポーネントが**異なる解釈**をすることで信頼境界が崩れる、パーサ差異（request normalization/traversal）系の一般原理そのものである。

2. **ゲートウェイ突破後のプリミティブ獲得**: 認証ゲートウェイを越えると、攻撃者は本来保護されているはずの内部APIに未認証で到達できる。パストラバーサルにより設定ファイル等の読み取り・操作の足場も得る。

3. **コマンド実行へ橋渡し**: 内部到達後、package-update サービスの欠陥（CVE-2026-34910）を突く。このサービスはユーザ指定のパッケージ名を `fmt.Sprintf` で組み立てて `sh -c` 経由で実行するが、入力検証を欠くためシェルのメタ文字（`;` `|` `` ` `` `$()` 等）がそのまま解釈される。攻撃者が握った「任意文字列をコマンド文字列に混入させるプリミティブ」が、そのまま任意コマンド実行＝root RCE という最終被害に直結する。

4. **クラスの一般原理**: 全体は「認証ロジックの信頼境界破り（アクセス制御欠落）」→「特権ヘルパ（更新サービス）の入力検証不備」という2種の弱点を連結した典型的な**未認証RCEチェーン**。単体では権限が限られる欠陥でも、連鎖により被害が段階的に増幅される点が要注意である。

※本稿は概念レベルの説明にとどめ、実際に武器化可能なリクエスト構文・PoC・正確なパス等は記載しない。

### 検知・暫定緩和・恒久対策

- **検知の着眼点**: nginx／プロキシログで、`/api/auth/validate-sso/` 等の認証免除プレフィックスを含みつつ、パーセントエンコードや `../` を伴う不自然なURIを監視。認証を経ていないのに内部APIへ到達しているアクセス、package-update 系エンドポイントへの想定外リクエストを相関分析する。Bishop Fox 公開の検知ツールで脆弱性有無を安全に判定可能。
- **暫定緩和**: 管理インターフェイス（UniFi OS の Web/API）をインターネットへ露出させない。到達元をVPN・管理セグメント・許可IPに限定し、外部からの直接アクセスを遮断する。
- **恒久対策**: UniFi OS Server を 5.0.8 以降（unifi-core 5.0.153）へ、機器ファームウェアを 5.1.12 以降へ更新する。修正は nginx の「生URI vs 正規化URI」比較の追加（認証バイパス封鎖）と、更新パスの入力検証追加（コマンドインジェクション封鎖）で構成される。
- **運用示唆**: エッジ／ネットワーク管理機器はランサムウェア攻撃者の初期侵入口として狙われやすい。管理面の露出最小化とKEV優先のパッチ運用を徹底する。

## 📖 用語解説（このニュースとのつながり）

- **認証ゲートウェイ / サブリクエスト認証**: 各サービスへ通す前に nginx が別途「認証確認用の内部リクエスト」を投げて可否を判断する仕組み。今回はこの判定に使うURIと転送先URIが食い違い、バイパスされた。
- **URI正規化（normalization）**: `%2F`→`/` のデコードや `../` の畳み込みなど、URIを標準形に整える処理。認証判定と転送で正規化のタイミングがズレるとバイパスの温床になる。
- **パストラバーサル（CWE-22）**: `../` で本来の範囲外のファイル階層へ抜け出す攻撃。ここでは内部ファイルへの到達に利用。
- **コマンドインジェクション（CWE-78）**: ユーザ入力がOSコマンドの一部として実行される欠陥。`sh -c` に未検証の文字列を渡した結果、任意コマンドが実行される。
- **RCEチェーン**: 単体では限定的な複数の弱点を連結し、最終的にリモートコード実行へ到達させる攻撃構成。今回は「認証回避＋パストラバーサル＋コマンド注入」の3連鎖。
- **CISA KEV**: 米CISAが「実際に悪用が確認された脆弱性」だけをまとめた優先対応カタログ。登録は“今まさに攻撃されている”ことの強いシグナル。

## 影響範囲 / 推奨対応

UDM／UDM-Pro／UDM-SE／UDM-Pro-Max／UDRシリーズ／Express 7／UNVRファミリ／Cloud Key／Cloud Gateway など、UniFi OS を搭載する広範な機器が対象。特に管理インターフェイスをインターネットに露出している構成は即時危険。UniFi OS Server は 5.0.8 以降、機器ファームウェアは 5.1.12 以降へ速やかに更新し、更新まで管理面をVPN／許可IPの内側へ隔離すること。侵害痕跡（不審なSSH鍵追加、想定外のプロセス／設定変更）の確認も推奨。
