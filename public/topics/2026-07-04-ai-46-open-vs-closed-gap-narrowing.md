# オープン vs クローズドの差が縮小 — DeepSeek V4 が GPT-5.5 級で 34 倍安

> トピック: 開発トピック｜出典: llm-stats｜種別: ai

## 一行サマリー
オープンウェイトの DeepSeek V4 がクローズドな GPT-5.5 に匹敵する性能を、約34分の1のコストで提供したと報告され、オープンとクローズドの実力・価格差が急速に縮まっている（数値は要確認）。

## 🔰 初学者向け（何が・なぜ重要か）
少し前まで、最も賢い AI は大手企業の「クローズド」なモデル（API でしか使えない）に限られ、公開されている「オープン」なモデルは一段劣ると見なされていました。ところが状況が変わりつつあります。

DeepSeek V4 のようなオープンモデルが、トップクラスのクローズドモデルに肩を並べる性能を、桁違いに安いコストで出せるようになったと報告されています。「34倍安い」というのは、同じ仕事をさせたときの費用が数十分の一になりうるということです。これは、これまで高価な API に頼っていた開発者や企業が、自社で動かせる安価なオープンモデルで十分戦えるようになったことを意味します。AI 活用の裾野が一気に広がる転換点です。

## 🧠 専門的解説（技術詳細・設計・使いどころ）
### 技術的詳細（何が変わったか）
llm-stats の集計では、DeepSeek V4 が GPT-5.5 級のベンチ性能を大幅に低いコストで達成したとされる。「34倍安」「GPT-5.5級」といった具体値・比較条件（対象ベンチ、API 価格 vs 自ホスト費用など）は要確認だが、上位オープンモデルがフロンティア近傍に達したという傾向は他ソースとも整合する。

### 仕組み / 背景（なぜこう設計されたか）
効率化技術（MoE によるスパース活性化、蒸留、良質データ、推論最適化）の成熟で、より小さい実効計算で高性能を出せるようになった。オープンウェイトは自ホストで単位コストを圧縮でき、寛容ライセンスなら商用利用も容易。結果としてクローズド API の価格プレミアムが正当化しにくくなっている。

### 実務での使いどころ・移行の勘所
コスト感度の高い大量推論・バッチ処理・オンプレ要件ではオープンモデル自ホストが有力。ただし自ホストは GPU 運用・スケーリング・保守の負担が伴い、レイテンシ SLA や安全対策の作り込みも必要。少量・可変負荷ならマネージド API の方が総コストで有利な場合もあるため、TCO（総保有コスト）で比較する。

## 📖 用語解説（このトピックとのつながり）
- **オープンウェイト**: 重み公開モデル。自ホストで単価を下げられる。
- **クローズドモデル**: API 提供モデル。運用は楽だが単価が高い傾向。
- **MoE（Mixture of Experts）**: 一部の専門家のみ活性化し効率化する構造。低コスト化の一因。
- **TCO（総保有コスト）**: 運用込みの総費用。オープン/クローズド比較の判断軸。

## 影響範囲 / 推奨アクション
モデル調達の意思決定者・大量推論を扱う開発者に大きく影響。API 一択を見直し、自ホストのオープンモデルを TCO・SLA・安全性の観点で比較評価すること。
