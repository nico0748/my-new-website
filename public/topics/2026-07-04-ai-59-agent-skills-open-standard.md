# Agent Skills がオープン標準に ― OpenAI も採用、Anthropic が「MCP に続く2枚目の標準」を狙う

> トピック: 開発トピック｜出典: thenewstack.io / pulsemcp.com｜種別: ai

## 一行サマリー
Anthropic が「Agent Skills」の仕様をオープン標準として公開し、OpenAI もこれを採用した。かつて MCP（Model Context Protocol）をエージェントのツール接続標準に押し上げたのと同じ手法で、今度は「エージェントに“スキル（手順・能力）”を配布・共有する」ための共通規格を業界標準にしようとする動き。エージェント開発の相互運用性が一段進む。

## 🔰 初学者向け（何が・なぜ重要か）
AI エージェントに新しい能力（例：「請求書を処理する手順」）を教えるとき、各社バラバラのやり方だと、片方で作った“できること”を別のエージェントに持っていけません。**Agent Skills** は、この「エージェントに手順・能力を持たせるパッケージ」の書き方を共通ルールにする試みです。

料理に例えると、レシピ（スキル）を全員が同じ書式で書くようにする、という話です。同じ書式なら、あるお店のレシピを別のお店でもそのまま使える。今回はさらに、そのルールを Anthropic だけでなく OpenAI も採用したのが大きい。標準は「みんなが使う」ことで初めて価値が出るからです。以前 MCP（AI が外部ツールを使うための共通規格）が広く普及したのと同じ流れが、今度は「スキル」で起きようとしています。

## 🧠 専門的解説（技術詳細・設計・使いどころ）
### 技術的詳細（何が変わったか）
- Anthropic が **Agent Skills 仕様をオープン標準**として公開。**OpenAI もこれを採用**（2026-06〜07、要確認で一次情報を確認）。
- 位置づけは「MCP がツール接続の標準になったのと同様に、Skills を“能力配布”の標準にする」狙い。スキルの構造（手順・リソース・メタデータ）を規格化し、エージェント間で共有可能にする。
- 関連して、スキルのパッケージ配布を扱う **レジストリ／パッケージマネージャ**的な取り組みも登場（例: 学術的提案 "Skilldex" 等、要確認）。Claude Code 側も `/dataviz` などのスキル追加が進む。

### 仕組み / 背景（なぜこう設計されたか）
エージェントの能力は大きく「モデルの地力」「ツール接続（MCP）」「スキル（特定タスクの手順・知識のまとまり）」に分けられます。MCP が“外部ツールへの配線”を標準化したのに対し、Agent Skills は“そのツールをどう使って何を達成するか”という手順・ノウハウの持ち運びを標準化します。スキルを規格化された形（説明・使う道具・実行手順・スコープ）でパッケージ化すれば、作ったスキルを再利用・共有・配布でき、車輪の再発明を減らせます。

なぜオープン標準か。標準は採用者数がすべてで、単独ベンダー仕様では“持ち運べる”価値が生まれません。Anthropic は MCP で「先に開いて広く採ってもらい、事実上の標準を取る」戦略が奏功した実績があり、Skills でも同じ手を打っています。OpenAI の採用は、この標準に競合の重みが乗ったことを意味し、相互運用の現実味を高めます。トレードオフ／論点として、標準の主導権を誰が握るか（ガバナンス）、複数のスキル・マーケットプレイス（Anthropic/Vercel/OpenAI/Cline 等）が乱立した場合の互換性、そしてスキル配布に伴うセキュリティ（信頼できないスキルの実行リスク）が挙げられます。

### 実務での使いどころ・移行の勘所
- 自社エージェントの定型能力（社内手順、ドメイン特化ワークフロー）をスキルとしてパッケージ化すれば、ツールやモデルを跨いで再利用しやすくなる。
- 標準準拠で書いておくと、将来のベンダー切替や複数エージェント併用時の移植コストが下がる。
- 配布・実行するスキルの出所と権限（スコープ）を検証する運用を最初から組み込む。マーケット乱立期は互換性を見極めて採用。

## 📖 用語解説（このトピックとのつながり）
- **Agent Skills**: エージェントに手順・能力を持たせるパッケージの仕様。本トピックの主役。
- **MCP (Model Context Protocol)**: エージェントが外部ツールへ接続する標準。Skills はその“能力版”という位置づけ。
- **オープン標準**: 誰でも実装できる公開規格。採用者数が価値を決める。
- **スキルレジストリ / パッケージマネージャ**: スキルを配布・共有する仕組み。再利用の基盤。
- **ガバナンス**: 標準の意思決定・管理の主体。乱立や主導権が論点になる。

## 影響範囲 / 推奨アクション
エージェントを構築・運用する AI 開発者全般に影響する。標準準拠でスキルを設計しておくと相互運用と移植性で有利。まずは社内の定型ワークフローをスキル化して再利用性を検証し、配布・実行時のセキュリティ（出所・権限）を運用に組み込むのが推奨アクション。
