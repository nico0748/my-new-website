# Valkey 8.1 — Redis 代替の本命、Redis 8.0 比で3倍超スループットの報告

> トピック: Valkey 8.1（Redis 代替）｜出典: Valkey 公式｜種別: OSS データストア更新

## 一行サマリー

Valkey は Redis のライセンス変更を機に生まれた Linux Foundation 傘下の OSS フォークで、AWS・Google・Oracle が支持し、8.1 では Redis 8.0 比で3倍超のスループットを示すベンチマークも報告される、Redis 置き換えの有力候補。

## 🔰 初学者向け（何が・なぜ重要か）

多くの Web サービスは「よく使うデータを手元に置いて高速に返す」ために、**インメモリデータストア**（メモリ上に置く超高速な保管庫）を使います。その定番が長らく **Redis** でした。キャッシュ、セッション管理、順位表、キューなど用途は幅広いです。

ところが Redis は数年前にライセンスを変更し、「完全な自由ソフトウェアではなくなった」ことで、コミュニティやクラウド各社が反発しました。そこで Redis のコードから分岐（フォーク）して作られた、より自由なライセンスの後継が **Valkey（ヴァルキー）** です。中立的な Linux Foundation の下で開発され、AWS・Google・Oracle といった大手が支援しています。

Valkey は Redis とほぼ同じ使い方ができる（互換）ので、乗り換えの負担が小さいのが強み。さらに 8.1 では性能面も磨かれ、条件次第では Redis より大幅に速いという報告もあります。「Redis の資産をそのまま活かしつつ、自由なライセンスと高性能を得たい」――そんな場面で選ばれています。

## 🧠 専門的解説（技術詳細・設計・使いどころ）

### 技術的詳細（何が変わったか）

- **Valkey 8.1** が Redis 代替として推奨される段階に。ベンチマークで Redis 8.0 比 **3倍超のスループット** を示すとの報告。
- Valkey は Redis のライセンス変更（RSALv2/SSPL への移行）を受けて生まれた OSS フォークで、**Linux Foundation 傘下**。BSD ライセンスを維持。
- AWS・Google・Oracle が採用・支持を表明済み。多くのマネージド Redis 互換サービスが Valkey ベースへ移行。
- Redis とのプロトコル/コマンド互換を重視し、既存クライアント・ツールをそのまま使える設計（細かな互換差は要確認）。

### 仕組み / 背景（なぜこう設計されたか）

Valkey が高スループットを狙う核は「マルチスレッド化された I/O とコマンド処理」にある。オリジナルの Redis は長らく「コマンド実行はシングルスレッド」という設計で、これが一貫性と実装単純さを保つ一方、マルチコアを活かしきれないボトルネックだった（GIL 問題と同型の構図）。Valkey は I/O 処理やコマンドのディスパッチを複数スレッドへ分散し、コア数に応じてスループットをスケールさせる方向で改良を重ねている。これがベンチで大差が出る主因だ。

背景のライセンス経緯も設計思想に直結する。Redis 社が SSPL 等へ移行したのは、クラウド事業者が Redis をマネージド提供して収益化する構図への対抗だった。しかしこれは OSI 承認の自由ソフトウェアではなくなることを意味し、コミュニティとクラウド各社が「中立・自由な後継」を求めて Valkey を立ち上げた。Linux Foundation 下で BSD を維持するのは、特定企業の意向で再びライセンスが変わるリスクを構造的に排除するためだ。トレードオフとして、Redis 本家の新機能（本家独自の拡張）と Valkey の機能が今後分岐していく可能性があり、両者の互換性は永続保証ではない点に留意が要る。

### 実務での使いどころ・移行の勘所

キャッシュ、セッションストア、レート制限、ランキング、軽量キューなど Redis の典型用途はそのまま Valkey で置き換え可能。移行は基本的にエンドポイント差し替えと動作確認で済むことが多いが、(1) 使用中の Redis モジュール/独自拡張が Valkey にあるか、(2) 運用ツール・監視・クライアントライブラリの対応、(3) データ移行（RDB/AOF の互換）を事前確認したい。マネージド利用なら、多くのクラウドが提供する Valkey 互換サービスへ移ることでライセンス懸念とコストの両面でメリットが出る。逆に Redis 本家の最新独自機能に強く依存している場合は、機能差の見極めが必要。

## 📖 用語解説（このトピックとのつながり）

- **インメモリデータストア**: データを主記憶に置いて超高速に読み書きする保管庫。Redis/Valkey が代表。
- **フォーク**: 既存 OSS のコードを分岐して別プロジェクト化すること。Valkey は Redis のフォーク。
- **SSPL / BSD ライセンス**: 前者は制約の強いライセンス（Redis が移行）、後者は自由度の高い OSS ライセンス（Valkey が維持）。
- **マルチスレッド I/O**: 入出力処理を複数スレッドで並列化する設計。Valkey の高スループットの源。
- **RDB / AOF**: Redis 系の永続化形式（スナップショット/追記ログ）。移行時の互換確認ポイント。

## 影響範囲 / 推奨アクション

- 影響: Redis を使う全サービス、特にライセンス懸念やコスト最適化、マルチコアスケールを求めるチーム。
- 推奨: 新規は Valkey を第一候補に検討。既存 Redis はモジュール依存・運用ツール互換・データ移行を確認のうえ段階移行。本家独自機能依存時は機能差を精査。
