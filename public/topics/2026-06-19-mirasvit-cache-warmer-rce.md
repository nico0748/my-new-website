# Mirasvit Full Page Cache Warmer CVE-2026-45247 — Magentoのクッキーで未認証RCE、KEV登録・実環境悪用中

> トピック: 脆弱性ニュース｜出典: CISA KEV / Halo Security / Indusface / Imperva｜種別: 信頼できないデータのデシリアライズ（PHPオブジェクトインジェクション）→未認証RCE

## 一行サマリー

Magento（Adobe Commerce）向け拡張「Mirasvit Full Page Cache Warmer」が、クライアントから送られてくるクッキーを検証せずにPHPの`unserialize()`へ渡すため、認証なしで任意コードを実行できる（CVSS 9.8）。CISAが2026年6月3日にKEV（実際に悪用された脆弱性カタログ）へ登録し、ECサイト改ざん・カード情報窃取の踏み台になり得るため、v1.11.12以降への即時更新が必須。

## 🔰 初学者向け（何が・なぜ問題か）

ネットショップの多くは「Magento」というショッピングサイトの土台ソフトで動いています。今回問題が見つかったのは、その表示を速くするための追加部品（拡張機能）「Mirasvit Full Page Cache Warmer」です。

この部品は、利用者のブラウザに「クッキー」という小さなメモを渡し、次に来たときそのメモを読み返して状態を思い出します。ところが、メモの中身を「信用できる手書きメモ」としてそのまま受け取り、書かれている指示の通りに部品を組み立て直してしまいます。クッキーは利用者（＝攻撃者にもなりうる人）が自由に書き換えられるものなので、攻撃者が細工したメモを送ると、サーバが攻撃者の思い通りの「部品」を組み立て、最終的に攻撃者のプログラムを実行してしまいます。

例えるなら、店員が「お客様が持参したメモに書かれた組み立て図のとおりに、店の裏で機械を組み立てる」運用をしていて、悪意ある客が「金庫を開ける機械」の図を渡せばその通り作ってしまう、という状態です。ログインも不要なので、誰でも一発で店の裏側を乗っ取れてしまう、というのが怖い点です。すでに実際の攻撃が確認されています。

## 🧠 専門的解説（技術詳細・メカニズム・検知対応）

### 技術的詳細（種別・前提条件）

- **CVE**: CVE-2026-45247 / **CVSS v3.1**: 9.8（Critical）/ ベクトル `AV:N/AC:L/PR:N/UI:N/S:U/C:H/I:H/A:H`
- **CWE**: CWE-502（Deserialization of Untrusted Data、いわゆるPHPオブジェクトインジェクション）→ RCE
- **影響製品**: Mirasvit Full Page Cache Warmer for Magento 2（Adobe Commerce）/ **影響バージョン**: 1.11.12 未満 / **修正版**: 1.11.12（2026-05-25公開）
- **根本原因**: セッション状態をクッキーに格納し、リクエストごとにその一部をクラス制限なしでネイティブ`unserialize()`へ渡す。クッキーはクライアント由来のため、復元されるオブジェクトを攻撃者が制御できる。
- **悪用状況**: CISAが2026-06-03にKEV登録（連邦機関の是正期限2026-06-06）。公開ベンダ（Imperva等）が実環境での攻撃観測を報告。
- **前提条件**: 脆弱なバージョンの拡張が稼働し、エンドポイントが到達可能であること。認証は不要。

### メカニズム（攻撃が成立する仕組み）

※以下は脆弱性クラスの一般原理に基づく概念説明。武器化手順・PoC・正確なガジェットチェーンは記載しない。

1. **信頼境界の取り違え**: クッキーはクライアントが完全に制御できる外部入力であり、信頼境界の外側にある。本来は検証・署名・型制限が必須だが、本件はその値を「信頼できる内部状態」として扱う。
2. **無制限デシリアライズというプリミティブ**: クラス許可リストを指定せずに`unserialize()`を呼ぶと、アプリ内に存在する任意のクラスのオブジェクトを攻撃者が再構築できる。これが「オブジェクト生成プリミティブ」を攻撃者に与える。
3. **マジックメソッドの連鎖（POPチェーン）**: 復元時や破棄時に自動で走るマジックメソッド（`__wakeup`/`__destruct`等）を持つクラスを起点に、コード内の既存メソッドを数珠つなぎ（プロパティ・オリエンテッド・プログラミング）にして、ファイル書き込みやコマンド実行といった「使える動作」へ到達させる。
4. **未認証から実行への橋渡し**: 認証を必要としない経路でこのプリミティブを行使できるため、匿名の攻撃者がRCEへ直行する。Magento環境では、Webルートへのスクリプト設置やDB資格情報の窃取を経て、サイト改ざん・スキマー（カード情報窃取コード）の埋め込みに発展しうる。
   - 一般原理: 「外部入力 → 無制限デシリアライズ → POPチェーン → RCE」は、PHP/Java/Python/.NET 各言語に共通する古典的かつ強力なクラス。根治には「そもそも信頼できない入力をデシリアライズしない／許可クラスを厳格に限定する」しかない。

### 検知・暫定緩和・恒久対策

- **検知の着眼点**: 当該拡張のエンドポイントへの異常なリクエスト、クッキー値が`O:`や`a:`で始まる長大なシリアライズ文字列、PHP-FPM/Apacheからの不審な子プロセス、Webルート配下に追加された未知の`.php`ファイルや改変タイムスタンプ、外向きの不審通信。
- **暫定緩和**: 即時更新できない場合は、当該拡張を無効化、もしくはWAFで該当クッキー・エンドポイントへのシリアライズ様パターンを遮断。管理画面・該当URLをIP制限の背後へ。
- **恒久対策**: v1.11.12 以降へ更新。公開実績のある環境は侵害前提で、Webルート・cron・管理者アカウント・支払い周りのコード（スキマー混入）を点検する。

## 📖 用語解説（このニュースとのつながり）

- **デシリアライズ（CWE-502）**: 文字列やバイト列を、プログラム内のオブジェクトへ戻す処理。本件では攻撃者が中身を制御でき、RCEの起点となった。
- **PHPオブジェクトインジェクション**: 攻撃者が制御するシリアライズ文字列をPHPに復元させ、意図しないオブジェクト・処理を発火させる手口。
- **POPチェーン（ガジェットチェーン）**: 既存クラスのメソッドを数珠つなぎにして攻撃を成立させる連鎖。デシリアライズ脆弱性を「実害」に変える橋渡し。
- **KEV**: CISAが「実際に悪用が確認された」と認定した脆弱性カタログ。登録＝今まさに狙われている、の強いシグナル。
- **スキマー**: ECサイトに仕込まれ、入力されたカード情報を盗むコード。Magento改ざんの典型的な最終目的。

## 影響範囲 / 推奨対応

影響は、Mirasvit Full Page Cache Warmer 1.11.12 未満を稼働するすべての Magento 2 / Adobe Commerce サイト。認証不要・CVSS 9.8・KEV登録・実環境悪用というすべての危険条件が揃っているため、最優先で v1.11.12 以降へ更新する。更新後も、公開期間に侵害された可能性を前提に、Webルートのファイル整合性、cron、管理者・APIアカウント、決済処理コードを必ず点検すること。
