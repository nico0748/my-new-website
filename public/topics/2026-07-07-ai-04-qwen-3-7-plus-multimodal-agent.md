# Qwen3.7-Plus — 視覚・深い推論・ツール実行・自律反復を1モデルに、価格は Max の約1/6

> トピック: Alibaba Qwen3.7-Plus の Bailian 投入｜出典: MarkTechPost（一次情報は Alibaba Cloud Bailian 告知、要確認）｜種別: 商用マルチモーダル・エージェントモデルの投入

## 一行サマリー

Alibaba の Qwen チームは2026年6月1日、5月の旗艦 Qwen3.7-Max に続いてマルチモーダル・エージェント版の Qwen3.7-Plus を Bailian プラットフォームに投入し、画像・動画理解、段階的な深い推論、ツール呼び出し、出力の検証・テスト、タスク完了までの自律反復を、Max の約6分の1のトークン単価と100万トークン文脈で提供する。

## 🔰 初学者向け（何が・なぜ重要か）

Qwen（通義千問）は中国 Alibaba の AI モデルシリーズで、オープンソース版の人気も高く、世界の AI 開発で存在感を増しています。今回の Qwen3.7-Plus のポイントは「一人で最後までやり切る AI」を安く提供することです。

従来のチャット AI は「聞かれたら答える」だけでした。Qwen3.7-Plus は、(1) 問題を段階的にじっくり考える、(2) 必要なら自分でプログラムを書いて直す、(3) 外部の道具（API や検索）を呼び出す、(4) 出来上がったものを実際に動かして確かめる、(5) 駄目なら直してやり直す——という一連の流れを自動で繰り返します。料理に例えると、レシピを答えるだけでなく、実際に作り、味見をし、しょっぱければ調整して完成まで持っていく「料理人」型です。

もう一つの狙いは価格です。旗艦の Max と同系統の能力を約6分の1の単価で出すことで、「エージェントを大量に、常時動かす」用途のコストの壁を下げに来ています。読める文脈も100万トークン（本数百冊分の文字量）と大きく、長い資料仕事に向きます。

## 🧠 専門的解説（技術詳細・設計・使いどころ）

### 技術的詳細（何が変わったか）

- **投入時期**: Qwen3.7-Max が2026年5月20日の Apsara Summit（杭州）で発表され、6月1日に Qwen3.7-Plus が Bailian に投入。
- **能力**: 画像・動画理解（マルチモーダル）、深い推論（step-by-step）、自己プログラミング（自コードの記述・修正）、ツール呼び出し、出力の実行・検証、タスク完了までの自律反復ループ。
- **コンテキスト**: 100万トークン。長文書・大規模エージェントワークフロー向け。
- **価格**: Max 比で約1/6のトークン単価（具体的な単価表は要確認）。
- **提供形態**: 2026年6月時点で Qwen 3.7 系は Alibaba Cloud 経由の API 専用（オープンウェイト版は未公開。Qwen チームは従来、後追いでオープン版を出す傾向があるが 3.7 の扱いは要確認）。
- **評価**: Vision Arena で Qwen3.7-Plus-Preview が総合16位、ビジョン分野でラボとして5位相当との集計。

### 仕組み / 背景（なぜこう設計されたか）

Qwen3.7-Plus の設計は「エージェント能力のモデル内在化」という2026年の潮流の中国側代表例である。従来、検証・反復のループは LangGraph などの外部フレームワークがオーケストレーションしてきたが、ループの制御自体をモデルの学習対象にすると、(1) ステップ間の文脈受け渡しロスが消える、(2) ツール選択・撤退判断がモデルの方策として最適化される、という利点がある。OpenAI の Codex 系、Anthropic の agentic 訓練、Unisound U2（07-06 既報）と同じ方向で、「エージェント FW の仕事がモデルに吸われる」圧力が世界的に強まっている。

Max/Plus の2段構えは推論経済学の設計でもある。フロンティア級（Max）は高価で常用に向かず、エージェントは1タスクで数十〜数百回の呼び出しを行うため、単価が1/6なら「反復回数で品質を稼ぐ」戦略が成立する。teacher-student 蒸留や MoE のアクティブパラメータ削減で能力単価を下げる手法が一般的だが、Plus の内部構成は非公開（要確認）。

API 専用での投入は、オープンウェイト路線で名を売った Qwen の商業化フェーズを示す。オープン版（Qwen3.6 系まで）でエコシステムを作り、最先端は自社クラウドで収益化する「open-core」型のモデル戦略である。

### 実務での使いどころ・移行の勘所

- **長文書×視覚のエージェント処理**: 図表入り資料の一括解析、動画コンテンツの構造化、スクリーンショット駆動の UI テストなど、視覚＋反復が要る処理が単一モデルで組める。
- **コスト設計**: 「Plus で反復し、最終確認だけ Max/他社フロンティア」という階層化がコスト効率的。ルーターやフォールバック チェーンに組み込む価値がある。
- **注意点**: Alibaba Cloud 依存となるため、データ所在・輸出規制・社内ポリシーの確認が先決。中国系 API の利用可否は組織によって扱いが分かれる。EU AI Act の GPAI 透明性要件（本日別記事）への対応状況も選定要素になる。
- ベンチマーク（Vision Arena 16位等）はプレビュー時点の値であり、GA 後の再評価を推奨。

## 📖 用語解説（このトピックとのつながり）

- **Bailian（百煉）**: Alibaba Cloud のモデルサービス基盤。Qwen3.7 系の提供窓口。
- **自律反復（autonomous iteration）**: 出力を検証し、失敗なら修正して再試行するループをモデル自身が回すこと。本モデルの中核機能。
- **マルチモーダル**: テキスト以外（画像・動画等）も入力に扱える能力。Plus は動画理解まで含む。
- **MoE（Mixture of Experts）**: 一部の専門家サブネットのみ活性化する構造。能力単価を下げる常套手段（Plus での採用有無は要確認）。
- **open-core 戦略**: 基盤をオープンにし最先端を商用で売るビジネスモデル。Qwen の Max/Plus は商用側。

## 影響範囲 / 推奨アクション

- マルチベンダー戦略のチーム: 視覚系エージェントの選択肢として評価リストに追加（データガバナンス審査とセット）。
- オープンウェイト利用者: Qwen3.6 系までのオープン版と 3.7 API 版の性能差を確認し、依存の置き方を判断。
