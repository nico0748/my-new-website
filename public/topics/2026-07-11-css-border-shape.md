# CSS border-shape が Chrome 147 に — 四角以外の「箱の形」そのものを CSS で定義する

> トピック: CSS `border-shape`（Chrome 147）｜出典: Chrome for Developers・una.im・CSS-Tricks｜種別: Web 標準 / CSS

## 一行サマリー

多角形・円・任意パスなど、四角形以外の枠（ボーダー）を CSS だけで作れる `border-shape` プロパティが Chrome 147 で利用可能になった。要素を単にマスクする `clip-path` と違い、`border-shape` は「箱そのものの形」を再定義してボーダーを描き、内側だけをクリップする。ストロークする変種と、2 つの形状の間を塗る変種の 2 系統を持つ。

## 🔰 初学者向け（何が・なぜ重要か）

Web の要素は基本「四角い箱」でできている。角を丸める `border-radius` はあったが、「五角形の枠」「切り欠きのあるチケット型」「星形の縁取り」のような、四角から大きく外れた枠を作るのは難しかった。SVG を重ねたり、画像を使ったり、複雑な小細工が必要だった。

`border-shape` は、その「箱の輪郭」を CSS で自由に指定できるようにする。円、楕円、多角形、さらには `shape()` 関数や SVG のパス文字列で、好きな形の枠を描ける。たとえるなら、これまで既製の四角い額縁しか選べなかったのが、額縁の形を自分でデザインできるようになったイメージ。

似た機能の `clip-path` は「要素をその形で切り抜く（はみ出しを隠す）」だけだが、`border-shape` は「枠の形を定義し、装飾し、内側をクリップする」ところまでやる。つまり見た目の縁取りと切り抜きを一体で扱える。

## 🧠 専門的解説（技術詳細・設計・使いどころ）

### 技術的詳細（何が変わったか）

- **提供**: Chrome 147 で安定提供（Chrome 147 はネスト可能なアニメーションや自動テキスト配色なども同梱）。
- **受け付ける形状**: 基本図形 `circle()` / `ellipse()` / `inset()` / `polygon()`、複雑なパスを CSS 内で描く `shape()` 関数（Interop 2026 の一部）、SVG パスデータ風の `path("M 10 10 L 90 10 …")` 文字列。
- **2 つの変種**: 1 つの形をストロークする変種と、2 つの形状の間を塗る（枠の太さを 2 形状で表す）変種。
- **`clip-path` との本質的な違い**: `clip-path` は要素を単にマスクするだけだが、`border-shape` は「ボックスそのもの」を再定義する。同じ図形値を受け付けるが役割が違う。

### 仕組み / 背景（なぜこう設計されたか）

CSS のボックスモデルは矩形前提で、角の丸めは `border-radius` という「例外的な装飾」で扱ってきた。非矩形デザインの需要（クーポン券の切り欠き、吹き出し、装飾枠）に対しては、これまで背景 SVG・`clip-path`・擬似要素の合わせ技で「見た目だけ」非矩形に見せる回避策が主流だった。しかしそれらは「枠線（ボーダー）」「影」「アウトライン」を形に沿わせるのが難しく、破綻しがちだった。

`border-shape` は「ボーダーの形を第一級のプロパティにする」設計。要素の輪郭そのものを図形として定義するため、ボーダー・装飾・内側クリップが一貫して同じ形に従う。`clip-path` が「描画結果を後から切り抜く」のに対し、`border-shape` は「箱の定義段階で形を決める」ため、線の太さや装飾を形に正しく載せられる。トレードオフは、任意パスやアニメーションを多用するとレイアウト/描画コストが上がること、そして現状 Chromium 先行で、非対応ブラウザ向けには矩形フォールバックの用意が要る点。

### 実務での使いどころ・移行の勘所

チケット/クーポンの切り欠き、吹き出し、装飾的なカード枠、ブランド固有のシェイプなど、これまで画像や SVG に頼っていた非矩形 UI を CSS 化できる。移行の勘所は、(1) `clip-path` で「隠していただけ」の非矩形を、枠線・影が要るものは `border-shape` へ、(2) `shape()` は Interop 2026 対象で相互運用が進む一方、`path()` 文字列は保守性が落ちるので複雑形状は設計を吟味、(3) 非対応環境では矩形＋`border-radius` にフォールバックさせる。装飾中心の用途から段階導入するのが安全。

## 📖 用語解説（このトピックとのつながり）

- **border-shape**: 箱の輪郭そのものを図形で定義し、枠線を描く新プロパティ。
- **clip-path**: 要素を指定形状でマスク（切り抜き）する既存機能。border-shape とは役割が別。
- **shape() 関数**: CSS 内で複雑なパスを記述する関数。Interop 2026 対象で相互運用が進む。
- **polygon() / circle() / ellipse()**: 基本図形の指定。border-shape / clip-path 共通の値。
- **Interop**: 主要ブラウザが相互運用性を高める年次の取り組み。shape() が含まれる。

## 影響範囲 / 推奨アクション

- 非矩形 UI（切り欠き・吹き出し・装飾枠）を画像/SVG 無しで CSS 表現でき、保守性が上がる。
- 枠線・影が形に沿う必要があるものは clip-path から border-shape へ置き換え検討。
- Chromium 先行のため、非対応環境向けに矩形フォールバックを用意。
