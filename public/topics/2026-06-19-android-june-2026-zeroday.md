# Android 6月パッチ CVE-2025-48595 — Framework整数オーバーフローのゼロデイ権限昇格、標的型で悪用

> トピック: 脆弱性ニュース｜出典: BleepingComputer / SC Media / Android Security Bulletin｜種別: 整数オーバーフロー→ローカル権限昇格（ゼロデイ）

## 一行サマリー

Googleの2026年6月Androidセキュリティ更新は計124件の不具合を修正し、うち1件 CVE-2025-48595 は Android Framework の整数オーバーフローによるゼロデイ権限昇格で、ユーザー操作なしにローカルで権限を奪える。限定的・標的型の悪用（商用スパイウェアや国家関与の可能性）が確認されており、Android 14以降の端末は最新パッチへの即時更新が必要。

## 🔰 初学者向け（何が・なぜ問題か）

スマホのAndroidには、アプリと本体の機能を仲介する「Framework（土台の仕組み）」という共通部品があります。今回見つかった欠陥（CVE-2025-48595）は、この土台が「数の計算を誤る」ことで、本来は権限の低いアプリやコードが、管理者なみの強い権限を勝手に手に入れてしまう、というものです。

「整数オーバーフロー」とは、決められた桁数を超える大きな数を入れると、数値が一周して小さな値になってしまう現象です。例えるなら、9999までしか表示できないメーターが10000で0に戻るようなもの。プログラムが「サイズは0だ」と勘違いしてしまい、メモリの扱いを誤り、攻撃者がその隙に入り込みます。

怖いのは、利用者が何もしなくても（怪しいボタンを押さなくても）成立しうる点と、すでに「特定の人物を狙う攻撃」で実際に使われている点です。こうした手口は、ジャーナリストや要人を狙う高度なスパイウェアでよく見られます。Android 14以降を使っている人は、すぐに更新してください。

## 🧠 専門的解説（技術詳細・メカニズム・検知対応）

### 技術的詳細（種別・前提条件）

- **CVE**: CVE-2025-48595 / **深刻度**: High（Frameworkの権限昇格）/ 正確なCVSS値は要確認
- **CWE**: CWE-190（Integer Overflow or Wraparound）→ メモリ破壊→ローカル権限昇格
- **影響コンポーネント**: Android Framework / **影響範囲**: Android 14以降（ベンダにより範囲が異なる場合あり、要確認）
- **悪用状況**: Googleが「限定的・標的型の悪用の可能性」を表明。実環境でのゼロデイ悪用。6月更新は計124件を修正、うち18件がCritical。
- **前提条件**: ローカル（端末上）でコードを動かせること（不正アプリ等）。ユーザー操作不要との報告。
- **悪用の流れ（高位）**: 細工した入力でFramework内の整数演算を破綻 → メモリ確保/境界計算の誤り → 権限昇格。

### メカニズム（攻撃が成立する仕組み）

※以下は脆弱性クラスの一般原理に基づく概念説明。武器化手順・PoC・正確なオフセットは記載しない。

1. **境界計算に使う値の汚染**: サイズ長やインデックスの計算に、攻撃者が影響を与えられる値が用いられる。信頼境界（アプリ→Framework）を越えて入った値の検証が不十分。
2. **整数オーバーフロー/ラップアラウンド**: 計算結果が型の上限を超えて巻き戻り、本来より小さい（または不正な）サイズ値になる。プログラムは「安全なサイズ」と誤認する。
3. **メモリ操作の不整合**: 誤ったサイズに基づいてバッファ確保やコピーを行うと、ヒープオーバーフロー等のメモリ破壊が生じる。これが攻撃者の「メモリ書き換えプリミティブ」になる。
4. **昇格への橋渡し**: 高い権限で動くFrameworkプロセス内でメモリ破壊を制御できれば、制御フローやオブジェクトを操作して、低権限コンテキストから特権コンテキストへ昇格する。ユーザー操作不要の経路であれば、悪意アプリのインストール後に静かに成立しうる。
   - 一般原理: 「サイズ計算の汚染 → 整数オーバーフロー → メモリ破壊 → 権限昇格」はモバイル/カーネル領域の定番チェーン。境界計算は符号・桁あふれに対し安全な算術で行うことが根治策。

### 検知・暫定緩和・恒久対策

- **検知の着眼点**: 端末管理（MDM/EMM）でのパッチレベル監査、不審アプリの権限要求・サイドロード履歴、要人端末での挙動異常。一般利用者がOSレベルの悪用を直接検知するのは困難なため、更新適用が最善の防御。
- **暫定緩和**: 信頼できないアプリ（特にサイドロード）を入れない、Google Play Protectを有効化、要人・高リスク利用者はロックダウン系の保護機能の活用を検討。
- **恒久対策**: 2026年6月セキュリティパッチレベル以降へ更新。ベンダ配信の遅い端末は配信状況を確認し、可能な限り早期に適用。

## 📖 用語解説（このニュースとのつながり）

- **Android Framework**: アプリとOS中核を仲介する共通の仕組み。ここが破られると多くのアプリ・機能に影響する。
- **整数オーバーフロー（CWE-190）**: 数が型の上限を超えて巻き戻る不具合。サイズ計算に絡むとメモリ破壊の起点になる。
- **権限昇格（ローカル）**: 端末上の低権限コードが、より強い権限を得ること。本件の最終被害。
- **ゼロデイ**: 修正が出る前に悪用されている脆弱性。本件は標的型で実利用が確認された。
- **パッチレベル**: Androidの「いつ時点の修正が入っているか」を示す指標。更新済み判定の目安になる。

## 影響範囲 / 推奨対応

影響は、2026年6月パッチ未適用の Android 14以降端末（ベンダ差あり、要確認）。一般利用者にも関係するが、特に標的型攻撃のリスクが高いジャーナリスト・要人・行政関係者は最優先で更新を。組織はMDMでパッチレベルを点検し、サイドロード制限とPlay Protect有効化を徹底する。OS更新の配信が遅い端末は、メーカーの配信状況を確認のうえ早期適用すること。
