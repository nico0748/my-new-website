# Go 1.27（8月予定）— ジェネリックメソッドと、標準 encoding/json の v2 化

> トピック: Go 1.27（2026 年 8 月予定・WIP）｜出典: go.dev リリースノート・byteiota｜種別: 言語（プレビュー）

## 一行サマリー

2026 年 8 月に予定される Go 1.27 は、長らく要望されてきたジェネリックメソッド（メソッドが独自の型パラメータを宣言できる）を導入し、標準の `encoding/json` を長期計画の v2 実装に載せ替える。v2 は不正な UTF-8 や重複キーを拒否する厳格な既定を採用し、性能も大幅に改善する。※未リリースのためリリースノートは作業中で、詳細は確定前。

## 🔰 初学者向け（何が・なぜ重要か）

Go は「シンプルさ」と「速さ」で人気の言語。1.27 では、これまでできなかった2つの大きなことが入る予定。

ひとつめは「ジェネリックメソッド」。Go は 2022 年に型を抽象化するジェネリクスを入れたが、**メソッド**には独自の型パラメータを持たせられなかった。たとえるなら「関数には可変の型を渡せるのに、オブジェクトの操作（メソッド）には渡せない」という不便があった。1.27 でメソッドも型パラメータを宣言できるようになり、ある型に紐づく汎用処理を、パッケージ全体ではなくその型の名前空間の中に書けるようになる。

ふたつめは、JSON を扱う標準ライブラリの刷新。Go の JSON は便利だが、細かな「クセ」（想定外の入力を黙って受け入れる等）があった。新しい v2 実装はより厳格で速く、標準の `encoding/json` がその上に載る。日常的に JSON を扱う Go 開発者にとって影響が大きい。

## 🧠 専門的解説（技術詳細・設計・使いどころ）

### 技術的詳細（何が変わったか・予定）

- **リリース時期**: 2026 年 8 月予定。未リリースのためリリースノートは作業中（＝内容は確定前・要確認）。
- **ジェネリックメソッド**: メソッド宣言が独自の型パラメータを持てるようになる。従来はパッケージスコープの関数として書くしかなかった汎用処理を、特定データ型の名前空間内に置ける。ただしインターフェイスのメソッドは型パラメータを宣言できず、インターフェイスメソッドをジェネリックメソッドで実装することもできない。
- **encoding/json の v2 化**: `encoding/json/v2` は `encoding/json` の大改訂。`Marshal` / `MarshalWrite` / `MarshalEncode` / `Unmarshal` / `UnmarshalRead` / `UnmarshalDecode` を提供し、いずれも可変長の `Options` 引数で挙動を設定できる。標準 `encoding/json` が v2 実装に裏打ちされる。
- **v2 の厳格な既定**: JSON 文字列中の不正な UTF-8 を拒否し、オブジェクト内の重複キーを拒否する。v1 より相互運用的で安全側の既定。

### 仕組み / 背景（なぜこう設計されたか）

ジェネリックメソッドが当初のジェネリクスで見送られたのは、型推論や実装の複雑さ、インターフェイスとの整合（メソッド集合の一意性）という難問があったため。1.27 では「インターフェイスメソッドは対象外」と範囲を限定することで、型システムの健全性（インターフェイス充足の判定可能性）を保ちつつ、具体型のメソッドにだけ型パラメータを許す折衷設計を取った。これにより、コレクション型に `Map[U]`・`Filter` のような汎用操作を、その型のメソッドとして自然に生やせる。

encoding/json の v2 化は「後方互換を保ちながら基盤を差し替える」典型例。v1 は歴史的経緯からリフレクション多用で遅く、`null`・重複キー・不正 UTF-8 の扱いが緩かった。v2 はストリーミング指向の新実装で、多くの構造体でゼロヒープ割り当て・最大 10 倍高速なアンマーシャルを狙う。標準 `encoding/json` を v2 の上に載せることで、既存コードは基本そのまま速くなる一方、厳格化された既定（重複キー拒否など）が「これまで通っていた緩い入力」を弾く可能性がある——ここが移行時の主な非互換リスクだ。

### 実務での使いどころ・移行の勘所

汎用コレクション/データ構造ライブラリはジェネリックメソッドで表現力が上がる。JSON を多用するサービスは v2 化で高速化が期待できる。移行の勘所は、(1) まず `GOEXPERIMENT=jsonv2` 等で v2 挙動を試し、重複キー・不正 UTF-8 を含む入力が既存で通っていないか洗い出す、(2) 厳格化で弾かれる入力があれば `Options` で緩和するか、上流データを修正、(3) ジェネリックメソッドは「インターフェイス実装には使えない」制約を前提に設計、(4) 8 月リリースまでは WIP のため、本番前提にせず nightly/RC で検証。

## 📖 用語解説（このトピックとのつながり）

- **ジェネリクス / 型パラメータ**: 型を抽象化して汎用コードを書く仕組み。1.27 でメソッドにも拡張。
- **ジェネリックメソッド**: 独自の型パラメータを宣言できるメソッド。具体型のみ対象。
- **encoding/json v2**: 標準 JSON の大改訂実装。厳格な既定と高速化。標準 API がこの上に載る。
- **Marshal / Unmarshal**: Go 値と JSON の相互変換。v2 は Options で挙動を設定可能。
- **ストリーミング / ゼロアロケーション**: 逐次処理でメモリ割り当てを抑える方式。v2 の性能の源。

## 影響範囲 / 推奨アクション

- 汎用コレクション/ライブラリはジェネリックメソッドで API が改善。JSON 多用サービスは高速化。
- v2 の厳格既定（重複キー・不正 UTF-8 拒否）で既存入力が弾かれないか事前検証。
- 8 月リリースまで WIP。RC/nightly での検証にとどめ、本番投入は正式版後に。
