# 名前のみコンテナクエリが Baseline 入り — 「その名前の親がいるか」だけで分岐できる

> トピック: 名前のみコンテナクエリ（Baseline Newly available・2026年5月 / Chrome 148 ほか）｜出典: web.dev Baseline digest｜種別: CSS 機能の相互運用達成

## 一行サマリー

条件式を書かずに「指定した名前のコンテナが祖先に存在するか」だけでスタイルを切り替えられる「名前のみコンテナクエリ」が、2026年5月に Baseline Newly available に到達した。コンポーネントが「どの文脈（サイドバー内・カード内など）に置かれているか」でレイアウトを変える書き方が、より素直に表現できる。

## 🔰 初学者向け（何が・なぜ重要か）

レスポンシブ対応は長年「画面全体の幅」を見て切り替えてきた（メディアクエリ）。だが本当に知りたいのは「このカードが今どのくらいの幅の場所に置かれているか」であることが多い。それに応える仕組みが「コンテナクエリ」で、親要素の大きさに応じて子のデザインを変えられる。

今回の「名前のみコンテナクエリ」は、そこからさらに一歩進んだ使い方。幅の条件（例：400px 以上）を書かず、「`sidebar` という名前のコンテナの中にいるかどうか」だけで分岐できる。たとえば同じ「記事カード」を、サイドバーに置いたときは縦積み、本文エリアに置いたときは横並び、と「置かれた場所の役割」で切り替えたい場面がある。名前だけで判定できると、この「文脈による出し分け」がとても短く、読みやすく書ける。

## 🧠 専門的解説（技術詳細・設計・使いどころ）

### 技術的詳細（何が変わったか）

コンテナクエリは祖先に `container-type`（および任意の `container-name`）を宣言し、`@container <name> (min-width: ...)` のように問い合わせる。今回 Baseline 化したのは「サイズ条件を伴わず、名前だけで問い合わせる」記法で、web.dev の 2026年5月 Baseline digest では Chrome 148 で Baseline Newly available になったと告知された。指定名のコンテナが祖先チェーンに存在すればマッチする、というシンプルな判定を提供する（正確な各エンジンのバージョンは MDN で要確認）。

### 仕組み / 背景（なぜこう設計されたか）

コンポーネント指向の CSS では「部品は置かれる場所を知らずに書きたい」という原則がある。従来のサイズ条件付きコンテナクエリは「幅がいくつか」で分岐するため、"どこに置かれたか" という意味情報を幅の閾値で代用する必要があり、閾値のマジックナンバー化を招いた。名前のみクエリは「文脈（named container）を明示的に宣言し、部品はその名前の有無で反応する」という、意味ベースの分岐を可能にする。トレードオフは、コンテナ名という新たな命名規約をチームで統一・管理する必要が生じる点と、名前の有無だけでは表現できない連続的なサイズ適応にはやはりサイズ条件が要る点。両者は排他ではなく、意味（名前）とサイズ（幅）を組み合わせて使うのが自然な設計になる。

### 実務での使いどころ・移行の勘所

デザインシステムで「同一コンポーネントを複数コンテキストに配置する」ケースに最適。カード・リストアイテム・ウィジェットを「`sidebar` 内／`main` 内／`grid` 内」で出し分ける規約を、幅の魔法数を排して宣言できる。移行の勘所は、まずレイアウト領域に一貫した `container-name` を付与するルールを設計すること、未対応ブラウザ向けにサイズ条件付きクエリや通常フローのフォールバックを用意すること。過度な入れ子コンテナはデバッグを難しくするため、コンテナは「意味のある領域」に限定して命名するのが実務的。

## 📖 用語解説（このトピックとのつながり）

- **コンテナクエリ**: 画面全体ではなく親要素の状態に応じてスタイルを切り替える仕組み。本機能はその「名前だけで判定する」派生。
- **container-name / container-type**: 祖先要素をクエリ対象として宣言するプロパティ。名前のみクエリはこの `container-name` を参照する。
- **Baseline Newly available**: 主要ブラウザの最新版で使えるようになった段階。広く普及した「Widely available」の前段。
- **メディアクエリとの違い**: メディアクエリは画面/デバイス基準、コンテナクエリは「親要素」基準で分岐する。

## 影響範囲 / 推奨アクション

- コンポーネントベース CSS の「文脈依存の出し分け」が簡潔になる。デザインシステムのバリアント設計を見直す好機。
- レイアウト領域への `container-name` 命名規約を先に整備してから導入する。
- 旧ブラウザ向けフォールバックを併用し、プログレッシブに適用する。
