# BeyondTrust Remote Support/PRA の未認証RCE、VShell・SparkRAT で実悪用進行（CVE-2026-1731）

> トピック: 脆弱性ニュース ｜ 出典: BeyondTrust BT26-02 / Rapid7 / GreyNoise / CISA KEV ｜ 種別: 未認証リモートコード実行（WebSocket 処理の不備）

## 一行サマリー

特権リモートアクセス製品 BeyondTrust Remote Support(RS) と Privileged Remote Access(PRA) に、認証情報なしで乗っ取れる致命的RCE（CVSSv4 9.9）が存在。細工した WebSocket メッセージだけで成立し、CISA KEV 登録済み。攻撃では VShell や SparkRAT が投下されており、自己ホスト型の未更新インスタンス（約8,500台が露出報告）は最優先で更新すべき。

## 🔰 初学者向け（何が・なぜ問題か）

BeyondTrust の Remote Support / PRA は、情報システム部門やヘルプデスクが「離れた場所のPCやサーバに安全に入って作業する」ための遠隔サポート製品です。いわば“社内の合鍵を束ねて配る管理室”で、多くの端末へ入る権限が集まる、極めて価値の高い設備です。

今回の穴は、その管理室の「常時つながる連絡窓口」にありました。Web の裏で使われる WebSocket という“開きっぱなしの通話回線”を処理する部品（thin-scc-wrapper）が、送られてきたメッセージを十分に確認せずに受け取ってしまう不具合です。例えるなら、受付が「合言葉の確認もせず、電話越しの指示をそのまま実行してしまう」状態。攻撃者は身分証（認証情報）もいらず、ただインターネット越しに細工したメッセージを1本送るだけで、管理室のサーバに自分の命令を実行させられます。

管理室が乗っ取られると、そこにつながる多数の端末へ攻撃が波及します。実際の攻撃では、乗っ取ったサーバに VShell や SparkRAT という“遠隔操作の道具”が仕込まれており、そこを足場に社内深部へ侵入されます。SaaS版は自動で修正済みですが、自社で運用している（自己ホスト型の）製品は手動更新しないと危険なまま、という点が要注意です。

## 🧠 専門的解説（技術詳細・メカニズム・検知対応）

### 技術的詳細（種別・前提条件）

- **CVSS**: v4 で 9.9（near-maximum）Critical。
- **CWE**: 未認証の入力処理不備によるRCE（CWE-306 認証欠如／CWE-20 入力検証不備 系に相当、詳細は要確認）。
- **脆弱性の性質**: `thin-scc-wrapper` コンポーネントの WebSocket 接続処理における未認証リモートコード実行。認証情報・ユーザー操作とも不要、攻撃複雑度は低。
- **前提条件**: インターネット到達可能な該当エンドポイントへ WebSocket 接続できること。
- **影響バージョン**: Remote Support(RS) 25.3.1 以前、Privileged Remote Access(PRA) 24.3.4 以前。
- **修正/対応**: BeyondTrust が BT26-02 を公開（2026年2月6日）。SaaS は 2月2日に自動パッチ適用。自己ホスト型は手動更新が必要。約8,500台の自己ホスト・インスタンスが露出と Hacktron AI が報告（AI による変種解析で発見）。
- **悪用状況**: GreyNoise・watchTowr・Arctic Wolf 等が実悪用を確認。PoC 公開後24時間以内に攻撃観測。2026年2月13日に CISA KEV 登録。攻撃では VShell / SparkRAT が投下（継続的な悪用が報告されており、未更新の自己ホスト機は現在も危険）。

### メカニズム（攻撃が成立する仕組み）

本件は「常時接続の双方向チャネル（WebSocket）を、認証確立前に高い権限で処理してしまう」という設計上の弱点に起因する。

1. **信頼境界の位置**: WebSocket は HTTP からアップグレードされる“開きっぱなしの双方向回線”で、リアルタイム制御に使われる。問題の `thin-scc-wrapper` は、この回線から届くメッセージを、認証が確立する前段階で受理・処理していた。ここに「未認証の相手が特権処理に触れる」信頼境界の穴が生じる。

2. **どのコンポーネントの不備か**: WebSocket メッセージのパース/ディスパッチ処理で、送信者の権限確認や入力の妥当性検証が不足していた。攻撃者は正規クライアントを装う必要すらなく、細工メッセージを送るだけで内部処理を誘導できる。

3. **攻撃者が握るプリミティブ**: 未認証で特権処理を呼び出せることで、攻撃者はサーバ側での操作実行プリミティブを獲得する。これがコマンド/コード実行経路に橋渡しされ、製品を動かすプロセス権限での任意コード実行（RCE）に至る。

4. **最終被害への連鎖**: RS/PRA は多数の被管理端末への“特権アクセスのハブ”であるため、サーバ掌握は「そこから到達できる全端末への横展開」を意味する。実攻撃では VShell/SparkRAT を常駐させ、認証情報窃取・追加侵入・持続化に用いる。1製品の陥落が広範な内部侵害の起点となる。

5. **クラスの一般原理**: 「認証確立前に到達可能な常時接続チャネル（WebSocket/管理API）× 入力・権限検証の欠落」は、エッジ/リモートアクセス製品で反復する高危険パターン。境界に置かれ、かつ特権が集中する装置ほど、単一の未認証RCEが致命傷になる。

※武器化可能な具体的メッセージ構造・PoC・エンドポイントパスは記載しない（概念レベル）。

### 検知・暫定緩和・恒久対策

- **検知の着眼点（IoC/挙動）**: RS/PRA アプライアンスへの外部からの想定外 WebSocket 接続・異常なアップグレード要求を監視。サーバ上での想定外プロセス生成、VShell/SparkRAT 由来のビーコン通信・永続化痕跡（不審サービス/cron/スケジュールタスク）を EDR で検出。管理者アカウントの不審な作成・利用を相関。
- **暫定緩和**: 該当エンドポイントのインターネット直接公開を停止し、到達元を VPN/許可IPに限定。SaaS 移行や WAF/リバースプロキシでの WebSocket 制御も検討。露出インスタンスの棚卸しを最優先で実施。
- **恒久対策**: RS を 25.3.1 超、PRA を 24.3.4 超の修正版へ即時更新（自己ホスト型は手動適用が必須）。適用前に侵害痕跡調査を行い、痕跡があれば資格情報の全ローテーションと封じ込め。
- **運用示唆**: 特権リモートアクセス製品は“合鍵の集約点”。KEV 登録済み・実悪用中の前提で、露出面の即時遮断とパッチを最優先とする。

## 📖 用語解説（このニュースとのつながり）

- **Remote Support / PRA**: 遠隔サポート・特権リモートアクセス製品。多数端末への特権が集中するため陥落時の影響が甚大。
- **WebSocket**: HTTP からアップグレードされる双方向・常時接続の通信。今回、認証前処理の穴が突かれた。
- **未認証RCE（pre-auth RCE）**: 認証なしで任意コードを実行できる最悪クラスの脆弱性。CVSS 9.9。
- **VShell / SparkRAT**: 攻撃で投下された遠隔操作ツール。侵害後の持続化・横展開に用いられる。
- **CISA KEV**: 実悪用が確認された脆弱性の公式カタログ。登録＝優先対応の強いシグナル。

## 影響範囲 / 推奨対応

BeyondTrust Remote Support(≤25.3.1) / PRA(≤24.3.4) の自己ホスト型が主リスク（SaaS は自動修正済み）。約8,500台の露出報告があり、PoC 公開後24時間で攻撃が始まった実績から猶予はない。修正版へ即時更新し、更新までは該当エンドポイントの外部公開を停止すること。VShell/SparkRAT の痕跡調査と、侵害時の資格情報全ローテーションを併せて実施する。
