# CSS Gap Decorations が Chromium で安定化 — grid / flex の「すき間」に線を引く標準機能

> トピック: CSS Gap Decorations（Chrome 149 で安定版）｜出典: Chrome for Developers｜種別: Web 標準 / CSS

## 一行サマリー

grid や flexbox の要素間の「すき間（gap）」に、これまで擬似要素や背景画像などの回避策なしでは引けなかった仕切り線を、CSS だけで描けるようにする Gap Decorations が Chrome / Edge 149（2026 年 6 月 2 日安定版）で利用可能になった。多段組みの `column-rule` を grid・flex に一般化した機能で、線幅・色・インセットのアニメーションにも対応する。

## 🔰 初学者向け（何が・なぜ重要か）

Web ページで要素をタイル状に並べるとき、`gap` プロパティで要素と要素の間隔を空けられる。ところが「その間隔に一本の仕切り線を引きたい」という、表やダッシュボードでありがちな要望を叶える標準的な方法がこれまで無かった。

そのため開発者は、各要素に `border` を付けて重なりを調整したり、ダミーの要素（擬似要素）を差し込んだり、背景画像で線に見せかけたり…という「ハック」を使ってきた。どれも面倒で、要素の数が変わると崩れやすい。

Gap Decorations は、この「すき間に線を引く」を CSS の正式なプロパティにしたもの。たとえるなら、これまで手作業で定規を当てて線を書き込んでいたのが、「ここに罫線を入れて」と一言指定するだけで済むようになった。しかも対応していないブラウザでは線が出ないだけで、レイアウト自体は普通に表示される（＝安全に足せる進化）。

## 🧠 専門的解説（技術詳細・設計・使いどころ）

### 技術的詳細（何が変わったか）

- **提供**: Chrome / Edge 149 から安定版（Chrome 149 は 2026 年 6 月 2 日にステーブル）。grid・flexbox・multicol のすき間を装飾できる。
- **新プロパティ群**: 多段組みの `column-rule`（幅・スタイル・色）を出発点に、`column-rule-inset` / `row-rule-inset`（線の端を内側に詰める）、`column-rule-visibility-items` / `row-rule-visibility-items`（どのすき間に線を出すかの制御）などを追加。
- **アニメーション**: 線幅・色・インセットのアニメーションに対応。
- **段階的強化（progressive enhancement）**: 装飾目的なら未対応ブラウザで単に線が出ないだけで、gap は通常どおり描画される。

### 仕組み / 背景（なぜこう設計されたか）

CSS はもともと multicol（段組み）に `column-rule` という「段と段の間に引く線」を持っていた。しかし grid / flexbox の gap には対応する仕組みが無く、レイアウトエンジンは「すき間」を空白としてしか扱えなかった。Gap Decorations は、この `column-rule` の概念を **2 次元のレイアウト全般に一般化**し、行方向・列方向それぞれのすき間に対して「線を描くレイヤー」をレイアウト計算の一部として定義する設計。

要素ごとの `border` で線を模倣する従来手法の課題は、(1) 端の要素で線が余る/足りない、(2) 要素数が動的に変わると破綻、(3) 折り返し時に意図しない位置へ線が出る、といった点だった。Gap Decorations は「線はすき間に属する」とモデル化することで、要素数や折り返しに追従して自然に線が引かれる。`*-inset` や `*-visibility-items` は「端のすき間だけ線を消す」「特定のすき間だけ引く」といった細かな制御を、要素側ではなくすき間側の宣言で完結させる。トレードオフは、現状 Chromium 系のみの対応で、装飾以外の用途（線に意味を持たせる場合）では代替表現のフォールバック設計が要る点。

### 実務での使いどころ・移行の勘所

価格表・比較表・カード一覧・ダッシュボードのパネル区切りなど、「タイル状レイアウトに罫線を入れたい」場面に有効。移行の勘所は、(1) 既存の `border` ハックや擬似要素による罫線を Gap Decorations に置き換えて DOM を単純化、(2) 端のすき間の線を `*-inset` / `*-visibility-items` で整える、(3) 装飾目的なら未対応ブラウザ向けフォールバック不要だが、線が情報を担う場合は別途担保。Chromium 系限定のため、当面は「あると嬉しい強化」として段階導入するのが安全。

## 📖 用語解説（このトピックとのつながり）

- **gap**: grid / flex で要素間の間隔を空けるプロパティ。その「すき間」に線を引くのが本機能。
- **column-rule**: 多段組みで段間に線を引く既存プロパティ。Gap Decorations の原型。
- **progressive enhancement（段階的強化）**: 対応環境でだけ上乗せされる機能設計。未対応でも壊れない。
- **inset**: 線の端を内側へ詰める指定。端のすき間の見栄えを整える。
- **Chromium**: Chrome / Edge 等の基盤。現状この機能は Chromium 系で先行。

## 影響範囲 / 推奨アクション

- カード/表/ダッシュボードの罫線を CSS だけで表現でき、DOM とスタイルを簡素化できる。
- 既存の border ハック・擬似要素罫線の置き換え候補。端のすき間は inset / visibility で調整。
- Chromium 系限定のため、線に意味がある UI ではフォールバックを検討。
