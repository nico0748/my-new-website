# Ubiquiti UniFi OS に CVSS 10.0 の最大深刻度3連脆弱性（CVE-2026-34908/34909/34910）— アクセス制御欠落からroot RCEへ連鎖・KEV悪用中

> トピック: 脆弱性ニュース ｜ 出典: CISA KEV / BleepingComputer / SecurityWeek / Ubiquiti Security Advisory Bulletin 064 ｜ 種別: 不適切なアクセス制御（CWE-284）＋パストラバーサル（CWE-22）＋不適切な入力検証（CWE-20）の連鎖による未認証root RCE

## 一行サマリー

中小企業・家庭で広く使われるUbiquitiのネットワーク統合OS「UniFi OS」に、CVSS 10.0（最大値）の脆弱性が3件見つかり、CISAが6月23日にKEV（実環境で悪用中の脆弱性カタログ）へ登録した。アクセス制御の欠落（CVE-2026-34908）・パストラバーサル（CVE-2026-34909）・コマンドインジェクション（CVE-2026-34910）を連鎖させると未認証でroot権限のコード実行に至る。連邦機関は6月26日までの是正を命じられており、UniFi OS Server 5.0.8への即時更新が必要。

## 🔰 初学者向け（何が・なぜ問題か）

UniFi（ユニファイ）は、Wi-Fiアクセスポイントや防犯カメラ、ネットワーク機器をまとめて管理するための「司令塔ソフト」です。会社のオフィスや店舗、自宅のネットワーク全体を、このUniFi OSという一つの管理画面（コントローラー）から面倒みる仕組みになっています。つまり、ここを乗っ取られると**ネットワークまるごと**が攻撃者の手に落ちます。

今回見つかったのは、その司令塔に潜む3つの欠陥です。たとえるなら、(1) **受付の鍵が壊れていて誰でも建物に入れてしまう**（アクセス制御の欠落）、(2) **館内の通路設計が甘く、関係者専用エリアの書類棚まで歩いて行けてしまう**（パストラバーサル＝本来見えてはいけないファイルまでたどれる）、(3) **館内放送のマイクに細工した言葉を吹き込むと、警備員がその通りに動いてしまう**（コマンドインジェクション＝攻撃者の文字列を機器が命令として実行する）、という3点です。

恐ろしいのは、この3つを**つなげて使う**と、外部の何の権限もない攻撃者が、最終的に機器の最高権限（root＝管理者の中の管理者）で好きなプログラムを動かせてしまうことです。しかもCISAは「すでに実際の攻撃で使われている」と警告しています。3件すべてが深刻度の最大値である **CVSS 10.0** に評価されており、これは「これ以上ない危険度」を意味します。

## 🧠 専門的解説（技術詳細・メカニズム・検知対応）

### 技術的詳細（種別・前提条件）

- **CVSS**: 3件いずれも **10.0（Critical／最大値）**。ネットワーク経由・未認証・低複雑度で悪用可能。
- **CWE分類**: CVE-2026-34908＝CWE-284（不適切なアクセス制御）、CVE-2026-34909＝CWE-22（パストラバーサル／ディレクトリトラバーサル）、CVE-2026-34910＝CWE-20相当（不適切な入力検証によるOSコマンドインジェクション）。
- **前提条件**: 攻撃者が管理インターフェースにネットワーク到達できること。インターネットに露出したUniFi OS管理ポートが主な攻撃面。
- **影響製品・バージョン**: UniFi OS Server。修正版は **UniFi OS Server 5.0.8**（正確な対象範囲はUbiquiti Security Advisory Bulletin 064で要確認）。
- **悪用状況**: **CISAが2026年6月23日にKEV登録**。少なくとも1件が実環境で悪用中。連邦民間機関（FCEB）にはBOD 26-04に基づき**2026年6月26日までの是正（パッチ適用または使用停止）**が命じられている。

### メカニズム（攻撃が成立する仕組み）

以下は概念レベルの説明であり、武器化可能な手順・PoC・正確なペイロード等は含めない。

1. **信頼境界の所在**: UniFi OSの管理APIは、外部からのHTTPリクエストを受ける正面玄関である。ここで「誰が・何を要求できるか」を正しく仕分けられるかが信頼境界となる。
2. **第1段：認可チェックの欠落（CVE-2026-34908）**: 本来は認証済み管理者のみが叩けるべきエンドポイントが、認可（権限の確認）を欠いており、未認証の要求でも設定変更などの特権操作を受け付けてしまう。攻撃者はここで「正規ユーザーのふり」をする必要すらなく足場を得る。
3. **第2段：パストラバーサルでの情報奪取（CVE-2026-34909）**: ファイルパスを含む入力に対する正規化・検証が不十分なため、`../`相当の相対参照で本来アクセスできない基盤OS上のファイル（設定・資格情報・トークン等）を読み出せる。これにより次段の足がかりとなる秘密情報を握る。
4. **第3段：コマンドインジェクションでのroot実行（CVE-2026-34910）**: 入力検証の不備により、攻撃者が与えた文字列が**シェルコマンドの一部として連結・実行**される。UniFi OSの該当処理が高権限で動くため、注入されたコマンドは **root権限**で実行される。
5. **連鎖による致命性**: 単体でも危険な3つを「認可回避 → 情報読み出し → コマンド実行」と橋渡しすることで、未認証の外部攻撃者が最終的に機器の完全制御（root RCE）を獲得する。攻撃者が握るプリミティブ（認証回避・任意ファイル読み取り・任意コマンド実行）が、そのまま最終被害（ネットワーク機器の乗っ取り）へ直結する典型的な脆弱性チェーンである。

脆弱性クラスとしては、「組み込み／アプライアンス管理プレーンにおける認可欠落・パス検証欠落・シェル連結」という、ネットワーク機器で最も頻出する3点セットに該当する。

### 検知・暫定緩和・恒久対策

- **検知の着眼点**: 管理インターフェースへの未認証アクセスの成功、`../`等を含む異常なパス要求、UniFi OSプロセスからの予期しない子プロセス（シェル・ネットワークツール）生成、設定の不審な変更履歴。アクセスログでの管理エンドポイントへの外部IPからの到達を重点監視。
- **暫定緩和（パッチ前）**: 管理インターフェースを**インターネットから遮断**し、VPN／管理用VLAN経由のみに限定。ファイアウォールで管理ポートへのアクセス元を厳格に制限。即時更新できない機器はネットワークから隔離。
- **恒久対策**: **UniFi OS Server 5.0.8への即時アップデート**。3件は連鎖前提のため、すべてを塞ぐ最新版適用が必須。
- **運用示唆**: ネットワーク機器の管理プレーンは「インターネットに晒さない」を恒常原則とする。KEV登録案件はBOD期限（6/26）を待たず最優先で対応。

## 📖 用語解説（このニュースとのつながり）

- **UniFi OS**: Ubiquiti製ネットワーク機器を統合管理するOS／コントローラー。意味＝「ネットワークの司令塔」。本件＝ここが乗っ取られるとネットワーク全体が危険に。
- **アクセス制御欠落（CWE-284）**: 権限確認をすべき処理でそれを怠る欠陥。本件＝未認証で特権操作を許す第1段。
- **パストラバーサル（CWE-22）**: `../`等で想定外のファイルへ到達する欠陥。本件＝基盤OSの秘密情報を読み出す第2段。
- **コマンドインジェクション**: 入力がシェル命令として実行される欠陥。本件＝root権限でのコード実行を生む第3段。
- **KEV / BOD 26-04**: CISAの「悪用中脆弱性カタログ」と連邦機関への是正命令。本件＝6/26までの対応が義務付け。

## 影響範囲 / 推奨対応

影響を受けるのは、UniFi OS Serverで管理されるすべての環境。特に**管理インターフェースをインターネットに露出している**中小企業・店舗・在宅ネットワークが高リスク。推奨対応は、(1) UniFi OS Server 5.0.8への即時更新、(2) 管理インターフェースのインターネット露出を即座に遮断（VPN/管理VLAN化）、(3) 露出していた機器は侵害前提で設定・認証情報・接続機器を点検、(4) 不審な設定変更・子プロセス生成の監視。
