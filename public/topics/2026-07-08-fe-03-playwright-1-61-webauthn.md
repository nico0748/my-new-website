# Playwright 1.61：パスキー（WebAuthn）を実機なしでテストできる仮想オーセンティケータ

> トピック: Playwright 1.61 リリース｜出典: Playwright 公式リリースノート / GitHub Releases｜種別: マイナーリリース（E2E テストツール）

## 一行サマリー

Playwright 1.61 は、パスキー認証（WebAuthn）を物理セキュリティキーなしでテストできる仮想オーセンティケータ（`browserContext.credentials`）、`page.localStorage` を直接扱う API、フレーキーテスト向けの動画保持モードを追加した。パスワードレス認証を採用するアプリの E2E テストが一気に現実的になる。

## 🔰 初学者向け（何が・なぜ重要か）

パスキーは「パスワードの代わりに、指紋認証や顔認証、端末内の鍵でログインする」仕組み（WebAuthn / FIDO2）。安全でパスワード不要なので急速に普及しているが、テストが難しかった。なぜなら、ログインを自動テストしようとすると、本物の指紋センサーやセキュリティキーが必要になり、CI（自動テスト環境）では再現できなかったからだ。

Playwright はブラウザを自動操作して Web アプリをテストするツール。その 1.61 で「仮想オーセンティケータ」が入った。これは「本物のセキュリティキーのふりをするソフトウェア」で、テストコードから「今からパスキーを登録する」「ログイン要求に応答する」を自在に指示できる。実機のハードウェアなしで、パスキーの登録もログインも自動テストできるようになる。

つまり、これまで「手動でしか確認できなかったパスワードレスログイン」が、自動テストの守備範囲に入る。パスキー対応を進めているサービスにとって、地味だが大きな前進だ。

## 🧠 専門的解説（技術詳細・設計・使いどころ）

### 技術的詳細（何が変わったか）

- **仮想オーセンティケータ（Credentials virtual authenticator）**: `browserContext.credentials` 経由で利用。テスト内でパスキーを登録し、`navigator.credentials.create()` / `navigator.credentials.get()` のセレモニーに実機なしで応答できる。
- **`page.localStorage` API**: ページの localStorage を直接読み書きする API。従来 `page.evaluate()` で回りくどく操作していたのを簡潔化。
- **動画保持モード（video retention modes）**: フレーキー（不安定）なテスト向けに、失敗時のみ動画を残すなど保持ポリシーを制御。
- 直前の 1.60（2026年5月）では `locator.drop()`（ファイルやクリップボード類似データの外部ドラッグ&ドロップをシミュレート）、HAR / トレース記録への WebSocket リクエスト記録などが追加された。

### 仕組み / 背景（なぜこう設計されたか）

WebAuthn テストの根本課題は「ハードウェア依存」だ。パスキーの本質は「秘密鍵が端末（Authenticator）に閉じていて外に出ない」ことにあり、これはセキュリティ上の強みだが、そのままだと自動テストで鍵操作を再現できない。Chrome DevTools Protocol（CDP）には以前から仮想オーセンティケータの下位機構があり、Playwright 1.61 はそれを高レベル API として使いやすく公開した形だ。

仮想オーセンティケータは、実際の暗号鍵ペアをソフトウェア上に生成し、WebAuthn のチャレンジ・レスポンス（アテステーション／アサーション）をプログラムから制御できるようにする。テストは「ユーザーが指紋を当てた」という物理操作を、API 呼び出しに置き換える。トレードオフとして、仮想オーセンティケータはあくまで「プロトコル上の振る舞い」を再現するもので、実機の生体センサーやプラットフォーム固有の UI までは検証できない。E2E テストとしての回帰防止には十分だが、実機受け入れテストの完全代替ではない。

`page.localStorage` の追加も同じ思想で、「よく使うのに回りくどかった操作」を一級 API に昇格させ、テストコードの意図を読みやすくしている。

### 実務での使いどころ・移行の勘所

- パスキー / パスワードレス認証を導入済み・導入予定のサービスは、ログインフローの回帰テストを CI に組み込める。
- 認証済み状態の前提を作るセットアップで `page.localStorage` を使うとテストが簡潔になる。
- フレーキーテストの調査では動画保持モードで「失敗時だけ録画」に絞り、ストレージと実行時間を節約。
- 1.60/1.61 はマイナー更新だが仮想オーセンティケータは破壊的変更ではないため、既存スイートに追加テストとして足しやすい。

## 📖 用語解説（このトピックとのつながり）

- **WebAuthn / FIDO2**: パスワードレス認証の標準。公開鍵暗号で、秘密鍵は端末（Authenticator）に閉じる。Playwright 1.61 の仮想オーセンティケータが対象とする。
- **パスキー**: WebAuthn を使った端末同期型の認証情報。生体認証などでログインする。
- **仮想オーセンティケータ**: 実機の代わりに WebAuthn の応答を返すソフトウェア。CDP 由来で Playwright が高レベル API 化。
- **アテステーション / アサーション**: パスキー登録時（create）と認証時（get）にオーセンティケータが返す署名付きデータ。
- **フレーキーテスト**: 環境依存で成否が揺れる不安定なテスト。動画保持モードは原因調査を助ける。

## 影響範囲 / 推奨アクション

- パスキー対応アプリは 1.61 へ更新し、仮想オーセンティケータで登録・ログインの E2E テストを追加する。
- localStorage を使ったテストセットアップを `page.localStorage` に置き換えて簡潔化。
- フレーキーテストは動画保持モードで録画ポリシーを最適化。
- 実機固有の生体 UI 検証は別途手動／受け入れテストで担保する（仮想オーセンティケータは完全代替ではない）。
