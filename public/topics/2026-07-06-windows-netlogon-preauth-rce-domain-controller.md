# Windows Netlogon の未認証RCEでドメインコントローラー乗っ取り、実環境で悪用進行（CVE-2026-41089）

> トピック: 脆弱性ニュース ｜ 出典: Microsoft(2026年5月Patch Tuesday) / CCB(ベルギー) / Help Net Security / 各社解析 ｜ 種別: スタックバッファオーバーフロー（認証前RCE）

## 一行サマリー

Windows のドメイン認証の中核「Netlogon」に、認証情報なし・ユーザー操作なしでドメインコントローラー（DC）を SYSTEM 権限で乗っ取れる致命的脆弱性（CVSS 9.8）が見つかり、実環境での悪用が確認された。DC が落ちれば Active Directory ドメイン全体が制圧されるため、2026年5月パッチの即時適用が急務。

## 🔰 初学者向け（何が・なぜ問題か）

会社のWindowsネットワークにはたいていActive Directory（AD）という「社員名簿＋入館ゲートの管理室」があります。その管理室の本体が「ドメインコントローラー（DC）」で、「誰が正しい社員か」「どのドアを開けていいか」を一手に握る、まさに社内の“総合受付兼警備本部”です。

「Netlogon（ネットログオン）」は、この受付が社員証（アカウント）を確認するときに使う専用の連絡回線だと思ってください。今回の穴は、その連絡回線に届いた“書類の長さ”をきちんと測らずに受け取ってしまう不具合です。攻撃者が想定より長い書類（データ）を送りつけると、受付の作業机（メモリ）からデータがあふれ出し、机の裏に置いてあった「次に何をするか」のメモまで書き換えられてしまいます。これがバッファオーバーフローという現象です。

書き換えられた結果、攻撃者は受付＝DC に自分の命令を実行させられます。しかも社員証（ログイン情報）は一切不要で、誰かがクリックする必要もありません。ネットワークで DC に話しかけられる位置にいれば、警備本部そのものを最高権限（SYSTEM）で乗っ取れるということです。総合受付を奪われれば、全社員のドアがすべて開いてしまうのと同じで、被害は組織全体に及びます。

## 🧠 専門的解説（技術詳細・メカニズム・検知対応）

### 技術的詳細（種別・前提条件）

- **CVSS**: 9.8 Critical（`AV:N/AC:L/PR:N/UI:N/S:U/C:H/I:H/A:H` 相当）。
- **CWE**: CWE-121（スタックベースのバッファオーバーフロー）。
- **脆弱性の性質**: Netlogon RPC インターフェースにおける未認証・0クリックのリモートコード実行。SYSTEM 権限で実行される。
- **前提条件**: 標的が「ドメインコントローラーとして構成された Windows Server」であること、および攻撃者から Netlogon（RPC）に到達できるネットワーク位置。認証情報・ユーザー操作は不要。
- **影響範囲**: DC として稼働する現行サポート対象の Windows Server 各バージョン。
- **修正版**: 2026年5月12日（May 2026 Patch Tuesday）の更新で修正。Microsoft の WARP（Windows Attack Research and Protection）チームが内部発見。
- **悪用状況**: 2026年5月29日に実環境での悪用が確認され、ベルギーの国家サイバーセキュリティ機関 CCB が優先的なパッチ適用を警告。DC 完全掌握＝ドメイン乗っ取りに直結するため優先度は最上位（要確認：KEV登録状況は各自で最新確認を）。

### メカニズム（攻撃が成立する仕組み）

本件は「認証前に到達できるプロトコル処理で、外部入力の長さ検証が不十分」という、境界防御が最も薄い場所に生じた古典的かつ最悪クラスのメモリ破壊である。

1. **信頼境界の位置**: Netlogon は「まだ認証が済んでいない相手」とも会話を始めるプロトコルであり、DC 上で高い権限を持つプロセスが処理する。つまり「認証前 × 高権限 × ネットワーク到達可能」という三拍子が揃った、攻撃者にとって理想的な入口。

2. **不備のあるコンポーネント**: Netlogon のリクエスト処理で、受信フィールドの長さ・境界を検証しないままスタック上のバッファへコピーする箇所が存在した。攻撃者が制御する“オーバーサイズのフィールド”がバッファ容量を超えて書き込まれると、隣接するスタック領域（保存されたリターンアドレスや制御データ）が上書きされる。

3. **攻撃者が握るプリミティブ**: スタック上の制御データを上書きできることで、攻撃者は「プログラムの実行フローを奪う」プリミティブを得る。ここから、DC 上のプロセス権限（SYSTEM 相当）で任意コード実行へ橋渡しされる。

4. **最終被害への連鎖**: SYSTEM でのコード実行を得た攻撃者は、ドメイン管理者アカウントの新規作成、Kerberos 資格情報の窃取（ゴールデンチケット等）、GPO 改ざんによる全端末への横展開といった“ドメイン全制圧”へ進みうる。DC 1台の陥落が組織全体の陥落を意味する点が本件の恐ろしさ。

5. **クラスの一般原理**: 「認証前に届く RPC/ネットワークプロトコルでの長さ検証欠落 → スタック/ヒープ破壊 → 制御奪取」は、ネットワークサービスにおける最重量級の脆弱性クラス。近代 OS は ASLR/スタックカナリア等の緩和を備えるが、プロトコル固有の情報漏えいや実装差で回避されうるため、緩和頼みにせずパッチ適用が本筋。

※武器化可能な具体的パケット構造・オフセット・PoC は記載しない（概念レベル）。

### 検知・暫定緩和・恒久対策

- **検知の着眼点（IoC/挙動）**: DC 以外の内部ホストが Netlogon RPC セッションを開始していないか監視（DC-to-DC 以外の発信は要調査）。侵害後兆候として、Windows セキュリティイベント **4720（アカウント作成）＋4728（管理グループ追加）** の組み合わせ、DC 上での **4688** による SYSTEM 権限の想定外プロセス生成、正規変更管理を経ない GPO 改変を相関。`C:\Windows\System32` 以外に出現する `msimg32.dll` は要即時隔離とする報告あり（要確認）。奇形の CLDAP リクエストや内部IPからの反復接続、Netlogon 交換における過大フィールドのパケットも着眼点。
- **暫定緩和**: DC への Netlogon/RPC 到達を管理セグメントに限定し、一般クライアント網からの不要な到達を遮断。DC の露出面最小化と内部セグメンテーションを強化。
- **恒久対策**: 2026年5月の累積更新を全 DC へ即時適用。適用状況を全 DC で棚卸しし、未適用機の隔離を検討。
- **運用示唆**: DC は「単一障害点かつ最高価値標的」。パッチSLAを最短化し、適用後も上記イベントIDで遡及ハンティングを行う。

## 📖 用語解説（このニュースとのつながり）

- **Netlogon**: Windows ドメインで認証・セキュアチャネルを担うプロトコル/サービス。今回はここの実装不備が突かれた。
- **ドメインコントローラー（DC）**: AD ドメインの認証・ポリシーを司るサーバ。陥落＝ドメイン全制圧。
- **バッファオーバーフロー（CWE-121）**: 入力がバッファ容量を超えて隣接メモリを破壊する欠陥。制御奪取の起点。
- **0クリック / 認証前（pre-auth）**: 被害者の操作もログインも不要で成立する攻撃。最も危険な条件。
- **SYSTEM 権限**: Windows で最も強力なローカル権限。DC 上での SYSTEM 実行はドメイン管理者相当以上の脅威。

## 影響範囲 / 推奨対応

ドメインコントローラーとして稼働する Windows Server が対象。未認証・0クリックで DC を SYSTEM 掌握できるため、AD を運用する全組織が影響を受けうる。2026年5月の更新を全 DC に即時適用し、適用までは Netlogon/RPC の到達元を管理セグメントに限定すること。実環境で悪用済みのため、適用後もイベントID 4720/4728/4688 や不審な Netlogon 発信の遡及調査を強く推奨する。
