# エレコム製無線LANルーターに認証回避＋OSコマンドインジェクション（CVE-2026-40621 / CVE-2026-42062）

> トピック: 脆弱性ニュース ｜ 出典: JVN (JVN#03037325) / エレコム / INTERNET Watch ｜ 種別: 認証回避＋OSコマンドインジェクション

## 一行サマリー

国内ベンダー・エレコム製の無線LANルーター／アクセスポイント計21製品に、認証なしで操作される欠陥（CVE-2026-40621）と、認証なしで任意OSコマンドを実行される欠陥（CVE-2026-42062）を含む複数の脆弱性。2つを連鎖させると外部から機器を完全掌握されうるため、最新ファームへの更新が必須。

## 🔰 初学者向け（何が・なぜ問題か）

対象は、家庭や職場でインターネットに繋ぐための国産 Wi-Fi ルーターなどです。今回は大きく2つの弱点が見つかりました。

1つ目（CVE-2026-40621）は**「特定のURLを知っていれば、ログインしなくても操作できてしまう」**問題です。本来ならパスワードで守られている管理機能に、"合鍵になる住所"を直接叩くだけで入れてしまうイメージです。

2つ目（CVE-2026-42062）は**「機器に命令を注入して、好きなコマンドを実行できてしまう」**問題です。設定値を入れる欄に、値のふりをした"命令"を紛れ込ませると、ルーターがそれを本物の命令として実行してしまいます。

怖いのは、この2つが**組み合わさる**ときです。1つ目で認証をすり抜け、2つ目で命令を実行すれば、攻撃者はパスワードを一切知らないまま、ルーターを乗っ取れます。乗っ取られると通信の盗み見、偽サイト誘導、他機器への攻撃の踏み台化などが起こりえます。対策はシンプルで、メーカーが出した**対策済みファームウェアへ更新する**ことです。

## 🧠 専門的解説（技術詳細・メカニズム・検知対応）

### 技術的詳細（種別・前提条件）

- **脆弱性種別**: CVE-2026-40621 = 認証回避（CWE-288/CWE-306: 認証の欠落・不備）／ CVE-2026-42062 = OSコマンドインジェクション（CWE-78）。
- **CVSS**: 「要確認」（JVN/NVD の各エントリ参照。認証不要でのコマンド実行相当のため実害は高い）。
- **前提条件**: 管理インターフェイスへ到達可能なこと（LAN 内、あるいはリモート管理が有効なら WAN からも）。認証は**不要**。
- **影響機種（例・v で示す版以前）**: WRC-X3000GS2-B/W (v1.09)、WRC-X6000QS-G (v1.14)、WRC-XE5400GS-G / WRC-XE5400GSA-G (v1.13)、WRC-BE72XSD-B/BA (v1.1.1)、WRC-BE65QSD-B (v1.1.0)、WRC-W702-B (v1.1.0)、WAB-BE187-M (v1.1.10)、WAB-BE72-M / WAB-BE36-M / WAB-BE36-S (v1.1.3) 他、計21製品。
- **修正版**: 各機種の対策済み最新ファームウェア（エレコム公式アドバイザリ参照）。
- **悪用状況**: 実悪用の広域報告は現時点で「要確認」。ただし認証不要RCEに連鎖しうるため、ルーター系はスキャン対象になりやすく優先度は高い。

### メカニズム（攻撃が成立する仕組み）

以下は「型の説明」であり、具体的なエンドポイント名・注入文字列・PoCは示さない。

1. **信頼境界**: 利用者／攻撃者 → ルーター管理機能。ここは本来「認証済みの管理者のみ」を通す境界。
2. **認証回避の不備（CVE-2026-40621）**: 一部の管理機能に**アクセス制御が掛かっていない**か、URL を直接指定すると認証チェックを経ずに到達できる（強制ブラウジング／認可欠落）。攻撃者は"URL を知る"だけで管理操作のプリミティブを得る。
3. **コマンド注入の不備（CVE-2026-42062）**: 管理機能が受け取ったパラメータを、シェルコマンドの組み立てに**無害化せずそのまま連結**している。攻撃者はパラメータに区切り文字や制御構文を混ぜ、データを命令として解釈させる。
4. **橋渡し（連鎖）**: 認証回避（手順2）で"認証を通さずに"コマンド注入エンドポイント（手順3）へ到達 → 認証不要の任意コマンド実行が成立。単体でも危険な2欠陥が連鎖して被害が最大化する。
5. **最終被害**: 機器の完全掌握、設定改ざん（DNS 書き換えによる中間者化）、内部ネットワーク侵入、ボットネット組み込み。
- **一般原理**: 「認可（この人はこの操作をしてよいか）の欠落」と「入力を命令として扱ってしまうインジェクション」は、それぞれ単体でも重大だが、連鎖すると"認証不要RCE"という最悪クラスになる。組込み機器では両者が同居しやすい。

### 検知・暫定緩和・恒久対策

- **検知の着眼点**: 管理画面への外部アクセス、認証を伴わない設定変更、DNS 設定やファーム改ざんの痕跡、機器からの不審なアウトバウンド。
- **暫定緩和（更新前）**: リモート管理（WAN 側管理）を無効化、管理アクセスを信頼セグメントに限定、管理者パスワードを強固なものに変更、不要なポート開放を停止。
- **恒久対策**: 各機種の**対策済み最新ファームウェアへ更新**。更新後は設定の再点検（DNS・管理者アカウント・リモート管理設定）。
- **運用示唆**: 国産ルーターでも認可欠落＋インジェクションの複合は珍しくない。IoT 機器はファーム更新の運用（自動更新設定・定期点検）を徹底する。

## 📖 用語解説（このニュースとのつながり）

- **認証回避（強制ブラウジング）**: ログインを経ずに、URL を直接指定して保護された機能へ到達する攻撃。CVE-2026-40621 の本質。
- **OSコマンドインジェクション**: 入力値に混ぜた文字列を、機器がOSコマンドとして実行してしまう欠陥。CVE-2026-42062 に該当し、機器掌握につながる。
- **脆弱性の連鎖（Exploit Chain）**: 単体では限定的な複数欠陥を組み合わせ、認証回避→コマンド実行のように被害を拡大する手法。本件の危険度が高い理由。
- **DNS 書き換え**: ルーター掌握後、名前解決を攻撃者サーバに向けて偽サイトへ誘導する典型的悪用。中間者攻撃の入口。
- **リモート管理**: WAN 側から管理画面に入れる機能。有効だと外部から連鎖攻撃を受けやすく、無効化が重要な緩和策。

## 影響範囲 / 推奨対応

- **影響範囲**: エレコム製無線LANルーター／アクセスポイント計21製品（上記対象版以前）。家庭・SOHO・小規模拠点での利用が中心。
- **推奨対応**: (1) 対象機種の最新ファームへ更新、(2) 更新前はリモート管理無効化・管理アクセス制限、(3) 更新後にDNS/管理者設定の改ざん有無を点検、(4) 自動更新の有効化と定期棚卸し。
