# Cisco Unified CM / SME 未認証SSRF CVE-2026-20230 がroot権限奪取に悪用（PoC公開）

> トピック: 脆弱性ニュース ｜ 出典: Cisco PSIRT / BleepingComputer / CISA KEV ｜ 種別: サーバーサイドリクエストフォージェリ（CWE-918）→ ファイル書き込み→ root昇格

## 一行サマリー

Cisco Unified Communications Manager（Unified CM）と Unified CM SME に未認証のSSRF脆弱性があり、攻撃者がOS上にファイルを書き込んで最終的にroot権限へ昇格できる。CVSSは8.6だがCiscoは“Critical”評価。公開PoCがあり実環境で悪用中、CISAは6/25にKEV登録。前提はWebDialerサービスの有効化のみ。

## 🔰 初学者向け（何が・なぜ問題か）

Cisco Unified CMは、企業の**社内電話交換機（IP電話の頭脳）**です。内線・通話制御を一手に担う、止まると業務が止まる種類のサーバーです。

SSRF（サーバーサイドリクエストフォージェリ）は、平たく言うと「**受付係に“代わりにこの住所へ行ってきて”と命じて、本来外部から触れない奥の部屋を代理で操作させる**」攻撃です。攻撃者自身は奥に入れなくても、信頼されているサーバー自身に内部向けの操作を肩代わりさせます。

今回はそれが「サーバー内部にファイルを置く」ところまで悪用でき、置いたファイルを足がかりに**root（最高権限）**まで上り詰められます。しかもログイン不要。条件は「WebDialer」という発信補助機能がオンになっていることだけ（初期設定ではオフ）。電話の頭脳を乗っ取られれば、盗聴・なりすまし発信・社内ネットワークへの侵入の踏み台と、被害は通話にとどまりません。

## 🧠 専門的解説（技術詳細・メカニズム・検知対応）

### 技術的詳細（種別・前提条件）

- **CVE**: CVE-2026-20230 / **CVSS v3.1**: 8.6（High。ただしCiscoは“Critical”——スコープ変更を伴う攻撃連鎖を反映）/ **種別**: SSRF（CWE-918）。特定HTTPリクエストの入力検証不備に起因。
- **対象**: Cisco Unified CM および Unified CM SME。**影響版**: 14.x（14SU6 より前）、15.x（15SU5 より前）。**修正版**: 14SU6、15SU5。
- **前提条件**: WebDialer サービスが有効であること（**既定では無効**）。未認証で到達可能。
- **悪用状況**: 実環境で悪用（active exploitation）、公開PoCあり、CISA KEV登録（2026-06-25）。

### メカニズム（攻撃が成立する仕組み）

「外部の未認証リクエスト ↔ サーバーが内部権限で行う後続処理」という信頼境界で、リクエスト先（宛先URL/リソース）の検証が不十分なことが起点。

1. **入力検証不備→SSRF成立**: WebDialer関連の特定HTTPリクエストで、サーバーが参照・取得する宛先を攻撃者が部分的に制御できる。サーバーは“自分自身を信頼された主体”として内部リソースへアクセスするため、外部から直接触れない面が代理操作される（CWE-918）。
2. **SSRFからファイル書き込みへの転化**: 単なる内部読み取りにとどまらず、攻撃者は応答処理やパス制御の不備を介して**OS上の任意（または準任意）ファイル書き込み**プリミティブを獲得。これが「読めるだけ」のSSRFと一線を画す危険点。
3. **ファイル書き込み→root昇格**: 書き込んだファイル（起動スクリプト/設定/権限の高いコンテキストで読まれる資産等）を踏み台に、後続でroot権限のコード実行へ橋渡し。スコープが変わる（アプリ境界を越えてOS最高権限へ）ため、CiscoはCVSS値より重い“Critical”と判断。
4. **被害の波及**: rootを取られたUC基盤は、通話傍受・経路改ざん・横展開の拠点になる。

脆弱性クラスとしての一般原理（SSRF＋安全でないファイル書き込み）を示すにとどめ、**武器化手順・正確なパス・PoCの具体は記載しない**。

### 検知・暫定緩和・恒久対策

- **検知の着眼点**: WebDialer関連エンドポイントへの未認証/異常リクエスト、UC基盤からの想定外の内部・外部通信、新規/改変されたファイルやcron/起動スクリプト、rootでの異常プロセス。Cisco提供のIoC・ログ指標。
- **暫定緩和**: 不要なら **WebDialerサービスを無効化**（既定無効に戻す）。管理/サービスインタフェースの露出をネットワークで限定。
- **恒久対策**: 14SU6 / 15SU5 以降へ更新。更新後も侵害有無（追加ファイル・アカウント・永続化）を点検。

## 📖 用語解説（このニュースとのつながり）

- **Unified CM（CUCM）**: シスコのIP電話制御基盤。今回の標的で、止まると通話業務が停止する。
- **SSRF（CWE-918）**: サーバーに“代理アクセス”をさせる攻撃。今回は内部操作→ファイル書き込みに発展した。
- **WebDialer**: クリック発信を補助するサービス。これが有効だと攻撃面が開く（既定は無効）。
- **root昇格**: OSの最高権限を奪うこと。ファイル書き込みプリミティブからの橋渡しで到達。
- **スコープ変更（CVSS）**: 脆弱なコンポーネントを越えて別境界に影響が及ぶこと。Ciscoが“Critical”と重く見た理由。

## 影響範囲 / 推奨対応

Unified CM 14.x/15.x を運用する企業・組織が対象。国内でもシスコUCは大企業・自治体で広く稼働。**WebDialerの要否を即確認（不要なら無効化）し、14SU6/15SU5へ更新、露出を遮断、侵害痕跡を点検**すること。PoC公開済みのため猶予は小さい。
