# Google OmniFlash と Interactions API — any-to-any 生成と会話で編集する新世代 API

> トピック: Google OmniFlash（Omni ファミリー）と Interactions API GA｜出典: ThursdAI / Google｜種別: 新モデル/新 API のリリース

## 一行サマリー

Google が「any-to-any（任意の入出力を跨ぐ）」Omni ファミリーの第一弾 OmniFlash を発表。最長10秒の動画を生成し、Interactions API 経由の会話的なマルチターン編集で精密に手直しできる。この Interactions API は同週に GA（一般提供）到達し、生成物を「作って終わり」ではなく「対話しながら仕上げる」ワークフローを標準化する。

## 🔰 初学者向け（何が・なぜ重要か）

これまでの生成 AI は「テキスト→画像」「テキスト→動画」のように、入口と出口が決まっているものが主流でした。any-to-any（エニー・トゥ・エニー）とは、テキスト・画像・音声・動画などを自由に組み合わせて入れ、自由な形で出せる、という考え方です。OmniFlash はその第一弾で、動画を生み出せます。

もう一つの目玉が Interactions API という「対話しながら直す」仕組みです。従来は、生成した動画が気に入らなければプロンプト（指示文）を書き直して一から作り直していました。新しい方式では「もう少し明るく」「この人物を右に」と会話で指示を重ね、少しずつ完成へ近づけられます。ちょうど人間のデザイナーに口頭で修正を頼むような感覚です。作って一発勝負ではなく、対話で仕上げる——この使い勝手の変化が、制作系の AI 活用を大きく実用的にします。

## 🧠 専門的解説（技術詳細・設計・使いどころ）

### 技術的詳細（何が変わったか）

- Google が **OmniFlash** を発表。any-to-any の **Omni ファミリー**第一弾で、最長10秒の動画生成に対応。
- **Interactions API** 経由の会話的マルチターン編集をサポート。生成物を対話で逐次修正できる。
- Interactions API は同週に **GA（一般提供）**到達。
- 併せて画像生成の NanoBanana 2 Lite なども同時期に登場（別記事参照）。
- モデルの詳細仕様（解像度、料金、コンテキスト、対応モダリティの範囲）は一次情報での確認を推奨（要確認）。

### 仕組み / 背景（なぜこう設計されたか）

「any-to-any」は、複数モダリティ（テキスト・画像・音声・動画）を単一の表現空間で扱う統合モデルの方向性を指す。個別の専用モデル（画像用・動画用）を繋ぎ合わせるより、共通の内部表現で入出力を跨げるほうが、モダリティ間の一貫性（例: 参照画像の人物を動画でも保つ）を保ちやすい。OmniFlash が「Omni ファミリーの第一弾」と位置づけられるのは、この統合基盤の上に順次モダリティ・能力を載せていく設計思想の表れだ。

Interactions API の本質は「生成をステートフルな対話に変える」ことにある。従来のテキスト→動画は基本ステートレスで、修正はプロンプト全体の再指定になり、以前の結果との連続性を保ちにくい。会話的編集は、直前の生成状態をコンテキストとして保持し、「差分の指示」を適用する。これは LLM のマルチターン会話と同じ原理を生成メディアに持ち込むもので、ユーザーは全体を作り直さず局所修正を積める。トレードオフは状態管理コストと一貫性の維持——編集を重ねるほど、意図しないドリフト（少しずつ元から離れる）や、以前の指示との矛盾が起きうる。ゆえに API 側は編集履歴・バージョン・ロールバックの扱いが重要になる。制作フローとしては「粗く一発生成 → 会話で詰める」が定石化する。

### 実務での使いどころ・移行の勘所

短尺動画（広告・SNS・プロトタイプ）、モックアップ、コンテンツのバリエーション量産で有望。会話的編集は非エンジニアのクリエイターにも扱いやすく、制作イテレーションを速める。導入時は、(1) 出力の権利・利用規約（商用可否、学習利用の扱い）、(2) 生成物の来歴・電子透かし（真正性）対応、(3) 料金がターン数・秒数に応じて増える課金構造、(4) 一貫性ドリフトへの対処（重要フレームは固定する等）を確認する。既存の画像/動画パイプラインへ組み込む場合、Interactions API のステートフルなセッション管理をアプリ側の状態とどう同期するかが設計ポイント。仕様更新が速いため、依存は疎結合に保ちたい。

## 📖 用語解説（このトピックとのつながり）

- **any-to-any**: 任意モダリティを跨いで入出力する生成方式。Omni ファミリーの思想。
- **OmniFlash**: Google の any-to-any 第一弾。動画生成が可能。本トピックの主役。
- **Interactions API**: 生成物を会話で逐次編集するステートフル API。今回 GA 到達。
- **マルチターン編集**: 差分指示を積み重ねて仕上げる手法。作り直し不要の鍵。
- **ドリフト**: 編集を重ねるうち元意図から徐々に離れる現象。一貫性管理の課題。

## 影響範囲 / 推奨アクション

- 影響: 制作・マーケ・コンテンツ生成に AI を使うチーム。イテレーション速度と表現の幅に波及。
- 推奨: 短尺動画/モックで検証。権利・来歴・課金構造・一貫性ドリフトを確認し、セッション状態をアプリと疎結合に統合。仕様は一次情報で確認。
