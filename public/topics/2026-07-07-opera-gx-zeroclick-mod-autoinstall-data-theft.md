# Opera GX にゼロクリックで悪性MODを自動インストールされる致命的欠陥

> トピック: 脆弱性ニュース ｜ 出典: The Hacker News / Threat-Modeling.com / Opera ｜ 種別: 拡張（MOD）の無承認自動インストール → 情報窃取

## 一行サマリー
ゲーマー向けブラウザ Opera GX に、悪意あるWebサイトがユーザーのクリックやApprovalなしにブラウザ拡張（GX Mod）を勝手にインストールできる欠陥が判明。研究者はページ訪問だけでログイン中のGmailアドレスを丸ごと復元するPoCを実演した。バージョン 130.0.5847.89 で修正済み。

## 🔰 初学者向け（何が・なぜ問題か）
Opera GX は、見た目や機能を「MOD（モッド）」という着せ替え部品でカスタマイズできる、ゲーマーに人気のブラウザです。普通、こうした拡張機能を入れるときは「本当にインストールしますか？」という確認ボタンを押す必要があります。この確認こそが、勝手なソフトを入れられないための最後の砦です。

今回の欠陥は、その**確認ボタンを一切出さずに、Webサイト側が裏で勝手にMODを取り付けられてしまう**というものです。しかも「クリック不要（ゼロクリック）」。悪意あるサイトを開いた瞬間、目に見えない仕掛け（隠しフレーム）を通じてMODが静かに導入されます。

そのMODは、あなたが見ているページの表示を細工することで、**画面に出ている情報（たとえばログイン中のGmailアドレスなど）をこっそり盗み出せる**ことが実演されました。ユーザーは何もクリックしていないのに情報を抜かれる——「玄関の鍵をかけ忘れた」どころか「勝手に合鍵を作って侵入された」ような状態です。幸い最新版で塞がれているため、Opera GX を使っている人は今すぐ更新すべきです。

## 🧠 専門的解説（技術詳細・メカニズム・検知対応）

### 技術的詳細（種別・前提条件）
- **種別**: ブラウザ拡張（GX Mod）の無承認・ゼロクリック自動インストールを許す欠陥。CVE は未採番。Opera のバグバウンティで **P1（Critical）** 評価、最高額 $5,000 が支払われた。
- **前提条件**: 被害者が Opera GX で攻撃者の用意した Web ページを開くこと。ユーザーのクリックや承認は不要。
- **影響/修正**: バージョン **130.0.5847.89** で修正。`opera://about` で更新を確認可能。Opera は「実環境での悪用の証拠は確認していない」としている。
- **制約**: GX Mod は JavaScript を実行できず権限も持たないが、**カスタムCSSでサイトの表示を改変できる**ため、これが情報窃取のプリミティブとして機能した。

### メカニズム（攻撃が成立する仕組み）
承認プロンプト欠落 + サイドチャネル的なCSS悪用による情報漏えいの一般原理として、以下のように成立する（概念レベル。武器化手順は示さない）。

1. **信頼境界の誤り**: 本来「拡張のインストール」はユーザーの明示同意を要する強い操作で、Webコンテンツ（低信頼）とブラウザ機能（高信頼）の境界を越える。Opera GX の MOD パイプラインは、この境界で**承認プロンプトを出さずに自動でダウンロード＆有効化**する実装になっていた。
2. **配信トリガーの悪用**: 悪意あるページが、MODパッケージ（`.crx` 相当）を指す隠し iframe を読み込ませることでインストール処理を起動できた。ユーザー操作を介さず（ゼロクリック）にパイプラインを踏ませる点が核心。
3. **プリミティブの獲得**: 導入されたMODは JavaScript も権限も持たないが、**任意のカスタムCSSを対象ページに注入**できる。CSS は本来「見た目」だけだが、セレクタと属性依存の背景読み込み等を組み合わせると、"ページ上に特定の文字列/属性が存在するか" を外部リクエストの発生有無として観測できる（CSSベースのデータ窃取＝サイドチャネル）。
4. **情報復元への橋渡し**: この観測を1文字ずつ積み重ねることで、ログイン中ユーザーのGmailアドレス等の可視情報を段階的に再構成できる。研究者は1回のページ訪問・ゼロクリックでフルアドレス復元を実演した。

脆弱性クラスとしては「重要操作の同意欠如（無承認インストール）」と「CSSインジェクションによる情報漏えい（サイドチャネル）」の連鎖。単体では無害に見えるCSSが、承認欠落と結びついて実害に転じた点が学び。

### 検知・暫定緩和・恒久対策
- **検知の着眼点**: 覚えのない GX Mod が有効化されていないか（拡張/MOD一覧の点検）。サイト表示の不自然な改変。ブラウザからの想定外の外向きリクエスト。
- **暫定緩和**: 信頼できないサイトの閲覧を避ける。MOD 一覧を点検し不審なものを削除。
- **恒久対策**: Opera GX を **130.0.5847.89 以降** へ更新（`opera://about`）。組織的にはブラウザバージョンのインベントリ管理と自動更新の徹底。
- **運用示唆**: コンシューマ向け派生ブラウザ（ゲーマー向け等）も業務端末に混在し得るため、管理対象に含めてパッチ状況を把握する。

## 📖 用語解説（このニュースとのつながり）
- **Opera GX / GX Mod**: ゲーマー向け Opera 派生ブラウザと、その見た目・機能を変える拡張部品。本件はこのMODが無断導入された。
- **ゼロクリック**: 被害者の操作を一切必要とせず成立する攻撃。閲覧しただけで被害が及ぶため危険。
- **無承認インストール**: 通常必要な「インストールしますか？」の同意を回避して拡張が入る欠陥。本件の起点。
- **CSSインジェクション／サイドチャネル**: スクリプトを使わずCSSの挙動差から情報を推測・抽出する手口。本件の窃取メカニズム。
- **.crx / iframe**: 拡張パッケージ形式と、ページ内に別文書を埋め込む仕組み。隠しiframeがインストール起動に悪用された。

## 影響範囲 / 推奨対応
Opera GX 利用者全般（特に業務端末に混在するケース）。攻撃面は Opera GX 利用者に限られるが、ゼロクリック・無承認という性質から危険度は高い。130.0.5847.89 以降へ即時更新し、身に覚えのないMODを点検・削除する。
