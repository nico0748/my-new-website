# 北朝鮮系「PolinRider」、npm/Packagist/Go/Chromeに108個の悪性パッケージを投下

> トピック: 脆弱性ニュース ｜ 出典: The Hacker News / Socket ｜ 種別: ソフトウェアサプライチェーン攻撃（悪性パッケージ配布）

## 一行サマリー
北朝鮮系の脅威アクター（Contagious Interview 系）が、npm・Packagist・Go modules・Chrome拡張にまたがり計108個の悪性パッケージ／拡張を公開する「PolinRider」キャンペーンを継続中。開発者を偽の採用課題で騙してマルウェアを導入させる手口で、今年最大級のサプライチェーン攻撃の一つ。

## 🔰 初学者向け（何が・なぜ問題か）
現代のソフトウェア開発は、料理でいえば「市販の出来合いパーツ（部品）」をたくさん組み合わせて作ります。npm や Go、PHP の Packagist は、その部品を無料で配っている「巨大な部品倉庫」です。開発者は毎日、この倉庫から部品を取り寄せてアプリに組み込みます。

今回の攻撃は、**その部品倉庫に「見た目はまともだが中に毒が仕込まれた部品」を108個も混ぜ込んだ**ものです。うっかりその部品を取り寄せて自分のアプリに使うと、開発者のパソコンや、作ったアプリに毒（マルウェア）が入り込みます。1つの毒入り部品が、それを使う何千ものアプリ・企業へ連鎖的に広がる——これが「サプライチェーン攻撃」の怖さです。

さらに攻撃者は「偽の求人・採用面接」も使います。開発者に「技術課題をやってください」と偽のコーディング課題を渡し、その中に悪性パッケージを仕込んで実行させる（Contagious Interview＝"感染する面接"）。**信頼している倉庫や、正当に見える採用プロセスが武器になる**ため、警戒しにくいのが厄介な点です。

## 🧠 専門的解説（技術詳細・メカニズム・検知対応）

### 技術的詳細（種別・前提条件）
- **種別**: 協調的なソフトウェアサプライチェーン攻撃。単一 CVE はなし。
- **規模**: 108個の悪性パッケージ／拡張（npm 19、Composer/Packagist 10、Go modules 61、Chrome 拡張 1）。悪性リリース成果物は 162 個を特定（Socket の Karlo Zanki 氏の報告）。
- **帰属**: 北朝鮮系で「Contagious Interview」キャンペーンに関連づけられるアクター。キャンペーンは進行中で新規パッケージが継続出現。
- **前提条件**: 被害者（多くは開発者）が悪性パッケージを取得・実行すること。偽の採用課題（技術アセスメント/コーディングチャレンジ）を通じた誘導が併用される。

### メカニズム（攻撃が成立する仕組み）
サプライチェーン汚染の一般原理として、以下のように最終被害へ橋渡しされる（概念レベル）。

1. **信頼境界の悪用**: 開発エコシステムは「レジストリ上のパッケージは概ね信頼できる」という前提で回っている。攻撃者はこの"暗黙の信頼"を境界の弱点として突く。正規に見える名前・説明・バージョンで悪性物を公開し、正当パッケージと見分けにくくする（typosquatting／メンテナ乗っ取り／正規リポジトリ改変など）。
2. **実行トリガーの獲得**: 多くのエコシステムでは依存導入時やビルド時に**インストールスクリプト（例: postinstall）やモジュール初期化が自動実行**される。ここが「取り寄せ＝コード実行」に化けるプリミティブ。開発者が明示的にコードを書かなくても、依存を入れた瞬間に攻撃コードが走る。
3. **初期アクセスの確立**: 実行された悪性コードは、情報窃取（認証情報・トークン・鍵）、追加ペイロード取得、開発端末のバックドア化を行う。開発者は往々にしてクラウド鍵やソース権限を持つため、被害インパクトが大きい。
4. **二次拡散への橋渡し**: 窃取した認証情報でCI/CDやリポジトリを侵害すれば、被害者が公開する製品にも毒が混入し、その利用者へ連鎖する（サプライチェーンの多段汚染）。「偽採用課題」は、レジストリ経由より直接的に標的開発者へ実行を仕込む補助経路。

脆弱性クラスは「信頼された配布チャネル＋自動実行フックの悪用」であり、技術的欠陥というより"信頼の運用"の問題。防御は個別バグ修正でなく供給網の検証プロセスで行う。

### 検知・暫定緩和・恒久対策
- **検知の着眼点（IoC/ログ）**: 既知IoCに一致するパッケージ名・バージョン・ハッシュ。ビルド/インストール時の想定外の外向き通信。新規・低ダウンロード数パッケージの突然の採用。メンテナアカウントの不審なリリース履歴。
- **暫定緩和**: 依存の**最小パッケージ年齢（minimum package age）ポリシー**を導入し、公開直後のパッケージを未検証で採用しない。lockfile 固定と署名検証。
- **恒久対策**: npm/Composer/Go 依存を既知IoCで棚卸し。Socket・Snyk・`npm audit` 等のレジストリスキャンを CI に組込み。開発者教育（偽採用課題への警戒）。ビルドはインストールスクリプトを無効化した隔離環境で実行。
- **運用示唆**: 開発端末とCIを"最小権限＋隔離"で運用し、クラウド鍵の有効期間短縮とローテーションを徹底する。

## 📖 用語解説（このニュースとのつながり）
- **サプライチェーン攻撃**: ソフトの部品供給網に毒を混ぜ、利用者へ連鎖感染させる手口。本件の本質。
- **npm / Packagist / Go modules**: それぞれ JavaScript / PHP / Go の公開パッケージ倉庫。本件はこの4系統（＋Chrome拡張）に悪性物が投下された。
- **Contagious Interview**: 偽の採用・面接課題で開発者に悪性コードを実行させる北朝鮮系の手口。本件の誘導経路。
- **postinstall スクリプト**: パッケージ導入時に自動実行される仕組み。"取り寄せ＝コード実行" に化ける攻撃面。
- **IoC（侵害の痕跡）**: 攻撃を示す指標（ハッシュ・ドメイン・パッケージ名等）。検知の手掛かり。

## 影響範囲 / 推奨対応
npm・Composer・Go・Chrome拡張を利用する全ての開発組織。既知IoCで依存を棚卸しし、最小パッケージ年齢ポリシー・署名検証・CIでのレジストリスキャンを導入。開発者へ偽採用課題の手口を周知し、ビルドは隔離環境・最小権限で実行する。
