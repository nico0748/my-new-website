# Microsoft Defenderの権限昇格「BlueHammer」（CVE-2026-33825）、ランサムウェア攻撃で悪用中 — CISAがKEV更新

> トピック: 脆弱性ニュース ｜ 出典: BleepingComputer / SecurityWeek / CISA(KEV) ｜ 種別: Microsoft Defender のアクセス制御粒度不足によるローカル権限昇格（LPE）

## 一行サマリー

Microsoft Defender のローカル権限昇格「BlueHammer」（CVE-2026-33825）が、ランサムウェア攻撃で悪用されていると CISA が KEV 更新で確認。原因は Defender のアクセス制御の粒度不足で、認証済みの攻撃者がローカルで SYSTEM 権限へ昇格し、SAM データベース（ローカルアカウントのハッシュ）にアクセスできる。2026-04-02 に研究者が PoC 付きで公開（ゼロデイ）、Microsoft は 2026-04-14 に修正、CISA は 04-22 に KEV 追加済み。パッチ未適用の Windows は早急な適用が必要。

## 🔰 初学者向け（何が・なぜ問題か）

Windows には最初から「Microsoft Defender」というウイルス対策の仕組みが入っています。本来は“守る側”のソフトです。ところが今回の問題は、**その守り手であるDefenderの権限管理が甘く、逆に攻撃者の踏み台にされてしまう**というものです。

コンピューターには権限のランクがあります。ふつうの利用者は「一般席」、管理者は「SYSTEM」という最上位席です。BlueHammer は、すでにパソコンに入り込んだ攻撃者（一般席）が、Defenderの穴を使って**最上位のSYSTEM席に飛び級で座れてしまう**バグです。SYSTEM を握られると、パスワードの金庫（SAMデータベース＝ローカルアカウントの暗号化パスワードが入った場所）を開けられ、セキュリティ機能を無効化し、他のPCへ広がり、最終的に**ランサムウェア（身代金要求ウイルス）で全体を暗号化**する準備が整ってしまいます。

たとえるなら、ビルの警備会社の制服（Defender）を悪用して、一般来訪者が警備員に成りすまし、マスターキーの保管庫を開けてしまうようなものです。守り手の権限が強いぶん、乗っ取られたときの被害も大きくなります。

やっかいなのは、この脆弱性が**パッチが出る前に攻撃コードごと公開されてしまった（ゼロデイ）**こと、そして現在は**ランサムウェア集団が実際に使っている**ことです。修正はすでに出ているので、Windows Update を適用していないPCは早急に更新する必要があります。

## 🧠 専門的解説（技術詳細・メカニズム・検知対応）

### 技術的詳細（種別・前提条件）

- **CVE / 種別**: CVE-2026-33825。Microsoft Defender における**アクセス制御の粒度不足（insufficient granularity of access control）**に起因するローカル権限昇格（LPE）。CVSS 値は**要確認**（LPEとして中〜高帯と推定）。
- **前提条件**: 攻撃者が対象 Windows 上で**すでに認証済みのローカル実行権限**を保有していること。単独でのリモート初期侵入には使えず、侵入後の昇格・横移動フェーズで用いられる。
- **影響 / 到達点**: SYSTEM 権限への昇格。到達後は SAM データベース（ローカルアカウントのパスワードハッシュ）へアクセス可能となり、資格情報窃取・防御無効化・横移動・ランサムウェア展開の前段が成立。
- **公開・修正タイムライン**: 2026-04-02 に研究者「Nightmare Eclipse」が MSRC の開示対応への抗議として PoC 付きで公開（＝ゼロデイ）。Microsoft は 2026-04-14 にパッチ提供。CISA は 2026-04-22 に KEV 追加、FCEB に 2026-05-07 までの適用を指示。
- **今回の更新点**: CISA が KEV を更新し、本脆弱性が**ランサムウェアキャンペーンで悪用中**と明記（BleepingComputer 2026-07-01 報）。Microsoft 自身は本記事時点で「悪用済み」タグを付していない（見解に差、**要確認**）。

### メカニズム（攻撃が成立する仕組み）

CVE の内部詳細は限定的なため、概念レベルで分解する（武器化手順・PoCは扱わない）。

1. **信頼境界**: 境界は「一般ユーザー権限で操作できる領域」と「SYSTEM/セキュリティサブシステムが管理する特権領域」の間。Defender はセキュリティ製品として高い権限で動作し、そのコンポーネントや対象リソースへのアクセス可否が厳密に制御されているべき箇所。
2. **不備の所在（アクセス制御の粒度不足）**: Defender が扱うオブジェクト（プロセス・ファイル・レジストリ・デバイス等）への権限チェックが**十分に細かく行われていない**。本来は特権主体のみが操作できるべき対象に、低権限の主体が到達・操作できる隙が生じる。これがLPEの起点。
3. **攻撃者が握るプリミティブ**: 粒度不足を突くことで、攻撃者は「特権コンテキストで動作する処理に、自分の制御下の入力・操作を差し込む」プリミティブを獲得する。アクセス制御欠落クラスでは、これが特権操作の“肩代わり”につながる。
4. **最終被害への橋渡し**: 特権操作を通じて SYSTEM 相当のコンテキストを掌握すると、SAM データベースの読み出し（ローカル資格情報の入手）、セキュリティ制御の停止、永続化、横移動が可能になる。ランサムウェア攻撃者はこの昇格をキルチェーンに組み込み、初期の足場から環境全体の暗号化・恐喝へと橋渡しする。
5. **クラスとしての位置づけ**: 「アクセス制御の欠落／粒度不足（CWE-284系）」は、権限境界の設計・実装のミスに起因する古典的だが致命的なクラス。特にセキュリティ製品自体が特権で動くため、そこにこの欠陥があると“守り手が最短の昇格経路”に反転する。メモリ破壊ではなく認可判定の穴である点が本質。

### 検知・暫定緩和・恒久対策

- **検知の着眼点**: 非管理者プロセスからの SYSTEM 昇格イベント、`lsass`/SAM への異常アクセス、Defender コンポーネントに対する想定外の操作、セキュリティサービスの停止・改ざん、資格情報ダンプの痕跡（レジストリハイブのコピー等）。EDR による昇格・防御無効化の相関検知。
- **暫定緩和**: 最小権限の徹底（一般ユーザーにローカル実行の自由度を与えすぎない）、初期侵入経路（フィッシング・露出サービス）の封じ込め、Defender の改ざん防止（tamper protection）有効化、重要資産の分離。
- **恒久対策**: 2026-04 の Microsoft パッチ（CVE-2026-33825 対応）を適用。KEV 掲載＆ランサムウェア悪用中のため、未適用端末は緊急対応。適用状況の全社棚卸しと、露出端末の優先パッチ。
- **運用示唆**: LPE 単体は初期侵入とセットで危険度が跳ね上がる。初期アクセスの遮断（メール・資格情報保護）と昇格の検知を二層で固める。

## 📖 用語解説（このニュースとのつながり）

- **Microsoft Defender**: Windows 標準のセキュリティ機能。本来は防御側だが、権限管理の不備で昇格の踏み台に転用された。
- **ローカル権限昇格（LPE）**: 端末内で低い権限から SYSTEM 等の高権限へ格上げする攻撃。BlueHammer はこの典型。
- **SYSTEM 権限**: Windows における最上位のローカル権限。握られると事実上その端末を完全支配できる。
- **SAM データベース**: ローカルアカウントのパスワードハッシュを保管する領域。SYSTEM 昇格後にアクセスされ、資格情報窃取に使われる。
- **KEV（Known Exploited Vulnerabilities）**: CISA が「実際に悪用が確認された脆弱性」をまとめた目録。掲載＝優先対応の強いシグナル。
- **ゼロデイ**: 修正が出る前に悪用・公開される脆弱性。BlueHammer は PoC がパッチ前に公開された。
- **アクセス制御の粒度不足（CWE-284系）**: 権限チェックが粗く、本来許してはいけない操作を低権限主体に許してしまう欠陥。本件の根本原因。

## 影響範囲 / 推奨対応

Microsoft Defender を搭載しパッチ未適用の Windows 端末が対象。すでにランサムウェア攻撃で悪用中かつ CISA KEV 掲載のため、**最優先で 2026-04 の修正を適用**する。あわせて、初期侵入の遮断（フィッシング対策・露出サービス縮小）、Defender の改ざん防止有効化、EDR による SYSTEM 昇格・資格情報ダンプの検知、適用状況の全社棚卸しを行う。LPE は初期アクセスと組み合わさって被害が拡大するため、二層防御を徹底する。
