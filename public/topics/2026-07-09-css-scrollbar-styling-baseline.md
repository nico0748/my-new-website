# 標準スクロールバー装飾（scrollbar-width / scrollbar-color）が全ブラウザ対応 — Baseline 入り

> トピック: 標準スクロールバー系プロパティが Baseline｜出典: MDN / web.dev｜種別: CSS 機能の相互運用達成

## 一行サマリー

スクロールバーの太さと色を標準 CSS で指定できる `scrollbar-width` / `scrollbar-color` が、Safari を含む主要ブラウザで足並みが揃い Baseline に到達した。ベンダー独自の `::-webkit-scrollbar` に頼らなくても、細いスクロールバーやテーマに合わせた配色をクロスブラウザで実現できる。

## 🔰 初学者向け（何が・なぜ重要か）

Web サイトの右端に出るスクロールバー。ダークテーマなのにスクロールバーだけ明るくて浮いて見える、という経験はよくあります。これを整えたくても、これまでは「WebKit（Chrome/Safari）だけで効く特殊な書き方」と「Firefox 用の書き方」を別々に用意する必要があり、面倒でした。

今回、`scrollbar-width`（太さ）と `scrollbar-color`（つまみと溝の色）という**標準の書き方**が、すべての主要ブラウザで使えるようになりました（Baseline 入り）。「スクロールバーを細くする」「暗い背景に馴染む色にする」といった調整を、1種類の素直な CSS で全ブラウザに効かせられます。細かい部分ですが、ダッシュボードやサイドバーの多いアプリでは見た目の完成度がぐっと上がります。

## 🧠 専門的解説（技術詳細・設計・使いどころ）

### 技術的詳細（何が変わったか）

- `scrollbar-color`（2025年12月）に続き `scrollbar-width` も相互運用が成立し、標準スクロールバー装飾が Baseline に。
- `scrollbar-width`: `auto` / `thin` / `none` の3値。`none` は「バーを隠すがスクロールは可能」（オーバーフロー挙動は維持）。
- `scrollbar-color: <thumb> <track>`: つまみ（thumb）と溝（track）の2色を指定。`auto` でブラウザ既定に戻す。
- これらは論理プロパティで、要素単位・`:root` 単位のどちらでも指定可能。`overflow` を持つ任意のスクロールコンテナに効く。
- 従来の `::-webkit-scrollbar` 系疑似要素は非標準ながら現存。細粒度の造形が必要な場合の使い分けが残る。

### 仕組み / 背景（なぜこう設計されたか）

スクロールバーは OS とブラウザが描画する UI 部品であり、Web からの制御は歴史的に「非標準の WebKit 疑似要素」か「Firefox 独自プロパティ」に分裂していた。前者はピクセル単位で造形できる反面、標準化されず、Safari/Firefox 間で挙動が割れた。標準化にあたっては「細かく造形する自由度」より「クロスブラウザで安全に太さと色を整えられる最小限」を優先し、`scrollbar-width`（3段階）と `scrollbar-color`（2色）という**意図的に抑制した API**が採られた。

トレードオフは明確だ。角丸や影、ホバー時の造形といった凝った表現は標準プロパティでは不可能で、そこは引き続き `::-webkit-scrollbar`（非標準）に頼るしかない。だが「アクセシビリティと OS 一貫性を壊さない範囲で最小限を標準化する」という設計判断により、大多数のユースケース（細くする・配色を合わせる）はクロスブラウザで安全に満たせるようになった。過剰にスクロールバーを消す・細くすると操作性を損なうため、標準側が値を絞ったのは合理的でもある。

### 実務での使いどころ・移行の勘所

ダーク/ライトのテーマ切替を持つアプリ、サイドバーやチャット欄など内部スクロール領域の多い UI、デザインシステムの一貫性が求められる製品で効果的。移行は「`::-webkit-scrollbar` で太さ・色だけ調整していた箇所」を標準プロパティへ置換するところから。`scrollbar-width: none` で完全に隠す場合は、スクロール可能であることが視覚的に分かる別の手掛かり（フェード影など）を添え、キーボード/タッチ操作を損なわないよう配慮する。凝った造形が要る箇所のみ WebKit 疑似要素を併用する。

## 📖 用語解説（このトピックとのつながり）

- **scrollbar-width**: スクロールバーの太さ（auto/thin/none）。本トピックの中核。
- **scrollbar-color**: つまみと溝の2色指定。テーマ整合の要。
- **::-webkit-scrollbar**: 非標準の造形用疑似要素。細かい表現が要る場合の使い分けが残る。
- **Baseline**: 主要ブラウザで相互運用が成立した状態。今回この2プロパティが到達。
- **オーバーフロー（overflow）**: 要素からあふれた内容のスクロール挙動。スクロールバー装飾の対象領域を決める。

## 影響範囲 / 推奨アクション

- 影響: テーマを持つ全 Web UI。細部の見た目品質とクロスブラウザ一貫性に寄与。
- 推奨: WebKit 疑似要素での太さ・色調整を標準プロパティへ置換。`none` 使用時は操作性の手掛かりを維持。
