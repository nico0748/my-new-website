# FortiBleed — 約74,000台のFortinet VPN認証情報が流出（弱いSHA-256ハッシュの解読）

> トピック: 脆弱性ニュース ｜ 出典: BleepingComputer / Help Net Security / CISA / JPCERT/CC(at260019) ｜ 種別: 認証情報漏えい（CVEなし・国内でも注意喚起）

## 一行サマリー

「FortiBleed」と呼ばれる大規模キャンペーンで、約74,000台のFortinet機器のVPN/管理者認証情報が流出。原因は新しい脆弱性ではなく、**アップグレード後に残る弱いSHA-256ハッシュのパスワードをオフライン解読された**こと。JPCERT/CC も国内向けに注意喚起した。

## 🔰 初学者向け（何が・なぜ問題か）

Fortinet の FortiGate は、多くの企業が「社外から社内に安全に入るための入口（VPN）」として使う機器です。今回、その入口の**ユーザー名やパスワードが約74,000台分も流出**しました。世界中のネットに露出した FortiGate の約半数が影響し得る規模です。

原因は「新しい穴」ではなく、**古い設計の後遺症**でした。FortiGate を古い版から新しい版へアップグレードすると、管理者パスワードが**弱い方式（SHA-256）で保存されたまま**残り、管理者が一度ログインし直すまで新しい強い方式に切り替わりません。パスワードの保存には本来「解読に膨大な時間がかかる」丈夫な方式を使うべきですが、SHA-256 は総当たり解読が速く進みます。

攻撃者は**45台のGPU（画像処理用の高速チップ）を並べた解読装置**でこの弱いハッシュを一気に解き、実際に使える生パスワードを大量に手に入れました。これは「盗んだ金庫を高性能ドリルで次々こじ開けた」ようなもの。CVE（脆弱性番号）は付いておらず**当てるパッチも無い**ため、対策は「パスワードを全部変える」ことが基本になります。

## 🧠 専門的解説（技術詳細・メカニズム・検知対応）

### 技術的詳細（種別・前提条件）

- **CVE**: なし。Fortinet は「新規脆弱性ではない」と説明。**パッチ不在**。
- **種別**: 認証情報漏えい／弱いパスワードハッシュのオフライン解読（**CWE-916: 計算量不足のパスワードハッシュ**の考え方に該当）。
- **規模**: 約 **73,000〜74,000 台**、194カ国。インターネット到達可能な FortiGate の**約50%**が影響し得ると推定。
- **流出内容**: ユーザー名・メール・**平文パスワード**（解読済み）を含む。研究者 Bob Diachenko が発端を発見。
- **根本原因**: アップグレード時、管理者パスワードが**弱いSHA-256ハッシュのまま保持**され、管理者が手動で再ログインするまで強い保存方式へ移行しない仕様。
- **攻撃基盤**: 45-GPU のオフライン解読インフラで大規模にハッシュを解読。
- **国内対応**: JPCERT/CC が **2026-06-23** に「Fortinet製品に関連する認証情報の漏えいに関する注意喚起」（at260019）を公開。CISA も機器ハードニングを勧告。

### メカニズム（攻撃が成立する仕組み）

これは「脆弱性の悪用」ではなく「**弱い保存方式＋移行の隙**を突いた資格情報の再構成」である。型の説明に留め、解読手順は示さない。

1. **信頼の前提**: パスワードは平文で保存せず、**一方向ハッシュ**にして保存する。安全性は「ハッシュから元パスワードを逆算するのが計算的に非現実的」という前提に依存する。
2. **保存方式の弱さ**: パスワード保存に**汎用高速ハッシュ（SHA-256）**を使うと、GPU で毎秒膨大な候補を試せてしまう。本来はソルト付きで**意図的に遅い**方式（bcrypt/scrypt/Argon2 等）を使うべき。
3. **移行の隙（プリミティブの入手）**: アップグレード時に旧来の弱いハッシュが残置され、管理者の再ログインまで強い方式へ更新されない。攻撃者がこの弱いハッシュ群を入手できれば、**オフラインで無制限に総当たり**できる状況が生まれる。
4. **橋渡し**: 45-GPU の解読インフラで弱ハッシュを解き、**有効な平文パスワード**を復元。これが VPN/管理者ログインという**正規の認証プリミティブ**に直結する。
5. **多層の悪用**: 復元済み資格情報は、低次の犯罪者による日和見的アクセス販売から、国家関連アクターによる標的型侵入・諜報まで、**同一の資格情報プール**を土台に多層で悪用される。

一般原理として、(a) パスワード保存に高速ハッシュを使う設計、(b) バージョン移行時に旧秘密が残置される移行ロジックの不備、は「脆弱性番号は付かないが実害は甚大」という典型例。防御の要は暗号方式そのものではなく**秘密のローテーション**になる。

### 検知・暫定緩和・恒久対策

- **検知の着眼点**: 正規資格情報を使った不審なVPNログイン（未知の地理/時間帯、複数同時）、管理者ログイン後の設定変更、既知の流出データセットと自組織アカウントの突合。
- **暫定緩和**: インターネット公開の管理面/SSL-VPN を可能な限り閉じ、許可IP/多要素の背後へ。
- **恒久対策（最重要）**: **全 VPN・管理者パスワードの即時リセット**（特にインターネット公開機）、既存セッションの全終了、**MFA の強制**、強力なパスワードポリシー。アップグレード後は管理者が明示的に再ログインして保存方式を更新。
- **運用示唆**: 「CVE が無い＝安全」ではない。**資格情報の衛生（定期ローテーション・MFA・露出最小化）**が最終防衛線であることを示す事案。

## 📖 用語解説（このニュースとのつながり）

- **FortiBleed**: 今回の大規模認証情報流出キャンペーンの通称。脆弱性名ではない。
- **VPN（Virtual Private Network）**: 社外から社内へ安全に接続する仕組み。FortiGate の SSL-VPN が主な入口で、資格情報が流出した。
- **パスワードハッシュ / SHA-256**: パスワードを一方向変換して保存する仕組み。SHA-256 は高速すぎて解読されやすく、パスワード保存には不向き。
- **ソルト付き低速ハッシュ（bcrypt/Argon2）**: 意図的に計算を遅くし総当たりを困難にする方式。本来こちらを使うべきだった。
- **オフライン解読**: 盗んだハッシュを攻撃者の手元で無制限に総当たりすること。GPU で高速化される。
- **MFA（多要素認証）**: パスワード流出時の最後の砦。今回のような資格情報漏えいに対する主要な緩和策。

## 影響範囲 / 推奨対応

- **影響範囲**: インターネット公開の FortiGate/SSL-VPN を運用する組織全般。194カ国・最大約74,000台。国内組織も対象になり得る（JPCERT 注意喚起あり）。
- **推奨対応**: (1) 全 VPN・管理者パスワードを即時リセット、(2) 既存セッション全終了、(3) MFA 強制と公開面の最小化、(4) 流出データと自組織アカウントの突合、(5) 不正ログイン痕跡の遡及調査。
