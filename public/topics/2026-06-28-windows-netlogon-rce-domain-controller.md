# Windows Netlogon の未認証RCEでドメインコントローラ全面侵害（CVE-2026-41089, 実環境で悪用・CVSS 9.8）

> トピック: 脆弱性ニュース｜出典: BleepingComputer / Help Net Security / SecurityWeek / NVD｜種別: スタックベースのバッファオーバーフロー（CWE-121）による未認証リモートコード実行

## 一行サマリー
Active Directoryの認証を司る Windows Netlogon に、未認証の攻撃者がネットワーク越しに任意コードを実行できるスタックバッファオーバーフローが見つかった（CVE-2026-41089, CVSS 9.8）。ドメインコントローラ（DC）上でコード実行=AD丸ごと掌握につながり、すでに実環境で悪用が確認されている「Zerologon級」の重大脆弱性。

## 🔰 初学者向け（何が・なぜ問題か）
会社のWindowsネットワークには「ドメインコントローラ（DC）」という“門番にして名簿管理人”がいます。誰がログインできるか、どんな権限を持つかを一手に管理する、組織で最も重要なサーバーです。

その門番が使う「Netlogon」という受付窓口に、設計上の弱点がありました。Netlogonは仕事の性質上、ネットワーク上の誰からでも話しかけられるよう常に窓口を開けています。今回の脆弱性は、その窓口に「想定より長すぎる申請書」を提出すると、受付の机（メモリ）からはみ出して隣の領域を上書きできてしまう、というもの。たとえるなら、記入欄からはみ出した文字が、本来書き換えてはいけない処理手順の指示書まで上書きしてしまい、攻撃者の指示どおりに門番が動いてしまう状態です。

ログイン情報（IDやパスワード）すら不要で、ネットワークに届きさえすれば成立します。門番を乗っ取られれば、組織全員のアカウント・権限が攻撃者の手に落ち、ランサムウェアの一斉展開まで一直線。だからこそ最優先で塞ぐべき穴です。

## 🧠 専門的解説（技術詳細・メカニズム・検知対応）

### 技術的詳細（種別・前提条件）
- **CVE**: CVE-2026-41089。**CVSS 3.1**: 9.8（MSRC）。**CWE**: CWE-121（スタックベースのバッファオーバーフロー）。
- **悪用状況**: 実環境で悪用が確認（Centre for Cybersecurity Belgium が2026-05-29に「actively exploited」と更新）。PoC公開の動きあり。「Zerologon級」と評される。
- **前提条件**: 攻撃者がDCのNetlogon RPCインターフェース（通常SMB/445やRPCエンドポイント経由）に到達できること。**認証は不要**（DCが設計上このインターフェースを公開しているため）。
- **影響範囲**: DCとして動作しうる全Windows Serverエディション（出典では2008 R2〜2025、Standard/Datacenter）。
- **開示/修正**: Microsoftが2026-05-12に開示（WARPチームが報告）。6月Patch Tuesday系の更新で対応。**正確な修正KBは環境別に要確認**。

### メカニズム（攻撃が成立する仕組み）
1. **露出した信頼境界**: Netlogonの RPC インターフェースは、ドメイン参加・セキュアチャネル確立のために**未認証段階から到達可能**であることが前提。つまり「認証前のコードパス」が攻撃面として常時開いている。
2. **境界長検証の不備**: 攻撃者が制御する長さ/内容のフィールドを含む特製リクエストを送ると、固定長スタックバッファに対する境界チェックが不足しているため、**スタック領域を越えて隣接データを上書き**できる（古典的スタックBOF）。
3. **制御フローの奪取**: スタック上の戻りアドレスや関数ポインタ等の制御データが上書き可能になると、攻撃者は実行フローを自分のコードへ誘導するプリミティブを握る。これが「任意コード実行」への橋渡し。
4. **被害の最大化**: 実行先が**DCのSYSTEM文脈**であるため、コード実行はそのままドメイン全体の信頼基盤の掌握を意味する。資格情報アクセス、ディレクトリ偵察、ポリシー改ざん、横展開、ランサムウェア準備へ連鎖しうる。
   - 一般原理: 「未認証で到達できる入力パース処理」に古典的メモリ破壊（境界外書き込み）が存在すると、認証の壁を経ずに最深部の制御を奪われる。とりわけ認証基盤コンポーネントでは、単一の境界バグが組織全体の侵害に直結する。
   - 本記述は概念レベルに留め、オフセットや武器化手順・PoCコードは記載しない。

### 検知・暫定緩和・恒久対策
- **検知の着眼点**: DCに対する異常なNetlogon/RPCトラフィック、外部や非管理セグメントからの445/RPCアクセス、DC上のlsass/netlogon関連プロセスのクラッシュ・再起動、直後の不審な子プロセス・資格情報アクセス挙動。
- **暫定緩和**: DCへのSMB/RPC到達を信頼セグメントに限定（ネットワーク分離・FW）。管理プレーンの隔離。露出DCのインターネット遮断。
- **恒久対策**: 該当する6月のセキュリティ更新を全DCへ即時適用（環境別KBを確認）。適用後もNetlogonセキュアチャネルの監視を継続。
- **運用示唆**: DCは「最後の砦」。パッチ適用の遅延がそのまま全社侵害リスクになるため、メンテナンス枠を待たず緊急適用を検討。

## 📖 用語解説（このニュースとのつながり）
- **Netlogon**: Windowsの認証・ドメイン信頼を担うサービス/RPCインターフェース。本件の欠陥箇所であり、未認証で到達可能な点が致命的。
- **ドメインコントローラ（DC）**: AD環境で認証・ポリシーを集中管理するサーバー。ここでのコード実行は全社掌握に等しい。
- **スタックバッファオーバーフロー（CWE-121）**: スタック上の固定長領域を越えて書き込み、制御データを破壊する欠陥。本件の根本原因。
- **RPC（Remote Procedure Call）**: ネットワーク越しに関数を呼ぶ仕組み。Netlogonの受付窓口にあたる。
- **Zerologon**: 2020年のNetlogon重大脆弱性（CVE-2020-1472）。今回は深刻度の比較対象として引用される。

## 影響範囲 / 推奨対応
DCとして稼働する全Windows Serverが対象。AD環境を持つ組織は最優先で6月更新を全DCへ適用し、適用までDCへのRPC/SMB到達を信頼境界内に制限する。露出していた可能性があれば、Netlogonプロセスのクラッシュ履歴・資格情報アクセス・横展開痕跡を侵害調査する。
