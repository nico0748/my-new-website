# GamersFirst Anti-Cheat ドライバの特権昇格（CVE-2026-12166/12167/12168）— JVN公表・write-what-whereでSYSTEM

> トピック：アンチチートドライバのカーネル権限昇格 ｜ 出典：JVN (JVNVU#98406800) / CERT/CC VU#639124 ｜ 種別：不適切なアクセス制御・入力検証（CWE-284 / CWE-822 / CWE-476）

## 一行サマリー

オンラインゲームの不正対策ドライバ「Little Orbit製 GamersFirst Anti-Cheat（GFAC）」に複数のローカル特権昇格・DoS脆弱性が見つかり、JPCERT/CC経由でJVN（JVNVU#98406800）が注意喚起。低権限ユーザーがカーネルへ任意書き込み（write-what-where）を行いSYSTEM権限を奪取できる。

## 🔰 初学者向け（何が・なぜ問題か）

オンラインゲームには、チート（不正行為）を防ぐために「アンチチート」というソフトが一緒に入ります。その多くはPCの最も深い層＝カーネル（OSの中枢）で動く「ドライバ」として動作し、ゲームの動きを監視します。強い権限を持つ代わりに、そこに欠陥があると被害も大きくなります。

今回の GamersFirst Anti-Cheat のドライバ（GFAC.sys）には、例えるなら「本来は信頼できる係員しか使えないはずの“中枢への直通電話”が、実は誰でもかけられる状態になっていた」問題があります（CVE-2026-12167）。さらにその電話口で「この住所にこの荷物を置いてきて」と指示すると、係員が住所を確認せずにOSの中枢の好きな場所へ物を置いてしまう（CVE-2026-12168、任意のカーネルメモリ書き込み）。加えて、特定の指示で係員が固まって建物全体が止まる（CVE-2026-12166、クラッシュ＝DoS）欠陥もあります。

これを悪用すると、そのPCで動く**権限の低いプログラム**（たとえば不正アプリやマルウェア）が、自分の権限を一気に**SYSTEM（Windowsの最高権限）**まで引き上げられます。すでにPC内に潜り込んだ攻撃者にとって“最後のひと押し”になり、完全な乗っ取りへつながります。ネット越しの遠隔攻撃ではなく「ローカル」＝すでにそのPCで何か動かせる状態が前提ですが、ゲームPCは家庭にも多く、影響は無視できません。

## 🧠 専門的解説（技術詳細・メカニズム・検知対応）

### 技術的詳細（種別・前提条件）

- **CVE**：
  - CVE-2026-12166：`GFAC_Sys_x64.sys` の初期化／リクエスト処理におけるNULLポインタ参照（CWE-476）→ クラッシュ（DoS）。
  - CVE-2026-12167：ミニフィルタ通信ポートのセキュリティ記述子が不十分（CWE-284, 不適切なアクセス制御）→ 低権限ユーザーがドライバへ接続し本来信頼プロセス専用の機能へアクセス可能。
  - CVE-2026-12168：通信ポート経由メッセージ内の**ユーザー指定メモリアドレスを検証せず書き込み**（CWE-822/CWE-123, write-what-where）→ 任意カーネルメモリ書き込み→SYSTEM昇格。
- **公表**：JVN JVNVU#98406800（JPCERT/CC調整）、CERT/CC VU#639124。
- **前提条件**：**ローカル**（当該PC上でコードを実行できる低権限ユーザー）。GFACドライバがインストール・稼働していること。リモート単体では成立しない。
- **CVSS**：要確認（複数脆弱性の集合。write-what-whereによるSYSTEM昇格は高深刻度相当）。
- **悪用状況**：公開時点で実環境悪用の確証情報は確認できず（**要確認**）。ただしwrite-what-whereは信頼性の高い昇格プリミティブで、悪用容易性は高い。

### メカニズム（攻撃が成立する仕組み）

低権限からSYSTEMへの昇格が「接続の解放」→「任意書き込み」→「トークン改ざん」で橋渡しされます（概念レベル。武器化手順・オフセット・PoCは記載しません）。

1. **信頼境界の崩壊（CVE-2026-12167）**：カーネルドライバがユーザー空間と通信するために開く**ミニフィルタ通信ポート**は、本来「信頼された保護プロセス（アンチチート本体）だけが接続できる」よう、厳格なセキュリティ記述子（アクセス制御）で守るべき。ここが緩いため、**任意の低権限プロセスがドライバの特権機能に接続**できる。これが攻撃の入口。
2. **入力検証の不備（CVE-2026-12168）**：接続後、ドライバはポート経由のメッセージに含まれる「宛先アドレス」と「書き込む値」を、**検証せずにそのままカーネル空間へ書き込む**。攻撃者は「どこに（where）・何を（what）」書くかを完全制御できる——これが古典的かつ強力な **write-what-where プリミティブ**。
3. **プリミティブ→SYSTEM**：任意カーネル書き込みが得られると、攻撃者は自プロセスの**セキュリティトークン**（権限を表すカーネル構造体）を書き換え、特権を持つトークンにすり替える等の手口でSYSTEMへ昇格する。カーネル構造体を直接改変できるため、権限モデルそのものを内側から破れる。
4. **可用性影響（CVE-2026-12166）**：初期化・要求処理のNULLポインタ参照は、細工要求でカーネルパニック（BSOD）を誘発し、システム停止（DoS）を招く。
5. **クラス一般原理**：署名済みカーネルドライバは、(a) ユーザー↔カーネル間IPC（IOCTL/通信ポート）のアクセス制御、(b) 受領データ（特にアドレス・長さ・ポインタ）の境界検証、の2点を守れないと、低権限からのカーネル昇格路になる。「特権ヘルパ（強い権限で動く補助コンポーネント）の入力検証不備」の典型で、アンチチート/EDR/仮想化などカーネルドライバ全般に共通する落とし穴。

### 検知・暫定緩和・恒久対策

- **検知の着眼点**：GFACドライバの通信ポートへ、アンチチート本体以外の**予期しないプロセスが接続**していないか。カーネルクラッシュ（BSOD, Bugcheck）の頻発。EDRによる不審なトークン操作・カーネル書き込み挙動の検知。
- **暫定緩和**：ベンダ修正が出るまで、当該ゲーム/GFACを使わない端末では**アンインストール**を検討。業務PCにゲーム由来のカーネルドライバを入れない運用（アプリ許可リスト）。脆弱ドライバのブロック（Microsoftの脆弱ドライバブロックリスト等）の活用。
- **恒久対策**：Little Orbit / GamersFirst の修正版ドライバへ更新（提供状況は**要確認**）。JVN/CERT/CCの続報を追跡。
- **運用示唆**：ローカル昇格は「侵入後の権限拡大」を担う後段だが、初期侵入（フィッシング等）と組み合わせると完全掌握に直結する。BYOVD（脆弱ドライバ持ち込み）文脈でも悪用され得るため、ドライバ資産の可視化が重要。

## 📖 用語解説（このニュースとのつながり）

- **カーネルドライバ / ミニフィルタ**：OSの中枢（カーネル）で動く拡張。アンチチートは監視のため高権限で動く。今回はその通信ポートのアクセス制御と入力検証が甘かった。
- **write-what-where**：攻撃者が「任意の場所に任意の値」を書ける最上位級のメモリ破壊プリミティブ。CVE-2026-12168の本質で、SYSTEM昇格を確実にする。
- **セキュリティトークン**：プロセスの権限を表すカーネル内の構造体。書き換えられると権限昇格が成立する。
- **ローカル特権昇格（LPE）**：すでにそのPCで動ける低権限の攻撃者が、より高い権限（SYSTEM）を奪う攻撃。侵入後の“完全掌握”の鍵。
- **JVN / JPCERT/CC**：日本の脆弱性情報の公式窓口・調整機関。今回の注意喚起の出所で、国内利用者への周知経路。

## 影響範囲 / 推奨対応

GamersFirst Anti-Cheat（GFAC）を導入したWindows端末が対象。リモート単体では成立しないが、マルウェア感染後の権限昇格に直結するため軽視は禁物。修正版ドライバの提供状況を確認し、未提供の間は不要端末での該当ゲーム/ドライバのアンインストール、脆弱ドライバブロックの適用、EDRによるカーネル異常挙動監視で暫定対応する。業務PCへのゲーム由来カーネルドライバ混入を運用ルールで抑止する。
