# Vite 8.0 — Rust 製バンドラ Rolldown に一本化、ビルド最大10〜30倍高速化

> トピック: Vite 8.0（Rolldown 統合）｜出典: Vite 公式ブログ｜種別: ビルドツールのメジャー更新

## 一行サマリー

Vite 8.0（2026年3月12日リリース）は、これまで開発は esbuild・本番は Rollup と分かれていたバンドラを Rust 製の Rolldown 一本に統一し、プラグイン互換を保ったままビルドを最大10〜30倍高速化する、Vite 2 以来最大のアーキテクチャ変更。

## 🔰 初学者向け（何が・なぜ重要か）

Vite（ヴィート）は、いまのフロントエンド開発で最も広く使われている「開発の土台ツール」です。コードを保存した瞬間に画面へ反映させたり（開発時）、公開用に全部をぎゅっとまとめたり（本番ビルド）する役割を持ちます。

これまでの Vite には少し不思議な事情がありました。開発中は「esbuild」、本番用の組み立ては「Rollup」という、別々の道具を内部で使い分けていたのです。速さ重視の場面と最適化重視の場面で、性格の違う2台のエンジンを積み替えていたイメージです。これだと「開発では動いたのに本番ビルドだけ挙動が違う」といった食い違いが起きやすくなります。

Vite 8 では、この2台を **Rolldown（ロールダウン）** という Rust 製の1台のエンジンに統合しました。Rust は非常に速い言語なので、ビルドが桁違いに速くなります。しかも、これまで作られた大量の Vite プラグイン（拡張機能）はほぼそのまま使えるよう配慮されています。「速くなって、しかも開発と本番の食い違いも減る」――それが Vite 8 の嬉しさです。

## 🧠 専門的解説（技術詳細・設計・使いどころ）

### 技術的詳細（何が変わったか）

- 2026年3月12日に Vite 8.0 が正式リリース。**Rolldown（Rust 製バンドラ）を唯一の統一バンドラ**として採用。
- 従来は開発＝esbuild、本番＝Rollup の二本立て。8.0 でこれを Rolldown へ一本化。
- 公称ビルド高速化は最大 10〜30 倍。実例として Linear が本番ビルド 46 秒 → 6 秒（約87%減）、GitLab は Vite 7 比で 2.5 分 → 22 秒（元の webpack 比では約43倍）と報告。
- プラグイン互換性を維持する設計。移行は2段階を推奨：まず Vite 7 上で `vite` を `rolldown-vite` に差し替え、その後 Vite 8 へ上げる。
- 2025年12月の Vite 8 beta で Rolldown を統合済み。beta 期間中に Rolldown 自体が beta → RC へ進行。

### 仕組み / 背景（なぜこう設計されたか）

Vite が二本立てだった理由は歴史的なものだ。開発時は「起動の速さ」が命で、事前バンドルに esbuild（Go 製・超高速）を使う。一方、本番は tree-shaking やチャンク分割など「最適化の質」が命で、成熟した Rollup（JS 製）を使っていた。だがこの構成は、開発と本番で解決アルゴリズム・出力形態が微妙に異なり、環境差バグの温床になる。また Rollup が JS 製ゆえ大規模プロジェクトの本番ビルドが遅い、という限界もあった。

Rolldown は「Rollup 互換の API と挙動を、Rust ネイティブ実装で再現する」ことを目標に作られたバンドラだ。これにより (1) 開発と本番で同一エンジンを使えるため環境差が縮む、(2) Rust によるネイティブ実行と並列化でビルドが大幅高速化、という2つの課題を同時に解く。トレードオフは、Rollup プラグインの一部（特に内部 API に深く依存するもの）が完全互換にならない可能性と、まだ新しいゆえのエッジケース。だからこそ公式は「まず rolldown-vite で試し、差分を確認してから 8 へ」という慎重な移行パスを敷いている。

### 実務での使いどころ・移行の勘所

本番ビルドが遅い中〜大規模 SPA、モノレポ、CI 時間が課題のチームで効果が大きい。移行は「rolldown-vite へ差し替え → ビルド出力・チャンク分割・ソースマップの差分を検証 → 8 へ」の順で。特に確認すべきは、Rollup 前提のカスタムプラグイン、`manualChunks` などのチャンク戦略、そして CSS/アセット処理の細部。React/Vue/Svelte いずれのプラグインも公式対応が進むが、コミュニティプラグインは個別に互換状況を確認したい。Turbopack（Next.js 専用）と違い、Rolldown は Vite エコシステム全般（Nuxt, SvelteKit, Astro など）へ波及するため影響範囲は広い。

## 📖 用語解説（このトピックとのつながり）

- **Vite**: フロントエンド開発の土台ツール（dev サーバー＋ビルド）。本トピックの主役。
- **Rolldown**: Rust 製バンドラ。Rollup 互換を狙い、Vite 8 で開発・本番を一本化する中核。
- **esbuild / Rollup**: 従来 Vite が開発/本番で使い分けていた2つのバンドラ。8 で Rolldown に置き換え。
- **tree-shaking**: 使われないコードを削って出力を小さくする最適化。バンドラの品質を左右する。
- **チャンク分割（code splitting）**: 出力を複数ファイルに分け、必要な分だけ読み込ませる手法。移行時の差分確認ポイント。

## 影響範囲 / 推奨アクション

- 影響: Vite を使う広範なエコシステム（Nuxt/SvelteKit/Astro 含む）。ビルド時間・環境差バグ・CI コストに直結。
- 推奨: まず `rolldown-vite` で試験導入し、ビルド出力（チャンク・ソースマップ）の差分検証。コミュニティプラグインの互換確認後に Vite 8 へ。
