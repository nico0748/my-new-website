# Cisco Catalyst SD-WAN Manager：任意ファイル書き込みからroot奪取、ゼロデイ悪用を確認（CVE-2026-20262）

> トピック: 脆弱性ニュース ｜ 出典: BleepingComputer / Cisco PSIRT / SecurityAffairs ｜ 種別: 任意ファイル書き込み→権限昇格（root）

## 一行サマリー

企業の広域ネットワークを集中管理する「Cisco Catalyst SD-WAN Manager（旧vManage）」に、低権限の認証ユーザーが任意のファイルを書き込み、最終的にroot権限を奪える脆弱性（CVSS 6.5）が見つかった。Ciscoは実環境でのゼロデイ悪用を確認しており、CISA KEV登録済み（是正期限6/29）。

## 🔰 初学者向け（何が・なぜ問題か）

SD-WAN Managerは、会社の複数拠点をつなぐネットワーク全体を一つの画面から操る“司令塔”です。司令塔を乗っ取られると、社内ネットワークの設定・経路・通信を攻撃者が自由に操れてしまいます。

今回の弱点は「ファイルのアップロード受付」にありました。例えるなら、書類の提出窓口で“提出先のフォルダ名”を職員がチェックせず、利用者の言いなりに保存していたようなもの。攻撃者は本来置けないはずの場所（システムの重要な設定ファイルがある場所など）に、細工したファイルを忍び込ませられます。そのファイルが後でシステムに高い権限（root＝最高管理者）で読み込まれることで、攻撃者の命令がroot権限で動いてしまう――これが「ファイル書き込み→権限昇格」の流れです。

しかもこの欠陥は、修正が出る前から実際の攻撃に使われていた“ゼロデイ”でした。米国の政府機関には「期限内に必ず直せ」という命令（KEV登録）も出ています。

## 🧠 専門的解説（技術詳細・メカニズム・検知対応）

### 技術的詳細（種別・前提条件）

- **CVE**: CVE-2026-20262
- **CVSS**: 6.5（ただし実環境でのゼロデイ悪用とroot到達を踏まえ、運用上の優先度はHigh相当として扱う）
- **脆弱性クラス**: Web UIのファイルアップロード処理における**ユーザー入力の検証不備（任意ファイル書き込み / Arbitrary File Write）**。最終的にroot権限昇格へ繋がる。
- **前提条件**: **少なくとも書き込み権限を持つ有効な認証情報**が必要（＝認証済みの低～中権限ユーザー）。Web UIに到達できるネットワーク位置。
- **影響対象**: Cisco Catalyst SD-WAN Manager（vManage）。デプロイ形態は on-prem / Cloud-Pro / Cisco Managed Cloud / FedRAMP を含む。
- **修正版（例）**: 20.9.9.1以前→20.9.9.2、20.12.7.1以前→20.12.7.2、20.15.4.4以前→20.15.4.5、20.15.5.2以前→20.15.5.3、20.18.3→20.18.3.1、26.1.1.1以前→26.1.1.2。
- **悪用状況**: Cisco PSIRTが**限定的な実環境悪用（in-the-wild / ゼロデイ）**を確認。CISA KEV登録済み（是正期限 2026-06-29）。

### メカニズム（攻撃が成立する仕組み）

※脆弱性クラスの一般原理を概念レベルで説明する。具体的な書き込みパス・PoC等は記載しない。

1. **信頼境界とアップロードの責務**: SD-WAN ManagerのWeb UIは、認証済みユーザーからのファイルアップロードを受け付ける。ここでの設計要件は「保存先パスとファイル種別をサーバ側が厳格に制約する」こと。本件はこの制約（パス検証・正規化・許可リスト）が不十分だった。
2. **任意ファイル書き込みプリミティブの獲得**: 攻撃者は、アップロード時の保存先指定（パス・ファイル名）を細工した不正なHTTPリクエストを送る。検証不備により、本来書き込めないシステム領域へファイルを設置できる＝「任意の場所に任意の内容を書き込む」プリミティブを握る。
3. **書き込み→実行への橋渡し**: 単なる書き込みが特権実行に化けるのは、「そのファイルが後でroot権限のプロセスに読み込まれる」設計箇所を狙うため。例として、(a) 高権限デーモンが読む設定ファイル/スクリプトを上書きする、(b) 定期実行（cron相当）やサービス起動時に実行されるパスへファイルを置く、(c) 既存の正規ファイルを改ざんしてロジックを乗っ取る、といったクラスが知られる。
4. **root権限昇格の成立**: 上記いずれかにより、攻撃者が用意した処理がrootコンテキストで実行され、システム全体の制御を奪取。
5. **被害の最終形**: SD-WAN管理基盤はネットワーク全体の構成・経路・証明書を握るため、root奪取は拠点間通信の傍受・改ざん・横展開へ直結する。

クラスとしては「アクセス制御＋入力検証の欠落（任意ファイル書き込み）が、特権実行パスと結合して権限昇格に至る」典型例である。CVSSの数値（6.5）は前提条件（要認証）を反映するが、ゼロデイ悪用とroot到達という結果の重さを軽視してはならない。

### 検知・暫定緩和・恒久対策

- **恒久対策**: 上記の固定リリースへ即時アップデート（KEV是正期限は既に到来。未対応組織は最優先で対応）。
- **暫定緩和（パッチ前）**: (1) Web UI / 管理インターフェースへのアクセスを**管理セグメント・許可IPに限定**、(2) 管理者・運用アカウントの権限最小化と棚卸し、(3) 不要な低権限アカウントの無効化、(4) 管理面へのMFA徹底。
- **検知の着眼点**: 想定外パスへのファイル生成・更新、Web UIプロセスによる高権限領域への書き込み、アップロード後に発生する新規プロセスやサービス再起動、認証済み低権限ユーザーによる管理API（ファイルアップロード系エンドポイント）への異常アクセス、設定/スクリプトファイルのハッシュ変化。

## 📖 用語解説（このニュースとのつながり）

- **SD-WAN / vManage**: 複数拠点のWANをソフトウェアで集中管理する仕組みと、その管理コンソール。乗っ取りの影響がネットワーク全体に及ぶ理由。
- **任意ファイル書き込み（Arbitrary File Write）**: 攻撃者が望む場所に望む内容を書ける状態。本件の中核プリミティブ。
- **権限昇格（Privilege Escalation）**: 低い権限から高い権限（root）へ昇格すること。書き込みが実行に化ける橋渡しが鍵。
- **ゼロデイ**: 修正が存在しない段階で悪用される脆弱性。本件はパッチ前に実攻撃で使われた。
- **CISA KEV**: 米CISAが「実際に悪用されている」と認定した脆弱性カタログ。連邦機関に期限付き是正を課す。民間でも優先対応の指標になる。

## 影響範囲 / 推奨対応

Cisco Catalyst SD-WAN Manager（vManage）の各デプロイ形態が対象。ゼロデイ悪用が確認されKEV登録済みのため、該当バージョンは直ちに固定リリースへ更新する。更新前は管理インターフェースへの到達を厳格に制限し、低権限アカウントの棚卸しと管理面の監視を強化する。
