# .NET 10 が耐量子暗号を標準搭載 — ML-KEM/ML-DSA を BCL から直接使える

> トピック: .NET 10（LTS）の耐量子暗号サポート｜出典: Microsoft Learn / .NET Blog｜種別: ランタイム/標準ライブラリの機能

## 一行サマリー

.NET 10（LTS）は `System.Security.Cryptography` に NIST 標準化済みの耐量子（post-quantum）アルゴリズム ML-KEM（FIPS 203）・ML-DSA（FIPS 204）・SLH-DSA（FIPS 205）の型を追加し、将来の量子コンピュータによる暗号解読に備えた鍵交換・署名を標準ライブラリから直接扱えるようにした。3月の更新を経て 2026年に本格運用フェーズへ入っている。

## 🔰 初学者向け（何が・なぜ重要か）

いま Web の通信や署名を守っている暗号（RSA や楕円曲線暗号）は、「巨大な数の計算がとても難しい」ことに安全性を頼っている。ところが十分に強力な量子コンピュータが実現すると、この難しさが崩れ、既存暗号が破られてしまうと考えられている。まだ実機は来ていないが、「今の暗号で守った通信を今のうちに記録しておき、将来の量子コンピュータで後からまとめて解読する（Harvest Now, Decrypt Later＝今収穫して後で復号）」という脅威が現実的に語られている。

そこで、量子コンピュータでも破りにくい新しい暗号「耐量子暗号（PQC）」への移行が世界的に進んでいる。.NET 10 は、この新暗号を「標準ライブラリに最初から入っている道具」として提供した。開発者は外部ライブラリを探し回らず、標準の暗号 API の延長で新方式の鍵交換や署名を書ける。地味だが、数年がかりの移行の土台を敷く重要な一歩だ。

## 🧠 専門的解説（技術詳細・設計・使いどころ）

### 技術的詳細（何が変わったか）

Microsoft Learn の .NET 10 概要によれば、`System.Security.Cryptography` に3つの非対称 PQC アルゴリズムの型が追加された：鍵カプセル化の **ML-KEM（FIPS 203、旧 Kyber 系）**、格子ベース署名の **ML-DSA（FIPS 204、旧 Dilithium 系）**、ハッシュベース署名の **SLH-DSA（FIPS 205、旧 SPHINCS+ 系）**。.NET 10 は LTS で 2028年11月10日までサポート（2025年11月リリース、2026年3月に累積更新）。ランタイム側は JIT のインライン化・脱仮想化・スタック割り当て改善、AVX10.2 対応、NativeAOT 強化なども入る。なお PQC 型の対応範囲や OS のネイティブ暗号への依存関係はプラットフォーム差があり、正確な API 形状・対応状況は公式ドキュメントで要確認。

### 仕組み / 背景（なぜこう設計されたか）

現行の公開鍵暗号（RSA・ECDH/ECDSA）は素因数分解・離散対数の困難性に依存し、ショアのアルゴリズムを実行できる量子計算機によって理論上破れる。NIST は複数年の公募・評価を経て 2024 年に PQC を標準化し、格子問題やハッシュ関数の安全性に基づく方式を選定した。ML-KEM は「共有鍵を安全に配送する」鍵カプセル化、ML-DSA/SLH-DSA は「なりすまし防止の署名」を担う。.NET が標準 BCL に組み込んだ狙いは、PQC を「特別なライブラリ」ではなく既存の暗号 API と同じ抽象で扱えるようにし、TLS スタックや署名処理へ段階的に織り込めるようにすること。トレードオフとして、PQC は鍵・署名サイズが従来より大きく（帯域・ストレージ増）、成熟度も従来暗号ほど枯れていないため、当面は「従来暗号と PQC を併用するハイブリッド」構成で移行するのが定石になる。

### 実務での使いどころ・移行の勘所

長期間秘匿すべきデータ（医療・金融・国防・長寿命な署名）を扱うシステムほど「今から」PQC への布石を打つ意味が大きい。移行の勘所は、いきなり全面置換ではなく、鍵交換はハイブリッド（従来 + PQC）から始めること、鍵・署名サイズ増によるプロトコル/ストレージへの影響を見積もること、そして相互運用のため対向システムやライブラリの PQC 対応状況を確認すること。.NET 10 の型はあくまで基盤であり、TLS 全体の PQC 化は OS・リバースプロキシ・証明書エコシステムの対応と足並みを揃える必要がある。まずは暗号アジリティ（アルゴリズムを差し替えやすい設計）を整えておくのが現実的な準備になる。

## 📖 用語解説（このトピックとのつながり）

- **耐量子暗号（PQC）**: 量子コンピュータでも破りにくい暗号方式。.NET 10 が ML-KEM/ML-DSA/SLH-DSA を標準化。
- **ML-KEM / ML-DSA / SLH-DSA**: それぞれ NIST 標準の鍵カプセル化・格子署名・ハッシュ署名。旧称は Kyber/Dilithium/SPHINCS+。
- **Harvest Now, Decrypt Later**: 現在の暗号通信を記録し将来解読する脅威モデル。PQC 前倒し移行の動機。
- **ハイブリッド暗号 / 暗号アジリティ**: 従来暗号と PQC を併用する構成／アルゴリズムを差し替えやすい設計。移行戦略の要。

## 影響範囲 / 推奨アクション

- 長期秘匿データを扱うシステムは、PQC 移行の設計（ハイブリッド・暗号アジリティ）を今から始める価値が高い。
- .NET 10 の PQC 型は基盤。TLS 全体の PQC 化は OS・プロキシ・証明書エコシステムの対応と足並みを揃える。
- 鍵/署名サイズ増の帯域・ストレージ影響を見積もり、対向システムの相互運用を検証する。
