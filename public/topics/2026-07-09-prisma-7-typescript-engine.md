# Prisma 7 — Rust クエリエンジンを撤廃し純 TypeScript/WASM 化、バンドル85〜90%削減

> トピック: Prisma 7（クエリエンジンの TypeScript 化）｜出典: Prisma 公式ドキュメント｜種別: ORM のメジャー・アーキテクチャ更新

## 一行サマリー

TypeScript 向け ORM の Prisma がバージョン 7 で、長年同梱してきた Rust 製クエリエンジンを撤廃し、クライアントを純 TypeScript/WASM 実装へ刷新した。エンジンは約1.6MB（gzip 600KB）へと85〜90%小さくなり、大きな結果セットで最大3.4倍高速、サーバーレスのコールドスタートは最大9倍改善したとされる。

## 🔰 初学者向け（何が・なぜ重要か）

ORM（オーアールエム）は、データベースを「SQL を直接書かずに、プログラムの言葉で操作できるようにする道具」です。Prisma はその代表格で、型（データの形）を厳密にチェックしてくれる安心感から広く使われています。

これまでの Prisma には隠れた事情がありました。中身に「Rust という別言語で書かれた部品（クエリエンジン）」を同梱しており、これがファイルサイズを膨らませ、特にサーバーレス環境（使うたびに一瞬で起動する仕組み）で「起動が遅い」原因になっていたのです。Prisma 7 では、この Rust 部品をやめ、全部を TypeScript/WASM で作り直しました。結果、部品はぐっと小さくなり、起動も速くなりました。利用者にとっては「同じ書き味のまま、軽く・速くなった」大型アップデートです。

## 🧠 専門的解説（技術詳細・設計・使いどころ）

### 技術的詳細（何が変わったか）

- Prisma 7 で **Rust クエリエンジンを撤廃**、クライアント全体を純 TypeScript／WASM 実装へ移行（プロジェクト史上最大のアーキテクチャ変更）。
- 新エンジンは約 1.6MB（gzip 600KB）で、従来比 **85〜90% のサイズ削減**。
- 大きな結果セットで最大 **3.4倍高速**、サーバーレスのコールドスタートが最大 **9倍改善**との報告。
- ネイティブバイナリの同梱・配布が不要になり、デプロイ構成（Lambda 等のレイヤ、Docker イメージサイズ）が簡素化。
- スキーマファースト（`schema.prisma`）の設計思想と型生成の基本 API は維持。数値は一次情報での再確認を推奨（要確認）。

### 仕組み / 背景（なぜこう設計されたか）

Prisma が Rust エンジンを採る設計は、当初は合理的だった。クエリのパース・プランニング・結果整形といった重い処理をネイティブコードで高速に行い、JS ランタイムの負荷を抑える狙いだ。しかしこの構成には副作用があった。(1) プラットフォームごとのバイナリ（Linux/macOS、glibc/musl、arm/x64）を同梱するためバンドルが肥大、(2) 起動時にネイティブプロセスを立ち上げるオーバーヘッドがサーバーレスのコールドスタートを直撃、(3) エッジ（Cloudflare Workers 等）ではネイティブバイナリが動かせず対応が困難、という三重苦だ。

純 TypeScript/WASM 化は、これらを一挙に解く。JS/WASM は各ランタイムに単一成果物で載るためプラットフォーム分岐が消え、プロセス起動が不要になりコールドスタートが縮む。エッジでも同じコードが動く。トレードオフは、重い処理を JS/WASM で行うため理論ピーク性能ではネイティブに劣り得る点だが、実測では JS エンジンの最適化と余計なプロセス間通信の除去が効き、多くのワークロードでむしろ高速化したという。これは「移植性とコールドスタートを、ピーク性能より優先する」というサーバーレス時代の設計判断であり、競合 Drizzle が最初から SQL 近接・軽量路線を採ったこととも符合する。

### 実務での使いどころ・移行の勘所

サーバーレス／エッジ主体の構成、コールドスタートや Lambda のパッケージサイズが課題だったチームで恩恵が大きい。既存 Prisma プロジェクトのアップグレードは、まず開発環境で生成クライアントの差分・型の互換・生 SQL（`$queryRaw`）挙動を検証し、CI でクエリ結果の回帰を確認する。ネイティブバイナリ前提のデプロイ手順（バイナリターゲット指定、Docker のマルチステージ）は簡素化できるので、あわせて見直すとよい。モノリシックで複雑なリレーションを持つ用途では引き続き Prisma が強く、サーバーレスの軽量用途では Drizzle との比較検討も現実的。数値効果は自プロジェクトで実測して判断する。

## 📖 用語解説（このトピックとのつながり）

- **ORM**: SQL を書かずに DB を型付きで操作する道具。Prisma はその代表。本トピックの主役。
- **クエリエンジン**: クエリの解析・実行・整形を担う中核。Prisma 7 で Rust から TS/WASM へ。
- **コールドスタート**: サーバーレスで初回起動にかかる遅延。今回の主要改善点。
- **WASM（WebAssembly）**: 各ランタイムで動く移植可能なバイナリ形式。エッジ対応の鍵。
- **スキーマファースト**: `schema.prisma` を単一の真実にする設計。7 でも維持。

## 影響範囲 / 推奨アクション

- 影響: Prisma 利用の全バックエンド、特にサーバーレス／エッジ構成。
- 推奨: 開発環境で型・生 SQL・クエリ回帰を検証 → デプロイ手順（バイナリ/Docker）を簡素化。数値効果は自環境で実測。
