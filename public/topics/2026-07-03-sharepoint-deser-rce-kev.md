# Microsoft SharePoint Server の RCE(CVE-2026-45659)が実環境で悪用、CISA KEV 登録

> トピック: 脆弱性ニュース ｜ 出典: The Hacker News / CISA KEV ｜ 種別: 信頼できないデータのデシリアライズ(CWE-502) による RCE

## 一行サマリー

Microsoft SharePoint Server の RCE 脆弱性 CVE-2026-45659(CVSS 8.8、信頼できないデータのデシリアライズ)が実環境で悪用され、CISA が KEV カタログへ登録、米連邦機関に 2026年7月4日までの適用を義務付けた。サイトメンバー相当の低権限で認証済みなら遠隔コード実行に至りうる。

## 🔰 初学者向け（何が・なぜ問題か）

SharePoint は「社内の共有フォルダ＋掲示板＋書類倉庫」を Web でまとめたようなサーバー製品で、多くの企業がイントラの中心に据えています。今回の問題は、そのサーバーが「送られてきたデータの箱」を中身を確認せずに開けてしまうところにあります。

コンピュータの世界では、プログラムの状態(オブジェクト)を保存・送信できる形に変換することを「シリアライズ」、それを元に戻すことを「デシリアライズ(復元)」と呼びます。宅配便に例えるなら、荷物を箱詰めして送り、受け取り側が箱を開けて中身を組み立て直すイメージです。ここで受け取り側が「差出人不明の箱でも中身を疑わずに、書いてある通りに組み立てて動かす」設計になっていると、攻撃者は箱の中に「組み立てた瞬間に悪意あるコマンドを実行する仕掛け」を仕込めます。

CVE-2026-45659 では、SharePoint に**普通のメンバー権限でログインできる程度の相手**(管理者でなくてよい)が、細工したデータを送り込むだけでサーバー上で任意のプログラムを動かせてしまいます。イントラの中心を乗っ取られると、社内文書の窃取・改ざん、他サーバーへの横移動の踏み台化につながります。すでに「実際に攻撃で使われている」ことが確認され、米政府が緊急で「直せ」と通達を出した、という点が今回の重さです。

## 🧠 専門的解説（技術詳細・メカニズム・検知対応）

### 技術的詳細（種別・前提条件）

- **CVE**: CVE-2026-45659 / **CVSS**: 8.8(要確認: 提供元により基準スコア表記が異なる場合あり)
- **種別(CWE)**: CWE-502 Deserialization of Untrusted Data(信頼できないデータのデシリアライズ)
- **前提条件**: ネットワーク到達性＋**認証済みユーザー**。報道では「最低でもサイトメンバー権限」で成立するとされ、管理者・昇格権限は不要。ユーザー操作は不要。
- **影響製品**: SharePoint Server Subscription Edition / SharePoint Server 2019 / SharePoint Enterprise Server 2016。
- **修正**: Microsoft が 2026年5月に対処済み(該当月の累積更新)。
- **悪用状況**: 実環境での悪用を CISA が確認し KEV 登録。FCEB(米連邦民間行政機関)への期限は 2026年7月4日。攻撃主体・最終目的は本記事時点で公表されておらず**要確認**。

### メカニズム（攻撃が成立する仕組み）

デシリアライズ RCE の一般原理と、本件で報じられている性質から、成立の流れを概念レベルで分解する(武器化手順・PoC・具体的ガジェットチェーンは記載しない)。

1. **信頼境界の誤り**: SharePoint の一部エンドポイントは、クライアント側から渡されるオブジェクトを復元(デシリアライズ)して処理する。ここで「認証済みユーザーが送るデータ＝信頼できる」と暗黙に扱う設計が信頼境界の取り違えになる。実際には認証済みでも入力の中身は攻撃者制御下にある。
2. **攻撃者が握るプリミティブ**: 攻撃者は「復元されるオブジェクトの型と中身」を制御できる。.NET 系のデシリアライズでは、復元時にプロパティのセッターやコールバックが自動的に走る「型」が存在し、これらを数珠つなぎにすると、復元という一見無害な操作の副作用として任意メソッド呼び出しへ橋渡しできる(いわゆるガジェットチェーンの一般原理)。
3. **検証の欠落**: 復元対象の型を許可リストで制限する仕組み(型バインダ/シリアライズフィルタ)が無い、または回避可能だと、想定外の危険な型が復元されてしまう。
4. **最終被害への橋渡し**: 復元の副作用がプロセス権限でのコマンド実行(RCE)に到達する。SharePoint はサービスアカウントで動くため、成功すればサーバー権限の掌握、ひいては認証情報の窃取・他資産への横移動につながる。
5. **クラスとしての位置づけ**: 本件は「アクセス制御の穴」ではなく「復元処理が攻撃者データを実行可能な副作用に変換してしまう」デシリアライズ・クラスの典型。認証はあくまで入口の敷居を下げる要素であり、根本原因は入力型の無制限な復元にある。

> 注: 上記は CWE-502 型の一般原理に基づく概念説明であり、CVE-2026-45659 固有の内部実装(具体的な脆弱エンドポイント・ガジェット)は公表情報が限られるため断定を避ける。

### 検知・暫定緩和・恒久対策

- **検知の着眼点**: SharePoint の IIS/ULS ログで、認証済みだが低権限のアカウントからの異常な POST、`w3wp.exe` からの想定外の子プロセス生成(cmd/powershell 等)、短時間での多数の逆シリアライズ関連エラー。EDR で SharePoint サービスアカウント配下の異常なプロセスツリーを監視。
- **暫定緩和(パッチ前)**: 外部公開している SharePoint はまず境界(WAF/リバースプロキシ)でアクセス元を制限。不要な匿名/ゲスト共有を停止し、サイトメンバー権限の棚卸しを行い、不審アカウントを無効化。
- **恒久対策**: Microsoft の 2026年5月更新(および以降の累積更新)を適用。適用済みかを版管理で確認し、KEV 期限(7/4)を運用の優先度基準に据える。
- **運用示唆**: 「認証済み＝安全」という前提を捨てる。デシリアライズを扱う内製コードでは型の許可リスト化(AllowedTypes/シリアライズフィルタ)を徹底する。

## 📖 用語解説（このニュースとのつながり）

- **デシリアライズ(逆シリアライズ)**: 保存・送信用に変換したデータを、プログラムが使えるオブジェクトへ復元する処理。今回はこの復元が攻撃の起点。
- **ガジェットチェーン**: 復元時に自動実行される既存クラスのメソッドを連鎖させ、任意コード実行へ導く手法の総称。本件 RCE の橋渡し原理。
- **CWE-502**: 「信頼できないデータのデシリアライズ」を指す脆弱性分類。今回の根本原因のカテゴリ。
- **CISA KEV**: 米 CISA が「実際に悪用が確認された脆弱性」をまとめた目録。掲載＝優先対応すべき、の強いシグナル。本件は 7/2 に追加。
- **FCEB 期限**: 米連邦民間機関に課される修正期限。本件は 2026/7/4。民間でも優先度判断の目安になる。

## 影響範囲 / 推奨対応

オンプレの SharePoint Server(Subscription Edition / 2019 / 2016)を運用する組織が対象。イントラの中核であるほど被害が拡大しやすい。対応は、2026年5月以降の累積更新の適用確認を最優先とし、外部公開インスタンスはアクセス制限とアカウント棚卸しを即時実施。適用状況とログ監視の結果を記録し、KEV 期限を社内 SLA に反映する。
