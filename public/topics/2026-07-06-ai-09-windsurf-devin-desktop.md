# Windsurf が Devin Desktop に統合 — AIコーディング勢力図の再編

> トピック: Windsurf → Devin Desktop｜出典: The New Stack / 各種報道｜種別: 製品統合・リブランド（2026-06-02）

## 一行サマリー

2026年6月2日、Devin を手がける Cognition が AI コーディングエディタ Windsurf を「Devin Desktop」へ統合・リブランド。旧無料 Windsurf エディタは Devin Free（$0、Tab 補完・インライン編集が無制限）に、Devin Pro は $20/月で全モデルとクラウドエージェントを提供。IDE型のローカル支援と自律クラウドエージェントを一つの製品線に束ねる動き。

## 🔰 初学者向け（何が・なぜ重要か）

AI でコードを書く道具は、大きく2種類あります。ひとつは「エディタに寄り添って補完・編集を手伝う」タイプ（Windsurf や Cursor など）。もうひとつは「タスクを丸ごと任せると自律的に実装してくる」クラウド型エージェント（Devin など）。これまで別物だったこの2つが、今回1つの製品線にまとまりました。

具体的には、Devin を作る Cognition という会社が、人気の AI エディタ Windsurf を取り込み「Devin Desktop」という名前に統合。無料版はそのまま Devin Free として使え、月20ドルの Pro にすると強いモデルや、クラウドで自律的に働くエージェントまで使える——という形に整理されました。

なぜ重要かというと、AI コーディングの世界が「手元で手伝う道具」と「勝手にやってくれる道具」に分かれていたのが、境界を越えて1つのスタックに融合し始めているからです。開発者にとっては「小さな編集は手元で、大きな作業はクラウドエージェントに」を同じ製品でシームレスに行き来できる未来が近づいています。ツールの離合集散が激しい分野なので、どれに乗るかの見極めも大切になります。

## 🧠 専門的解説（技術詳細・設計・使いどころ）

### 技術的詳細（何が変わったか）

- **統合・リブランド（2026-06-02）**: Cognition（Devin の開発元）が Windsurf を Devin Desktop へ折り込み。
- **プラン再編**: 旧無料 Windsurf エディタ = Devin Free（$0、Tab 補完・インライン編集が無制限）。Devin Pro = $20/月（全モデル利用＋Devin Cloud エージェント）。
- **業界文脈**: Cursor は同時期に Teams 席を再編（Standard $32/席・月〈年払い〉、Premium $96/席・月〈Standard の5倍利用〉）。OpenAI は 6/26 に GPT-5.6 を発表し Codex へ展開。Claude Code は Claude Sonnet 5（6/30）を搭載。
- 「Cursor / Claude Code / Codex / Antigravity が一つの AI コーディングスタックへ収斂しつつある」という観測が広がる。

### 仕組み / 背景（なぜこう設計されたか）

AI コーディング製品は、(A) IDE 内で低レイテンシに補完・編集を返す「対話的支援」と、(B) タスクを受けてバックグラウンドで多段実装する「自律エージェント」という、要件の異なる2モードを持つ。前者は即応性とローカル文脈（開いているファイル、カーソル位置）が命で、後者は long-horizon の計画・実行・検証と、隔離環境（サンドボックス）での安全な実行が命だ。両者を別製品で持つと、開発者は「小さな修正は Cursor 的ツール、大きな作業は Devin」と使い分ける必要があり、文脈の分断が生じる。Devin Desktop はこの2モードを1つのクライアントに束ね、「手元の編集」と「クラウドへ委譲する自律作業」を同じUI・同じ文脈で往復できるようにする狙いだ。ビジネス的には、無料エディタで裾野を広げ、Pro でクラウドエージェント（計算コストの高い自律実行）を収益化する典型的なフリーミアム設計。トレードオフは、ローカル対話とクラウド自律という性質の違うUXを一貫させる難しさと、急速な統廃合による製品継続性リスク（乗り換えコスト、ロックイン）だ。

### 実務での使いどころ・移行の勘所

チーム導入では、「日々の補完・小修正はローカル（Free/低コスト帯）、大きめの自律作業はクラウドエージェント（Pro 帯）」という役割分担で費用対効果を測るのが実務的。ただしこの分野はツールの統廃合・価格改定が頻繁なので、(1) 生成コードのレビュー/CI ゲートを前提に据える、(2) 特定製品へのロックインを避けるため設定・プロンプト資産を移植可能に保つ、(3) クラウドエージェントに与える権限（リポジトリ、シークレット、ネットワーク）を最小化する、が要点。価格・プラン内容は流動的なので、契約前に一次情報で最新を確認すること。

## 📖 用語解説（このトピックとのつながり）

- **IDE型 AI 支援**: エディタ内で補完・インライン編集を即応で返すタイプ。旧 Windsurf 系。
- **自律クラウドエージェント**: タスクを受け隔離環境で多段実装する自律型。Devin 系。
- **フリーミアム**: 無料で裾野を広げ、有料で高コスト機能を収益化するモデル。Free/Pro の設計。
- **ロックイン**: 特定製品への依存で乗り換えが困難になる状態。統廃合が激しい分野での注意点。

## 影響範囲 / 推奨アクション

- 対象: AI コーディングツールを選定・運用する開発チーム。
- アクション: ローカル補完とクラウド自律作業を役割分担して費用対効果を測る。生成コードは必ずレビュー/CI を通す。設定・プロンプト資産は移植可能に保ちロックインを回避。エージェント権限は最小化。価格/プランは一次情報で最新確認。
