# CSS の lh 単位が Baseline 入り — 行の高さに合わせた寸法指定が標準に

> トピック: CSS `lh` 相対単位が Baseline Newly available｜出典: web.dev（2026年5月ダイジェスト）｜種別: Web 標準／ブラウザ相互運用

## 一行サマリー

2026年5月、要素の計算後 line-height を表す CSS の相対単位 `lh` が全主要ブラウザで利用可能（Baseline Newly available）になり、アイコンバッジや背景ハイライトを「テキスト1行の高さ」にぴったり合わせる指定が、計算やマジックナンバーなしで書けるようになった。

## 🔰 初学者向け（何が・なぜ重要か）

Web ページで「文字の高さにぴったり合った丸バッジ」や「1行分だけ光る背景」を作りたいとき、これまでは行の高さをピクセルで手打ちしたり、JavaScript で計算したりしていました。文字サイズや行間を変えるとその数字も直さなければならず、地味に面倒でした。

`lh` は「いまのこの要素の1行の高さ」をそのまま単位として使える仕組みです。たとえるなら、定規を持ち歩かなくても「この行の背丈ぶん」と指さすだけで寸法を指定できるようなもの。`height: 1lh` と書けば「ちょうど1行分の高さ」になり、フォントや行間を変えても自動で追従します。

これが「Baseline 入り」したというのは、Chrome・Safari・Firefox といった主要ブラウザすべてで安心して使えるようになった、という意味です。つまり「一部のブラウザで動かない」を気にせず本番で使える段階になりました。

## 🧠 専門的解説（技術詳細・設計・使いどころ）

### 技術的詳細（何が変わったか）

`lh` は「その要素の計算済み line-height」を長さとして参照する相対単位。ルート要素の line-height を参照する `rlh` と対になる。2026年5月、Safari の対応追加により全主要エンジンで実装が揃い、web.dev の Baseline 判定で "Newly available" に到達した。

用途例:
- `height: 1lh` / `min-height: 1lh` で「1行ぶんの高さ」の箱を作る
- インライン装飾（ハイライト、下線帯）を行高に一致させる
- アイコンとテキストの縦位置を、行ボックスに対して厳密に揃える

同じ5月のダイジェストには、`:open` 擬似クラスや `@container` の `style()` クエリなど「状態・文脈を CSS で扱う」機能群も並ぶ。

### 仕組み / 背景（なぜこう設計されたか）

従来、行の高さに寸法を合わせるには `em` を使った近似（`line-height` の値を別途掛ける）か、固定 px か、JS 計測しかなかった。`em` はフォントサイズ基準であって line-height 基準ではないため、`line-height` を変えるとズレる。`lh` は「line-height そのもの」を一次単位に昇格させることで、この不一致を根本から解消する。

設計上の要点は、`lh` が「計算後の line-height」を参照する点。`line-height: normal` のようにフォント依存で確定する値でも、レイアウト時に確定した実寸を返すため、可視結果と一致する。トレードオフは循環参照の回避で、`line-height` 自体を `lh` で指定するような自己参照は避ける必要がある。全体として、CSS が「ハードコードされた数値」から「意味のある相対量」へ移行する 2026 年の潮流（`lh`, コンテナ相対、論理プロパティ等）の一部と位置づけられる。

### 実務での使いどころ・移行の勘所

デザインシステムのコンポーネント（バッジ、チップ、インラインアイコン、コードハイライト）で効果が大きい。フォントスケールや密度（compact/comfortable）を切り替える UI では、`lh` に寄せておくと行高変更に自動追従し、メンテが激減する。移行時の注意は、古いブラウザをサポート対象に含む場合のフォールバック（`height: 1.4em` などの近似値を先に書く）。Baseline Newly available は「最新版で使える」段階であり、企業のレガシー端末を含む Baseline Widely available（十分に普及）まで待つべきかは要件次第で判断する。

## 📖 用語解説（このトピックとのつながり）

- **lh 単位**: 要素の計算後 line-height を表す相対長さ単位。本トピックの主役。
- **rlh 単位**: ルート要素の line-height を参照する版。`lh` と対。
- **Baseline**: 主要ブラウザでの利用可否を示す web.dev の指標。"Newly available"（新たに全対応）と "Widely available"（普及済み）がある。
- **line-height**: 行ボックスの高さを決める CSS プロパティ。`lh` はこの計算値を参照する。

## 影響範囲 / 推奨アクション

インライン装飾やバッジ寸法を持つデザインシステムは `lh` 採用で保守性が上がる。レガシー端末対応が必要な場合は近似値フォールバックを併記。まずは非クリティカルな装飾から段階導入を推奨。
