# CSS @scope が Baseline 2026 入り — プリプロセッサなしでスタイルの適用範囲を封じ込め

> トピック: CSS `@scope` が Baseline 2026｜出典: web-standards.dev / MDN｜種別: CSS 機能の相互運用達成

## 一行サマリー

スタイルの適用範囲を「ここからここまで」と DOM のサブツリー単位で限定できる CSS `@scope` が、主要ブラウザで足並みが揃い Baseline 2026 に到達した。ネイティブ CSS ネストと合わせ、小規模プロジェクトで Sass などのプリプロセッサを残す理由がまた一つ消えた。

## 🔰 初学者向け（何が・なぜ重要か）

CSS の悩みの定番が「スタイルが意図せず他の場所まで効いてしまう」ことです。たとえば「カード内のリンクだけ色を変えたい」のに、書き方を誤ると別のリンクまで巻き込んでしまう。これを避けるため、これまでは長いクラス名を付けたり、CSS Modules や Sass のような道具で「見えない壁」を作ってきました。

`@scope`（スコープ）は、この「壁」を CSS 標準の機能で作れるようにするものです。「このまとまりの中だけ、このスタイルを適用する」と範囲を宣言でき、しかも「ここから先は対象外」という下限も指定できます。今回のニュースは、この機能が全主要ブラウザで使えるようになり、安心して本番投入できる段階（Baseline）になった、という話です。追加ツールなしで、素の CSS だけでスタイルの漏れを防げるようになりました。

## 🧠 専門的解説（技術詳細・設計・使いどころ）

### 技術的詳細（何が変わったか）

- `@scope` が主要ブラウザで相互運用に達し Baseline 2026 として整理された。合わせてネイティブ CSS ネストも Baseline に定着。
- 構文は `@scope (<scope-root>) to (<scope-limit>) { ... }`。ルート選択子で適用開始点を、`to` で下限（この境界より内側は対象外）を指定できる。
- インライン利用も可能: `<style>@scope { ... }</style>` を DOM のあるサブツリー内に置くと、その親要素をルートとしてスコープが張られる。
- スコープ内では `:scope` でルート自身を参照できる。詳細度（specificity）はスコープのルート選択子分が加算される点に注意。
- 近接性（proximity）ルール: 同一詳細度なら、スコープルートに DOM 上で近い宣言が優先される。

### 仕組み / 背景（なぜこう設計されたか）

CSS のセレクタは本来グローバルで、「どこまで効かせるか」の下限を素直に書く手段がなかった。子孫結合子（空白）は「子孫すべて」に効くため、ネストしたコンポーネント（例: カードの中にさらにカード）でスタイルが染み出す「テーマ漏れ」が起きやすい。`@scope` は上限（ルート）だけでなく**下限（limit）**を持つことで、この染み出しを構文レベルで断ち切る。これは CSS Modules や BEM が命名規約で人力にやっていた「境界付け」を、言語機能として正式化したものだ。

トレードオフは詳細度と近接性の理解コスト。`@scope` はルート選択子分の詳細度を足すため、`!important` の乱用や詳細度戦争を避ける設計が要る。また近接性ルールは「後勝ち」ではなく「近い方が勝つ」ため、従来のカスケード直感と異なる場面がある。とはいえ、レイヤー（`@layer`）やネストと組み合わせれば、プリプロセッサに頼らず宣言的にスタイルの境界とカスケードを制御できるという一般的な方向性は明確だ。

### 実務での使いどころ・移行の勘所

デザインシステムのコンポーネント、記事本文（`.prose` 的な領域）内の限定スタイル、テーマの入れ子（ダーク領域の中の一部だけライト）などに有効。CSS Modules や styled-components の「スコープ隔離」目的の一部は `@scope` で代替できるが、コンポーネント単位のビルド時ハッシュ隔離が要る大規模構成では併用が現実的。移行はまず「テーマ漏れが起きていた局所」から。詳細度・近接性の挙動をチームで共有し、`@layer` と組み合わせて優先順位を明示的に設計するのが安全。

## 📖 用語解説（このトピックとのつながり）

- **@scope**: スタイルの適用範囲を DOM サブツリー単位で限定する CSS 機能。本トピックの主役。
- **scope-limit（to 句）**: 「ここから先は対象外」という下限指定。テーマ漏れ防止の要。
- **:scope**: スコープのルート要素自身を指す擬似クラス。
- **詳細度 / 近接性（specificity / proximity）**: どの宣言が勝つかを決める規則。@scope 特有の近接性に注意。
- **CSS Modules / BEM**: 従来スタイル隔離に使われてきた手法。@scope が言語機能として代替・補完する。

## 影響範囲 / 推奨アクション

- 影響: CSS を書く全フロントエンド。プリプロセッサ/CSS-in-JS 依存の見直し余地。
- 推奨: テーマ漏れの局所から段階導入。`@layer` と併用して優先順位を明示。詳細度・近接性の挙動をチームで共有。
