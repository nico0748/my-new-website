# Laravel 13 — ファーストパーティ「Laravel AI SDK」搭載、PHP フルスタックが AI ネイティブへ

> トピック: Laravel 13 の新機能と AI SDK｜出典: Laravel 公式リリースノート｜種別: メジャーバージョン（3月リリース、13.18 系まで更新が進み話題継続中）

## 一行サマリー

Laravel 13（2026年3月17日リリース、現行 13.18 系）は、テキスト生成・ツール呼び出しエージェント・埋め込み・音声・画像・ベクトルストア統合を単一 API で扱うファーストパーティの Laravel AI SDK を導入し、PHP のフルスタックフレームワークとして「AI 機能を標準装備で書ける」段階に入った。

## 🔰 初学者向け（何が・なぜ重要か）

Laravel は PHP で最も人気のあるフレームワークで、ウェブアプリに必要な部品（ログイン、データベース、メール送信など）が最初から揃っている「全部入りの工具箱」です。バージョン13の目玉は、その工具箱に「AI の工具一式」が公式に加わったことです。

これまで PHP アプリに AI 機能（チャット、要約、画像生成など）を入れるには、OpenAI 用・Anthropic 用・Google 用と、それぞれ別々のライブラリや自前の HTTP 通信コードを書く必要がありました。Laravel AI SDK は「どの AI 会社を使うかに関わらず、同じ書き方で呼べる共通の窓口」です。電源プラグの変換アダプタのように、裏側の違いを吸収してくれます。

対応範囲も広く、文章生成だけでなく、AI に道具を持たせて自律的に作業させる「エージェント」、文章を数値の並びに変換して検索に使う「埋め込み」、音声や画像、そしてそれらを保存する「ベクトルストア」まで一つの流儀で扱えます。「PHP は AI 時代に乗り遅れた」と言われがちだった状況への、公式からの回答といえます。

## 🧠 専門的解説（技術詳細・設計・使いどころ）

### 技術的詳細（何が変わったか）

- **Laravel AI SDK（ファーストパーティ）**: テキスト生成 / ツール呼び出しエージェント / 埋め込み / 音声 / 画像 / ベクトルストア統合を統一 API で提供。対応プロバイダの全リストとエージェントループの詳細仕様は公式ドキュメントで要確認。
- リリース日: 2026年3月17日。以降パッチが継続し、2026年7月時点の最新は 13.18 系。
- Laravel の年次メジャーサイクル（Q1 リリース）に沿った版。その他のフレームワーク本体の変更点（ディレクトリ構成やスターターキットの変更等）は要確認。

### 仕組み / 背景（なぜこう設計されたか）

フレームワークが AI SDK を「ファーストパーティ」で持つ意味は、単なるラッパー提供ではない。(1) 認証情報・設定の管理を FW の config/秘密管理に統合、(2) キュー・ジョブ（Laravel なら Horizon/Queue）と AI 呼び出しの統合（長時間のエージェント実行をジョブ化）、(3) テスト用のフェイク実装（HTTP をモックせず `AI::fake()` 的に差し替え）といった「FW の他の部品との噛み合わせ」を公式が保証できる点にある。これはコミュニティ製ラッパー（Prism 等）や各社公式 SDK の寄せ集めでは得にくい一貫性である。

設計思想としては Vercel AI SDK（JS）が確立した「プロバイダ抽象＋ストリーミング＋ツール呼び出しの標準化」を PHP/Laravel の慣用句（ファサード、Eloquent 的な流暢 API）で再構成したものと位置付けられる。JS エコシステムで起きた「FW が AI プリミティブを内蔵する」流れ（Vercel AI SDK、Spring AI、Django/Rails のコミュニティ動向）の PHP 版であり、Web フレームワーク間の競争軸が「CRUD の速さ」から「エージェント機能の書きやすさ」へ移っていることを示す。

トレードオフ: プロバイダ抽象は各社固有の最先端機能（プロンプトキャッシュ、構造化出力の方言、バッチ API 等）を最小公倍数に丸めがちで、性能・コスト最適化を突き詰める段階では抽象を破って直接 API を叩く箇所が残る。抽象化の漏れをどこまで「エスケープハッチ」として設計できているかが SDK の寿命を決める。

### 実務での使いどころ・移行の勘所

- 既存 Laravel アプリへの AI 機能追加（サポートチャット、コンテンツ要約、セマンティック検索）は、サードパーティ選定の議論を飛ばして公式 SDK から始めるのが標準ルートになる。
- ベクトルストア統合により、pgvector 等を使う RAG 構成が Eloquent 的な操作感で書ける（対応ストアは要確認）。小〜中規模の RAG なら専用ベクトルDB を建てず Postgres で足りるケースが多い。
- Laravel 12 以前からのアップグレードは通常のメジャー移行手順（互換性リストの確認）で。AI SDK 自体は新規追加なので既存コードへの破壊はない。
- 大規模・多言語組織では「AI ロジックを PHP に置くか、独立サービス（Python）に切り出すか」の境界設計が論点。プロトタイプ〜中規模は SDK 内製、重い学習/推論基盤は分離が目安。

## 📖 用語解説（このトピックとのつながり）

- **ファーストパーティ SDK**: フレームワーク開発元自身が保守する公式ライブラリ。コミュニティ製との違いは他機能との統合保証と寿命。
- **ツール呼び出しエージェント**: LLM に関数（道具）を渡し、必要に応じて呼ばせて結果を再投入するループ。SDK はこのループの定型部分を隠蔽する。
- **埋め込み（Embeddings）**: テキストをベクトルに変換したもの。セマンティック検索や RAG の基礎で、SDK が生成と保存を一続きに扱う。
- **ベクトルストア**: 埋め込みを保存し類似検索する仕組み。pgvector のような既存 DB 拡張も含む。
- **プロバイダ抽象**: OpenAI/Anthropic/Google 等の API 差を吸収する層。利便性と最先端機能アクセスのトレードオフを孕む。

## 影響範囲 / 推奨アクション

- Laravel チーム: AI 機能の PoC は公式 SDK ベースへ集約し、独自ラッパーの新規開発は停止を検討。
- Laravel 11/12 運用中: 13 系へのアップグレード計画に AI SDK 活用の有無を織り込む（サポート期限は eosl 情報を確認）。
