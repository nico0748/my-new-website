# Cisco Catalyst SD-WAN Manager CVE-2026-20262 — 低権限の任意ファイル書き込みからroot奪取、ゼロデイ悪用

> トピック: 脆弱性ニュース｜出典: Cisco PSIRT / BleepingComputer / SecurityWeek / The Hacker News｜種別: 入力検証不備→任意ファイル書き込み→権限昇格(root)

## 一行サマリー

ネットワーク管理基盤 Cisco Catalyst SD-WAN Manager（旧 vManage）のファイルアップロード処理に入力検証不備があり、低権限の認証済み攻撃者が細工した HTTP リクエストでサーバ上の任意の場所にファイルを書き込み、最終的に root 権限へ昇格できる。CISA が 2026-06-15 に KEV へ追加し限定的な実環境悪用を確認済み。回避策はなくアップグレードのみが対策で、全デプロイ形態（オンプレ/クラウド/FedRAMP）が対象。

## 🔰 初学者向け（何が・なぜ問題か）

SD-WAN Manager は、企業内に多数あるネットワーク機器（拠点をつなぐルータ群）を、一つの管理画面からまとめて設定・監視する「ネットワークの司令塔」です。ここを握られると、社内ネットワーク全体に号令をかけられるため、非常に重要な装置です。

この司令塔には「ファイルをアップロードする」窓口があります。本来は決められた保管庫にだけファイルを置くべきですが、アップロードされたファイルの中身や置き場所を十分にチェックしていませんでした。そのため、ログインできる権限の低い利用者が、細工したリクエストを送ることで「保管庫の外」にファイルを書き込めてしまいます。

例えるなら、社員証さえあれば入れる受付の人が、本来は触れない配電盤や金庫室にまで荷物を置けてしまうようなものです。攻撃者はこの「好きな場所にファイルを置ける力」を使って、システムが最高権限（root、何でもできる管理者）で実行するファイルを上書きし、装置を完全に乗っ取ります。

低い権限さえあれば成立し、すでに実際の攻撃で使われています。Cisco は修正版を出していますが、回避策はなく更新するしかありません。

## 🧠 専門的解説（技術詳細・メカニズム・検知対応）

### 技術的詳細（種別・前提条件）

- **CVE**: CVE-2026-20262 / **CVSS**: 要確認（Cisco PSIRT のスコアを参照。性質上 High 相当）
- **CWE**: CWE-20（不適切な入力検証）／ CWE-434・任意ファイル書き込み → 権限昇格
- **影響コンポーネント**: Web UI / API のファイルアップロード処理
- **根本原因**: アップロード時のユーザー入力の検証が不十分で、細工した HTTP リクエストにより本来想定しない場所へファイルを作成・上書きできる。書き込んだファイルが root 権限で処理されることを悪用して任意コマンド実行・権限昇格に至る。
- **影響範囲**: 全デプロイ形態（オンプレ、Cisco SD-WAN Cloud-Pro、Cisco SD-WAN Cloud（Cisco Managed）、SD-WAN for Government（FedRAMP））。デバイス設定に依存せず影響。
- **悪用状況**: 2026年6月に限定的な実環境悪用を Cisco が観測。CISA は 2026-06-15 に KEV 追加、連邦民間機関の対応期限を 2026-06-29 に設定。
- **前提条件**: 低権限であっても認証済みであること（authenticated, low-privilege）。**回避策なし**、修正版へのアップグレードが唯一の対策。
- **関連**: 同月内に Cisco は別の SD-WAN ゼロデイ（CVE-2026-20182）も修正しており、SD-WAN 管理面が連続して標的化されている。

### メカニズム（攻撃が成立する仕組み）

※以下は脆弱性クラスの一般原理に基づく概念説明。武器化手順・PoC・正確なパスは記載しない。

1. **入力検証の不備（信頼境界の誤り）**: アップロード処理がクライアント由来のファイル名/保存先/内容を十分に検証せず受け入れる。外部入力は信頼境界の外にあるため、本来は厳格な正規化・許可リスト検証が必要だが、それが欠けている。
2. **任意ファイル書き込みプリミティブの獲得**: 検証不備により、攻撃者は想定された保管ディレクトリの外へファイルを作成・上書きできる。これが「場所を選んで書ける」プリミティブとなる。
3. **権限差の悪用（低権限→root への橋渡し）**: 書き込んだファイルが、より高い権限（root）を持つコンポーネントやサービスによって読み込み・実行・参照される経路を狙う。攻撃者は低権限ユーザーとして書き込むだけだが、その成果物を特権プロセスに処理させることで実効的に権限昇格する。
   - 脆弱性クラスの一般原理: 「低権限ユーザーが書き込めるファイルを、特権プロセスが検証せず実行/読込する」構図は古典的な権限昇格パターン。信頼境界（誰が書いたか）と実行権限（誰が動かすか）の不一致が橋渡しを生む。
4. **司令塔の掌握**: root を得ることで、管理対象ネットワーク全体の設定変更・監視回避・横展開が可能になり、被害が SD-WAN Manager 単体を超えて広がる。

### 検知・暫定緩和・恒久対策

- **検知の着眼点**: 管理 UI/API への異常なアップロードリクエスト、想定外ディレクトリでのファイル生成・更新タイムスタンプ、root 権限プロセスからの不審な子プロセス、低権限アカウントによる連続したアップロード試行、SD-WAN Manager のシステムログ上の権限昇格痕跡。
- **暫定緩和**: 公式の回避策は提供されていない。更新までの間は、管理面のアクセスを信頼できる管理ネットワーク・ジャンプホストに限定し、不要な低権限アカウントを棚卸し、アカウントの強度と監査を強化する。
- **恒久対策**: 影響を受ける全ブランチで Cisco が提供する修正ソフトへアップグレード（唯一の対策）。悪用が観測されているため、更新後に侵害調査（追加ファイル・新規/改ざんアカウント・設定変更）を実施する。

## 📖 用語解説（このニュースとのつながり）

- **SD-WAN Manager（旧 vManage）**: 多数のネットワーク機器を集中管理する司令塔。掌握されると社内ネット全体に影響する。
- **権限昇格（Privilege Escalation）**: 低い権限から、より高い権限（本件では root）を奪う行為。被害が一気に拡大する。
- **任意ファイル書き込み（CWE-434/CWE-20）**: 場所や内容を検証せずファイルを受け入れる不備。特権プロセスが処理するとRCE/昇格へ直結。
- **root**: UNIX 系で何でもできる最上位権限。これを得ると装置を完全制御できる。
- **ゼロデイ（zero-day）**: 修正提供前から悪用されていた脆弱性。本件は限定的ながら実環境悪用が先行した。
- **回避策なし（no workaround）**: 設定変更等で塞げず、更新でしか対処できない状態。アップグレードが必須。

## 影響範囲 / 推奨対応

影響は、Cisco Catalyst SD-WAN Manager の全デプロイ形態（オンプレ/各種クラウド/FedRAMP）で、デバイス設定に依存せず広範。低権限の認証アクセスがあれば root 奪取に至るため、内部脅威・侵害済みアカウント経由のリスクも高い。推奨対応は、回避策がないため影響ブランチの修正ソフトへ速やかにアップグレードし、管理面アクセスを最小化、アカウントを棚卸しすること。悪用観測済みのため、更新後も侵害調査を行う。
