# DGM3103SCT：Web管理画面ログイン後にroot権限で任意コマンド実行（CVE-2026-56808 / JVN#28979424）

> トピック: 脆弱性ニュース ｜ 出典: JVN（Japan Vulnerability Notes）／ AVTECH Security ｜ 種別: OSコマンドインジェクション（CWE-78、認証後→root RCE）

## 一行サマリー

JVN が AVTECH 製ネットワーク機器 DGM3103SCT の OS コマンドインジェクション（CVE-2026-56808）を6月30日に公表。Web 管理画面にログインできるユーザーが root 権限で任意コマンドを実行可能（CVSS 4.0＝8.6 / CVSS 3.0＝7.2）。影響はファームウェア 3.2.5.4 以前で、最新版へのアップデートが必要。国内でも JVN 経由で注意喚起された機器のため、該当設置環境は要確認。

## 🔰 初学者向け（何が・なぜ問題か）

DGM3103SCT は、監視カメラ系などで使われるネットワーク機器（メーカーは台湾の AVTECH社）です。設定は「Web 管理画面」からブラウザで行います。今回の欠陥は、その管理画面にログインできる人が悪用できるタイプです。

問題は**OS コマンドインジェクション**という種類です。管理画面には、機器名やネットワーク設定などを入力する欄があります。本来こうした欄には"ただの文字"が入る想定ですが、この機器では入力された文字がそのまま機器内部の「命令（OS コマンド）」の一部として実行されてしまいます。たとえるなら、注文票の「備考欄」に書いた文章を、店員がそのまま"厨房への指示書"として実行してしまうようなものです。

しかも実行される権限が最高権限の **root**。攻撃者が管理画面に入り込めれば（初期パスワードの使い回しや弱いパスワード、あるいは正規権限の悪用で）、機器を完全に掌握し、監視映像の窃取や社内ネットワークへの侵入の踏み台にできます。

## 🧠 専門的解説（技術詳細・メカニズム・検知対応）

### 技術的詳細（種別・前提条件）

- **CVE**: CVE-2026-56808 ／ 識別子: JVN#28979424。
- **公開日**: 2026年6月30日（JVN）。
- **CVSS**: CVSS 4.0 基本値 **8.6**、CVSS 3.0 基本値 **7.2**（要確認：レポート値）。
- **種別（CWE）**: OS コマンドインジェクション（CWE-78）。
- **前提条件**: 当該機器の **Web 管理画面にログイン可能**であること（認証後）。認証を得た第三者、または正規権限を悪用できる者が対象。
- **影響製品/バージョン**: AVTECH Security 製 DGM3103SCT、ファームウェア **Ver. 3.2.5.4 およびそれ以前**。
- **修正版**: 開発者提供情報に基づく最新版へのアップデート（該当バージョンは JVN／ベンダー情報を要確認）。
- **悪用状況**: 本記事時点で実悪用の報告は確認されず（要確認）。ただし IoT/監視系機器はボットネット・侵入の標的になりやすい。

### メカニズム（攻撃が成立する仕組み）

※本節は**概念レベル**。武器化手順・PoC・具体的コマンドは記載しない。

1. **信頼境界の誤り**: Web 管理画面のパラメータ（設定値・ネットワーク項目等）が、機器内部でシェルコマンドの組み立てに使われる。認証済みユーザーの入力を"信頼できる文字列"として扱い、コマンド文字列に連結してしまう設計がそもそもの綻び。
2. **不十分な無害化**: 入力に対しメタ文字（`;` `|` `&` `` ` `` `$()` 等、コマンド区切り・置換に使われる記号）のエスケープ・除去・許可リスト検証が不足。ユーザー入力が"データ"の枠を越えて"コマンド"として解釈される余地が残る。
3. **コマンド実行への橋渡し**: 組み立てられた文字列が `system()` 等でシェルに渡され実行される。攻撃者が握るプリミティブは「コマンド文字列への任意文字の混入」で、これがそのまま任意コマンド実行に直結する。
4. **root への昇格不要の被害**: 当該処理が root 権限デーモンで動作しているため、追加の権限昇格なしに最初から root 相当の実行が得られる。機器の完全掌握・永続化・横展開に繋がる。
5. **一般原理（型の説明）**: OS コマンドインジェクションは「ユーザー入力を可変部としてシェルコマンドを動的生成する」設計に内在する古典的欠陥。対策の本筋は、シェルを介さない API 呼び出し、引数の配列渡し（シェル解釈を経由させない）、厳格な入力検証（許可リスト）。認証後であっても"管理者＝信頼"を過信せず入力を無害化すべき、という教訓。

### 検知・暫定緩和・恒久対策

- **検知の着眼点**: 管理画面設定項目への異常な記号・長大入力、設定変更を契機とした予期しない子プロセス／シェル起動、機器からの不審なアウトバウンド通信、管理アカウントの異常ログイン。
- **暫定緩和（更新前）**: 管理画面をインターネットに公開しない（VPN／管理LAN限定）。初期・弱いパスワードを変更し多要素・アクセス制限を適用。不要な遠隔管理機能を無効化。
- **恒久対策**: 開発者提供の修正ファームウェアへ更新。
- **運用示唆**: 「認証後だから低リスク」と軽視しない。管理画面が外部到達可能だと、資格情報窃取や使い回しで認証の壁は容易に越えられる。IoT/監視機器は"設置して放置"されがちで、資産棚卸しとファーム更新運用の整備が重要。

## 📖 用語解説（このニュースとのつながり）

- **OS コマンドインジェクション（CWE-78）**: 入力がシェルコマンドの一部として実行される欠陥。今回の本体で、"備考欄が指示書になる"構図。
- **JVN（Japan Vulnerability Notes）**: JPCERT/CC と IPA が運営する国内の脆弱性情報ポータル。本件は JVN 経由で国内向けに周知された。
- **root 権限**: 機器の最高権限。処理が root で動くため、注入成功が即・完全掌握に繋がる。
- **認証後（Authenticated）脆弱性**: ログイン後に成立する欠陥。管理画面の外部公開・弱いパスワードがあると"認証済み"の前提は簡単に満たされてしまう。
- **CVSS 4.0 / 3.0**: 深刻度指標の新旧版。本件は 4.0 で 8.6、3.0 で 7.2 と評価軸により値が異なる（比較時は版に注意）。

## 影響範囲 / 推奨対応

AVTECH DGM3103SCT（ファーム 3.2.5.4 以前）を設置する組織・施設。管理画面を外部公開している、または初期/弱いパスワードのままの環境は特にハイリスク。(1) 最新ファームへの更新、(2) 管理画面の外部非公開化とアクセス制限、(3) 管理アカウントのパスワード強化、(4) 設定変更を契機とする異常プロセス／通信の監視。
