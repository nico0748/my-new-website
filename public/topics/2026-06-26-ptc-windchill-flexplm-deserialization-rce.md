# PTC Windchill / FlexPLM 入力検証不備によるデシリアライズ→未認証RCE CVE-2026-12569（KEV登録）

> トピック: 脆弱性ニュース ｜ 出典: CISA KEV / PTC Trust Center / NVD ｜ 種別: 入力検証不備（CWE-20）→ 安全でないデシリアライズ→ 未認証RCE

## 一行サマリー

製造業の製品データ管理（PLM）基盤 PTC Windchill PDMLink と FlexPLM に、入力検証不備に起因する安全でないデシリアライズの脆弱性があり、未認証の遠隔攻撃者が任意コード実行（システム全体の掌握）に至りうる。CISAは6/25にKEV登録。国内の製造業で広く使われる基幹系のため国内影響大。

## 🔰 初学者向け（何が・なぜ問題か）

Windchill（ウィンドチル）やFlexPLMは、メーカーが**製品の設計図・部品表・仕様書を一元管理する“会社の金庫兼図書館”**のようなシステムです（PLM=製品ライフサイクル管理）。自動車・電機・アパレルなど、ものづくり企業の中枢で動いています。

このシステムは、外部から送られてきたデータを受け取って“元の形に組み立て直す”処理（デシリアライズ＝荷ほどき）を行います。ここで問題なのは、**送られてきた荷物の中身をろくに確認せずに荷ほどきしてしまう**こと。攻撃者は荷物の中に「組み立てた瞬間に勝手にプログラムが動き出す仕掛け」を仕込んで送りつけられます。

しかもログイン不要（未認証）で、ネット越しに届けば成立します。結果として、攻撃者は設計図の宝庫であるサーバーを乗っ取り、**機密の設計データを盗む・改ざんする・暗号化して身代金を要求する**といったことが可能になります。会社の“レシピ”が丸ごと危険にさらされる、というイメージです。

## 🧠 専門的解説（技術詳細・メカニズム・検知対応）

### 技術的詳細（種別・前提条件）

- **CVE**: CVE-2026-12569 / **CVSS v4.0**: 9.3（Critical、Network/PR:None/UI:None）/ **種別**: 入力検証不備（CWE-20）に起因する安全でないデシリアライズ（CWE-502）→ 未認証RCE。
- **対象**: PTC Windchill PDMLink および FlexPLM。影響版は 11.2.1.0 / 12.0.2.0 / 12.1.2.0 / 13.0.2.0 / 13.1.0.0〜13.1.3.0、および 11.0 M030 より前のリリース等（NVD/PTC Advisory参照、細部は**要確認**）。
- **修正版**: PTC公式アドバイザリ（Trust Center）の指示版へ更新（恒久パッチ/緩和の適用、版番号は**要確認**）。
- **悪用状況**: CISA KEV登録（2026-06-25）＝実環境での悪用根拠ありとCISAが判断。一部の脆弱性DBは「公開時点でin-the-wild不明」と記す情報もあり、悪用範囲の断定は避けるが、KEV掲載を踏まえ最優先扱いが妥当。

### メカニズム（攻撃が成立する仕組み）

「未認証の外部入力 ↔ アプリ内部のオブジェクト復元」という信頼境界で、入力の素性検証を欠いたまま危険なデシリアライズが行われる、という安全でないデシリアライズの一般原理に沿う。

1. **入力検証の不備（CWE-20）**: 公開エンドポイントが、受信したシリアライズ済みデータの型・出所・整合性を十分検証せずに処理パスへ渡す。ここで信頼境界が正しく引かれていない。
2. **安全でないデシリアライズ（CWE-502）**: 検証されない直列化データをオブジェクトへ復元する過程で、攻撃者が制御するクラス/ガジェットが実体化する。Javaエコシステム等では、復元時に副作用を持つ「ガジェットチェーン」が、データの“荷ほどき”だけでコード実行プリミティブへ転化しうる。
3. **プリミティブの橋渡し**: 攻撃者は「任意オブジェクト生成」というプリミティブを足がかりに、最終的にプロセス権限での任意コード実行（RCE）へ到達。認証を要しないため、到達性さえあれば初期侵入から一気に深部へ。
4. **被害の波及**: PLMは設計IP・部品表・サプライヤ情報の宝庫。RCE成立は機密漏えい・改ざん・ランサム展開・横展開（Active Directory連携先など）の起点になる。

脆弱性クラスとしての説明であり、**具体的なガジェット・PoC・ペイロード構造は記載しない**（概念レベル）。CVE個別の内部実装の詳細が未公開の部分は断定を避ける。

### 検知・暫定緩和・恒久対策

- **検知の着眼点**: Windchill/FlexPLMアプリサーバ（Java/Apache等）の異常な子プロセス生成、想定外のアウトバウンド接続、デシリアライズ系の例外スタックトレース急増、未認証での大きなPOST/特定エンドポイントへの連続アクセス。WAF/IDSでの直列化データ署名パターン。
- **暫定緩和**: 当該システムをインターネットから隔離（VPN/許可IP限定）、リバースプロキシで未認証アクセスを制限、PTCが提示する緩和策（該当機能の無効化等）の適用。
- **恒久対策**: PTC Trust Centerの指示に従い修正版へ更新。更新後も侵害有無（Webシェル・追加アカウント・不審ジョブ）を点検。資産棚卸しで露出インスタンスを特定し露出を最小化。

## 📖 用語解説（このニュースとのつながり）

- **PLM / Windchill / FlexPLM**: 製品の設計〜廃棄までのデータを管理する基幹。今回はこの“金庫”が狙われた。
- **デシリアライズ（CWE-502）**: 受信データをオブジェクトに復元する処理。中身を検証しないと、復元＝コード実行になりうる。
- **入力検証不備（CWE-20）**: 受け取ったデータの素性チェック漏れ。今回の根本原因で、デシリアライズの暴発を許した。
- **ガジェットチェーン**: 既存クラスを連鎖させてコード実行に持ち込む手口の総称（概念）。
- **KEV / BOD 26-04**: 実悪用が確認された脆弱性のCISAリストと、連邦機関に迅速是正を課す指令。優先度判断の指標。

## 影響範囲 / 推奨対応

国内製造業（自動車・電機・産機・アパレル等）でWindchill/FlexPLMは広く採用されており、外部公開やVPN経由でアクセス可能な構成は高リスク。**最優先でPTCアドバイザリに沿った更新/緩和を適用し、露出を遮断、侵害痕跡を点検**すること。設計IP保護の観点から経営レベルでの優先対応が望ましい。
