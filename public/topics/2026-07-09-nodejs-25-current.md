# Node.js 25（Current）— ESM/CJS 相互運用の進化とデバッグ強化、LTS ではない実験ライン

> トピック: Node.js 25（Current リリース）｜出典: Node.js 公式リリース情報｜種別: ランタイム更新

## 一行サマリー

Node.js 25 は奇数番のため LTS には入らず約6か月で EOL する「Current（先行）」ライン。ESM/CJS 相互運用の継続改善、デバッグ/プロファイリング強化、メモリ消費削減などを含み、本番の安定運用には偶数番の Node.js 24（Active LTS）が引き続き推奨される。

## 🔰 初学者向け（何が・なぜ重要か）

Node.js は「ブラウザの外で JavaScript を動かす」実行環境で、サーバーや CLI ツールの定番土台です。Node.js にはバージョンの付け方にルールがあり、**偶数番（24 など）は長期サポート（LTS）**、**奇数番（25 など）は短命の実験ライン**という慣習があります。奇数番は最新機能をいち早く試す場で、長く使うのには向きません。

Node.js 25 はその奇数番。約半年で役目を終える代わりに、次の LTS に入る予定の新機能を先取りできます。今回のテーマは、昔ながらの書き方（CommonJS＝`require`）と新しい標準（ESM＝`import`）を混ぜて使うときの相互運用の改善や、問題を早く見つけるためのデバッグ・計測ツールの強化、そしてメモリを節約する改良などです。要するに「本番はまだ 24 で、25 は新機能のお試し用」という位置づけを理解しておくのが大事です。

## 🧠 専門的解説（技術詳細・設計・使いどころ）

### 技術的詳細（何が変わったか）

- **Node.js 24 は Active LTS**（EOL 2028年4月30日）で、業務クリティカルには安全な選択肢。
- **奇数番（19/21/23/25）は LTS に入らず**、Current として約6か月で EOL、長期セキュリティ支援なし。
- Node.js 25 は Current 段階で主に実験用途向け。`NODE_MODULE_VERSION` は 141 に更新（ネイティブアドオンの ABI が変わるため再ビルドが必要な指標）。
- EOL 済み API は完全撤去、ESM/CJS 相互運用の継続的進化、デバッグ/プロファイリングの改善、メモリ消費削減の実行時改良（サーバーレス/マイクロサービスに有利）。
- 具体的な API 追加点はリリースノートで要確認。

### 仕組み / 背景（なぜこう設計されたか）

Node のリリースモデルは「安定を求める本番」と「最新を試す先行検証」を両立させるために、偶数=LTS/奇数=Current の二系統を採る。新機能はまず Current で世に出し、フィードバックとバグ修正を経て、次の偶数番で LTS 化される。こうすることで、破壊的変更や未成熟機能が長期サポート環境に直接流れ込むのを防ぐ。`NODE_MODULE_VERSION`（ABI バージョン）が上がるのは、V8 や内部 API が変化しネイティブアドオンのバイナリ互換が切れることを示すため。メジャー更新時に `node-gyp` 等で再ビルドが要るのはこのためだ。

ESM/CJS 相互運用が毎バージョン話題になるのは、Node が歴史的に CommonJS（同期 `require`）で発展し、後から ECMAScript 標準の ESM（非同期・静的解析可能な `import`）を取り込んだ経緯による。両者はモジュール解決・実行タイミング・トップレベル await の扱いが異なり、「ESM から CJS を読む」「CJS から ESM を読む」の各方向で細かな制約が残ってきた。近年は同期的に ESM を `require` できる方向など、相互運用の摩擦を減らす改善が続いており、25 もその延長にある。トレードオフは、相互運用を緩めるほどエッジケースの挙動が増え、ツール（バンドラ・型システム）側の追随が必要になる点。

### 実務での使いどころ・移行の勘所

Node.js 25 は「次の LTS に入る機能を先に検証する」用途で使う。CI に 25 のジョブを足して、自社コード・依存・ネイティブアドオンが新 ABI・新挙動で動くかを事前に確認しておくと、次期 LTS への移行が滑らかになる。逆に、本番サービスやライブラリの推奨サポート対象は 24（Active LTS）に据えるのが定石。奇数番を本番採用すると、半年後に EOL してセキュリティ更新が止まるリスクを負う。ネイティブアドオンを含むプロジェクトは `NODE_MODULE_VERSION` 変更に伴う再ビルドを忘れずに。

## 📖 用語解説（このトピックとのつながり）

- **LTS / Current**: 長期サポート（偶数番）と短命の先行版（奇数番）。25 は Current。
- **ESM / CommonJS（CJS）**: `import` の標準モジュールと `require` の旧来方式。相互運用が継続テーマ。
- **NODE_MODULE_VERSION（ABI）**: ネイティブアドオンのバイナリ互換を示す番号。25 で 141 に更新。
- **ネイティブアドオン**: C/C++ で書かれた Node 拡張。ABI 変更時は再ビルドが必要。
- **EOL（End of Life）**: サポート終了。奇数番は約6か月で到達。

## 影響範囲 / 推奨アクション

- 影響: Node.js を使うサーバー/ツール全般。特にネイティブアドオン利用者と、次期 LTS を見据えるチーム。
- 推奨: 本番は Node.js 24（LTS）を継続。25 は CI で先行検証に活用し、ABI 変更に伴う再ビルドと ESM/CJS 挙動差を確認。
