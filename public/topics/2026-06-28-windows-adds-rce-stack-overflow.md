# Windows Active Directory Domain Services のスタックオーバーフローでDC上RCE（CVE-2026-45648, 6月Patch Tuesday）

> トピック: 脆弱性ニュース｜出典: Rapid7 / Cisco Talos / Windows Forum（6月Patch Tuesday）｜種別: スタックベースのバッファオーバーフロー（CWE-121）による認証後リモートコード実行

## 一行サマリー
Active Directory Domain Services（AD DS）にスタックバッファオーバーフローがあり、認証済みの攻撃者がネットワーク越しにドメインコントローラ上で任意コードを実行できる（CVE-2026-45648, CVSS 8.8）。記録的だった2026年6月Patch Tuesdayの重要RCEのひとつで、未認証のNetlogon欠陥（CVE-2026-41089）と並びDC防御の要となる。

## 🔰 初学者向け（何が・なぜ問題か）
前述のNetlogonの記事と同じく、これも組織の“門番にして名簿管理人”であるドメインコントローラ（DC）を狙う脆弱性です。違いは「最初の一歩に、何らかの正規アカウントが必要」な点。つまり、フィッシングや使い回しパスワードなどで一般社員のアカウントを1つ手に入れた攻撃者が、その足場からDCを乗っ取りに行ける、という想定です。

AD DS（DCの中核機能）が処理する“申請書”の記入欄に、長すぎる内容を流し込むと、机（メモリ）からはみ出して処理手順を上書きできてしまう——構造はNetlogonと同じ「はみ出し書き込み（バッファオーバーフロー）」です。一般アカウントという小さな足がかりが、組織全体の支配権へ一気に昇格しうるため、「認証が必要だから後回し」とは言えない危険度です。

## 🧠 専門的解説（技術詳細・メカニズム・検知対応）

### 技術的詳細（種別・前提条件）
- **CVE**: CVE-2026-45648。**CVSS**: 8.8。**CWE**: CWE-121（スタックベースのバッファオーバーフロー）。
- **悪用状況**: 6月Patch Tuesdayで修正。**実環境での悪用は本稿時点で確証なし＝要確認**（同時期に悪用が確認されたのは未認証のCVE-2026-41089）。
- **前提条件**: **認証済み（authorized）の攻撃者**であること。ネットワーク経由でAD DSの該当処理へ到達できること。ユーザー操作は不要。
- **影響範囲（出典記載）**: Windows Server 2022（21H2/22H2）、Windows Server 2025（24H2）。**修正KB**: KB5094128 / KB5094125（出典記載、環境別に要確認）。

### メカニズム（攻撃が成立する仕組み）
1. **認証後に到達する攻撃面**: AD DSがドメインユーザーからの正規リクエストを処理する過程に、攻撃者が長さ/内容を制御できるフィールドのパース処理が存在する。低権限でも認証さえ通れば到達可能な内部処理面。
2. **境界外書き込み**: 固定長スタックバッファに対する境界チェック不足により、攻撃者制御データがスタック上の隣接領域（戻りアドレス/制御データ）を上書きできる（古典的スタックBOF）。
3. **制御フロー奪取→コード実行**: 制御データの破壊により実行フローを攻撃者コードへ誘導するプリミティブが成立。実行コンテキストがDCの特権プロセスであるため、認証後RCE＝事実上のドメイン掌握に橋渡しされる。
4. **権限の質的転換**: 「一般ドメインユーザー」という水平的な足場から、「DC上のコード実行」という垂直的な最高権限へ昇格する点が本質。初期侵入（資格情報窃取等）と組み合わさると被害が連鎖する。
   - 一般原理: 認証“後”のメモリ破壊は「正規ユーザーなら誰でも特権昇格できる」内部脅威・横展開の温床。認証の有無は深刻度を下げる要素だが、資格情報が容易に漏れる現実では“認証必須”は十分な緩和にならない。
   - 本記述は概念レベルに留め、オフセットや武器化手順・PoCは記載しない。

### 検知・暫定緩和・恒久対策
- **検知の着眼点**: DC上のAD DS/lsass関連プロセスの異常終了・再起動、低権限アカウントによる異常なディレクトリ操作、直後の特権昇格・横展開挙動。
- **暫定緩和**: 6月更新適用までDCへのアクセスを最小化、特権分離、低権限アカウントの監視強化、資格情報の使い回し排除とMFA徹底で初期足場の獲得を阻害。
- **恒久対策**: 6月Patch Tuesday更新（KB5094128/KB5094125 等、環境別に確認）を全DCへ適用。
- **運用示唆**: 未認証のCVE-2026-41089と本件はセットでDCを脅かす。両方を同一メンテナンスで確実に塞ぐ。

## 📖 用語解説（このニュースとのつながり）
- **AD DS（Active Directory Domain Services）**: ドメインの認証・ディレクトリを担うDCの中核機能。本件の欠陥箇所。
- **認証後（authenticated）RCE**: 攻撃に正規アカウントが必要なRCE。未認証より一段ハードルは高いが、資格情報漏えいで前提が満たされやすい。
- **スタックベースのバッファオーバーフロー（CWE-121）**: 固定長スタック領域を越えた書き込みで制御データを破壊する欠陥。Netlogon欠陥と同種。
- **Patch Tuesday**: Microsoftが毎月第2火曜に更新を公開する日。2026年6月は206件・記録的規模だった。
- **特権昇格（横→縦）**: 一般ユーザー権限からDCのSYSTEM級権限へ跳ね上がること。本件の最終被害像。

## 影響範囲 / 推奨対応
AD DSを稼働するWindows Server 2022/2025が対象。資格情報を1つ握った攻撃者にDC掌握を許しうるため、6月更新を全DCへ適用する。あわせて資格情報の使い回し排除・MFA・特権分離で初期侵入のハードルを上げ、未認証のCVE-2026-41089と同時に対処する。
