# Fortinet FortiClient EMS のアクセス制御不備でゼロデイRCE（CVE-2026-35616, 実環境で悪用・KEV登録）

> トピック: 脆弱性ニュース ｜ 出典: watchTowr / Tenable / The Hacker News / CyberScoop ｜ 種別: 不適切なアクセス制御（CWE-284）→ 認証回避→コード実行

## 一行サマリー

エンドポイント管理サーバ Fortinet FortiClient EMS のAPIにアクセス制御不備があり、認証なしの細工リクエストでサーバ上のコード実行に至る（CVE-2026-35616、CVSS 9.1）。ベンダー公表前から実環境で悪用されたゼロデイで、CISA KEVに登録済み。緊急ホットフィックスの適用が必須。

## 🔰 初学者向け（何が・なぜ問題か）

会社では何百台ものパソコンに同じセキュリティソフト（FortiClient）を入れて、それを**1台の管理サーバ（EMS）からまとめて配信・設定**します。この管理サーバは、全社の端末を束ねる「司令塔」です。司令塔を乗っ取られれば、攻撃者は全端末へ命令を出せる立場になります。

今回の問題は、この司令塔サーバの「窓口（API）」が、**本来は社内管理者しか触れないはずの操作を、ログインしていない相手にも実行させてしまう**ことです。たとえるなら、本社ビルの最重要フロアの扉に鍵がかかっておらず、受付を素通りして誰でも入れてしまう状態です。

攻撃者は認証を一切せずに、細工したリクエストを送るだけで管理サーバ上でプログラムを動かせます。しかもこの欠陥は、Fortinetが注意喚起を出す**前から実際に攻撃に使われていた（ゼロデイ）**ことが確認され、CISAの「悪用確認済みリスト（KEV）」にも載りました。司令塔を握られると、全社の端末が一斉に危険にさらされます。

## 🧠 専門的解説（技術詳細・メカニズム・検知対応）

### 技術的詳細（種別・前提条件）

- **CVE**: CVE-2026-35616
- **種別（CWE）**: CWE-284 不適切なアクセス制御（認証・認可の回避）。
- **CVSS**: 9.1（出典により9.8表記あり、**要確認**）。
- **前提条件**: FortiClient EMSの管理API/サーバへネットワーク到達できること。**認証不要・ユーザー操作不要**。
- **影響バージョン**: FortiClient EMS 7.4.5 / 7.4.6。**7.2系以前は影響なし**。
- **修正**: 緊急ホットフィックス提供。恒久修正は7.4.7予定（適用状況は環境ごとに**要確認**）。
- **悪用状況**: 2026-03-31にwatchTowrのセンサーが悪用を観測（公表は4/4）。CISA KEVに2026-04-06登録。同製品では別のSQLインジェクション（CVE-2026-21643）も近接して悪用されており、攻撃者の注目度が高い。

### メカニズム（攻撃が成立する仕組み）

1. **信頼境界**: FortiClient EMSは管理者向けのAPIエンドポイントを提供する。これらは「認証・認可を通過した管理者のみがアクセスできる」前提で特権操作（設定変更・配布・サーバ側処理）を行う。境界は「APIゲートウェイでの認証/認可チェック」にある。
2. **アクセス制御の欠落**: 一部のAPIパス/操作で、認証・認可チェックが正しく適用されていなかった（または別経路で回避できた）。これにより攻撃者は「認証済み管理者だけが呼べるはずの機能」を**未認証で呼び出すプリミティブ**を得る。
3. **特権機能の悪用→コード実行**: 未認証で到達できた特権機能を通じ、サーバ側でファイル操作・コマンド実行・配布機構などを起動できる経路が存在し、これが**サーバ上のコード実行**へ橋渡しされる。EMSは多数の管理端末を制御するため、サーバ掌握は配下端末への影響に拡大しうる。
4. **プリミティブから最終被害へ**: 「認証回避による特権API到達」が、特権機能の連鎖を介して「管理サーバ完全掌握」に至る。エンドポイント管理基盤の制圧は、横展開・大規模侵害の起点となる。

一般原理として、**認証と認可を各エンドポイントで一貫して強制しない（中央集権的なポリシー適用の欠如）**設計は、アクセス制御欠落（Broken Access Control）の典型。武器化手順・具体的なAPIパスは記載しない。

### 検知・暫定緩和・恒久対策

- **検知の着眼点**: EMS管理APIへの未認証アクセス、想定外のIPからの管理操作、EMSサーバ上の予期しない子プロセス生成やファイル作成、Fortinet/CISAが公開するIoC。watchTowr/Tenableの侵害確認手順を参照。
- **暫定緩和**: 緊急ホットフィックスを即適用。EMS管理インターフェースを**インターネットから遮断**し、管理アクセスを専用セグメント/VPNに限定。
- **恒久対策**: 7.4.7（恒久パッチ）へ更新。同製品の関連CVE（CVE-2026-21643等）も併せて対処。
- **運用示唆**: ゼロデイ＋KEV登録のため、既に侵害された可能性を前提に、EMSサーバと配下端末の侵害有無を点検（不正アカウント・改ざんされた配布設定）。

## 📖 用語解説（このニュースとのつながり）

- **FortiClient EMS**: 多数の端末のセキュリティ設定を集中管理するサーバ。今回の司令塔＝脆弱な対象。
- **アクセス制御不備（CWE-284 / Broken Access Control）**: 認証・認可を正しく強制しない欠陥。今回の根本原因。
- **ゼロデイ**: 修正提供前から悪用される脆弱性。今回は公表前に攻撃が観測された。
- **KEV**: CISAの悪用確認済み脆弱性カタログ。登録＝早急な対応が必要な実害級。
- **横展開（ラテラルムーブメント）**: 侵入後に組織内へ攻撃範囲を広げる動き。管理基盤掌握はその起点になる。

## 影響範囲 / 推奨対応

FortiClient EMS 7.4.5 / 7.4.6 を運用する組織が対象（7.2以前は非該当）。**緊急ホットフィックス即適用**、管理インターフェースの外部遮断、7.4.7への更新を実施。ゼロデイ・KEV登録のため、EMSサーバと配下端末の侵害有無を併せて確認すること。
