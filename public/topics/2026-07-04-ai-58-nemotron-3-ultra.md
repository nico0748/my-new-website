# NVIDIA Nemotron 3 Ultra ― Mamba-2×Attention ハイブリッドMoEで長時間エージェントを効率化

> トピック: 開発トピック｜出典: developer.nvidia.com｜種別: ai

## 一行サマリー
NVIDIA が 2026-06-04 に公開した **Nemotron 3 Ultra** は、**Mamba-2 と Attention を組み合わせたハイブリッド MoE**（総 550B / アクティブ 55B）で、1M トークンの長文脈と thinking/non-thinking モード、ツール利用を備える。長時間走り続けるエージェント向けに「推論の速さと効率」を狙った設計で、NVIDIA NIM と Hugging Face から利用できる。

## 🔰 初学者向け（何が・なぜ重要か）
AI エージェント（自律的に何ステップも作業を続ける AI）は、長く動くほど「毎回長い履歴を読み直す」コストが重くなります。今の主流である Transformer（Attention 方式）は賢い一方で、文脈が長くなると計算量が急増しやすいという弱点があります。

Nemotron 3 Ultra は、この弱点を補うために **Mamba-2** という別方式を混ぜています。Mamba 系は「長い列を一定の効率で流し読みする」のが得意で、Attention は「重要な箇所にピンポイントで注目する」のが得意。両方を組み合わせることで、「長時間・長文脈でも効率よく、それでいて賢く」を目指しています。速く走り続けられて燃費も良い、ハイブリッド車のようなイメージです。長く働くエージェントほど、この効率が効いてきます。

## 🧠 専門的解説（技術詳細・設計・使いどころ）
### 技術的詳細（何が変わったか）
- **リリース: 2026-06-04**。アーキテクチャは **Mamba-2 / Attention のハイブリッド Mixture-of-Experts**。
- 規模は **総 550B / アクティブ 55B**。**1M トークン**のコンテキストに対応。
- **thinking / non-thinking モード**を切替可能で、ツール利用（tool-use）指向の機能を提供。
- 提供形態は **NVIDIA NIM** と **Hugging Face**（Nemotron 3 には Super 120B A12B 等の下位バリアントも存在、要確認）。

### 仕組み / 背景（なぜこう設計されたか）
Transformer の Attention は、系列長 N に対して計算・メモリが概ね N² で増える性質があり、超長文脈や長時間エージェントでコストが跳ね上がります。一方 **状態空間モデル（SSM）系の Mamba-2** は、系列を線形に近い計算量で流せる代わりに、任意位置への“ピンポイント参照”は Attention ほど得意ではありません。ハイブリッド設計は、層の一部を Mamba-2（長い文脈を効率的に運ぶ担当）、一部を Attention（重要箇所への精密な参照担当）に割り当て、両者の長所を取り、短所を補い合います。

さらに **MoE** により、総 550B の容量を持ちつつ1トークンの実効計算を 55B に抑え、推論スループットとコストを改善します。thinking モードは難しい推論に計算を厚く割り当て、non-thinking は軽い応答を速く返す——「必要なときだけ深く考える」制御です。トレードオフは、ハイブリッド構成の実装・チューニングの複雑さと、Attention 純正モデルに比べた挙動特性の違い（長距離の厳密な参照が要る一部タスクでの検証が必要）です。長時間エージェントという用途に絞った最適化である点を理解して使うのが肝要です。

### 実務での使いどころ・移行の勘所
- 長時間・長文脈で走り続けるエージェント（自律リサーチ、監視、長い対話・ワークフロー）で効率の恩恵が大きい。
- NVIDIA NIM 経由なら自社インフラ（GPU）での本番デプロイがしやすく、データ主権・レイテンシ制御に有利。
- thinking/non-thinking の使い分けでコストと品質のバランスを取れる。長距離の厳密参照が要るタスクは事前検証を。

## 📖 用語解説（このトピックとのつながり）
- **Nemotron 3 Ultra**: NVIDIA の長時間エージェント向けハイブリッド MoE モデル。本トピックの主役。
- **Mamba-2 / 状態空間モデル (SSM)**: 長い系列を線形に近い計算量で処理する方式。効率の源泉。
- **Attention / Transformer**: 重要箇所へ精密に注目する主流方式。長文脈で計算量が増えやすい。
- **ハイブリッド MoE**: Mamba と Attention を混ぜ、専門家の一部だけ起動する構造。効率と性能を両立。
- **thinking / non-thinking モード**: 推論の深さを切替える制御。難タスクにだけ計算を厚くする。
- **NVIDIA NIM**: NVIDIA のモデル推論マイクロサービス。自社 GPU での本番配備に使う。

## 影響範囲 / 推奨アクション
長時間・長文脈エージェントを自社インフラで動かしたい AI 開発者に関係する。効率重視のアーキテクチャなので、Attention 純正モデルで推論コストが問題化している用途に検討価値がある。NIM/HF で試し、thinking/non-thinking のコスト効果と長距離参照精度を自社タスクで確認するのが妥当。
