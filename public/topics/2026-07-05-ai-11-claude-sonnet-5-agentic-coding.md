# Claude Sonnet 5 が Claude Code の既定に — ネイティブ 1M トークンで長尺コーディング

> トピック: Claude Sonnet 5 の Claude Code 既定化・ネイティブ 1M コンテキスト・導入価格｜出典: Anthropic 公式｜種別: AI コーディングエージェント（開発ツール）

## 一行サマリー

2026年6月30日に登場した Claude Sonnet 5 が Claude Code の既定モデルになり、ネイティブ 1M トークンのコンテキストと、$2/$10 per Mtok（8月31日までの導入価格）でエージェント的コーディングに投入された。Sonnet 4.6 比で推論・ツール利用・コーディングが向上し、「大規模コードベースを丸ごと文脈に入れて作業する」開発体験が現実的になっている。

## 🔰 初学者向け（何が・なぜ重要か）

「コーディングエージェント」とは、AI がコードを読み、書き、テストを回し、複数ファイルにまたがる修正を自分で進める仕組みです。その代表格が Anthropic の Claude Code で、その頭脳となる既定モデルが新しい Claude Sonnet 5 に切り替わりました。

大きいのは「一度に読める量（コンテキスト）」が最大100万トークンになったこと。トークンはおおまかに単語のかけらのようなもので、100万トークンは分厚い本を何冊も同時に開いておけるイメージです。これまでは大きなプロジェクトだと「一部しか見せられない」ため、AI が全体像を掴めず的外れな修正をしがちでした。丸ごと見渡せると、関連ファイルの整合を取った修正や、広い範囲のリファクタリングが任せやすくなります。

さらに一定期間の割引価格も出ており、実際に使って試しやすい状況です。AI に「小さな断片」ではなく「プロジェクト全体」を任せる方向が進んでいます。

## 🧠 専門的解説（技術詳細・設計・使いどころ）

### 技術的詳細（何が変わったか）

- **Claude Sonnet 5（2026年6月30日）**: Sonnet 4.6 比で推論・ツール利用・コーディング・ナレッジワークが向上。最も「エージェント的」な Sonnet と位置づけ。
- **Claude Code の既定モデル化**: ネイティブ 1M トークンのコンテキストウィンドウを備え、Claude Code の標準として提供。
- **導入価格**: 8月31日まで $2/$10 per Mtok（入力/出力）のプロモ価格。
- 併せて Claude Enterprise 向けに管理者アナリティクス、モデル単位のエンタイトルメント、支出アラートなどの運用機能を追加。

（数値・提供条件は時期により変動するため、正確な最新値は Anthropic 公式で要確認。）

### 仕組み / 背景（なぜこう設計されたか）

コーディングエージェントの性能は「モデルの賢さ」だけでなく「どれだけ関連文脈を一度に保持できるか」に強く依存する。大規模コードベースでは、変更対象と、その依存先・呼び出し元・テスト・設定が広範に散らばる。コンテキストが小さいと、必要な断片を都度検索して継ぎ足す（RAG 的な）運用になり、取りこぼしや不整合が起きやすい。ネイティブ 1M トークンは、この「断片主義」を緩和し、関連ファイル群をまとめて文脈に載せて一貫した推論をさせる方向の設計だ。

トレードオフは明確で、長大な文脈は入力トークン量＝コストとレイテンシを押し上げ、また「入れれば入れるほど精度が上がる」わけではない（無関係情報が多いと注意が薄まる、いわゆる文脈の希釈）。したがって実務では「全部入れる」より「関連の高いものを賢く選んで大きく入れる」ハイブリッドが有効で、リポジトリ地図やツール検索と併用する。エージェント能力（ツール利用・自己修正ループ）の向上は、単発生成より「テストを回して直す」反復ワークフローの信頼性に効く。導入価格は採用障壁を下げ、実運用データを集める狙いとも読める。

### 実務での使いどころ・移行の勘所

大規模・多ファイルのリファクタリング、テスト駆動の修正、レガシー把握など「広い文脈」が効く作業に向く。ただしコンテキストを盛るほどコストが増えるため、無制限に詰め込まず、関連範囲を絞ったうえで大きめに与える運用が費用対効果で有利。CI と組み合わせ、エージェントの変更は必ずテスト/レビューを通す。企業導入では新設の管理者アナリティクス・支出アラートでコストと利用を可視化し、モデルエンタイトルメントでアクセスを制御する。他社のコーディングエージェント/モデルとは、コンテキスト長・ツール利用の堅さ・価格を要件に照らして比較する。

## 📖 用語解説（このトピックとのつながり）

- **Claude Sonnet 5**: Anthropic の新モデル。Claude Code の既定頭脳。本トピックの主役。
- **Claude Code**: Anthropic のコーディングエージェント（CLI/エージェント）。Sonnet 5 を既定採用。
- **コンテキストウィンドウ**: モデルが一度に扱えるトークン量。ネイティブ 1M に拡大。
- **トークン**: テキストを分割した処理単位。コストとコンテキスト量の基準。
- **文脈の希釈**: 無関係情報が多いと注意が分散し精度が落ちる現象。長文脈運用の注意点。

## 影響範囲 / 推奨アクション

大規模コードベースの把握・リファクタリング・テスト駆動修正をエージェントに任せやすくなった。コンテキストは「関連を絞って大きく」与えコストを制御、変更は必ずテスト/レビュー経由に。企業は管理者アナリティクスと支出アラートで利用・コストを可視化する。
