# Angular 22 — 「signal-first」時代へ、Signal Forms とゾーンレス標準化が進む

> トピック: Angular 22（signal-first）｜出典: Angular 公式（angular.dev）｜種別: メジャーフレームワーク更新

## 一行サマリー

Angular 22（2026年6月3日リリース）は「signal-first」を掲げ、Signal ベースの細粒度リアクティビティを中核に据えた世代。前バージョン21でゾーンレス変更検知が新規アプリ標準・Vitest がテストランナー標準・Signal Forms が実験導入され、22 はその路線をさらに前進させる。

## 🔰 初学者向け（何が・なぜ重要か）

Angular は Google が作る大規模向けのフロントエンドフレームワークです。企業の管理画面のように「画面が大きく複雑で、長く保守する」アプリで強みを発揮します。

これまでの Angular は「Zone.js」という仕組みで画面の再描画タイミングを判断していました。たとえるなら「部屋のどこかで物音がしたら、念のため部屋全体を見回して変わった所を探す」やり方です。安全ですが、部屋（アプリ）が大きいほど見回りのコストがかさみます。

新しい主役が **Signal（シグナル）** です。これは「値が変わったら、その値を使っている場所だけにピンポイントで知らせる」仕組み。物音がした場所だけを直接見に行くイメージで、無駄な見回りが消えて速くなります。Angular 22 が掲げる「signal-first」は、この Signal を Angular の設計の中心に据える、という宣言です。フォーム（入力画面）も Signal ベースの新しい「Signal Forms」で書けるようになり、値の変化と画面が素直につながります。

## 🧠 専門的解説（技術詳細・設計・使いどころ）

### 技術的詳細（何が変わったか）

- Angular 21 は 2025年11月20日リリース。ゾーンレス変更検知を新規アプリのデフォルト化、テストランナーを Karma から **Vitest** へ、**Signal Forms** を実験 API として導入、v19 以来 deprecated だった API を削除。
- 加えて Angular MCP Server（AI 連携）、Angular ARIA パッケージ（アクセシビリティ）などを追加。
- **Angular 22 は 2026年6月3日リリース**で「signal-first」世代と位置づけ。約6か月周期のリリースに沿う。細部の API 変更は公式リリースノートで要確認。

### 仕組み / 背景（なぜこう設計されたか）

Angular の変更検知は長らく Zone.js に依存してきた。Zone.js はブラウザの非同期 API（setTimeout, イベント, XHR など）をモンキーパッチで捕捉し、「非同期が起きた＝どこか変わったかも」としてコンポーネントツリーを走査する。この「保守的に全体を確認する」方式は正しさを担保する一方、大規模アプリでは変更のない枝まで走査してオーバーヘッドになる。

Signal は反対の発想を採る。Signal は値そのものが「読まれた場所（依存）」を記録し、値が更新されると依存先だけを無効化して再計算・再描画する。これは Solid や Vue の refs と同系統の「細粒度リアクティビティ」であり、更新の伝播が O(変化に関係する依存数) に近づく。ゾーンレス化は Zone.js を外して Signal の通知だけで再描画を駆動する方向で、バンドルから Zone.js を除ける（サイズ減）副次効果もある。トレードオフは、既存の「暗黙に全体再描画される」前提で書かれたコードや、Zone.js に依存するサードパーティとの互換性。移行にはスキマティクス（自動変換）が用意される。

### 実務での使いどころ・移行の勘所

新規の大規模業務アプリは最初からゾーンレス＋Signal で組むと、パフォーマンス設計がシンプルになる。既存アプリは (1) まずコンポーネント状態を段階的に Signal へ移し、(2) 変更検知戦略を見直し、(3) 最後にゾーンレスへ切り替える、という順が安全。Signal Forms はまだ実験段階を経ての採用となるため、既存の Reactive Forms 資産を一気に置き換えるのではなく、新規画面から試すのが現実的。テストは Karma 廃止に伴い Vitest への移行が必要になる点に注意。

## 📖 用語解説（このトピックとのつながり）

- **Signal**: 値と依存を追跡し、変化時に依存先だけを更新するリアクティブなプリミティブ。本トピックの中核。
- **細粒度リアクティビティ**: 変化に関係する箇所だけを再計算する更新モデル。Signal が実現する原理。
- **Zone.js / ゾーンレス**: 非同期を捕捉して全体走査する従来方式（Zone.js）と、それを外す新方式（ゾーンレス）。
- **Signal Forms**: Signal ベースの新フォーム API。値の変化と検証・UI が素直に連動する。
- **変更検知（change detection）**: 状態変化を画面へ反映する仕組み。Signal 化で走査コストが下がる。

## 影響範囲 / 推奨アクション

- 影響: Angular を採用する業務アプリ全般。パフォーマンス、テスト構成、フォーム実装に波及。
- 推奨: 新規は signal-first＋ゾーンレスで設計。既存は状態→変更検知→ゾーンレスの順で段階移行。Karma→Vitest 移行を計画。Signal Forms は新規画面から試験導入。
