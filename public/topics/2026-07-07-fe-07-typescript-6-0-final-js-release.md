# TypeScript 6.0 — JS 実装最後のメジャー、非推奨と移行フラグで 7.0 へ橋渡し

> トピック: TypeScript 6.0 リリース｜出典: Microsoft TypeScript 公式ブログ｜種別: メジャーバージョンリリース

## 一行サマリー

2026年3月23日にリリースされた TypeScript 6.0 は、従来の JavaScript ベースのコンパイラ codebase による最後のメジャー版で、機能追加より「Go ネイティブの 7.0 へ壊れずに移るための地ならし」に徹する。非推奨の整理・既定値変更・新しい移行フラグを導入し、7.0 で消える古い書き方を先に警告する。

## 🔰 初学者向け（何が・なぜ重要か）

TypeScript 6.0 は、派手な新機能で盛り上がるタイプのアップデートではありません。むしろ「引っ越し前の大掃除」に近い版です。この次の 7.0 で、コンパイラ（コードを点検する仕組み）が別言語（Go）で作り直され大幅に速くなるのですが、その引っ越しをスムーズにするために、6.0 が先に部屋を片付けてくれます。

具体的には、「もう使わないほうがいい古い機能」に警告ランプを点け、「7.0 では動きが変わる箇所」を前もって教えてくれます。つまり 6.0 の警告をきちんと片付けておけば、7.0 に上がるとき慌てずに済む、という設計です。急がば回れで、まず 6.0 を通しておくのが得策、というわけです。

## 🧠 専門的解説（技術詳細・設計・使いどころ）

### 技術的詳細（何が変わったか）

- JS ベースのオリジナル codebase による **最後のメジャー**。以降の新機軸は 7.0（Go 実装）へ。
- 一連の **非推奨（deprecation）** を整理し、7.0 で削除／変更される挙動を 6.0 段階で警告化。
- **既定値の変更**と**新しい移行フラグ**を導入。7.0 の新既定を 6.0 上で先行体験・切替できる。
- 破壊的変更は移行を促す範囲に限定。挙動互換の 7.0 と合わせ「6.0 で警告解消 → 7.0 へ」の順路が公式推奨。
- 2026年3月6日に RC、3月23日に GA。

### 仕組み / 背景（なぜこう設計されたか）

大規模言語プロジェクトが実装基盤を丸ごと差し替える（今回は JS→Go）とき、ユーザー影響を最小化する定石が「バージョンをまたいだ段階移行」である。7.0 で意味論を変えずに実装だけ差し替えるには、7.0 で困る書き方を旧実装のうちに炙り出しておく必要がある。6.0 の非推奨・移行フラグはその「先出し警告」であり、React の StrictMode や各種フレームワークの deprecation warning と同じ発想に立つ。

トレードオフは、フラグ期間中は同一メジャー内で挙動が分岐し、ドキュメントやトラブルシュートが一時的に複雑になること。一方で得られるのは、7.0 という大手術を「日常のマイナー更新の延長」に近づける予測可能性である。6.0 を「機能の版」ではなく「移行の版」と位置付けたのは、この安全性を優先した判断といえる。

### 実務での使いどころ・移行の勘所

- 直近のアクションは明快: **まず 6.0 に上げ、出てくる非推奨・移行フラグの警告をすべて解消する**。これが 7.0 への最短ルート。
- 移行フラグは「7.0 の新既定を今の 6.0 で試す」スイッチとして使える。CI に組み込み、挙動差分を早期に検知する。
- 古い `tsconfig` 設定や、非推奨 API に依存する周辺ツール（旧世代の型駆動ライブラリ等）を棚卸しする好機。
- 大量の型を抱えるプロジェクトほど、6.0 での整理が 7.0 の高速化（型検査 約10倍）の恩恵を素直に受ける前提条件になる。

## 📖 用語解説（このトピックとのつながり）

- **非推奨（deprecation）**: 将来削除予定の機能に付く警告。6.0 はこれを整理し、7.0 での削除に備える。
- **移行フラグ**: 次期メジャーの挙動を現行版で先行有効化するオプション。6.0 の目玉で、7.0 への段差を消す。
- **codebase（実装基盤）**: コンパイラそのもののソース。6.0 まで JS 実装、7.0 から Go 実装。
- **挙動互換**: 実装を変えても言語仕様・型付けの結果は変えない方針。7.0 が守り、6.0 が地ならしする。

## 影響範囲 / 推奨アクション

- 全 TypeScript プロジェクト: 6.0 へ更新 → 警告ゼロ化 → 7.0 RC を試験導入、の順で進める。
- ライブラリ作者: 非推奨 API 依存を解消し、6.0/7.0 双方で通るよう追随。
- 6.0 をスキップして 7.0 直行するのは非推奨（移行警告を取りこぼすため）。
