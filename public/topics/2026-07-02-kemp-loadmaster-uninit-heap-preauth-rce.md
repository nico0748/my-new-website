# Progress Kemp LoadMaster：未初期化ヒープ由来の無認証RCE、悪用試行を確認（CVE-2026-8037）

> トピック: 脆弱性ニュース ｜ 出典: The Hacker News / watchTowr Labs / eSentire TRU ｜ 種別: OSコマンドインジェクション（未初期化メモリ起因）

## 一行サマリー

負荷分散アプライアンス Kemp LoadMaster の API に、ログイン不要で root 権限のコマンド実行に至る致命的欠陥（CVSS 9.8）が判明。原因は入力チェック漏れではなく「文字列エスケープ処理の末尾ヌル欠落＋未初期化malloc」という低レベルなメモリ不備の組み合わせ。6月29日から悪用試行が観測されており、API有効機器は即時更新が必要。

## 🔰 初学者向け（何が・なぜ問題か）

Kemp LoadMaster は、たくさんのアクセスを複数のサーバへ振り分ける「交通整理の信号機」のような装置（ロードバランサ）です。企業システムの入口に置かれ、常にインターネットに面しています。

今回の欠陥は、その装置の「API（外部から操作するための窓口）」にあります。攻撃者は**ログインすらせずに**この窓口へ細工したデータを送るだけで、装置を管理する最高権限（root）で好きな命令を実行できてしまいます。入口の信号機を乗っ取られれば、そこを通る全通信の盗み見や、社内ネットワークへの侵入の足場にされます。

面白い（そして怖い）のは原因です。よくある「危険な文字を消し忘れた」タイプではなく、プログラムが文字列を扱う際の**とても細かなミス**——「文字列の終わりの目印（ヌル文字）を書き忘れた」ことと「新しく確保したメモリ領域を掃除せず、前のゴミデータが残ったまま使った」ことの合わせ技です。この2つが噛み合うと、プログラムは"想定外の隣のメモリ"まで読み進めてしまい、そこに攻撃者が仕込んだ命令が混ざり込む、という仕組みです。

## 🧠 専門的解説（技術詳細・メカニズム・検知対応）

### 技術的詳細（種別・前提条件）

- **CVE**: CVE-2026-8037
- **CVSS**: 9.8（Critical、ZDI採番）。一部レポートでは 9.6 表記あり（要確認）。
- **種別（CWE）**: OSコマンドインジェクション（CWE-78）に分類。根因は未初期化メモリの使用（CWE-457）と不適切なヌル終端処理（CWE-170：不適切なヌル終端）。古典的な入力サニタイズ不備ではない点が特徴。
- **前提条件**: 対象機器で **API 機能が有効**であること。無認証（pre-auth）でAPIに到達できれば成立。
- **影響バージョン**: Kemp LoadMaster GA 7.2.63.1 以前、および LTSF 7.2.54.17 以前（APIが有効な場合）。
- **修正版**: Progress が該当ビルドで修正。`malloc` を `calloc`（ゼロ初期化）へ変更し、エスケープ出力にヌル終端を追加。
- **悪用状況**: eSentire の Threat Response Unit が悪用試行を確認、活動開始は2026年6月29日。

### メカニズム（攻撃が成立する仕組み）

※本節は**概念レベル**。武器化手順・PoC・正確なオフセットは記載しない。

1. **エスケープ関数の二つの実装欠陥**: 引用符をエスケープする内部関数 `escape_quotes()` が、(a) `malloc()` で確保したバッファを初期化せず（＝前に使われたヒープの残骸データが残る）、(b) 生成したエスケープ文字列の末尾に**ヌル終端を書かない**。
2. **文字列の"終わり"が消える**: C言語の文字列は「ヌル文字までが1つの文字列」というルールで動く。終端が無いと、後続の `sprintf` 等が文字列の終わりを認識できず、**隣接するヒープメモリまで読み進めてしまう**（境界外読み取り）。
3. **攻撃者がヒープを"仕込む"**: `malloc` は解放済みチャンクを再利用しうるため、攻撃者は事前に別リクエストでコマンド断片をヒープ上に撒く（ヒープスプレー）。未初期化のバッファ隣に、攻撃者制御データが居座る状況を作る。
4. **コマンド行への混入**: エスケープ済みポインタが `sprintf`／`system()` 連鎖で `validuser` コマンド行の組み立てに使われる際、終端欠落により sprintf が撒かれたデータまで読み込み、**攻撃者制御の内容を含むコマンド行**が完成する。
5. **root RCE への橋渡し**: そのコマンド行が `system()` で実行される。LoadMaster の当該処理が高権限で動くため、無認証のまま root 相当の任意コマンド実行に至る。攻撃者が握るプリミティブは「隣接ヒープに任意データを置く力」＋「終端欠落で読み込ませる力」で、これがコマンド注入に橋渡しされる。
6. **一般原理**: メモリ安全性の欠陥（未初期化メモリ・終端不備）が、上位層のコマンド構築ロジックを汚染して RCE 化する典型。入力検証だけを見ていると見落とす"低レベル×高レベルの合流点"に脆弱性が生まれる。

### 検知・暫定緩和・恒久対策

- **検知の着眼点**: LoadMaster API エンドポイントへの無認証アクセス、`validuser` 系処理に絡む異常なパラメータ長・非印字バイト混入、API 経由の予期しないコマンド実行痕跡・子プロセス生成。
- **暫定緩和（パッチ前）**: 不要なら **API 機能を無効化**。API 管理面のインターネット直接公開を停止し、管理ネットワーク／許可IPに限定。
- **恒久対策**: 修正ビルド（`calloc` 化＋ヌル終端修正を含む版）への更新。
- **運用示唆**: エッジ機器の管理API露出は最小化が原則。ロードバランサ／ゲートウェイ類は "境界にあるが故に境界防御が効かない" ため、機器自体の露出面積管理が要。

## 📖 用語解説（このニュースとのつながり）

- **ロードバランサ（LoadMaster）**: アクセスを複数サーバへ分散する境界機器。乗っ取られると通過通信の傍受・内部侵入の踏み台になる。
- **未初期化メモリ（CWE-457）**: 確保した領域を掃除せず使うこと。今回は前の"ゴミデータ"に攻撃者データが混じり、悪用の起点になった。
- **ヌル終端の欠落（CWE-170）**: 文字列の終わりの目印を書き忘れる不備。sprintf が隣のメモリまで読む"境界外読み取り"を誘発した。
- **ヒープスプレー**: 攻撃者が狙った内容をヒープ上に大量配置する手法。未初期化バッファの隣に命令片を置くために使われる。
- **pre-auth（無認証）RCE**: ログイン前に成立する遠隔コード実行。CVSS 9.8 の主因で、境界機器では特に致命的。

## 影響範囲 / 推奨対応

API を有効化した Kemp LoadMaster（GA 7.2.63.1 以前 / LTSF 7.2.54.17 以前）を境界に配置している全組織。悪用試行が観測済みのため対応は急務。(1) API 有効・外部到達可否の確認、(2) 修正ビルドへの更新、(3) 更新前は API 無効化と管理面の遮断、(4) 6月29日以降の API アクセスログ精査。
