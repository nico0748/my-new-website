# FastStone Image Viewer に複数のファイル解析の脆弱性（境界外書き込みでコード実行の恐れ）

> トピック: 脆弱性ニュース ｜ 出典: JVN / JPCERT-CC 注意喚起 / Tenable・ZDI ｜ 種別: ファイル解析時の境界外書き込み・スタックバッファオーバーフロー（CWE-787/121）

## 一行サマリー

国内でも広く使われる画像ビューア「FastStone Image Viewer」に、細工した画像ファイル（GIF/PSD/CUR/PNG 等）の解析時に境界外書き込みやスタックオーバーフローを起こす複数の脆弱性があり、ファイルを開くだけで任意コード実行に至る恐れがある。JVN/JPCERT 経由で公表。最新版への更新と不審ファイルの取り扱い注意が必要（個別CVE・CVSSは要確認）。

## 🔰 初学者向け（何が・なぜ問題か）

「FastStone Image Viewer」は、写真や画像をサクサク見られる無料の定番ソフトです。今回の問題は、このソフトが画像ファイルを“読み解くとき”の処理に複数の欠陥が見つかったことです。

画像ファイルには「縦◯ピクセル・横◯ピクセル」「色の情報はここから」といった設計図（ヘッダ情報）が書かれています。ソフトはそれを信じて、必要な広さの“作業スペース（メモリ）”を用意して中身を展開します。ところが攻撃者が設計図にウソの数値を書き込んだ画像を作ると、ソフトは用意したスペースをはみ出して書き込んでしまいます。たとえるなら、A4 が入る前提の引き出しに、ラベルだけ「A4」と偽った巨大な紙を押し込んで、引き出しの周りの大事な書類まで押し潰してしまうイメージです。

この“はみ出し”をうまく誘導されると、ソフトの動きを乗っ取られ、最終的にあなたの PC で攻撃者の命令が実行される恐れがあります。怖いのは、メール添付やダウンロードした「画像をダブルクリックして開く」だけで成立しうること。出所不明の画像ファイルは安易に開かないこと、ソフトを最新版にすることが対策です。

## 🧠 専門的解説（技術詳細・メカニズム・検知対応）

### 技術的詳細（種別・前提条件）

- 種別: 画像ファイル解析時の境界外書き込み（Out-of-Bounds Write, CWE-787）／スタックバッファオーバーフロー（CWE-121）等のメモリ破壊。GIF・PSD の OOB write、CUR（BITMAPINFOHEADER の BitCount）でのスタックオーバーフロー、PNG tRNS チャンク解析でのスタック破壊などが報告。
- CVSS: 要確認（複数 CVE を集約。CVE 番号も JVN/ZDI 告知を要確認）。
- 前提条件: 攻撃者が用意した細工画像ファイルを、被害者が当該ソフトで「開く／プレビューする」こと（ユーザー操作必須のクライアントサイド攻撃）。ネットワーク越しの自動悪用ではなく、添付・配布・水飲み場経由でファイルを開かせる手口が中心。
- 影響: FastStone Image Viewer の脆弱バージョン（過去報告では 7.5 以下、直近は 8.3.0.0 系の複数解析バグも報告。範囲は要確認）。
- 修正版: ベンダ最新版へ更新（バージョンは JVN/ベンダ告知を要確認）。
- 悪用状況: 公開時点で広範な実環境悪用の報告は確認されていない（要確認）。ただしクライアント側メモリ破壊は標的型で武器化されやすい。

### メカニズム（攻撃が成立する仕組み）

注: CVE 詳細が集約・調整中のため「型の説明」にとどめ、武器化手順・オフセット・PoC は記載しない。

1. **ファイルヘッダ＝信頼できない入力**: 画像のヘッダ（寸法・ビット深度・チャンク長など）は外部由来＝攻撃者が自由に書ける値。解析器はこれを基にバッファ確保や反復回数を決める。ここが信頼境界。
2. **長さ・サイズ検証の欠落**: 解析時にヘッダ値と実データの整合（境界チェック）を十分に行わないと、宣言値と実バッファ容量に齟齬が生じる。
3. **境界外書き込みプリミティブの獲得**: 齟齬のまま展開・コピーを行うことで、割り当て領域の外（スタック上の戻りアドレス周辺やヒープのメタデータ）へ書き込む OOB write／スタックオーバーフローが成立する。これが攻撃者の握る基本プリミティブ。
4. **制御フロー奪取への橋渡し**: スタック上の制御データ（戻りアドレス等）やヒープ管理構造を上書きし、最終的にプロセスの実行フローを攻撃者の用意したコードへ誘導＝任意コード実行に至りうる（実際には ASLR/DEP 等の緩和回避を要し、別技法との連鎖が一般的）。
5. **一般原理**: 「外部由来の長さ・寸法値を信じてバッファ操作する」ファイルパーサは、メモリ破壊の典型攻撃クラス。多様なフォーマット（GIF/PSD/CUR/PNG）を扱うほど解析経路が増え、検証漏れの面積が広がる。

### 検知・暫定緩和・恒久対策

- 検知の着眼点: 画像ビューアプロセスの異常終了（クラッシュ）多発、当該プロセスからの想定外の子プロセス生成、メール/共有経由で配布された不審な拡張子・偽装画像。EDR でのビューアからのシェル起動。
- 暫定緩和: 出所不明の画像を当該ソフトで開かない、関連付けを信頼できるビューアに変更、サンドボックス／仮想環境での閲覧、メールゲートウェイでの危険ファイル検疫。
- 恒久対策: FastStone Image Viewer を最新版へ更新（JVN/ベンダ告知を確認）。組織は配布ソフト一覧から該当バージョンを洗い出し、更新または利用停止を判断。
- 運用示唆: クライアント側パーサ脆弱性は「ファイルを開かせる」ソーシャル手口と結合する。添付ファイル運用ポリシーと従業員教育を併せて見直す。

## 📖 用語解説（このニュースとのつながり）

- **ファイル解析（パーサ）**: 画像等の中身を読み解く処理。本件の脆弱箇所で、ヘッダ値を信じすぎたのが原因。
- **境界外書き込み（OOB write, CWE-787）**: 割り当て範囲外への書き込み。メモリ破壊→コード実行の起点。
- **スタックバッファオーバーフロー（CWE-121）**: スタック上の領域をはみ出す古典的欠陥。戻りアドレス書き換えに直結。
- **クライアントサイド攻撃**: 利用者がファイルを開く操作を必要とする攻撃。配布・添付と相性が良い。
- **JVN/JPCERT 注意喚起**: 国内の脆弱性公表・周知の枠組み。本件の出典。

## 影響範囲 / 推奨対応

FastStone Image Viewer を業務・個人で利用する Windows 環境が対象。JVN／ベンダ告知に従い最新版へ更新し、更新までは出所不明の画像を当該ソフトで開かない運用とする。組織は配布ソフトのバージョンを棚卸しし、メール・ファイル共有経由の偽装画像に対する検疫と注意喚起を強化する。個別 CVE・CVSS・影響範囲は確定情報を要確認。
