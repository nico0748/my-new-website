# WebMCP が Chrome 149 でオリジントライアル — Web ページが AI エージェント向けに「ツール」を公開する API

> トピック: WebMCP（Chrome 149 オリジントライアル）｜出典: Chrome for Developers・InfoQ｜種別: Web 標準 / エージェント

## 一行サマリー

Web ページ側が「AI エージェントが呼び出せる名前付きの関数（ツール）」をブラウザに登録できる WebMCP が、Chrome 149 でオリジントライアルとして始まった。`navigator.modelContext.registerTool` API で、フォーム操作・ナビゲーション・状態管理などを型付きツールとして宣言でき、エージェントは画面をピクセル単位で推測する代わりに、ページが公開した意図の明確なツールを直接叩ける。

## 🔰 初学者向け（何が・なぜ重要か）

最近は「AI エージェントが人の代わりにサイトを操作する（予約を取る、買い物をする）」使い方が広がっている。ところがエージェントは、人間のように画面を見てボタンの位置を推測しながらクリックするしかなく、見た目が少し変わると失敗したり、意図しない操作をしたりしやすい。

WebMCP は、その関係を逆にする。サイト側が「うちのページでは『商品を検索する』『カートに入れる』という操作を、こういう入力で呼べますよ」と、エージェント向けの取扱説明書（ツール一覧）をあらかじめ用意しておく仕組み。エージェントは推測をやめ、用意されたツールを名前で呼ぶだけでよくなる。

たとえるなら、これまで来客が家じゅうを探して物を使っていたのを、玄関に「電気はこのスイッチ、水はこの蛇口」とラベル付きの操作盤を置いてあげるようなもの。安全で確実、そして誤操作が減る。

## 🧠 専門的解説（技術詳細・設計・使いどころ）

### 技術的詳細（何が変わったか）

- **提供**: Chrome 149 で実験的オリジントライアル開始。Google と Microsoft のエンジニアが W3C Web Machine Learning Community Group で策定、初公表は 2026 年 2 月 10 日。
- **中核 API**: ブラウザに `navigator.modelContext.registerTool` が追加され、ページが名前付き・型付きの関数を登録。エージェントはそれを直接呼べる。
- **2 つの記述方法**:
  - **命令的 API**: 標準の JavaScript で、フォーム入力・ナビゲーション・状態管理などのツールを定義。
  - **宣言的 API**: 標準の HTML フォーム要素に注釈を付けるだけで WebMCP ツールを生成。
- **採用状況（要確認扱い）**: Gemini in Chrome が対応予定とされ、Expedia・Booking.com・Shopify・Credit Karma・Target 等が実験中と報じられている。

### 仕組み / 背景（なぜこう設計されたか）

従来のブラウザ自動化エージェントは「アクセシビリティツリーやスクリーンショットからボタンを推測してクリックする」アプローチで、DOM の変化・A/B テスト・動的レンダリングに弱く、誤操作リスクも高かった。WebMCP は、サーバー↔クライアント間の MCP（Model Context Protocol）の考え方を **ページ↔エージェント間に持ち込み**、ページが「呼べる操作」を構造化して公開する。

設計の勘所は「意図の明示」だ。ページ制作者が入力スキーマ（引数の型）付きでツールを宣言するため、エージェントは何を渡せばよいかを推測せずに済み、ページ側も「どの操作を許すか」を制御できる。命令的 API は複雑なアプリ状態を扱え、宣言的 API は既存のフォームに注釈を足すだけで済むため導入コストが低い。トレードオフは、(1) 公開したツールが新たな攻撃面になり得るため権限・確認フローの設計が要ること、(2) 現状オリジントライアルで仕様は流動的なこと、(3) エージェントとサイト双方の対応が揃って初めて価値が出る「両面市場」であること。

### 実務での使いどころ・移行の勘所

EC の検索・カート・チェックアウト、予約、フォーム入力など「エージェントに任せたい定型操作」を持つサイトが対象。移行の勘所は、(1) まず宣言的 API で主要フォームに注釈を足して低コストに試す、(2) 複雑な操作は命令的 API でツール化しつつ、破壊的操作には確認・認可を挟む、(3) オリジントライアル段階なので本番前提にせず、仕様変更に追従できる薄い実装から始める。セキュリティレビュー（公開ツールの権限範囲）を必ずセットで行うこと。

## 📖 用語解説（このトピックとのつながり）

- **WebMCP**: ページが AI エージェント向けにツールを公開する Web 標準の提案。Chrome 149 で試験導入。
- **navigator.modelContext.registerTool**: ページが名前付き・型付き関数を登録するブラウザ API。
- **MCP（Model Context Protocol）**: LLM/エージェントに外部ツールを繋ぐプロトコル。WebMCP はその Web ページ版。
- **オリジントライアル**: 特定サイトが本番で新機能を試せる Chrome の限定公開制度。仕様は未確定。
- **宣言的 / 命令的 API**: HTML 注釈で作る方式と JS で定義する方式。導入コストと表現力のトレードオフ。

## 影響範囲 / 推奨アクション

- エージェント経由の操作を確実・安全にしたいサイトは、主要操作の WebMCP ツール化を検討。
- まず宣言的 API で低コスト試験。破壊的操作は認可・確認フローを必須に。
- オリジントライアル段階のため本番依存は避け、公開ツールのセキュリティレビューを同時に実施。
