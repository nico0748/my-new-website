# WordPressプラグイン Gravity SMTP に未認証情報漏えい（CVE-2026-4020）— APIキー大量窃取、実環境で悪用中

> トピック: 脆弱性ニュース ｜ 出典: BleepingComputer / CrowdSec / SentinelOne ｜ 種別: 認可チェック不備（permission_callback欠陥）に起因する未認証の機微情報露出（CWE-862/CWE-200系）

## 一行サマリー

約10万サイトで使われるメール送信プラグイン Gravity SMTP に、ログイン不要で Amazon SES / Google / Mailjet / Resend / Zoho などの**APIキーやOAuthトークンを丸ごと盗める**情報漏えい脆弱性 CVE-2026-4020 が見つかり、すでに大規模な実環境悪用（約6週間で1,700万回の攻撃ブロック）が観測されている。該当バージョンを使っている場合は更新だけでなく**全鍵のローテーション**が必要となる。

## 🔰 初学者向け（何が・なぜ問題か）

WordPressで作ったサイトは、問い合わせ返信や会員登録の確認メールなど、いろいろなメールを自動送信します。ただWordPress標準のメール送信は届きにくいため、**Gravity SMTP** のようなプラグインを入れて、Amazonやグーグルなどの「ちゃんと届くメール配信サービス」を経由させるのが一般的です。このとき、配信サービスにログインするための**鍵（APIキー）**をプラグイン内に保存します。

今回の問題は、このプラグインに用意された**点検用の裏口（テスト用のURL）**が、誰でも・ログインなしでアクセスできる状態になっていたことです。本来「管理者だけが使える」ように鍵をかけておくべき扉に、鍵がかかっていませんでした。その扉を開けると、サイトの構成情報に加えて、保存されていた**配信サービスの鍵そのもの**が一覧で返ってきてしまいます。

たとえるなら、店の従業員専用のバックヤードのドアが開けっ放しで、しかもその中の棚に「取引先の金庫の合鍵」が無造作に置いてあった、というイメージです。攻撃者はその合鍵（APIキー）を持ち出し、あなたの名義で大量のスパムメールを送ったり、メール配信サービスの利用料を勝手に使い込んだり、取引先サービス側のデータにアクセスしたりできます。被害がサイト内にとどまらず、外部の連携サービスにまで広がる点が厄介です。

## 🧠 専門的解説（技術詳細・メカニズム・検知対応）

### 技術的詳細（種別・前提条件）

- **CVSS**: 5.3（ベンダー評価）／7.5（NVD評価）。評価は割れているが、漏えいする情報の性質（**有効なAPIキー・OAuthトークン**）を踏まえると実害は深刻。
- **CWE分類**: 認可チェックの欠如（CWE-862 Missing Authorization）に起因する機微情報の露出（CWE-200）。
- **前提条件**: Gravity SMTP **2.1.4以前**が稼働し、サードパーティのメール配信連携（SES/Google/Mailjet/Resend/Zoho等）が設定済みであること。攻撃に認証は不要。
- **漏えい内容**: 各配信サービスのライブAPIキー／OAuthトークン、WordPressバージョン、有効プラグイン一覧、サーバー情報、データベースのメタデータ等を含む大規模なシステムレポート。
- **修正版・対応**: Gravity SMTP **2.1.5以降**へ更新。脆弱バージョンで連携設定済みのサイトは**侵害前提**で全鍵・トークンをローテーション。
- **悪用状況**: **実環境で活発に悪用中**。脆弱性は2026年3月31日公開、CrowdSecが5月22日に検知対応、最初の実悪用が5月27日に観測、6月1日時点で412の攻撃元IP、約6週間で1,700万回の攻撃をブロック。

### メカニズム（攻撃が成立する仕組み）

以下は概念レベルの説明であり、武器化を助ける具体は最小限にとどめる。

1. **信頼境界の所在**: WordPressのREST APIエンドポイントは、各ルート登録時に `permission_callback` で「誰がこの機能を呼べるか」を定義する。ここが**認可の最前線**であり、機微情報を返すエンドポイントは管理者権限などに限定すべき。
2. **認可チェックの欠陥**: 問題のテスト用エンドポイント（`/wp-json/gravitysmtp/v1/tests/mock-data` 系）の `permission_callback` が**無条件で `true` を返す**実装になっていた。これにより認証状態に関わらず誰でも到達でき、認可の信頼境界が事実上無効化される。
3. **機微情報の露出**: 認証なしのHTTP GET（適切なクエリパラメータ付与）でエンドポイントを叩くと、プラグインが保持する設定情報＝**外部サービスの有効な資格情報**を含むレポートが応答として返る。攻撃者はメモリ破壊などの高度な手法を一切使わず、単純なリクエストだけで価値の高い秘密情報を取得できる。
4. **被害の外部波及**: 窃取されたAPIキー/トークンは、サイト本体ではなく**連携先サービス**（メール送信枠、クラウドメールAPI等）の悪用に直結する。なりすまし送信、スパム/フィッシング配信、課金枠の枯渇、連携サービス内データへのアクセスへ発展する。

脆弱性クラスとしては「機微データを返すAPIに認可チェックが欠落」という、Webアプリで最も多い欠陥型（Broken Access Control）に該当する。メモリ系の難しさはなく、**発見即・大量自動悪用**されやすいのが特徴。

### 検知・暫定緩和・恒久対策

- **検知の着眼点**: Webアクセスログで `gravitysmtp/v1/tests` 系パス・`/wp-json/` 配下への**認証なしGETの急増**、未知IPからの当該エンドポイントアクセス、連携サービス側での見覚えのない送信・API利用の急増。WAF/監視で当該ルートへのアクセスをアラート化。
- **暫定緩和（パッチ前）**: 直ちにプラグインを無効化、もしくはWAF/サーバ設定で当該RESTパスを外部から遮断。連携サービスのコンソールでAPIキーの利用状況を確認。
- **恒久対策**: Gravity SMTP 2.1.5以降へ更新。**更新だけでは漏えい済みの鍵は無効化されない**ため、SES/Google/Mailjet/Resend/Zoho等の**全APIキー・OAuthトークンを失効・再発行**。WordPress本体・他プラグインの棚卸しと最小権限化。
- **運用示唆**: プラグインに秘密情報を持たせる構成は、認可欠陥1つで連携先まで波及する。秘密はできる限り環境変数や秘密管理側に寄せ、RESTルートの `permission_callback` を定期監査する。

## 📖 用語解説（このニュースとのつながり）

- **SMTPプラグイン / Gravity SMTP**: WordPressのメール送信を外部配信サービス経由で確実化するプラグイン。本件＝ここに保存した鍵が盗まれる。
- **APIキー / OAuthトークン**: 外部サービスを利用するための認証用の秘密鍵。意味＝「取引先サービスの合鍵」。本件＝漏えいの主目標。
- **REST API / permission_callback**: WordPressの機能をHTTPで提供する仕組みと、その「呼び出し可否」を判定する関数。本件＝これが常に許可を返していたのが根本原因。
- **Broken Access Control（CWE-862）**: 本来制限すべき機能に認可チェックが無い欠陥。本件＝未認証で機微情報に到達できた理由。
- **鍵のローテーション**: 漏えい時に鍵を失効・再発行する運用。本件＝更新後に必須の対応。

## 影響範囲 / 推奨対応

影響を受けるのは、Gravity SMTP 2.1.4以前を使い、外部メール配信サービスと連携している全WordPressサイト（約10万サイト規模）。推奨対応は、(1) 2.1.5以降へ即時更新、(2) 連携サービス（SES/Google/Mailjet/Resend/Zoho等）の**全APIキー・OAuthトークンの失効と再発行**、(3) アクセスログでの当該RESTパスへの不審アクセス調査、(4) 連携サービス側での不正送信・課金の確認、(5) WAFで当該エンドポイントへの外部アクセス遮断。
