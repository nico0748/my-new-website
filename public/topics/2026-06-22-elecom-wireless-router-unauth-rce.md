# エレコム製無線LANルーター等に未認証OSコマンドインジェクション（CVE-2026-42062 / CVE-2026-40621）

> トピック: 脆弱性ニュース ｜ 出典: JVN#03037325 / エレコム セキュリティ情報 / INTERNET Watch ｜ 種別: OSコマンドインジェクション（CWE-78）／ 認証の欠如（CWE-306）

## 一行サマリー
エレコム製の無線LANルーター・無線アクセスポイント計21製品に複数の脆弱性。製品にアクセスできる攻撃者が**認証なしで任意のOSコマンドを実行**できる欠陥（CVE-2026-42062）と、**認証なしで製品を操作**できる認証回避（CVE-2026-40621）を含む。対策済みファームウェアへの更新が必須。国内で広く使われる機器のため影響が大きい。

## 🔰 初学者向け（何が・なぜ問題か）
家庭や小さなオフィスでインターネットの"玄関"になっているのが **無線LANルーター（Wi-Fiルーター）** です。今回はエレコム製の複数機種に、攻撃者にとって都合の良すぎる欠陥が見つかりました。

ひとつ目（CVE-2026-42062）は **OSコマンドインジェクション**。ルーターの設定画面などにある入力欄は、本来「数字や文字を受け取るだけ」のはずですが、ここに"命令文"を混ぜて送ると、ルーターがそれを命令と勘違いして実行してしまいます。たとえると、宅配の備考欄に「鍵を開けて家中の物を運び出して」と書いたら配達員が本当にやってしまう、という状態です。しかも**ログイン不要**で成立します。

ふたつ目（CVE-2026-40621）は **認証の欠如（認証回避）**。本来パスワードが必要な操作なのに、特定のやり方でアクセスすると**ログインを飛ばして**機器を操作できてしまいます。受付を素通りして社長室に入れるようなものです。

ルーターを乗っ取られると、通信の盗み見・改ざん、偽サイトへの誘導（DNS書き換え）、他者への攻撃の踏み台化、ボットネット組み込みなどに悪用されます。国内で普及している機器のため、家庭・SOHO問わず影響が広いのが特徴です。

## 🧠 専門的解説（技術詳細・メカニズム・検知対応）

### 技術的詳細（種別・前提条件）
- **CVE**: CVE-2026-42062（認証なしの任意OSコマンド実行 / OSコマンドインジェクション CWE-78）、CVE-2026-40621（認証なしで製品を操作 / 認証の欠如 CWE-306）。※JVN#03037325 には他の脆弱性も併記。
- **対象製品（一部）**: WRC-XE5400GSA-G v1.13以前、WRC-BE72XSD-B / -BA v1.1.1以前、WRC-BE65QSD-B v1.1.0以前、WRC-W702-B v1.1.0以前、WAB-BE187-M v1.1.10以前、WAB-BE72-M / WAB-BE36-M / WAB-BE36-S v1.1.3以前 など計21製品（無線LANルーター・無線AP）。
- **CVSS**: 要確認（JVN/エレコム公開情報参照）。未認証RCE相当のため実質Critical級。
- **悪用状況**: 本CVEの実環境悪用は要確認。ただしSOHO/家庭用ルーターの未認証RCEはボットネット（Mirai系等）に取り込まれやすく、公開直後から探索が増える傾向。
- **前提条件**: 「当該製品にアクセスできる」こと。管理画面がLAN内のみか、設定によりWAN側・無線経由で到達可能かで露出が変わる（インターネット直結・リモート管理有効だと危険度が大きく上昇）。

### メカニズム（攻撃が成立する仕組み）
1. **信頼境界**: ルーターのWeb管理I/Fやサービスは、ユーザー入力（HTTPパラメータ等）を受け取り、内部でOSコマンド（設定反映・診断ツール起動など）を呼ぶ実装が多い。ここが「ユーザー入力＝信頼できないデータ」と「シェルへ渡すコマンド文字列」の境界。
2. **OSコマンドインジェクション（CVE-2026-42062の核）**: 入力値をサニタイズ（`;` `|` `&` `$()` 等のシェルメタ文字の無害化）やallowlist検証せずにシェルへ連結すると、攻撃者は本来の引数に**追加コマンドを連結**できる。これにより「ルーターOS上の任意コマンド実行」というプリミティブを握る。
3. **認証回避との合わせ技（CVE-2026-40621）**: 認証の欠如により、本来ログインが必要なエンドポイント/操作に**未認証で到達**できる。コマンドインジェクション可能なエンドポイントが認証回避と重なると、"未認証 → 任意コマンド実行 → 機器完全掌握"が一直線に成立する。
4. **最終被害への橋渡し**: 任意OSコマンド実行は、設定改ざん（DNS/ファイアウォール）、永続化（不正プロセス/ファーム改変）、トラフィック傍受、踏み台化・ボット化へ橋渡しされる。家庭・SOHO機器は監視が薄く、侵害が長期に気付かれにくい。

> ※ 一般原理: OSコマンドインジェクションは「外部入力をシェルコマンド文字列へ未検証連結」で生じる（CWE-78）。対策の要諦はシェルを介さないAPI使用・引数配列渡し・厳格なallowlist。認証の欠如（CWE-306）はエンドポイント単位の認可チェック漏れ。具体的な悪用リクエストやパラメータは記載しない（概念レベル）。

### 検知・暫定緩和・恒久対策
- **恒久対策**: エレコムが提供する**対策済みファームウェアへ更新**（各機種の最新版へ）。
- **暫定緩和**: WAN側からの管理アクセス（リモート管理/WAN側Web UI）を無効化、管理I/FはLAN内限定に。既定パスワードの変更、UPnP/不要サービスの停止、無線管理アクセスの制限。インターネットに管理画面を晒さない。
- **検知の着眼点**: 管理画面への未認証アクセス試行、設定（DNS/管理者アカウント/ファイアウォール）の不審な変更、外部への予期しない通信（C2/スキャン）、ルーターの再起動多発や性能劣化。ISP/監視機器側でのアウトバウンド異常監視。
- **運用示唆**: SOHO/家庭機器も資産として棚卸し・更新管理する。EOL機種は買い替えを検討。

## 📖 用語解説（このニュースとのつながり）
- **OSコマンドインジェクション（CWE-78）**: 入力に紛れたコマンドを機器のOSが実行してしまう脆弱性。本件CVE-2026-42062の本質で、ルーター乗っ取りに直結。
- **認証の欠如 / 認証回避（CWE-306）**: 本来必要なログイン確認が抜けている欠陥。本件CVE-2026-40621で、未認証アクセスを許す。
- **シェルメタ文字**: `; | & $( )` などコマンドを区切る/起動する特殊文字。無害化漏れがインジェクションの原因。
- **ボットネット（例: Mirai系）**: 乗っ取った多数のIoT/ルーターを束ねDDoS等に使う攻撃網。未認証RCEのルーターは格好の標的。
- **リモート管理（WAN側管理）**: インターネット側から設定画面に入れる機能。有効だと攻撃面が激増するため無効化が推奨。

## 影響範囲 / 推奨対応
国内で広く使われるエレコム製Wi-Fiルーター/無線AP計21製品が対象。該当機種の利用者は**最新ファームへ即更新**し、WAN側管理の無効化・管理I/FのLAN限定・既定パスワード変更を実施する。設定改ざんやDNS書き換えの有無を点検する。
