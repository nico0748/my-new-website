# 三菱電機製家電 CVE-2026-5667 — 固定SSID＋固定パスワードのハードコード、近隣からWi-Fi接続で機器情報取得

> トピック: 脆弱性ニュース｜出典: JVN (JVNVU#99620284) / 三菱電機 PSIRT / Security NEXT｜種別: ハードコードされた認証情報の使用（CWE-798）

## 一行サマリー

三菱電機製の複数の Wi-Fi 対応家電に、製品へ固定で埋め込まれた SSID とパスワード（ハードコード認証情報、CWE-798）が存在する。購入後にルータへ一度も接続していない、または工場出荷状態へ戻して再設定せず放置している場合、電波が届く範囲の第三者が固定の資格情報で機器へ接続し、運転状態・設定温度・室内温度などの機器データを取得できる。JVN が 2026-06-15 公表、ファームウェア更新と適切な初期設定が必要。

## 🔰 初学者向け（何が・なぜ問題か）

最近のエアコンや家電は Wi-Fi につながり、スマホアプリで操作できます。その「初期接続」のために、製品の電源を入れると、まず家電自身が小さな Wi-Fi の電波（アクセスポイント）を出して、スマホをそこへつないで設定する仕組みがよくあります。

問題は、このときの接続用の名前（SSID）とパスワードが、製品にあらかじめ固定で書き込まれていて、しかも同じ型なら全部同じだった点です。つまり「合鍵が最初から全戸共通」のような状態です。本来は、設定が終わって家庭の Wi-Fi につながれば、この共通の入口は使われなくなります。

ところが、買ったままで一度も家の Wi-Fi につないでいない、あるいは初期化したまま放置していると、その共通の入口がずっと開いたままになります。すると、電波が届く範囲（隣の部屋や家の前を通る人）にいる他人が、固定のパスワードでその家電に接続し、「今エアコンが動いているか」「設定温度は何度か」「室温は何度か」といった情報をのぞけてしまいます。

直接おうちのネットに入られるわけではありませんが、生活パターン（在宅かどうか等）が読まれかねないため、適切に設定して入口を閉じることが大切です。

## 🧠 専門的解説（技術詳細・メカニズム・検知対応）

### 技術的詳細（種別・前提条件）

- **CVE**: CVE-2026-5667 / **JVN**: JVNVU#99620284 / **CVSS**: 要確認（JVN/ベンダー情報を参照。物理的な無線到達が前提のため一般に Medium 相当）
- **CWE**: CWE-798（ハードコードされた認証情報の使用）
- **影響コンポーネント**: 製品の Wi-Fi 初期接続用アクセスポイント（固定SSID＋固定パスワード）
- **根本原因**: 製品ファームウェアに固定の SSID とパスワードが埋め込まれており、未設定/初期化放置の状態ではこの資格情報で誰でも機器へ接続できる。
- **影響条件（成立の前提）**:
  1. 製品の Wi-Fi を有効にした状態で、購入後一度もルータへ接続していない、または
  2. 工場出荷状態に戻した後、再設定せず放置している場合。
- **想定される被害**: 攻撃者は機器の Wi-Fi 電波到達範囲内（近隣の部屋・通りすがり）から接続し、運転状態・設定温度・室内温度などの機器データを取得可能。
- **対象**: 三菱電機製の複数の家電製品/型番（広範）。詳細は JVN・三菱電機 PSIRT を参照。
- **前提条件**: 攻撃者が物理的に無線到達範囲内にいること（リモートからの広域攻撃ではない）。

### メカニズム（攻撃が成立する仕組み）

※以下は脆弱性クラスの一般原理に基づく概念説明。具体的な資格情報や手順は記載しない。

1. **共有秘密の埋め込み（信頼の前提崩壊）**: 認証は「攻撃者が知り得ない秘密」を前提に成立する。ところがSSID/パスワードが製品に固定で埋め込まれ、同型で共通だと、その秘密は「製品を入手・解析すれば誰でも知り得る公知の値」に堕する。秘密が秘密でなくなることで認証が形骸化する。
2. **初期状態の窓が開いたまま残る**: 本来は家庭Wi-Fiへの接続設定が完了すれば初期接続用APは役割を終える。しかし未設定/初期化放置だと、この共通資格情報のAPが稼働し続け、攻撃の窓が開いたまま残存する。
3. **無線到達による接続**: 攻撃者は固定の資格情報を用い、電波到達範囲から機器の初期接続APへ接続する。これにより機器が提供する情報取得インターフェースへアクセスできる。
   - 脆弱性クラスの一般原理: ハードコード認証情報（CWE-798）は IoT の典型的弱点。資格情報がデバイス固有でなく共通である点、利用者が変更できない/しない点が問題を深刻化させる。対策は「個体ごとに一意な初期資格情報」「初回設定の強制」「未設定時のインターフェース無効化」など。
4. **被害の範囲**: 本件は機器データの取得が中心で、家庭内ネットワークへの侵入やRCEに直結する性質ではないが、生活パターンの推測などプライバシー上の懸念がある。

### 検知・暫定緩和・恒久対策

- **検知の着眼点**: 製品が想定外に初期接続用APを発信し続けていないか、身に覚えのない機器接続。家庭環境では検知は難しく、設定状態の確認が現実的。
- **暫定緩和**: 製品をただちに家庭のWi-Fiルータへ正しく接続して初期設定を完了し、共通APの窓を閉じる。Wi-Fi機能を使わない場合は無効化。初期化後は放置せず必ず再設定する。
- **恒久対策**: 三菱電機が提供するファームウェア更新を適用する。ファーム更新が直ちに行えない場合は、上記の「確実な初期設定／未使用時の無効化」を運用で徹底する。対象製品・型番・更新可否は JVN・三菱電機 PSIRT で確認。

## 📖 用語解説（このニュースとのつながり）

- **ハードコード認証情報（CWE-798）**: ソフト/機器に固定で埋め込まれた資格情報。変更できず、同型で共通だと「公知の鍵」と化す。本件の根本原因。
- **SSID**: Wi-Fi ネットワークの名前。本件では初期接続用APの固定の名前が手がかりになる。
- **初期接続用アクセスポイント**: 家電が自ら出す設定用のWi-Fi。設定完了後は不要になるが、放置すると窓が開いたまま残る。
- **無線到達範囲**: 攻撃が成立する物理的な距離。隣室や建物前など近接が前提で、広域リモート攻撃ではない。
- **IoT セキュリティ**: ネット接続家電特有の弱点群。初期資格情報の一意化・初回設定強制が定石の対策。

## 影響範囲 / 推奨対応

影響は、Wi-Fi を有効にしたまま初期設定未了、または初期化後放置の三菱電機製対象家電。被害は近隣からの機器データ取得が中心で、深刻度は条件付きだが、生活情報の漏えい懸念がある。推奨対応は、対象製品を正しく家庭Wi-Fiへ接続して初期設定を完了し、使わない場合はWi-Fiを無効化、提供されるファームウェア更新を適用すること。対象型番と更新情報は JVN・三菱電機 PSIRT で確認する。
