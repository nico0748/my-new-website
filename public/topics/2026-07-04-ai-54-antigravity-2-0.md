# Google Antigravity 2.0 — デスクトップ＋CLI 二面構成の自律コーディング基盤

> トピック: 開発トピック｜出典: The New Stack（要一次確認）｜種別: ai

## 一行サマリー
Google は 2026-05-19 の I/O で Antigravity 2.0 を発表。統一ハーネスを「再設計されたデスクトップアプリ」と「新しい標準 CLI」の2つの面に分け、並列タスク用の専門サブエージェント、クロスプラットフォームな端末サンドボックス、認証情報マスキング、強化された Git ポリシーを追加した。

## 🔰 初学者向け（何が・なぜ重要か）
Antigravity は Google の AI コーディング環境で、AI が自分でコードを書き進める「エージェント型」の道具です。2.0 では、この道具を「見た目のあるアプリ（デスクトップ）」と「コマンドで操るツール（CLI）」の2つの入口に整理しました。画面で操作したい人にも、ターミナルで自動化したい人にも合わせられる、という考え方です。

新しさの中心は「安全に自律作業させる」工夫です。1つは「サンドボックス」。AI がコマンドを実行する場所を、本番環境から隔離した砂場に閉じ込め、万一おかしなことをしても被害が広がらないようにします。2つ目は「認証情報マスキング」。パスワードや API キーといった秘密が、AI の作業ログや画面にうっかり漏れないよう隠します。3つ目は「Git ポリシーの強化」で、AI による無断のコミットやプッシュに歯止めをかけます。

つまり Antigravity 2.0 は「AI にどんどん任せる」だけでなく「任せても事故らない」ための土台を固めた更新、と理解できます。

## 🧠 専門的解説（技術詳細・設計・使いどころ）
### 技術的詳細（何が変わったか）
Antigravity 2.0 は Google I/O（2026-05-19）で発表。従来の統一ハーネスを2つの surface に再編：(1) 再設計されたデスクトップアプリ、(2) 新規の standalone CLI。機能追加は、並列タスク用の specialized subagents、クロスプラットフォームの terminal sandboxing、credential masking（認証情報のマスキング）、hardened Git policies（強化された Git 運用ポリシー）。全体として「自律度」と「ガードレール」を同時に引き上げる方向。詳細な機能仕様・対応モデル・提供範囲は Google 公式の一次情報で要確認（本稿の一次情報が二次報道経由の点に留意）。

### 仕組み / 背景（なぜこう設計されたか）
エージェント型コーディングでは、モデルの賢さと同じくらい「ハーネス（実行を安全・効率よく回す枠組み）」が体験を左右する。デスクトップ/CLI の2面化は、対話的レビュー重視の開発者と、CI/自動化に組み込みたい開発者の双方に最適な入口を提供する設計だ。並列サブエージェントは、依存の少ないサブ課題を同時進行させて壁時間を縮める狙い（ただし課題分割と結果統合の難しさがトレードオフ）。terminal sandboxing は、エージェントが任意コマンドを実行する以上「隔離実行」が事故防止の要になるための必須機構。credential masking は、ログ・画面・モデル入力に秘密情報が混入する「意図せぬ漏えい」経路を塞ぐ。hardened Git policies は、自律エージェントが履歴を汚す/未承認変更を送る事故を、ポリシーで機械的に防ぐ発想だ。これらは前述の「コーディングツールの収束」で各社共通の論点になっている。

### 実務での使いどころ・移行の勘所
Google エコシステム（Gemini 系モデル、GCP）との親和性を活かす開発に好適。対話的に使うならデスクトップ、CI/自動化に組み込むなら CLI、と入口を選べる。導入の勘所は、(1) サンドボックスの隔離範囲とネットワーク権限を明示設定、(2) credential masking を過信せず、秘密情報はそもそもエージェントに触れさせない設計（シークレット管理の分離）を併用、(3) Git ポリシーで自動コミット/プッシュの可否とブランチ保護を定義、(4) 並列サブエージェントはコスト増を伴うため上限とログを設定。他ツール同様、生成物のテスト/レビューは前提。

## 📖 用語解説（このトピックとのつながり）
- **ハーネス（harness）**: エージェントを安全・効率よく動かす実行枠組み。2.0 の中核。
- **サンドボックス**: 実行を隔離する砂場環境。任意コマンド実行の事故を封じ込める。
- **credential masking**: 認証情報をログ/画面/入力から隠す機構。漏えい経路を塞ぐ。
- **specialized subagents（並列サブエージェント）**: 役割特化のエージェントを並列実行する仕組み。壁時間短縮。
- **Git ポリシー**: コミット/プッシュ/ブランチ保護の運用規則。自律変更の暴走を機械的に抑止。

## 影響範囲 / 推奨アクション
Google/Gemini 中心の開発チームに関係。用途に応じてデスクトップ/CLI を選択。サンドボックス範囲・ネットワーク権限・Git ポリシーを明示設定し、秘密情報はエージェントに触れさせない分離を併用。並列サブエージェントはコスト上限とログを設定。一次情報での機能確認を推奨。
