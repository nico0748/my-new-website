# CSS Anchor Positioning が Baseline 入り ― Floating UI を CSS だけで代替

> トピック: 開発トピック｜出典: southwellmedia｜種別: frontend

## 一行サマリー
要素を別の要素（アンカー）に対して配置できる **CSS Anchor Positioning** がクロスブラウザの Baseline に到達（時期は要確認）。ツールチップやポップオーバーの位置合わせを、Floating UI／Popper.js といった JS ライブラリなしで CSS だけで実現できる。

## 🔰 初学者向け（何が・なぜ重要か）
ボタンにマウスを乗せると出てくる小さな吹き出し（ツールチップ）や、メニューのポップアップ。これらを「ボタンのすぐ下、ただし画面端なら上に出す」といった具合にきっちり位置合わせするのは、実は昔から面倒な作業でした。要素の座標を JavaScript で計算し、スクロールやリサイズのたびに追従させる必要があり、そのために Floating UI や Popper.js といった専用ライブラリを読み込むのが定番でした。

CSS Anchor Positioning は、この「Aという要素を基準にBを配置する」を **CSS の言葉だけで書ける**ようにするものです。`anchor-name` で基準を名付け、`position-anchor` と `anchor()` で「その要素の右下に」と指定するだけ。ブラウザが位置計算とはみ出し回避を肩代わりしてくれます。JS を減らせると、ページは軽く・速く・壊れにくくなります。

## 🧠 専門的解説（技術詳細・設計・使いどころ）
### 技術的詳細（何が変わったか）
- `anchor-name`（基準要素に名前を付与）、`position-anchor`／`anchor()`（配置対象から基準を参照）が使用可能に。
- はみ出し時の再配置は `position-try-fallbacks`（旧 `position-try-options`、名称は要確認）で宣言的に指定。
- `popover` 属性や `:popover-open` と組み合わせ、ネイティブなポップオーバー UI を構築できる。

### 仕組み / 背景（なぜこう設計されたか）
従来の `position: absolute` は「最も近い位置指定された祖先」を基準にする制約があり、DOM 構造に配置が縛られていました。Anchor Positioning は DOM の親子関係に依存せず、名前で任意の要素を基準にできます。これにより、視覚的な位置関係と DOM 構造を分離でき、`overflow: hidden` なコンテナ内の要素にも吹き出しを正しく出せます。JS の座標計算を CSS 宣言に置き換えることで、リフロー時の追従もブラウザが最適化します。

### 実務での使いどころ・移行の勘所
- ツールチップ、ドロップダウン、コンテキストメニュー、コーチマークなど「何かの隣に出す UI」全般に有効。
- Floating UI 依存を減らせるが、複雑なフリップ/シフト挙動は移行前に挙動差を要確認。
- 古いブラウザ向けにはフォールバックが必要な場合あり。プログレッシブエンハンスメントで導入するのが安全。

## 📖 用語解説（このトピックとのつながり）
- **アンカー配置**: ある要素を別要素の位置に紐づけて配置する CSS 機能。本トピックの主役。
- **`anchor-name` / `anchor()`**: 基準を名付け、参照して座標を得るための CSS プロパティ/関数。
- **Floating UI / Popper.js**: 従来この配置を担っていた JS ライブラリ。代替対象。
- **popover 属性**: ネイティブなポップオーバーを作る HTML 機能。Anchor と好相性。
- **Baseline**: 主要ブラウザで安全に使えると認定された状態。採用判断の目安。

## 影響範囲 / 推奨アクション
UI コンポーネントを実装するフロントエンド開発者に影響大。新規のツールチップ/ポップオーバーは Anchor Positioning を第一候補に。既存の Floating UI 依存は、対応ブラウザ範囲を確認のうえ段階的に置換を検討。
