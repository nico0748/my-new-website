# 横河電機 FAST/TOOLS・CI Server に重要情報の平文送信の脆弱性（CVE-2026-11833）— CVSS 8.2・設定情報が漏えいの恐れ

> トピック: 脆弱性ニュース ｜ 出典: JVN / CISA ICS Advisory / Yokogawa Security Advisory ｜ 種別: 重要情報の平文送信（Cleartext Transmission of Sensitive Information, CWE-319）

## 一行サマリー

国内製造業の制御システム（SCADA）として広く使われる横河電機のFAST/TOOLSおよびCI Serverに、Webサーバーの応答にCI Serverの設定情報が含まれて返ってしまう「重要情報の平文送信」脆弱性CVE-2026-11833（CVSS 4.0で8.2、High）が判明し、JVNが6月23日に公表した。傍受・取得された設定情報が次の攻撃の足がかりにされる恐れがある。パッチ提供までは該当Webサーバーへのアクセス制限などネットワーク側の防御が必要。

## 🔰 初学者向け（何が・なぜ問題か）

FAST/TOOLSは、工場やプラントの設備を監視・制御する「SCADA（スキャダ）」と呼ばれる産業用システムです。発電・化学・水処理など、社会インフラの現場で機器の状態を集めて操作する中枢を担います。CI Serverはそれと連携する構成管理のサーバーです。こうした制御システムは、一度侵入されると物理的な設備に影響しうるため、特に慎重な防御が求められます。

今回の問題は、このシステムのWebサーバーが応答（ブラウザに返すデータ）の中に、本来見せるべきでない**CI Serverの設定情報を平文（暗号化されていない、そのまま読める状態）で含めてしまう**点です。

たとえるなら、受付で書類を渡すとき、表面の用件だけでなく**裏面に書かれた内部マニュアルや鍵のありか**まで一緒に渡してしまうようなものです。しかも封筒に入れず（平文＝暗号化なし）丸見えで渡すため、通信を覗き見できる位置にいる第三者がその情報を拾えます。攻撃者は得た設定情報をもとに、システムの構造を把握し、次のより踏み込んだ攻撃を組み立てられます。直接乗っ取られるわけではないものの、**侵入の偵察・準備を助けてしまう**点が問題です。

## 🧠 専門的解説（技術詳細・メカニズム・検知対応）

### 技術的詳細（種別・前提条件）

- **CVSS**: CVSS 4.0 ベーススコア **8.2（High）**。
- **CWE分類**: CWE-319（重要情報の平文送信）。Webサーバー応答にCI Serverの設定情報が含まれて返却される。
- **前提条件**: 攻撃者が該当Webサーバーへネットワーク到達、または通信経路を傍受できる位置にいること。
- **影響製品・バージョン**: FAST/TOOLS **R9.01〜R10.04**（対象パッケージ: RVSVRN, UNSVRN, HMIWEB, FTEES, HMIMOB）、CI Server **R1.01〜R1.04**（全パッケージ）。
- **修正・対応**: 横河のセキュリティアドバイザリおよびJVN（6/23公表）/ CISA ICSアドバイザリ（ICSA-26-041-01）を参照。恒久対策の提供状況・適用手順はベンダー情報で要確認。
- **悪用状況**: 本稿時点で実環境悪用の報告は確認されていない（要確認）。ただしOT環境では設定情報漏えいが横展開の起点になりやすい。

### メカニズム（攻撃が成立する仕組み）

以下は概念レベルの説明であり、武器化可能な手順・PoC等は含めない。

1. **信頼境界の所在**: 制御システムのWeb管理面は、本来「認可された運用者にのみ必要最小限の情報を返す」べき境界である。応答に何を載せるかの制御が信頼境界となる。
2. **過剰な情報の応答混入**: Webサーバーが応答を生成する際、CI Serverの**設定情報（システム構成・内部パラメータ等）を応答ボディに含めて返却**してしまう。これは「最小権限・最小開示」の原則からの逸脱で、本来内部に留めるべき情報が外部到達点に露出する。
3. **平文であることの増幅効果**: その情報が暗号化されず平文で送られるため、通信経路上で傍受可能な攻撃者（中間者、同一セグメント上の盗聴者）が容易に取得できる。機密性（Confidentiality）が直接損なわれる。
4. **偵察→横展開への橋渡し**: 得られた設定情報は、システム構成・接続関係・潜在的な認証情報の手がかりとなり、攻撃者の偵察を加速する。単体では即時のコード実行を生まないが、**他の脆弱性や弱い認証と組み合わせる**ことで侵入・横展開の前提を整える。

脆弱性クラスとしては、「アプリケーションが応答に過剰な内部情報を含める情報露出（Information Exposure）」と「暗号化欠如による平文送信」の複合であり、OT/ICS製品で頻出する設計上の機密性欠陥に該当する。

### 検知・暫定緩和・恒久対策

- **検知の着眼点**: 制御系Webサーバーへの外部・非運用セグメントからのアクセス、設定情報を含む応答の異常取得、平文HTTP通信のキャプチャ痕跡。OTネットワークの通信可視化（一方向監視）で異常フローを検出。
- **暫定緩和（パッチ前）**: 該当Webサーバーへの**アクセスを運用に必要な機器・セグメントへ限定**。OTネットワークをIT網から分離し、通信経路の盗聴を防ぐネットワーク制御（VLAN分離・暗号化トンネル）を適用。リモートアクセスはVPN経由に限定。
- **恒久対策**: 横河の提供する修正／回避策をベンダー手順に従い適用。
- **運用示唆**: ICS/OTは「ネットワークで囲って守る」が基本。製品単体の機密性欠陥は、セグメンテーションと最小到達性で被害を抑える。

## 📖 用語解説（このニュースとのつながり）

- **SCADA / FAST/TOOLS**: 産業設備を監視制御するシステム。意味＝「プラントの司令塔」。本件＝その設定情報が漏れうる。
- **CI Server**: FAST/TOOLSと連携する構成管理サーバー。本件＝その設定情報が応答に混入する。
- **平文送信（CWE-319）**: 機密情報を暗号化せず送る欠陥。本件＝傍受で情報が取得される。
- **情報露出（Information Exposure）**: 不要な内部情報を外部に開示する欠陥。本件＝応答に過剰な設定情報が載る。
- **OTセグメンテーション**: 制御網をIT網から分離する防御。本件＝最有力の暫定緩和。

## 影響範囲 / 推奨対応

影響を受けるのは、FAST/TOOLS R9.01〜R10.04 および CI Server R1.01〜R1.04 を運用するプラント・インフラ事業者。推奨対応は、(1) 横河のアドバイザリ／JVN（6/23）に基づく修正・回避策の適用、(2) 該当Webサーバーへのアクセスを運用機器・セグメントに限定、(3) OT網のIT網からの分離と通信暗号化、(4) 制御系Webへの外部アクセス・平文通信の監視。
