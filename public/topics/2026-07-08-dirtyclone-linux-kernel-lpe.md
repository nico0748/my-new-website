# Linuxカーネル『DirtyClone』でローカル権限昇格（CVE-2026-43503）— skbフラグ伝播漏れがroot奪取に

> トピック: 脆弱性ニュース ｜ 出典: JFrog Security Research / SecurityWeek / Ubuntu ｜ 種別: ローカル権限昇格（メモリ破壊）

## 一行サマリー

Linux カーネルのネットワーク処理（skbuff のフラグメント管理）にフラグ伝播漏れがあり、ローカル利用者が特権バイナリのメモリを書き換えて root に昇格できる欠陥『DirtyClone』（CVE-2026-43503, CVSS 8.8/kernel.org・7.0/Red Hat）。マルチテナントやコンテナ／Kubernetes 環境で特に危険。

## 🔰 初学者向け（何が・なぜ問題か）

Linux はサーバやクラウドの土台となるOSです。今回の欠陥は、そのOSの心臓部（カーネル）の中でも「ネットワークの小包（パケット）を扱う部品」に潜んでいました。

パケットは、中身のデータを効率よく扱うため「このデータは他とも共有中だから勝手に書き換えないで」という**注意書きの札（フラグ）**を持ちます。ところが、パケットを複製（クローン）したり中身をずらしたりする処理で、この札を**付け替え忘れる**ミスがありました。

札が外れると、本来"共有中で触ってはいけない"データが"書き換えてOK"と誤認されます。攻撃者はこれを利用し、`su` のような**管理者になるための特権プログラムの中身（メモリ上のコピー）を丸ごと差し替え**ます。次に誰かがそのプログラムを実行すると、差し替えた攻撃者のコードが管理者権限で動き、root（最上位権限）を奪われます。

たとえるなら、「開封厳禁」の封印シールを貼り忘れた荷物に、こっそり中身をすり替えられ、受け取った人がそれを"正規品"として使ってしまう、という状況です。しかもディスクにはファイルとして痕跡が残らない（メモリ上だけの改ざん）ため、発見が難しいのも特徴です。

## 🧠 専門的解説（技術詳細・メカニズム・検知対応）

### 技術的詳細（種別・前提条件）

- **脆弱性種別**: カーネルのメモリ管理不備（フラグ伝播漏れ）に起因する権限昇格。DirtyFrag 系（ページキャッシュ改ざん型 LPE）の変種。
- **CVSS**: 8.8（kernel.org）／ 7.0（Red Hat）。値の差は評価基準の違い。断定は避け両論併記。
- **前提条件**: ローカル実行権限に加え、`CAP_NET_ADMIN` を保持または取得できること。非特権ユーザー名前空間（user namespace）が有効な環境ではこの取得が容易になり、リスクが上がる。
- **影響範囲**: 主要ディストリの多くのカーネル。マルチテナントサーバ、CI/CD ランナー、コンテナホスト、Kubernetes クラスタが高リスク。
- **修正版**: mainline commit `48f6a5356a33`、Linux v7.1-rc5、および stable/LTS へのバックポート。各ディストリの更新を適用。
- **悪用状況**: 2026年6月に JFrog が公開。PoC 研究が出ており、痕跡がディスクに残らない点で検知回避性が高い。

### メカニズム（攻撃が成立する仕組み）

以下は「型の説明」であり、武器化可能な手順・正確なオフセット・PoCコードは示さない。

1. **信頼境界**: 非特権プロセス（ユーザー空間）→ カーネル（特権空間）。カーネルは自分が管理するメモリの整合性を保証する責務を負う。
2. **不備のあるコンポーネント**: `__pskb_copy_fclone()` と `skb_shift()` というヘルパで、フラグメント記述子を別の skbuff へ移す際に **`SKBFL_SHARED_FRAG` フラグを伝播させ損ねる**。このフラグは「このフラグメントは共有中＝コピーオンライトすべき」を示す印。
3. **攻撃者が握るプリミティブ**: フラグが落ちると、共有中のページが"占有・可変"として扱われる。攻撃者は特権バイナリ（例: `/usr/bin/su`）のページキャッシュをネットワークパケットに載せ、パケットをクローンし、攻撃者制御の IPsec トンネルを通す。復号ステップで**メモリ内バイナリを任意バイトに上書き**できる（＝任意メモリ書き込みプリミティブ）。
4. **橋渡し（昇格への連結）**: 上書き対象が"次に特権実行されるバイナリ"であるため、その実行時に攻撃者のコードが特権コンテキストで走る。任意書き込み → 特権バイナリ改ざん → root 実行、という橋渡し。
5. **最終被害**: root 権限奪取。コンテナからホストへのブレイクアウト、同一ホスト上の他テナントへの侵害に発展。
- **一般原理**: コピーオンライト（COW）系の欠陥は、「共有／占有の状態管理」が一箇所でも破れると、読み取り専用のはずの資源が書き換え可能になる。DirtyCOW 以来繰り返される"状態フラグの一貫性"問題の系譜。

### 検知・暫定緩和・恒久対策

- **検知の着眼点**: 非特権プロセスによる想定外の `CAP_NET_ADMIN` 取得、IPsec/ESP 関連モジュールの不審なロード、特権バイナリ実行後の異常挙動。ディスクに痕跡が残りにくいため、EDR はメモリ・syscall 監視を重視。
- **暫定緩和（パッチ前）**: 非特権ユーザー名前空間を制限（`sysctl -w kernel.unprivileged_userns_clone=0`）、不要な脆弱モジュール（`rxrpc`, `esp4`, `esp6`）をブラックリスト化。
- **恒久対策**: 各ディストリの修正カーネルへ更新（Ubuntu 等が提供済み）。更新後は再起動を確実に行う。
- **運用示唆**: マルチテナント／コンテナ基盤では"ローカル昇格＝致命的"。カーネル更新の SLA を短縮し、user namespace の許可範囲を最小化する。

## 📖 用語解説（このニュースとのつながり）

- **skbuff（sk_buff）**: Linux がネットワークパケットを表す内部構造体。この中のフラグメント管理でフラグ伝播が漏れたのが本件の起点。
- **SKBFL_SHARED_FRAG**: 「このフラグメントは共有中」を示す印。落ちると共有ページを可変扱いしてしまい、改ざんの余地を生む。
- **CAP_NET_ADMIN**: ネットワーク管理系の特権ケーパビリティ。本件の前提条件で、user namespace 有効環境では取得が容易になる。
- **コピーオンライト（COW）**: 共有中は読み取り専用にし、書き込み時に初めて複製する仕組み。状態管理が破れると読み取り専用が書き換え可能になる（DirtyCOW と同系統）。
- **コンテナブレイクアウト**: コンテナ内権限からホスト側権限へ抜け出す攻撃。root 昇格が成立すると現実的な脅威になる。

## 影響範囲 / 推奨対応

- **影響範囲**: 脆弱なカーネルを動かすサーバ／端末全般。特にマルチテナント・CI/CD・コンテナ／Kubernetes 環境。
- **推奨対応**: (1) 修正カーネルへ更新し再起動、(2) 更新前は user namespace 制限＋脆弱モジュールのブラックリスト化、(3) EDR でメモリ／ケーパビリティ取得の監視強化、(4) コンテナホストの権限最小化を再点検。
