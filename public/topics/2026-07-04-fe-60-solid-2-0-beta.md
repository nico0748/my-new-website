# SolidJS 2.0（beta）― 細粒度リアクティビティの先駆けがメジャー刷新へ

> トピック: 開発トピック｜出典: solidjs.com / TanStack Router releases（solid-js 2.0.0-beta 参照）｜種別: frontend

## 一行サマリー
「細粒度リアクティビティ（fine-grained reactivity）」の代表格 SolidJS が 2.0 の beta 段階に入っている（TanStack Router の依存更新で `solid-js` 2.0.0-beta 系が確認できる）。仮想 DOM を使わず、変更があった箇所だけを最小コストで更新する方式を、次のメジャーでさらに洗練させる動き。近年フレームワーク各社が追随する「Signals 潮流」の源流の一つ。

## 🔰 初学者向け（何が・なぜ重要か）
画面の一部だけが変わったとき、その一部だけを賢く描き直せると無駄がありません。SolidJS はこの「必要な最小限だけ更新する」ことを最初から徹底しているフレームワークです。多くのフレームワークが使う「仮想 DOM（画面の下書きを毎回作って差分を比べる方式）」を使わず、値と画面を直接ひも付けます。

例えるなら、雑誌のレイアウトを毎回まるごと組み直すのではなく、「この見出しだけ差し替える」と最初から配線しておくイメージです。今回の SolidJS 2.0 は、その配線方式（Signals）をより使いやすく・安定させるメジャー更新で、まだ beta（試験公開）段階です。最近 React・Angular・Vue などが相次いで取り入れている「Signals」という考え方の、いわば先輩格が次の世代へ進もうとしている、という話です。

## 🧠 専門的解説（技術詳細・設計・使いどころ）
### 技術的詳細（何が変わったか）
- **SolidJS 2.0 は beta**（`solid-js` 2.0.0-beta 系、`@solidjs/web` も beta 追随。正確な GA 時期・変更点は公式 CHANGELOG 参照、要確認）。
- 仮想 DOM を持たず、**Signals** ベースの細粒度リアクティビティで、依存する DOM ノードのみを直接更新する方式を継続・洗練。
- エコシステム（TanStack Router 等）が 2.0 beta への追随を進めており、実運用に向けた検証が進行中。

### 仕組み / 背景（なぜこう設計されたか）
細粒度リアクティビティの核心は「値（Signal）とその値を読む箇所（DOM の一部や計算）を、購読関係として直接つなぐ」ことです。値が変わると、その値を実際に使っている最小の依存だけが再実行されます。仮想 DOM 方式が「全体を再描画して差分を計算する」のに対し、Solid は「差分計算そのものを不要にする」設計で、再レンダリングのオーバーヘッドを構造的に排除します。

なぜ今メジャー更新か。React 19 の Compiler、Angular の Signals、Vue の Vapor Mode など、主要フレームワークが軒並み「細粒度・コンパイル駆動」へ寄ってきており、この分野の先駆者だった Solid も API の一貫性・型安全・エコシステム互換を整えるタイミングに来たためです。トレードオフとして、Signals モデルは「いつ・何が再実行されるか」の直感が仮想 DOM と異なり、学習コストと既存資産の移行コストが生じます。beta 段階ゆえ本番採用には慎重さが要ります。

### 実務での使いどころ・移行の勘所
- 高頻度更新（リアルタイムダッシュボード、エディタ、可視化）で、再レンダリングコストを構造的に抑えたい場面に強い。
- 既存 Solid 1.x からの移行は、2.0 の破壊的変更点とエコシステム（Router 等）の 2.0 対応状況を確認してから。
- 「Signals をちゃんと使いたい」チームの技術選定で、React/Angular/Vue の Signals 実装と比較検討する価値がある。

## 📖 用語解説（このトピックとのつながり）
- **SolidJS**: 仮想 DOM を使わない細粒度リアクティビティのフレームワーク。本トピックの主役。
- **細粒度リアクティビティ (fine-grained reactivity)**: 変更箇所だけを最小単位で更新する方式。
- **Signals**: 値と依存を購読関係で結ぶリアクティブな仕組み。近年多くのフレームワークが採用。
- **仮想 DOM**: 画面の下書きを作って差分計算する方式。Solid はこれを使わない。
- **beta**: 試験公開段階。API 変更を織り込んで検証すべきフェーズ。

## 影響範囲 / 推奨アクション
Signals 潮流の技術選定に関わるフロントエンド開発者に参考になる。既存 React 等プロジェクトへの直接影響はないが、新規で高頻度更新の UI を作るなら Solid 2.0 の動向を追う価値がある。本番採用は GA を待ちつつ、beta で PoC 検証するのが妥当。
