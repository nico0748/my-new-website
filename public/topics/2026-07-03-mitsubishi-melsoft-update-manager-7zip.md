# 三菱電機 MELSOFT Update Manager に 7-Zip 起因の複数脆弱性(JVNVU#95990609)

> トピック: 脆弱性ニュース ｜ 出典: JVN / JPCERT/CC / 三菱電機 ｜ 種別: 同梱 7-Zip 由来のヒープオーバーフロー(CWE-122)・NULL 参照(CWE-476)・リンク解釈(CWE-59)・パストラバーサル(CWE-22)

## 一行サマリー

三菱電機の FA 向けソフト MELSOFT Update Manager に、同梱する 7-Zip 由来の複数脆弱性(CVE-2025-53816/53817/55188/11001)が存在。細工した圧縮ファイルをユーザーが展開すると、情報の取得・改ざんや DoS を引き起こしうる。対象は SW1DND-UDM-M Ver.1.000A〜1.014Q。

## 🔰 初学者向け（何が・なぜ問題か）

MELSOFT Update Manager は、工場の制御機器を扱う三菱電機の FA(ファクトリーオートメーション)ソフト群を最新に保つための「更新管理ツール」です。このツールは内部で 7-Zip という有名な圧縮・解凍ソフトの機能を取り込んで使っています。

問題は「解凍(展開)」にあります。ZIP のような圧縮ファイルは、たくさんのファイルを一つにまとめた「引っ越しの段ボール箱」のようなもの。解凍とは、その箱を開けて中身を元の場所に並べ直す作業です。今回の脆弱性は、攻撃者が細工した「罠入りの箱」を用意し、それを利用者がうっかり展開すると起きます。中には、(1)展開ソフトを異常終了させて動けなくする(サービス妨害=DoS)仕掛けや、(2)「箱の外」——つまり本来置くはずのない場所へファイルを書き出させる仕掛け(パストラバーサル/リンク悪用)が含まれます。後者は、重要ファイルの上書きや、意図しない場所への配置を通じて改ざん・情報取得につながります。

つまり被害の入口は「怪しい圧縮ファイルを開くこと」。制御システムの更新ツールという性質上、影響が現場のシステムに及びうる点が重要です。

## 🧠 専門的解説（技術詳細・メカニズム・検知対応）

### 技術的詳細（種別・前提条件）

- **識別子**: JVNVU#95990609(親アドバイザリ)。含まれる CVE と種別:
  - CVE-2025-53816 … ヒープベースのバッファオーバーフロー(CWE-122)
  - CVE-2025-53817 … NULL ポインタデリファレンス(CWE-476)
  - CVE-2025-55188 … リンク解釈の問題(CWE-59)
  - CVE-2025-11001 … パストラバーサル(CWE-22)
- **前提条件**: 攻撃者が用意した**細工済み圧縮ファイルをユーザーが展開**すること(ユーザー操作が必要)。ネットワーク単独の遠隔・無認証攻撃ではなく、ソーシャル的な配布(メール添付・共有等)が典型的な起点。
- **影響製品/版**: MELSOFT Update Manager(SW1DND-UDM-M)Ver.1.000A〜1.014Q。
- **修正/対応**: 開発者提供の対応版へアップデート。即時更新できない場合は回避策・緩和策の適用が推奨(具体版は三菱電機の情報で**要確認**)。
- **悪用状況**: 本記事時点で実環境悪用の公表情報なし(**要確認**)。7-Zip 由来の上流脆弱性が同梱ソフトへ波及した「サプライチェーン(依存コンポーネント)型」。

### メカニズム（攻撃が成立する仕組み）

各 CVE のクラスに沿って、成立の一般原理を概念レベルで分解する(具体的な細工手順・PoC は記載しない)。

1. **共通の信頼境界**: 「外部から受け取った圧縮ファイル」は完全に攻撃者制御下のデータ。解凍処理はその中の**ヘッダ・エントリ名・格納サイズ・メタ情報**を信じて動くため、ここが信頼境界。
2. **ヒープオーバーフロー(CWE-122)**: 圧縮ヘッダに記載されたサイズ情報と実データの不整合を、確保済みバッファ境界を越えて書き込む形で突く。攻撃者が握るプリミティブは「ヒープ上の隣接領域の破壊」で、条件次第でクラッシュ(DoS)から、より深い制御奪取の可能性まで幅がある(本件の到達点は**要確認**)。
3. **NULL 参照(CWE-476)**: 想定外・破損したエントリを処理する際に、確保に失敗したポインタを検査せず参照し、プロセスを異常終了させる。主に可用性(DoS)への影響。
4. **リンク解釈の問題(CWE-59)/パストラバーサル(CWE-22)**: アーカイブ内エントリ名に相対経路(`../`)やシンボリックリンクを仕込むと、解凍先が「展開ディレクトリの外」へ抜け出す。攻撃者が握るプリミティブは「任意パスへのファイル書き出し/既存ファイル上書き」。書込先を起動時読込ファイルや実行パスへ向ければ、改ざん経由でより重い被害へ橋渡しされうる(いわゆる Zip Slip の一般原理)。
5. **依存コンポーネント波及の構造**: 根本原因は上流 7-Zip 側のパーサ実装にあり、それを取り込む MELSOFT Update Manager が影響を継承した。同梱ライブラリの更新追随の遅れが被害面になるという、サプライチェーン脆弱性の典型。
6. **クラスとしての位置づけ**: メモリ安全性の破れ(2,3)とパス制御の欠落(4)の複合。前者は主に DoS、後者は改ざん/情報操作に効く。いずれも「解凍という正常処理」が引き金である点が共通。

> 注: 各 CVE の正確な CVSS・到達可能な最悪影響は上流(7-Zip)および三菱電機の情報に依拠すべきで、ここでは CWE クラスの一般原理として説明する。

### 検知・暫定緩和・恒久対策

- **検知の着眼点**: MELSOFT Update Manager 実行時の異常終了・クラッシュの発生、展開ディレクトリ外への予期しないファイル生成・既存ファイルの改変、出所不明の圧縮ファイルの受領・展開操作。
- **暫定緩和(パッチ前)**: 出所不明の圧縮ファイルを本ツールで展開しない。ファイルは信頼できる経路(正規配布元)からのみ取得。展開は権限を絞った環境で行い、書込先を監視。
- **恒久対策**: 三菱電機提供の対応版(同梱 7-Zip を更新した版)へアップデート。FA 製品全体のパッチ運用に組み込む。
- **運用示唆**: FA/OT 環境は更新が遅れがち。同梱 OSS(7-Zip 等)の脆弱性は SBOM/依存管理の観点で追跡し、上流 CVE の波及を見落とさない体制にする。

## 📖 用語解説（このニュースとのつながり）

- **同梱ライブラリ(依存コンポーネント)**: 製品が内部で使う他社/OSS 製部品。今回は 7-Zip がそれで、上流の欠陥が製品へ波及。
- **Zip Slip / パストラバーサル(CWE-22)**: 圧縮内エントリ名の細工で展開先の外へファイルを書き出す手法。改ざんの起点。
- **リンク解釈の問題(CWE-59)**: シンボリックリンク等の扱いを悪用し、想定外の場所を操作させる不備。
- **ヒープオーバーフロー(CWE-122)/NULL 参照(CWE-476)**: いずれもメモリ安全性の破れ。主に DoS、条件により深い影響。
- **SBOM**: 製品に含まれる部品一覧。同梱 OSS の脆弱性追跡に有効で、今回のような波及の早期把握に役立つ。

## 影響範囲 / 推奨対応

MELSOFT Update Manager(SW1DND-UDM-M)Ver.1.000A〜1.014Q を利用する FA/OT 運用組織が対象。被害の入口は「細工圧縮ファイルの展開」であるため、まず出所不明ファイルを展開しない運用を徹底し、正規配布元からの取得に限定する。恒久対応として三菱電機提供の対応版へ更新し、同梱 OSS の脆弱性追跡を SBOM/依存管理へ組み込む。
