# Splunk Enterprise 認証不要の任意ファイル作成/切り詰め CVE-2026-20253（KEV登録・悪用確認）

> トピック: 脆弱性ニュース ｜ 出典: Splunk Advisory / The Hacker News / CISA ｜ 種別: 認証欠落 → 任意ファイル書き込み（CWE-306 系）

## 一行サマリー

ログ分析基盤 Splunk Enterprise の PostgreSQL サイドカー用エンドポイントに認証が掛かっておらず、ネットワーク到達可能な攻撃者が認証なしで任意のファイルを作成・切り詰めできる致命的欠陥（CVSS 9.8）。実環境での悪用が確認され、CISA は KEV に登録し緊急のパッチ適用を求めている。

## 🔰 初学者向け（何が・なぜ問題か）

Splunk は、会社中のサーバーやアプリが吐き出す「ログ（動作記録）」を一カ所に集めて検索・監視する、いわば「監視カメラの集中管理室」のような製品です。セキュリティ監視（SOC）の中心に置かれていることが多く、ここが乗っ取られると守る側の目が潰されます。

今回の問題は、Splunk が内部で使っているデータベース（PostgreSQL）に付随する「裏口の窓口」に、鍵（認証）が付いていなかったことです。たとえるなら、厳重に守られた管理室の正面玄関とは別に、誰でもノックなしで開けられる勝手口があり、その勝手口から「サーバー内の好きな場所に新しい書類を置いたり、既存の書類を空っぽにできた」ような状態です。

ファイルを自由に置けるということは、システムが起動時や定期実行時に読み込む設定ファイルやスクリプトを攻撃者が差し替えられる可能性があり、最終的にはサーバー上で任意のコマンドを実行（＝乗っ取り）できるところまで発展します。すでに実際の攻撃で使われているため、急いで更新する必要があります。

## 🧠 専門的解説（技術詳細・メカニズム・検知対応）

### 技術的詳細（種別・前提条件）

- 種別: 認証の欠落（CWE-306 相当）→ 認証不要の任意ファイル作成/切り詰め。最終的にコード実行（RCE）に連鎖しうる。
- CVSS: 9.8（Critical）。ベクトル CVSS:3.1/AV:N/AC:L/PR:N/UI:N/S:U/C:H/I:H/A:H（要確認・ベンダー値）。
- 影響バージョン: Splunk Enterprise 10.2 系の 10.2.4 未満、10.0 系の 10.0.7 未満。9.4 以前は非該当（要確認）。
- 修正版: 10.4.0 / 10.2.4 / 10.0.7 以上へアップグレード。
- 前提条件: 脆弱な PostgreSQL サイドカーサービスのエンドポイントにネットワーク到達できること。認証は不要。
- 悪用状況: Splunk PSIRT が限定的な実環境悪用を認知。CISA が KEV へ登録し、期限付きで修正を要求。

### メカニズム（攻撃が成立する仕組み）

注: 概念レベルにとどめ、武器化手順・PoC・正確なパスは記載しない。

1. **アクセス制御の欠落**: PostgreSQL サイドカー（メイン機能を補助する併走サービス）のエンドポイントが、認証・認可チェックなしに外部から呼び出せる状態だった。これが信頼境界の破れの起点。
2. **ファイル操作プリミティブの獲得**: 攻撃者は、この無認証エンドポイント経由で「任意パスへのファイル作成」または「既存ファイルの切り詰め（truncate）」という強力なプリミティブを得る。
3. **書き込みから実行への橋渡し**: 任意ファイル書き込みは、(a) 起動時・定期実行時に読み込まれるスクリプト/設定の上書き、(b) 認証・アクセス制御に関わるファイルの破壊（切り詰め）による別の不整合誘発、などを通じてコード実行や権限獲得へ連鎖しうる。任意書き込み→RCE は典型的な昇格パターン。
4. **一般原理**: 「補助サービス/サイドカーの認証を本体と同等に固めていない」設計は、認証欠落クラスの代表例。内部前提のコンポーネントがネットワークに露出することで攻撃面化する。

### 検知・暫定緩和・恒久対策

- 検知の着眼点: PostgreSQL サイドカー関連エンドポイントへの想定外アクセス、Splunk 関連ディレクトリでの予期しないファイル新規作成/サイズ0化、設定・起動スクリプトの改変、不審プロセス起動。
- 暫定緩和: ただちにアップグレードが原則（公式に回避策なし）。一時策として PostgreSQL サイドカーサービスの無効化、当該ポート/エンドポイントへのネットワークアクセス制限。
- 恒久対策: 10.4.0 / 10.2.4 / 10.0.7 以上へ更新。Splunk の管理面・内部サービスをインターネットから隔離。
- 運用示唆: 監視基盤そのものが標的になる前提で、Splunk のアクセス経路最小化と変更監視を強化する。

## 📖 用語解説（このニュースとのつながり）

- **Splunk Enterprise**: ログを集約・検索・可視化する基盤。SOC の中核に置かれることが多く、本件は守る側の中枢が狙われた点で重大。
- **サイドカー（sidecar）**: 主アプリに併走して補助機能を担う別プロセス/サービス。今回は PostgreSQL のサイドカーが無認証だった。
- **認証欠落（CWE-306）**: 必要な認証チェックを実装し忘れる欠陥。今回の根本原因の型。
- **任意ファイル書き込み / truncate**: 攻撃者が任意の場所にファイルを作る/中身を空にする操作。RCE への踏み台になる。
- **KEV（Known Exploited Vulnerabilities）**: CISA が「実際に悪用が確認された」脆弱性をまとめたカタログ。掲載＝至急対応の合図。

## 影響範囲 / 推奨対応

該当バージョンの Splunk Enterprise を運用する全組織が対象。修正版（10.4.0 / 10.2.4 / 10.0.7 以上）へ即時アップグレードする。即時更新が難しい場合は PostgreSQL サイドカーの無効化とネットワーク制限で暫定対処し、悪用痕跡（不審ファイル作成・設定改変）を調査する。
