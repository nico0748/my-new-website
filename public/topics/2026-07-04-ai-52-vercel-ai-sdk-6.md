# Vercel AI SDK 6 — エージェント・ツール承認・DevTools・MCP フル対応

> トピック: 開発トピック｜出典: Vercel 公式ブログ｜種別: ai

## 一行サマリー
Vercel AI SDK 6（2026-06）が公開され、エージェント（自律的なツール利用ループ）、ツール実行の承認（human-in-the-loop）、開発者向け DevTools、MCP のフルサポート、リランキング、画像編集などを追加した。TypeScript で AI 機能を組み込む定番ライブラリが、単発生成からエージェント構築の基盤へと拡張した。

## 🔰 初学者向け（何が・なぜ重要か）
Vercel AI SDK は、Web アプリに AI 機能（チャット、要約、画像生成など）を組み込むための TypeScript 製の道具箱です。どの AI 会社のモデルを使っても同じ書き方で呼べるようにしてくれるのが便利な点でした。

バージョン 6 の大きな変化は「エージェント」への対応です。従来は「質問を投げて答えをもらう」単発のやり取りが中心でしたが、エージェントは「目的を達成するために、AI が自分で道具（ツール）を選び、何度も使い、途中経過を見ながら進める」自律的な働き方です。SDK 6 はこれを標準で作れるようにしました。

加えて安全面の工夫として「ツール承認」が入りました。AI が「このファイルを消す」「メールを送る」といった危ない操作をしようとしたとき、実行前に人間の OK を挟める仕組みです。さらに開発中の動きを覗ける DevTools や、外部ツールをつなぐ共通規格 MCP のフル対応も入り、「AI エージェントを安全に、作りやすく」する方向へ大きく前進しました。

## 🧠 専門的解説（技術詳細・設計・使いどころ）
### 技術的詳細（何が変わったか）
Vercel AI SDK 6（2026-06）の主な追加は (1) Agents：モデルがツールを反復呼び出しして目的達成へ進む自律ループの抽象、(2) tool execution approval：ツール実行前に人手承認を挟む human-in-the-loop 機構、(3) DevTools：エージェントの推論・ツール呼び出しを可視化する開発支援、(4) MCP フルサポート：Model Context Protocol サーバをツール源として接続、(5) reranking：検索結果の再順位付け（RAG 品質向上）、(6) 画像編集。SDK は複数プロバイダ（Anthropic/OpenAI/Google 等）を統一 API で扱う設計を維持。各 API の安定/実験区分は公式ドキュメントで要確認。

### 仕組み / 背景（なぜこう設計されたか）
LLM アプリは「プロンプト→補完」の単発から、「計画→ツール実行→観察→再計画」を繰り返すエージェント型へ重心が移った。SDK がこのループ（いわゆる ReAct 的な反復）を標準化すると、開発者は毎回自前でループ・状態管理・ツール定義を書かずに済む。tool approval は、自律度が上がるほど「モデルが勝手に破壊的操作をする」リスクが増すことへの必然的な対策で、権限境界を人手承認で担保する設計だ。MCP フル対応は、ツール接続の断片化（プロバイダごとに独自形式）を共通プロトコルで解消する狙いで、既存 MCP サーバ資産をそのまま流用できる。reranking は、ベクトル検索の粗い上位候補を高精度モデルで並べ替えて RAG の適合度を上げる定石。トレードオフは、抽象化が増える分の学習コストと、エージェントの反復がトークン/コスト/レイテンシを押し上げる点だ。

### 実務での使いどころ・移行の勘所
Next.js を中心とした Web アプリに、チャット・検索補助・自律ワークフローを組み込む用途に好適。移行の勘所は、(1) 既存の単発生成コードは維持しつつ、自律処理が要る箇所からエージェント API を導入、(2) 破壊的/外部影響のあるツールには必ず approval を噛ませる、(3) RAG を持つなら reranking で品質を底上げ、(4) 既存 MCP サーバがあれば SDK 6 のフル対応で再利用。エージェントのコスト暴走を防ぐため、反復回数上限・タイムアウト・可観測性（DevTools/ログ）を最初から設定する。

## 📖 用語解説（このトピックとのつながり）
- **AI SDK**: 複数の LLM を統一 API で扱う TypeScript ライブラリ。Vercel 製が定番。
- **エージェント / ツール利用ループ**: モデルが道具を反復使用して目的達成する自律的処理。SDK 6 が標準化。
- **tool approval（human-in-the-loop）**: ツール実行前に人手承認を挟む安全機構。破壊的操作の抑止に。
- **MCP（Model Context Protocol）**: モデルと外部ツール/データを繋ぐ共通規格。SDK 6 がフル対応。
- **reranking（リランキング）**: 検索候補を再順位付けして適合度を上げる手法。RAG 品質向上の定石。

## 影響範囲 / 推奨アクション
TypeScript/Next.js で AI 機能を作るチームに関係。単発生成は維持しつつ自律処理からエージェント API を導入。破壊的ツールには approval を必須化し、反復上限・タイムアウト・可観測性を初期設定。RAG は reranking で品質を底上げ、既存 MCP サーバは再利用する。
