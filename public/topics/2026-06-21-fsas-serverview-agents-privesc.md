# エフサステクノロジーズ ServerView Agents for Windows に権限昇格の脆弱性（CVE-2026-27788/32325）

> トピック: 脆弱性ニュース ｜ 出典: JVN（JPCERT/CC・IPA）JVN#67883085 / JVNDB-2026-000077 ｜ 種別: 権限昇格（ローカル）

## 一行サマリー

国内サーバ管理ソフト ServerView Agents for Windows（エフサステクノロジーズ）に、サーバへログイン可能なローカル利用者が SYSTEM 権限を奪取できる権限昇格の脆弱性が2件（CVSS v3 7.8 / v4 8.5）報告された。V11.60.04 以前が対象で、最新版へのアップデートが必要。

## 🔰 初学者向け（何が・なぜ問題か）

ServerView Agents は、サーバ機（業務用の大型コンピュータ）の状態を見張る“健康管理ソフト”です。ファンの回転やディスクの寿命、温度などを監視し、異常があれば管理者に知らせます。多くの企業のサーバルームで動いている、いわば“常駐の保健室の先生”のような存在です。

今回見つかった問題は、このソフトの設定や置き場所の権限管理に不備があり、そのサーバに普通の利用者としてログインできる人が、こっそり最上位の権限（SYSTEM＝Windowsで一番強い権限）まで昇格できてしまう、というものです。

例えるなら、保健室には鍵がかかっているはずなのに、一般の生徒でも入れる戸棚にマスターキーが置きっぱなしになっていて、それを使えば校長室にも金庫にも入れてしまう、という状態です。攻撃者は最初は限られた権限しか持っていなくても、この踏み台を使ってサーバ全体を自由にできるようになります。

ポイントは「ログインできる人なら」という前提です。完全な外部の他人がいきなり悪用できるわけではありませんが、内部の不正利用者や、別の手段で一度侵入に成功した攻撃者にとっては、被害を一気に拡大する“昇格の足場”になります。サーバ監視ソフトは強い権限で動くため、こうした昇格欠陥は侵害の深刻度を大きく押し上げます。

## 🧠 専門的解説（技術詳細・メカニズム・検知対応）

### 技術的詳細（種別・前提条件）

- **CVE-2026-27788**: 重要なリソースに対する不適切なアクセス権の割り当て（CWE-732: Incorrect Permission Assignment for Critical Resource）。
- **CVE-2026-32325**: 権限管理の不備（CWE-268: Privilege Chaining／不適切な権限管理の類型）。
- **CVSS（IPA評価）**: v3 基本値 **7.8（重要）** AV:L/AC:L/PR:L/UI:N/S:U/C:H/I:H/A:H、v4 基本値 **8.5（重要）**。攻撃元区分はローカル、必要権限は低（PR:L）。
- **前提条件**: 当該製品がインストールされたサーバに**ログイン可能な利用者**であること（ローカル前提）。
- **想定される影響**: ローカル攻撃者が **SYSTEM 権限を取得**し得る（完全な機密性・完全性・可用性侵害）。
- **影響バージョン**: ServerView Agents for Windows **V11.60.04 およびそれ以前**。
- **修正版**: 開発者提供の最新版へアップデート（ベンダ告知 fsastech.com 2026/0529 参照）。回避策としてアップデート適用までのワークアラウンドが案内されている。
- **悪用状況**: 公開時点で実環境悪用の報告は確認されず（「要確認」）。報告者は株式会社ロック 細田氏、JPCERT/CC が調整。

### メカニズム（攻撃が成立する仕組み）

以下は概念レベルの説明であり、武器化可能な手順は含めない。

1. **信頼境界の所在**: SYSTEM 権限で動作する監視サービスと、その実行ファイル／設定ファイル／ディレクトリ等の“重要リソース”の保護境界。低権限ユーザーと特権サービスの間に正しいアクセス制御が必要。
2. **アクセス権の割り当て不備（27788 / CWE-732）**: 重要リソース（実行ファイル・サービス関連パス・設定等）に対して、本来は管理者のみ書き込み可とすべき箇所へ、低権限ユーザーが書き込み・置換できる緩い権限が設定されている。これが「低権限ユーザーが特権実行内容に介入できる」プリミティブを生む。
3. **権限管理の不備（32325 / CWE-268）**: 特権の継承・委譲の取り扱いに欠陥があり、低権限の操作が特権文脈へ波及する経路が存在する。
4. **昇格への橋渡し**: 攻撃者は緩い権限のリソースを書き換え／差し替えることで、SYSTEM 権限で動くサービスに自分の制御下にある内容を実行させる。低権限→特権サービスの実行内容掌握→SYSTEM 実行、という橋渡しで権限昇格が成立する。
5. **被害の増幅**: SYSTEM はWindows上の最上位権限であり、奪取されればローカルの全データ・全プロセスを制御可能。侵入後の永続化、防御無効化、横展開の起点となる。

脆弱性クラスとしては、**特権サービスが参照する重要リソースのパーミッション欠陥（insecure permissions / privilege escalation via writable service resource）**という、ローカル昇格の古典かつ実害の大きい型。原則的防御は「特権が触る資源は特権主体のみ書き込み可」とする最小権限・ACL厳格化。

### 検知・暫定緩和・恒久対策

- **検知の着眼点**: ServerView Agents 関連の実行ファイル／設定ファイルの不審な変更・差し替え、低権限ユーザーによる当該ディレクトリへの書き込み、SYSTEM 権限での想定外プロセス生成。ファイル整合性監視（FIM）とWindowsイベントログ（プロセス作成・権限変更）の突合。
- **暫定緩和（パッチ前）**: 開発者案内のワークアラウンド適用。対象サーバへのローカルログイン権限を最小化し、不要なローカルアカウントを棚卸し。重要リソースのACLを点検し、低権限書き込みを排除。
- **恒久対策**: V11.60.04 超の最新版へアップデート。適用後はファイル権限の再点検と昇格痕跡の調査。
- **運用示唆**: 監視・管理エージェントは高権限で常駐するため、ローカル昇格欠陥でも「侵害の最終段の増幅器」となる。内部脅威・多段侵入を前提に、特権サービス資源の権限堅牢化を平時から実施する。

## 📖 用語解説（このニュースとのつながり）

- **ServerView Agents for Windows**: エフサステクノロジーズ（旧富士通系）のサーバ管理／監視ソフトウェア。本件＝この常駐エージェントの権限管理不備が昇格経路に。
- **権限昇格（Privilege Escalation）**: 低い権限から高い権限を得ること。本件＝一般ユーザー→SYSTEM。
- **SYSTEM 権限**: Windowsにおける最上位のローカル権限。本件＝奪取されると端末全体を制御される。
- **CWE-732（不適切なアクセス権割り当て）**: 重要資源に緩すぎる権限を与える欠陥。本件＝27788の根。
- **CWE-268（権限管理の不備）**: 特権の継承・委譲の取り扱い欠陥。本件＝32325の根。
- **JVN / JPCERT/CC 調整**: 国内の脆弱性情報公表・開発者調整の枠組み。本件＝国内報告者からの届出をJPCERT/CCが調整し公表。

## 影響範囲 / 推奨対応

影響を受けるのは ServerView Agents for Windows V11.60.04 以前を運用する組織。悪用にはローカルログインが前提のため外部からの即時被害は限定的だが、内部不正・多段侵入時の権限昇格に直結し侵害深度を高める。推奨対応は、(1) 最新版へのアップデート、(2) 適用までの開発者案内ワークアラウンド適用、(3) 対象サーバのローカルログイン権限・ローカルアカウントの最小化、(4) 関連リソースのACL点検とファイル整合性監視。
