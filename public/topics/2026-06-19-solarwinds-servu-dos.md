# SolarWinds Serv-U CVE-2026-28318 — 未認証POSTでファイルサーバをクラッシュ、KEV登録・悪用中

> トピック: 脆弱性ニュース｜出典: The Hacker News / BleepingComputer / SecurityWeek / CISA KEV｜種別: 制御されない資源消費（DoS）／未認証

## 一行サマリー

ファイル転送サーバ SolarWinds Serv-U に、`Content-Encoding: deflate` を含む細工したPOSTリクエストで認証なしにサービスをクラッシュさせられる脆弱性（CVE-2026-28318、CVSS 7.5）がある。CISAが2026年6月5日にKEVへ登録し実環境での悪用を確認、修正版 15.5.4 HF1 への更新と外部公開の見直しが急務。

## 🔰 初学者向け（何が・なぜ問題か）

Serv-U は、社内外でファイルをやり取りするための「ファイル転送サーバ」です。取引先とのデータ受け渡しなどに使われ、インターネットに面して動いていることが多いソフトです。

今回の問題は、攻撃者が特殊な「圧縮して送るよ」という印（`Content-Encoding: deflate`）を付けたリクエストを送ると、サーバがその処理でつまずき、サービスごと落ちてしまう、というものです。ログインは不要で、リクエストを投げるだけ。情報が盗まれるタイプではありませんが、サーバが止まる＝業務が止まる（サービス妨害／DoS）という被害につながります。

例えるなら、受付窓口に「特定の折り方をした書類」を投げ込むだけで、受付係が混乱して窓口を閉めてしまうようなもの。攻撃者は鍵も身分証も要らず、窓口を繰り返し止められます。すでに実際に悪用されているため、放置は危険です。

## 🧠 専門的解説（技術詳細・メカニズム・検知対応）

### 技術的詳細（種別・前提条件）

- **CVE**: CVE-2026-28318 / **CVSS**: 7.5（High）/ 影響は可用性（A:H）中心
- **CWE**: CWE-400（Uncontrolled Resource Consumption）系のDoS
- **影響製品**: SolarWinds Serv-U（マルチプロトコル・ファイルサーバ）/ **修正版**: 15.5.4 HF1
- **悪用状況**: CISAが2026-06-05にKEV登録。連邦機関の是正期限は2026-06-19。実環境での悪用を確認。
- **前提条件**: 該当バージョンのServ-Uが到達可能なネットワークに公開されていること。認証不要。
- **トリガ**: `Content-Encoding: deflate` を伴う特別に細工されたPOSTリクエスト。なお当該サービスはこのヘッダ機能を必要としない。

### メカニズム（攻撃が成立する仕組み）

※以下は脆弱性クラスの一般原理に基づく概念説明。武器化手順・PoC・正確なペイロードは記載しない。

1. **不要機能の入力受理（攻撃面の露出）**: サービスは本来必要としない`Content-Encoding`系の入力を受理してしまう。使わない機能を受け付けること自体が攻撃面を広げている。
2. **検証なしの解凍処理**: 圧縮指定されたボディを、サイズ・整合性の上限チェックが不十分なまま展開・処理する経路に入る。ここで「制御されない資源消費」が起きうる素地が生まれる。
3. **資源枯渇・例外による停止**: 展開処理が過大なメモリ/CPUを消費する、もしくは想定外データで未処理例外に至り、サービスプロセスがクラッシュする。これが可用性喪失（DoS）の本体。
4. **未認証・低コストでの反復**: 認証境界の手前でトリガできるため、攻撃者は単一の軽量リクエストでサービスを落とし、再起動のたびに繰り返せる。可用性に対する継続的な脅威となる。
   - 一般原理: 「使わない入力を受理 → 上限チェックなしの処理 → 資源枯渇/例外 → クラッシュ」はDoSの典型クラス。根治には入力の早期拒否と資源上限の強制が必要。

### 検知・暫定緩和・恒久対策

- **検知の着眼点**: `Content-Encoding`（特に`deflate`）ヘッダを伴うServ-Uへの異常POST、サービスの突然のクラッシュ・再起動の反復、同一送信元からの短時間の連続リクエスト、プロセス監視上の異常終了。
- **暫定緩和**: アクセスを既知のIPに限定し、当該サービスが不要とする`content-encoding`を含むリクエストを境界（WAF/リバースプロキシ）で遮断。
- **恒久対策**: 15.5.4 HF1 へ更新。インターネット直結を見直し、必要な相手のみに限定（許可リスト）する。

## 📖 用語解説（このニュースとのつながり）

- **Serv-U**: SolarWinds社のファイル転送サーバ。外部公開されやすく、停止すると業務影響が大きい。
- **DoS（サービス妨害）**: サービスを停止・過負荷にして使えなくする攻撃。本件は情報窃取ではなく「止める」タイプ。
- **Content-Encoding: deflate**: 本文を圧縮して送る際の指定。本件ではこの処理経路がクラッシュの引き金になった。
- **CWE-400（制御されない資源消費）**: 入力に応じて資源（CPU/メモリ）が際限なく消費される欠陥クラス。
- **KEV**: 実際に悪用が確認された脆弱性カタログ。是正期限が設定され、優先対応の指標になる。

## 影響範囲 / 推奨対応

影響は、修正前バージョンの SolarWinds Serv-U を稼働し、ネットワークに到達可能なすべての環境。特にインターネット公開のファイルサーバは、業務停止リスクが直撃する。推奨対応は、ただちに 15.5.4 HF1 へ更新し、アクセス元を許可リストで限定、境界で不要な`content-encoding`を遮断すること。連邦機関でなくとも、KEV登録・悪用確認を踏まえ最優先で扱うべき。
