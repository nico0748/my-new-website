# Windows カーネルに CVSS 9.8 の未認証RCE（CVE-2026-45657）— TCP/IP処理のUse-After-FreeでSYSTEM実行・ワーム化の懸念

> トピック: 脆弱性ニュース ｜ 出典: Microsoft MSRC / Zero Day Initiative / The Hacker News / Tenable ｜ 種別: 解放後使用（Use-After-Free, CWE-416）に起因する未認証リモートコード実行（RCE）

## 一行サマリー

2026年6月のWindows月例更新で修正された中で最も危険とされるのが、Windowsカーネルの未認証RCE CVE-2026-45657（CVSS 9.8）。カーネルのTCP/IP処理におけるUse-After-Freeに起因し、ネットワーク経由・認証不要・ユーザー操作不要でSYSTEM権限のコード実行に至る。Zero Day Initiativeが「今月のバグ」と評し、**ワーム化（自己増殖）の懸念**も指摘される。公開・悪用はまだ確認されていないが、世界中でパッチ解析が進む中、6月更新の最優先適用が必須。

## 🔰 初学者向け（何が・なぜ問題か）

Windowsの最も奥にある中核部分を「カーネル」と呼びます。その中で、インターネット通信を処理する「**TCP/IP**」という基盤機能に欠陥が見つかりました。TCP/IPは、あらゆるネット通信の土台です。ここに穴があると、通信を受け取るだけで攻撃が成立しかねません。

今回の問題は **Use-After-Free（解放後使用）** という、「もう片付けて返したはずの作業机を、まだ自分のもののように使い続ける」タイプのメモリ破壊です。攻撃者はネットワーク越しに細工した通信を送るだけで、この混乱を引き起こし、最終的にWindowsの**最高権限であるSYSTEM**（管理者以上の、システムそのものの権限）で好きなプログラムを動かせます。

とりわけ恐ろしいのは2点です。1つは、**ログインも、利用者のクリックも一切不要**なこと。通信が届く位置に攻撃者がいれば成立します。もう1つは、専門家が指摘する**「ワーム化」の可能性**——感染した端末が次の端末へ自動で攻撃を広げ、ネットワーク内を雪だるま式に侵食する恐れです。過去の大規模感染（WannaCry等）も同種の"操作不要・自己増殖"型でした。今のところ実際の悪用は確認されていませんが、世界中の研究者がパッチを解析して攻撃手法を探っている最中で、時間との勝負です。

## 🧠 専門的解説（技術詳細・メカニズム・検知対応）

### 技術的詳細（種別・前提条件）

- **CVSS**: 9.8（Critical）。**未認証・ネットワーク経由・低複雑度・ユーザー操作不要**で悪用可能。
- **CWE分類**: CWE-416（Use-After-Free）。**WindowsカーネルのTCP/IP処理**における解放後使用。
- **前提条件**: 攻撃者が標的へネットワーク到達できること。認証・ユーザー操作は不要。
- **影響バージョン**: Windows 11（23H2〜26H1）、Windows Server 2022 / 2025（Server Coreを含む）。正確な対象ビルドはMSRC更新ガイドで要確認。
- **修正版**: **2026年6月の月例セキュリティ更新**で修正。Windows Update / WSUS / 資産管理ツールで該当KBを適用。
- **悪用状況**: 公開時点で**未公開・未悪用**（Microsoft）。ただしZDIは「今月のバグ」と評し**ワーム化**を懸念。研究者によるパッチ解析が進行中で、武器化リスクは高い。

### メカニズム（攻撃が成立する仕組み）

以下は概念レベルの説明であり、武器化可能な手順・PoC・正確なオフセット等は含めない。

1. **信頼境界の所在**: カーネルのTCP/IPスタックは、ネットワークから届く未認証のパケットを最初に処理する最前線である。パケット処理に伴うカーネルオブジェクトのライフサイクル管理が信頼境界となる。
2. **解放後使用（UAF）の発生**: 特定のシーケンスでパケット／接続状態を処理する際、参照中のカーネルオブジェクトが解放される（または解放済みオブジェクトが参照される）。攻撃者は細工した通信でこの状態を意図的に誘発する。
3. **カーネルメモリ操作プリミティブの獲得**: 解放されたスロットを攻撃者制御の内容で再占有し、カーネル空間のポインタ・データを操作する手がかりを得る。カーネル内UAFは強力な悪用基盤となりうる。
4. **SYSTEM権限でのコード実行**: カーネルは最高特権（SYSTEM/カーネル権限）で動作するため、制御奪取はそのまま**システム全体の掌握**を意味する。ユーザー権限の壁を経由しない。
5. **ワーム化の条件**: 「未認証・ネットワーク経由・操作不要」という3条件が揃うため、感染端末が同種の脆弱な端末へ自動で攻撃を伝播させる**自己増殖（ワーム）**が理論上成立しうる。これが被害を指数的に拡大させる懸念点。

脆弱性クラスとしては、「ネットワークスタック内のオブジェクト・ライフサイクル管理ミスに起因するカーネルUAF」であり、リモート・無操作で最高権限を奪える点で最も危険度の高い類型に属する。

### 検知・暫定緩和・恒久対策

- **検知の着眼点**: カーネルのクラッシュ（BSOD）・bugcheckの急増、TCP/IP関連の異常通信パターン、SYSTEM権限での予期しないプロセス生成や横方向通信。ネットワークIDS/IPSでの異常パケット検出。
- **暫定緩和（パッチ前）**: 境界・ホストファイアウォールで**不要なポート／プロトコルを遮断**し攻撃面を縮小。インターネットから直接到達可能なWindowsホストの露出を最小化。ネットワークセグメンテーションで横展開を抑制。
- **恒久対策**: **2026年6月月例更新の即時適用**（クライアント・サーバー双方、Server Coreも対象）。再起動を伴うため計画的かつ迅速に。
- **運用示唆**: "操作不要・未認証・ネットワーク経由"のカーネルRCEは最優先パッチ対象。ワーム化を想定し、内部セグメント間のフィルタリングと迅速な全社展開を準備する。

## 📖 用語解説（このニュースとのつながり）

- **カーネル / TCP/IP**: OSの中核と通信基盤処理。意味＝「OSの心臓部のネット処理」。本件＝ここに未認証RCEの穴。
- **Use-After-Free（CWE-416）**: 解放済みメモリを使う欠陥。本件＝カーネルメモリ操作の起点。
- **SYSTEM権限**: Windowsの最高権限。本件＝奪取＝システム全体掌握。
- **ワーム化（wormable）**: 操作不要で自己増殖する性質。本件＝感染が連鎖拡大しうる懸念。
- **月例更新（Patch Tuesday）**: Microsoftの定例セキュリティ更新。本件＝6月分で修正済み。

## 影響範囲 / 推奨対応

影響を受けるのは、Windows 11（23H2〜26H1）および Windows Server 2022/2025（Server Core含む）。インターネットに面した、あるいは攻撃者が到達しうるネットワーク上のWindowsホスト全般が潜在的標的。推奨対応は、(1) 2026年6月月例更新の即時適用（クライアント・サーバー双方）、(2) 不要ポート/プロトコルの遮断と露出最小化、(3) 内部セグメンテーションによる横展開抑制、(4) カーネルクラッシュ・異常通信・SYSTEMプロセス生成の監視。
