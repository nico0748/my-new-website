# PHP 8.5 — パイプ演算子 |> と URI 拡張、TAILCALL VM

> トピック: PHP 8.5｜出典: PHP.net 公式 / Zend / php.watch｜種別: メジャーリリース（2025-11-20、8.5.8 系が現行）

## 一行サマリー

PHP 8.5（2025年11月20日）は、関数を連鎖させて読みやすく書けるパイプ演算子 `|>` を目玉に、URI 拡張、クローン時のプロパティ変更、配列の先頭/末尾値を取る関数、そして再帰のスタック負荷を減らす TAILCALL VM を追加した。関数型スタイルと日常的な使い勝手を底上げするリリース。

## 🔰 初学者向け（何が・なぜ重要か）

PHP は WordPress や多くの Web サイト・EC を支える定番のサーバー言語です。8.5 で最も話題なのが「パイプ演算子 `|>`」。これは、ある処理の結果を次の処理へ「そのまま流し込む」書き方を可能にします。従来は `c(b(a($x)))` のように内側から外側へ読む入れ子になりがちで、順番が直感に反していました。パイプなら `$x |> a(...) |> b(...) |> c(...)` と、左から右へ「起きる順」に読めます。データを工場のベルトコンベアに載せ、工程を順に通していくイメージです。

ほかにも、配列（値の並び）の「最初の値」「最後の値」を一発で取る関数が標準で入り、これまで小さな回り道が必要だった作業が素直になりました。URL を安全に扱う「URI 拡張」も加わり、URL の分解・組み立てを標準機能でこなせます。

内部では「TAILCALL VM」という仕組みで、再帰的な処理（自分自身を呼び返すパターン）のメモリ消費を抑える改良も入りました。総じて「大きな革命」より「毎日のコードが素直で速くなる」実用アップデートです。

## 🧠 専門的解説（技術詳細・設計・使いどころ）

### 技術的詳細（何が変わったか）

- **パイプ演算子 `|>`**: 左辺の値を右辺の callable に渡して連鎖できる。ネストした関数呼び出しを直線的な流れに変換。8.5 で最も注目の機能。
- **URI 拡張**: URL/URI の解析・構築を標準で提供。
- **クローン時プロパティ変更**: `clone` の際にプロパティを書き換え可能に（イミュータブルなオブジェクトの「一部だけ違う複製」を作りやすい）。
- **配列関数**: 配列の先頭値・末尾値を取得するネイティブ関数を追加。
- **TAILCALL VM**: VM レベルで末尾呼び出し最適化を活用し、再帰パターンのスタックオーバーヘッドを削減。
- リリースは 2025-11-20。以後 8.5.8 等のセキュリティパッチ版が現行。

### 仕組み / 背景（なぜこう設計されたか）

パイプ演算子は関数型言語（F#、Elixir 等）で定番の構文で、「データ変換のパイプライン」を宣言的に書けるのが利点。PHP は近年、可読性と関数合成のしやすさを求める声に応え、`|>` を言語構文として導入した。内側から外側へ読む入れ子（deep nesting）は認知負荷が高く、変換の順序を追いにくい——これを左→右の逐次表現に置き換えることで、コードレビューやデバッグの負担を下げる。クローン時プロパティ変更は、値オブジェクト/イミュータブル設計で「1フィールドだけ差し替えた複製」を安全に作る現実的なニーズへの対応。TAILCALL VM は、コンパイラの末尾呼び出し最適化を PHP VM が活かすことで、再帰的アルゴリズムのスタック消費を抑える下回りの改善で、言語表現の変化ではなく実行効率の底上げに当たる。トレードオフとしては、`|>` の導入で複数の書き方が併存し、チーム内スタイルの統一が新たな論点になる点がある。

### 実務での使いどころ・移行の勘所

`|>` は、バリデーション→整形→変換のような一連のデータ処理を読みやすく書ける場面で効果的。既存の入れ子呼び出しを機械的にパイプへ書き換えるより、可読性が上がる箇所に絞って導入し、コーディング規約で使いどころを定めるのが実務的。URI 拡張は、これまで自前実装やライブラリに頼っていた URL 処理を標準へ寄せられる。8.5 への移行は 8.x 系のセマンティクスに沿うため比較的軽いが、非推奨・削除項目の確認と、依存フレームワーク（Laravel 等）の 8.5 対応表明を待って上げるのが安全。運用ではセキュリティパッチ版（8.5.8 系）へ追随すること。

## 📖 用語解説（このトピックとのつながり）

- **パイプ演算子 `|>`**: 左辺の値を右辺の関数へ流し込み、変換を左→右で連鎖させる構文。8.5 の目玉。
- **callable**: 呼び出し可能なもの（関数・メソッド等）。`|>` の右辺に置く対象。
- **末尾呼び出し最適化（TCO）**: 関数の最後で別関数を呼ぶ形をスタックを増やさず実行する最適化。TAILCALL VM が活用。
- **イミュータブル / clone**: 変更不可オブジェクトと、その「一部だけ違う複製」を作る操作。8.5 でクローン時変更が可能に。

## 影響範囲 / 推奨アクション

- 対象: PHP を使う Web/EC バックエンド、Laravel 等のフレームワーク利用者。
- アクション: `|>` は可読性が上がる箇所に絞って導入しスタイルを規約化。URL 処理は URI 拡張へ寄せる検討。移行はフレームワークの 8.5 対応表明を待ち、非推奨/削除項目を確認。セキュリティパッチ版へ追随。
