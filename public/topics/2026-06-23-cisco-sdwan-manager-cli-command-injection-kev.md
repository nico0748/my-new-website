# Cisco Catalyst SD-WAN Manager のCLIにroot権限コマンドインジェクション（CVE-2026-20245）— KEV登録・実環境で悪用、専用修正なし

> トピック: 脆弱性ニュース ｜ 出典: Cisco Security Advisory / CISA KEV / The Hacker News ｜ 種別: 出力エスケープ不備＋入力検証不備に起因するコマンドインジェクション（CWE-116/CWE-78系）→root権限昇格

## 一行サマリー

広域ネットワークを集中管理する Cisco Catalyst SD-WAN Manager（旧vManage）のCLIに、netadmin権限を持つ攻撃者が細工ファイルを与えることで**root権限で任意コマンドを実行できる**脆弱性 CVE-2026-20245 が見つかり、CISAのKEV（既知悪用脆弱性）に登録され実環境悪用が確認されている。本CVE専用のパッチは出ておらず、**先行修正（CVE-2026-20182）を含むバージョンへのアップグレード**で対処する必要がある。

## 🔰 初学者向け（何が・なぜ問題か）

会社の拠点（本社・支社・店舗）が増えると、それぞれの回線やルーターを1台ずつ手で設定するのは大変です。そこで使われるのが **SD-WAN** という仕組みで、全拠点のネットワークを**1つの管理画面からまとめて操作**できるようにします。その司令塔が Cisco Catalyst SD-WAN Manager（昔のvManage）です。ここを握れば、会社中のネットワーク機器に指示を出せる、いわば「ネットワークの中央管制室」です。

今回の問題は、その管制室の操作画面（CLI＝コマンド入力画面）が、利用者から渡されたファイルの中身を**十分に点検せずにシステムの命令として実行してしまう**点にあります。攻撃者があらかじめ管理者寄りの権限（netadmin）を持っていれば、細工したファイルを送り込むことで、本来は許されていない**最高権限（root）でのコマンド実行**にまで権限を引き上げられます。

たとえるなら、管制室に入れる中堅スタッフ（netadmin）が、点検書類に紛れ込ませた裏の指示書を読み上げさせることで、本来は所長しか押せない「全システム最高権限のボタン」を押させてしまうイメージです。しかもこの欠陥は**実際に悪用が確認**されており、米政府機関は早急な対処を命じています。中央管制室を奪われると、全拠点のネットワークが攻撃者の支配下に入りかねません。

## 🧠 専門的解説（技術詳細・メカニズム・検知対応）

### 技術的詳細（種別・前提条件）

- **CVSS**: 7.8（High）。**ローカル・認証済み**だが、結果がroot権限取得のため影響は大きい。
- **CWE分類**: 出力のエンコード/エスケープ不備（CWE-116 / Cisco表記「Improper Encoding or Escaping of Output」）と、ユーザー入力の検証不足によるOSコマンドインジェクション（CWE-78）。
- **前提条件**: 攻撃者が対象システム上で **netadmin 権限**を保有していること。完全な未認証ではないが、権限昇格（netadmin→root）として悪用される。攻撃ベクトルはCLIへの**細工ファイル供給**。
- **影響製品**: Cisco Catalyst SD-WAN Manager。関連で SD-WAN Controller / Validator にも権限昇格系の注意が及ぶ（Cisco Advisory 参照）。
- **修正版・対応**: 本CVE個別のパッチはなく、Ciscoは **2026年5月14日にリリースされた CVE-2026-20182 向け修正を含むSD-WANソフトウェアへのアップグレード**を推奨。正確な固定バージョンはCisco公式で要確認。
- **悪用状況**: **CISA KEV登録（2026年6月9日）／実環境で活発に悪用中**。FCEB（米連邦民政機関）に期限付きの是正が命じられた。

### メカニズム（攻撃が成立する仕組み）

以下は概念レベルの説明であり、武器化可能な手順・PoC等は含めない。

1. **信頼境界の所在**: SD-WAN Manager のCLIは、netadminなどの**運用権限**と、システム内部の**root権限**の間にある境界に立つ。CLIが受け取るファイルやパラメータは「権限の低い側からの入力」として本来は無害化（サニタイズ）されるべき。
2. **エスケープ/検証の不備**: CLIが供給されたファイル内容をOSコマンド文字列として組み立てる際、**出力エスケープと入力検証が不十分**なため、ファイル内に仕込まれたコマンド区切りや制御要素がそのままシェルに渡る。これがコマンドインジェクションのプリミティブとなる。
3. **権限昇格への橋渡し**: 注入されたコマンドは、CLI処理を担う**root権限のプロセス文脈**で実行される。結果として netadmin から root へと権限が一段引き上げられ、攻撃者は管理アプライアンスを完全掌握する。
4. **中央管制点としての波及**: SD-WAN Manager は全拠点機器のポリシー・構成を統制するため、ここのroot奪取は**全WANの設定改ざん・トラフィック傍受・横展開**に直結する。単一機器の侵害が広域ネットワーク全体の侵害へ拡大する。

脆弱性クラスとしては、「特権ヘルパ（root文脈で動くCLI処理）が低権限入力を不適切に扱うことによるコマンドインジェクション型の権限昇格」に該当する。入力の無害化境界が権限境界と一致していないのが本質的な欠陥。

### 検知・暫定緩和・恒久対策

- **検知の着眼点**: SD-WAN Manager上での**予期しないroot権限プロセスの生成**、CLI経由の不審なファイル操作・設定変更、新規アカウント作成や権限付与、管理アプライアンスからの異常なアウトバウンド通信。管理プレーンの監査ログ・コマンド履歴をSIEMへ集約し相関監視。
- **暫定緩和（パッチ前）**: netadmin相当の**特権アカウントを最小化**し、共有・過剰付与を排除。管理インターフェースをインターネットから遮断し、到達経路を管理セグメント／許可IPに限定。多要素認証と特権アクセス管理（PAM）で運用者操作を統制。
- **恒久対策**: Ciscoが指示する修正済みSD-WANソフトウェア（CVE-2026-20182向け修正を含むビルド）へ**即時アップグレード**。KEV登録済みのため、期限を意識した最優先対応とする。適用後は侵害有無のハント（不審プロセス・アカウント・設定差分）を実施。
- **運用示唆**: ネットワーク中央管制系（SD-WAN Manager等）は「奪取されると全社ネットが崩れる単一点」。特権最小化・管理プレーン分離・KEV連動の緊急パッチ運用を恒常原則化する。

## 📖 用語解説（このニュースとのつながり）

- **SD-WAN / Catalyst SD-WAN Manager（旧vManage）**: 広域網を集中管理する仕組みと、その司令塔。意味＝「ネットワークの中央管制室」。本件＝ここがroot奪取される。
- **netadmin / root**: 運用管理権限と、システム最高権限。本件＝netadminからrootへ昇格されるのが被害の核。
- **コマンドインジェクション（CWE-78）**: 入力に紛れた命令がそのままOSで実行される欠陥。本件＝細工ファイル経由で発生。
- **出力エスケープ不備（CWE-116）**: 特殊文字を無害化せずに使う欠陥。本件＝Ciscoが示す根本種別。
- **CISA KEV**: 実際に悪用が確認された脆弱性のカタログ。本件＝登録済みで対応の緊急度が最上位。

## 影響範囲 / 推奨対応

影響を受けるのは、Cisco Catalyst SD-WAN Manager を運用する全組織。広域ネットワークの中央管制点であるため、root奪取は全拠点機器への波及・設定改ざん・通信傍受・横展開に直結する。推奨対応は、(1) Cisco指示の修正済みSD-WANソフトウェア（CVE-2026-20182向け修正を含む）への即時アップグレード、(2) netadmin等の特権アカウント最小化とMFA/PAM適用、(3) 管理プレーンのインターネット露出排除と到達経路限定、(4) KEV登録を踏まえた期限付き是正と侵害ハントの実施。
