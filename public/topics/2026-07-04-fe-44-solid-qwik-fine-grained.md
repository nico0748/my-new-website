# Solid / Qwik の細粒度リアクティビティ — TanStack が横断サポート

> トピック: 開発トピック｜出典: TanStack｜種別: frontend

## 一行サマリー
Solid と Qwik は仮想 DOM に頼らない細粒度リアクティビティで高性能を実現するフレームワークで、TanStack がこれらを横断サポートすることで採用の敷居が下がっている。

## 🔰 初学者向け（何が・なぜ重要か）
React などの多くのフレームワークは、状態が変わると「コンポーネント全体を作り直して差分を比較する（仮想 DOM）」方式で画面を更新します。丁寧ですが、規模が大きくなると無駄な再計算が増えます。

Solid と Qwik は「細粒度リアクティビティ」という別のアプローチを取ります。これは「変わった値に直接ぶら下がっている画面の一点だけを更新する」やり方です。たとえるなら、部屋の模様替えのたびに部屋全体を撮り直して比べるのではなく、動いた家具だけをその場で動かすイメージです。無駄が少なく、非常に高速に動きます。

Qwik はさらに「resumability（再開可能性）」という独自の武器を持ち、サーバーで作った画面をブラウザで最小限の JavaScript だけで動かせるようにします。初回表示が軽いのが特徴です。そして TanStack（Query や Router などの人気ライブラリ群）がこれらを横断サポートすることで、「React 以外でも同じ道具が使える」状況が整いつつあります。

## 🧠 専門的解説（技術詳細・設計・使いどころ）
### 技術的詳細（何が変わったか）
Solid は signal ベースの細粒度リアクティビティで、コンポーネントは初回のみ実行され以降は再実行されない。Qwik は resumability により初期 JS をほぼゼロに近づけ、必要なコードだけを遅延読み込みする。TanStack Query / Router 等がマルチフレームワークアダプタを提供し、React 以外でも共通 API を使える。各バージョンや対応状況は要確認。

### 仕組み / 背景（なぜこう設計されたか）
仮想 DOM の差分計算コストと、SSR 後のハイドレーション（全コードの再実行）が性能の壁だった。signal による直接更新と resumability はこの2つを構造的に回避する。「再レンダリングを前提にしない」設計思想。

### 実務での使いどころ・移行の勘所
コンテンツ量が多く初期表示速度が重要なサイト、大量の状態更新が走るダッシュボードで効果が出やすい。ただし React に比べエコシステム・求人・情報量は限定的。TanStack 横断サポートは移行の緩衝材になるが、フレームワーク固有の書き方の学習コストは残る。

## 📖 用語解説（このトピックとのつながり）
- **細粒度リアクティビティ**: 変更点に直結した最小単位だけ更新する方式。Solid/Qwik の核。
- **signal**: 値と依存関係を追跡するリアクティブな最小単位。Solid の基盤。
- **resumability**: サーバー描画状態をブラウザで再開する Qwik の仕組み。ハイドレーション不要に近づける。
- **TanStack**: フレームワーク横断のライブラリ群。Solid/Qwik 採用の橋渡し役。

## 影響範囲 / 推奨アクション
性能を最優先する新規プロジェクトに検討価値がある。既存 React 資産が大きい場合は無理に移行せず、TanStack 共通ライブラリで将来の選択肢を確保しておくのがよい。
