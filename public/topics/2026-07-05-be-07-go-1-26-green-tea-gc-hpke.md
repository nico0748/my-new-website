# Go 1.26 — Green Tea GC が既定化、crypto/hpke と実験的 SIMD を追加

> トピック: Go 1.26 リリース｜出典: go.dev 公式リリースノート｜種別: 言語/ランタイム（メジャーアップデート）

## 一行サマリー

2026年2月にリリースされた Go 1.26 は、これまで実験扱いだった「Green Tea ガベージコレクタ」を既定で有効化し、cgo のオーバーヘッドを約30%削減、ポスト量子ハイブリッドに対応した `crypto/hpke` パッケージや実験的な `simd/archsimd`・`runtime/secret` を追加した。Go 1 互換性を保ちつつ、ランタイムと標準ライブラリの底上げが中心。

## 🔰 初学者向け（何が・なぜ重要か）

Go はサーバー開発でよく使われる言語で、「メモリの後片付け（ガベージコレクション＝GC）」を自動でやってくれます。ただ GC は動くたびにプログラムを少し止めるので、その止まる時間や無駄をいかに減らすかが性能の鍵でした。

Go 1.26 では「Green Tea GC」という新しい後片付けの方式が**標準で使われる**ようになりました。たとえるなら、掃除の段取りをより効率的なやり方に切り替えたイメージで、同じ仕事でも手が空く時間が増え、アプリが速く・滑らかに動きやすくなります。

加えて、通信を暗号化する新しい標準部品（`crypto/hpke`）が入り、しかも将来の「量子コンピュータでも破られにくい」方式に対応しています。今すぐ全員が使うものではありませんが、長く使うシステムのセキュリティを先取りする備えです。開発者が特別なことをしなくても、Go を上げるだけで恩恵を受けられるのが今回の性格です。

## 🧠 専門的解説（技術詳細・設計・使いどころ）

### 技術的詳細（何が変わったか）

- **Green Tea GC が既定有効化**: 1.25 で実験導入されたガベージコレクタが 1.26 で既定に。GC 効率の改善を狙う。
- **cgo オーバーヘッド約30%削減**: C 呼び出しの基本コストを低減。スライスのバッキングストアをスタック割り当てできる状況も拡大し、性能向上。
- **言語の小改善**: 組み込み `new` が式を引数に取り初期値を指定可能に。ジェネリック型が自身の型パラメータリスト内で自己参照可能に。
- **crypto/hpke（新規）**: RFC 9180 の Hybrid Public Key Encryption を実装。ポスト量子ハイブリッド KEM をサポート。
- **実験パッケージ**: `GOEXPERIMENT=simd` で有効化する `simd/archsimd`（アーキ固有 SIMD 操作）、機密情報の一時領域を安全に消去する `runtime/secret`、`runtime/pprof` の goroutineleak プロファイル。
- **プラットフォーム**: 1.26 は macOS 12 Monterey で動く最後のリリース（1.27 で macOS 13 以上が必要）。

### 仕組み / 背景（なぜこう設計されたか）

Go の GC は「低レイテンシ（停止時間の短さ）」を最優先に設計された並行マーク&スイープ方式で、スループットとのバランスが常に課題だった。Green Tea GC は、メモリのスキャン局所性やワークの分配を見直し、キャッシュ効率とスケーラビリティを改善する方向の再設計とされる。既定化は「十分な検証を経て安全に切り替えられる」段階に達したことを意味する。トレードオフとして、GC の挙動が変わるためレイテンシ/スループット特性がワークロード依存で微妙に動く可能性があり、性能クリティカルなサービスはベンチで確認するのが望ましい。

`crypto/hpke` の追加は、暗号の世界が「ポスト量子移行」に舵を切っている潮流の反映だ。HPKE は公開鍵で共有鍵を封じ、その鍵で本文を対称暗号化する「ハイブリッド」方式で、鍵確立部分にポスト量子 KEM を混ぜることで、将来量子計算機が実用化しても遡って復号されにくくする（harvest-now-decrypt-later 対策）。SIMD や `runtime/secret` の実験導入は、数値計算の高速化や機密データのメモリ残留対策という、特定用途での性能・安全性ニーズに応えるもの。

### 実務での使いどころ・移行の勘所

Go 1.26 への更新は Go 1 互換のため基本的に低リスクだが、GC 既定変更があるため、レイテンシに敏感なサービスは更新前後で p99 停止時間とスループットを計測すること。長寿命システムや規制産業では `crypto/hpke` のポスト量子ハイブリッドを、鍵交換のロードマップに組み込む価値がある。SIMD/`runtime/secret` は実験段階のため、用途が明確な数値処理や機密取り扱いに限定して評価する。macOS 12 対応が最後になる点は CI/開発環境の更新計画に反映する。

## 📖 用語解説（このトピックとのつながり）

- **Green Tea GC**: Go 1.26 で既定化された新しいガベージコレクタ。本トピックの中心。
- **cgo**: Go から C コードを呼ぶ仕組み。今回オーバーヘッドが削減された。
- **HPKE（RFC 9180）**: 公開鍵で共有鍵を封じるハイブリッド暗号方式。`crypto/hpke` が実装。
- **ポスト量子 / KEM**: 量子計算機に耐える鍵カプセル化機構。HPKE に組み込める。
- **SIMD**: 1命令で複数データを並列処理する仕組み。`simd/archsimd` が実験提供。

## 影響範囲 / 推奨アクション

Go サーバー全般が更新だけで GC 効率と cgo コストの改善を受けられる。レイテンシ敏感なサービスは更新前後の停止時間を計測。長寿命/規制系は `crypto/hpke` のポスト量子対応を鍵管理計画に検討。実験パッケージは用途限定で評価する。
