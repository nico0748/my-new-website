# @starting-style が Baseline 入り ― display:none からの入場アニメを JS なしで

> トピック: 開発トピック｜出典: modern-css｜種別: frontend

## 一行サマリー
CSS の **`@starting-style`** ルールがクロスブラウザ Baseline に到達（時期は要確認）。`display: none` から表示される要素や、新しく DOM に追加された要素の「入場アニメーション」を、JavaScript を使わず CSS だけで滑らかに実現できる。

## 🔰 初学者向け（何が・なぜ重要か）
モーダルやトーストが「フワッ」と現れる演出。一見簡単そうですが、CSS には長年ひとつの壁がありました。要素がまだ画面に「存在していない」（`display: none`）状態から現れる瞬間には、アニメーションの「開始地点の見た目」を指定する場所がなかったのです。そのため、いったん JavaScript で少し待ってからクラスを付け替える、といった小細工が必要でした。

`@starting-style` は、まさにこの「登場した瞬間の初期状態」を CSS で書ける場所を提供します。「現れる直前は透明で少し縮んでいる」と宣言しておけば、ブラウザが登場と同時にそこから通常状態へ滑らかに補間してくれます。開ける瞬間だけ手を貸していた JavaScript が不要になり、演出のコードがぐっとシンプルになります。

## 🧠 専門的解説（技術詳細・設計・使いどころ）
### 技術的詳細（何が変わったか）
- `@starting-style { ... }` 内に、要素が最初にレンダリングされる際の初期スタイルを宣言できる。
- `transition-behavior: allow-discrete` と組み合わせ、`display` や `overlay` など従来アニメ不可だった離散プロパティも遷移対象にできる。
- `popover` 属性やダイアログの表示/非表示遷移と組み合わせると、開閉アニメを完全に CSS 化できる。

### 仕組み / 背景（なぜこう設計されたか）
CSS トランジションは「変化前の値」と「変化後の値」の両方が必要ですが、`display: none` からの登場は変化前の値が存在しませんでした。`@starting-style` は「初回レンダリング時に適用する仮の開始値」を定義することでこの穴を埋めます。`allow-discrete` は、従来アニメできなかった `display` のような非連続な値も、遷移の最後（消える側）や最初（現れる側）で切り替えられるようにする補完機能です。設計思想は「演出のためだけの JS を排し、宣言的に閉じる」ことにあります。

### 実務での使いどころ・移行の勘所
- モーダル、トースト、ドロップダウン、`<dialog>`、popover の開閉アニメに最適。
- 既存の JS ベース入場アニメ（クラス付け替え）を CSS へ置換でき、依存コードを削減。
- 対応が不十分な環境ではアニメが無効化される（機能はする）ため、プログレッシブエンハンスメントとして安全に導入可能。

## 📖 用語解説（このトピックとのつながり）
- **`@starting-style`**: 要素登場時の初期スタイルを定義する CSS ルール。本トピックの主役。
- **`allow-discrete`**: `display` 等の非連続プロパティを遷移対象にする値。入退場アニメの鍵。
- **離散プロパティ**: 中間値を持たない `display` などのプロパティ。従来アニメ不可だった。
- **popover / <dialog>**: ネイティブの開閉 UI。`@starting-style` と好相性。
- **プログレッシブエンハンスメント**: 未対応環境でも壊れず、対応環境で強化する導入戦略。

## 影響範囲 / 推奨アクション
UI 演出を実装する開発者に影響。新規のモーダル/トースト演出は `@starting-style` を第一候補に。既存の JS 依存の入場演出は、対応ブラウザ範囲を確認して段階的に CSS 化を検討。
