# Kubernetes 1.37 が開発中 — GPU を「分割して割り当てる」DRA が前進

> トピック: Kubernetes 1.37（開発中・2026年8月26日リリース予定）｜出典: Kubernetes コントリビュータ情報 / VersionLog｜種別: フレームワーク メジャーリリース（開発版）

## 一行サマリー

Kubernetes v1.37 は開発サイクル中で、2026年8月26日リリース予定。目玉のひとつが Dynamic Resource Allocation（DRA）の「分割可能デバイス（partitionable devices）」対応で、1枚の GPU を複数ワークロードに柔軟に切り分けて割り当てる仕組みが前進する。現時点はアルファ段階のため本番利用は非推奨（正確な機能昇格状況は要確認）。

## 🔰 初学者向け（何が・なぜ重要か）

Kubernetes は「たくさんのコンテナ（小さなアプリの実行単位）を、大量のサーバーの上でうまく配置・管理してくれる」基盤ソフト。クラウドの標準土台になっている。

AI ブームで需要が急増しているのが「GPU の割り当て」。GPU は高価で数も限られるのに、従来の Kubernetes は「GPU 1枚をまるごと1つの処理に渡す」しかできず、小さな推論タスクに1枚まるごと占有させるのはもったいなかった。DRA（動的リソース割り当て）は、こうした特殊で高価なハードウェアを賢く配る新しい仕組みで、今回の「分割可能デバイス」対応で「GPU 1枚を複数の処理で分け合う」ことが正式な仕組みとして扱えるようになっていく。AI 推論をたくさん回す現場にとって、コスト効率に直結する重要な進化だ。

## 🧠 専門的解説（技術詳細・設計・使いどころ）

### 技術的詳細（何が変わったか）

Kubernetes の公式コントリビュータ情報および VersionLog によれば、v1.37.0 は 2026年8月26日リリース予定で、記事執筆時点はアルファ（v1.37.0-alpha.1）。継続テーマとして、GPU など「分割可能デバイス（partitionable devices）」に対する Dynamic Resource Allocation（DRA）の作業が挙げられている。DRA は従来の Device Plugin ベースの割り当てを置き換える方向の仕組みで、リソースをクレーム（ResourceClaim）として宣言し、スケジューラがそれに合致するデバイスを動的に確保する。分割可能デバイス対応は、物理 GPU を論理的に区画化して複数クレームへ割り当てる用途を想定する（各サブ機能の Alpha/Beta/GA 昇格の正確な状態は KEP と正式リリースノートで要確認）。

### 仕組み / 背景（なぜこう設計されたか）

初期 Kubernetes の GPU 割り当ては Device Plugin による「整数個のデバイスをまるごと確保」モデルで、AI 時代の多様なアクセラレータ（GPU の MIG 分割、共有、トポロジ制約付き割り当て等）を表現しきれなかった。DRA は「リソースを型付きのクレームとして宣言し、ドライバとスケジューラが動的にマッチング・確保する」という、より一般的で拡張可能なモデルへの作り直し。分割可能デバイスはその上で「1つの物理デバイスから複数の割り当て可能単位を切り出す」ことを表現できるようにする。トレードオフは、柔軟性と引き換えにスケジューリング・ドライバ実装の複雑さが増すこと、そして安定化に複数リリースを要し、機能が Alpha→Beta→GA と段階的にしか使えないこと。だからこそ本番投入は昇格状況の見極めが不可欠になる。

### 実務での使いどころ・移行の勘所

GPU 推論を大量に回す AI プラットフォーム、マルチテナントで高価なアクセラレータを共有したい基盤で価値が高い。移行の勘所は、DRA 関連機能はまだ段階的昇格の途上でありアルファ機能はフィーチャーゲートを明示的に有効化する必要があること、本番クラスタでのアルファ利用は避け、まず検証環境で GPU ドライバ（ベンダのリソースドライバ）の対応状況と合わせて評価すること。既存の Device Plugin ベース構成からの移行は、当面は並行運用しつつ DRA の安定化を待つのが現実的。リリース予定日（8月26日）や機能昇格は変動しうるため、確定情報は正式リリースノートで確認する。

## 📖 用語解説（このトピックとのつながり）

- **DRA（Dynamic Resource Allocation）**: GPU など特殊リソースをクレームとして宣言し動的に確保する仕組み。Device Plugin の後継的モデル。
- **partitionable devices（分割可能デバイス）**: 1つの物理デバイスを複数の割り当て単位に切り分けられる対象。GPU 共有の中核概念。
- **フィーチャーゲート**: 実験的機能を明示的に有効/無効化する仕組み。アルファ機能の利用に必要。
- **KEP（Kubernetes Enhancement Proposal）**: 機能提案・設計文書。正確な仕様と昇格状況の一次ソース。

## 影響範囲 / 推奨アクション

- AI 推論基盤における GPU のコスト効率・共有性が中期的に改善する見込み。基盤チームは DRA の進捗を追う価値がある。
- v1.37 はリリース前・DRA 関連は段階的昇格中。本番でのアルファ利用は避け、検証環境でベンダドライバと合わせて評価する。
- リリース日・機能昇格は変動しうるため、確定は正式リリースノート／KEP で確認する。
