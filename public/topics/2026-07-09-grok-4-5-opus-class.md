# Grok 4.5 公開 — xAI が「Opus 級」を主張、コーディング/エージェント特化の最新モデル

> トピック: Grok 4.5（2026-07-08 公開）｜出典: x.ai / TechCrunch / Artificial Analysis｜種別: 新モデルのリリース

## 一行サマリー

xAI が2026年7月8日、コーディング・エージェント・知識労働に強いとする最新モデル Grok 4.5 を一般公開した。Elon Musk は「Opus 級」と表現するが、xAI 自身が公開したベンチマークでも4指標中2勝2敗と評価はまだら。Artificial Analysis の Intelligence Index では54点で総合4位（Fable 5=60、Opus 4.8=56、GPT-5.5=55）と、トップ集団のやや下に位置する。

## 🔰 初学者向け（何が・なぜ重要か）

いま AI モデルの世界は、各社が数か月ごとに新モデルを出す激しい競争状態です。Grok は Elon Musk の会社 xAI が作る AI で、今回その最新版 4.5 が公開されました。売りは「プログラミングを書く力」と「エージェント（AI が自分で手順を考えて作業を進める使い方）」の強さです。

注目すべきは「宣伝と実測のギャップ」です。Musk は「最高峰の Opus に匹敵する」と表現しましたが、実際のベンチマーク（性能を測るテスト）を見ると、勝っている項目もあれば負けている項目もある、というのが正確なところ。第三者機関の総合スコアでは4位で、トップではありません。開発者にとっての教訓は、「AI モデルの発表は宣伝込みで受け取り、自分の用途で実際に試して判断する」こと。数字の一部だけを切り取った主張に流されないことが大切です。

## 🧠 専門的解説（技術詳細・設計・使いどころ）

### 技術的詳細（何が変わったか）

- 2026年7月8日、xAI が Grok 4.5 を一般公開（Musk の private beta 告知から11日後）。コーディング・エージェント・知識労働特化を標榜。
- ベンチマーク（xAI 公開値・第三者値の混在、要確認）:
  - DeepSWE 1.0: Grok 4.5 = 62.0%（Fable(max) 66.1%、GPT-5.5(xhigh) 64.31%、Opus 4.8(max) 55.75%）。
  - SWE Marathon 解決率（pass@1）: Grok 4.5 = 29.0%（Opus 4.8(max) 26.0% を上回る）。
  - SWE-Bench Pro 解決率: Grok 4.5 = 64.7%（Fable(max) 80.4%、Opus 4.8(max) 69.2%）。
  - Artificial Analysis Intelligence Index: 54 で総合4位（Fable 5=60、Opus 4.8=56、GPT-5.5=55）。
- xAI が選んだ4ベンチのうち、Opus 4.8 に対し2勝（DeepSWE 1.0、Terminal-Bench 2.1）2敗（DeepSWE 1.1 で6点差、SWE-Bench Pro で4.5点差）。
- 価格・コンテキスト長などの詳細は一次情報での確認を推奨（要確認）。

### 仕組み / 背景（なぜこう設計されたか）

近年のフロンティアモデル競争は「単一の総合力」から「用途別の強み」へと軸が移っている。Grok 4.5 が前面に出すのはコーディングとエージェントで、これは実務の AI 需要が「チャットの賢さ」より「タスクを最後まで自律遂行する力」へ移っていることの反映だ。SWE-Bench 系（実際の GitHub Issue を解けるか）や SWE Marathon（長時間の連続作業）といった評価が重視されるのは、エージェント用途では「1回の推論の賢さ」より「長い手順の中で失敗せずゴールへ到達する持久力」が価値を決めるためだ。

一方で今回の件は、ベンチマーク評価の難しさも示す。同じモデルでも「どのベンチを選び、どの設定（max/xhigh 等の推論予算）で測るか」で見え方が大きく変わる。ベンダーは自社が勝つ指標を選んで提示するインセンティブがあり、Grok 4.5 が「4指標中2勝」に留まるのも、有利なベンチを選んでなおその結果、という読み方ができる。トレードオフの一般原理として、推論予算（思考トークン）を積めばスコアは上がるがコストと遅延が増す。ゆえに「max 設定のスコア」は実運用のコスト制約下では再現しにくい。開発者は公表ベンチを鵜呑みにせず、自分のタスク分布・コスト予算で評価する必要がある。

### 実務での使いどころ・移行の勘所

コード生成やエージェント的な自動化を検討中なら、候補の1つとして自前タスクで A/B 比較する価値はある。ただし「Opus 級」という表現に依存せず、(1) 自分のコードベース/言語での実解決率、(2) 長時間タスクでの安定性（途中脱線・ループの有無）、(3) 単価×トークン消費の実コスト、(4) API のレート制限や可用性、を実測して判断する。既存のワークフロー（Claude/GPT 系）からの乗り換えは、ツール呼び出し（MCP 等）や既存プロンプト資産の互換性も要確認。ベンチ上位でも自分の用途で最良とは限らないため、少数の代表タスクでの比較を推奨。

## 📖 用語解説（このトピックとのつながり）

- **Grok 4.5**: xAI の最新 LLM。コーディング/エージェント特化を標榜。本トピックの主役。
- **SWE-Bench / SWE Marathon**: 実際のソフト課題を解けるか測る評価。エージェント性能の代表指標。
- **Artificial Analysis Intelligence Index**: 第三者の総合知能スコア。今回 Grok 4.5 は総合4位。
- **推論予算（max/xhigh 等）**: 思考トークンをどれだけ使うかの設定。スコアとコストのトレードオフ。
- **エージェント**: AI が手順を自律的に計画・実行する使い方。長時間の持久力が価値を決める。

## 影響範囲 / 推奨アクション

- 推奨: 「Opus 級」の宣伝に依存せず、自前タスクで実解決率・安定性・実コストを A/B 検証。数値は一次情報で再確認。
