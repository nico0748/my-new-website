# AI コーディングエージェントの収束 — Cursor / Claude Code / Codex / Antigravity が一つのスタックへ

> トピック: 開発トピック｜出典: The New Stack｜種別: ai

## 一行サマリー
2026 年前半、Cursor・Claude Code・Codex・Antigravity など主要 AI コーディングツールが、機能・使い方の面で互いに接近し「誰も計画していなかった一つのスタック」に収束しつつあると報じられている。エディタ内補完から、CLI／自律エージェント／クラウドの並列実行まで、開発者は複数ツールを役割分担で併用する運用が定着してきた。

## 🔰 初学者向け（何が・なぜ重要か）
数年前まで、AI に手伝ってもらうコーディングは「入力中に続きを提案してくれる（オートコンプリート）」が中心でした。今は違います。「この機能を作って」と頼むと、AI が自分でファイルを開き、複数箇所を編集し、テストを走らせ、直しまで進める——そんな自律的な「エージェント」型が主流になりつつあります。

この分野には Cursor（AI エディタ）、Claude Code（Anthropic の CLI 型）、Codex（OpenAI 系）、Antigravity（Google 系）など複数の有力ツールがあります。面白いのは、競い合ううちに各ツールの機能や使い勝手がどんどん似てきて、「どれを使っても近い体験」に収束してきたことです。記事はこれを「誰も計画していなかったが、結果的に一つの共通スタックになった」と表現しています。

実務では「1つに絞る」より「役割で使い分ける」のが定着しています。細かい編集はエディタ内、まとまった実装は CLI エージェント、時間のかかる作業はクラウドで並列に……といった具合です。どのツールも似てきたからこそ、乗り換えや併用がしやすくなった、というのが今の状況です。

## 🧠 専門的解説（技術詳細・設計・使いどころ）
### 技術的詳細（何が変わったか）
The New Stack は、Cursor・Claude Code・Codex・Antigravity が半年で機能的に接近し、事実上一つのスタックへ収束したと分析する。共通して見られる要素は、(1) エディタ内補完（インライン）から自律エージェント（計画→編集→テスト→修正のループ）への拡張、(2) CLI/ターミナル型ハーネスの整備、(3) クラウドでの並列タスク実行（複数サブエージェント）、(4) MCP を介した外部ツール接続、(5) サンドボックス・権限管理・Git ポリシーなどのガードレール。Google の Antigravity 2.0（I/O 2026-05-19 発表）はデスクトップアプリと標準 CLI の2面構成に再編され、並列サブエージェント・端末サンドボックス・認証情報マスキング・強化された Git ポリシーを追加した。各ツールの詳細機能とバージョンは各社一次情報で要確認。

### 仕組み / 背景（なぜこう設計されたか）
収束の背景には、基盤モデルの能力向上（長い自律作業をこなせる）と、MCP のような相互接続標準の普及がある。ツールがモデルとツール接続を共通化すると、差別化点はモデル性能・ハーネス（作業を安全に回す枠組み）・UX・価格へ移り、結果として各社の設計が似通う。並列サブエージェントは「大きなタスクを分割し、複数のエージェントが同時に別々のサブ課題を進める」設計で、壁時間の短縮を狙う。サンドボックスや Git ポリシー、認証情報マスキングは、自律度が上がるほど「意図せぬ破壊・情報漏えい」のリスクが増すことへの共通対策だ。トレードオフは、自律度と速度を追うほど監視・レビューの負荷が増し、コスト（トークン/実行時間）も膨らむ点。ツールが似通うほど、選定基準は「モデルの質×ハーネスの安全性×価格」に集約される。

### 実務での使いどころ・移行の勘所
役割分担が現実解。細粒度の編集・補完はエディタ内、機能単位の実装は CLI エージェント、長時間・大規模タスクはクラウド並列、という使い分けが定着している。ツールが似てきたぶん乗り換えコストは下がったが、(1) 権限・サンドボックス設定を最初に固める、(2) 破壊的操作・外部送信は人手承認を挟む、(3) 生成物は必ずテスト/レビューで検証、(4) コスト上限とログ（可観測性）を設定、が共通の勘所。価格体系は 2026 年前半に大きく動いた（後述の請求モデル変更参照）ため、契約更新前にコスト試算を行う。

## 📖 用語解説（このトピックとのつながり）
- **コーディングエージェント**: 計画→編集→テスト→修正を自律的に回す AI ツール。補完の次段階。
- **ハーネス（harness）**: エージェントを安全・効率よく動かす枠組み（サンドボックス、権限、Git ポリシー等）。差別化の要。
- **並列サブエージェント**: 大タスクを分割し複数エージェントで同時進行する仕組み。壁時間短縮を狙う。
- **MCP**: モデルと外部ツールを繋ぐ共通規格。ツール横断の相互運用と収束を後押し。
- **サンドボックス / 権限管理**: 実行を隔離し操作範囲を制限する安全機構。自律度上昇に伴い重要度増。

## 影響範囲 / 推奨アクション
コードを書く全チームに関係。単一ツール固定より役割分担での併用が有効。権限・サンドボックス・人手承認・コスト上限・ログを標準装備として整える。ツールが似通い乗り換えコストが下がったため、モデル質×ハーネス安全性×価格で定期的に選定を見直す。
