# CSS masonry が display:grid-lanes に決着 ― 長年の仕様論争が CSSWG 合意へ

> トピック: 開発トピック｜出典: css-tricks｜種別: frontend

## 一行サマリー
Pinterest 風の「石積み（masonry）レイアウト」をネイティブ CSS で実現する構文が、長い議論の末に **`display: grid-lanes`（呼称は要確認）**で CSSWG 合意に至った。Safari が先行実装しており、JS ライブラリなしの masonry がいよいよ現実的に。

## 🔰 初学者向け（何が・なぜ重要か）
写真ギャラリーでよく見る、高さがバラバラのカードが隙間なく詰め込まれたレイアウト——これを「masonry（石積み）」と呼びます。レンガを積むように、空いた隙間に次のカードを詰めていく配置です。これまでは CSS 標準では作れず、Masonry.js のような JavaScript ライブラリで無理やり座標計算して実現するのが定番でした。

問題は「これを CSS のどの仕組みに載せるか」で、標準化団体（CSSWG）内で数年にわたり意見が割れていました。「既存の Grid の一部にすべき」派と「別の独立した表示モードにすべき」派の対立です。今回、**`grid-lanes` という新しい表示値**として決着し、Safari が先行実装しました。決着したことで、ようやく JS なしの masonry が標準への道筋に乗った、という節目のニュースです。

## 🧠 専門的解説（技術詳細・設計・使いどころ）
### 技術的詳細（何が変わったか）
- masonry レイアウトが `display: grid-lanes`（正式名称・構文は要確認）という独立した表示モードとして CSSWG 合意。
- **Safari が先行実装**（他ブラウザは追随段階、要確認）。
- 「Grid に統合するか、独立モードにするか」という長年の設計論争が独立モード側で決着した点が重要。

### 仕組み / 背景（なぜこう設計されたか）
Grid は行と列を厳密に揃える二次元レイアウトで、行の高さが揃うのが本質でした。masonry は「列は揃えるが行は揃えない」性質で、Grid の厳密なグリッド線と概念が衝突します。これを Grid の一機能（`grid-template-rows: masonry`）にすると、Grid の他機能との組み合わせで意味が曖昧になる懸念がありました。独立した `grid-lanes` として切り出すことで、Grid の一貫性を保ちつつ masonry 固有の挙動を明確に定義できる、という判断です。トレードオフは「学ぶべき表示モードが増える」点。

### 実務での使いどころ・移行の勘所
- 画像ギャラリー、カード一覧、ダッシュボードのウィジェット配置などに有効。
- 現状は Safari 先行のため、他ブラウザ向けにはフォールバック（columns や JS）が当面必要。
- 仕様が固まりきる前の段階なので、正式名称・プロパティは変わりうる。本番採用は Baseline 化を待つのが安全。

## 📖 用語解説（このトピックとのつながり）
- **masonry レイアウト**: 高さの異なる要素を隙間なく積むレイアウト。本トピックの対象。
- **grid-lanes**: masonry を担う新表示モード（呼称は要確認）。今回の決着点。
- **CSSWG**: CSS の仕様を策定する作業部会。合意主体。
- **CSS Grid**: 二次元レイアウト機能。masonry を統合するか論争の相手だった。
- **Interop**: ブラウザ横断で挙動を揃える取り組み。masonry も対象領域。
