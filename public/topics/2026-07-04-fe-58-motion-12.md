# Motion 12.42 ― 旧 Framer Motion が独立、WAAPI ハイブリッドエンジンで120fpsアニメを標準化

> トピック: 開発トピック｜出典: motion.dev / GitHub motiondivision/motion｜種別: frontend

## 一行サマリー
アニメーションライブラリ「Framer Motion」は独立プロジェクト「Motion」へと改称され、import も `framer-motion` から `motion/react` へ移行した。最新版 12.42.2（2026-07 時点）は Web Animations API と ScrollTimeline を活かすハイブリッドエンジンで、ブラウザネイティブに近い120fpsのアニメーションを実現する。

## 🔰 初学者向け（何が・なぜ重要か）
Web 画面の「ふわっと動く」「スクロールに合わせて要素が現れる」といった動きを、簡単なコードで付けられるのが Motion（旧 Framer Motion）です。これまで Framer という会社の名前が付いていましたが、独立したプロジェクトになり、名前も import する場所も `motion/react` に変わりました。引っ越しで住所が変わったようなものです。

技術面での要点は「動きの計算をどこでやるか」です。従来は JavaScript（アプリのコード）が毎フレーム計算していましたが、Motion はできる部分をブラウザ本体の機能（Web Animations API）に任せます。厨房の下ごしらえをプロの機械に任せるイメージで、アプリ側の負担が減り、カクつかない滑らかな動き（120fps）が出しやすくなります。

## 🧠 専門的解説（技術詳細・設計・使いどころ）
### 技術的詳細（何が変わったか）
- **Framer Motion → Motion** に改称（2025 年に独立プロジェクト化）。import は `framer-motion` → `motion/react`。
- 最新版は **12.42.2**（2026-07-04 時点で「4日前公開」＝ 2026-06 末、要確認）。
- 直近の 12.x 更新例: **12.39.0（2026-05-18）** で animation sequence の `repeatType` / `repeatDelay` 対応、drag 挙動・React 19 互換・scroll 処理の改善。**12.35.2（2026-03-09）** で `styleEffect` のファイルサイズ削減。
- ハイブリッドエンジン: **Web Animations API (WAAPI)** と **ScrollTimeline** をネイティブ活用し、120fps を狙う。

### 仕組み / 背景（なぜこう設計されたか）
アニメーションの伝統的課題は「メインスレッドの奪い合い」です。JS が毎フレーム値を計算してスタイルを書き換える方式は、他の処理と CPU を取り合い、重い画面ではコマ落ち（jank）します。Motion のハイブリッドエンジンは、可能なアニメーションをブラウザのコンポジタスレッド側で走る WAAPI に委譲し、スクロール連動は ScrollTimeline に任せることで、メインスレッドの負荷を下げます。これがネイティブ相当の滑らかさの源泉です。

トレードオフとして、WAAPI で表現できないもの（レイアウトアニメーションや複雑な物理挙動）は引き続き JS 側で処理する必要があり、Motion は「WAAPI に載せられるものは載せ、載せられないものは JS」というハイブリッド判断を内部で行います。改称に伴う import パス変更は、既存コードの一括置換という移行コストを生みます。

### 実務での使いどころ・移行の勘所
- 既存 `framer-motion` 利用者は import を `motion/react` に置換（公式の upgrade guide あり）。挙動は概ね互換だが、drag/scroll 周りの変更点は要検証。
- スクロール駆動アニメ（ScrollTimeline）を多用する LP・プロダクトサイトで恩恵が大きい。
- 低スペック端末での jank 低減が主目的なら、レイアウトアニメの多用は避け、WAAPI に載る transform/opacity 中心に設計する。

## 📖 用語解説（このトピックとのつながり）
- **Motion（旧 Framer Motion）**: React/JS 向けアニメーションライブラリ。本トピックの主役。
- **Web Animations API (WAAPI)**: ブラウザ標準のアニメーション API。メインスレッド外で動かせるのが利点。
- **ScrollTimeline**: スクロール量を時間軸にできる標準機能。スクロール駆動アニメの基盤。
- **jank（コマ落ち）**: フレーム落ちによるカクつき。メインスレッド負荷が主因。
- **120fps**: 高リフレッシュレート端末での滑らかな描画目標。ネイティブ委譲で到達しやすい。

## 影響範囲 / 推奨アクション
React でアニメーションを使う多くのフロントエンド開発者に関係する。改称による import 変更は早めに追随しておくと将来の更新が楽。パフォーマンス重視のサイトでは、WAAPI に載るプロパティ中心の設計へ見直すことで体感を改善できる。
