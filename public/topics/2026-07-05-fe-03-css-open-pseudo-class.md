# CSS :open 擬似クラスが Baseline 入り — details/dialog の開閉を CSS だけで装飾

> トピック: `:open` 擬似クラスが Baseline Newly available（Safari 26.5 対応で全ブラウザ揃う）｜出典: web.dev（2026年5月ダイジェスト）｜種別: Web 標準

## 一行サマリー

Safari 26.5 が `:open` 擬似クラスに対応し、`<details>` や `<dialog>` などの「開いている状態」を CSS だけでスタイル分岐できる機能が全主要ブラウザで使えるようになった（Baseline Newly available）。開閉トグルの見た目切り替えに JavaScript や属性ハックが不要になる。

## 🔰 初学者向け（何が・なぜ重要か）

Web には「開いたり閉じたりする部品」がたくさんあります。折りたたみメニュー（`<details>`）、モーダルダイアログ（`<dialog>`）、日付を選ぶ入力欄などです。これらが「今、開いているのか閉じているのか」で見た目を変えたい場面はよくあります。たとえば、開いているときだけ矢印アイコンを回転させる、といった演出です。

これまでは「開いているか」を CSS から直接知る手段が乏しく、JavaScript でクラスを付け外ししたり、`[open]` 属性を使ったりと回り道が必要でした。`:open` 擬似クラスは、その要素が「開いた状態」かどうかを CSS が直接見分けられるようにするものです。たとえるなら、ドアに「開/閉」のランプが付いて、CSS がそのランプを見て装飾を切り替えられるようになった感覚です。

Safari が対応したことで主要ブラウザが出揃い、実務で安心して使える段階（Baseline 入り）になりました。

## 🧠 専門的解説（技術詳細・設計・使いどころ）

### 技術的詳細（何が変わったか）

`:open` は「開いた状態を持つ要素が開いているとき」にマッチする擬似クラス。対象は主に `<details>`、`<dialog>`（`show()` / `showModal()` で開いた状態）、および `<select>` や日付・色ピッカー等のポップアップを持つフォーム要素。2026年5月の Safari 26.5 対応により全主要エンジンで実装が揃い、Baseline Newly available になった。対になる `:closed` も仕様上定義される。

これにより、たとえば `details:open summary::after { transform: rotate(90deg); }` のように、開閉トグルの装飾を宣言的に書ける。

### 仕組み / 背景（なぜこう設計されたか）

`<details>` には従来から `[open]` 属性セレクタが使えたが、これは属性の有無に依存し、`<dialog>` や各種ポップアップのように「属性ではなく内部状態で開閉する」要素には統一的に適用できなかった。`:open` は「開いている」という UI 状態を、要素の種類に依らず横断的に表現する擬似クラスとして設計された。これは 2026 年の CSS が向かう「状態を意識する（state-aware）CSS」の一例で、`:has()`、`@container style()`、`:open` などにより、これまで JavaScript が担っていた「状態に応じた見た目切り替え」を宣言的スタイルへ移す流れの一部だ。

トレードオフは、`:open` はあくまで「開閉状態」の表現であって開閉ロジックそのものではない点。アニメーションの起点として便利だが、フォーカス管理やアクセシビリティ（開いたダイアログへの焦点移動など）は別途 JS/ARIA で担保する必要がある。

### 実務での使いどころ・移行の勘所

アコーディオン、FAQ、ドロワー、モーダルのトグル演出で JS を削減できる。既存の `[open]` セレクタ実装は、`<dialog>` やポップアップにも一貫適用したい場合に `:open` へ寄せると保守が楽になる。移行の注意点は、Baseline Newly available のため古いブラウザでは無視される点。装飾（回転・色変化など）は無くても機能が壊れない「プログレッシブエンハンスメント」として使うのが安全。中核的な表示/非表示の切り替えを `:open` だけに依存させないこと。

## 📖 用語解説（このトピックとのつながり）

- **:open / :closed 擬似クラス**: 要素の開閉状態にマッチするセレクタ。本トピックの主役。
- **<details> / <dialog>**: 折りたたみ・ダイアログ用のネイティブ HTML 要素。`:open` の主対象。
- **プログレッシブエンハンスメント**: 新機能を「あれば嬉しい装飾」として使い、非対応環境でも壊れない設計方針。
- **state-aware CSS**: 状態に応じたスタイル切り替えを宣言的に行う 2026 年の CSS 潮流。

## 影響範囲 / 推奨アクション

開閉 UI を持つコンポーネントは `:open` で JS 装飾を削減可能。ただし焦点管理・ARIA は引き続き必要。非対応環境で機能が壊れないよう、装飾用途に限定して段階導入するのが安全。
