# Ivanti Sentry に CVSS 10.0 の未認証OSコマンドインジェクション（CVE-2026-10520）— KEV登録・実環境悪用が進行

> トピック: 脆弱性ニュース ｜ 出典: Ivanti Security Advisory / The Hacker News / CISA KEV / Shadowserver ｜ 種別: OSコマンドインジェクションによる未認証リモートコード実行（root RCE）

## 一行サマリー

モバイル管理基盤の中継ゲートウェイ Ivanti Sentry に、ログイン不要でroot権限のコマンド実行を許す CVSS 10.0 満点の脆弱性 CVE-2026-10520 が見つかり、CISA の KEV（悪用が確認された脆弱性カタログ）にも登録された。公開PoCに基づく大量の悪用試行が観測され、一部はバックドアを仕込まれているため、R10.5.2 / R10.6.2 / R10.7.1 以降への即時更新が必須。

## 🔰 初学者向け（何が・なぜ問題か）

Ivanti Sentry は、社員のスマホやタブレット（モバイル端末）が社内のメールやシステムにつながるときの“中継・検問所”として働く装置です。端末と社内システムの間に立ち、「この端末は会社のルールを守っているか」を確認しながら通信を取り次ぎます。

今回の問題は、その検問所に対して、ログインもしていない外部の人が細工した命令を送りつけると、装置がそれを“設定の文字”ではなく“実行する命令”として勘違いして動かしてしまう、という欠陥です。しかも実行されるのは最上位のroot権限（装置の全権限）。つまり外部の見知らぬ人が、検問所の中で何でも好きな操作を実行できてしまいます。

たとえるなら、検問所の係員に「お名前は？」と尋ねたら、来訪者が名前の代わりに「金庫を開けて中身を渡せ」と書いた紙を出し、係員がそれを命令だと思って実行してしまうようなものです。本来は“ただの文字”であるはずの入力が、“命令”として実行されてしまうのが、この種の欠陥（コマンドインジェクション）の怖さです。

深刻なのは、ログイン不要で・満点(10.0)の危険度・PoC（攻撃の実証コード）が公開済みで・すでに世界中で悪用試行が起きている、という最悪の条件が揃っている点です。一部の機器はすでに侵入され、再侵入用の裏口（バックドア）まで仕込まれていると報告されています。

## 🧠 専門的解説（技術詳細・メカニズム・検知対応）

### 技術的詳細（種別・前提条件）

- **CVSS**: 10.0（Critical・満点）。**未認証・ネットワーク経由・低複雑度・ユーザー操作不要**で root RCE。
- **CWE分類**: CWE-78（OS Command Injection）。外部入力が適切に無害化されないままOSコマンド文脈へ渡る古典的かつ最高危険度の欠陥。
- **影響バージョン**: Ivanti Sentry R10.5.2 / R10.6.2 / R10.7.1 より前のバージョン。
- **修正版**: R10.5.2 / R10.6.2 / R10.7.1 以降。
- **悪用状況**: **実環境で活発に悪用**。Ivanti は6月12日付の更新で CISA KEV 追加を告知。Shadowserver は公開PoCに基づく「大量の悪用試行」を観測し、少なくとも複数のインスタンスがバックドア化されたと報告。
- **権限**: 取得される実行権限は root（システム最上位）。

### メカニズム（攻撃が成立する仕組み）

以下は概念レベルの説明であり、武器化可能な手順・PoC・正確なパラメータ等は含めない。

1. **信頼境界の所在**: Sentry は「外部からの未認証要求」を最前線で受ける中継ゲートウェイであり、ここで受け取った入力は本来「データ」として安全に処理されねばならない。
2. **無害化の欠落**: 受信した特定パラメータが、シェル等のOSコマンドを組み立てる処理へ渡る際に、メタ文字（`;` `|` `&` 等のコマンド区切り・連結記号）のエスケープや許可リスト検証が欠落している。これにより攻撃者は「データのはずの入力」に命令片を混入させ、コマンド文脈で解釈させるプリミティブを握る。
3. **認証前で成立する点**: この入力処理経路が認証チェックの前段に存在するため、攻撃者は資格情報なしで到達できる。これが「未認証 RCE」を成立させる核心。
4. **権限の橋渡し**: コマンドを実行するプロセスが root 権限（あるいは特権ヘルパ）で動作するため、注入された命令はそのまま最上位権限で実行される。アプリ層の入力不備が、特権分離の不足を介して機器全体の掌握（root RCE）へ直結する。
5. **悪用後の永続化**: 攻撃者は得た root 実行で Web シェルやバックドアを設置し、パッチ後も残存する足場を確保しようとする（観測済み）。よって「更新するだけ」では不十分で、侵害調査が必須となる。

脆弱性クラスとしては、認証前経路における入力無害化欠如に起因する OS コマンドインジェクション（CWE-78）の典型例であり、特権プロセスでの実行という条件が被害を root 全権掌握まで増幅している。

### 検知・暫定緩和・恒久対策

- **検知の着眼点**: Sentry への認証前リクエストでのコマンドメタ文字の出現、想定外の子プロセス生成（シェル起動）、Webシェル様ファイルの新規作成・改変、外部C2への不審通信、cron/起動スクリプトへの不審な追記。これらをWebアクセスログ・プロセス監査・FIM（ファイル整合性監視）で突き合わせる。Shadowserver/JPCERT 等の外部通知も活用。
- **暫定緩和（パッチ前）**: 管理／サービスインターフェースのインターネット露出を排除し、到達経路を許可IPに限定。WAF/IPSの最新シグネチャ適用。即時更新できない場合はサービス縮退も検討。
- **恒久対策**: R10.5.2 / R10.6.2 / R10.7.1 以降へ即時更新。更新後は **必ず侵害調査（IoCハント）** を実施し、バックドア・不審アカウント・改変ファイルを精査。被侵害が疑われる機器は証跡保全のうえ再構築し、関連資格情報を全面ローテーション。
- **運用示唆**: 公開PoC＋満点CVSS＋実悪用という条件下では「24時間以内の対処」を前提とする。境界・中継機器は被侵害前提の運用（早期検知と再構築計画）が不可欠。

## 📖 用語解説（このニュースとのつながり）

- **Ivanti Sentry**: モバイル端末管理（UEM/EMM）環境で、端末と社内システム間の通信を仲介・検査するゲートウェイ。意味＝モバイル接続の検問所。本件＝この検問所が未認証で乗っ取られる。
- **OSコマンドインジェクション（CWE-78）**: 入力が命令として解釈され任意のOSコマンドが実行される欠陥。意味＝「文字のはずの入力が命令になる」。本件＝root RCEの直接原因。
- **未認証RCE**: ログイン不要で任意コードを実行できる最悪クラスの脆弱性。本件＝攻撃コスト最小・被害最大。
- **KEV（Known Exploited Vulnerabilities）**: CISAが「実際に悪用が確認された」と認定した脆弱性カタログ。意味＝対処の最優先度を示す指標。本件＝6月にKEV登録済み。
- **バックドア**: 再侵入用に仕込まれる裏口。本件＝悪用済み機器で設置が観測され、パッチ後も残存しうる。
- **PoC（Proof of Concept）**: 脆弱性の悪用を実証するコード。本件＝公開済みで悪用が加速。

## 影響範囲 / 推奨対応

影響を受けるのは、修正前バージョンの Ivanti Sentry を公開運用する全組織。モバイル接続基盤の中継点であるため、侵害は社内メール・システムへのアクセス経路の掌握と横展開に直結する。推奨対応は、(1) R10.5.2 / R10.6.2 / R10.7.1 以降への即時更新、(2) 管理・サービス面のインターネット露出排除、(3) 更新後の侵害調査（バックドア・改変・不審アカウントの精査）、(4) 被侵害疑い機器の再構築と資格情報の全面ローテーション、(5) 外部スキャン通知・脅威情報の継続監視。
