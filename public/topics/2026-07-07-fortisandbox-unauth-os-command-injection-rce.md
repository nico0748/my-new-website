# FortiSandbox に認証不要のOSコマンドインジェクション、実環境で悪用（CVE-2026-25089 ほか）

> トピック: 脆弱性ニュース ｜ 出典: BleepingComputer / Fortinet PSIRT / Help Net Security ｜ 種別: OSコマンドインジェクション → 遠隔コード実行（RCE）

## 一行サマリー
「ファイルが安全かどうかを検査する装置」であるはずの Fortinet FortiSandbox に、ログイン不要で任意のOSコマンドを実行できる致命的な欠陥（CVE-2026-25089, CVSS 9.1〜9.8）が見つかり、6月中旬から実際の攻撃で悪用されている。防御装置そのものがネットワーク侵入の入口になり得るため、即時のアップデートが必要。

## 🔰 初学者向け（何が・なぜ問題か）
FortiSandbox は、たとえるなら「不審な小包を開ける前に、防爆室の中で開封して安全か確かめる係」です。メール添付やダウンロードファイルを隔離された仮想環境で実際に動かしてみて、「これはマルウェアだ／これは安全だ」と会社に代わって判断してくれる装置です。

問題は、その判断係の「受付窓口」（Web管理画面）に、**合言葉（ログイン）なしで裏口から命令を送り込める穴**があったことです。攻撃者は細工したリクエストを送るだけで、装置の中で自分の好きな命令（OSコマンド）を実行できてしまいます。「安全を確かめる係」の権限を乗っ取られると、検査を通り抜けてマルウェアを「安全」と偽装させたり、その装置を足がかりに社内ネットワークの奥へ侵入したりできます。**守るための装置が、逆に侵入の入口になってしまう**のが最大の怖さです。

しかもこの穴は「合言葉不要（認証不要）」で突けるため、インターネットに管理画面をさらしている組織は、誰でも狙える的になっています。すでに2026年6月中旬から実際に攻撃が観測されているため、「まだ被害の噂を聞かないから大丈夫」ではなく、今すぐの対処が求められます。

## 🧠 専門的解説（技術詳細・メカニズム・検知対応）

### 技術的詳細（種別・前提条件）
- **CVE-2026-25089**: FortiSandbox Web UI における OS コマンドインジェクション（CWE-78: Improper Neutralization of Special Elements used in an OS Command）。CVSS は情報源により 9.1（Critical）と 9.8 の両方が報じられており、**正確な確定値は要確認**。攻撃前提は「遠隔・認証不要（PR:N）」とされ、細工したHTTPリクエストの送信のみで成立する。
- **CVE-2026-26083**: 併せて報じられた CVSS 9.8 の脆弱性で、未パッチ環境での遠隔コード実行（RCE）につながるとされる。詳細な種別は**要確認**。
- **影響バージョン**: FortiSandbox 5.0.0〜5.0.5、4.4.0〜4.4.8、および 4.2.x 系。FortiSandbox Cloud / PaaS は 5.0.4〜5.0.5。
- **修正版**: FortiSandbox 5.0.6 以降、4.4.9 以降、Cloud/PaaS 5.0.6 以降。
- **悪用状況**: 2026年6月14〜16日の週末から3つの重大脆弱性で実環境の悪用が観測された（Defused Cyber が24時間以内に確認）。CVE-2026-25089 の悪用コードは AI 支援で作られた痕跡があり実装ミスも含む、との分析もある（**確度は要確認**）。国内では Fortinet 製品に関連する認証情報漏えいについて JPCERT/CC が 2026-06-23 に注意喚起を出しており、Fortinet 製品全般への攻撃圧力が高い状況。

### メカニズム（攻撃が成立する仕組み）
OSコマンドインジェクションの一般原理として、以下のステップで最終被害（RCE）へ橋渡しされる（概念レベルの説明で、武器化手順・PoC・オフセットは示さない）。

1. **信頼境界の誤り**: Web UI が受け取った外部入力（パラメータやフィールド値）を、内部でシェルコマンドの一部として組み立てる際に、`;` `|` `&` `$( )` などのシェルメタ文字を無害化（エスケープ/検証）せずに渡してしまう。ここで「ユーザ入力＝データ」であるべき信頼境界が、「入力＝実行される命令」へと踏み越えられる。
2. **認証境界の欠如**: 本来は管理者ログインを要求すべきエンドポイントが、認証チェックを経ずに到達可能になっている。これにより「未認証の外部者」が上記の入力ポイントに直接触れられる（PR:N の意味）。
3. **プリミティブの獲得**: 攻撃者は「装置上で任意コマンドを実行する」という強力なプリミティブを手に入れる。単なる情報漏えいではなく、コマンド実行という汎用能力が得られる点が致命的。
4. **RCE／永続化への橋渡し**: そのプリミティブを使い、リバースシェルの確立、Webシェル設置、認証情報や検査ポリシーの窃取・改ざん、装置を踏み台にした内部横展開（lateral movement）へと展開する。防御装置は通常ネットワークの中枢に接続され広い可視性を持つため、被害インパクトが増幅される。

脆弱性クラスとしては「特権を持つコンポーネント（アプライアンス）の入力検証不備」に該当し、"アクセス制御欠落 + 入力サニタイズ欠如" の二重の不備が連鎖して致命度を押し上げている典型例。

### 検知・暫定緩和・恒久対策
- **検知の着眼点**: FortiSandbox 管理画面（HTTPS）への外部IPからの想定外アクセス、通常運用にないプロセス生成、装置からの外向き通信（C2 の疑い）、検査ジョブや設定テーブルの不審な変更。Web アクセスログにシェルメタ文字を含む異常なリクエストがないか。
- **暫定緩和（パッチ前）**: 管理インターフェースをインターネットから遮断し、信頼できる管理セグメント/IPのみに制限。不要な外部公開を停止。
- **恒久対策**: 上記修正版へ即時アップデート。アップデート後も、パッチ前に侵害を受けた可能性を前提にログ監査と認証情報のローテーションを実施。
- **運用示唆**: 「セキュリティ製品自体が攻撃対象」という前提でアプライアンスを最小権限・最小露出で運用し、パッチ適用SLAを一般サーバより短く設定する。

## 📖 用語解説（このニュースとのつながり）
- **サンドボックス（Sandbox）**: 隔離された実行環境。本件では FortiSandbox が疑わしいファイルを安全に「試し実行」して脅威判定する装置を指す。その装置自体が乗っ取られた。
- **OSコマンドインジェクション（CWE-78）**: アプリが外部入力をOSのコマンド文字列に組み込む際の無害化不足により、攻撃者が任意コマンドを紛れ込ませられる欠陥。本件の中核。
- **認証不要（Unauthenticated / PR:N）**: 攻撃にログイン情報が不要であること。誰でも突けるため危険度が跳ね上がる。
- **RCE（Remote Code Execution）**: 遠隔から任意コードを実行される、最も深刻な結果の一つ。本件のゴール。
- **CVSS**: 脆弱性の深刻度スコア（0〜10）。9以上は Critical。本件は 9.1〜9.8 と報告。

## 影響範囲 / 推奨対応
インターネットに管理画面を露出している FortiSandbox（および Cloud/PaaS）は最優先で危険。修正版（5.0.6 / 4.4.9 以降等）へ即時更新し、管理面の外部露出を遮断、パッチ前の侵害を想定した監査と認証情報ローテーションを行う。Fortinet 製品全般が攻撃者の注目を集めているため、他 Forti 製品のパッチ状況も併せて点検すること。
