# Windows HTTP.sys の整数オーバーフローRCE（CVE-2026-47291）— 巨大リクエストでカーネルコード実行

> トピック：Windowsの中核HTTPスタックを狙うRCE ｜ 出典：Microsoft MSRC / Windows Forum ｜ 種別：整数オーバーフロー（CWE-190）に起因するリモートコード実行

## 一行サマリー

Windowsの中核HTTP処理ドライバ「HTTP.sys」に整数オーバーフロー起因のリモートコード実行（CVE-2026-47291）が見つかり、オーバーサイズのリクエストで悪用可能。IIS等を外部公開するサーバでは深刻で、Patch Tuesdayで修正済み（Critical）。

## 🔰 初学者向け（何が・なぜ問題か）

Webサーバは、外から届くリクエスト（「このページをください」といった通信）を受け取って処理します。Windowsでは、その一番最初の受付を「HTTP.sys（エイチティーティーピー・エスワイエス）」という部品が担っています。しかもこれはOSの中枢（カーネル）で動く、とても強い権限を持つ受付です。

今回の問題は「整数オーバーフロー」というバグです。たとえるなら、受付が荷物の大きさをメモする欄に「3桁まで」しか書けないのに、攻撃者が異常に大きなサイズ（例：1000）の荷物を送ると、桁があふれて「000」のように誤った小さい数字として記録されてしまう、という現象です。すると受付は「小さい荷物用」の狭い置き場を用意したのに、実際には巨大な荷物が押し込まれ、隣の大事な領域まではみ出して壊してしまいます。

攻撃者はこの“はみ出し”を精密に操ることで、最終的にサーバ上で自分のプログラムをカーネル権限で動かせる可能性があります。HTTP.sysはインターネットに面したWebサーバ（IISなど）の最前線にいるため、**認証前・ネットワーク越し**に届くリクエストだけで狙える点が特に危険です。サーバの完全な乗っ取りに直結し得ます。

## 🧠 専門的解説（技術詳細・メカニズム・検知対応）

### 技術的詳細（種別・前提条件）

- **CVE**：CVE-2026-47291
- **深刻度**：Critical（RCE）。CVSS値は要確認。
- **CWE**：CWE-190（整数オーバーフロー／ラップアラウンド）→ ヒープ等のメモリ破壊 → RCE。
- **影響コンポーネント**：Windows の HTTP プロトコルスタック **HTTP.sys**（カーネルモードドライバ）。IISやHTTP.sysを利用する各種サービスが攻撃面。
- **前提条件**：**オーバーサイズのリクエスト**を送れること。HTTP.sysが受け付けるエンドポイント（Webサービス等）がネットワーク到達可能であること。認証は不要と見られる（ネットワーク面の未認証攻撃、**要確認**）。
- **悪用状況**：Patch Tuesdayで修正。公開時点で実環境悪用の確証情報は確認できず（**要確認**）。ただしHTTP.sys系の過去脆弱性（例：ワーム化が懸念されたRCE）と同様、ネットワーク面・カーネル特権という条件から高リスク。
- **対策**：該当のWindows更新プログラムを適用。

### メカニズム（攻撃が成立する仕組み）

整数オーバーフロー起点のメモリ破壊がRCEへ橋渡しされる、古典的だが強力なクラスです（概念レベル。武器化手順・PoC・オフセットは記載しません）。

1. **サイズ計算の破綻**：HTTP.sysがリクエスト（ヘッダやボディ長など）を解析する過程で、長さ・件数などを表す整数値が計算に使われる。攻撃者が**異常に大きな値**を与えると、加算・乗算の結果が整数型の上限を超えて**ラップアラウンド（桁あふれで小さな値に化ける）**する。
2. **確保サイズと実データの不一致**：破綻した（過小な）計算結果に基づいてメモリ領域（バッファ）を確保する一方、実際にコピー/処理されるデータは想定より大きい。結果、**確保領域を越える書き込み（ヒープオーバーフロー等のメモリ破壊）**が発生する。信頼境界＝「外部から来た長さ情報を信じて内部のメモリ確保に使う」ところの検証不備が核心。
3. **プリミティブの獲得**：メモリ破壊により、攻撃者は隣接するカーネルヒープ上の構造体（オブジェクトのヘッダ、関数ポインタ、サイズ情報等）を制御可能なプリミティブを得る。
4. **RCEへの橋渡し**：この破壊を精密に操作し、制御データ（例：関数ポインタや実行フローに影響する構造体）を上書きすることで、最終的に**カーネル権限での任意コード実行**へ到達し得る。HTTP.sysはカーネルモードで動くため、成功時の権限は最高位。
5. **クラス一般原理**：整数オーバーフローRCEは、(a) 外部由来の数値を境界チェックなしにサイズ計算へ用いる、(b) その結果でメモリ確保／コピー長を決める、という設計が揃うと成立する。プロトコル解析（HTTP/TLS/画像デコード等）は攻撃者制御の長さフィールドが多く、この種のバグが生まれやすい。CVEの正確な内部経路は非公開のため、ここでは**型の説明**に留める。

### 検知・暫定緩和・恒久対策

- **検知の着眼点**：異常に大きなヘッダ/ボディ長を含むリクエスト、プロトコル仕様を逸脱する巨大リクエストのアクセス/WAFログ。HTTP.sys由来のカーネルクラッシュ（Bugcheck）や`HTTP`関連イベントの異常。
- **暫定緩和**：境界のWAF/リバースプロキシで**リクエストサイズ上限**（ヘッダ長・ボディ長）を厳格化し、オーバーサイズ要求を遮断。不要なHTTP.sys利用サービスの公開停止・アクセス元制限。
- **恒久対策**：該当Windows更新プログラムを速やかに適用。外部公開サーバを最優先。
- **運用示唆**：HTTP.sysはIIS以外にも多数のWindowsサービスが内部利用しており、「IISを入れていないから無関係」とは限らない。プレ認証・ネットワーク面・カーネル特権という三拍子が揃うため、公開資産は最優先でパッチ管理する。

## 📖 用語解説（このニュースとのつながり）

- **HTTP.sys**：Windowsカーネルで動くHTTP受付ドライバ。IIS等の最前線であり、ここのRCEはサーバ完全掌握に直結する。
- **整数オーバーフロー（CWE-190）**：数値が型の上限を超え、桁あふれで小さな値に化ける現象。今回はサイズ計算の破綻→メモリ破壊の起点。
- **ヒープオーバーフロー／メモリ破壊**：確保領域を越える書き込みで隣接データを壊す状態。制御データを上書きできればRCEに発展する。
- **プレ認証（未認証）攻撃**：ログイン不要で成立する攻撃。到達性だけで狙えるため危険度が跳ね上がる。
- **カーネルモード**：OSの最高権限で動く実行モード。ここでのコード実行は防御をほぼ無効化する。

## 影響範囲 / 推奨対応

HTTP.sysを利用する外部公開Windowsサーバ（IIS等）が最優先の対象。該当Windows更新プログラムを速やかに適用し、適用までは境界でリクエストサイズ上限を厳格化してオーバーサイズ要求を遮断する。IIS未導入でもHTTP.sysを内部利用するサービスがあるため、資産の棚卸しと公開面の縮小を併せて実施する。
