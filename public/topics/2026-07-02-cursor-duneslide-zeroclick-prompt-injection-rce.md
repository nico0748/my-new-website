# Cursor IDE "DuneSlide"：ゼロクリックのプロンプトインジェクションからサンドボックス脱出・RCE（CVE-2026-50548 / CVE-2026-50549）

> トピック: 脆弱性ニュース ｜ 出典: The Hacker News / Cato AI Labs / SecurityWeek ｜ 種別: プロンプトインジェクション連鎖・シンボリックリンク解決不備（サンドボックス脱出→RCE）

## 一行サマリー

AI コードエディタ Cursor に、"見た目は普通の一言のプロンプト" だけで安全機構（サンドボックス）を突破し開発者PCで任意コマンドを実行できる2つの欠陥（いずれもCVSS 9.8）が判明。Cato AI Labs が "DuneSlide" と命名。攻撃者が仕込んだ Web 検索結果や MCP サーバ応答を AI エージェントが読むだけで、ユーザー操作ゼロ（ゼロクリック）で成立する。Cursor 3.0 で修正済み。

## 🔰 初学者向け（何が・なぜ問題か）

Cursor は、AI が自動でコードを書いたり調べ物をしたりしてくれる人気の「AI 搭載コードエディタ」です。AI エージェントは、Web 検索の結果や外部ツール（MCP サーバ）の返事など、**外の情報を読み込んで**作業を進めます。

問題は、その "外から読み込む情報" に攻撃者がこっそり悪意ある指示を紛れ込ませられる点です。これを**プロンプトインジェクション**と呼びます。たとえるなら、真面目な部下（AI）に渡した参考資料の余白に、悪人が「ついでにこの金庫を開けて中身を外に送れ」と書き込んでおくようなもの。部下は資料の一部だと思って従ってしまいます。

Cursor には、AI が勝手に危険な操作をしないよう「安全な檻（サンドボックス）」があります。ところが DuneSlide では、攻撃者の仕込んだ指示によって AI が檻の外へ抜け出し、**あなたのPCで好きなコマンドを実行**できてしまいます。しかも被害者は「普通の質問を一度しただけ」で発火する（ゼロクリック）。乗っ取られれば、PC だけでなくエディタがログイン中のクラウドやSaaS環境まで危険にさらされます。

## 🧠 専門的解説（技術詳細・メカニズム・検知対応）

### 技術的詳細（種別・前提条件）

- **CVE**: CVE-2026-50548, CVE-2026-50549（Cato AI Labs 発見、"DuneSlide"）。
- **CVSS**: 両者とも 9.8（CVSS 3.1）／9.3（CVSS 4.0）。Critical。
- **種別（CWE）**: 
  - プロンプトインジェクション（AI/LLM 特有、CWE-1427 等の LLM 指示注入系）
  - CVE-2026-50549 はパス解決ロジックのシンボリックリンク追従不備（CWE-59：Link Following / CWE-61 系）。
- **前提条件**: ユーザーが**罪のないプロンプトを1回入力**するだけ。AI エージェントが**信頼できないソース**（MCP サーバ応答、汚染された Web 検索結果 等）から攻撃者制御コンテンツを取り込むことが引き金。特別な権限・明示的操作は不要（ゼロクリック）。
- **影響バージョン**: Cursor 3.0 未満のすべて。
- **修正版**: Cursor 3.0（2026年4月2日リリース）で両欠陥を修正。
- **開示経緯**: Cato が2月19日報告 → 当初却下 → 2月26日エスカレーション → 再オープン・トリアージのうえ 3.0 で修正。
- **悪用状況**: 実環境での広範な悪用の断定情報はなし（要確認）。ただしゼロクリック・高CVSSのため露出リスクは高い。

### メカニズム（攻撃が成立する仕組み）

※本節は**概念レベル**。実際の武器化手順・PoC・具体的ペイロードは記載しない。

1. **信頼境界の誤り（間接プロンプトインジェクション）**: AI エージェントは「ユーザーの指示」と「外部から取り込んだデータ」を明確に区別できない。攻撃者は Web ページ／検索結果／MCP サーバ応答に指示文を埋め込む。エージェントがそれを"文脈"として読むと、埋め込まれた指示を**自分のタスクとして実行**してしまう（indirect prompt injection）。ここで攻撃者は「エージェントに任意の意図を注入する」プリミティブを得る。
2. **サンドボックス脱出への橋渡し（CVE-2026-50549 のパス解決不備）**: 注入された指示は、プロジェクトディレクトリ内に**外部ファイルを指すシンボリックリンク**を作らせる。Cursor の正規化（canonicalization）ステップが失敗した場合——たとえばリンク先が存在しない／読み取り権限が無い等——エージェントは**検証されていない元のシンボリックリンクのパスを信頼**するフォールバック挙動を取る。これにより「プロジェクト内の安全な領域だけを触る」というサンドボックスの前提（信頼境界）が破られ、外部パスへの書き込み等が可能になる。
3. **RCE への連鎖**: サンドボックスが無力化されると、続くコマンドは"ユーザー本人の権限"で実行される。任意ファイル書き込み → 起動時・ツール実行時に読まれるファイルの汚染などを介し、非サンドボックスの遠隔コード実行に至る。攻撃者が握るのは「エージェントの意図（注入）」＋「パス検証の綻び（symlink 追従）」で、この2枚が合わさって"入力"から"実行"へ橋渡しされる。
4. **一般原理**: (a) LLM エージェントにおける「データと命令の混同」は本質的な弱点で、外部コンテンツを取り込む限り指示注入の余地が残る。(b) パス正規化の失敗時に"未検証の元パスを信頼するフォールバック"は、TOCTOU／リンク追従（CWE-59）の典型的アンチパターン。安全側に倒すなら失敗時は拒否すべき。両者が重なると"読むだけ"でRCEになる。

### 検知・暫定緩和・恒久対策

- **検知の着眼点**: プロジェクト配下での不審な symlink 生成、エージェントによる外部パスへの書き込み、AI セッション中の予期しない子プロセス起動・アウトバウンド通信、MCP 応答／検索結果に紛れる指示様文字列。
- **暫定緩和（更新前）**: 信頼できない MCP サーバ・外部検索ソースをエージェントに接続しない。自動実行（auto-run）系の権限を絞り、コマンド実行やファイル書き込みに人間承認を挟む。作業を最小権限アカウント／隔離環境で行う。
- **恒久対策**: Cursor 3.0 以降へ更新。
- **運用示唆**: AI コーディング/エージェント環境は「外部コンテンツ＝潜在的な命令」という前提で設計する。MCP・検索・リポジトリ等の"入力源の信頼度"を管理し、エージェントの権限は最小化・承認ゲート化する。

## 📖 用語解説（このニュースとのつながり）

- **プロンプトインジェクション（間接型）**: 外部データに指示を埋め込み AI に実行させる攻撃。今回の起点で、"資料の余白に命令を書く"構図。
- **ゼロクリック**: 被害者の明示操作なしで成立すること。普通の質問1回で発火するため、本件の危険度を押し上げる要因。
- **サンドボックス脱出**: AI の安全な実行範囲を破ること。CVE-2026-50549 の symlink 不備が"檻の外"への橋渡しになった。
- **シンボリックリンク追従（CWE-59）**: リンクを辿って想定外ファイルにアクセスさせる不備。正規化失敗時に未検証パスを信頼した点が核心。
- **MCP サーバ**: AI エージェントに外部データ／ツールを渡す仕組み。今回はその応答が"攻撃者制御コンテンツ"の運び手になり得た。

## 影響範囲 / 推奨対応

Cursor 3.0 未満を利用する全開発者・組織。外部 MCP や Web 検索をエージェントに接続している環境は特にハイリスク。乗っ取られると開発PCに加え、エディタがサインイン中のクラウド／SaaS ワークスペースにも波及。対応は (1) Cursor 3.0 以降へ即時更新、(2) 信頼できない入力源の切り離し、(3) 自動実行権限の縮小と承認ゲート導入、(4) 開発環境の隔離と最小権限化。
