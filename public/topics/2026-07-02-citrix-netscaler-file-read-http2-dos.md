# Citrix NetScaler：無認証の任意ファイル読み取りとHTTP/2 Bomb DoSなど6件を修正（CVE-2026-8451 ほか）

> トピック: 脆弱性ニュース ｜ 出典: The Hacker News / SecurityWeek / CyberScoop ｜ 種別: 境界外読み取り・メモリ解放漏れ・任意ファイル読み取り（DoS/情報漏えい）

## 一行サマリー

Citrix が NetScaler ADC / Gateway の6件の脆弱性を修正。無認証で任意ファイルを読める CVE-2026-10816（CVSS 7.1）や、細工した HTTP/2 リクエストで機器を停止させる "HTTP/2 Bomb" 型 DoS（CVE-2026-13474）を含む。過去の CitrixBleed を想起させるメモリ系欠陥で、パッチ適用に加え一部は設定変更（Http2SmallWndTimeout）も必要。

## 🔰 初学者向け（何が・なぜ問題か）

NetScaler（ADC / Gateway）は、社外から社内システムやVPNへ安全に入るための「玄関ゲート」の役割を持つ機器です。常にインターネットに面しているため、攻撃者に最初に狙われる場所でもあります。

今回まとめて直った欠陥のうち特に重要なのは2種類です。ひとつは**「無認証の任意ファイル読み取り」**。ログインなしで、本来見えてはいけない機器内部のファイルを覗けてしまう問題です。設定や認証情報の断片が漏れれば、次の攻撃の材料になります（過去に大問題となった "CitrixBleed" と同じ系統の怖さ）。

もうひとつは**「HTTP/2 Bomb」型のサービス妨害（DoS）**。攻撃者が特殊なウェブ通信（HTTP/2）を送りつけると、機器がメモリを解放しきれずに詰まり、**玄関ゲートそのものが機能停止**してしまいます。ゲートが落ちれば、正規利用者もVPNや業務システムに入れなくなります。

## 🧠 専門的解説（技術詳細・メカニズム・検知対応）

### 技術的詳細（種別・前提条件）

- **対象**: Citrix NetScaler ADC / NetScaler Gateway。ベンダー速報 **CTX696604**。
- **CVE（6件）**: CVE-2026-8451, CVE-2026-8452, CVE-2026-8655, CVE-2026-10816, CVE-2026-10817, CVE-2026-13474。
  - **CVE-2026-10816（CVSS 7.1）**: 無認証の任意ファイル読み取り。NSIP／Cluster Management IP／管理有効な SNIP への到達が前提。
  - **CVE-2026-8452**: メモリオーバーフロー由来の予測不能挙動・DoS。機器が **Gateway** または **AAA 仮想サーバ**として構成されている場合。
  - **CVE-2026-13474**: 実効寿命後のメモリ解放漏れ（CWE-401）による DoS。**不正な HTTP/2 リクエスト**で発火（"HTTP/2 Bomb"）。
- **修正版**: NetScaler ADC / Gateway 14.1-72.61 以降、および 13.1 系は 13.1-63.18 以降。
- **追加対応**: CVE-2026-13474 は HTTP Strict Profile 未使用構成の場合、アップグレードのみでは不完全。**Http2SmallWndTimeout を 30 秒に手動設定**する必要あり。
- **悪用状況**: 本速報時点で広範な実悪用の断定情報はなし（要確認）。ただし NetScaler は攻撃者の常時標的であり、CitrixBleed 系の前例から早期適用が強く推奨される。

### メカニズム（攻撃が成立する仕組み）

※本節は**概念レベル**。武器化手順・PoC は記載しない。

1. **任意ファイル読み取り（CVE-2026-10816）の型**: 管理系IF（NSIP/SNIP/CMIP）で受け付ける入力が、参照ファイルの決定に不十分な検証のまま使われると、境界外や想定外パスの読み取りに繋がる。信頼境界＝「管理面に触れられる者＝認証済み管理者」という前提が、無認証到達可能な経路の存在によって崩れ、攻撃者が"読む"プリミティブを握る。CitrixBleed 系と同様、露出したメモリ／ファイル内容にセッションや設定の機微情報が含まれると被害が連鎖する。
2. **HTTP/2 Bomb DoS（CVE-2026-13474）の型**: HTTP/2 は1接続で多数のストリームを多重化する。小さなウィンドウのストリームを大量／特定パターンで送ると、機器側が各ストリームに割り当てたメモリを実効寿命後も解放できず（CWE-401：メモリ解放漏れ）、リソースが枯渇して処理不能＝DoS に至る。攻撃者が握るのは「安価なリクエストで高価なメモリ確保を強制する非対称性」で、これがサービス停止へ橋渡しされる。緩和に Http2SmallWndTimeout（小ウィンドウ・ストリームのタイムアウト短縮）が要るのはこのため。
3. **メモリオーバーフロー DoS（CVE-2026-8452）の型**: Gateway/AAA 構成時の処理でメモリ境界を超える書き込み・オーバーフローが起き、予測不能な挙動やクラッシュを誘発する。認証・アクセス制御の入口機能が落ちること自体が可用性被害となる。
4. **共通原理**: いずれも「境界機器のメモリ管理の綻び」に起因する。RCE ではなくとも、機微情報の読み取りと可用性喪失は、認証基盤・VPN の入口を担う NetScaler では重大インパクトになる。

### 検知・暫定緩和・恒久対策

- **検知の着眼点**: NSIP/SNIP/CMIP 管理面への無認証・外部アクセス、想定外パスのファイル参照エラー、HTTP/2 ストリーム数・小ウィンドウ多重の急増、機器のメモリ使用率スパイクと応答遅延／再起動。
- **暫定緩和**: 管理面（NSIP/管理有効SNIP）をインターネットから遮断し管理ネットワーク限定に。CVE-2026-13474 は **Http2SmallWndTimeout=30秒**を設定。不要なら該当仮想サーバ構成を見直す。
- **恒久対策**: 14.1-72.61 以降 / 13.1-63.18 以降へ更新。HTTP Strict Profile 未使用構成は設定変更を忘れず併用。
- **運用示唆**: NetScaler は "パッチ＋設定" の二段構えが必要なケースがある。アップグレードだけで安心せず、ベンダー速報の追加手順を必ず確認する。

## 📖 用語解説（このニュースとのつながり）

- **NetScaler ADC / Gateway**: 社外からの入口を担う ADC／VPN ゲートウェイ。今回の欠陥は"入口機器の情報漏えい・停止"に直結する。
- **任意ファイル読み取り**: 本来アクセス不可のファイルを読める欠陥。無認証（CVE-2026-10816）であるほど危険で、CitrixBleed 系の情報漏えいに連なる。
- **HTTP/2 Bomb**: 安価なHTTP/2リクエストで機器のメモリを枯渇させる DoS 手法。今回 Http2SmallWndTimeout の設定が緩和策として要求された。
- **メモリ解放漏れ（CWE-401）**: 使い終わったメモリを解放しない不備。積み重なると資源枯渇＝DoS に至る、CVE-2026-13474 の根因。
- **Http2SmallWndTimeout**: 小さいウィンドウのHTTP/2ストリームを打ち切る設定値。DoS 緩和のため 30 秒への手動設定が求められる。

## 影響範囲 / 推奨対応

インターネット公開の NetScaler ADC / Gateway を運用する全組織。特に Gateway / AAA 仮想サーバ構成、管理IFが外部到達可能な構成はハイリスク。(1) 14.1-72.61 / 13.1-63.18 以降へ更新、(2) CVE-2026-13474 は Http2SmallWndTimeout=30秒 を設定、(3) 管理面の外部遮断、(4) 情報漏えい兆候（無認証ファイル参照）の調査。
