# RadiX AX6600 ルータ OSコマンドインジェクション（JVN#20769211）— 管理画面からroot奪取

> トピック: 脆弱性ニュース｜出典: JVN（Japan Vulnerability Notes）/ JVN iPedia｜種別: OSコマンドインジェクション（CWE-78）

## 一行サマリー

WiFi 6 ゲーミングルータ「RadiX AX6600」（Micro-Star International 提供）の Web 管理コンソールに OS コマンドインジェクション（CVSS 7.2）があり、管理者としてログインした状態で root 権限の任意コマンドを実行されうる。対策はメーカー提供の最新ファーム（v781521 以降）への更新。

## 🔰 初学者向け（何が・なぜ問題か）

家庭やオフィスのインターネット入り口にある「ルータ」は、外と中をつなぐ門番のような機器です。RadiX AX6600 はゲーミング向けの高性能ルータで、設定はブラウザの管理画面（Web コンソール）から行います。

今回の問題は、その管理画面の入力欄に細工した文字列を入れると、ルータの中で「本来は実行されないはずの命令（OSコマンド）」をこっそり紛れ込ませて実行できてしまう欠陥です。これを OS コマンドインジェクション（命令の差し込み）と呼びます。

たとえるなら、受付に「お名前は？」と書く欄があるのに、そこへ名前ではなく「金庫を開けて」という指示書を書き込むと、受付がそのまま実行してしまうようなものです。しかもルータの内部では root（最高管理者）として動くため、ルータを完全に乗っ取られる可能性があります。

ただし前提として「管理者としてログイン済み」であることが必要です。とはいえ、初期パスワードのままだったり、管理画面が外部に露出していたりすると、攻撃者が管理者になりすましてこの穴を突けてしまいます。最新ファームへの更新が確実な対策です。

## 🧠 専門的解説（技術詳細・メカニズム・検知対応）

### 技術的詳細（種別・前提条件）

- **識別子**: JVN#20769211 / JVNDB-2026-000046 / **CVE**: 要確認（JVN記載を参照）
- **CVSS**: 7.2（CVSS v3.0 基本値、High）
- **CWE**: CWE-78（OS Command Injection / OSコマンドインジェクション）
- **製品**: RadiX AX6600 WiFi 6 Tri-Band Gaming Router（提供: Micro-Star International Co., Ltd.）
- **影響バージョン**: ファームウェア v781521 より前 / **修正版**: v781521 以降
- **悪用状況**: 公表は 2026-06-17（JVN）。実環境での広域悪用報告は要確認だが、ルータ系のコマンド注入はボット組み込み・恒久バックドア化に直結しやすく注意が必要。
- **前提条件**: Web コンソールに管理者としてログインできること。実行される権限は root。

### メカニズム（攻撃が成立する仕組み）

※以下は脆弱性クラスの一般原理に基づく概念説明。武器化手順・PoCは記載しない。

1. **外部入力の不十分な検証（信頼境界の誤り）**: 管理コンソールの特定パラメータが、内部でOSコマンドの一部として組み立てられる。ユーザ入力は信頼境界の外にあるのに、シェルに渡す前の無害化（メタ文字のエスケープ／許可リスト化）が不足している。
2. **コマンド境界の越境**: シェルでは `;` `|` `&` `$()` などが「コマンドの区切り・連結・展開」を意味する。これらが素通りすると、攻撃者の文字列が本来の一語ではなく「追加の命令」として解釈される。これが任意コマンド実行プリミティブ。
3. **特権での実行へ橋渡し**: ルータの管理機能はしばしば root で動作する。注入されたコマンドも root 権限で実行されるため、設定の書き換え・恒久的なバックドア設置・ファーム改ざんといった完全掌握に直結する。
4. **認証境界との関係**: 本件は管理者ログインが前提だが、初期/弱パスワード、管理画面のWAN側露出、CSRF 等と組み合わさると、認証の壁が実質的に崩れ未認証相当の悪用に発展しうる。
   - 脆弱性クラスの一般原理: コマンドインジェクションは「データとして扱うべき入力を、コードとして解釈する経路に渡す」ことで成立する。組み込み機器の管理UIはシェル呼び出しを多用するため特に頻出する。

### 検知・暫定緩和・恒久対策

- **検知の着眼点**: 管理コンソールへの異常なログイン、設定パラメータに混入したシェルメタ文字、ルータからの想定外の外向き通信（C2/ボット参加の兆候）、ファーム/設定の予期せぬ変更。
- **暫定緩和**: 管理画面を WAN（インターネット）側から到達不能にする、管理アクセスを内部の限られた端末/VLAN に制限、初期パスワードの変更と多要素化、リモート管理機能の無効化。
- **恒久対策**: ファームウェアを v781521 以降へ更新。更新後はパスワード再設定と設定の点検（不審なポート転送・DNS設定・スケジュールの有無）を実施。

## 📖 用語解説（このニュースとのつながり）

- **OSコマンドインジェクション（CWE-78）**: 入力をOSの命令文に紛れ込ませて実行させる攻撃。本件の本体で、root実行ゆえに被害が大きい。
- **Web 管理コンソール**: ルータの設定用ブラウザ画面。ここの入力検証不備が突かれた。
- **root 権限**: 機器内の最高権限。ここで任意実行できると機器を完全掌握される。
- **シェルメタ文字（`; | & $()`）**: コマンドの区切りや展開を表す記号。無害化されないと注入が成立する。
- **JVN / JVNDB**: 日本の脆弱性情報ポータルとデータベース。国内ベンダー・流通製品の一次情報源。

## 影響範囲 / 推奨対応

影響は RadiX AX6600 ルータでファーム v781521 より前を使用している環境。家庭・SOHO・店舗ネットワークの入り口機器であり、乗っ取られると配下端末全体の通信傍受や踏み台化につながる。推奨対応は、最新ファーム（v781521 以降）への更新、管理画面のWAN側露出の停止、管理パスワードの強化。更新後はポート転送やDNS設定など不審な変更の有無を点検する。
