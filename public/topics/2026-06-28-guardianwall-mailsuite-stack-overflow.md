# GUARDIANWALL MailSuite に未認証RCE級のスタックBOF、実環境で悪用済み（CVE-2026-32661, 国内・CVSS 9.8）

> トピック: 脆弱性ニュース｜出典: JPCERT/CC 注意喚起（at260013）/ IPA / JVN#35567473 / キヤノンITソリューションズ｜種別: スタックベースのバッファオーバーフロー（CWE-121）によるリモートコード実行

## 一行サマリー
国内で広く使われるメールセキュリティ製品 GUARDIANWALL MailSuite（キヤノンITソリューションズ）に、特定コマンド `pop3wallpasswd` のスタックバッファオーバーフローがあり、細工リクエストで任意コードを実行される（CVE-2026-32661, CVSS v3.0 9.8 / v4.0 9.3）。オンプレ版ではすでに悪用が確認されており、メールの“関所”が乗っ取りの起点になりうる。

## 🔰 初学者向け（何が・なぜ問題か）
GUARDIANWALL MailSuite は、会社のメールを出入りする際にチェックする“関所（メールゲートウェイ）”です。情報漏えい防止やフィルタリングのため、社内外のメールがほぼ必ずここを通ります。

その関所の内部にある `pop3wallpasswd` という処理に、これまでの記事と同じ「はみ出し書き込み（バッファオーバーフロー）」の弱点がありました。攻撃者が関所のWebサービスに“長すぎる細工リクエスト”を送ると、処理用のメモリからはみ出して内部の指示を上書きでき、関所のサーバー上で攻撃者のプログラムを動かせてしまいます。

関所は全メールが通る要所なので、ここを乗っ取られるとメールの盗み見・改ざん、社内ネットワークへの侵入の踏み台にされる恐れがあります。しかも開発元は「オンプレミス版で実際の攻撃を確認済み」と公表しており、“いつか”ではなく“すでに”狙われている状況です。

## 🧠 専門的解説（技術詳細・メカニズム・検知対応）

### 技術的詳細（種別・前提条件）
- **CVE**: CVE-2026-32661。**JVN**: JVN#35567473。**CWE**: CWE-121（スタックベースのバッファオーバーフロー）。
- **CVSS**: v3.0 基本値 9.8 / v4.0 基本値 9.3。
- **悪用状況**: 開発者がオンプレミス版で**本脆弱性を悪用した攻撃を確認済み**と公表（国内で実害リスク高）。
- **前提条件**: 該当製品のWebサービスに到達し、細工リクエストを送れること。脆弱箇所は製品内コマンド `pop3wallpasswd`。
- **影響バージョン**: オンプレ版 Ver 1.4.00 〜 Ver 2.4.26。SaaS版（GUARDIANWALL Mailセキュリティ・クラウド）は2026-04-30のメンテナンス以前。**修正**: SaaS版は同メンテで修正済み、オンプレ版は提供パッチを適用。

### メカニズム（攻撃が成立する仕組み）
1. **公開サービスの入力面**: 製品のWebサービス経由で `pop3wallpasswd` コマンド処理に外部入力が到達する。メールゲートウェイは外部通信を受ける性質上、攻撃面が露出しやすい。
2. **境界長検証の不備**: 攻撃者が長さ/内容を制御できる入力に対し、固定長スタックバッファの境界チェックが不足。境界外書き込みでスタック上の制御データ（戻りアドレス等）を破壊できる（古典的スタックBOF）。
3. **制御フロー奪取→RCE**: 制御データ破壊により実行フローを攻撃者コードへ誘導するプリミティブを獲得。製品サービスの実行コンテキストで任意コード実行に橋渡しされる。
4. **被害の拡大**: メールゲートウェイは全メールフローと社内外の境界に位置するため、RCE成立後はメールの傍受・改ざん、認証情報の収集、内部ネットワークへの横展開の足場となりうる。
   - 一般原理: 「インターネット境界に置かれた製品（VPN/メールGW/ファイアウォール等）の未認証〜低認証パースにメモリ破壊」がある場合、境界そのものが侵入口になる。境界機器は最優先のパッチ対象。
   - 本記述は概念レベルに留め、オフセットや武器化手順・PoCは記載しない。

### 検知・暫定緩和・恒久対策
- **検知の着眼点**: GUARDIANWALL関連プロセス（pop3wallpasswd/Webサービス）の異常終了・再起動、製品サーバーから生成される不審な子プロセス・外部通信、Webサービスへの異常に長い/不正なリクエスト。
- **暫定緩和**: 製品WebサービスへのアクセスをIP制限・境界FWで最小化、管理面のインターネット非公開化、可能なら一時的に外部到達を遮断。
- **恒久対策**: オンプレ版は開発元提供パッチを即時適用。SaaS版は2026-04-30メンテで修正済みのため最新状態を確認。
- **運用示唆**: 既に悪用確認済みのため、適用前提で「侵害済みかどうか」を前向きに調査（プロセス/通信/メールログの異常）。

## 📖 用語解説（このニュースとのつながり）
- **GUARDIANWALL MailSuite**: キヤノンITソリューションズのメールセキュリティ製品。国内導入が多く、全メールが通る“関所”であることが被害規模に直結。
- **メールゲートウェイ（関所）**: 送受信メールを検査・制御する境界機器。侵害されると傍受・踏み台化の影響が大きい。
- **pop3wallpasswd**: 本件の脆弱箇所となった製品内コマンド。ここでスタックBOFが発生。
- **スタックベースのバッファオーバーフロー（CWE-121）**: 固定長スタック領域を越えた書き込みで制御を奪う欠陥。本連載で繰り返し登場する古典的かつ致命的な型。
- **JPCERT/CC 注意喚起・JVN**: 国内の脆弱性情報を調整・公表する公的窓口。国内製品の対応判断はまずここを確認する。

## 影響範囲 / 推奨対応
GUARDIANWALL MailSuite オンプレ版 Ver 1.4.00〜2.4.26 を運用する国内組織が対象。既に悪用確認済みのため最優先で開発元パッチを適用し、適用までWebサービスへの到達をIP制限・境界FWで絞る。SaaS版利用者は2026-04-30メンテ以降であることを確認。露出していた場合はプロセス・通信・メールログの異常を侵害調査する。
