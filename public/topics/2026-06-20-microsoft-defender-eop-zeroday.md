# Microsoft 6月パッチ CVE-2026-41091 — Defenderの権限昇格ゼロデイを実環境で悪用、記録的208件の一部

> トピック: 脆弱性ニュース ｜ 出典: BleepingComputer / Security Affairs / Zero Day Initiative ｜ 種別: 権限昇格（Elevation of Privilege）

## 一行サマリー

Microsoftの2026年6月パッチ（過去最大級の約208件）に、Windows標準のセキュリティ機能「Microsoft Defender」の権限昇格ゼロデイ CVE-2026-41091（CVSS 7.8）が含まれ、修正前から実環境で悪用されていた。すでに端末に入り込んだ攻撃者が管理者級の権限を奪う踏み台になる。

## 🔰 初学者向け（何が・なぜ問題か）

Microsoft Defender は、Windowsに最初から入っている「警備員（ウイルス対策ソフト）」です。普段はパソコンを守る側ですが、今回はその警備員自身に弱点が見つかりました。

この弱点は「権限昇格（けんげんしょうかく）」と呼ばれるタイプです。たとえるなら、すでに建物に入り込んだ侵入者（=一般ユーザー権限で動くマルウェアや、フィッシングで実行された不正プログラム）が、警備員のスキを突いて「マスターキー（管理者権限）」を手に入れるようなもの。これ単体では建物に「入る」ことはできませんが、いったん入った後に被害を一気に拡大させる“格上げ”の手段になります。

特に問題なのは、Microsoftが修正パッチを出す前から、この弱点が**実際の攻撃に使われていた**点（これを「ゼロデイ」と呼びます）。さらに今回の6月の更新は過去最大級の約208件もの修正を含み、管理者にとっては適用作業が大仕事になっています。Defenderは多くのWindows端末で動いているため、影響範囲が広いのが特徴です。

## 🧠 専門的解説（技術詳細・メカニズム・検知対応）

### 技術的詳細（種別・前提条件）

- **CVE/CVSS**: CVE-2026-41091、CVSS 7.8（Important/High）。種別は Microsoft Defender の Elevation of Privilege（権限昇格）。
- **前提条件**: ローカルでコード実行できる足場が前提（既存マルウェア、初期侵入後など）。リモート単独での侵入口ではなく「侵入後の権限格上げ」に用いられる。複数研究者がクレジットされており、複数経路からの悪用が示唆される。
- **悪用状況**: 修正前から実環境で悪用が確認された**ゼロデイ**。6月パッチ全体では約208 CVE（過去最大級）を修正し、複数のゼロデイ/公開済み脆弱性を含む。
- **修正版**: 2026年6月のWindows月例更新（Patch Tuesday, 6/9公開）で修正。Defender はクラウド更新で配信される場合があるため、エンジン/プラットフォームのバージョン確認が重要。
- **同時修正の注目株**: Windows Kernel の未認証RCE CVE-2026-45657（CVSS 9.8）など重大なRCEも含む（要・併せて適用）。
- ※ 詳細な内部不備は一部「要確認」。本記事は公開情報に基づく型の整理を含む。

### メカニズム（攻撃が成立する仕組み）

1. **信頼の高い常駐コンポーネントが標的**: Defender はSYSTEM相当の高い権限で常時稼働する。こうした「常に高権限で動く保護コンポーネント」は、権限昇格の典型的な狙い目になる。
2. **低権限からの橋渡し**: 攻撃者はまず一般ユーザー権限でコードを実行できる状態（初期侵入）にある。Defender側の処理の不備（権限境界での入力/操作の検証不足など、権限昇格クラスに共通の型）を突くことで、低権限プロセスから高権限の動作を誘発する。
3. **プリミティブの獲得**: 権限昇格が成立すると、攻撃者は「高権限での任意操作」というプリミティブを握る。これによりセキュリティ製品の無効化、永続化（自動起動の仕込み）、認証情報の窃取が可能になる。
4. **最終被害への接続**: 端末の完全掌握→ドメイン内横展開（ラテラルムーブメント）の起点になる。保護ソフト自体が乗っ取られると、防御の“目”を潰されるため被害検知も難しくなる。

一般原理として、権限昇格（EoP）は「権限境界をまたぐ操作の検証不備」「高権限プロセスが低権限の入力/オブジェクトを信頼してしまう」ことから生じる。単体では侵入できないが、初期侵入バグやフィッシングと連鎖して致命化する“最後のひと押し”になりやすい。武器化手順やPoCはここでは扱わない（概念レベル）。

### 検知・暫定緩和・恒久対策

- **検知の着眼点**: Defender エンジン/プラットフォームの異常な改変・無効化、SYSTEM権限での見慣れないプロセス生成、EoP直後に典型的な永続化・認証情報アクセス（LSASS アクセス等）の挙動。
- **暫定緩和**: 最小権限運用とアプリ実行制限（攻撃の前提となる初期コード実行を抑止）、EDRの改ざん防止（tamper protection）有効化、特権アカウントの分離。
- **恒久対策**: 2026年6月のWindows月例更新を全端末へ適用。Defender のエンジン/プラットフォーム/シグネチャを最新化。併せて CVE-2026-45657 等の重大RCEも優先適用。

## 📖 用語解説（このニュースとのつながり）

- **Microsoft Defender**: Windows標準のセキュリティ機能。本件は「守る側」自身に権限昇格の穴があった。
- **権限昇格（EoP）**: 低い権限から高い権限を奪う攻撃。本件は侵入後の被害拡大手段。
- **ゼロデイ**: 修正が出る前に悪用される脆弱性。本件は実環境悪用が確認済み。
- **Patch Tuesday（月例更新）**: Microsoftが毎月第2火曜に出すまとめ更新。今回は約208件で過去最大級。
- **ラテラルムーブメント**: 侵入後に組織内の他端末・サーバーへ広がる動き。権限昇格はその起点になる。

## 影響範囲 / 推奨対応

Microsoft Defender を有効化した広範なWindows環境。リモート単独の侵入口ではないが、初期侵入後の被害を決定づける権限昇格であり、実環境悪用済みのため緊急度は高い。**6月の月例更新を全端末に適用**し、Defender のバージョンを確認。EDRの改ざん防止と最小権限運用で初期コード実行の抑止も並行する。同月の Windows Kernel RCE（CVE-2026-45657）等も忘れず適用。
