# キヤノン EOS Network Setting Tool 複数脆弱性（JVNVU#98100934）— FTP認証情報の漏えい懸念

> トピック: 脆弱性ニュース｜出典: JVN / Canon PSIRT (CP2026-005) / Canon サポート｜種別: 証明書検証不備・平文プロトコル等（複数）

## 一行サマリー

キヤノンのカメラ用ソフト「EOS Network Setting Tool」（EOS Utility 同梱）に複数の脆弱性（CVE-2026-9258〜9262、最大CVSS 6.5）があり、FTP/FTPS/SFTP の接続テスト機能を通じて認証情報が窃取されうる。サーバ証明書の検証不備や既定が平文FTPになっている点が原因。対策は EOS Utility を 3.20.21 以降へ更新すること。

## 🔰 初学者向け（何が・なぜ問題か）

一眼カメラ（キヤノン EOS）には、撮った写真をネットワーク経由でサーバ（FTP）へ自動転送する機能があり、その設定を行うPCソフトが「EOS Network Setting Tool」です。プロのカメラマンが現場から写真を送るときなどに使われます。

このソフトには「接続テスト」機能があり、設定したサーバへちゃんとつながるか試せます。ところが、その通信のやり方に複数の弱点が見つかりました。たとえば、相手サーバの「身分証（電子証明書）」をきちんと確認しないため、偽サーバになりすまされても気づけません。また、初期設定が「中身を暗号化しない平文のFTP」になっているため、通信を盗み見されるとログイン情報（ユーザー名・パスワード）が筒抜けになります。

たとえるなら、宅配の受け取りで相手の身分証を確認せず、しかも荷物の中身を透明な袋で運ぶようなものです。途中で待ち伏せた攻撃者に、サーバへの鍵（認証情報）を奪われてしまう恐れがあります。深刻度は中程度ですが、奪われた認証情報は他の侵入の足場になりうるため、更新が推奨されます。

## 🧠 専門的解説（技術詳細・メカニズム・検知対応）

### 技術的詳細（種別・前提条件）

- **識別子**: JVNVU#98100934 / Canon PSIRT CP2026-005
- **CVE**: CVE-2026-9258, 9259, 9260, 9261, 9262（5件）
- **代表例**: CVE-2026-9259 = サーバ証明書の検証不備（CVSS 6.5, Medium）／ CVE-2026-9262 = 既定で非セキュアな（平文）FTPプロトコルを使用
- **CWE**: CWE-295（Improper Certificate Validation）/ CWE-319（Cleartext Transmission of Sensitive Information）相当
- **影響製品**: EOS Network Setting Tool Ver.1.5.0 以前（Windows/macOS）。EOS Utility 3.12.0〜3.20.20 に同梱。
- **影響箇所**: FTP/FTPS/SFTP の通信テスト機能
- **悪用状況**: 公表 2026-06-15。悪用には中間者（MITM）位置の取得など前提が必要で、広域の自動悪用報告は要確認。
- **修正版**: EOS Utility 3.20.21 以降。

### メカニズム（攻撃が成立する仕組み）

※以下は脆弱性クラスの一般原理に基づく概念説明。武器化手順・PoCは記載しない。

1. **信頼の検証欠落（証明書検証不備）**: FTPS/SFTP の接続テストで、相手サーバが提示する証明書の正当性（発行者チェーン・ホスト名一致・有効期限）を十分検証しない。これにより「正規サーバである」という信頼前提が成立しないまま通信が確立する。
2. **中間者の介在余地**: 攻撃者が同一ネットワークや経路上（公共Wi-Fi、ARPスプーフィング等）に位置すると、偽サーバを正規になりすませる。クライアントは検証不備ゆえ偽装を見抜けない。
3. **平文プロトコルによる露出**: 既定が暗号化なしのFTPの場合、認証情報や転送内容がネットワーク上を平文で流れる。中間者は復号すら不要で、傍受だけで認証情報を取得できる（情報露出プリミティブ）。
4. **認証情報窃取から横展開へ橋渡し**: 奪われたFTP認証情報は、転送先サーバへの不正アクセス、保存写真の窃取・改ざん、さらに資格情報の使い回しを突いた他システムへの横展開の足場になりうる。
   - 脆弱性クラスの一般原理: 「通信相手を検証しない」「機密を平文で送る」は、暗号化トランスポートの前提（相手認証＋秘匿性）を満たさない典型的失敗。暗号スイートを使っていても検証を省けば安全性は崩れる。

### 検知・暫定緩和・恒久対策

- **検知の着眼点**: FTP の平文認証トラフィック、想定外の証明書（自己署名・ホスト名不一致）での接続成立、転送先サーバへの不審なログイン元IP。
- **暫定緩和**: 接続テスト・転送を信頼できるネットワーク（VPN/専用線）でのみ実施、平文FTPを避け SFTP/FTPS を明示選択、転送用アカウントを最小権限化しパスワードを使い回さない。
- **恒久対策**: EOS Utility を 3.20.21 以降へ更新。更新後は転送設定を暗号化必須に見直し、過去に平文で運用した期間の認証情報はローテーション（再設定）する。

## 📖 用語解説（このニュースとのつながり）

- **EOS Network Setting Tool**: キヤノン一眼の通信設定ソフト。FTP転送の設定・テストに使い、本件の弱点はこのテスト機能にある。
- **証明書検証（CWE-295）**: 通信相手が本物かを電子証明書で確かめる処理。これを省くと偽サーバになりすまされる。
- **平文FTP（CWE-319）**: 暗号化なしでファイル・認証情報を送るプロトコル。傍受で中身が丸見えになる。
- **中間者攻撃（MITM）**: 通信経路上に割り込み、盗聴・改ざんする攻撃。証明書検証不備＋平文がこれを容易にする。
- **JVNVU / PSIRT**: JVN の海外連携番号（VU）と、ベンダー（キヤノン）の製品セキュリティ窓口の公式アドバイザリ。

## 影響範囲 / 推奨対応

影響は EOS Network Setting Tool Ver.1.5.0 以前（EOS Utility 3.12.0〜3.20.20 同梱）を使う Windows/macOS 環境。とくにFTP転送を多用するスタジオ・報道・イベント撮影の運用で、認証情報窃取のリスクがある。推奨対応は、EOS Utility を 3.20.21 以降へ更新し、転送は SFTP/FTPS を明示選択、信頼できる経路でのみ実施すること。過去に平文運用していた場合は転送用アカウントの認証情報を再設定する。
