# ThingsBoard CVE-2026-53676 — IoTプラットフォームのプロトタイプ汚染、テナント管理者からサンドボックス内コード実行

> トピック: 脆弱性ニュース｜出典: JVN#16937365 / JVNDB-2026-000086 / CIRCL｜種別: プロトタイプ汚染（CWE-1321）→サンドボックス内コード実行

## 一行サマリー

オープンソースのIoTプラットフォーム ThingsBoard に、プロトタイプ汚染の脆弱性 CVE-2026-53676（JVN#16937365、CVSS v3.0 7.2 / v4.0 8.6）があり、テナント管理者（TENANT_ADMIN）権限でログインできる利用者がサンドボックス内で任意コードを実行しうる。国内の研究者（LAC）が報告、v4.3.1.2 未満が影響、最新版への更新が必要。

## 🔰 初学者向け（何が・なぜ問題か）

ThingsBoard は、たくさんのIoT機器（センサーや装置）からデータを集め、ダッシュボードで見たり制御したりするための土台ソフトです。工場・ビル・インフラなどの「機器の集中管理室」のような役割を担います。

今回の欠陥は「プロトタイプ汚染」と呼ばれるものです。JavaScript（このソフトの一部で使われる言語）には、すべての部品が共有する「設計図のひな型（プロトタイプ）」があります。攻撃者がこのひな型をこっそり書き換えると、その後に作られるすべての部品に悪い性質が伝わってしまいます。

例えるなら、ある会社の「全社員共通のマニュアル」に一行だけ悪い指示を紛れ込ませると、以後すべての社員がその誤った指示に従ってしまう、という状態です。本件では、ある程度の権限を持つ利用者（テナント管理者）がこの書き換えを行うと、本来は隔離された安全区画（サンドボックス）の中でプログラムを動かせてしまいます。完全な乗っ取りではないものの、管理者権限の悪用や内部からの足がかりにつながるため危険です。

## 🧠 専門的解説（技術詳細・メカニズム・検知対応）

### 技術的詳細（種別・前提条件）

- **CVE**: CVE-2026-53676 / **JVN**: JVN#16937365 / **JVNDB**: JVNDB-2026-000086
- **CVSS**: v3.0 基本値 7.2（High）/ v4.0 基本値 8.6（High）
- **CWE**: CWE-1321（Improperly Controlled Modification of Object Prototype Attributes＝プロトタイプ汚染）
- **影響製品/バージョン**: ThingsBoard v4.3.1.2 未満 / **修正**: 開発者提供の最新版へ更新
- **前提条件**: テナント管理者（TENANT_ADMIN）権限で当該製品にログインできること。
- **影響**: サンドボックス化されたコンテキスト内での任意コード実行。報告者は LAC Co., Ltd. の今井裕貴氏。
- **悪用状況**: 2026-06-16 にJVN公表。実環境での悪用報告は現時点で要確認。

### メカニズム（攻撃が成立する仕組み）

※以下は脆弱性クラスの一般原理に基づく概念説明。武器化手順・PoCは記載しない。

1. **共有プロトタイプへの到達**: JavaScriptでは、オブジェクトは共通の親（`Object.prototype`等）を参照する。`__proto__`/`constructor`/`prototype`といった特殊キーを含む入力を、再帰マージやプロパティ代入で無検証に処理すると、共有プロトタイプに書き込めてしまう。
2. **プロトタイプ汚染プリミティブ**: 攻撃者がキー・値を制御してプロトタイプの属性を追加・改変できると、以後生成・参照される全オブジェクトに「汚染された既定値」が混入する。これが汎用的な「挙動改変プリミティブ」になる。
3. **ガジェットとの接続**: 汚染された属性が、後段のコード（テンプレート評価、オプション解釈、スクリプト実行のサンドボックス設定など）で参照されると、想定外の分岐や設定上書きが起き、サンドボックス内でのコード実行というガジェットに到達する。
4. **権限前提と被害の橋渡し**: 本件はTENANT_ADMINを前提とするため未認証ではないが、テナント管理者の正規操作の範囲を超えて、サンドボックス内コード実行という新たなプリミティブを得る。マルチテナント環境では、テナント境界・基盤側への影響可能性を評価する必要がある。
   - 一般原理: 「特殊キーを含む入力の無検証マージ → プロトタイプ汚染 → ガジェット連鎖」はNode.js/JSエコシステム共通のクラス。根治は`__proto__`等のキー拒否、`Object.create(null)`やMap利用、安全なマージ実装。

### 検知・暫定緩和・恒久対策

- **検知の着眼点**: `__proto__`/`constructor`/`prototype`を含む異常なリクエストボディ、テナント管理者アカウントによる不審な設定変更・スクリプト投入、サンドボックス実行ログの異常。
- **暫定緩和**: 更新まではテナント管理者権限の付与を最小化し、当該機能へのアクセスを制限。入力検証（特殊キーの拒否）を境界で補強。
- **恒久対策**: v4.3.1.2 以降（開発者提供の最新版）へ更新。テナント管理者アカウントの棚卸しと、権限分離・監査ログの強化。

## 📖 用語解説（このニュースとのつながり）

- **ThingsBoard**: IoT機器のデータ収集・可視化・制御を担うプラットフォーム。インフラ・産業用途で利用され、停止・侵害の影響が大きい。
- **プロトタイプ汚染（CWE-1321）**: JSの共有ひな型を書き換え、全オブジェクトの挙動を改変する攻撃。本件のコード実行の起点。
- **サンドボックス**: 影響範囲を限定するための隔離実行環境。本件はその「内側」での実行だが、設定や前提次第で影響が広がりうる。
- **TENANT_ADMIN**: テナント単位の管理者権限。本脆弱性の悪用前提となる権限レベル。
- **JVN / JVNDB**: 日本のJPCERT/CC・IPAが運営する脆弱性情報ポータル。国内向けの一次情報源。

## 影響範囲 / 推奨対応

影響は、ThingsBoard v4.3.1.2 未満を運用し、テナント管理者アカウントが存在する全環境。特に複数テナントを収容する基盤や、産業・インフラ用途では、内部からの権限悪用・横展開の起点となりうる。推奨対応は、開発者提供の最新版へ更新し、テナント管理者権限を最小化、入力検証と監査ログを強化すること。実環境での悪用状況は要確認だが、CVSS High かつ国内一次情報での公表を踏まえ計画的に更新する。
