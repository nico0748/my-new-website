# NEC Aterm シリーズに複数脆弱性（CVE-2026-4620 ほか）— OSコマンドインジェクション等でLAN側から乗っ取り、21モデル影響・9機種はEOL

> トピック: 脆弱性ニュース ｜ 出典: JVN（JVN#80890147 / JVN#89339669）/ NEC（NV26-003/NV26-001）/ ScanNetSecurity ｜ 種別: OSコマンドインジェクション／パストラバーサル／権限チェック欠如ほか（国内）

## 一行サマリー

NEC製Wi-Fiルータ「Aterm」シリーズの21モデルに、権限チェック欠如（CVE-2026-4309）、パストラバーサル（CVE-2026-4619）、OSコマンドインジェクション（CVE-2026-4620／CVE-2026-4622）、危険な隠し機能（CVE-2026-4621）の計5件が判明。いずれもLAN側からの悪用が前提だが、成立すれば任意コマンド実行や設定改ざん・情報漏えいに至る。12機種はファーム更新提供、残り9機種はサポート終了（EOL）のため買い替えが唯一の対策。

## 🔰 初学者向け（何が・なぜ問題か）

Atermは家庭や職場で広く使われるNEC製の無線LANルータ（インターネットの“玄関口”になる機器）です。今回、この機器に複数の弱点が見つかりました。

主な問題は、ルータに送る指示の中に**攻撃者がこっそり別のコマンドを紛れ込ませると、ルータがそれを実行してしまう**欠陥（OSコマンドインジェクション）です。たとえるなら、「設定を変えて」という依頼書の余白に「ついでに裏口の鍵も開けて」と書き足すと、受付が気づかず両方やってしまうような状態です。ほかにも、本来触れない場所のファイルを指定できてしまう（パストラバーサル）、権限の確認が甘くて設定を勝手に変えたり機器情報を抜けたりする（権限チェック欠如）、メーカーが仕込んだ危険な隠し機能がある、といった弱点が重なっています。

救いは、これらが**LAN側（同じネットワークの内側）からの操作を前提**としている点で、インターネット越しに直接いきなり乗っ取られるものではありません。ただし、家庭内に入り込んだマルウェアや、公衆Wi-Fiのように内側に他人がいる環境では現実的な脅威です。ルータを乗っ取られると通信の盗み見や改ざんの土台にされます。

対策として、対象21モデルのうち12機種には修正ファームが提供されますが、**残り9機種はサポート終了（EOL）で修正が出ず、買い替えが唯一の対策**です。まず自分の機種を確認し、更新か買い替えを検討してください。

## 🧠 専門的解説（技術詳細・メカニズム・検知対応）

### 技術的詳細（種別・前提条件）

- **CVE / 件数**: 計5件。
  - CVE-2026-4309: 権限チェックの欠如（認可不備）。
  - CVE-2026-4619: パストラバーサル。
  - CVE-2026-4620 / CVE-2026-4622: OSコマンドインジェクション（任意OSコマンド実行）。
  - CVE-2026-4621: セキュリティ上問題のある隠し機能（hidden functionality）。
- **識別子**: JVN#80890147（NV26-003, OSコマンドインジェクション）、JVN#89339669（NV26-001, 複数脆弱性）。**CVSS値は各JVN/NEC advisoryで要確認**。
- **影響範囲**: Atermシリーズ21モデル。
- **前提条件**: いずれも**LAN側からのアクセスが前提**（WAN側からの直接悪用は想定外）。認可不備系は認証状態により悪用条件が変わるため各Advisoryを確認。
- **悪用状況**: 実悪用報告は現時点で確認されていない（要確認）。
- **修正状況**: 12機種はファームウェアアップデート提供。残り9機種は**サポート終了（EOL）**で修正提供なし＝機器交換が対策。

### メカニズム（攻撃が成立する仕組み）

複数クラスの脆弱性が同居するが、最も深刻なOSコマンドインジェクション（CVE-2026-4620/4622）を中心に概念レベルで整理する（武器化手順・具体ペイロードは記載しない）。

1. **信頼境界の所在**: ルータの管理機能（Web UI等）は、利用者入力を受けて内部でシステムコマンドや設定処理を呼び出す。ここで「入力は無害化されている」という前提が置かれる。
2. **入力検証／認可の不備**: OSコマンドインジェクションでは、入力値のサニタイズが不十分なため、攻撃者が制御する文字列が**シェルコマンド組み立てにそのまま流入**する（CWE-78相当）。並行して権限チェック欠如（CVE-2026-4309）は、本来認可が必要な操作を**認可判定なしに実行**させ、パストラバーサル（CVE-2026-4619）は**想定外パスのファイル参照**を許す。
3. **プリミティブの獲得**: これらにより攻撃者は「任意コマンド実行」「設定改ざん」「機器情報・ファイル読み取り」といったプリミティブを、LAN内から獲得する。隠し機能（CVE-2026-4621）はさらに強力な操作面を与え得る。
4. **最終被害への橋渡し**: 任意コマンド実行はルータの掌握＝通信経路の支配に直結する。乗っ取られたルータはDNS改ざん・トラフィック傍受・内部横展開の拠点となり、家庭／小規模ネットワーク全体の信頼を崩す。

一般原理として、OSコマンドインジェクションは**外部入力を検証せずコマンド文字列へ連結する**設計不備、認可不備は**操作前の権限確認を欠く**設計不備であり、境界機器（ルータ）で起きると影響が面的に波及する。

### 検知・暫定緩和・恒久対策

- **検知の着眼点**: 管理UIへの想定外リクエスト、設定の不審な変更（DNS/ポート転送/管理者）、LAN内からの管理面への異常アクセス、機器の再起動・挙動変化。家庭用ルータはログが乏しく、設定監視が中心。
- **暫定緩和**: 管理画面のパスワード強化とアクセス元制限、リモート管理（WAN側管理）の無効化、ゲスト/内部セグメント分離でLAN側攻撃面を縮小。不要機能の無効化。
- **恒久対策**: 対象12機種は提供ファームへ即時更新。**EOLの9機種はサポート対象機種へ買い替え**。更新後は管理者資格情報を変更し設定を点検。

## 📖 用語解説（このニュースとのつながり）

- **OSコマンドインジェクション（CWE-78）**: アプリが外部入力を検証せずOSコマンドとして実行してしまう欠陥。今回のCVE-2026-4620/4622が該当し、ルータ乗っ取りの主因。
- **パストラバーサル**: `../` 等でディレクトリを遡り、本来アクセスできないファイルを参照する欠陥。設定・秘密情報の漏えいにつながる。
- **権限チェック欠如（認可不備）**: 操作前に権限を確認しない欠陥。認証済みでも越権操作を許し、設定改ざん・情報取得を可能にする。
- **隠し機能（hidden functionality）**: 製品に仕込まれた文書化されていない機能。悪用されると強力な操作面となる。
- **EOL（サポート終了）**: 修正提供が終わった状態。脆弱性が出ても直せず、買い替えが唯一の対策になる。

## 影響範囲 / 推奨対応

NEC Aterm 21モデルが対象。LAN側前提だが、内部にマルウェアや第三者がいる環境では現実的脅威で、乗っ取りは通信傍受・改ざんの土台になる。まず機種を確認し、12機種は提供ファームへ更新、EOLの9機種は買い替え。リモート管理無効化・管理パスワード強化・セグメント分離で攻撃面を縮小すること。
