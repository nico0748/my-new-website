# NVFP4 — Blackwell がハード対応する 4bit 浮動小数、学習も推論も脱量子化なしで高速化

> トピック: NVFP4 量子化フォーマット｜出典: NVIDIA・AI Infrastructure・vLLM/llm-compressor ドキュメント｜種別: 推論基盤 / 量子化

## 一行サマリー

NVFP4 は NVIDIA Blackwell GPU がハードウェアで直接扱える 4bit 浮動小数の量子化形式。ブロック単位の FP8「マイクロスケーリング」と E2M1 の FP4 値を組み合わせ、FP8 比で 2〜3 倍の演算スループットと約 1.8 倍のメモリ削減を、精度劣化を最小限に抑えつつ実現する。重みも活性も NVFP4 なら脱量子化なしで計算でき、`llm-compressor` で変換し vLLM で実行できる。

## 🔰 初学者向け（何が・なぜ重要か）

LLM は膨大な数値（重み）でできていて、動かすには大量のメモリと計算力が要る。そこで「数値を粗く丸めてデータ量を減らす」量子化という技術が使われる。数値を 16bit や 8bit、さらに 4bit に縮めれば、メモリも計算も軽くなる。ただし粗くしすぎると賢さ（精度）が落ちるのが悩みどころだ。

NVFP4 は「4bit まで縮めても精度をあまり落とさない」ように工夫した新しい 4bit フォーマット。ポイントは、小さなブロックごとに「拡大率（スケール）」を持たせて、値の大小の幅を上手に表現すること。たとえるなら、全部を同じ物差しで測るのではなく、区画ごとに最適な目盛りの物差しを使い分けて、粗い 4bit でも精度を保つイメージ。

しかも最新の Blackwell GPU はこの 4bit をそのまま計算できるので、「4bit を一度 16bit に戻してから計算する（脱量子化）」手間が要らない。結果として、より大きなモデルを、より安く・速く動かせる。

## 🧠 専門的解説（技術詳細・設計・使いどころ）

### 技術的詳細（何が変わったか）

- **形式**: ハードウェア対応のブロック単位 4bit 浮動小数。1 要素は 4bit（符号 1・指数 2〔バイアス 1〕・仮数 1）の E2M1。
- **ハイブリッド構成**: ブロックごとの FP8「マイクロスケーリング」＋ E2M1 の FP4 値を組み合わせ、大幅なメモリ削減と計算高速化を、精度劣化を最小に保って実現。
- **性能**: FP8 比で 2〜3 倍の演算スループット、約 1.8 倍のメモリ削減。
- **脱量子化不要**: Blackwell Tensor Core が NVFP4 を直接扱うため、重み・活性が両方 NVFP4 なら推論中もコンパクトな 4bit のまま計算でき、脱量子化が不要。
- **ソフトウェア**: `llm-compressor` で NVFP4 に変換し、NVFP4 対応の vLLM で実行。精度回復には量子化認識蒸留（QAD）等も研究されている。

### 仕組み / 背景（なぜこう設計されたか）

低ビット量子化の難所は「1 つのテンソル内で値の絶対値の幅（ダイナミックレンジ）が広い」こと。単純に全体を 1 つのスケールで 4bit に丸めると、小さな値が潰れて精度が崩れる。NVFP4 は **細かいブロックごとに FP8 のスケール係数を持たせる「マイクロスケーリング」** で、局所的なレンジに合わせて目盛りを最適化する。各値自体は E2M1（2bit 指数・1bit 仮数）の FP4 で、指数を持つ浮動小数のため整数 4bit（INT4）より広いレンジを表現できる——これが精度維持の鍵だ。

設計の妙は「ブロックスケールは高精度（FP8）、値は超低精度（FP4）」という二層構造で、メモリはほぼ 4bit のまま、表現力はブロックスケールで補う点。さらに Blackwell の Tensor Core が NVFP4 演算をネイティブ実行するため、従来の「4bit で保存→計算前に高精度へ戻す」往復が消え、メモリ帯域と演算の両方で得をする。トレードオフは、(1) Blackwell 系ハードに依存する、(2) ブロックスケールの初期化や外れ値処理が甘いと精度が落ちるため、PTQ（学習後量子化）の質やブロックスケール初期化、必要なら量子化認識蒸留での回復が要る、(3) すべての層を 4bit にすると敏感な層で劣化するため、層ごとの精度混在の設計判断が要ることだ。

### 実務での使いどころ・移行の勘所

大規模 MoE や長文脈モデルを Blackwell 系で安く速くサービングしたい場面、メモリ制約で載らないモデルを収めたい場面に効く。移行の勘所は、(1) まず `llm-compressor` で NVFP4 化し、vLLM で精度・スループットを実測、(2) 精度が落ちる層は高精度に残す混在量子化や、量子化認識蒸留で回復、(3) 効果は Blackwell 前提のため、旧世代 GPU では利点が限定的、(4) 重み・活性の両方を NVFP4 にして初めて脱量子化を省ける点を踏まえ、W4A4 構成の可否を検証。ベンチ値は構成依存が大きいので自モデルでの検証必須（要確認）。

## 📖 用語解説（このトピックとのつながり）

- **量子化**: 数値のビット幅を減らしメモリ/計算を軽くする技術。粗いほど精度リスク。
- **NVFP4 / E2M1**: 4bit 浮動小数形式。指数 2・仮数 1 で INT4 より広いレンジを表現。
- **マイクロスケーリング**: ブロックごとに FP8 スケールを持たせ局所レンジに適応する仕組み。
- **脱量子化（dequantization）**: 低ビット値を計算前に高精度へ戻す処理。NVFP4 は不要にできる。
- **W4A4 / 量子化認識蒸留（QAD）**: 重み活性とも 4bit の構成と、精度回復の学習手法。

## 影響範囲 / 推奨アクション

- Blackwell 系で大規模モデルを低コスト・高スループットでサービング可能。W4A4 で脱量子化を省略。
- `llm-compressor`→vLLM で精度/速度を実測し、敏感な層は混在量子化や QAD で回復。
- 効果はハード依存。旧世代 GPU では利点が限られる点に留意し、自モデルで検証（要確認）。
