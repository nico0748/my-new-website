# Google Chrome V8 ゼロデイ CVE-2026-11645 が実環境で悪用（境界外メモリアクセス）

> トピック: 脆弱性ニュース ｜ 出典: The Hacker News / Google Chrome Release / CISA ｜ 種別: 境界外メモリアクセス（OOB read/write, CWE-787/125）

## 一行サマリー

Chrome の JavaScript/WebAssembly エンジン「V8」に境界外メモリアクセスの脆弱性 CVE-2026-11645（CVSS 8.8）があり、Google は「実環境で悪用するエクスプロイトが存在する」と認めた。悪意あるWebページを開くだけで悪用されうるため、Chrome および Chromium 系ブラウザの即時更新が必要。

## 🔰 初学者向け（何が・なぜ問題か）

Chrome の中には「V8」という、Webページの JavaScript（ページを動かすプログラム）を高速に実行するエンジンが入っています。私たちが普段見るほぼ全てのWebサイトはこの V8 の上で動いています。

今回の問題は、V8 が処理のためにメモリ（作業机）を扱うときに、本来割り当てられた範囲の「外」を読み書きしてしまう欠陥です。たとえるなら、自分の机の引き出しだけを使うべきなのに、隣の人の引き出しまで勝手に開けて中身を書き換えてしまうようなものです。攻撃者がこの「はみ出し」を巧妙に誘導すると、ブラウザの動作を乗っ取り、最終的にあなたの端末で不正なコードを動かせる可能性があります。

特に怖いのは、被害に遭うのに難しい操作が不要なことです。攻撃者が用意した、あるいは改ざんされたWebページを「開く」だけで成立しえます。すでに実際の攻撃で使われている（ゼロデイ）ため、Chrome を最新版に更新することが最も重要な対策です。普段使いの全員に関係します。

## 🧠 専門的解説（技術詳細・メカニズム・検知対応）

### 技術的詳細（種別・前提条件）

- 種別: V8（JavaScript/WebAssembly エンジン）の境界外メモリアクセス（Out-of-Bounds read/write、CWE-787/125 の型）。
- CVSS: 8.8（High、要確認）。攻撃元はネットワーク（Web）。ユーザー操作は「悪性ページを開く」程度。
- 影響: Chrome 安定版の脆弱バージョン、および同 V8 を用いる Chromium 系ブラウザ（Edge, Brave 等、各社更新が必要）。
- 修正版: Google が安定版チャネルで修正を配布。最新版へ更新で解消。
- 悪用状況: Google が「CVE-2026-11645 のエクスプロイトが実環境に存在する」と公表。詳細は更新普及のため非開示。CISA KEV へ登録（2026-06-09、要確認）。

### メカニズム（攻撃が成立する仕組み）

注: 概念レベルにとどめ、武器化手順・PoC・オフセット等は記載しない。

1. **JIT/最適化と型の前提**: V8 は高速化のため、変数の型や配列長について最適化時の前提を置く。攻撃者が JavaScript でその前提を崩す状態（例: 想定外の型・長さ変化）を作ると、境界チェックが実態とずれる。
2. **境界外アクセスのプリミティブ獲得**: ずれた前提のまま配列等にアクセスさせることで、割り当て領域の外を読む/書くプリミティブ（OOB read/write）が成立する。読みは情報漏えい（メモリレイアウト把握）、書きは制御データ改変の足がかり。
3. **サンドボックス内コード実行への橋渡し**: OOB write を起点に、オブジェクトのメタデータやポインタを操作し、最終的にレンダラープロセス内で任意コード実行へ至りうる。実害化には通常さらにサンドボックス回避を要する（多くは別バグと連鎖）。
4. **一般原理**: JIT を持つスクリプトエンジンにおける「型混同/境界チェック迂回」は、ブラウザ攻撃の主要クラス。Web ページという誰でも触れる入力面が攻撃面になる点が脅威の本質。

### 検知・暫定緩和・恒久対策

- 検知の着眼点: 端末側でのブラウザ子プロセスの異常起動、不審なドメインへの誘導（水飲み場攻撃・悪性広告）、EDR でのブラウザからの想定外プロセス生成。
- 暫定緩和: 信頼できないリンク/広告を開かない、社内では更新の強制配布。標的型の懸念がある場合はサイト分離・拡張機能の最小化。
- 恒久対策: Chrome / Chromium 系を最新安定版へ即時更新（再起動で適用）。組織は管理ポリシーで自動更新を担保。
- 運用示唆: ブラウザはOSと並ぶ攻撃面。資産横断で各 Chromium 系ブラウザのバージョンを点検する。

## 📖 用語解説（このニュースとのつながり）

- **V8**: Chrome の JavaScript/WebAssembly 実行エンジン。本脆弱性の所在。
- **境界外メモリアクセス（OOB）**: 割り当て範囲外を読み書きする欠陥。情報漏えいや制御奪取の起点になる。
- **JIT コンパイル**: スクリプトを実行時に機械語へ最適化する仕組み。型の前提崩しが攻撃の入口になりやすい。
- **サンドボックス**: ブラウザが危険な処理を隔離する仕組み。RCE 実害化には通常その回避が別途必要。
- **ゼロデイ / KEV**: 修正前から悪用される脆弱性／CISA の実悪用カタログ。掲載＝至急更新の合図。

## 影響範囲 / 推奨対応

Chrome 利用者全般と、Chromium を基盤とする Edge・Brave などの利用者が対象。各ブラウザを最新安定版へ即時更新し、再起動で適用する。組織は自動更新ポリシーの徹底とバージョン点検を行い、当面は不審なリンク・広告経由のアクセスに注意する。
