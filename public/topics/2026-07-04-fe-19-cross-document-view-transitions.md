# クロスドキュメント View Transitions ― MPA のページ遷移を CSS でアニメ化

> トピック: 開発トピック｜出典: southwellmedia｜種別: frontend

## 一行サマリー
別々の HTML ページ間の遷移を滑らかにアニメーションできる **クロスドキュメント View Transitions（MPA 対応）**が利用可能に（Baseline 時期は要確認）。SPA を組まなくても、通常のリンク遷移が SPA のようなヌルッとした画面切替になる。

## 🔰 初学者向け（何が・なぜ重要か）
スマホアプリでは、一覧の写真をタップするとその写真が拡大しながら詳細画面へ滑らかに移ります。一方、昔ながらのウェブ（複数の HTML ページで構成される MPA）では、リンクを踏むと画面が一瞬「パッ」と白く消えて次のページが出る、味気ない切替が普通でした。この滑らかさを出すために、多くのサイトはわざわざ SPA（1枚の HTML で全部やる方式）を採用してきました。

クロスドキュメント View Transitions は、**普通のリンク遷移でも SPA のような滑らかな切替**を、CSS の数行だけで実現します。「このページとこのページの切替はフェードで」「共通のロゴは位置を補間して移動」と宣言するだけ。SPA の複雑な仕組みを持ち込まなくても、体験を上げられます。シンプルな構成のまま、見栄えだけ現代化できるのが魅力です。

## 🧠 専門的解説（技術詳細・設計・使いどころ）
### 技術的詳細（何が変わったか）
- 同一オリジンの MPA 間で、`@view-transition { navigation: auto; }` を両ページに宣言するだけで遷移アニメが有効化。
- `view-transition-name` を共有した要素は、ページをまたいで位置・サイズが補間される（モーフィング）。
- SPA 向けの `document.startViewTransition()` に加え、ドキュメントをまたぐ遷移が標準化された点が新しい。

### 仕組み / 背景（なぜこう設計されたか）
従来の View Transitions API は SPA（同一ドキュメント内の DOM 差し替え）専用でした。しかしウェブの大半は依然として複数ページの MPA です。クロスドキュメント版は、ブラウザが遷移前ページの「スナップショット」と遷移後ページの状態を保持し、その間を補間することで、ドキュメントが切り替わっても連続した見た目を作ります。JS 主導の SPA 化に頼らずとも滑らかさを得られるようにする、「プラットフォームで解決する」思想の表れです。

### 実務での使いどころ・移行の勘所
- コンテンツ主体のサイト、Astro/静的サイト、サーバ描画中心の MPA と特に好相性。
- 一覧→詳細の「共有要素モーフィング」がリッチな体験を低コストで実現。
- 未対応ブラウザでは単に通常遷移になる（壊れない）ため、プログレッシブエンハンスメントとして安全。過剰な演出は避ける。

## 📖 用語解説（このトピックとのつながり）
- **View Transitions**: 画面遷移をアニメ化する仕組み。SPA 版と MPA 版がある。
- **クロスドキュメント**: 別々の HTML ページをまたぐこと。今回の新対応領域。
- **MPA / SPA**: 複数ページ構成／単一ページ構成。MPA でも滑らかにできるのが要点。
- **view-transition-name**: ページ間で補間する共有要素を紐づける名前。
- **共有要素モーフィング**: 一覧の要素が詳細へ変形移動する演出。体験向上の核。

## 影響範囲 / 推奨アクション
MPA・静的サイト・コンテンツ主体サイトの開発者に恩恵大。新規サイトはまず `@view-transition` を宣言して基本のフェードを入れ、一覧→詳細で共有要素名を付けるのが低コストで効果的。当ポートフォリオのような複数ページ構成でも段階導入しやすい。
