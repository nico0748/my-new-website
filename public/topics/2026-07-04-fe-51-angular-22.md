# Angular 22 リリース — Signals 中心のリアクティビティが本格化

> トピック: 開発トピック｜出典: Angular GitHub Releases｜種別: frontend

## 一行サマリー
Angular の最新メジャー Angular 22 が 2026 年前半にリリースされ、Signals ベースのリアクティビティと Zone.js なし（zoneless）構成の成熟が進んだ。長らく「重厚なフレームワーク」と見られてきた Angular が、細粒度リアクティビティの潮流に本格的に舵を切った節目となる。

## 🔰 初学者向け（何が・なぜ重要か）
Web 画面は「データが変わったら、その部分だけ描き直す」ことが理想です。ところが従来の Angular は Zone.js という仕組みで「何かが起きたら、とりあえずアプリ全体を見回して変化を探す」やり方をとっていました。これは確実ですが、規模が大きくなると無駄な確認作業が増え、動作が重くなりがちでした。

Angular 22 が推し進める Signals（シグナル）は、値そのものに「私が変わったら、私を使っている場所だけ教えてね」という通知機能を持たせる考え方です。電球（画面）と配線（依存関係）を明示的につないでおき、スイッチ（データ）を押した電球だけが点く、というイメージです。全部屋を見回らなくて済むので速く、無駄がありません。

つまり Angular 22 は「全体を巡回して変化を探す」旧来方式から「変わった値が変わった場所だけを直接更新する」新方式へと、フレームワークの土台を移し替えている段階だと理解すると分かりやすいです。

## 🧠 専門的解説（技術詳細・設計・使いどころ）
### 技術的詳細（何が変わったか）
Angular 22 は 2026 年前半のメジャーリリース（Angular 21 は 2025 年 11 月安定版、その後 22 が RC を経て公開）。近年の Angular は Signals（`signal` / `computed` / `effect`）と、これに連動した `input()` / `output()` / `model()` などの Signal ベース API を段階的に安定化してきた。加えて Zone.js を使わない zoneless な変更検知が実運用可能なレベルに達し、ビルド時のハイドレーション（incremental hydration）や制御フロー構文（`@if` / `@for` / `@defer`）と組み合わせて初期表示・実行時性能を底上げする。正確な各機能の stable/experimental 区分と最小バージョンは公式リリースノートで要確認。

### 仕組み / 背景（なぜこう設計されたか）
従来の Zone.js 方式は「非同期処理をパッチして、処理後にコンポーネントツリーを走査（dirty checking）する」ため、実装が変更を意識しなくても動く堅牢さがある反面、変更検知のコストがツリー規模に比例して膨らむ弱点があった。Signals は依存グラフを実行時に自動構築し、変わった値の下流だけを再評価する「pull ベースの細粒度リアクティビティ」を採る。これは Solid や Vue Vapor、Svelte runes と同じ系譜で、業界全体が仮想 DOM 全走査からグラフ駆動の局所更新へ向かう流れの一部だ。トレードオフとして、Zone.js 前提の既存ライブラリ・パターンとの互換や、effect の実行タイミング設計に注意が要る。

### 実務での使いどころ・移行の勘所
新規プロジェクトは zoneless + Signals を前提に設計するとメリットが最大化する。既存の大規模アプリは、まず制御フロー構文と standalone components への移行を済ませ、状態を段階的に Signals へ寄せてから zoneless 化を検討するのが安全。RxJS を捨てる必要はなく、非同期ストリームは RxJS、UI 状態は Signals という役割分担が現実的。`ng update` の自動マイグレーションを活用しつつ、変更検知に依存した副作用コードを洗い出すことが移行の勘所になる。

## 📖 用語解説（このトピックとのつながり）
- **Signals（シグナル）**: 値と依存関係を結びつけ、変化を局所的に伝播させるリアクティブ・プリミティブ。Angular 22 の中核。
- **Zone.js / zoneless**: 非同期をパッチして変更検知を駆動する従来の仕組みと、それを使わない新方式。Signals により zoneless が実用段階に。
- **変更検知（change detection）**: 状態変化を画面に反映する処理。全走査からグラフ駆動の局所更新へ移行中。
- **制御フロー構文（`@if` / `@for` / `@defer`）**: テンプレート内の分岐・繰り返し・遅延読み込みを担う新構文。Signals と好相性。
- **incremental hydration**: SSR 済み HTML を、必要な部分だけ後から対話可能にする手法。初期表示を軽くする。

## 影響範囲 / 推奨アクション
Angular を採用する企業・チーム全般に影響。新規は zoneless + Signals を既定に。既存は standalone / 制御フロー構文への移行を先行させ、状態管理を段階的に Signals へ。RxJS 資産は非同期ストリーム用途で共存可能。移行時は `ng update` と ESLint による規約チェックを併用する。
