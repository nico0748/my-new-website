# Joomla Content Editor（JCE）アクセス制御不備 CVE-2026-48907（CVSS 10.0）でPHP Webシェルを設置される

> トピック: 脆弱性ニュース ｜ 出典: The Hacker News / BleepingComputer / CISA KEV ｜ 種別: アクセス制御不備（CWE-284）→ 任意PHPコード実行

## 一行サマリー

Joomla用の人気エディタ拡張「JCE（Joomla Content Editor）」に最大深刻度（CVSS 10.0）のアクセス制御不備があり、未認証の攻撃者が不正な「エディタプロファイル」を作成してPHP Webシェルをアップロード・実行できる。実環境で自動化された攻撃が進行中で、CISAはKEV登録・連邦機関に6/19までのパッチ適用を命じた。

## 🔰 初学者向け（何が・なぜ問題か）

Joomla（ジュームラ）は、ブログや企業サイトを作るための「ホームページ作成ソフト」（CMS=コンテンツ管理システム）です。JCEはその上で文章や画像を編集するための「高機能ワープロ部品」だと思ってください。

このワープロ部品には「誰がどの編集機能を使えるか」を決める**入館証チェック**があります。今回の不具合は、本来ログインしている管理者しか触れないはずの「編集者の権限設定（プロファイル）を新しく作る」ボタンに、**入館証を持たない人（未認証＝ログインすらしていない外部の人）でも手が届いてしまう**というものです。

攻撃者はまず「何でもアップロードしてよい」という都合のいい権限プロファイルを勝手に新規作成し、その権限を使って**PHPという“サーバー上で動くプログラム”ファイル**を投げ込みます。これが置けてしまうと、サイトのサーバーを遠隔から操る「裏口（Webシェル）」が完成します。一度裏口を作られると、パッチを当てた後でも裏口は残るため、見つけて消すまで侵入され続けます。被害イメージは「玄関の鍵を直しても、すでに作られた合鍵で勝手口から入られ続ける」状態です。

## 🧠 専門的解説（技術詳細・メカニズム・検知対応）

### 技術的詳細（種別・前提条件）

- **CVE**: CVE-2026-48907 / **CVSS**: 10.0（Critical・最大値）/ **種別**: 不適切なアクセス制御（CWE-284、実害は任意PHPコード実行）。
- **対象**: Widget Factory「Joomla Content Editor（JCE）」 1.0.0 〜 2.9.99.4。
- **修正版**: JCE 2.9.99.5（2026-06-03 公開）。
- **前提条件**: 当該拡張が導入されたJoomlaサイトが外部公開されていること。**認証不要・ユーザー操作不要**でネットワーク経由から成立。
- **悪用状況**: 実環境で悪用（active exploitation）・公開エクスプロイトあり・攻撃は自動化、数百サイト規模で被害報告。CISA KEV登録、FCEBは2026-06-19までの是正命令。

### メカニズム（攻撃が成立する仕組み）

本件は「信頼境界（未認証ユーザー ↔ 管理者専用の設定変更機能）」をまたぐ権限チェック欠落が、ファイルアップロードという危険プリミティブに直結した典型例。

1. **アクセス制御の欠落**: JCEのエディタプロファイル作成・編集を担うエンドポイントが、呼び出し元の認証/権限を十分検証していない。本来は管理者ロールが前提のはずの操作が、未認証リクエストで到達可能になっている（信頼境界の取り違え）。
2. **ポリシーの書き換え（攻撃者がプリミティブを獲得）**: 攻撃者は「アップロード可能な拡張子に .php を含む」「許可ディレクトリを緩める」等の**緩いプロファイルを新規作成**する。これにより本来禁止されるはずの実行可能ファイルのアップロードが“正規機能の範囲内”として通ってしまう。
3. **任意ファイル設置→コード実行への橋渡し**: 緩めたプロファイル経由でPHPファイル（Webシェル）をWeb公開ディレクトリへ設置。以後、攻撃者がそのURLにアクセスするだけでサーバー上で任意PHPが実行され、未認証RCEが完成する。
4. **永続化**: 設置されたWebシェルはパッチ適用後も残存し、バックドアとして機能する。

脆弱性クラスとしては「アクセス制御欠落（Missing Authorization）＋危険なファイルアップロード（Unrestricted File Upload, CWE-434）」の合流。**実際の武器化手順・PoCコード・具体的なリクエスト構造は本稿では扱わない**（概念レベルにとどめる）。

### 検知・暫定緩和・恒久対策

- **検知の着眼点**: Webルート配下に身に覚えのない .php ファイルが出現していないか。JCEのプロファイル設定が短時間に新規作成・変更された監査ログ。アクセスログでエディタ管理系エンドポイントへの未認証POSTや、不審なUser-Agent/自動化痕跡。設置済みWebシェルへの定期的なアクセス。
- **暫定緩和**: 即時パッチが基本。困難な場合はWAFで当該エンドポイントへの未認証アクセスを遮断、アップロードディレクトリのPHP実行を無効化（`.htaccess`等で実行権を剥奪）。
- **恒久対策**: JCE 2.9.99.5 以降へ更新。**更新は入口を塞ぐだけで、侵害済みサイトの清掃にはならない**点に注意——Webシェル・不正プロファイル・追加された管理ユーザーの有無を必ず精査し、必要に応じてクリーンな状態から再構築。

## 📖 用語解説（このニュースとのつながり）

- **CMS / Joomla**: Webサイトを作る土台ソフト。今回はそのプラグイン（JCE）が穴になった。
- **エディタプロファイル**: 「誰が・何を・どこへ」アップロード/編集できるかを定義する設定。攻撃者はこれを勝手に作って制限を外した。
- **Webシェル**: サーバー上で動く遠隔操作用の不正スクリプト。設置されると“裏口”になる。
- **アクセス制御不備（CWE-284）**: 「この操作は誰がしてよいか」のチェック漏れ。今回の根本原因。
- **KEV**: CISAが「実際に悪用が確認された」と認定した脆弱性リスト。掲載＝最優先で対処すべき印。

## 影響範囲 / 推奨対応

JCE 1.0.0〜2.9.99.4 を使う全Joomlaサイトが対象。国内でも中小企業・自治体・教育機関のサイトでJoomla+JCEは一定数稼働しており、未認証・自動化攻撃のため露出していれば早晩スキャンされる。**最優先で2.9.99.5以降へ更新し、既存侵害（Webシェル・不正プロファイル・不審管理者）の有無を点検**すること。
