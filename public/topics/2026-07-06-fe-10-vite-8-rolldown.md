# Vite 8 — Rolldown が既定バンドラに（Rust化で最大10〜30倍）

> トピック: Vite 8｜出典: Vite 公式ブログ / rolldown-vite リポジトリ｜種別: メジャーリリース（2026-03-12）

## 一行サマリー

Vite 8（2026年3月）は、Rust 製バンドラ Rolldown を全ユーザーに既定採用し、esbuild と Rollup を Rolldown・Oxc ベースのツールへ置き換えた。プラグイン互換を保ちつつビルドを最大10〜30倍高速化し、移行用の `rolldown-vite` パッケージは役目を終えてアーカイブ済み。

## 🔰 初学者向け（何が・なぜ重要か）

Vite は今やフロントエンド開発の土台的なビルドツールで、Astro・Nuxt・SvelteKit・多くの React プロジェクトが内部で使っています。ビルドツールの仕事は、たくさんの小さなファイル（コード）を、ブラウザが効率よく読み込める形にまとめる（＝バンドルする）こと。Vite はこれまで、開発中は esbuild、本番ビルドは Rollup という2つの別々の道具を使い分けていました。

Vite 8 は、この2つを Rolldown という「Rust で書かれた1つの高速な道具」に一本化しました。Rust はメモリ効率が高くとても速い言語で、これによりビルド時間が桁で短くなります。ある実例では本番ビルドが46秒から6秒（約87%減）になったと報告されています。開発中と本番で同じ道具を使うので、「開発では通ったのに本番ビルドで挙動が違う」というズレも減ります。

さらに嬉しいのは、既存のプラグイン（拡張機能）がそのまま使えるよう互換性を保っている点。速くなるのに、多くの場合ほぼ設定変更なしで乗れる——これが大きな価値です。

## 🧠 専門的解説（技術詳細・設計・使いどころ）

### 技術的詳細（何が変わったか）

- **Rolldown が単一の既定バンドラに**: opt-in なしで全ユーザーに適用。開発の esbuild と本番の Rollup を、Rolldown・Oxc ベースのツールへ統一。
- **性能**: プラグイン互換を維持しつつ、最大10〜30倍高速なビルド。実例では Linear の本番ビルドが46秒→6秒（約87%減）。
- **移行パッケージの終了**: 技術プレビューだった `rolldown-vite` は Vite 8 安定版で不要となり、リポジトリは2026-03-19にアーカイブ（読み取り専用）化。
- リリースは 2026-03-12。

### 仕組み / 背景（なぜこう設計されたか）

Vite の速さは元来「開発時はネイティブ ESM でバンドルを避け、esbuild で依存を事前バンドルする」ことに由来した。だが本番は最適化のため Rollup でバンドルする必要があり、開発と本番で挙動が微妙に食い違う二重構成が長年の課題だった。Rolldown は Rollup 互換のAPIを Rust で実装したバンドラで、これに一本化すれば開発・本番の道具差を解消でき、かつ Rust 化で大規模プロジェクトのビルドが桁で速くなる。Oxc（Rust 製の JS/TS ツールチェーン）と組み合わせることで、パース・変換・バンドルまでネイティブ実装で貫ける。トレードオフは、成熟した Rollup プラグイン資産との互換をどこまで完全に保てるか——公式は互換維持を掲げるが、特殊なプラグインやカスタム設定では検証が要る。フロントエンド全体で進む「JS 実装ツールの Rust 置換」の到達点の一つだ。

### 実務での使いどころ・移行の勘所

Vite を使う既存プロジェクトは、Vite 8 への更新でビルド高速化の恩恵をほぼ自動で受けられる可能性が高い。移行時の確認ポイントは、(1) カスタム Rollup プラグイン／`rollupOptions` の互換、(2) esbuild 固有の変換に依存した設定、(3) CI のビルド時間・成果物の同一性検証。大規模モノレポほど短縮効果が大きい。上位フレームワーク（Astro/Nuxt/SvelteKit 等）は追随して Vite 8 系へ上げるため、フレームワーク側のサポート表明を待って合わせるのが安全。数値（10〜30倍・87%減）はプロジェクト構成に強く依存するため、自環境でベンチして裏取りすること。

## 📖 用語解説（このトピックとのつながり）

- **バンドラ**: 多数のモジュールを最適化して1つ（または数個）にまとめる道具。Vite 8 では Rolldown。
- **Rolldown**: Rollup 互換APIを Rust で実装した高速バンドラ。Vite 8 の既定。
- **Oxc**: Rust 製の JS/TS ツールチェーン（パーサ/リンタ等）。Rolldown と組み合わせて使われる。
- **esbuild / Rollup**: 従来 Vite が開発/本番で使い分けていた Go/JS 製ツール。Vite 8 で Rolldown に統一。

## 影響範囲 / 推奨アクション

- 対象: Vite を直接、または Astro/Nuxt/SvelteKit 経由で使う全フロントエンド。
- アクション: Vite 8 更新後、カスタム Rollup プラグインと `rollupOptions` の互換、CI ビルド時間・成果物同一性を検証。上位フレームワークは公式サポート表明に合わせて更新。性能値は自環境で計測する。
