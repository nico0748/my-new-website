# Cisco Catalyst SD-WAN Manager の出力エスケープ不備でroot奪取（CVE-2026-20245）

> トピック: 脆弱性ニュース ｜ 出典: The Hacker News / Cisco Security Advisory / CISA KEV ｜ 種別: コマンドインジェクション・権限昇格

## 一行サマリー
Cisco Catalyst SD-WAN Manager のCLIに入力検証の不備（CVE-2026-20245, CVSS 7.8）があり、netadmin権限を持つ攻撃者が細工したファイルのアップロードを通じてroot権限でコマンドを実行できる。すでに実環境で悪用が確認され、CISAのKEVに登録済みのため早急なパッチ適用が必要。

## 🔰 初学者向け（何が・なぜ問題か）
SD-WAN Manager は、社内の複数拠点をつなぐネットワーク（SD-WAN）を一括管理する「司令塔」のような製品です。各拠点のルーターに「こう動け」と設定を配るのがこの司令塔の役目です。

今回の問題は、司令塔に「中くらいの権限で入れる管理者アカウント（netadmin）」を持っている人が、特定の細工をしたファイルをアップロードすると、本来できないはずの「機械の一番偉い権限（root）」を奪えてしまう、というものです。たとえるなら、ビルの「フロア管理者カード」しか持っていない人が、ちょっとした小細工で「ビル全体のマスターキー」を作れてしまうようなものです。

司令塔を乗っ取られると、攻撃者は各拠点のルーターに勝手な設定を配れます。実際に「エッジ機器（拠点側のルーター）へ設定変更が押し込まれた」事例も観測されており、ネットワーク全体が攻撃者の意のままになる危険があります。

## 🧠 専門的解説（技術詳細・メカニズム・検知対応）

### 技術的詳細（種別・前提条件）
- **CVE / CVSS**: CVE-2026-20245、CVSS 7.8（High）。
- **種別（CWE）**: CWE-116（Improper Encoding or Escaping of Output / 出力の不適切なエンコード・エスケープ）。製品CLIにおける利用者供給入力の検証不足に起因。
- **前提条件**: 攻撃者は対象システム上で **netadmin 権限**を保有していること（事前認証は必要だが管理権限の昇格が成立）。
- **影響バージョン / 修正版**: Catalyst SD-WAN release 20.18.2.1 以前が影響。**20.18.3.1** で修正（2026-06-12 リリース）。SD-WAN Controller / Manager / Validator が関連（要確認: 製品ごとの該当範囲はCisco Advisoryで確認）。
- **悪用状況**: Cisco が限定的な悪用を確認。CISA KEV 登録済み。Google Mandiant の研究者が発見・報告。

### メカニズム（攻撃が成立する仕組み）
本件は「管理機能の出力／入力処理に対するエスケープ不備」が、ファイルアップロード経路を通じてOSコマンド実行へ橋渡しされる典型例である（CVE詳細は限定公開のため概念レベルで説明する）。

1. **信頼境界**: netadmin が操作するCLI／管理機能は「管理者が入力する値は安全」という暗黙の前提で実装されがち。ここが第一の弱点。
2. **不備の所在**: アップロードされたファイル名・内容など利用者供給データが、内部でシェルやコマンド文字列を組み立てる際に適切にエスケープ／エンコードされない（CWE-116）。
3. **獲得プリミティブ**: 攻撃者は「コマンド文字列への任意文字列の混入（コマンドインジェクション）」というプリミティブを得る。管理プロセスが高権限で動くため、混入されたコマンドはその権限で実行される。
4. **昇格の成立**: netadmin（限定管理者）→ root（OS最上位）へと権限が跳ね上がる。管理プレーンを掌握した攻撃者は、エッジ機器への設定配布機能を悪用して下流ネットワークへ影響を波及できる。

一般原理として、管理者専用機能であっても「認証済み＝信頼できる」と扱ってはならない。入力の信頼境界はユーザーの権限レベルではなく、データの出所で引くべきである。

### 検知・暫定緩和・恒久対策
- **検知の着眼点**: 管理UI/CLIでの予期しないファイルアップロード、root権限での想定外プロセス生成、エッジ機器への突発的な設定プッシュ（config変更履歴）を監視。
- **暫定緩和**: 管理インターフェースへのアクセスを管理ネットワーク／既知IPに限定。netadmin アカウントの棚卸しと最小権限化、認証情報のローテーション。
- **恒久対策**: 20.18.3.1 以降へアップグレード。管理プレーンのネットワーク分離（アウトオブバンド管理）を徹底。

## 📖 用語解説（このニュースとのつながり）
- **SD-WAN Manager**: SD-WAN全体を集中管理する管理プレーン製品。今回はここが乗っ取り対象。
- **CWE-116（出力エスケープ不備）**: データを別の文脈（シェル等）へ渡す際に特殊文字を無害化しない欠陥。本件のコマンドインジェクションの根本原因。
- **netadmin**: 製品上の管理者ロールの一つ。今回の攻撃の「前提権限」。
- **管理プレーン**: 機器の設定・制御を司る層。ここを奪われると配下機器すべてに影響が及ぶ。
- **KEV**: CISAが「実際に悪用が確認された」脆弱性をまとめたカタログ。登録は優先対応のサイン。

## 影響範囲 / 推奨対応
影響は Catalyst SD-WAN Manager（20.18.2.1 以前）を運用する組織。SD-WANで多拠点を束ねる企業ネットワークでは、1台の侵害が全拠点に波及しうる。直ちに 20.18.3.1 へ更新し、netadmin 権限の保有者を最小化、管理面のネットワーク分離と監査ログ確認を実施する。
