# Mistral Large 3 — Apache 2.0 の 675B MoE、Blackwell 向け NVFP4 版も提供

> トピック: Mistral Large 3 のオープンウェイト公開 ｜ 出典: Mistral AI 公式 / LMArena ｜ 種別: 新モデルリリース

## 一行サマリー

欧州の Mistral AI が旗艦オープンウェイトモデル Mistral Large 3 を Apache 2.0 で公開。675B 総 / 41B 活性の MoE で 262K 文脈を持ち、多言語・画像理解に強い。vLLM・Red Hat と組んだ NVFP4 版チェックポイントも用意され、Blackwell NVL72 や 8×A100/H100 単一ノードで効率的に動く。

## 🔰 初学者向け（何が・なぜ重要か）

Mistral AI はフランス発の AI 企業で、「オープンウェイト（重みを公開して誰でも使える）」路線を代表する存在だ。米中の巨大企業に対し、欧州から高性能モデルを自由なライセンスで出すことで存在感を保ってきた。

**Mistral Large 3 はその旗艦モデルで、ライセンスが Apache 2.0**——商用でも改造でも自由に使える緩い条件だ。中身は MoE（混合エキスパート）で、全体 6750億パラメータのうち実際に動くのは 410億。巨大な知識量を持ちつつ動作コストを抑える、いまの大型モデルの定石を踏む。**262K の文脈長**で長い文書もまとめて読め、**多言語**（特に多言語会話に強い）と**画像理解**にも対応する。

もうひとつ実務的に大きいのが、**NVFP4 という 4bit 形式の配布版**が用意されたこと。これは「モデルを軽く圧縮して、少ない GPU でも速く動かす」ための形式で、NVIDIA の新しい Blackwell 世代 GPU や、比較的入手しやすい 8 枚構成の H100/A100 サーバー 1 台で回せる。つまり「自由に使えて、多言語・画像もこなし、現実的なハードで動く」——自前で高性能 AI を運用したい組織の選択肢になる。

## 🧠 専門的解説（技術詳細・設計・使いどころ）

### 技術的詳細（何が変わったか）

- **公開**: 2025-12-02。**Apache 2.0** ライセンスで、ベースと指示チューニング版の双方を提供。Hugging Face／Mistral プラットフォーム（API）で利用可。
- **アーキテクチャ**: スパース MoE、**41B 活性 / 675B 総**パラメータ。NVIDIA H200 GPU 3000 枚でスクラッチ学習。
- **文脈長**: 262K。多言語サポートが強力で、画像理解も備える。
- **性能**: ポストトレーニング後、一般プロンプトで最良クラスのオープンウェイト指示モデルと同等。LMArena の OSS 非推論モデル部門で #2 デビューとの報。
- **量子化配布**: vLLM・Red Hat と協働し **NVFP4 形式**のチェックポイントを公開。Blackwell NVL72 や 8×A100 / 8×H100 単一ノードで効率実行。
- ※ 2025-12 公開のため"最新"というより 2026 の旗艦オープンウェイトとして継続的に参照される。ベンチ・順位は公表ベースで独立検証は「要確認」。

### 仕組み / 背景（なぜこう設計されたか）

**スパース MoE のねらい**。41B 活性 / 675B 総という比率は、「密モデル 41B 級の推論コストで、675B 相当の容量を引き出す」設計。ルータが入力ごとに一部エキスパートを選び、実効計算を抑える。多言語・多ドメインのように"局所的に異なる専門性"を要するタスクで、エキスパートの棲み分けが効きやすい。代償は全エキスパートを保持するメモリ負荷で、単一ノードに載せるには量子化がほぼ前提になる。

**なぜ NVFP4 版を用意したか**。NVFP4 は NVIDIA Blackwell が対応する 4bit 浮動小数フォーマットで、FP8 等より小さいメモリ・高いスループットで推論できる。ベンダ（vLLM/Red Hat）と組んで公式の量子化チェックポイントを出すことは、「重みを配るだけでなく、現実的なハードで回せる状態まで面倒を見る」という実運用志向の表れだ。4bit 化は精度劣化のリスクを伴うが、フォーマットと較正を作り込むことで実用精度を保つ狙い。トレードオフは、最新 GPU（Blackwell）で最も効く一方、旧世代では恩恵が限られる点。

**オープンウェイト戦略**。Apache 2.0 で旗艦を出すことは、クローズド API 勢に対し「自前で回せる・改造できる高性能」を武器にする戦略。特に欧州のデータ主権・規制要件を重視する組織に訴求する。

### 実務での使いどころ・移行の勘所

多言語プロダクト、画像＋テキストのマルチモーダル処理、データを外に出せない自社運用に向く。導入の勘所はハードウェアと量子化——フル精度は 675B ゆえ重く、現実には NVFP4 等の量子化版を単一ノード（8×H100/A100 や Blackwell）で回す構成が基本。最新の DeepSeek V4 系や他のオープンウェイトと、自社タスクでの精度・コストを比較評価するとよい。2025年末公開のため、より新しいモデルが出ている点も踏まえ、"多言語・欧州・Apache 2.0"という強みが要件に合うかで採否を判断する。

## 📖 用語解説（このトピックとのつながり）

- **オープンウェイト / Apache 2.0**: 重み公開かつ商用・改変に寛容なライセンス。Mistral の路線。
- **スパース MoE / 活性パラメータ**: 一部エキスパートだけ起こす構造と実効計算量。41B 活性が効率の鍵。
- **文脈長**: 一度に扱える入力量。Large 3 は 262K。
- **NVFP4**: NVIDIA Blackwell 対応の 4bit 浮動小数フォーマット。省メモリ・高速推論を可能に。
- **量子化**: 重みを低ビット化して軽量・高速化する技術。単一ノード実行に必須級。
- **LMArena**: 人間投票ベースのモデル比較アリーナ。OSS 部門 #2 デビューの根拠。
- **マルチモーダル**: テキストに加え画像等も扱う能力。Large 3 は画像理解対応。

## 影響範囲 / 推奨アクション

- 多言語・マルチモーダル・自社運用が要件の組織に有力。特に欧州のデータ主権重視環境。
- フル精度は重いため NVFP4 等の量子化版＋単一ノード構成が現実的。Blackwell で最も効く。
- 2025末公開ゆえ、より新しいオープンウェイト（DeepSeek V4 等）と自社タスクで精度・コスト比較を。
