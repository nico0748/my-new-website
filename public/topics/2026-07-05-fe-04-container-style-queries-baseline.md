# カスタムプロパティの Container Style Queries が Baseline 入り（Firefox 151 対応）

> トピック: `@container` の `style()` クエリ（カスタムプロパティ）が Baseline Newly available｜出典: web.dev（2026年5月ダイジェスト）｜種別: Web 標準

## 一行サマリー

2026年5月、Firefox 151 が `@container` の `style()` クエリに対応し、カスタムプロパティ（CSS 変数）の値に応じてスタイルを分岐する「コンテナ・スタイルクエリ」が全主要ブラウザで利用可能になった。親要素が持つ変数の状態を、子孫が CSS だけで参照して見た目を切り替えられる。

## 🔰 初学者向け（何が・なぜ重要か）

これまで CSS には、「画面幅がこれ以上なら」で切り替えるメディアクエリや、「この箱の幅がこれ以上なら」で切り替えるコンテナクエリがありました。どちらも「大きさ」を基準にした切り替えです。

今回 Baseline 入りしたのは「大きさ」ではなく「状態（値）」で切り替える仕組みです。CSS 変数（カスタムプロパティ）に `--theme: dark` や `--tone: warning` のような値を入れておき、「もし近くの箱の `--tone` が warning なら、この文字を赤くする」といった分岐が CSS だけで書けます。

たとえるなら、部屋に「今日の気分」という名札を貼っておき、その名札を見た家具たちが勝手に色を変えるようなもの。親が状態を宣言すれば、子孫はそれを読んで自律的に装飾を変えられます。Firefox が対応したことで主要ブラウザが出揃い、実務投入できる段階になりました。

## 🧠 専門的解説（技術詳細・設計・使いどころ）

### 技術的詳細（何が変わったか）

コンテナクエリには「サイズクエリ（`@container (min-width: ...)`）」と「スタイルクエリ（`@container style(...)`）」の2系統がある。今回 Baseline 入りしたのは後者、特に**カスタムプロパティを条件にする** `style()` クエリ。例:

```css
@container style(--tone: warning) {
  .label { color: var(--accent-amber); }
}
```

祖先要素で `--tone: warning` が設定されていれば、そのコンテナ配下の `.label` にスタイルが適用される。2026年5月に Firefox 151 が対応し、全主要エンジンで実装が揃って Baseline Newly available になった。

### 仕組み / 背景（なぜこう設計されたか）

従来、「親の状態に応じて子の見た目を変える」には、親にクラスを付け替える JS か、変数を直接使った値の伝播（`color: var(--tone-color)`）しかなかった。前者は JS 依存、後者は「値を差し込む」ことはできても「値に応じて全く別のルール群へ分岐する」ことは苦手だった。`style()` クエリは、カスタムプロパティを**条件式**として使えるようにし、状態に応じたルールブロックの切り替えを宣言的に可能にした。

設計思想は「関心の分離」だ。状態の宣言（親が `--tone` を決める）と、状態への反応（子が `@container style()` で装飾を決める）を分離することで、コンポーネントの再利用性が上がる。トレードオフは、現状 `style()` クエリが実務では主にカスタムプロパティ条件で使われる点（任意の CSS プロパティ値のクエリは仕様・実装が段階的）で、複雑な条件を組むと可読性が落ちやすい。カスケードやコンテナの入れ子との相互作用も設計時に意識が要る。

### 実務での使いどころ・移行の勘所

テーマ切り替え（light/dark、brand バリアント）、トーン（info/success/warning/error）に応じたコンポーネント装飾、密度モードなど「親が状態を宣言し子孫が反応する」設計に最適。デザインシステムで JS のクラス付け替えを削減できる。移行時は、非対応ブラウザで分岐が無視されるため、既定スタイルを先に定義し `style()` は上書き強化として使う。値の命名規則（`--tone` などのトークン設計）を先に固めておくと破綻しにくい。

## 📖 用語解説（このトピックとのつながり）

- **Container Style Queries**: コンテナが持つスタイル（主にカスタムプロパティ値）に応じて分岐する `@container style()`。本トピックの主役。
- **カスタムプロパティ（CSS 変数）**: `--name: value` で定義し `var()` で参照する値。`style()` の条件に使う。
- **Container Size Queries**: 祖先要素のサイズで分岐する従来のコンテナクエリ。スタイルクエリと対。
- **関心の分離**: 状態の宣言と状態への反応を分ける設計方針。本機能が後押しする。

## 影響範囲 / 推奨アクション

テーマ・トーン駆動のデザインシステムは `style()` クエリで JS を削減可能。トークン命名を先に設計し、既定スタイル＋上書きの形で段階導入するのが安全。非対応環境では既定スタイルが効くよう設計する。
