# 相対色 oklch() + color-mix() が Baseline 入り ― ネイティブな色操作が CSS で完結

> トピック: 開発トピック｜出典: southwellmedia｜種別: frontend

## 一行サマリー
知覚均等な色空間 **oklch()**、既存色を加工する **相対色構文（`from`）**、2色を混ぜる **color-mix()** が揃って Baseline に到達（時期は要確認）。Sass や JS を使わず、CSS だけで色の明度調整・ホバー色生成・混色ができるようになった。

## 🔰 初学者向け（何が・なぜ重要か）
ボタンにマウスを乗せると少し暗くなる、ブランドの基準色から明るい/暗いバリエーションを作る——こうした「色を少しだけいじる」作業は、デザインで日常的に発生します。従来 CSS だけではこれが苦手で、色を暗くしたければ Sass の `darken()` を使ったり、あらかじめ全バリエーションを手で用意したりしていました。`#00e5cc` を少し暗くした値を、いちいち別の 16 進数で書いていたわけです。

新しく Baseline 入りした一連の機能は、これを **CSS の中だけで動的に**行えるようにします。`oklch()` は人間の目に均等な色空間で「明るさだけ 10% 下げる」が直感的にでき、相対色構文は「この既存色を元に明度だけ変える」を書け、`color-mix()` は「この色とあの色を 30% 混ぜる」を書けます。基準色を 1 つ CSS 変数で持てば、そこから派生色を自動生成できる、というのが大きな価値です。

## 🧠 専門的解説（技術詳細・設計・使いどころ）
### 技術的詳細（何が変わったか）
- **oklch()**: 明度 L・彩度 C・色相 H を指定する知覚均等な色空間。明度操作が直感的で破綻しにくい。
- **相対色構文**: `oklch(from var(--accent) calc(l - 0.1) c h)` のように既存色を分解して再構成できる。
- **color-mix()**: `color-mix(in oklch, var(--a), var(--b) 30%)` で 2 色を任意比率で混合。
- 3 機能が揃って Baseline 化し、実運用で組み合わせられるようになった（時期は要確認）。

### 仕組み / 背景（なぜこう設計されたか）
従来の `hsl()`／`rgb()` は、同じ明度指定でも色相によって見た目の明るさがバラつく問題（知覚非均等）がありました。oklch は人間の知覚に沿って設計され、「L を一定に保てば見た目の明るさも揃う」ため、体系的なカラースケール生成に向きます。相対色と color-mix を CSS ネイティブにしたのは、Sass などプリプロセッサや JS に頼らず、**CSS 変数と組み合わせて実行時に色を計算**できるようにするためです。テーマ切替やダークモードで威力を発揮します。

### 実務での使いどころ・移行の勘所
- 単一のブランド色から hover/active/border 用の派生色を CSS だけで生成できる。
- Sass の色関数依存を減らし、CSS 変数ベースのデザイントークン運用に統一しやすい。
- ガンマ/色域の扱いが従来と異なるため、既存色を機械的に oklch へ置換すると微妙な色ズレが出うる。目視確認を推奨。

## 📖 用語解説（このトピックとのつながり）
- **oklch()**: 知覚均等な色空間。体系的なカラースケール生成の土台。
- **相対色構文（from）**: 既存色を分解・再構成して派生色を作る CSS 機能。
- **color-mix()**: 2 色を任意比率で混ぜる関数。ホバー色や中間色に有用。
- **知覚均等**: 数値変化と見た目変化が比例する性質。oklch の利点。
- **デザイントークン**: 色などデザインの最小単位。CSS 変数 + oklch で動的運用可能。

## 影響範囲 / 推奨アクション
デザインシステムやテーマを扱う開発者に影響大。基準色を CSS 変数で 1 つ持ち、相対色/color-mix で派生色を生成する設計に移行すると保守性が上がる。既存パレットを oklch 化する際は色ズレを目視確認すること。当ポートフォリオのアクセント色（`--accent` #00e5cc）派生にも応用余地あり。
