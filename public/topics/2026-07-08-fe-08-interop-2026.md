# Interop 2026：ブラウザ横断の互換性を進める 20 の重点領域（Anchor Positioning・cross-document View Transitions ほか）

> トピック: Interop 2026｜出典: web.dev / WebKit ブログ / Mozilla Hacks / Igalia｜種別: 業界横断イニシアチブ

## 一行サマリー

Interop 2026 は、Apple・Google・Igalia・Microsoft・Mozilla が協働して Web プラットフォーム機能のブラウザ間互換性を高める年次プロジェクト。33 提案からなる 20 の重点領域と 4 つの調査領域が合意され、Anchor Positioning、Dialog/Popover 改善（`closedby` 属性・`:open` 疑似クラス・`popover="hint"`）、cross-document View Transitions、WebRTC/WebTransport などが並ぶ。

## 🔰 初学者向け（何が・なぜ重要か）

同じ Web の機能でも、Chrome では動くのに Safari では動かない、といった「ブラウザ差」は開発者を長年悩ませてきた。差があると、開発者は各ブラウザ向けの回避コードを書いたり、便利な新機能の採用を諦めたりする羽目になる。

Interop はこの「ブラウザ差」を減らすための、主要ブラウザベンダーが毎年集まって行う共同プロジェクトだ。「今年はこの機能を、全ブラウザで同じように動くよう優先して直そう」という重点領域を決め、共通のテストスイート（Web Platform Tests）の合格率を競うように上げていく。開発者にとっては「来年にはこの機能が、どのブラウザでも安心して使えるようになる」という予測可能性が得られる。

2026 の重点には、要素を別要素の位置に合わせて配置する Anchor Positioning（ツールチップ配置などが楽になる）や、ページ遷移をアニメーションで滑らかにつなぐ cross-document View Transitions などが入っている。これらが全ブラウザで揃えば、JavaScript ライブラリに頼らず標準機能でリッチな UI を作れるようになる。

## 🧠 専門的解説（技術詳細・設計・使いどころ）

### 技術的詳細（何が変わったか）

- 運営: Apple・Google・Igalia・Microsoft・Mozilla の主要レンダリングエンジン貢献者。
- **20 の重点領域（33 提案）＋ 4 の調査領域**で合意。State of HTML / State of CSS 調査の上位課題を反映。
- 主な重点領域:
  - **Anchor Positioning**: ある要素を別要素の位置基準で配置（ツールチップ等）。
  - **Dialog / Popover 改善**: `dialog` の `closedby` 属性（閉じるユーザー操作を指定）、`:open` CSS 疑似クラス、`popover="hint"` グローバル属性（他の auto popover に従属するヒント型ポップオーバー）。
  - **Container Style Queries と View Transitions**: 2026 は **cross-document View Transitions**（ページ間遷移の滑らかなアニメーション）へ拡張。
  - **WebRTC / WebTransport**: WebRTC は 2025 からの継続で合格率 91.6% に到達。
- 調査領域: アクセシビリティテスト（ブラウザ間で一貫したアクセシビリティツリー生成）、JPEG XL、モバイルテスト、WebVTT。

### 仕組み / 背景（なぜこう設計されたか）

Interop の核心は「Web Platform Tests（WPT）という共通テストの合格率を、複数ベンダーが同時に押し上げる」という運用にある。仕様が同じでも、実装が微妙に食い違えば互換性は生まれない。そこで「合格すべきテスト」を機能ごとに束ね、各ブラウザの合格率を公開ダッシュボードで可視化することで、実装の収束を促す。競争ではなく協調のフレームワークだ。

重点領域の選定に State of HTML / State of CSS の調査（開発者の「使いたいのに互換性が不安」という声）を使うのが特徴。開発者の実需に近い機能から優先的に互換性を固めるため、Interop で扱われた機能は「近く Baseline に到達しやすい」目安にもなる。

2026 で cross-document View Transitions が入った意味は大きい。単一ページ内の遷移アニメーションは各ブラウザで先行実装されていたが、ページをまたぐ（MPA / 通常のリンク遷移）アニメーションは実装差が残っていた。これが揃えば、SPA を組まなくても「ネイティブアプリのような滑らかな画面遷移」を標準機能だけで実現しやすくなる。トレードオフというより、標準化の性質上「合意形成に時間がかかる」点が制約で、重点入り＝即全ブラウザ完成ではなく「今年中に収束を目指す」段階である点は理解しておく必要がある。

### 実務での使いどころ・移行の勘所

- Interop 2026 の重点機能は「今後 1 年で互換性が固まる見込み」の指標として採用計画に使える。
- Anchor Positioning はツールチップ / ポップオーバー / メニュー配置の JS ライブラリ置き換え候補。
- cross-document View Transitions は MPA サイトの遷移体験改善に有効（SPA 化せずリッチな遷移が可能に）。
- ただし「重点入り」は完成保証ではない。導入時点の実対応は caniuse / WPT ダッシュボードで確認し、フォールバックを用意する。

## 📖 用語解説（このトピックとのつながり）

- **Web Platform Tests（WPT）**: ブラウザ共通の適合性テスト群。Interop はこの合格率を指標に互換性を測る。
- **Anchor Positioning**: 要素を別要素基準で配置する CSS 機能。Interop 2026 重点領域。
- **View Transitions**: ビューの切り替えをアニメーション化する API。2026 は cross-document（ページ間）へ拡張。
- **`popover="hint"`**: 他の auto popover に従属するヒント型ポップオーバー属性。Dialog/Popover 改善の一部。
- **Baseline**: 主要ブラウザで安定利用できる指標。Interop 重点機能は Baseline 到達の先行指標になりやすい。

## 影響範囲 / 推奨アクション

- Anchor Positioning・cross-document View Transitions を JS ライブラリ置き換えの候補として評価。
- 重点入り機能でも導入時は WPT / caniuse で実対応を確認し、フォールバックを用意。
- アクセシビリティツリー一貫性（調査領域）の進展を、支援技術対応の品質向上として注視する。
