# Oracle PeopleSoft のゼロデイRCE（CVE-2026-35273）— ShinyHunters が100超組織を侵害

> トピック: 脆弱性ニュース ｜ 出典: The Hacker News / Google Cloud (Mandiant) / SecurityWeek ｜ 種別: 未認証リモートコード実行（CVSS 9.8・ゼロデイ悪用）

## 一行サマリー

Oracle PeopleSoft（PeopleTools）に**ログイン不要でサーバを乗っ取れる**ゼロデイRCE（CVE-2026-35273, CVSS 9.8）が存在し、恐喝集団 ShinyHunters が100超の組織（大学が中心、Nissan なども含む）でデータ窃取に悪用した。

## 🔰 初学者向け（何が・なぜ問題か）

PeopleSoft は、大学や大企業が**人事・給与・学生情報**などを扱う大型の業務システムです。中には氏名・住所・給与・社会保障番号のような機微な個人情報が大量に入っています。

今回の穴は「**ログインすら要らない**」タイプの最も危険なものでした。攻撃者は正規のIDもパスワードも持たず、ネット越しにHTTPで話しかけるだけで、サーバに**自分の命令を実行させられる**のです。家にたとえるなら、玄関の鍵を開ける必要すらなく、外から呼びかけるだけで家の中の人が言いなりに動いてしまうようなもの。

しかも Oracle が修正情報を出す前から攻撃が始まっていた**ゼロデイ**（＝防御側が知らないうちに悪用される穴）でした。恐喝集団 ShinyHunters がこれを使って100を超える組織に侵入し、データを盗んで「公開されたくなければ金を払え」と脅迫。被害は大学が約7割を占め、日本関連では Nissan の名も挙がっています。個人情報を預けている側にとっても他人事ではない事案です。

## 🧠 専門的解説（技術詳細・メカニズム・検知対応）

### 技術的詳細（種別・前提条件）

- **CVE**: CVE-2026-35273 ／ **CVSS 3.1: 9.8（Critical）**
- **製品/コンポーネント**: Oracle PeopleSoft Enterprise **PeopleTools**、**Updates Environment Management** コンポーネント。
- **種別（CWE）**: 未認証リモートコード実行。攻撃連鎖の性質から**デシリアライズ/ガジェットチェーン系（CWE-502 の可能性）**と推定されるが、Oracle は詳細を限定開示のため**要確認**。
- **前提条件**: **認証不要・ユーザー操作不要**、HTTP でのネットワーク到達性のみ。
- **悪用状況**: **ゼロデイとして実環境で悪用**。Mandiant が **UNC6240（＝ShinyHunters）**に帰属、活動期間は **5/27〜6/9**。Oracle は **6/10** に帯域外（out-of-band）でアドバイザリ公開。300超インスタンス・100超組織が被害、うち**約68%が高等教育機関**（多くが米国）。
- **修正**: Oracle の緊急アドバイザリに従い該当 PeopleTools を更新。

### メカニズム（攻撃が成立する仕組み）

攻撃者は「**古い脆弱性＋ゼロデイを組み合わせたガジェットチェーン**」を用いたと報告されている。以下は脆弱性クラスの型の説明であり、武器化手順・PoC・オフセットは示さない。

1. **信頼境界**: PeopleTools のような業務基盤は、外部からのHTTPリクエストを受け付ける公開面を持つ。ここで「未認証の入力」を安全に扱えるかが信頼境界。
2. **未認証の攻撃面**: Updates Environment Management という管理系コンポーネントが、認証前に処理されるリクエストを受理していたと推定される。認証の関所より手前に**攻撃者が触れる処理経路**が存在した。
3. **攻撃者が握るプリミティブ**: 未認証RCEでは、攻撃者は最終的に**サーバ上での任意コード実行**という最上位のプリミティブを得る。ガジェットチェーンでは、単体では無害な複数の処理（オブジェクト復元、リフレクション、ライブラリの副作用など）を数珠つなぎにして、この実行能力を組み立てる。
4. **橋渡し**: 「入力→内部オブジェクト化→意図しないメソッド実行」という流れ（デシリアライズ系の典型）で、データが**コード実行**へ橋渡しされる。認証が不要なため、この橋を渡るのに正規権限は要らない。
5. **侵入後の目的**: RCE 取得後は Web シェル設置・データ窃取・恐喝という定石。ShinyHunters は破壊より**データ窃取＋二重恐喝**を志向する。

一般原理として、業務基盤の「管理/更新系コンポーネント」は権限が強く、かつ複雑な入力（シリアライズ済みオブジェクト等）を扱うため、**未認証で到達できると致命的**になりやすい。ゼロデイ×既知バグの連鎖は、単独では低リスクの欠陥が組み合わせで凶悪化することを示す。

### 検知・暫定緩和・恒久対策

- **検知の着眼点**: PeopleTools/Updates Environment Management への異常なPOST、認証前エンドポイントへの不審アクセス、Webシェル設置痕、想定外プロセスからの外向き通信、5/27〜6/9 前後の侵害兆候の遡及調査（IoC は Mandiant/Arctic Wolf レポート参照）。
- **暫定緩和（パッチ前）**: 該当コンポーネント/管理面をインターネットから遮断、WAF で既知の悪用パターンを遮断、公開範囲の最小化。
- **恒久対策**: Oracle 緊急アドバイザリの**該当パッチを即時適用**。侵害前提でのフォレンジック（データ窃取有無・横展開）を実施し、資格情報をローテーション。
- **運用示唆**: ゼロデイ悪用済み事案は「パッチ適用＝完了」ではない。**侵害されていた前提**で証跡調査と通知義務（個人情報保護）の検討が必要。

## 📖 用語解説（このニュースとのつながり）

- **PeopleSoft / PeopleTools**: 大学・大企業向けの人事給与等の業務基盤。機微な個人情報を大量に保持し、標的価値が高い。
- **未認証RCE**: ログイン不要で遠隔からコードを実行される最上位クラスの脆弱性。今回の中核。
- **ゼロデイ**: 防御側が対策を持たないうちに悪用される脆弱性。本件は Oracle 公開前から悪用されていた。
- **ガジェットチェーン / デシリアライズ**: 単体では無害な処理を連結し、データからコード実行を組み立てる攻撃技法。未認証RCEの背後に多い。
- **ShinyHunters / UNC6240**: データ窃取と二重恐喝で知られる犯罪グループ。Mandiant の追跡名が UNC6240。
- **二重恐喝**: データを暗号化/窃取したうえ「公開するぞ」と脅す手口。今回はデータ窃取＋公開脅迫が中心。

## 影響範囲 / 推奨対応

- **影響範囲**: インターネット到達可能な PeopleSoft/PeopleTools（特に Updates Environment Management）を持つ組織。高等教育機関が最多被害、企業（Nissan 等）にも波及。
- **推奨対応**: (1) Oracle 緊急アドバイザリのパッチを即時適用、(2) 5/27〜6/9 を含む期間の侵害調査（Webシェル・データ持ち出し）、(3) 認証情報全ローテーション、(4) 管理/更新コンポーネントの公開停止、(5) 個人情報漏えい時の通知・法対応の検討。
