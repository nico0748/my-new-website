# vLLM の Model Runner V2 — 推論スケジューリングを刷新し GB200 で最大56%高スループット

> トピック: vLLM の Model Runner V2（MRV2、v0.20 系）｜出典: vLLM 公式ドキュメント（docs.vllm.ai）｜種別: 推論基盤の主要機能

## 一行サマリー

オープンソースの LLM 推論エンジン vLLM が Model Runner V2（MRV2、v0.20.0+）を導入。GB200 で最大56%のスループット向上、H100/Blackwell での FP8 推論、既定の継続的バッチング、SSE ストリーミング、マルチモーダル対応を備え、大規模な LLM サービングの効率をさらに引き上げた。

## 🔰 初学者向け（何が・なぜ重要か）

ChatGPT のような AI サービスは、裏側で「モデルに質問を渡して答えを生成する」計算（推論）を、膨大なユーザーぶん同時にこなしています。この推論を、いかに速く・安く・たくさん捌くかが、AI サービスの原価とレスポンスを決めます。vLLM は、この「推論を効率よく捌く」ためのオープンソースエンジンで、多くの AI 企業の土台になっています。

今回の Model Runner V2 は、その処理の段取り（スケジューリング）を作り直したバージョンです。ざっくり言うと「GPU という高価な計算機を、待ち時間なく・ムダなく働かせる」ための改良で、最新の GPU では従来より半分近く多くの処理をこなせるようになりました。

たとえるなら、行列のできるラーメン店が、厨房の動線と注文の捌き方を見直して、同じ人数・同じ設備でより多くの客をさばけるようにした、というイメージ。AI を提供する側にとっては、そのままコスト削減と応答改善に直結します。

## 🧠 専門的解説（技術詳細・設計・使いどころ）

### 技術的詳細（何が変わったか）

vLLM の安定版（v0.20 系、2026年5月時点で v0.20.2 等）に含まれる主な要素:

- **Model Runner V2（MRV2、v0.20.0+）**: GB200 で最大56%高いスループット。
- **FP8 推論**: H100・Blackwell GPU で大幅なスループット改善。
- **継続的バッチング（continuous batching）が既定**。
- **ストリーミング**: サーバ送信イベント（SSE）を内蔵サポート。
- **マルチモーダル対応**: 画像とテキスト。

MRV2 は、2025年に既定化された V1 エンジン（スケジューリング負荷の削減、ゼロコピーの DMA 転送など）の系譜を継ぐ改良。正確なベンチマーク条件・数値は公式ドキュメント／リリースノートで要確認。

### 仕組み / 背景（なぜこう設計されたか）

LLM 推論のボトルネックは、GPU メモリ帯域と「アイドル時間」だ。各リクエストは可変長のトークン列を逐次生成するため、単純に固定バッチで処理すると、短い応答が終わっても長い応答を待ってしまい GPU が遊ぶ。vLLM が広めた**継続的バッチング**は、生成が終わったスロットに新しいリクエストを即座に差し込み続けることで、GPU を常に満載に近づける。加えて vLLM の核である PagedAttention は、注意機構のキャッシュ（KV cache）を OS の仮想メモリのようにページ単位で管理し、メモリ断片化を抑えて同時実行数を増やす。

MRV2 が改善するのは、この処理を回す「スケジューラとモデル実行の段取り」だ。高い同時実行下ではスケジューリングそのもののオーバーヘッドが無視できなくなるため、V1/MRV2 はスケジューラを簡素化し、トークンのサンプリングや出力処理での余計なコピーを減らす（ゼロコピー DMA）。FP8 は数値を8ビット浮動小数へ量子化し、メモリ帯域と計算量を削減して最新 GPU の性能を引き出す設計。トレードオフは、量子化による精度低下の可能性と、最新機能ほど特定 GPU（Blackwell/GB200 等）に最適化が寄る点。汎用性と最先端性能のどちらを取るかで構成が変わる。

### 実務での使いどころ・移行の勘所

自前で LLM をホスティングし、トラフィックが多い・原価を下げたいチームに直接効く。既存 vLLM 利用者は、v0.20 系へ更新して MRV2 と FP8 の恩恵を評価する価値がある。移行の勘所は、(1) FP8 量子化を使うなら、対象モデル・タスクで精度回帰がないかをオフライン評価で確認、(2) 最大スループットの数値は特定 GPU 前提のことが多いので、自環境の GPU での実測が必須、(3) 継続的バッチングやストリーミングの設定がレイテンシ SLA と噛み合うかを負荷試験で検証。クラウド GPU コストが支配的なワークロードほど、この最適化の投資対効果は大きい。

## 📖 用語解説（このトピックとのつながり）

- **vLLM**: 高スループットな LLM 推論エンジン（OSS）。本トピックの主役。
- **Model Runner V2 (MRV2)**: スケジューリングとモデル実行を刷新した新実行系。
- **継続的バッチング**: 空いたスロットに逐次リクエストを差し込み GPU を満載に保つ手法。
- **PagedAttention / KV cache**: 注意機構のキャッシュをページ単位で管理しメモリ効率を高める仕組み。
- **FP8 量子化**: 数値を8ビットにして帯域・計算を削減する手法。精度とのトレードオフを持つ。

## 影響範囲 / 推奨アクション

LLM を自前ホスティングする高トラフィックなチームはコスト・応答改善が見込める。v0.20 系更新で MRV2/FP8 を評価する価値あり。FP8 は精度回帰をオフライン評価し、スループット数値は自環境 GPU で実測すること。
