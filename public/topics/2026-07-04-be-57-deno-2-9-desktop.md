# Deno 2.9（2026-06）— `deno desktop` で Web 技術からネイティブアプリを作る

> トピック: 開発トピック｜出典: Deno 公式ブログ｜種別: backend

## 一行サマリー
JavaScript/TypeScript ランタイム Deno の 2.9 が 2026-06-25 に公開され、Web 技術でネイティブなデスクトップアプリを作る `deno desktop`、npm/pnpm/yarn/Bun からの第一級の移行支援、CSS モジュールのインポート、スナップショット/パラメータ化テスト、より小さい `deno compile --bundle` バイナリ、Node.js 26 互換などを追加した。

## 🔰 初学者向け（何が・なぜ重要か）
Deno は、JavaScript や TypeScript を動かすための土台（ランタイム）で、Node.js の後発として「安全・標準準拠・道具込み」を掲げています。TypeScript をそのまま実行でき、リンタ・フォーマッタ・テスト・パッケージ管理まで一体で持っているのが特徴です。

2.9 の目玉は `deno desktop` です。これは「Web の技術（HTML/CSS/JS）で書いた画面を、パソコンにインストールして使える普通のアプリ（ネイティブアプリ）にする」機能です。これまで Electron などの別ツールが担っていた領域を、Deno 単体で扱えるようにする、という方向性です。

もう一つ実務的に嬉しいのが「引っ越し支援」です。npm・pnpm・yarn・Bun といった他ツールで作られたプロジェクトから、Deno へ乗り換える作業を第一級でサポートします。「試したいけど移行が面倒」という壁を下げる改良です。全体として、Deno が「サーバー用の実行環境」から「アプリ開発の総合基盤」へ守備範囲を広げている、と読めます。

## 🧠 専門的解説（技術詳細・設計・使いどころ）
### 技術的詳細（何が変わったか）
Deno 2.9 は 2026-06-25 リリース。主な追加は (1) `deno desktop`：Web 技術でネイティブデスクトップアプリを構築、(2) npm/pnpm/yarn/Bun からの第一級移行サポート、(3) CSS モジュールのインポート、(4) スナップショットテストとパラメータ化テスト、(5) `deno compile --bundle` の生成バイナリ縮小、(6) Node.js 26 互換。Deno は元々 TypeScript ネイティブ実行、パーミッションベースのセキュリティ、Web 標準 API、統合ツールチェーン（`deno fmt` / `deno lint` / `deno test`）を備える。各機能の安定/実験区分は公式ブログ・リリースノートで要確認。

### 仕組み / 背景（なぜこう設計されたか）
Deno の一貫した思想は「Web 標準を土台に、開発に必要な道具を最初から同梱する」ことだ。`deno desktop` は、Chromium/Electron 系が抱える「巨大なバンドル・重い常駐・独自 API」の課題に対し、Deno のランタイムとセキュリティモデルを活かした軽量な代替を狙う設計と読める。移行サポートの強化は、Node/npm 資産という巨大な既存エコシステムとの相互運用が普及の鍵だという現実的判断による。`deno compile` のバイナリ縮小は、単一実行ファイル配布（配布先にランタイムを要求しない）を実用的にするための最適化だ。トレードオフとして、`deno desktop` のような新機能は成熟に時間を要し、Electron ほどのプラグイン/実績エコシステムはまだ薄い。Node 互換も完全ではなく、ネイティブアドオン依存のパッケージでは差が出ることがある。

### 実務での使いどころ・移行の勘所
TypeScript を設定レスで動かしたい小〜中規模サービス、CLI ツール、エッジ関数（Deno Deploy）に好適。`deno desktop` は社内ツールや軽量デスクトップアプリの候補になり得るが、本番採用は成熟度を見極めて。既存 Node プロジェクトの移行では、2.9 の移行支援を使いつつ、(1) ネイティブアドオン依存の有無、(2) `deno compile --bundle` での配布要件、(3) パーミッション設定（`--allow-net` 等）による最小権限運用を確認する。テスト機能の拡充（スナップショット/パラメータ化）で、移行後のテスト資産も Deno 内で完結させやすくなった。

## 📖 用語解説（このトピックとのつながり）
- **ランタイム**: JS/TS を実行する土台。Deno は TypeScript ネイティブ・道具同梱が特徴。
- **`deno desktop`**: Web 技術でネイティブデスクトップアプリを作る 2.9 の新機能。Electron 的領域への進出。
- **`deno compile`**: 単一実行ファイルを生成する機能。2.9 でバイナリが小型化し配布が容易に。
- **パーミッションモデル**: ネットワーク/ファイル等の権限を明示付与する Deno の安全設計。最小権限運用の要。
- **Node 互換**: npm 資産を使うための互換層。2.9 で Node.js 26 に対応、ただし完全互換ではない。

## 影響範囲 / 推奨アクション
TypeScript 中心のバックエンド/ツール開発に関係。設定レスな TS 実行やエッジ関数で有力。`deno desktop` は成熟度を見極めつつ社内ツールから試す。Node からの移行は 2.9 の支援を活用し、ネイティブ依存とパーミッション設定を確認して段階的に。
