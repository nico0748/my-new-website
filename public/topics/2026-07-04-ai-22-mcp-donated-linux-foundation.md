# MCP を Agentic AI Foundation に寄贈 — Linux Foundation 傘下で中立標準へ

> トピック: 開発トピック｜出典: Wikipedia（Model Context Protocol）｜種別: ai

## 一行サマリー
Model Context Protocol（MCP）は 2025年12月（要確認）に Linux Foundation 傘下の Agentic AI Foundation へ寄贈され、単一企業の管理から中立的なオープンガバナンスへ移行した。これにより長期的な標準としての信頼性が大きく高まった。

## 🔰 初学者向け（何が・なぜ重要か）
技術規格が一社の持ち物のままだと、その会社の都合で方針が変わったり、囲い込まれたりする心配がつきまとう。だからこそ「みんなで使う共通ルール」は、中立な団体に預けて運営するのが定石だ。道路交通のルールを特定の自動車メーカーが決めるのではなく、公的機関が管理するのと同じ発想である。

MCP を発案した Anthropic は、この規格を Linux Foundation という業界横断の中立組織に寄贈した。これは「MCP はもう Anthropic だけのものではなく、業界全体の公共財です」という宣言に近い。企業が MCP に投資しても、後で方針が急変して梯子を外される心配が減るため、採用の後押しになる。

## 🧠 専門的解説（技術詳細・設計・使いどころ）
### 技術的詳細（何が変わったか）
技術仕様そのものが変わったわけではなく、統治構造（ガバナンス）が変わった点が本質。Linux Foundation 傘下の Agentic AI Foundation（要確認：名称・設立時期）に移管されたことで、仕様策定は複数ベンダーが参加する委員会形式に近づき、特定企業が一方的に破壊的変更を入れにくくなる。オープンな商標・貢献ライセンスの整備も進む。

### 仕組み / 背景（なぜこう設計されたか）
Kubernetes（CNCF 寄贈）や多くの OSS が辿った道と同じで、「発案企業が中立財団へ寄贈 → 業界標準として定着」という王道パターン。エージェント通信の A2A なども同様の中立化を進めており、AI エージェント基盤全体が「特定ベンダー中立」を競争軸にしつつある。

### 実務での使いどころ・移行の勘所
経営判断として MCP 採用を検討する際、「一社依存リスク」という反対論への強力な反証材料になる。ただしガバナンス移行直後は仕様の細部が固まりきっていない可能性があり、バージョン差異には注意。長期契約や大規模実装では、財団のロードマップと採用バージョンを明示しておくとよい。

## 📖 用語解説（このトピックとのつながり）
- **Linux Foundation**: 多数の重要 OSS を中立運営する非営利団体。MCP の新しい後見役。
- **Agentic AI Foundation**: AI エージェント関連標準を扱う傘下組織（要確認）。MCP の受け皿。
- **オープンガバナンス**: 複数当事者で仕様を運営する統治形態。ロックイン回避が狙い。
- **A2A**: エージェント間通信プロトコル。MCP と並ぶ中立標準化の動き。
- **CNCF**: Kubernetes を擁する Linux Foundation 傘下組織。中立化の先行事例。

## 影響範囲 / 推奨アクション
MCP 採用を検討する企業・アーキテクトに影響する。ベンダーロックイン懸念で採用をためらっていたなら、中立化は採用の後押し材料になる。導入時は財団の公式仕様バージョンを基準にし、独自拡張は明示的に分離しておくこと。
