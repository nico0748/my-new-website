# Linux カーネル「DirtyClone」CVE-2026-43503（cloned skb でファイルバックメモリ汚染→root）

> トピック: 脆弱性ニュース ｜ 出典: The Hacker News / JFrog Security Research / Cybersecurity News ｜ 種別: 局所的権限昇格（ファイルバックページ汚染 / SKB クローンの安全フラグ欠落）

## 一行サマリー

JFrog が公開した Linux LPE「DirtyClone」（CVE-2026-43503、CVSS 8.8）は、SKB（ネットワークバッファ）のクローン処理で安全フラグが欠落するため、読み取り専用のファイルバックページを書き換え可能な送信バッファとして扱わせ、特権バイナリを汚染して root を奪える。pedit COW と同系統で、非特権名前空間が有効な環境が高リスク。

## 🔰 初学者向け（何が・なぜ問題か）

これは同日紹介した「pedit COW」と**よく似た発想**の弱点ですが、突くポイントが違います。

Linux は、ネットワークでデータを送るとき「skb（ソケットバッファ）」という入れ物にデータを詰めます。効率化のため、この入れ物を**複製（クローン）**して使い回すことがあります。本来、ファイル由来の読み取り専用メモリは「これは共有・読み取り専用だから書き換え禁止」という安全ラベルが貼られています。

ところが DirtyClone では、入れ物を複製する処理の途中で**この安全ラベルが剥がれて**しまいます。すると攻撃者は、読み取り専用であるはずのファイル（例: ログイン用プログラム `/bin/su`）のメモリを、書き換え可能な「送信用データ」として扱わせ、中身を上書きできます。

攻撃者は `/bin/su` のメモリをネットワークパケットに紐づけ、自分が制御する暗号トンネル（IPsec）を通します。複合（復号）処理の段階で、そのプログラムのログインチェック部分を攻撃者の好きなバイト列に書き換えてしまうのです。結果、一般ユーザが root を取れます。pedit COW と合わせ「ファイルの読み取り専用コピーを書き換えて特権を奪う」攻撃が立て続けに公表された点が重要です。

## 🧠 専門的解説（技術詳細・メカニズム・検知対応）

### 技術的詳細（種別・前提条件）

- **CVE**: CVE-2026-43503（通称「DirtyClone」。DirtyFrag/Fragnesia と同系の変種）。
- **CVSS**: 8.8（High）。
- **種別**: ファイルバックページキャッシュの不正書き込みによる局所的権限昇格（LPE）。直接原因は SKB クローン時の安全フラグ `SKBFL_SHARED_FRAG` 欠落。
- **影響範囲**: 非特権ユーザ名前空間が有効な現代的ディストリ。Debian（既定で脆弱）、Fedora（既定で脆弱）、Ubuntu（24.04+ は AppArmor 名前空間制限で部分緩和だが対象として掲載）、Kubernetes/マルチテナント/コンテナ環境が最高リスク。
- **前提条件**: ローカルアカウントと、非特権ユーザ名前空間が利用可能であること。
- **悪用状況**: JFrog が2026年6月25日に本変種の動作する PoC ウォークスルーを初公開（実環境での広範な悪用は要確認）。
- **修正**: 変種群は2026年5月21日に mainline へマージ（v7.1-rc5, コミット 9e171fc1d7d7）。ベンダ修正カーネルへ更新。

### メカニズム（攻撃が成立する仕組み）

pedit COW と同じ「ファイルバックページを書き換えて特権バイナリを汚染する」ゴールに、別の経路（SKB クローン）で到達する。概念レベルで分解する。

1. **安全フラグの欠落**: SKB のクローンを行う `__pskb_copy_fclone()` が、クローン時に `SKBFL_SHARED_FRAG` 安全フラグを落としてしまう。このフラグは元々、先行する DirtyFrag 緩和でファイルバックページを保護するために導入されたもの。これが欠けることで「共有・読み取り専用のフラグメント」が、書き換え可能な扱いに退化する。`skb_shift()` も影響を受ける。
2. **読み取り専用ページを書き込み可能扱いに**: 攻撃者は `/usr/bin/su` 等の特権バイナリのメモリページをネットワークパケットの断片（フラグメント）として「配線」し、カーネルにそれをクローンさせる。フラグ欠落により、本来書き換え禁止の page cache が書き込み対象になる。
3. **攻撃者が握るプリミティブ**: 「ファイルバックページの内容を任意に上書き」というプリミティブ。pedit COW と最終的に同じ強さ（特権バイナリ汚染）に至るが、トリガが「パケットのクローン経路」である点が異なる。
4. **被害への橋渡し**: クローンしたパケットを攻撃者制御の IPsec トンネルへ通し、その復号処理の書き込みで、`/bin/su` のログインチェックを攻撃者の選んだバイト列へ差し替える。汚染されたバイナリの実行で root 権限のコードに到達する。

一般原理として、本件は「一度導入した緩和フラグを、別経路（クローン/シフト）で取りこぼすと防御が無効化される」回帰（regression）の典型。ネットワークバッファ管理とファイルバックページ保護の境界整合性が崩れると、読み取り専用前提が破れる。武器化に必要な正確な手順・PoC コードは記載しない。

### 検知・暫定緩和・恒久対策

- **検知の着眼点**: 非特権プロセスによる skb クローンを伴う異常なネットワーク操作、攻撃者制御の IPsec トンネル確立、特権バイナリ実行直後の権限昇格、ELF イメージ整合性の異常（EDR/FIM）。pedit COW と共通の IoC が多い。
- **暫定緩和（パッチ前）**: 非特権ユーザ名前空間を制限（`kernel.unprivileged_userns_clone=0`、Ubuntu の AppArmor 名前空間制限を有効化）。不要な IPsec/xfrm 機能の制限。**業務影響の事前検証は要確認**。
- **恒久対策**: ベンダ修正カーネル（mainline v7.1-rc5 相当のバックポート）へ更新し再起動。
- **運用示唆**: pedit COW（CVE-2026-46331）と DirtyClone は「ファイルバックページ汚染による root 昇格」という同一クラス。両者をまとめて棚卸し・パッチ計画に組み込むのが効率的。コンテナ/K8s 基盤は最優先。

## 📖 用語解説（このニュースとのつながり）

- **SKB（socket buffer / skb）**: カーネルがネットワークデータを格納する入れ物。本件はこの「クローン処理」が攻撃の入口。
- **SKBFL_SHARED_FRAG**: 共有フラグメントを保護する安全フラグ。これが落ちることでファイルバックページが書き換え可能になるのが核心。
- **ファイルバックページ / ページキャッシュ**: ファイル由来の読み取り専用メモリ。`/bin/su` 等の特権バイナリがここで汚染される。
- **非特権ユーザ名前空間**: 一般ユーザが限定特権を得る仕組み。攻撃の前提を満たすため、制限が緩和策になる。
- **回帰（regression）**: 過去に入れた修正・緩和が別経路で無効化される現象。本件は DirtyFrag 緩和フラグの取りこぼしという回帰。

## 影響範囲 / 推奨対応

非特権ユーザ名前空間が有効な Linux（Debian/Fedora 既定、Ubuntu 等）でローカルユーザが root を取得可能。Kubernetes・マルチテナントクラウド・コンテナ基盤が最高リスク。修正カーネルへの更新・再起動を最優先とし、未適用の間は非特権名前空間の制限で攻撃面を縮小する。pedit COW（CVE-2026-46331）と併せて点検すること。
