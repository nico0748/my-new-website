# BeyondTrust Remote Support / PRA に CVSS 9.9 の未認証RCE（CVE-2026-1731）— WebSocket経由でOSコマンド実行・PoC公開後に悪用拡大

> トピック: 脆弱性ニュース ｜ 出典: BeyondTrust BT26-02 / Rapid7 / BleepingComputer / Unit42 / GreyNoise ｜ 種別: OSコマンドインジェクション（CWE-78）に起因する未認証リモートコード実行（RCE）

## 一行サマリー

特権アクセス管理ベンダーBeyondTrustのRemote Support（RS）およびPrivileged Remote Access（PRA）に、未認証で悪用できるCVSS 9.9のOSコマンドインジェクション脆弱性CVE-2026-1731が見つかった。インターネットに面したエンドポイントへ細工したWebSocketメッセージを送るだけで、資格情報も操作も不要でOSコマンドを実行できる。PoC公開後24時間以内に実環境悪用が確認され、VShellやSparkRATの投下が観測されている。約8,500のオンプレ実例が露出対象とされ、即時パッチが必須。

## 🔰 初学者向け（何が・なぜ問題か）

BeyondTrustのRemote SupportやPRAは、**IT管理者やサポート担当が、社内外の端末・サーバーに安全に遠隔接続するための「特権アクセスの玄関」**です。本来は「鍵を厳重に管理して、信頼できる人だけを通す」ための製品——つまりセキュリティを守る側の道具です。それだけに、ここが破られると**最も信頼された経路がそのまま攻撃路に変わる**という、皮肉で深刻な事態になります。

今回の弱点は、この製品が通信に使う **WebSocket**（ブラウザとサーバーがリアルタイムにやり取りする仕組み）にあります。攻撃者が細工したメッセージを送ると、製品がその中身を**点検せずに装置内部の命令として実行**してしまいます。

たとえるなら、警備の厳重な受付に「特別な合言葉」のメッセージを投げ込むと、受付係がその文面に紛れた命令まで実行してしまうイメージです。ログインも、パスワードも、利用者のクリックも要りません。しかも、攻撃の難易度は低く、実証コード（PoC）が公開された直後から本物の攻撃が始まり、遠隔操作型のマルウェア（VShellやSparkRAT）が仕込まれた例も確認されています。守りの要であるこの製品をインターネットに晒している組織は、即座のパッチ適用が欠かせません。

## 🧠 専門的解説（技術詳細・メカニズム・検知対応）

### 技術的詳細（種別・前提条件）

- **CVSS**: **9.9（Critical, CVSS v4）**。未認証・ネットワーク経由・低複雑度・ユーザー操作不要で悪用可能。
- **CWE分類**: CWE-78（OSコマンドインジェクション）。細工したWebSocketメッセージを通じてサイトユーザーの文脈でOSコマンドを実行。
- **前提条件**: インターネットに面したRS/PRAエンドポイントへ到達できること。資格情報・ユーザー操作は不要。
- **影響製品・バージョン**: BeyondTrust **Remote Support 25.3.1以前**、**Privileged Remote Access 24.3.4以前**。
- **対応**: BeyondTrustはアドバイザリBT26-02（2026年2月6日公開）で開示。自動更新有効のインスタンスには自動適用済み。自己ホスト型は手動適用が必要。RS 21.3未満／PRA 22.1未満は新バージョンへの移行が必要。
- **悪用状況**: **PoC公開後24時間以内に実環境悪用を確認**（GreyNoise・watchTowr・Arctic Wolf等が独立に確認）。Unit42はVShell／SparkRAT投下を観測。約11,000インスタンスがインターネット露出、うち約8,500がオンプレで未適用なら脆弱。

### メカニズム（攻撃が成立する仕組み）

以下は概念レベルの説明であり、武器化可能な手順・PoC等は含めない。

1. **信頼境界の所在**: RS/PRAはインターネットに面したエンドポイントで、認証前のWebSocket接続を受理する。ここで「外部から来るメッセージ内容」と「サーバー側で実行されるコマンド」を厳格に分離できるかが信頼境界となる。
2. **未認証経路の存在**: 該当エンドポイントは認証前に到達可能であり、攻撃者は資格情報なしでWebSocketメッセージを送信できる。これにより攻撃の前提（到達性）が容易に満たされる。
3. **入力のコマンド混入（インジェクション）**: 受信したメッセージ内のデータが、サーバー側でOSコマンドを構築する際に**無害化されずに混入**する。攻撃者が与えた文字列がデータではなくコマンドの一部として解釈され、任意コマンド実行のプリミティブを得る。
4. **サイトユーザー文脈での実行と常駐化**: 注入コマンドは製品が動くサイトユーザーの権限で実行される。観測された攻撃では、これを起点に**Webシェルや遠隔操作マルウェア（VShell/SparkRAT）を投下**し、永続的な遠隔操作・横展開の足場を築く。
5. **"守りの製品"であることの増幅**: RS/PRAは特権アクセスを束ねるため、侵害は管理対象端末・資格情報への広範なアクセスへ直結する。信頼の集約点が単一障害点となる。

脆弱性クラスとしては、「認証前に到達可能なエンドポイントでの入力検証欠如によるコマンドインジェクション」であり、リモートアクセス／VPN系製品で頻発する"認証前RCE"の典型に該当する。

### 検知・暫定緩和・恒久対策

- **検知の着眼点**: RS/PRAエンドポイントへの異常なWebSocket接続・メッセージ、製品プロセスからの予期しない子プロセス（シェル・スクリプト）生成、VShell/SparkRATのIoC、サイトユーザー文脈での不審なアウトバウンド通信。
- **暫定緩和（パッチ前）**: 管理／接続エンドポイントの**インターネット露出を最小化**し、アクセス元をallowlist制限。自動更新サービスの有効化状況を確認。露出していた自己ホスト型は侵害調査を優先。
- **恒久対策**: **修正版へのアップデート**（RS 25.3.1超／PRA 24.3.4超。古いバージョンは対応版へ移行）。自動更新未契約の自己ホスト型は手動適用。
- **運用示唆**: セキュリティ製品こそ"認証前RCE"の標的になりやすい。露出面の最小化と迅速なパッチ運用、侵害想定の事後調査を標準化する。

## 📖 用語解説（このニュースとのつながり）

- **Remote Support / PRA**: BeyondTrustの特権リモートアクセス製品。意味＝「特権アクセスの玄関」。本件＝守りの要が攻撃路化。
- **WebSocket**: 双方向リアルタイム通信の仕組み。本件＝細工メッセージの送信経路。
- **OSコマンドインジェクション（CWE-78）**: 入力がOSコマンドとして実行される欠陥。本件＝未認証RCEの本体。
- **PoC（実証コード）**: 脆弱性を再現する検証コード。本件＝公開後24時間で悪用開始。
- **VShell / SparkRAT**: 遠隔操作型マルウェア。本件＝侵害後に投下され常駐化。

## 影響範囲 / 推奨対応

影響を受けるのは、Remote Support 25.3.1以前／PRA 24.3.4以前を運用し、エンドポイントをインターネットに露出している組織（特にオンプレ自己ホスト型）。推奨対応は、(1) 修正版への即時アップデート（自動更新未契約は手動適用）、(2) エンドポイントのインターネット露出最小化とアクセス元制限、(3) PoC公開後の悪用拡大を踏まえた侵害調査（Webシェル/VShell/SparkRATの痕跡）、(4) 製品プロセスからの不審な子プロセス・通信の監視。
