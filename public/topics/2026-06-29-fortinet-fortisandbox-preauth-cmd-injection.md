# FortiSandbox の3つの認証不要欠陥が実環境で悪用（CVE-2026-39813/39808/25089）

> トピック: 脆弱性ニュース ｜ 出典: The Hacker News / BleepingComputer / Qualys / Help Net Security ｜ 種別: 認証回避（パストラバーサル）＋OSコマンドインジェクション

## 一行サマリー
マルウェア検査用アプライアンス FortiSandbox に、認証不要で悪用できる3件の重大欠陥（いずれもCVSS 9.1）が見つかり、攻撃者が実環境で悪用を開始した。パストラバーサルによる認証回避とOSコマンドインジェクションを組み合わせると、セキュリティ機器自体が乗っ取られる恐れがある。

## 🔰 初学者向け（何が・なぜ問題か）
FortiSandbox は、怪しいファイルを「隔離された実験室（サンドボックス）」の中で実際に動かして、マルウェアかどうかを見分ける検査装置です。組織の防御の要となる機器です。

ところが、その検査装置の管理用の「窓口（API/Web UI）」に欠陥がありました。攻撃者は **ログインしていない状態（認証不要）** で、特殊な細工をしたリクエストを送るだけで、本来は鍵がかかっているはずの内部機能に入り込めます（パストラバーサル＝「順路を無視して裏口へ回り込む」イメージ）。さらに別の欠陥を使うと、装置に対して **好きなコマンドを実行させる**ことができます。

たとえるなら、空港の手荷物検査機そのものが乗っ取られ、検査員になりすました侵入者が好き勝手に操作できる状態です。守るための機械が、逆に攻撃の足がかりにされてしまうのが怖いところです。

## 🧠 専門的解説（技術詳細・メカニズム・検知対応）

### 技術的詳細（種別・前提条件）
- **CVE / CVSS**:
  - CVE-2026-39813（CVSS 9.1）: FortiSandbox JRPC API のパストラバーサルによる認証回避。2026年4月に修正済み。
  - CVE-2026-39808（CVSS 9.1）: OSコマンドインジェクション（認証不要、細工HTTPリクエスト）。2026年4月に修正済み。
  - CVE-2026-25089（CVSS 9.1）: FortiSandbox / Cloud / PaaS WEB UI のOSコマンドインジェクション（認証不要）。直近で修正。
- **種別（CWE）**: パストラバーサル（CWE-22）系の認証回避＋OSコマンドインジェクション（CWE-78）。
- **前提条件**: いずれも **認証不要（unauthenticated）**。管理インターフェースへネットワーク到達できることが条件。
- **悪用状況**: Fortinet が3件の悪用を観測（24時間以内のスパイク）。ただし CVE-2026-25089 の悪用コードはAIで生成された痕跡があり不完全との報告。動作する公開エクスプロイトは未確認（要確認）。

### メカニズム（攻撃が成立する仕組み）
認証回避とコマンド実行という2種のプリミティブが連鎖し、事前認証RCEに近い状態を作る典型的なチェーンである（概念レベルで解説）。

1. **信頼境界**: 管理API/Web UI は「認証を通過した者だけが内部関数に到達する」前提。第一の境界は認証層。
2. **境界の突破（CVE-2026-39813）**: JRPC API のパス解決にパストラバーサルの不備があり、`../` 的な経路操作で認証チェックを迂回し、本来保護された関数へ直接到達する。ここで「認証回避」というプリミティブを獲得。
3. **コマンド実行（CVE-2026-39808 / 25089）**: 到達した関数が、利用者供給入力をOSコマンド文字列に組み込む際にサニタイズしない。攻撃者は「任意コマンド実行」というプリミティブを得る。
4. **被害の成立**: 認証回避×コマンド実行が連結すると、未認証の攻撃者がアプライアンスOS上で任意コマンドを走らせられる。セキュリティ検査機器そのものの掌握＝防御網の内側への足場確保につながる。

一般原理として、パストラバーサルは「正規ルートの外を指す入力を、正規リソースとして解決してしまう」アクセス制御欠落の一種。認証を「経路で迂回できる」設計は、認証ロジックそのものの欠陥となる。

### 検知・暫定緩和・恒久対策
- **検知の着眼点**: 管理API/Web UIへの `../` や異常なパスを含むHTTPリクエスト、認証ログのない内部関数呼び出し、アプライアンス上の想定外プロセス・アウトバウンド通信。
- **暫定緩和**: 管理インターフェースをインターネットから遮断し管理セグメントに限定。アクセス元IPを許可リスト化。
- **恒久対策**: Fortinet の修正版へ更新（4月修正分＋直近修正分を両方適用）。侵害有無の調査（コマンド実行痕跡・設定改ざん）を実施。

## 📖 用語解説（このニュースとのつながり）
- **FortiSandbox**: 不審ファイルを隔離実行して判定するマルウェア検査機。今回の標的。
- **パストラバーサル（CWE-22）**: 入力でファイル/関数の場所を不正に指し、保護領域へ回り込む欠陥。本件の認証回避の核。
- **OSコマンドインジェクション（CWE-78）**: 入力がOSコマンドに混入し任意実行される欠陥。本件の最終的なRCE原因。
- **認証不要（pre-auth）**: ログイン前に悪用できること。攻撃ハードルが極端に低く危険度が高い。
- **JRPC API**: 製品内部の遠隔手続き呼び出しインターフェース。今回の侵入口。

## 影響範囲 / 推奨対応
FortiSandbox／FortiSandbox Cloud／PaaS を利用する組織が対象。検査機器が侵害されると、検知回避や横展開の起点になりうる。管理面のインターネット露出を直ちに排除し、4月以降の修正をすべて適用、悪用痕跡（コマンド実行ログ・不審プロセス）を確認すること。
