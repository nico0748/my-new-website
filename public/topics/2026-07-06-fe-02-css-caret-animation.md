# CSS caret-animation プロパティ（Chrome 140）— テキストカーソルの点滅を CSS で制御する

> トピック: CSS caret-animation｜出典: Chrome リリースノート / web.dev｜種別: 新CSSプロパティ

## 一行サマリー

Chrome 140 で追加された CSS の `caret-animation` プロパティは、入力欄で点滅するテキストカーソル（キャレット）の点滅アニメーションを開発者が制御できるようにするもので、`auto`（従来どおりブラウザ既定の点滅）と `manual`（点滅を止めて自前アニメーションを適用可能にする）の2値を取る。

## 🔰 初学者向け（何が・なぜ重要か）

文章を入力する欄をクリックすると、「ここに文字が入りますよ」という縦棒がチカチカ点滅します。これがキャレット（テキストカーソル）です。これまでこの点滅はブラウザが勝手に決めていて、色（`caret-color`）は変えられても、点滅の仕方には手を出せませんでした。

そのため凝ったデザインをしたい人は、本物のキャレットを隠して「偽物のカーソル」を自作していました。しかしこれは面倒で、日本語入力（変換中の候補表示）などで挙動がずれる原因にもなります。

新しい `caret-animation: manual` は「点滅はこちらで面倒みるので、ブラウザは点滅させないで」と伝える指示です。これで本物のキャレットをそのまま使いつつ、色をなめらかに変化させる・虹色にする、といった演出を CSS のアニメーションだけで実現できます。ターミナル風 UI やエディタ系サイトで「らしさ」を出すのに向いています。

## 🧠 専門的解説（技術詳細・設計・使いどころ）

### 技術的詳細（何が変わったか）

- **新プロパティ `caret-animation`**: 値は `auto`（既定・ブラウザ制御の点滅）と `manual`（開発者制御）。`manual` にすると既定の点滅が停止し、`caret-color` に対する CSS アニメーション/トランジションが有効になる。
- **`caret-color` との協調**: これまで `caret-color` にアニメーションを当てても既定の点滅と競合して意図どおり動かなかった。`manual` によりキャレット色のアニメーションが素直に反映される。
- **搭載バージョン**: Chrome 140 で出荷。クロスブラウザの Baseline 化状況（Safari / Firefox の対応時期）は要確認。
- 本文の仕様値・搭載時期は Chrome リリースノート/web.dev の記述に基づく。

### 仕組み / 背景（なぜこう設計されたか）

キャレットはブラウザが描画する「合成された UI 要素」で、DOM 要素ではないため CSS から直接掴めない。従来アニメーション可能だったのは `caret-color`（色）だけで、点滅そのものはユーザーエージェント（ブラウザ）のタイマー制御下にあった。`caret-color` にキーフレームを当てても、ブラウザ側の点滅タイマーと二重制御になり結果が破綻していた。

`caret-animation` の役割は、この「点滅の主導権」をブラウザから開発者に明示的に移譲するスイッチだ。`manual` を宣言した時点でブラウザは点滅タイマーを手放し、キャレットの見た目は完全に CSS のアニメーション計算に委ねられる。トレードオフとして、`manual` にすると点滅は自動では起きなくなるため、開発者が `@keyframes` で明示的に演出を書かない限りキャレットは点滅しない（＝アクセシビリティ上の「入力位置の視認性」を自分で担保する必要がある）。過度に凝った演出は、点滅がフォーカス位置を示すという本来の役割を損なう恐れがある点に注意。

### 実務での使いどころ・移行の勘所

ターミナル／コードエディタ風の演出、ブランドカラーに沿ったキャレット表現などが主用途。実装は「本物のキャレットを隠して偽カーソルを作る」ハックの置き換えになり、日本語 IME 変換中の表示ずれや、選択・コピー時の不整合といった副作用を避けられる。ただし現時点では Chrome 系先行のため、未対応ブラウザでは `auto` 相当にフォールバックする前提で、装飾はあくまで「あれば嬉しい」程度に留めるのが安全。フォーカス位置の可読性を損なわない範囲で使う。

## 📖 用語解説（このトピックとのつながり）

- **キャレット（caret）**: テキスト入力位置を示す点滅する縦棒。本トピックはこの点滅を制御する。
- **`caret-color`**: キャレットの色を指定する既存プロパティ。`caret-animation: manual` と組んで初めてアニメーションが素直に効く。
- **ユーザーエージェント（UA）**: ブラウザ本体。従来キャレットの点滅は UA が制御していた。
- **Baseline**: 主要ブラウザで安全に使える状態を示す指標。本機能の全ブラウザ対応は要確認。
- **IME**: 日本語などの入力変換系。偽キャレット実装が壊れやすかった箇所で、本機能はそれを回避する。

## 影響範囲 / 推奨アクション

- 対象: 入力欄を装飾したいフロントエンド、特にターミナル/エディタ系 UI。
- 推奨: 偽キャレット実装を `caret-animation: manual` + `caret-color` アニメへ置換検討。
- 注意: 未対応ブラウザ向けフォールバックを用意。演出でフォーカス視認性を損なわない。
