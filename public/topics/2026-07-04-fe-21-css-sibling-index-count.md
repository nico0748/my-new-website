# CSS sibling-index() / sibling-count() — 兄弟要素の位置をCSSだけで数える

> トピック: 開発トピック｜出典: Southwell Media｜種別: frontend

## 一行サマリー
`sibling-index()` と `sibling-count()` は、要素が同じ親の中で「何番目か」「兄弟が全部で何個か」を CSS の数式として取得できる新関数で、2026 年後半に Baseline 入りが見込まれている（時期は要確認）。これまで JavaScript やインライン `style` に頼っていた「連番に応じた遅延アニメーション」などを、CSS 単独で書けるようになる。

## 🔰 初学者向け（何が・なぜ重要か）
リストや行が縦にたくさん並んでいるとき、「上から順に少しずつ遅れてフェードインする」といった演出をしたいことがあります。これまでは各要素に「あなたは3番目だよ」という情報を JavaScript で埋め込むか、`--i: 3` のような変数を手で書いて渡す必要がありました。要素が増減するたびに書き換えが必要で、地味に面倒だったのです。

`sibling-index()` は、この「あなたは何番目？」という問いに CSS 自身が答えてくれる関数です。たとえば `animation-delay: calc(sibling-index() * 50ms)` と書けば、1番目は 0ms、2番目は 50ms、3番目は 100ms……と自動でずれてくれます。要素をいくつ足しても引いても、番号を手で振り直す必要がありません。

`sibling-count()` は「兄弟が全部で何個あるか」を返します。全体の個数が分かると、「全部で N 個のうち自分は何割の位置か」といった相対計算ができ、グラデーションや配色を要素数に応じて自動調整するような表現が CSS だけで完結します。要するに「並び順を数える仕事を、人間からブラウザへ渡す」機能です。

## 🧠 専門的解説（技術詳細・設計・使いどころ）
### 技術的詳細（何が変わったか）
`sibling-index()` は現在の要素の 1 始まりのインデックスを整数として返し、`sibling-count()` は同じ親を持つ兄弟要素の総数を返す。どちらも `calc()` の中で数値として使えるため、`animation-delay`・`transition-delay`・`transform` の量・`--custom-prop` の算出など、数値プロパティ全般に流し込める。Baseline 化の時期は「2026 後半予定」とされるが、正確な対応ブラウザ・バージョンは要確認。

### 仕組み / 背景（なぜこう設計されたか）
従来は `:nth-child(n)` セレクタで「n 番目だけに別スタイル」を当てることはできたが、これは「番号を条件に分岐する」仕組みで、番号そのものを数値として計算に使うことはできなかった。連番アニメーションのために `--i` を全要素へ手書きしたり、JS で `element.style.setProperty('--i', i)` を回したりする定番ハックが生まれていた。`sibling-index()` はこの値を CSS のファーストクラスの数値として公開することで、マークアップと JS からその責務を切り離す。

### 実務での使いどころ・移行の勘所
リスト・グリッド・ナビゲーションのスタッガー（時間差）アニメーション、要素数に応じた色相の割り当て、階段状レイアウトなどが主戦場。既存の `--i` 手動渡し実装は、対応ブラウザが揃えば `calc(sibling-index() * ...)` へ置換でき、テンプレートから番号付けのロジックを削除できる。ただし Baseline 到達前は旧実装をフォールバックとして残すか、`@supports` で分岐するのが安全。SSR/CSR いずれでもマークアップに番号を焼き込む必要がなくなる点が保守上の利点。

## 📖 用語解説（このトピックとのつながり）
- **Baseline**: 主要ブラウザで安全に使える状態を示す指標。この関数は 2026 後半に到達見込み（要確認）で、それまでは実験的機能。
- **`:nth-child()`**: 「n 番目」を条件に分岐する既存セレクタ。番号を数値として使えない点が `sibling-index()` との差。
- **`calc()`**: CSS 内で数式を書く関数。`sibling-index()` の戻り値はここに埋め込んで初めて意味を持つ。
- **スタッガーアニメーション**: 複数要素を少しずつ時間差で動かす演出。本関数の代表的な用途。

## 影響範囲 / 推奨アクション
フロントエンド全般、特にアニメーション演出を CSS 中心で組むチームに影響。対応が広がるまでは `@supports (sibling-index: 1)` 等での機能検出とフォールバック併用を推奨。新規実装では JS による番号注入を減らす前提で設計を検討するとよい。
