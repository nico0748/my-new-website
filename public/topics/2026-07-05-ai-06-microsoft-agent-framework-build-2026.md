# Microsoft Agent Framework が BUILD 2026 で拡張 — Agent Harness・Hosted Agents・CodeAct

> トピック: Microsoft Agent Framework の BUILD 2026 発表｜出典: Microsoft 公式 DevBlogs｜種別: フレームワーク機能追加

## 一行サマリー

Microsoft が BUILD 2026 で Agent Framework の新機能を発表。エージェント実行を標準化する Agent Harness、運用をマネージド化する Hosted Agents、コードを行動として生成・実行させる CodeAct などを追加し、Semantic Kernel と AutoGen を統合した同フレームワークを本番運用向けにさらに前進させた。

## 🔰 初学者向け（何が・なぜ重要か）

AI エージェント（自分で判断して道具を使い、多段の作業をこなす AI）を業務で使うには、「動かす仕組み」「運用する場所」「安全に実行する方法」をきちんと整える必要があります。Microsoft Agent Framework は、同社がこれらをまとめて提供する開発基盤で、以前バラバラだった Semantic Kernel と AutoGen という2つの流れを統合したものです。

BUILD 2026（Microsoft の開発者向け大型イベント）では、この基盤に3つの機能が加わりました。エージェントの実行手順を型どおりにそろえる「Agent Harness」、自分でサーバを用意しなくても動かせる「Hosted Agents（ホスト型）」、そして AI に「コードを書いてそれを実行する」形で作業させる「CodeAct」です。

たとえるなら、これまで「エンジンだけ」売っていたのが、車体・整備工場・安全装置まで一式そろえたパッケージに近づいた、というイメージ。エージェントを「作れる」段階から「業務で安定して回せる」段階へ引き上げる動きです。

## 🧠 専門的解説（技術詳細・設計・使いどころ）

### 技術的詳細（何が変わったか）

BUILD 2026 で発表された Microsoft Agent Framework の主な追加点:

- **Agent Harness**: エージェントの実行を標準化する仕組み（ループ、ツール呼び出し、エラー処理などの定型化）。
- **Hosted Agents**: エージェントをマネージド環境で実行できるホスティング。自前インフラ構築を不要にする方向。
- **CodeAct**: モデルにコードを生成させ、それを行動（アクション）として実行させるパラダイム。

Microsoft Agent Framework は Semantic Kernel（エンタープライズ向けオーケストレーション）と AutoGen（マルチエージェント研究基盤）を統合した後継で、今回の発表はそれを本番運用向けに強化するもの。各機能の GA/プレビュー状況や対応言語は公式リリースで要確認。

### 仕組み / 背景（なぜこう設計されたか）

エージェント開発は「同じような足回り（実行ループ、ツール接続、リトライ、監視、ホスティング）を毎回作り直す」という重複に悩まされてきた。Agent Harness は、この足回りを標準化して「エージェントの中身（何をするか）」に集中できるようにする発想だ。Hosted Agents は、スケジュール実行や長時間ジョブを「プラットフォーム機能」として提供するもので、2026年に各社（Anthropic の Managed Agents、OpenAI の Codex スケジュール実行等）が競って出している「エージェント運用のマネージド化」潮流に沿う。

CodeAct が興味深い。従来のツール使用エージェントは「あらかじめ定義された関数（ツール）を JSON で呼ぶ」形が主流だが、CodeAct は「モデルにコードそのものを書かせ、それを実行環境で走らせる」ことで行動を表現する。コードは、条件分岐・ループ・変数・既存ライブラリ呼び出しを自然に含められるため、固定的な関数呼び出しより表現力が高く、複雑な多段操作を少ない往復で実現できる。トレードオフは明白で、生成コードを実行することは強力な反面、サンドボックス化・権限制限・監査を厳格にしないと重大なセキュリティリスクになる。だからこそ Harness による標準化と Hosted による統制された実行環境がセットで意味を持つ。全体として「表現力の高い実行（CodeAct）＋標準化された足回り（Harness）＋統制された運用（Hosted）」という三点で、エージェントの本番化を狙う設計だ。

### 実務での使いどころ・移行の勘所

Microsoft/Azure 中心のエンタープライズで、Semantic Kernel や AutoGen をすでに使っているチームには自然な進化先。定型の足回りを Harness に任せ、運用は Hosted で軽くできる。CodeAct は、固定ツールでは表しきれない複雑なデータ操作・多段処理に有効だが、導入時はサンドボックス（隔離実行）、最小権限、実行ログの監査を必須要件として設計すること。移行の勘所は、まず既存エージェントの「足回り」を Harness の標準に載せ替え、CodeAct は影響範囲の限定された内部業務から試すこと。マネージド Hosted は運用を楽にする一方でロックインを生むため、可搬性（自前実行への退避経路）も意識したい。

## 📖 用語解説（このトピックとのつながり）

- **Microsoft Agent Framework**: Semantic Kernel と AutoGen を統合したエージェント開発基盤。本トピックの土台。
- **Agent Harness**: 実行ループ・ツール呼び出し等の足回りを標準化する仕組み。
- **Hosted Agents**: マネージド環境でエージェントを実行するホスティング。運用のマネージド化潮流の一部。
- **CodeAct**: モデルにコードを生成・実行させて行動を表現するパラダイム。表現力が高い反面サンドボックスが必須。
- **サンドボックス**: 生成コードを隔離実行し被害を封じ込める仕組み。CodeAct の安全性の要。

## 影響範囲 / 推奨アクション

Microsoft/Azure 中心でエージェントを運用するチームに有効。CodeAct 採用時はサンドボックス・最小権限・監査ログを必須設計とする。既存エージェントの足回りを Harness に載せ替え、CodeAct は限定的な内部業務から段階導入するのが安全。
