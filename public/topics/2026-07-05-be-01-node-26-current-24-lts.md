# Node.js 26 が Current・24 が Active LTS — 組み込み TypeScript と UUIDv7 が標準に

> トピック: Node.js リリースライン更新（26 = Current、24 = Active LTS）｜出典: Node.js 公式（nodejs.org）｜種別: ランタイム／LTS 更新

## 一行サマリー

2026年半ば時点で Node.js 26 が Current ライン（5月出荷、10月に LTS 予定）、Node.js 24 が Active LTS。24 系は `crypto.randomUUIDv7()`、`IncomingMessage` の `req.signal`、組み込み TypeScript 型ストリップの安定化などを備え、奇数版の 25 は6月1日に EOL を迎えた。

## 🔰 初学者向け（何が・なぜ重要か）

Node.js は、ブラウザの外で JavaScript を動かすための土台（ランタイム）で、世界中のサーバやツールで使われています。Node.js には「安定版（LTS）」と「先行版（Current）」があり、本番運用では長期サポートされる LTS を選ぶのが定石です。

いま押さえるべきは3点です。ひとつめ、最新の先行版が26、長期サポート版が24になったこと。ふたつめ、これまで TypeScript を動かすには変換ツールが必要でしたが、Node.js が「TypeScript の型注釈をそのまま剥がして実行する」機能を標準で持つようになり、設定なしで `.ts` を動かしやすくなったこと。みっつめ、時系列で並び替えやすい新しい ID 形式「UUIDv7」を生成する関数が標準で入ったことです。

たとえるなら、以前は別売りだった便利アタッチメントが、本体に最初から付いてくるようになったイメージ。「追加ツールを入れなくても、そこそこのことが素の Node.js でできる」方向に進んでいます。

## 🧠 専門的解説（技術詳細・設計・使いどころ）

### 技術的詳細（何が変わったか）

- **リリースライン**: Node.js 26 は2026年5月5日に初出荷され Current。10月に LTS 昇格予定。Node.js 24 は Active LTS で EOL は2028年4月30日。奇数版の 25 は2026年6月1日に EOL（奇数版は LTS に入らず約半年で EOL）。
- **24 系の主な機能**: `crypto.randomUUIDv7()`、`IncomingMessage` への `req.signal`、テストランナーの実行順ランダム化、npm 11 同梱、新しめの V8、**組み込み TypeScript 型ストリップの安定化**。

具体的なバージョン番号（24.16 LTS、26.4 等）はリリースごとに進む。正確な最新パッチは公式リリース表で要確認。

### 仕組み / 背景（なぜこう設計されたか）

Node.js の「偶数=LTS / 奇数=非 LTS」体制は、新機能の実験（奇数版）と本番安定（偶数版 LTS）を分離するための設計だ。奇数版を本番に使うと約半年で EOL になり無保証になるため、実運用は必ず偶数 LTS を選ぶ、という運用規律を前提にしている。

技術面で象徴的なのが「TypeScript 型ストリップの標準化」だ。これは TypeScript を**型検査せずに**、型注釈だけを構文的に取り除いて実行する方式（type stripping）。従来は ts-node や tsx、事前ビルドが必要だったが、ランタイムが直接 `.ts` を扱えることで開発時の摩擦が減る。トレードオフは、これが「型チェックではない」点。型の正しさは別途 `tsc --noEmit` 等で担保する必要があり、`enum` など一部のランタイム意味を持つ構文には制約が残る。`randomUUIDv7()` は、ランダムな UUIDv4 と違い先頭にミリ秒精度のタイムスタンプを含むため、データベースのインデックス局所性が良く、挿入性能・時系列ソートに有利という背景から標準化された。

### 実務での使いどころ・移行の勘所

新規プロジェクトは 24 LTS を基準にし、`.ts` を素で動かせる恩恵を CI/開発時に活用できる（本番ビルドで型チェックは維持）。25 系を使っているなら EOL 済みのため 24 LTS か 26 への移行が必須。UUIDv7 は主キー設計で UUIDv4 の代替として有効で、B-Tree インデックスの断片化を抑えられる。移行の勘所は、型ストリップに頼る場合でも型検査パイプラインを別に残すこと、そして LTS 昇格スケジュール（26 は10月 LTS 予定）に合わせて本番の採用時期を計画すること。

## 📖 用語解説（このトピックとのつながり）

- **LTS / Current**: 長期サポート版と先行版。本番は LTS（偶数）を選ぶ。本トピックの軸。
- **型ストリップ（type stripping）**: 型検査せず型注釈だけ除去して実行する方式。24 系で標準安定化。
- **UUIDv7**: 先頭にタイムスタンプを含む時系列ソート可能な UUID。`crypto.randomUUIDv7()` で生成。
- **EOL**: サポート終了。EOL 版はセキュリティ修正が出ない。

## 影響範囲 / 推奨アクション

25 系利用者は EOL 済みのため 24 LTS か 26 へ即移行を推奨。新規は 24 LTS 基準。UUIDv7 は主キー設計の改善に有効。型ストリップ活用時も型検査を別工程で維持すること。
