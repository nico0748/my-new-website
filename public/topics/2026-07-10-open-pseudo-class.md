# :open 擬似クラスが Baseline 入り — details/dialog/select の「開いた状態」を CSS だけで装飾

> トピック: `:open` 擬似クラス（Baseline Newly available・Safari 26.5 で足並み）｜出典: web.dev Baseline digest / MDN｜種別: CSS 機能の相互運用達成

## 一行サマリー

要素が「開いている」状態にマッチする CSS 擬似クラス `:open` が、Safari 26.5 の対応で主要ブラウザに足並みが揃い Baseline Newly available に到達した。`<details>` `<dialog>` の開閉、`<select>` `<input>` のピッカー展開などを、JS でクラスを付け替えずに CSS だけでスタイル分岐できる。

## 🔰 初学者向け（何が・なぜ重要か）

Web には「開いたり閉じたりする部品」がいくつもある。折りたたみ（`<details>`）、モーダル（`<dialog>`）、ドロップダウン（`<select>`）など。これらが「今開いているか」に応じて見た目を変えたい——たとえば開いているときだけ矢印を回す、背景を強調する——という要望はとても多い。

従来はこの判定を JavaScript でやっていた。「開いた」というイベントを監視して、要素にクラスを付けたり外したりする定型コードを書く必要があった。`:open` 擬似クラスは、この状態を CSS が直接知れるようにする。`details:open { ... }` と書けば、開いている間だけそのスタイルが当たる。状態管理の一部を CSS に任せられるので、JS が減り、コードが読みやすく壊れにくくなる。

## 🧠 専門的解説（技術詳細・設計・使いどころ）

### 技術的詳細（何が変わったか）

`:open` は「開いた状態を持つ要素が開いているとき」にマッチする擬似クラス。対象は主に `<details>` `<dialog>`（表示中）、および `<select>` や一部 `<input>`（ピッカー/ドロップダウンが展開中）。web.dev の 2026年5月 Baseline digest によれば、Safari 26.5 で `:open` が Baseline Newly available になり、Chromium 系・Firefox と合わせて相互運用が成立した。対になる `:closed` を含む状態擬似クラス群として整理されている（各要素での正確な適用範囲は MDN で要確認）。

### 仕組み / 背景（なぜこう設計されたか）

これまで「開閉状態のスタイリング」は要素ごとにバラバラだった。`<details>` は `[open]` 属性セレクタで拾えたが、`<dialog>` はトップレイヤ表示、`<select>` のネイティブピッカー展開は CSS から観測できず、状態表現の手段が統一されていなかった。`:open` は「開いている、という UI 状態」を横断的に一つの擬似クラスへ抽象化する設計で、宣言的 CSS の表現力を状態にまで広げる流れ（`:has()`、`@starting-style` などと同じ「JS がやっていたことを CSS に戻す」潮流）の一部。トレードオフは、`[open]` 属性セレクタなど既存手法との重複が生まれ「どちらで書くべきか」の指針統一が要ること、ネイティブ要素以外のカスタム開閉 UI には適用されない（あくまでブラウザが状態を持つ要素向け）こと。

### 実務での使いどころ・移行の勘所

アコーディオン、FAQ、ディスクロージャ、モーダル、ネイティブセレクトの装飾で威力を発揮する。`<details>` の `[open]` ベースの既存 CSS は `:open` に統一でき、`<dialog>`/`<select>` にも同じ書き味を広げられる。移行の勘所は、対応が揃ったとはいえ Newly available 段階なので、古い環境向けに `[open]` 属性セレクタ等のフォールバックを残すこと。カスタム実装（div ベースの独自アコーディオン）には効かないため、可能ならネイティブ要素へ寄せる設計判断とセットで検討すると効果が最大化する。

## 📖 用語解説（このトピックとのつながり）

- **擬似クラス**: 要素の状態（hover、focus、open など）に応じてスタイルを当てるセレクタ。`:open` は「開いた状態」用。
- **`<details>` / `<dialog>`**: それぞれ折りたたみ・モーダルのネイティブ要素。`:open` の主対象。
- **トップレイヤ**: `<dialog>` などが最前面に表示される描画レイヤ。従来 CSS から状態を拾いづらかった。
- **Baseline Newly available**: 主要ブラウザ最新版で使える段階。旧版考慮のフォールバックはまだ推奨。

## 影響範囲 / 推奨アクション

- 開閉 UI の状態スタイリングを CSS に一本化でき、状態管理 JS を削減できる。
- 既存の `[open]` ベース CSS を `:open` へ統一しつつ、旧環境向けフォールバックを併記する。
- カスタム開閉部品はネイティブ要素への置換を検討すると `:open` の恩恵を受けやすい。
