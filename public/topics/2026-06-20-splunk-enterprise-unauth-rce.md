# Splunk Enterprise CVE-2026-20253 — 認証なしでファイル操作からRCE、実環境で悪用中

> トピック: 脆弱性ニュース｜出典: Splunk / Orca Security / NetSPI / Help Net Security｜種別: 認証欠落→任意ファイル操作→未認証RCE

## 一行サマリー

ログ分析基盤 Splunk Enterprise に同梱される PostgreSQL サイドカーのエンドポイントが認証を一切持たず（CVSS 9.8）、ネットワークから到達できる攻撃者が認証なしで任意ファイルの作成・切り詰めを行い、最終的に遠隔コード実行（RCE）へ連鎖できる。CISA が KEV に追加し実環境での悪用を確認済みのため、10.0.x / 10.2.x 系は即時更新と侵害調査が必要。

## 🔰 初学者向け（何が・なぜ問題か）

Splunk は、サーバやアプリが吐き出す大量のログ（動作記録）を集めて検索・可視化する「企業の監視カメラ室」のようなソフトです。セキュリティ監視の中枢に置かれることが多く、社内のあらゆるシステムの情報が集まる重要な拠点です。

この Splunk の内部には、データを管理するために PostgreSQL という「データベースの補助エンジン」が一緒に動いています。本来、外部から触れる窓口にはすべて「あなたは誰ですか？」という受付（認証）があるべきです。ところが、この補助エンジンの一部の窓口には受付がまったく無く、誰でもノックすれば中の作業を頼めてしまう状態でした。

例えるなら、厳重に施錠された監視カメラ室の正面玄関とは別に、裏口のドアが鍵もかからず開けっ放しになっていて、そこから入れば「サーバの中に好きなファイルを置く」「ファイルの中身を空にする」といった操作を、名乗らずに実行できてしまうイメージです。攻撃者はこの「好きな場所にファイルを書ける力」を足がかりに、自分のプログラムをサーバに実行させ、Splunk が動くマシンを乗っ取れます。

監視の中枢が乗っ取られると、攻撃の痕跡を消されたり、集約された機密情報を盗まれたりするため、被害は Splunk 一台にとどまりません。すでに実際の攻撃で使われているため、急いで更新する必要があります。

## 🧠 専門的解説（技術詳細・メカニズム・検知対応）

### 技術的詳細（種別・前提条件）

- **CVE**: CVE-2026-20253 / **CVSS**: 9.8（Critical）
- **CWE**: CWE-306（重要な機能に対する認証の欠如）を起点に、CWE-73（外部から制御可能なファイルパス）／任意ファイル書き込み→RCE へ連鎖
- **影響コンポーネント**: Splunk Enterprise に同梱される PostgreSQL サイドカーサービスのエンドポイント
- **根本原因**: 当該エンドポイントが認証制御を完全に欠いており、ネットワーク到達可能な攻撃者が未認証でファイル操作（作成・切り詰め）を実行できる。研究者は PostgreSQL の `lo_export` 機能を悪用してスクリプトを書き込み・実行する形で RCE へ連鎖できることを実証。
- **影響バージョン**: Splunk Enterprise 10.0.x および 10.2.x 系（公開 2026-06-10）／**修正版**: 各系の修正リリースへ更新（ベンダー告知参照）
- **悪用状況**: 2026-06-18 に Splunk が限定的な実環境悪用を認め、CISA が KEV に追加。未認証・ネットワーク到達のみで成立するため危険度は最大級。
- **前提条件**: 該当バージョンの Splunk Enterprise が稼働し、PostgreSQL サイドカーのエンドポイントへネットワーク到達可能であること。認証情報は不要。

### メカニズム（攻撃が成立する仕組み）

※以下は脆弱性クラスの一般原理に基づく概念説明。武器化手順・PoC・正確なパス/オフセットは記載しない。

1. **認証境界の欠落（信頼境界の誤り）**: 本来「内部専用」を意図したサイドカーのエンドポイントが、ネットワークから到達可能な位置に認証なしで露出していた。攻撃者は受付（認証）を通らずに特権的な機能を直接呼び出せる。
2. **任意ファイル操作プリミティブの獲得**: 認証なしでファイルの作成・切り詰めが可能になることで、攻撃者は「場所を選んでファイルを書ける／既存ファイルを空にできる」という強力なプリミティブを手にする。これは単体でも改ざん・DoS に使えるが、最大の脅威は実行への橋渡しにある。
3. **書き込みから実行への橋渡し**: データベース機能（`lo_export` 相当の大規模オブジェクトのエクスポート）を悪用し、攻撃者が制御するスクリプトをサーバの実行され得る場所へ書き込む。続いてそれを起動させることで、Splunk が動作するホスト上で任意コードが実行され RCE が成立する。
   - 脆弱性クラスの一般原理: 「認証欠落」+「任意ファイル書き込み」の組み合わせは、自動実行されるパス（起動スクリプト・cron・Webルート配下）や DB のファイル出力機能に到達できると、ほぼ確実に RCE へ昇格する。多層防御の最初の壁（認証）が無いことが致命的。
4. **追加の副次被害**: 同根の不備から、保存型XSSやSSRFも報告されており、Splunk アプリ環境の完全な掌握につながる。

### 検知・暫定緩和・恒久対策

- **検知の着眼点**: PostgreSQL サイドカーのポート/エンドポイントへの外部からの予期せぬアクセス、Splunk ホスト上での見覚えのないファイル生成・タイムスタンプ更新、`splunkd`/PostgreSQL プロセスからの不審な子プロセス生成、エクスポート機能の異常な呼び出し、Web 層の保存型XSS/SSRF の兆候。
- **暫定緩和**: 即時更新できない場合は、サイドカーのエンドポイント/ポートをネットワークレベルで内部限定に制限（外部・横方向到達の遮断）、Splunk 管理面の公開停止、セグメンテーションと IP 制限の徹底。
- **恒久対策**: 10.0.x / 10.2.x の修正リリースへ更新。公開実績のあるインスタンスは「パッチ＋侵害調査」をセットで実施し、追加ファイル・新規アカウント・改ざんされた設定・痕跡消去の有無を点検する。

## 📖 用語解説（このニュースとのつながり）

- **Splunk Enterprise**: ログを集約・検索・可視化する監視基盤。セキュリティ監視の中枢に置かれるため、侵害の影響が全社に波及しやすい。
- **サイドカー（sidecar）**: 主アプリに付随して一緒に動く補助サービス。本件では PostgreSQL がこれに当たり、その露出した窓口が穴になった。
- **CWE-306（認証の欠如）**: 重要機能に認証をかけ忘れる欠陥。本件で「誰でも呼べる」状態を生んだ根本原因。
- **任意ファイル書き込み/切り詰めプリミティブ**: 攻撃者が場所を選んでファイルを書ける/空にできる能力。実行への橋渡しに使われる。
- **lo_export**: PostgreSQL でラージオブジェクトをファイルへ書き出す機能。本件では書き込みプリミティブを RCE へ昇格させる経路として悪用され得る。
- **KEV（Known Exploited Vulnerabilities）**: CISA が「実際に悪用が確認された」脆弱性をまとめた目録。掲載は最優先対応の合図。

## 影響範囲 / 推奨対応

影響は、Splunk Enterprise 10.0.x / 10.2.x が稼働し、当該サイドカーのエンドポイントへネットワーク到達可能な環境全般。未認証・ネットワーク到達のみで RCE に至るため、インターネット公開はもちろん、社内ネットからの到達でもリスクが高い。推奨対応は、ただちに修正リリースへ更新し、サイドカーのポート/エンドポイントを内部限定に制限すること。公開・到達実績のあるインスタンスは侵害された前提で、ファイル・アカウント・設定・ログ痕跡を調査する。
