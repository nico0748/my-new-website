# OpenAI と Broadcom が自社推論チップ「Jalapeño」を発表 — 推論コスト約半減を狙う専用 ASIC

> トピック: Jalapeño 推論チップ（2026年6月24日発表）｜出典: OpenAI 公式 / TechCrunch / Tom's Hardware｜種別: AI 推論ハードウェア

## 一行サマリー

OpenAI と Broadcom が、LLM 推論に特化して設計した OpenAI 初の自社チップ「Jalapeño」を 2026年6月24日に発表した。GPU 比で推論トークンあたり約50%のコスト削減を狙う専用 ASIC で、2026年末の初期展開を目標とする。AI の「使うたびにかかる費用（推論コスト）」を構造的に下げる動きとして注目される。

## 🔰 初学者向け（何が・なぜ重要か）

AI モデルには2つの費用局面がある。ひとつは「モデルを作る（学習・トレーニング）」費用、もうひとつは「作ったモデルを使って答えを出す（推論・inference）」費用。チャットや画像生成が世界中で毎秒大量に呼ばれる今、後者の「使うたびの費用」が積み上がり、AI 企業にとって最大級のコストになっている。

これまで推論は主に NVIDIA の GPU で動かしてきた。GPU は万能で高性能だが、「あらゆる計算向け」に作られているぶん、推論だけを回すには無駄な部分もある。Jalapeño は「推論だけを速く安くやる」ことに絞って専用設計したチップ（ASIC＝特定用途向け集積回路）。料理でいえば、なんでも作れる万能包丁ではなく、その料理専用のよく切れる道具を用意したようなもの。専用化により、同じ答えを出すのにかかる電力・費用を大きく下げられる、というのが狙いだ。AI がもっと安く使えるようになる土台の話で、開発者・利用者双方に効いてくる。

## 🧠 専門的解説（技術詳細・設計・使いどころ）

### 技術的詳細（何が変わったか）

OpenAI 公式および各報道によれば、Jalapeño は OpenAI 初の「Intelligence Processor」で、Broadcom と共同開発した LLM 推論特化のアクセラレータ。フロンティアモデルにとって重要なカーネル・メモリ移動・ネットワーキング・サービング（配信）パターンに合わせてアーキテクチャを最適化したとされる。Tom's Hardware は「レチクルサイズ級の大型 ASIC を約9か月という短期開発サイクルで実現」と報じ、OpenAI 側は自社 AI モデルを設計支援に用いたと述べている。性能面では、Broadcom CEO の発言としてGPU 現行世代比で推論トークンあたり約50%のコスト削減、電力あたり性能（perf/watt）も現行最先端を大きく上回るとされる（いずれも初期ラボ検証値で、実運用性能は要確認）。初期展開は2026年末目標で、初回生産分の約40%を Microsoft が購入する見込みと報じられている。

### 仕組み / 背景（なぜこう設計されたか）

汎用 GPU は幅広い演算に対応するため、推論に不要な機能や汎用性のためのオーバーヘッドを含む。一方、LLM 推論は「巨大な重み行列の読み出し（メモリ帯域律速）」「アテンションのカーネル」「トークンを次々生成する逐次処理」「多数リクエストのバッチング/配信」という比較的定型のパターンに支配される。ASIC 化の発想は、この定型パターンにハードウェアを合わせ込み、メモリ移動・ネットワーク・演算ユニットを推論に最適配置することで、単位電力あたりのスループットを引き上げること。垂直統合（自社でモデルもチップも設計）により、モデルの実行特性とハードを共設計できるのが強み。トレードオフは、ASIC は特定ワークロードに強い反面、モデルアーキテクチャが大きく変われば柔軟性で GPU に劣りうること、巨大 ASIC の歩留まり・供給・電力/冷却といった製造・運用リスクを抱えること、そして NVIDIA エコシステム（CUDA 等のソフト資産）から外れるためソフトウェアスタックの整備が必要なこと。

### 実務での使いどころ・移行の勘所

これは一般開発者が直接買うチップではなく、まず OpenAI 自身の推論基盤（および提携クラウド）のコスト構造に効く話。開発者への波及は「API 経由の推論単価が中期的に下がりうる」「大手が自社推論 ASIC を持つ流れ（Google TPU、Amazon 系、Microsoft 系に続く動き）が加速する」という形で現れる。使いどころの勘所は、当面は発表・初期検証段階であり、実際の可用性・価格反映・対応モデル範囲は2026年末以降の展開と実測を待つ必要があること。アプリ側の設計としては、特定ハード/プロバイダに密結合せず、推論バックエンドを差し替え可能に保つ（モデル抽象・ルーティング層を挟む）ことが、こうしたハード競争の恩恵を受けつつリスクを分散する現実的な備えになる。

## 📖 用語解説（このトピックとのつながり）

- **推論（inference）**: 学習済みモデルを実行して出力を得る処理。運用時の主要コスト。Jalapeño はここに特化。
- **ASIC**: 特定用途向けに設計した集積回路。汎用 GPU よりワークロード適合時に効率が高い。
- **perf/watt（電力あたり性能）**: 同じ電力でどれだけ処理できるか。データセンター運用の主要指標。
- **垂直統合 / 共設計**: モデルとハードを同一組織が最適化すること。実行特性に合わせた最適化が可能になる。

## 影響範囲 / 推奨アクション

- AI 推論コストの構造的低下と、大手による自社推論 ASIC 化の潮流が加速する。API 推論単価の中期的低下要因。
- 現段階は発表・初期検証。可用性・価格反映・対応範囲は2026年末以降の展開と実測を待つ（数値は初期ラボ値で要確認）。
- アプリ側は推論バックエンドを差し替え可能に保ち（モデル抽象/ルーティング層）、特定ハード・プロバイダへの密結合を避ける。
