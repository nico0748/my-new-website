# MySQL 9.7 LTS登場 — Enterprise機能がCommunityへ、Hypergraphオプティマイザとデータマスキング

> トピック: MySQL 9.7 LTS｜出典: Oracle MySQL Release Notes / InfoQ｜種別: メジャーLTSリリース

## 一行サマリー

MySQL 9.7.0 が 8.4 以来となる新しい LTS（長期サポート）系列として 2026-04-21 にリリースされ、Hypergraph オプティマイザや JSON Duality View、in-database JavaScript など、これまで Enterprise 版に限られていた機能の一部を Community 版に開放した。6-16 には CPU（重要セキュリティパッチ）である 9.7.1 も公開されている。

## 🔰 初学者向け（何が・なぜ重要か）

MySQL は世界で最も広く使われるオープンソースのリレーショナルデータベースの一つです。バージョンには2種類あり、新機能をどんどん試す「Innovation 系列（9.0〜9.6）」と、機能を固めて数年間手厚くサポートする「LTS 系列」があります。LTS は本番システムが安心して長く乗り続けられる「安定の岸」のようなものです。今回の 9.7 は、8.4 以来ひさびさに現れたその安定の岸で、これまで積み上がってきた 9.x の新機能をまとめて持ち込んだ点が重要です。

特に大きいのが「これまで有償の Enterprise 版でしか使えなかった機能の一部が、無料の Community 版でも使えるようになった」ことです。データベースを選ぶとき、機能が有償版に閉じ込められていると個人開発者や小さなチームは手が出せません。その壁が下がったことは、MySQL を使い続ける人にとって素直に嬉しいニュースです。

もう一つの目玉「Hypergraph オプティマイザ」は、SQL を実行するとき「どのテーブルをどの順で結合すれば一番速いか」を計算する頭脳部分の新エンジンです。テーブルをたくさん結合する複雑なクエリで、より賢い実行計画を立てられるようになります。

## 🧠 専門的解説（技術詳細・設計・使いどころ）

### 技術的詳細（何が変わったか）

- **リリース**: 9.7.0 = 2026-04-21（LTS 系列の起点）、9.7.1 = 2026-06-16（Critical Security Patch Update）。
- **Community へ開放された機能群**: これまで Enterprise 限定だった一部の能力が Community Server に移された（具体的な対象機能は公式リリースノートで要確認）。逆に **Dynamic Data Masking（動的データマスキング）** は Enterprise Edition 向けの新機能として追加。
- **Hypergraph オプティマイザ**: 複雑なクエリの最適化を改善する新しい結合順序探索エンジン。従来の貪欲法ベースのオプティマイザに対し、ハイパーグラフ表現で結合空間を探索する。
- **JSON Duality View**: リレーショナルなテーブル群を JSON ドキュメントとして読み書きできる二重ビュー。Community Server で **DML（更新系）** がサポートされた。
- **MySQL REST Service（MRS）の改善**、**OpenID 認証** のサポート、**in-database JavaScript**（DB 内で JavaScript ストアドを実行）など開発者向け機能を拡充。
- **運用機能**: クラスタのスロットリング可視化のための flow-control モニタリング、レプリケーション applier 統計の拡張。

### 仕組み / 背景（なぜこう設計されたか）

MySQL の Innovation/LTS 二本立ては、新機能を素早く届けたい要求と、本番を長期安定運用したい要求の綱引きを分離するための設計です。Innovation 系列で 9.0〜9.6 と積み上げた機能を、実績を見ながら LTS へ「合流」させるのが 9.7 の役割です。ユーザーは Innovation で先取りするか、LTS で腰を据えるかを選べます。

Hypergraph オプティマイザは、結合するテーブルが増えるほど組合せ爆発を起こす「結合順序決定」を、ハイパーグラフというデータ構造で表して探索する近年の研究成果（DuckDB などでも採用が進む方向）を取り込んだものです。従来の MySQL オプティマイザは結合順序を貪欲に決めるため、多テーブル結合で最適解を外すことがありました。ハイパーグラフ探索はより網羅的に良い計画を選べる一方、探索コストとのトレードオフがあります。

Enterprise 機能の Community 開放は、PostgreSQL や各種フォーク（MariaDB 等）との競争の中で、コミュニティの信頼と採用を維持する戦略的な判断です。「MySQL 9.7 Is Out and the Community Wins」という Oracle 公式ブログの見出しがそれを象徴しています。

### 実務での使いどころ・移行の勘所

- **新規に LTS を選ぶなら 9.7 が起点**。8.4 LTS からの移行や、9.x Innovation を追ってきたチームの「着地先」として自然。
- Hypergraph オプティマイザは複雑な分析系クエリで効く可能性が高いが、既存クエリの実行計画が変わりうるため、**移行前にプランのリグレッションテスト**を推奨。
- JSON Duality View の DML 対応は、ドキュメント指向 API とリレーショナルモデルを両立させたいアプリで有効。ただし更新の一貫性・制約の扱いは要検証。
- Community に開放された機能を採用する前に、対象バージョン（9.7.0 か 9.7.1 か）とエディションで実際に使えるか、公式リリースノートで確認すること。
- 9.7.1 は CPU（重要セキュリティ修正）なので、本番は速やかにパッチ適用の検討を。

## 📖 用語解説（このトピックとのつながり）

- **LTS（Long-Term Support）**: 数年単位で手厚くサポートされる安定系列。9.7 がその新起点。
- **Hypergraph オプティマイザ**: 結合順序をハイパーグラフとして探索する最適化エンジン。多テーブル結合の実行計画を改善。
- **JSON Duality View**: 同じデータをリレーショナルにも JSON ドキュメントにも見せる二重ビュー。今回 Community で更新系が解禁。
- **Dynamic Data Masking**: クエリ結果の機微データを動的にマスクする機能。9.7 では Enterprise 向けに追加。
- **CPU（Critical Patch Update）**: Oracle 製品の四半期セキュリティ修正。9.7.1 がこれに該当。
- **Innovation 系列**: 新機能を先行投入する短命系列（9.0〜9.6）。LTS の 9.7 と対になる。

## 影響範囲 / 推奨アクション

- 新規本番 DB を MySQL で立てるなら、Innovation ではなく **LTS の 9.7** を第一候補に。
- 複雑クエリを多用するシステムは、Hypergraph オプティマイザによる**実行計画変化を事前検証**。
- Community 版ユーザーは、開放された機能の対象を公式リリースノートで確認し、恩恵を受けられるか棚卸し。
- 9.7.1 のセキュリティ修正内容を確認し、パッチ適用計画を立てる。
- Enterprise 利用者は Dynamic Data Masking で PII 保護要件を満たせるか評価。
