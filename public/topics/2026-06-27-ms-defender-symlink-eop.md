# Microsoft Defender のリンク追従の脆弱性 CVE-2026-41091 が実環境で悪用（SYSTEM 権限昇格）

> トピック: 脆弱性ニュース ｜ 出典: The Hacker News / Microsoft / Help Net Security / CISA KEV ｜ 種別: リンク追従（シンボリックリンク追従、CWE-59）による権限昇格

## 一行サマリー

Windows 標準のウイルス対策 Microsoft Defender（マルウェア対策エンジン）にリンク追従の脆弱性 CVE-2026-41091（CVSS 7.8、High）があり、低権限ユーザーが SYSTEM 権限を奪取できる。実環境で悪用が確認され CISA KEV にも登録されているため、Defender エンジンの最新版（v1.1.26040.8 以降）への更新が必須。

## 🔰 初学者向け（何が・なぜ問題か）

Microsoft Defender は、Windows にあらかじめ入っている「ウイルス対策ソフト」です。怪しいファイルを見つけて隔離・削除するために、パソコンの中をとても強い権限（SYSTEM=管理者よりさらに上の最強権限）で動いています。普段は頼もしい味方ですが、今回その「強い権限」が逆手に取られました。

問題の正体は「ショートカット（リンク）の追いかけ方」の不注意です。Windows には、あるファイルを指し示す「近道（シンボリックリンク）」を作る仕組みがあります。たとえるなら「この棚を開けてね」と書いた付箋（リンク）ですが、その付箋の貼り先を後からこっそり「金庫を開けてね」に貼り替えられるイメージです。Defender は付箋の指す先を十分に確認せず、強い権限のまま言われた場所を開けて操作してしまいます。

そのため、ほとんど権限のない一般ユーザーでも、巧妙に用意した「付箋」を Defender に踏ませることで、本来触れないシステムの重要ファイルを Defender の強権で書き換えさせ、最終的に自分が SYSTEM 権限を握る、という乗っ取りができてしまいます。すでに実際の攻撃で使われている（悪用確認済み）ため、Defender を最新の状態にすることが何より大切です。

## 🧠 専門的解説（技術詳細・メカニズム・検知対応）

### 技術的詳細（種別・前提条件）

- 種別: Microsoft Malware Protection Engine（マルウェア対策エンジン）におけるリンク追従の脆弱性（symbolic link following、CWE-59 の型）。結果は局所的権限昇格（LPE / EoP）。
- CVSS: 7.8（High、要確認）。攻撃元はローカル。前提として「対象端末で認証済みの低権限ユーザーとしてコードを実行できる」こと（＝初期侵入後の権限昇格に使われる二段目のバグ）。ユーザー操作は不要に近い。
- 影響: Microsoft Malware Protection Engine v1.1.26030.3008 など脆弱バージョン。Defender が有効な Windows 端末は広範に該当しうる。
- 修正版: マルウェア対策エンジン v1.1.26040.8 で修正（要確認）。エンジンは通常バックグラウンドで自動更新されるが、オフライン・隔離端末では未更新の恐れ。
- 悪用状況: Microsoft が「実環境で悪用観測」と公表。CISA が KEV へ登録し、連邦機関に期限付き是正を指示（2026-06-03 期限、要確認）。同時期に Defender の DoS（CVE-2026-45498）も悪用が報じられている。

### メカニズム（攻撃が成立する仕組み）

注: 概念レベルにとどめ、武器化手順・PoC・正確なパス等は記載しない。

1. **特権コンポーネントが低権限の入力を扱う**: Defender はファイルスキャン等のため、低権限ユーザーが作成・配置したファイルやパスを SYSTEM 権限で開く。ここで「誰が用意した経路か」という信頼境界をまたぐ。
2. **リンクの貼り替え（信頼境界の悪用）**: 攻撃者（低権限ユーザー）は、Defender が後でアクセスする場所にシンボリックリンク／ジャンクションを仕込み、その指し先を保護されたシステム資源へ向ける。Defender はリンクの最終解決先を十分検証せず追従する。
3. **TOCTOU 的な隙**: 「Defender がパスを確認した時点」と「実際に開く時点」の間でリンク先をすり替えられると、検証をすり抜けて意図しない対象を操作させられる（チェックと使用の時間差＝TOCTOU）。これにより攻撃者は SYSTEM 権限での任意ファイル操作（書き込み／作成／切り詰め）というプリミティブを獲得する。
4. **権限昇格への橋渡し**: SYSTEM での任意書き込みプリミティブは、サービスが読み込む DLL・設定・スケジュール等を改変させることで、最終的に SYSTEM としての任意コード実行＝完全な権限昇格へ昇華しうる。
5. **一般原理**: 「高権限プロセスが低権限ユーザー管理下のパスを、リンク解決を厳密化せずに辿る」設計は、リンク追従／TOCTOU の典型的攻撃クラス。AV/EDR のような常時 SYSTEM 稼働コンポーネントは攻撃価値が高い。

### 検知・暫定緩和・恒久対策

- 検知の着眼点: 低権限ユーザーによる予期せぬシンボリックリンク／ジャンクション作成、Defender 関連プロセスからの保護領域への書き込み、権限昇格直後に出現する新規サービス・スケジュールタスク。EDR でのトークン昇格イベント。
- 暫定緩和: マルウェア対策エンジンの自動更新が機能していることを確認。隔離・エアギャップ端末はオフライン定義／エンジン更新を手動適用。一般ユーザー権限の最小化と不要なローカル実行の抑制。
- 恒久対策: Defender マルウェア対策エンジンを修正版（v1.1.26040.8 以降、要確認）へ更新。プラットフォーム／定義更新の配信状況を資産横断で点検。
- 運用示唆: 「ウイルス対策自身が攻撃面になる」前提で、AV/EDR コンポーネントのバージョン管理を脆弱性管理対象に明示的に組み込む。

## 📖 用語解説（このニュースとのつながり）

- **権限昇格（EoP / LPE）**: 低い権限から高い権限を奪う攻撃。本件は一般ユーザー→SYSTEM。初期侵入後の「足場固め」に使われる。
- **SYSTEM 権限**: Windows で最も強い権限。Defender がこの権限で動くため、悪用されると被害が致命的になる。
- **シンボリックリンク追従（CWE-59）**: リンクの指し先を検証せず辿る欠陥。本脆弱性の中核。
- **TOCTOU**: 検査時と使用時の時間差を突く攻撃。リンクのすり替えを成立させる隙。
- **KEV**: CISA の「実際に悪用された脆弱性」カタログ。登録＝至急対応の合図。

## 影響範囲 / 推奨対応

Microsoft Defender が有効な Windows 端末が広く対象。マルウェア対策エンジンを修正版へ更新し（多くは自動更新で適用済み）、オフライン端末は手動更新する。組織はエンジン／プラットフォームのバージョンを点検し、低権限アカウントの管理と EDR による昇格検知を強化する。初期侵入を許さない多層防御と併せて運用する。
