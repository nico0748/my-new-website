# Mastra が A2A を既定サポート — TypeScript エージェント同士を協調させる

> トピック: Mastra の Agent-to-Agent（A2A）既定対応と native MCP｜出典: Mastra 公式ブログ｜種別: AI エージェント基盤（機能追加）

## 一行サマリー

TypeScript 製エージェントフレームワーク Mastra が、2026年5月19日にエージェント間連携プロトコル A2A（Agent-to-Agent）を既定サポートし、外部の A2A エージェントを Mastra のサブエージェントとして動かしたり、Client SDK から呼び出せるようになった。加えて MCP（Model Context Protocol）にもネイティブ対応し、「ツール連携＝MCP、エージェント連携＝A2A」の二本柱を TypeScript で扱える。

## 🔰 初学者向け（何が・なぜ重要か）

AI エージェントが1体で全部やるのは限界があります。得意分野の違うエージェントが**互いに仕事を頼み合える**と、より複雑なことができます。そのための「エージェント同士の共通語」が A2A（Agent-to-Agent）というプロトコルです。

Mastra は、JavaScript/TypeScript でエージェントを作るための人気フレームワーク。今回そこに A2A が標準搭載され、他所で動いているエージェントを「自分の部下（サブエージェント）」のように呼び出して働かせられるようになりました。たとえるなら、社外の専門業者に電話一本で仕事を発注できる共通の受発注ルールが整った、というイメージ。

さらに、道具（検索・DB など）を使うための共通規格 MCP にも対応済み。「他のエージェントに頼む（A2A）」と「道具を使う（MCP）」の両方を、TypeScript の慣れた書き方で扱えるのが強みです。Web 開発者がそのまま AI エージェント開発に踏み込みやすくなります。

## 🧠 専門的解説（技術詳細・設計・使いどころ）

### 技術的詳細（何が変わったか）

- **A2A 既定サポート（2026年5月19日）**: Mastra のエージェントと Client SDK が Agent-to-Agent プロトコルに標準対応。外部 A2A エージェントを Mastra のサブエージェントとして実行、または Client SDK 経由で呼び出せる。
- **native MCP**: 直近で MCP のネイティブ対応を追加。MCP サーバー（ツール群）をエージェントに接続できる。
- 位置づけ: Mastra は LangChain / CrewAI / Microsoft Agent Framework / LlamaIndex Workflows / Google ADK / OpenAI Agents SDK と並ぶ主要フレームワークの一つ（TypeScript ネイティブが差別化）。

（プロトコルの実装細目・相互運用範囲は公式ドキュメントで要確認。）

### 仕組み / 背景（なぜこう設計されたか）

エージェント時代の相互運用は「2つの軸」に整理されつつある。1つは**エージェントとツール**の接続で、これを担うのが MCP（モデルにツール/データを標準手順で繋ぐ）。もう1つは**エージェントとエージェント**の連携で、これを担うのが A2A（あるエージェントが別のエージェントに、能力の広告・タスク委譲・結果受領を標準化された方法で行う）。Mastra が両対応することは、この二層モデルを TypeScript エコシステムで完結させる意味を持つ。

A2A の設計思想は「エージェントを、内部実装を隠したサービスとして相互接続する」こと。各エージェントは自分の能力（できること）を広告し、他エージェントはそれを発見して委譲する。これによりベンダーやフレームワークをまたいだ協調が可能になる。トレードオフは、プロトコル越しの委譲はネットワーク境界・認証・エラー処理・遅延といった分散システムの難しさを持ち込む点で、単一プロセス内のサブエージェント呼び出しより運用が複雑になる。MCP と A2A を分けて扱う設計は、「ツール呼び出し」と「エージェント委譲」を混同しないための整理でもある。

### 実務での使いどころ・移行の勘所

TypeScript/Node 中心のチームが、フロント〜バック〜エージェントを一貫した言語で組みたい場合に有力。社内外の専門エージェント（検索特化、データ分析特化など）を A2A で束ねる構成や、MCP でツール群を差し込む構成が作りやすい。導入時は、A2A 越しの委譲に伴う認証・タイムアウト・再試行・観測性を設計し、ローカルのサブエージェントで済む処理を無用にネットワーク化しない切り分けを行う。他フレームワーク（Python 系の LangGraph 等）との比較では、言語エコシステムと A2A/MCP 対応状況を要件に照らす。

## 📖 用語解説（このトピックとのつながり）

- **Mastra**: TypeScript 製の AI エージェントフレームワーク。本トピックの主役。
- **A2A（Agent-to-Agent）**: エージェント同士の能力広告・タスク委譲を標準化するプロトコル。
- **MCP（Model Context Protocol）**: モデルにツール/データを繋ぐ標準プロトコル。A2A と対をなす。
- **サブエージェント**: 別エージェントを部品として呼び出して働かせる構成。
- **能力広告（capability advertisement）**: エージェントが自分のできることを他者に示す仕組み。A2A の要素。

## 影響範囲 / 推奨アクション

TypeScript 中心のチームは、MCP（ツール）と A2A（エージェント連携）を一貫言語で扱えるようになった。専門エージェントの束ね役に有効。A2A 委譲は分散システムの作法（認証・再試行・観測性）を設計し、ローカルで済む処理の過剰なネットワーク化を避ける。
