# Langflow の公開フロー実行エンドポイントで未認証RCE（CVE-2026-33017, KEV・実環境で悪用）

> トピック: 脆弱性ニュース｜出典: The Hacker News / SonicWall / GitHub Advisory（GHSA-vwmf-pq79-vjvx）｜種別: 未認証リモートコード実行（コードインジェクション）

## 一行サマリー
ローコードAI構築ツール Langflow の「ログイン不要で公開フローを実行する」エンドポイントが、利用者の入力したPythonコードをそのまま `exec()` に渡してしまうため、ネット越しに誰でも任意コードを実行できる（CVE-2026-33017, CVSS 9.8）。公開からわずか20時間で悪用が始まり、Monero採掘マルウェアの設置に使われている。

## 🔰 初学者向け（何が・なぜ問題か）
Langflow は「AIの処理の流れ（フロー）」をブロックを並べるように組み立てられるツールです。便利機能として「ログインしなくても、お試し用に公開フローを動かせる」入り口（公開エンドポイント）が用意されていました。

問題は、その入り口が「フローを動かすついでに、リクエストに含まれていたプログラムの断片もそのまま実行してしまう」点です。たとえるなら、店先に「ご自由にお試しください」と置かれた調理マシンが、客が紙に書いて投げ込んだ任意のレシピを無検査でそのまま調理・実行してしまうようなもの。客（攻撃者）は「サーバーを乗っ取るレシピ」を投げ込むだけで、店の厨房（サーバー本体）を自由に操れてしまいます。

しかもLangflowは初期設定で「自動ログイン（AUTO_LOGIN）」が有効なため、誰でも管理者級のトークンを取得して公開フローを作れてしまいました。結果として、インターネットに露出したLangflowサーバーは、ID・パスワードなしで丸ごと乗っ取られ、攻撃者は暗号通貨Monero（モネロ）を勝手に採掘するプログラムを仕込みました（サーバーの計算資源を盗まれる被害）。

## 🧠 専門的解説（技術詳細・メカニズム・検知対応）

### 技術的詳細（種別・前提条件）
- **CVE**: CVE-2026-33017。**CVSS**: 出典により9.8（一部9.3）と表記揺れあり（要確認）。**種別**: 信頼できない入力のコード実行（コードインジェクション→RCE、CWE-94相当）。
- **悪用状況**: CISA KEV登録済み（出典では2026-03-25登録と記載＝**日付は要確認**）。実環境で**Monero採掘キャンペーン**に悪用。公開後**約20時間**で悪用開始、PoC公開前から攻撃発生。
- **前提条件**: Langflowインスタンスがインターネットに露出していること。初期設定の `AUTO_LOGIN=true` により未認証の訪問者が superuser トークンを取得可能。
- **影響バージョン**: 〜1.8.2 を含む全バージョンが影響。**修正版**: 1.9.0（出典時点でPyPI/GitHub Releasesの最新が1.8.3で、修正版の配布が追いついていないとの指摘あり＝**要確認**）。
- **エンドポイント**: `POST /api/v1/build_public_tmp/{flow_id}/flow`（公開フローのビルド実行）。

### メカニズム（攻撃が成立する仕組み）
1. **信頼境界の誤設定**: 「公開フロー実行」エンドポイントは“未認証ユーザーがアクセスする”ことを前提に設計されているにもかかわらず、その先で**サーバー側コード実行（`exec()`）という最高権限の操作**に到達できてしまう。すなわち、外部の untrusted 入力がサーバー内部の信頼コード実行面へ橋渡しされている。
2. **入力の無検証実行**: リクエストの任意パラメータ（フロー定義中の data 相当）に含まれるPython文字列が、サンドボックス・許可リスト・構文制限なしで `exec()` に渡る。これは「データのはずの入力を、コードとして解釈・実行する」典型的コードインジェクションのアンチパターン。
3. **認証前提の崩壊**: `AUTO_LOGIN` が既定で有効なため、本来必要なはずの認証が事実上スキップされ、攻撃者は superuser トークンを取得して公開フローを“正規手順で”作成できる。設計上の安全側のガード（認証）が運用既定値で無効化されている。
4. **被害への橋渡し**: 攻撃者は「任意コード実行」というプリミティブを握ると、cron/常駐プロセスとして採掘バイナリを設置し、必要に応じて権限・横展開へ拡張できる。サービスが特権アカウントで動作している場合、被害はホスト全体に及ぶ。
   - 一般原理: 「ユーザー入力を eval/exec/デシリアライズに流す」設計は、入力検証や最小権限の有無に関わらず、untrusted データを実行可能コードへ昇格させる経路を作る。AI/ローコード基盤は「ユーザー定義ロジックの実行」が機能そのものであるため、この境界設計を誤ると致命的になりやすい。
   - 本記述は概念レベルに留め、武器化可能な手順・PoC・正確なペイロードは記載しない。

### 検知・暫定緩和・恒久対策
- **検知の着眼点**: `/api/v1/build_public_tmp/` への外部からのPOST、未認証セッションによるフロー作成、Webサービスプロセスから生成される子プロセス（python/シェル/採掘バイナリ）、外部マイニングプール（Monero）への通信、CPU使用率の異常上昇。
- **暫定緩和**: Langflowをインターネットから隔離（VPN/IP制限/リバースプロキシ認証）。`AUTO_LOGIN=false` に設定。サービスを非特権アカウントで実行。公開フロー機能を使わないなら無効化。
- **恒久対策**: 1.9.0以降へアップデート（配布状況を確認）。AIフロー実行は専用サンドボックス/コンテナで最小権限・ egress 制限の下に隔離。
- **運用示唆**: 「ユーザー定義ロジックを実行するAI基盤」は、認証既定値とコード実行境界を必ず棚卸しする。

## 📖 用語解説（このニュースとのつながり）
- **コードインジェクション**: データとして渡された文字列がコードとして実行されてしまう欠陥。本件では公開エンドポイントのPython `exec()` がこれにあたる。
- **`exec()`（Python）**: 文字列をPythonコードとして実行する関数。untrusted入力を渡すと即RCEになる。本件の中核。
- **AUTO_LOGIN**: Langflowの初期設定で認証を自動化する仕組み。既定有効が未認証アクセスを許す要因となった。
- **KEV（Known Exploited Vulnerabilities）**: CISAが「実際に悪用が確認された」脆弱性をまとめたカタログ。登録は優先対応のシグナル。
- **Monero（モネロ）採掘**: 匿名性の高い暗号通貨をサーバー資源で無断採掘する攻撃。乗っ取り後の収益化手段として本件で観測。

## 影響範囲 / 推奨対応
インターネットに露出したLangflow（〜1.8.2を含む）が対象。AI開発・実験用に立てたインスタンスが放置・露出していると即侵害のリスク。直ちに外部到達性を遮断し、`AUTO_LOGIN=false`、非特権実行、1.9.0以降への更新（配布状況確認）を実施。既に露出していた場合は子プロセス・採掘通信・不審フローの有無を侵害調査する。
