# Safari 26.5 — Anchor Positioning と Scroll-driven Animations の「実戦投入級」安定化

> トピック: Safari 26.5（WebKit）の機能更新｜出典: WebKit 公式ブログ｜種別: ブラウザ定期リリース（ベータ）

## 一行サマリー

Safari 26.5 は CSS Anchor Positioning の実運用パターンに関する複数の修正（3段以上のアンカー連鎖、ビューポートリサイズ時のメディアクエリ再評価、unitless zero の受理など）と、Scroll-driven Animations の信頼性修正4件を含み、新機能の「仕様追加」から「本番で壊れない」段階への成熟を示すリリースとなった。

## 🔰 初学者向け（何が・なぜ重要か）

ブラウザの新機能には2つの段階があります。「使えるようになった」段階と、「安心して使える」段階です。ニュースになりやすいのは前者ですが、実際の仕事で大事なのは後者で、Safari 26.5 はまさにその「地固め」のリリースです。

たとえば「アンカー配置」（吹き出しをボタンにくっつけて表示する CSS の新機能）は、単純な例では動いても、「吹き出しの中にさらに吹き出し」のような3段重ねにすると正しく配置されないバグがありました。また「スクロールに合わせて絵が動く」機能（スクロール駆動アニメーション）も、ページを戻ったときにアニメーションが復元されない、一時停止が効かないといった細かい不具合がありました。26.5 はこうした「現場で踏む穴」を集中的に塞いでいます。

Safari は iPhone のブラウザエンジンでもあるため、Safari で安定すると「全ブラウザで使える」と言える機能が一気に増えます。地味に見えて、フロントエンド全体の底上げになる更新です。

## 🧠 専門的解説（技術詳細・設計・使いどころ）

### 技術的詳細（何が変わったか）

Anchor Positioning 関連の修正:
- ビューポートリサイズ時、anchor positioning と viewport units 併用下でメディアクエリが再評価されない問題を修正。
- 3つ以上連鎖するアンカー配置要素が正しく解決されない問題を修正。
- `anchor()` のフォールバック値が unitless zero を受け付けない問題を修正。
- `display: contents` の要素が `anchor-scope` 使用時にアンカースコープを確立しない問題を修正。

Scroll-driven Animations 関連の修正:
- scroll animation timeline の range name サポートを修正。
- `animation-play-state: paused` を動的に設定した際に正しく一時停止しない問題を修正。
- view timeline アニメーションが 0%/100% 閾値付近で不正な進捗値を返す問題を修正。
- back-forward cache からの復帰後に animation timeline が復元されない問題を修正。

その他: CSS `random()` 関数から `element-shared` キーワードを削除（新しい既定動作がユースケースをカバーし、CSSWG が仕様から削除したことに追従）。対象 OS は iOS/iPadOS/visionOS/macOS 26.5 beta ほか、macOS Sequoia/Sonoma にも配信。正式リリース日は要確認（本記事執筆時点ではベータ）。

### 仕組み / 背景（なぜこう設計されたか）

アンカー連鎖の問題は、アンカー解決がレイアウトの後段で行われる（配置対象のサイズ・位置が確定してから次のアンカーを解決する）ため、依存グラフが深くなると解決順序と無効化（invalidation）の管理が難しくなることに起因する。ネストしたポップオーバー（メニュー→サブメニュー→ツールチップ）は実アプリで普通に発生する構造であり、この修正は「デモは動くが本番で壊れる」溝を埋めるもの。

`random()` の `element-shared` 削除は、CSS 標準化の健全なサイクルを示す例でもある。実装フィードバックを受けて CSSWG が仕様から冗長なキーワードを外し、実装が追従して削除する——「一度出荷したら永遠に残る」Web の互換性原則の中で、ベータ段階での早期削除により将来の負債を回避した。

トレードオフ: この種の修正リリースは新機能がないため注目されにくいが、Interop（ブラウザ間相互運用性テスト）のスコア向上に直結し、開発者が polyfill や UA 分岐を消せる時期を早める。

### 実務での使いどころ・移行の勘所

- Floating UI からネイティブ Anchor Positioning への移行を検討中のチームは、Safari 26.5 の安定化で「ネスト構造を含む本格移行」の検証を始められる。移行判断は Baseline 到達（全エンジン安定）を基準に。
- scroll-driven animations を採用済みのサイトは、bfcache 復帰・pause 制御まわりの自前ワークアラウンドを 26.5 以降で削減できるか確認。
- `random()` の `element-shared` を先行利用していた実験的コードは書き換えが必要（新既定動作で代替）。

## 📖 用語解説（このトピックとのつながり）

- **WebKit**: Safari のレンダリングエンジン。iOS では全ブラウザの土台（EU の代替エンジン容認の動向は別論点）。
- **anchor-scope**: どのアンカー名がどの範囲で解決されるかを制御するプロパティ。`display: contents` との組み合わせバグが今回修正された。
- **view timeline**: 要素がビューポートに入ってから出るまでを進捗軸にするタイムライン。0%/100% 境界の精度が今回改善。
- **bfcache（back-forward cache）**: 戻る/進むでページを瞬時に復元する仕組み。アニメーション状態の復元漏れは体感品質に直結する。
- **Baseline**: 全主要ブラウザで安定利用できる機能セットの目印。修正リリースの積み重ねが Baseline 入りを支える。

## 影響範囲 / 推奨アクション

- ポップオーバー/ツールチップ/スクロール演出をネイティブ CSS へ寄せる計画の検証環境に Safari 26.5 beta を加える。
- 既存のアンカー配置ワークアラウンド（リサイズ時の強制再計算等）に「Safari 26.5 以降不要」の目印を付けて技術的負債として管理。
