# ソニー Optical Disc Archive Software のインストーラ権限不備でSYSTEM権限実行（CVE-2026-50255 / JVN#79926428）

> トピック: 脆弱性ニュース ｜ 出典: JVN / IPA / JPCERT/CC ｜ 種別: 不適切な既定アクセス権設定（権限昇格）

## 一行サマリー
ソニー製 Optical Disc Archive Software（Windows版）のインストーラに、不適切な既定アクセス権設定（CVE-2026-50255）があり、ローカルの一般ユーザーがSYSTEM権限で任意コードを実行できる。影響は 5.5.3 以前。IPA経由で報告され、JPCERT/CCが開発者と調整した国内報告事例。

## 🔰 初学者向け（何が・なぜ問題か）
Optical Disc Archive Software は、業務用の光ディスク（大容量アーカイブ）を読み書きするためのソフトです。今回の問題はその**インストーラ（導入プログラム）**にあります。

ソフトを入れるとき、関連するファイルやフォルダに「誰が触っていいか」という権限が設定されます。ここの設定が緩すぎて、**一般の利用者でも書き換えられる**状態になっていました。攻撃者（その端末を使える一般ユーザー）は、本来管理者しか触れない場所に細工を仕込み、システムが最高権限（SYSTEM）でそれを実行するよう仕向けられます。

たとえるなら、会社のビルで「掃除用具入れ」の鍵が誰でも開けられる状態になっていて、その中に仕掛けをすると、ビルの管理人（最高権限）がそれを実行してしまうようなものです。結果として、普通の社員アカウントが管理者並みの権限を奪える＝**権限昇格**が起きます。

## 🧠 専門的解説（技術詳細・メカニズム・検知対応）

### 技術的詳細（種別・前提条件）
- **CVE / 識別子**: CVE-2026-50255、JVN#79926428（JVNDB-2026-000084）。深刻度はHigh相当（CVSS数値は要確認）。
- **種別（CWE）**: CWE-276（Incorrect Default Permissions / 不適切な既定アクセス権設定）。
- **前提条件**: 攻撃者が対象端末で **ローカルの一般ユーザー権限**を保有していること（ローカル権限昇格）。
- **影響バージョン**: Optical Disc Archive Software for Windows **5.5.3 以前**。修正版の適用が必要（ベンダ案内を確認）。
- **悪用状況**: IPAへ届出、JPCERT/CCが調整。実環境での悪用報告は現時点で要確認（KEV未登録）。

### メカニズム（攻撃が成立する仕組み）
インストール時に作成されるリソースの権限設計の甘さが、特権プロセスによる「攻撃者制御物の実行」へ橋渡しされる典型的なローカル権限昇格である（概念レベルで説明）。

1. **信頼境界**: SYSTEM権限で動くサービスやインストール関連コンポーネントは、「自分が読み込む実行ファイル/ライブラリ/スクリプトは信頼できる場所にある」という前提で動く。
2. **不備の所在（CWE-276）**: インストーラが作る実行ファイルやディレクトリの既定ACL（アクセス権）が緩く、一般ユーザーに書き込み・置換が許されている。ここで「特権プロセスが参照するファイルを攻撃者が制御できる」プリミティブが発生。
3. **置換／仕込み**: 攻撃者は当該パスに悪意あるDLL/実行ファイルを置く、あるいは正規ファイルを置換する。
4. **昇格の成立**: SYSTEM権限のプロセスがそのファイルを読み込み・実行した瞬間、攻撃者のコードがSYSTEM権限で走る。一般ユーザー→SYSTEMへの昇格が完成する。

一般原理として、特権プロセスが触れるファイル・ディレクトリは「非特権ユーザーが書き換え不可能」でなければならない（最小権限と書き込み権の分離）。既定ACLの緩さは、DLLハイジャックや実行ファイル置換に直結する。

### 検知・暫定緩和・恒久対策
- **検知の着眼点**: 当該インストールディレクトリ配下のファイル作成・置換、SYSTEMプロセスによる非標準パスからのモジュール読み込み、一般ユーザーによる特権ディレクトリへの書き込み。
- **暫定緩和**: 対象ディレクトリのACLを見直し、一般ユーザーの書き込み権を剥奪。当該ソフトの利用端末を信頼できる利用者に限定。
- **恒久対策**: ベンダ提供の修正版へ更新（5.5.3 より後）。インストール後のACL監査を運用に組み込む。

## 📖 用語解説（このニュースとのつながり）
- **CWE-276（不適切な既定アクセス権）**: 作成物の初期権限が緩すぎる欠陥。本件の根本原因。
- **権限昇格（LPE）**: 低い権限から高い権限を奪うこと。本件は一般ユーザー→SYSTEM。
- **ACL（アクセス制御リスト）**: 誰が何をできるかの権限設定。緩いACLが攻撃面になる。
- **DLLハイジャック／実行ファイル置換**: 特権プロセスが読む対象を攻撃者が差し替える手口。本件の悪用パターン。
- **JVN / JPCERT/CC**: 国内の脆弱性情報窓口・調整機関。本件の一次情報源。

## 影響範囲 / 推奨対応
Optical Disc Archive Software（Windows版 5.5.3以前）を導入した端末が対象。放送・映像・アーカイブ業務など業務用途で使われるため、共用端末では一般ユーザーによる昇格リスクがある。修正版へ更新し、インストールディレクトリのACLを点検・是正する。
