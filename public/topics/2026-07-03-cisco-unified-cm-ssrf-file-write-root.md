# Cisco Unified CM の SSRF 脆弱性（CVE-2026-20230）— ファイル書き込みからroot昇格、実攻撃で悪用

> トピック：IP電話基盤への未認証攻撃 ｜ 出典：The Hacker News / Cisco Advisory / SecurityWeek / CISA KEV ｜ 種別：サーバサイドリクエストフォージェリ（CWE-918）

## 一行サマリー

企業のIP電話（VoIP）基盤である Cisco Unified Communications Manager に、未認証で悪用可能なSSRF脆弱性（CVE-2026-20230、CVSS 8.6）が見つかり、PoC公開後に「ファイル書き込み→root昇格」の実攻撃が観測され、CISAのKEVに登録・短期の是正期限が設定された。

## 🔰 初学者向け（何が・なぜ問題か）

Cisco Unified Communications Manager（略してUnified CM／CUCM）は、社内の電話・通話を制御する“電話交換機のソフト版”です。誰がどこに発信できるか、通話をどうつなぐかを管理する、オフィス通信の心臓部です。

今回の欠陥は「SSRF（サーバサイドリクエストフォージェリ）」というタイプです。たとえるなら、受付係（サーバ）に「この住所に代わりにお使いに行ってきて」と頼める仕組みがあり、本来は限られた用事にしか使えないはずが、細工した頼み方をすると、受付係が“本来行ってはいけない社内の奥（内部システムや自分自身）”へ勝手に使いに行かされてしまう、というものです。攻撃者は自分では直接触れない場所へ、サーバの権限を借りてアクセスさせられます。

さらに悪いことに、今回の攻撃はそこから一歩進み、サーバのファイルシステムに「攻撃者の好きなファイルを書き込む」ことにつながります。書き込めるファイル次第で、最終的にサーバの最高権限（root）を奪える、と報告されています。電話基盤を乗っ取られると、盗聴、通話の改ざん、社内ネットワークへの侵入拠点化などにつながります。しかも実際に悪用が始まっているため、早急なパッチが必要です。

## 🧠 専門的解説（技術詳細・メカニズム・検知対応）

### 技術的詳細（種別・前提条件）

- **CVE**：CVE-2026-20230
- **CVSS**：8.6（High）
- **CWE**：CWE-918（SSRF）。派生してファイル書き込み→特権昇格に連鎖。
- **影響製品**：Cisco Unified CM および Unified CM SME。
- **前提条件**：**未認証・リモート**で特定のHTTPリクエストを送れること。ただし悪用成立には **WebDialer サービスが有効**である必要がある（既定では無効）。この前提が実環境の被害範囲を左右する。
- **悪用状況**：2026年6月22日に悪用試行が観測され、6月23日に実環境悪用が複数ソースで報道。CISAは6月25日にKEV登録し、連邦機関に6月28日までの是正を要求。
- **修正版**：Unified CM / Unified CM SME の 14SU6 および 15SU5 で修正。

### メカニズム（攻撃が成立する仕組み）

SSRFを起点に、ファイル書き込みプリミティブを経てroot権限へ橋渡しされる連鎖です（概念レベル。武器化手順・PoC・オフセットは記載しません）。

1. **入力検証の欠落（信頼境界の越境）**：特定のHTTPリクエストにおいて、サーバが処理するURL/宛先が十分に検証されない。攻撃者は「サーバに代理アクセスさせる宛先」を制御できるプリミティブ（＝SSRF）を握る。
2. **内部リソースへの到達**：SSRFにより、外部からは直接到達できない内部エンドポイントやローカルサービス（サーバ自身のlocalhost含む）へ、サーバの権限でリクエストを送らせる。ここで“信頼された内部通信”という前提が破られる。
3. **書き込みプリミティブへの昇格**：内部到達を足がかりに、攻撃者はOS上の**ファイルへ任意内容を書き込む**ことが可能になる（報告ベース）。SSRF単体は「読み・アクセス」に見えても、到達先の機能次第で「書き込み」に化けるのが危険な点。
4. **root昇格への橋渡し**：任意ファイル書き込みが得られると、攻撃者は起動時に実行されるスクリプト、cron/サービス定義、設定ファイル等、**高権限で解釈されるファイル**を狙って上書き・投入する。これにより後続処理がroot権限で攻撃者コードを実行し、アプライアンスの完全掌握（root）に至る。
5. **クラス一般原理**：SSRFは「サーバに任意宛先へリクエストさせられる」プリミティブであり、それ自体より“到達先で何ができるか”が被害を決める。到達先に無認証の管理機能やファイル操作機能があると、情報漏えいにとどまらずRCE/特権昇格へ連鎖する。WebDialerのような追加サービスが有効だと攻撃面が拡大する。

### 検知・暫定緩和・恒久対策

- **検知の着眼点**：Unified CMのWebアクセスログで、内部宛先やlocalhostへ誘導するような不自然なパラメータを含むリクエスト。想定外のファイル生成・改変（起動スクリプトやサービス定義の変更時刻）。WebDialer関連エンドポイントへの異常アクセス。
- **暫定緩和**：**不要なら WebDialer サービスを無効化**（既定は無効）。管理・Webインタフェースへのアクセス元をVPN/管理セグメントに限定。境界での不審リクエスト遮断。
- **恒久対策**：14SU6 / 15SU5 へアップデート。悪用確認済みのため、適用前に露出していた機器は侵害調査（改変ファイル・不審アカウント・永続化の痕跡）を実施。
- **運用示唆**：通信基盤アプライアンスは「境界に置きがち＋パッチ遅延しがち」で狙われやすい。KEV是正期限（6/28）が短いことは緊急度の高さを示す。

## 📖 用語解説（このニュースとのつながり）

- **Unified CM（CUCM）**：Ciscoの企業向けIP電話・通話制御基盤。今回の標的で、乗っ取られると通信の盗聴・改ざんや侵入拠点化に直結。
- **SSRF（サーバサイドリクエストフォージェリ）**：サーバに攻撃者指定の宛先へリクエストを送らせる脆弱性。本件の起点で、内部到達→ファイル書き込みへ発展した。
- **ファイル書き込みプリミティブ**：攻撃者が任意の内容・場所にファイルを書ける能力。高権限で読まれるファイルを狙うことでroot昇格の橋になる。
- **WebDialer**：クリック発信などを担うUnified CMのサービス。悪用の前提条件で、無効化が有効な緩和策。
- **KEV／是正期限**：CISAが悪用確認済み脆弱性に設定する対応期限。6/28という短さが緊急度を物語る。

## 影響範囲 / 推奨対応

Unified CM / Unified CM SME を運用する組織は最優先で 14SU6 / 15SU5 へ更新。当面は WebDialer の有効/無効を確認し、不要なら無効化、管理面のネットワーク露出を絞る。6月22日以降にインターネット到達可能だった機器は、ファイル改変・永続化・不審アカウントの侵害調査を必ず実施すること。
