# エフサステクノロジーズ製 ServerView Agents for Windows のローカル権限昇格でSYSTEM奪取（CVE-2026-27788/CVE-2026-32325）

> トピック: 脆弱性ニュース ｜ 出典: JVN#67883085 / エフサステクノロジーズ / IPA ｜ 種別: 不適切なアクセス権割当（CWE-732）＋権限昇格（CWE-268）

## 一行サマリー

富士通系のエフサステクノロジーズが提供するサーバ管理ソフト「ServerView Agents for Windows」に、ローカルの一般ユーザが最高権限のSYSTEMを奪取できる2件の権限昇格脆弱性（CVSS 8.5）が公表された。V11.60.04 以前が対象で、最新版へのアップデートまたはワークアラウンド適用が推奨される。国内データセンタ・企業サーバで広く使われる管理ソフトのため要注意。

## 🔰 初学者向け（何が・なぜ問題か）

ServerView Agents は、富士通（現エフサステクノロジーズ）製サーバの状態（温度・ファン・ディスク・ハードの故障予兆など）を監視・管理する“サーバの健康診断ソフト”です。企業のサーバルームやデータセンタで、当たり前のように入っている縁の下の力持ちです。

今回の問題は「権限昇格（けんげんしょうかく）」と呼ばれるタイプです。権限昇格とは、本来は権限の低い利用者が、こっそり“もっと偉い立場”に成り上がってしまう欠陥のこと。今回の場合、そのサーバにログインできる一般ユーザ（あるいは、なんらかの方法で一般ユーザとして忍び込んだ攻撃者）が、Windowsで最も強い「SYSTEM権限」（何でもできる管理者以上の全能権限）を横取りできてしまいます。

なぜそんなことが起きるのか。原因の一つは「ファイルやフォルダに対する鍵のかけ方（アクセス権）がゆるすぎた」こと（CWE-732）。たとえるなら、SYSTEM権限で自動的に動く重要プログラムの置き場所に、誰でも書き込める“ゆるい鍵”がかかっていた状態です。攻撃者はそこへ細工した中身を置いておき、あとでシステムがそれを高い権限で実行するのを待つだけで、権限を奪えてしまいます。

単体では「まず一般ユーザとしてサーバに入れること」が前提なので、外からいきなり全部奪えるわけではありません。しかし攻撃は普通、他の穴で足がかりを得た後にこうした権限昇格で“本丸”を取りにきます。つまりこの脆弱性は、侵入者が最終的にサーバを完全掌握するための「昇進ルート」を与えてしまうのです。

## 🧠 専門的解説（技術詳細・メカニズム・検知対応）

### 技術的詳細（種別・前提条件）

- **CVE-2026-27788**: 重要なリソースに対する不適切なアクセス権の割当（CWE-732）／CVSS 4.0 基本値 8.5、CVSS 3.0 基本値 7.8（`AV:L/AC:L/PR:L/UI:N/S:U/C:H/I:H/A:H`）。
- **CVE-2026-32325**: 権限昇格（CWE-268）／CVSS 4.0 基本値 8.5、CVSS 3.0 基本値 7.8。
- **前提条件**: 当該製品がインストールされたサーバに**ログイン可能な**ローカル攻撃者（ローカル・低権限）。ネットワーク越しの単独悪用ではない。
- **影響バージョン**: ServerView Agents for Windows V11.60.04 およびそれ以前。
- **修正版/対応**: 開発者提供情報に基づき最新版へアップデート。適用までの間はベンダ推奨のワークアラウンド適用が推奨。
- **悪用状況**: 現時点で実環境での悪用報告は確認されていない（要確認）。報告者は株式会社ラック 飯田雅裕 氏、早期警戒パートナーシップに基づきIPA/JPCERT/CCが調整。

### メカニズム（攻撃が成立する仕組み）

本件は「高権限で動くコンポーネントが参照するリソースの保護が不十分」という、ローカル権限昇格（LPE）の典型パターンに該当する（CVE詳細の内部実装は非公開のため以下は型の説明を含む）。

1. **信頼境界の破れ（緩いACL）**: CWE-732 は、重要リソース（実行ファイル、ライブラリ、設定ファイル、サービスが読む作業ディレクトリ等）に対し、低権限ユーザが書き込み・改変できてしまうアクセス権設定を指す。SYSTEM等の高権限プロセスが、低権限ユーザの改変可能な資源を信頼して読み込む時点で、権限の信頼境界が崩れている。

2. **獲得プリミティブ＝高権限プロセスへの入力制御**: 低権限攻撃者は、緩いACLを突いて「後で高権限プロセスが読み込む/実行する対象」の中身を差し替える。代表的な型として、書き込み可能なパスへの悪性DLL配置（DLLハイジャック）、実行バイナリ/スクリプトの置換、引用符なしサービスパス（unquoted service path）や検索パスの悪用などがある（一般型の列挙であり本件の断定ではない）。

3. **SYSTEMへの橋渡し**: 差し替えた資源を高権限プロセス（サービス／自動起動コンポーネント）が実行・読込した瞬間、攻撃者の制御下のコードがSYSTEM権限で走る。低権限→SYSTEM という最終被害への橋渡しが成立する。CWE-268（権限昇格）は、こうした権限管理の欠陥そのものを指す。

4. **クラスの一般原理**: サーバ管理エージェントはSYSTEM権限で常駐するため攻撃価値が高い。「高権限プロセス × 低権限で改変可能な資源」の組み合わせは、Windows製品で反復して現れるLPEの王道。対策の要諦は、実行・読込対象の資源へ低権限ユーザの書き込みを許さない厳格なACL設計である。

### 検知・暫定緩和・恒久対策

- **検知の着眼点**: ServerView Agents のインストールディレクトリ・サービス関連パスに対する、低権限ユーザによるファイル作成/改変を監視（ファイル整合性監視）。SYSTEM権限プロセスによる想定外の子プロセス生成、未署名DLLのロードを検出。イベントログでサービス起動異常を確認。
- **暫定緩和**: パッチ適用までの間、ベンダ推奨のワークアラウンドを適用。加えて、対象サーバへの対話ログイン可能なアカウントを最小化し、一般ユーザのローカルログインを制限。関連ディレクトリのACLを見直し、低権限の書き込みを排除。
- **恒久対策**: ServerView Agents for Windows を開発者提供の最新版へアップデート。エフサステクノロジーズの告知（resources/security/2026/0529.html）を確認。
- **運用示唆**: 権限昇格は多段攻撃の“詰め”に使われる。境界防御だけでなく、サーバ内の最小権限・資源ACLの健全性を定期点検する。

## 📖 用語解説（このニュースとのつながり）

- **ServerView Agents**: 富士通系サーバのハードウェア監視・管理エージェント。SYSTEM権限で常駐するため昇格の標的価値が高い。
- **権限昇格（LPE, CWE-268）**: 低い権限のユーザがより高い権限を不正取得する欠陥。今回は最終的にSYSTEMを奪取。
- **不適切なアクセス権割当（CWE-732）**: 重要リソースに緩い権限を与えてしまう欠陥。低権限ユーザによる改変を許し、昇格の起点になる。
- **SYSTEM権限**: Windowsで最も強力な実行権限。取得＝サーバの実質的完全掌握。
- **DLLハイジャック / 引用符なしサービスパス**: 緩いACLや検索パスの不備を突いて高権限プロセスに悪性コードを読ませるLPEの代表的手法（一般型）。
- **JVN / 早期警戒パートナーシップ**: 国内の脆弱性情報を調整・公開する枠組み。本件はラック社の報告をIPA/JPCERT/CCが調整して公表。

## 影響範囲 / 推奨対応

ServerView Agents for Windows V11.60.04 以前を稼働する富士通系サーバ（データセンタ・企業サーバ）が対象。ローカルログイン前提のため単独での外部侵入は起きないが、他の侵入経路と組み合わせた権限昇格・完全掌握のリスクがある。最新版へアップデートし、適用までベンダ推奨ワークアラウンドを適用。対象サーバのローカルログインアカウント最小化と関連ディレクトリのACL見直しを併せて実施すること。
