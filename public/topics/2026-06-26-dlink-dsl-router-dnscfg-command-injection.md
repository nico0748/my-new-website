# サポート終了 D-Link DSL ルーターのコマンドインジェクション CVE-2026-0625 を実環境で悪用（パッチ提供なし）

> トピック: 脆弱性ニュース ｜ 出典: SecurityWeek / Dark Reading / The Hacker News ｜ 種別: OS コマンドインジェクション（CWE-78）

## 一行サマリー

サポート終了済みの D-Link DSL ゲートウェイ（DSL-2740R 等）の DNS 設定エンドポイント dnscfg.cgi に、入力検証不備によるコマンドインジェクション CVE-2026-0625（CVSS 9.3）があり、攻撃者が遠隔で任意コマンドを実行している。D-Link はパッチを出さず機器の廃棄・買い替えを推奨している。

## 🔰 初学者向け（何が・なぜ問題か）

家庭や小規模オフィスのインターネット接続を支える「ルーター」は、家のネットワークの玄関であり、すべての通信が通る門番です。今回問題になったのは、その D-Link 製ルーターのうち、すでに何年も前に販売・サポートが終了した古い機種です。

問題の箇所は「DNS（電話帳のように、サイト名を住所＝IPアドレスに変換する仕組み）の設定を受け付ける窓口」です。本来はサーバー名を書く欄なのに、攻撃者がそこに紛れ込ませた「命令文」を、ルーターがそのまま実行してしまう欠陥があります。たとえるなら、宅配の宛名欄に「（ついでに金庫を開けて）」と書いたら、配達員が本当に金庫を開けてしまうようなものです。

その結果、攻撃者はルーターを乗っ取り、あなたの通信を盗み見たり、偽サイトへ誘導（DNS書き換え）したり、他者を攻撃する踏み台に使えます。最も厄介なのは、メーカーが「もう修理しません＝パッチは出しません」と明言している点です。直す薬がないため、対策は「古い機種を新しいサポート対象機に買い替える」ことになります。

## 🧠 専門的解説（技術詳細・メカニズム・検知対応）

### 技術的詳細（種別・前提条件）

- 種別: OS コマンドインジェクション（CWE-78）。dnscfg.cgi（DNS 設定処理）でのユーザー入力の検証/サニタイズ不備。
- CVSS: 9.3（Critical、要確認）。遠隔から悪用、無認証の DNS 改変（DNSChanger）挙動と関連。
- 影響機種（要確認）: DSL-2740R / DSL-2640B / DSL-2780B / DSL-526B などの 2016〜2019 年頃のファームウェア亜種。いずれもサポート終了（EOL）。
- 修正版: **なし**。D-Link はパッチを提供せず、機器の廃棄・買い替えを推奨。
- 悪用状況: 2025年11月下旬以降、実環境で悪用継続。VulnCheck が2025年12月16日に D-Link へ報告。

### メカニズム（攻撃が成立する仕組み）

注: 概念レベルにとどめ、武器化手順・PoC は記載しない。

1. **信頼できない入力の取り込み**: dnscfg.cgi は DNS サーバー設定としてユーザー入力を受け取る。設計上は「サーバーのアドレス文字列」を想定。
2. **検証/エスケープの欠落**: その入力を OS のシェルコマンド組み立てに渡す際、メタ文字（コマンド区切り等）を無害化しないため、入力の一部がデータではなく「コマンド」として解釈される。これがインジェクションの核心。
3. **コマンド実行プリミティブの獲得**: 攻撃者は DNS 設定要求に見せかけて任意コマンドを注入し、ルーター上で実行する力を得る。組み込み機器では実行権限が高いことが多い。
4. **被害への橋渡し**: 乗っ取り後は、DNS の書き換え（利用者を偽サイトへ誘導＝DNSChanger）、通信の盗聴・改ざん、ボットネット組み込みによる踏み台化などに連鎖する。
5. **一般原理**: 「Web 管理 CGI が入力をシェルへ無検証で渡す」のは IoT/ルーターで頻出のコマンドインジェクションクラス。EOL でパッチが出ない点が被害を長期化させる。

### 検知・暫定緩和・恒久対策

- 検知の着眼点: ルーターの DNS 設定が身に覚えなく変更されていないか、管理画面の外部公開有無、当該機種からの不審な外向き通信、ISP からの不正通信通知。
- 暫定緩和: 機器の管理インターフェースをインターネットから遮断、リモート管理(WAN 側 GUI)を無効化、DNS 設定を確認・是正。ただし根本解決にはならない。
- 恒久対策: **EOL 機種をサポート対象の新機種へ交換**（メーカー推奨）。買い替えまでの間はインターネット直結を避ける。
- 運用示唆: 家庭/SMB 環境でも資産棚卸しを行い、サポート切れネットワーク機器を計画的に置き換える。

## 📖 用語解説（このニュースとのつながり）

- **ルーター/DSLゲートウェイ**: 家庭・小規模拠点のネット接続を担う機器。乗っ取られると全通信が危険に晒される。
- **コマンドインジェクション（CWE-78）**: 入力に紛れた命令を機器が実行してしまう欠陥。今回の根本原因。
- **dnscfg.cgi**: DNS 設定を扱う管理用 CGI。本件の脆弱な入口。
- **DNS / DNSChanger**: サイト名を住所に変換する仕組み／その設定を悪意で書き換え偽サイトへ誘導する手口。
- **EOL（End of Life）**: サポート終了。パッチが出ないため、買い替え以外に根本対策がない。

## 影響範囲 / 推奨対応

DSL-2740R / DSL-2640B / DSL-2780B / DSL-526B などサポート終了済み D-Link DSL ルーターの利用者（家庭・小規模拠点）が対象。パッチは提供されないため、サポート対象機種への買い替えが唯一の恒久策。当面はリモート管理の無効化、DNS 設定の確認、管理画面の非公開化で被害を抑える。
