# WatchGuard Firebox（Fireware OS）に複数のRCE（CVE-2026-13053 ほか）— 認証済み管理者がCLI/Web UIからコード実行・任意ファイル書き込み

> トピック: 脆弱性ニュース ｜ 出典: Cyber Security News / WatchGuard PSIRT / Threat-Modeling.com ｜ 種別: 境界外書き込み／パストラバーサル（認証済みRCE）

## 一行サマリー

企業向けファイアウォール「WatchGuard Firebox」のOS（Fireware OS）に、認証済み管理者が管理CLIやWeb UIから任意コード実行・任意ファイル書き込みに至る3件の脆弱性（CVE-2026-13053／13050／13054, いずれもCVSS v4.0 8.6）が公開された。境界外書き込み2件とパストラバーサル1件で、成立すれば機器の完全掌握につながる。広範な影響バージョンが対象で、最新版への更新が必要。

## 🔰 初学者向け（何が・なぜ問題か）

ファイアウォールは、社内ネットワークと外の世界の間に立つ“防火壁つきの門番”です。WatchGuard Fireboxはその門番を担う企業向け機器で、通信の許可・遮断を管理します。

今回の弱点は、この門番の**管理画面（コマンド操作＝CLI／ブラウザ操作＝Web UI）**にありました。管理者としてログインした状態で特殊な操作を行うと、機器が想定外の動きをして、攻撃者の命令を実行したり、機器内の好きな場所にファイルを書き込んだりできてしまいます。たとえるなら、門番室の操作盤に細工した指示を入れると、盤が誤作動して門番の権限を丸ごと奪われる——そんなイメージです。

ポイントは、これらの悪用には**「管理者としてログインできること」が前提**という点です。つまり、まず管理者のパスワードを盗む・推測する、あるいは別の穴で管理画面に入り込む、といった一段階が必要です。しかし一度そこを越えられれば、ファイアウォールそのものを乗っ取られ、通信の傍受・改ざん・遮断や、社内への侵入拡大の拠点にされてしまいます。門番が敵側につく被害の大きさから、早急な更新が求められます。

## 🧠 専門的解説（技術詳細・メカニズム・検知対応）

### 技術的詳細（種別・前提条件）

- **CVE / CVSS**: CVE-2026-13053、CVE-2026-13050、CVE-2026-13054。いずれも **CVSS v4.0 8.6**。
  - **CVE-2026-13053**: Fireware OSのCLIコマンドハンドラにおける**境界外書き込み**。特権認証ユーザーが細工CLIコマンドで任意コード実行。
  - **CVE-2026-13050**: `networkd` プロセスの**境界外書き込み**。管理Web UIへの細工リクエストで任意コード実行。
  - **CVE-2026-13054**: 管理Web UIの**パストラバーサル**。ログイン済み攻撃者がFireboxファイルシステム上の任意の場所にファイル書き込み。
- **影響バージョン**: Fireware OS 11.0〜11.12.4_Update1、12.0〜12.12、12.5〜12.5.18、および 2025.1〜2026.2。
- **前提条件**: **特権（管理者）認証**が必要。管理CLIまたは管理Web UIへの到達が前提。未認証単独では成立しない。
- **悪用状況**: 公開時点で本3件の広範な実悪用報告は確認されていない（要確認）。ただしFireboxは過去に実悪用事例があり、認証情報侵害と連鎖すると危険度が高い。
- **修正**: 更新版Fireware OS（2025.1.4 / 12.11.6 / 12.5.15 系など）が提供。運用バージョンに応じ最新へ更新。

### メカニズム（攻撃が成立する仕組み）

本件は「管理面の入力を、境界検証・パス正規化なしに危険な処理へ渡す」欠陥群である。概念レベルで整理する（武器化手順・オフセット・PoCは記載しない）。

1. **信頼境界と前提**: 管理CLI／Web UIは、認証済み管理者からの入力を受けて内部の設定・ネットワーク処理を実行する。ここでは「管理者入力は整形されており安全」という暗黙の前提が置かれている。
2. **境界外書き込みの成立（CVE-2026-13053/13050）**: CLIコマンドハンドラや `networkd` が、入力の長さ・範囲を十分に検証しないまま固定サイズのメモリ領域へ書き込む。これにより**隣接メモリを上書きする境界外書き込み（メモリ破壊）**プリミティブが生じ、制御データの改ざんを通じてコード実行へ昇格する。
3. **任意ファイル書き込みの成立（CVE-2026-13054）**: Web UIが受け取るパスを正規化せず、`../` 等でファイルシステムを遡れるため、攻撃者は**任意パスへのファイル書き込み**プリミティブを得る。起動時・定期実行時に読み込まれる資産へ書き込めば、これもコード実行・永続化へ橋渡しされる。
4. **最終被害への橋渡し**: いずれも認証済み管理者権限が起点だが、成立すれば機器（ファイアウォール）の完全掌握に至る。境界機器の掌握は、通信ポリシーの改変・傍受・遮断、内部侵入の足場化という面的被害を生む。

一般原理として、これは**入力長・パスの検証不備（境界外書き込み＋パストラバーサル）**という2クラスの実装欠陥で、管理面という高権限インターフェースに存在する点が危険性を高めている。認証前提でも、資格情報侵害やCSRF等と連鎖すると実害化する。

### 検知・暫定緩和・恒久対策

- **検知の着眼点**: 管理CLI／Web UIへの異常な入力・長大パラメータ、`networkd` のクラッシュ／再起動、想定外パスへのファイル生成、管理者ログイン後の不審操作、設定・ポリシーの無断変更。
- **暫定緩和**: 管理面（CLI/Web UI）を**インターネットから遮断**し、管理専用ネットワーク／VPN経由に限定。管理者アカウントのMFA・強パスワード化と最小化。管理アクセス元をACLで制限。
- **恒久対策**: 更新版Fireware OSへ即時アップデート。管理資格情報のローテーション、管理面の露出点検、侵害兆候の調査を併せて実施。

## 📖 用語解説（このニュースとのつながり）

- **Fireware OS / Firebox**: WatchGuardのファイアウォール製品と、その専用OS。ネットワーク境界の要で、掌握されると影響が広範。
- **境界外書き込み（Out-of-Bounds Write）**: 確保領域の外にデータを書き込む欠陥。制御データを上書きしてコード実行へ昇格し得る。今回のCLI/networkdの2件が該当。
- **パストラバーサル**: `../` 等でパスを遡り、意図しない場所のファイルを操作する欠陥。今回は任意ファイル書き込みに直結。
- **認証済みRCE**: 悪用に事前の認証（管理者ログイン）が必要なコード実行。資格情報侵害や他脆弱性との連鎖で実害化する点に注意。

## 影響範囲 / 推奨対応

広範なFireware OSバージョン（11.x〜12.x、2025.1〜2026.2）を搭載するWatchGuard Fireboxが対象。悪用には管理者認証が要るが、成立時は機器の完全掌握＝境界防御の崩壊につながる。更新版への即時適用、管理面のインターネット非公開化、管理者MFA・強パスワード、アクセス元制限を実施。管理資格情報の漏えい対策も並行して行うこと。
