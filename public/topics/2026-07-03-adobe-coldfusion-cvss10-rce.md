# Adobe ColdFusion に CVSS 10.0 の複数脆弱性、ファイルアップロード/パストラバーサルで RCE

> トピック: 脆弱性ニュース ｜ 出典: The Hacker News / BleepingComputer / Adobe(APSB26-68) ｜ 種別: 無制限ファイルアップロード(CWE-434)・パストラバーサル(CWE-22)・不適切な入力検証(CWE-20) による RCE

## 一行サマリー

Adobe が ColdFusion と Campaign Classic で最大深刻度(CVSS 10.0)の複数脆弱性を修正。無制限ファイルアップロード、パストラバーサル、入力検証不備により、いずれもユーザー操作不要で任意コード実行に至りうる。ColdFusion 2025(Update 9 以前)/2023(Update 20 以前)が対象。

## 🔰 初学者向け（何が・なぜ問題か）

ColdFusion は、Web アプリを作るための「土台(アプリケーションサーバー)」で、企業の業務システムや会員サイトの裏側で動いていることがあります。今回の問題は複数ありますが、代表的なのは「ファイルの受け取り方が甘い」ことです。

Web サイトには、写真や書類をアップロードする窓口がよくあります。安全な設計なら「受け取っていいのは画像だけ」「置いていい場所はこのフォルダだけ」と厳しく制限します。ところが今回の脆弱性では、その制限が甘く、攻撃者が「プログラムとして実行できるファイル(≒サーバーを操るための命令書)」を、しかも実行できてしまう場所に置ける状態でした。これは、宅配の受付が「食品だけ」と言いながら、実際は爆発物でも受け取って倉庫の真ん中に置いてしまうようなものです。

さらに「パストラバーサル」という、ファイルの置き場所を指定する文字列に細工して**本来触れない場所へ抜け出す**手口も併発します。結果として、攻撃者はサーバー上で好きなプログラムを動かせ(=RCE)、システム全体を掌握しかねません。しかも一部はログイン不要・ユーザー操作不要で、CVSS は満点の 10.0。極めて危険なため、Adobe は緊急のアップデートを出しました。

## 🧠 専門的解説（技術詳細・メカニズム・検知対応）

### 技術的詳細（種別・前提条件）

- **主な CVE(いずれも CVSS 10.0)**:
  - CVE-2026-48276, CVE-2026-48283 … 無制限ファイルアップロード(CWE-434)
  - CVE-2026-48277, CVE-2026-48281, CVE-2026-48316 … 不適切な入力検証(CWE-20)
  - CVE-2026-48282 … パストラバーサル(CWE-22)、いずれも任意コード実行につながる
- **種別(CWE)**: CWE-434 / CWE-22 / CWE-20 の複合。加えて Adobe はデシリアライズ対策として MySQL Java Connector の更新とシリアルフィルタ見直しを推奨。
- **前提条件**: ネットワーク到達性。CVSS 10.0 群は「ユーザー操作不要」で、認証要件は CVE により異なるため個別に**要確認**(満点スコアの多くは低権限/未認証を含意)。
- **影響バージョン**: ColdFusion 2025(Update 9 以前)、ColdFusion 2023(Update 20 以前)。
- **修正版**: ColdFusion 2025 → Update 10、ColdFusion 2023 → Update 21。Campaign Classic も該当更新あり。
- **悪用状況**: 本記事時点で広範な実悪用の断定情報はなし(**要確認**)。ColdFusion は歴史的に公開後短期間で悪用される傾向が強く、優先適用が必要。

### メカニズム（攻撃が成立する仕組み）

複数クラスが絡むため、代表的な 2 系統(アップロード→実行、パストラバーサル→書込)を概念レベルで分解する(具体的パス・PoC は記載しない)。

1. **信頼境界**: アップロード窓口やファイル操作 API は「外部入力(ファイル名・種別・保存先・中身)」を受け取る箇所であり、ここが最も外側の信頼境界。ここでの検証欠落が全体の起点。
2. **無制限ファイルアップロード系(CWE-434)**: 拡張子・MIME・マジックバイトの検証が不十分だと、実行可能なスクリプト(ColdFusion なら .cfm/.cfc 相当)をアップロードできる。さらに保存先が Web から到達可能かつ実行対象のディレクトリだと、アップロード後に URL アクセスするだけでサーバー上で実行され、Web シェル化する。攻撃者が握るプリミティブは「任意ファイルの設置＋実行」。
3. **パストラバーサル系(CWE-22)**: 保存パスやファイル参照に `../` 等の相対経路や絶対パスを注入できると、想定ディレクトリの外へ「脱出」して任意の場所へ書き込み/読み出しできる。書込先を起動時に読み込まれる設定やスクリプト置き場に向ければ、間接的にコード実行へ橋渡しされる。
4. **入力検証不備系(CWE-20)**: パラメータの型・範囲・許可値の検証が甘い箇所は、上記 2 系統の前提条件を崩す(検証をすり抜ける)テコとして働き、RCE の成立確率を上げる。
5. **最終被害への橋渡し**: いずれも「攻撃者制御の入力 → ファイルシステム上の任意書込/実行 → サーバー権限でのコード実行」という共通経路。ColdFusion はサービス権限で動くため、成功時の影響はホスト掌握に及ぶ。
6. **クラスとしての位置づけ**: 本件は「入力検証と出力先制御の欠落」というアプリケーション層の古典的かつ致命的なクラス。メモリ破壊ではなく、正規機能(アップロード/ファイル操作)の設計不備が武器化される点が特徴。

> 注: 個々の CVE の正確な認証要件・内部経路は Adobe 情報の範囲で判断すべきで、ここでは CWE クラスの一般原理として説明する。

### 検知・暫定緩和・恒久対策

- **検知の着眼点**: Web ルート配下に見覚えのない .cfm/.cfc/スクリプトが出現していないか、アップロードディレクトリからの実行(cfml エンジンの実行ログ)、`../` を含む異常なファイル操作パラメータ、ColdFusion サービスからの想定外プロセス生成。
- **暫定緩和(パッチ前)**: アップロードディレクトリの実行権を剥奪(スクリプト実行を無効化)、Web からの直接到達を遮断。管理コンソールを外部公開しない。WAF でパストラバーサルパターンと不審な拡張子アップロードを遮断。Adobe のシリアルフィルタ/接続ドライバ更新推奨を適用。
- **恒久対策**: ColdFusion 2025 → Update 10、2023 → Update 21 へ更新。Campaign Classic も該当更新を適用。
- **運用示唆**: 公開 ColdFusion は資産棚卸しの上で「そもそも公開が必要か」を再評価。過去侵害の痕跡(既設 Web シェル)がないかもあわせて点検する。

## 📖 用語解説（このニュースとのつながり）

- **無制限ファイルアップロード(CWE-434)**: 危険な種類のファイルを制限なく受け付けてしまう不備。今回の主要 RCE 経路。
- **パストラバーサル(CWE-22)**: 相対パス等で想定外のディレクトリへ抜け出す手法。任意書込/読出の起点。
- **Web シェル**: サーバー上に設置され、遠隔でコマンドを実行できる悪性スクリプト。アップロード脆弱性の典型的な最終形。
- **シリアルフィルタ**: Java のデシリアライズで許可する型を絞る仕組み。Adobe が併せて強化を推奨。
- **CVSS 10.0**: 深刻度の満点。ユーザー操作不要・広範な影響を示唆し、最優先対応の指標。

## 影響範囲 / 推奨対応

ColdFusion 2025(Update 9 以前)/2023(Update 20 以前)、および Campaign Classic の該当版を運用する組織が対象。特にインターネット公開インスタンスは最優先。対応はまず最新 Update への適用、次いでアップロードディレクトリの実行無効化・管理画面の非公開化・WAF での防御を実施。あわせて既存の侵害痕跡(不審ファイル)を点検する。
