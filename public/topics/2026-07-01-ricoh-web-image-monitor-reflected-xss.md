# リコー製プリンタ／複合機のWeb Image Monitorに反射型XSS、管理画面が悪用される恐れ（CVE-2025-41393）

> トピック: 脆弱性ニュース ｜ 出典: JPCERT/CC / Ricoh セキュリティ情報 / 各種ベンダ解析 ｜ 種別: 反射型クロスサイトスクリプティング（XSS）

## 一行サマリー

リコー製の多くのレーザープリンタ・複合機（MFP）の管理用Web画面「Web Image Monitor」に、入力値の処理不備による反射型XSSの脆弱性があり、JPCERT/CC調整で2026年6月30日に公表された。管理者が細工リンクを踏むと、ブラウザ上で攻撃者のスクリプトが実行され得る。

## 🔰 初学者向け（何が・なぜ問題か）

オフィスの複合機（コピー・スキャン・印刷をする機械）には、パソコンのブラウザから設定をいじれる管理画面があります。リコー製品ではこれを「Web Image Monitor」と呼びます。IT担当者がここから機器の設定や状態を確認します。

今回の弱点は、この管理画面が利用者の入力を“そのまま”画面に表示してしまうミス（反射型XSS）です。例えるなら、受付に置いた掲示板に、来た人が書いた内容を職員が無検査でそのまま貼り出してしまう状態。悪意ある人が「掲示」に紛れて命令文（スクリプト）を仕込んでおくと、それを見た人のブラウザがその命令を実行してしまいます。

具体的には、攻撃者が細工したURLを作り、複合機の管理者にメール等で踏ませます。管理者がそのリンクを開くと、管理者のブラウザ上で攻撃者のスクリプトが動き、管理画面の操作内容や認証情報が盗まれたり、設定を勝手に変えられたりする恐れがあります。複合機はスキャンした書類や宛先情報など機微なデータを扱うため、軽視できません。

## 🧠 専門的解説（技術詳細・メカニズム・検知対応）

### 技術的詳細（種別・前提条件）

- **CVE**: CVE-2025-41393（JPCERT/CC調整。公表 2026-06-30。対応するJVN番号は要確認）
- **深刻度**: Medium（反射型XSS。CVSS数値は要確認）
- **脆弱性クラス**: 反射型クロスサイトスクリプティング（Reflected XSS / CWE-79）。入力のサニタイズ／出力エンコーディング不備。
- **前提条件**: (1) 攻撃者が細工したURLを用意し、(2) Web Image Monitorにアクセスできる利用者（特に管理者）にそのリンクを踏ませる（ソーシャルエンジニアリング）。管理画面がネットワーク到達可能であること。
- **影響対象**: Web Image Monitorを実装する複数のリコー製レーザープリンタ／MFP。対象機種一覧はリコー公式（ricoh.com）の機種別告知を要確認。
- **対策**: リコーが対象機種向けに更新ファームウェアを順次提供（提供状況は機種により異なるため公式情報を確認）。

### メカニズム（攻撃が成立する仕組み）

※脆弱性クラスの一般原理を概念レベルで説明する。具体的なペイロードは記載しない。

1. **信頼境界と出力の責務**: Web Image Monitorは、URLパラメータやフォーム入力など外部由来の値をHTTPレスポンス（HTML）に反映する。Webアプリの基本要件は「外部入力を出力する際に、HTML文脈に応じてエスケープ（無害化）する」こと。本件はこの出力エンコーディングが不十分だった。
2. **反射ポイントの存在**: ユーザーが送った入力（典型的にはGETパラメータ）が、その応答ページ内にほぼそのまま“反射”して埋め込まれる箇所がある。ここにスクリプトとして解釈される文字列を混ぜ込めると、ブラウザがそれをコードとして実行する。
3. **デリバリ（攻撃の届け方）**: 反射型XSSは保存されないため、攻撃者は「悪性パラメータを含むURL」を作り、標的（管理者）にメール・チャット等で踏ませる必要がある。標的のブラウザがそのURLを開いた瞬間に、標的の権限・セッションのコンテキストでスクリプトが実行される。
4. **悪用プリミティブ**: 実行されたスクリプトは、(a) 管理画面のセッション情報の窃取、(b) 管理者になりすました設定変更リクエストの送出（CSRF的悪用）、(c) 画面の改ざん・フィッシング、といった操作を行いうる。
5. **被害の最終形**: 複合機の設定改ざん（スキャン転送先の書き換え等による情報の横取り）、認証情報の窃取、内部ネットワーク偵察の足場化。プリンタ／MFPは“ネットワーク内の見落とされがちな機器”であり、侵入の踏み台になりやすい。

クラスとしては「出力エンコーディング欠如による反射型XSS」。サーバ自体のRCEではないが、管理者を狙うことで実害（設定改ざん・情報窃取）に繋がる点が要注意。

### 検知・暫定緩和・恒久対策

- **恒久対策**: 対象機種の更新ファームウェアを適用（リコー公式の機種別告知を確認）。
- **暫定緩和（パッチ前）**: (1) Web Image Monitorを**業務LANから隔離**し、管理セグメント／許可端末からのみアクセス可能にする、(2) 不要なら外部・一般ユーザーからの管理画面到達を遮断、(3) 管理者に対し**出所不明のリンクを踏ませない**運用周知（本件はリンク誘導が起点）、(4) 管理操作は専用端末から実施。
- **検知の着眼点**: 管理画面URLへの不自然なパラメータ付きアクセス、短時間での設定変更（スキャン転送先・宛先帳の変更）、想定外のIPからの管理アクセス。

## 📖 用語解説（このニュースとのつながり）

- **Web Image Monitor**: リコー製プリンタ／複合機のWeb管理画面。今回の脆弱性の舞台。
- **反射型XSS（Reflected XSS）**: 送った入力がその場の応答にそのまま反映され、スクリプトとして実行される脆弱性。被害発動には標的が細工リンクを踏む必要がある。
- **出力エンコーディング（エスケープ）**: 外部入力を画面に出す際、`<` や `"` などを無害な表記に変換してコード実行を防ぐ基本対策。本件の不足点。
- **JPCERT/CC調整**: 報告者・ベンダ・JPCERT/CCが連携し、修正準備を整えてから公表する国内の脆弱性調整プロセス。
- **MFP（複合機）**: 印刷・コピー・スキャン・FAXを兼ねる機器。機微データを扱い、社内ネットワークの踏み台にもなり得る。

## 影響範囲 / 推奨対応

Web Image Monitorを搭載するリコー製プリンタ／MFPを利用する組織が対象。対象機種は公式告知で確認し、提供され次第ファームウェアを更新する。更新までは管理画面のアクセス制限と、管理者へのリンク誘導（フィッシング）注意喚起で緩和する。国内オフィスでの設置台数が多く、組織内に複数潜在しがちな点に留意。
