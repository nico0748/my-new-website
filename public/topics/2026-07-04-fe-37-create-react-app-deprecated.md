# Create React App 非推奨 — Vite への移行が定石に

> トピック: 開発トピック｜出典: openpr｜種別: frontend

## 一行サマリー
長年 React の標準スターターだった Create React App（CRA）が 2025-02 に公式に非推奨となり、新規プロジェクトは Vite や Next.js などのフレームワークへ移行するのが定石になった。

## 🔰 初学者向け（何が・なぜ重要か）
Create React App（CRA）は「React プロジェクトを1コマンドで始められる」公式の雛形ツールで、長い間、初心者が最初に触れる入口でした。ところが CRA は内部のビルド設定が古く、遅く、メンテナンスも滞り、現代の開発では足かせになっていました。

React チームは 2025 年に「CRA はもう推奨しません」と正式に宣言しました。たとえるなら、長年使われた入門用の自動車教習コースが「今の道路事情に合わない」として閉鎖され、新しいコース（Vite やフレームワーク）へ案内が切り替わったようなもの。これから React を始める人は CRA を使うべきではなく、高速な Vite でセットアップするか、ルーティングやサーバー機能まで欲しいなら Next.js のようなフレームワークを選ぶのが標準になりました。

## 🧠 専門的解説（技術詳細・設計・使いどころ）
### 技術的詳細（何が変わったか）
2025-02、React 公式ドキュメントが CRA を非推奨とし、代替として Vite・Parcel・Next.js・React Router（framework mode）などを案内した。CRA は webpack/Babel ベースで起動・ビルドが遅く、RSC・最新バンドラ機能への追随も難しかった。新規は Vite ベースのテンプレート（`create-vite` 等）か、要件次第でメタフレームワークを選ぶ流れが定着。

### 仕組み / 背景（なぜこう設計されたか）
CRA は「設定を隠蔽して手軽に始める」思想だったが、その隠蔽ゆえに拡張が難しく、eject すると複雑な設定が露出した。エコシステムが esbuild/SWC/Rolldown といった高速ネイティブツールへ移り、CRA の webpack 前提が時代遅れになった。React 側も「単なるライブラリではなくフレームワーク上での利用」を推奨する方針に転じ、CRA の役割は終わったと判断された。トレードオフは、既存 CRA 資産の移行コスト。

### 実務での使いどころ・移行の勘所
新規は迷わず Vite（SPA 用途）か Next.js/React Router framework mode（SSR・ルーティング込み）を選ぶ。既存 CRA からの移行は、`react-scripts` を外して Vite 設定へ置換し、環境変数（`REACT_APP_` → `VITE_`）やエントリ HTML の扱いを調整する。テストは Jest から Vitest への移行も併せて検討するとよい。

## 📖 用語解説（このトピックとのつながり）
- **Create React App（CRA）**: 旧標準の React スターター。非推奨化された当事者。
- **Vite**: 高速な後継スターター/ビルドツール。SPA 移行の第一候補。
- **メタフレームワーク**: Next.js 等、ルーティング/SSR まで含む上位枠組み。CRA の別の代替。
- **eject**: CRA の隠蔽設定を露出させる操作。拡張の難しさの象徴。
- **webpack**: CRA の内部バンドラ。ネイティブ高速ツールに置き換えられつつある。

## 影響範囲 / 推奨アクション
React 開発者全般が対象。新規は CRA を選ばず Vite かフレームワークを採用する。既存 CRA プロジェクトは、保守性・ビルド速度改善のため Vite への移行を計画し、環境変数・エントリ・テスト基盤の読み替えを進める。
