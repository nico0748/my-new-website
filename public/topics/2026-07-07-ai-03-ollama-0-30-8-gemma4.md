# Ollama v0.30.8 — Gemma 4 QAT 対応と Apple Silicon で約90%高速化、ローカル LLM が「コーディングエージェントの土台」へ

> トピック: Ollama v0.30.x 系の6月アップデート｜出典: Ollama 公式リリース（GitHub）｜種別: ローカル推論ランタイムの機能リリース

## 一行サマリー

ローカル LLM 実行ツール Ollama は v0.30.8（2026年6月12日）までの一連の更新で Gemma 4 の QAT（量子化認識学習）ウェイトに対応し、マルチトークン予測と自動チューニングにより Apple Silicon での Gemma 4 トークン生成を最大約90%高速化、さらに Claude Code や opencode の自動インストール連携などエージェント用途への統合を強めた。

## 🔰 初学者向け（何が・なぜ重要か）

Ollama は「自分のパソコンの中で AI モデルを動かす」ための定番ツールです。クラウドの AI と違い、データが外に出ない・通信費がかからない・オフラインでも動くのが利点で、`ollama run gemma4` のような1行でモデルを起動できる手軽さで広く使われています。

今回の更新には2つの見どころがあります。1つめは速度です。「マルチトークン予測」という技術で、AI が文章を作るとき1語ずつではなく数語まとめて予測し、まとめて検証する方式を取り入れました。書類を1文字ずつ清書する代わりに、下書きを数文字ずつ写して間違いだけ直すイメージで、Mac（Apple Silicon）での生成速度が9割近く速くなったとされています。

2つめは「QAT」対応です。モデルを小さく圧縮（量子化）すると普通は賢さが少し落ちますが、QAT は「最初から圧縮される前提で訓練しておく」ことで、小さいのに賢さをあまり失わないモデルを作る手法です。Google の Gemma 4 の QAT 版が使えるようになり、家庭の GPU やノート PC でも実用的な品質が出しやすくなりました。ローカル AI は「試す玩具」から「開発の道具」に近づいています。

## 🧠 専門的解説（技術詳細・設計・使いどころ）

### 技術的詳細（何が変わったか）

v0.30.x 系（6月の点リリース群、最新 v0.30.8 = 6月12日）の主な変更:
- **Gemma 4 QAT ウェイト対応**（6月5日の点リリース）。
- **Apple Silicon での Gemma 4 高速化**: マルチトークン予測と自動チューニングにより、トークン生成が最大約90%高速（公称値。ハードウェア・量子化条件依存のため実測は要確認）。MLX / llama.cpp エンジンの更新も同梱。
- **プロンプト/KV キャッシュ再利用の改善**: 同一プレフィックスの再計算を削減。
- **エージェント統合**: thinking 能力の自動検出、**Claude Code / opencode の自動インストール連携**、Codex 系モデルのドリフト検出改善。
- プラットフォーム修正: Windows の設定パス、Vulkan / CUDA / MLX、生成処理まわりの修正。
- Hermes Desktop の追加（6月7日）等、デスクトップ向けの動きも並行。

### 仕組み / 背景（なぜこう設計されたか）

**マルチトークン予測（MTP）/投機的デコード**の原理: 自己回帰生成は1トークンごとに巨大な行列計算を直列実行するためメモリ帯域律速になる。複数トークンをまとめて予測し、本体モデルで一括検証（外れたら破棄して通常生成に戻る）すれば、1回のフォワードパスで複数トークンが確定し、帯域あたりのスループットが上がる。Apple Silicon はユニファイドメモリで帯域が比較的潤沢だが GPU 演算器数は限られるため、「検証の並列化で演算を埋める」この手法と相性が良い。代償は、予測が外れやすいテキスト（コードのような低エントロピー部分は当たりやすく、創作は外れやすい）での効果低下と実装複雑性。

**QAT** は学習時に量子化誤差を順伝播に織り込み、重みが「量子化後も性能が残る場所」に収束するよう訓練する手法。事後量子化（PTQ）比で低ビット（4bit 級）でも劣化が小さい。ベンダー（Google）自身が QAT 版を配布する流れは、「量子化はコミュニティの後加工」から「公式のリリース形態」への転換を意味し、ローカル実行を一級市民として扱う姿勢の表れ。

**Claude Code / opencode 連携**は、Ollama が「チャット UI の裏方」から「コーディングエージェントのモデル供給層」へ軸足を移していることを示す。エージェントは同じ文脈を繰り返し送るため、プロンプト/KV キャッシュ再利用の改善もこの用途に直結する。

### 実務での使いどころ・移行の勘所

- **機密コードベースでのコーディング支援**: クラウドにコードを送れない環境で、Ollama + ローカルモデル + Claude Code/opencode 系 CLI の組み合わせが現実的な選択肢になってきた。ただしフロンティアモデルとの品質差は残るため、補完・定型作業中心と割り切る。
- **RAG のローカル実行**: 埋め込み生成と小型 LLM をローカルで回す構成（この vault の inbox 処理のような用途）は、QAT モデルで品質・速度の両立がしやすくなった。
- モデル選定: VRAM/メモリ 16GB 級なら Gemma 4 QAT の中型、8GB 級は小型量子化版が目安（具体的なサイズ別推奨は要確認）。
- 運用注意: Ollama は既定でローカルポートに API を開く。社内展開時は待ち受けアドレスと認証（リバースプロキシ）を必ず設計に入れる。

## 📖 用語解説（このトピックとのつながり）

- **量子化 / QAT / PTQ**: 重みの低ビット化。QAT は学習時から織り込む方式で、今回 Gemma 4 の公式 QAT 版が Ollama で使える。
- **マルチトークン予測**: 複数トークンの同時予測＋検証で生成を高速化する技術。Apple Silicon での約90%高速化の主因。
- **KV キャッシュ**: 過去トークンの注意計算結果の保存。再利用改善はエージェントの長い反復対話で効く。
- **MLX**: Apple Silicon 向け ML フレームワーク。Ollama のバックエンドの一つとして更新された。
- **opencode / Claude Code**: 端末で動くコーディングエージェント。Ollama が自動インストール連携を持ち、ローカルモデルを供給する。

## 影響範囲 / 推奨アクション

- ローカル LLM 運用者: v0.30.8 へ更新し、Gemma 4 系は QAT 版への差し替えを検討。
- Mac ユーザー: 速度改善が大きいため、旧版で「遅くて諦めた」ワークフローの再評価を推奨。
