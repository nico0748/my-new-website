# Apple MLX の分散推論 — Thunderbolt RDMA で Mac をクラスタ化、M5 の Neural Accelerator も活用

> トピック: Apple MLX の分散推論／M5 対応｜出典: Apple Developer（WWDC26）／Apple ML Research｜種別: ローカル AI 実行基盤の進化

## 一行サマリー

Apple のオンデバイス機械学習フレームワーク MLX が、macOS 26.2+ の Thunderbolt RDMA（JACCL バックエンド）で複数の Mac をつなぎ分散推論できるようになった。4ノードの Thunderbolt クラスタで最大3倍の高速化と、単機のメモリを超えるモデルのホストが可能。さらに新しい M5 チップの Neural Accelerator（行列演算専用ユニット）を活用し、Apple Silicon 上のローカル推論を一段引き上げる（WWDC26 で発表）。

## 🔰 初学者向け（何が・なぜ重要か）

MLX は「Mac の中だけで（クラウドに送らず）AI モデルを動かす」ための Apple 製の道具です。手元でモデルを動かせれば、プライバシーが守れ、通信費もかからず、オフラインでも使えます。ただ大きなモデルは、1台の Mac のメモリに収まりきらない、という壁がありました。

今回の目玉は2つ。1つは **複数の Mac を Thunderbolt ケーブルでつないで"1つの大きなマシン"のように使う**こと。高速な接続（Thunderbolt RDMA）で Mac 同士がメモリを分け合い、4台つなげば単機では載らない巨大モデルも動かせて、しかも最大3倍速くなります。もう1つは、新しい **M5 チップに載った「Neural Accelerator」**という、AI 計算（行列のかけ算）を専門に速くする回路を MLX が使えるようになったこと。要するに「手元の Mac だけで、より大きく・より速く AI を動かせる」時代に一歩進んだ、という話です。ローカル AI を重視する開発者にとって実用的な前進です。

## 🧠 専門的解説（技術詳細・設計・使いどころ）

### 技術的詳細（何が変わったか）

- **分散推論（MLX v0.30.1〜）**: JACCL バックエンド経由で **Thunderbolt 上の RDMA**（macOS 26.2+）を使い、複数 Mac をクラスタ化。4ノードの Thunderbolt クラスタで最大3倍の高速化、単機メモリを超えるモデルのホストが可能。
- **M5 の Neural Accelerator 対応**: M5 GPU 内蔵の行列積専用ユニットを MLX が活用し、推論を高速化。
- **Ollama × MLX**: Ollama 0.19（2026年3月31日）が MLX を統合、Apple Silicon で高速化。
- **WWDC26**: macOS 26.2 で Thunderbolt RDMA をサポート、MLX の分散推論で4ノード最大3倍を提示。ローカルエージェントを Mac 上で走らせるセッションも。
- MLX は2023年12月公開の Apple Silicon 向け配列演算ライブラリ。数値は一次ソースで要確認。

### 仕組み / 背景（なぜこう設計されたか）

Apple Silicon の強みは **ユニファイドメモリ**——CPU/GPU/Neural Engine が同じ大容量メモリを共有し、大きなモデルの重みをまるごと載せやすい。MLX はこの統合メモリを前提に、コピーを避けた効率的な配列演算を提供する。だが単機のメモリ上限（数十〜百数十 GB 級）を超えるモデルは載らない。そこで「複数機のメモリを束ねる」分散推論が要る。ここで効くのが **RDMA（Remote Direct Memory Access）**——CPU を介さずマシン間で直接メモリを読み書きする方式で、通信のオーバーヘッドとレイテンシを大幅に下げる。Thunderbolt を RDMA の物理層に使うことで、専用の高価なネットワーク機器なしに、Mac 数台を低レイテンシで結べる。これで「メモリ容量の壁」を横方向のスケールで越える。

M5 の Neural Accelerator は、Transformer 推論の律速である行列積（GEMM）を専用回路でこなす。汎用 GPU コアより電力効率・スループットで有利で、MLX がこれを叩けるようになると同じ Mac でより大きなモデル・より速い生成が可能になる。トレードオフは、分散推論のスケーリングは通信帯域と並列化効率に縛られ、ノードを増やすほど線形には伸びない（4ノードで3倍＝効率は逓減）点、そして最新機能が macOS 26.2+ / M5 という新しいハード・OS を要求する点。クラウド GPU に対する優位はコスト・プライバシー・オフライン性で、劣位は最上位モデルの生スループット。ローカル AI の現実解を「単機」から「手元の小クラスタ」へ押し広げたのが今回の意味である。

### 実務での使いどころ・移行の勘所

- プライバシー要件が厳しい／オフライン前提／API コストを避けたい用途で、Mac 上のローカル推論を実運用レベルに引き上げたいケースに向く。
- 単機に載らない中〜大型モデルを、手元の複数 Mac（Thunderbolt 接続）で動かす選択肢が現実に。開発機の遊休リソースをクラスタ化する使い方も。
- 手軽に始めるなら Ollama（MLX 統合）経由でモデルをローカル実行。より作り込むなら MLX で直接、分散・量子化を制御。
- 分散のスケーリングは逓減する。ノード追加のコスト対効果と、macOS 26.2+ / M5 世代の要件（新しいハード前提）を見込む。生スループット最優先ならクラウド GPU（vLLM/SGLang）と使い分け。

## 📖 用語解説（このトピックとのつながり）

- **MLX**: Apple Silicon 向けのオンデバイス機械学習フレームワーク。ユニファイドメモリを活かす。
- **ユニファイドメモリ**: CPU/GPU/Neural Engine が共有する大容量メモリ。大きなモデルを載せやすい Apple Silicon の要。
- **RDMA（Thunderbolt RDMA）**: CPU を介さずマシン間で直接メモリを読み書きする通信。分散推論の低レイテンシ化の核。
- **Neural Accelerator（M5）**: M5 GPU 内の行列積専用ユニット。Transformer 推論の律速を高速化し、MLX が活用。

## 影響範囲 / 推奨アクション

- ローカル AI 重視: Mac 単機／小クラスタでの推論を評価。Ollama(MLX) で手軽に、MLX 直叩きで作り込み。
- 大型モデル: Thunderbolt クラスタで単機メモリ超えを検証。スケーリング逓減とハード要件を見込む。
- 使い分け: 生スループット最優先はクラウド GPU、プライバシー・コスト・オフラインはローカル MLX。
