# Ivanti Sentry 認証バイパス CVE-2026-10523（未認証で管理者作成・最大級チェーン）

> トピック: 脆弱性ニュース ｜ 出典: BleepingComputer / Help Net Security / Rapid7 / Ivanti Advisory ｜ 種別: 認証バイパス（CWE-288/CWE-306）

## 一行サマリー

モバイル/アプリのゲートウェイ製品 Ivanti Sentry に、未認証の攻撃者が任意の管理者アカウントを作成できる認証バイパス CVE-2026-10523（CVSS 9.9）が判明。併存する RCE 系 CVE-2026-10520（CVSS 10.0、KEV登録・実環境悪用）と組み合わせると、認証なしでアプライアンスを完全掌握できる。

## 🔰 初学者向け（何が・なぜ問題か）

Ivanti Sentry は、社員のスマホやアプリが社内システムにアクセスする際の「関所（ゲートウェイ）」のような製品です。社外と社内の境界に置かれ、通信を取り次ぎ、本人確認の鍵も預かっています。境界にあるからこそ、ここを破られると一気に内側へ入られてしまいます。

今回の欠陥は、この関所の**本人確認の仕組みそのものを素通りできてしまう**もの（認証バイパス）です。攻撃者はログインせずに「自分を管理者として登録しろ」と命じ、勝手に管理者アカウントを作れます。一度管理者になれば設定も中身も自由に触れます。

さらに悪いことに、同じ製品にはもう一つ「管理権限があれば OS のコマンドを root（最高権限）で実行できる」欠陥（CVE-2026-10520）もありました。攻撃者は「①素通りで管理者になる → ②最高権限でコマンド実行」という二段ロケットで、関所を丸ごと乗っ取れます。実際にこの悪用は始まっており、米 CISA は連邦機関に**異例の3日以内パッチ**を命じました。

## 🧠 専門的解説（技術詳細・メカニズム・検知対応）

### 技術的詳細（種別・前提条件）

- **CVE**: CVE-2026-10523（認証バイパス）。連鎖相手は CVE-2026-10520（OS コマンドインジェクション → root RCE, CVSS 10.0）。
- **CVSS**: CVE-2026-10523 = 9.9（Critical）/ CVE-2026-10520 = 10.0（Critical）。
- **種別**: 認証バイパス（CWE-288 代替経路による認証回避 / CWE-306 認証の欠如の類型）。連鎖相手は OS コマンドインジェクション（CWE-78）。
- **影響バージョン**: Ivanti Sentry 10.5.1 / 10.6.1 / 10.7.0 およびそれ以前すべて。
- **修正版**: R10.5.2 / R10.6.2 / R10.7.1。
- **悪用状況**: CVE-2026-10520 は2026年6月11日に CISA KEV 登録。watchTowr の公開 PoC を用いた大量の悪用試行を Shadowserver が観測、確認19台中少なくとも2台がバックドア設置済み。CVE-2026-10523 単独でも完全乗っ取りが可能。

### メカニズム（攻撃が成立する仕組み）

本件は「認証ロジックの欠陥（broken authentication）」と「特権ヘルパへの入力検証不備（command injection）」が連鎖して最大被害に至る典型例。概念レベルで分解する。

1. **信頼境界の取り違え（10523）**: 管理機能の一部が、リクエストの出自を正しく認証せずに処理する経路を持っていた。攻撃者はこの代替経路を通じて、本来「認証済み管理者のみ」が呼べるはずのアカウント管理機能に到達する。これが「未認証 → 管理者アカウント生成」というプリミティブを与える。
2. **権限の確立**: 生成した管理者アカウントで正規の管理 API にログインでき、以降は「正当な管理者」として扱われる。アクセス制御の前提（＝この操作者は信頼できる）が崩れる。
3. **被害への橋渡し（10520との連鎖）**: 管理権限が前提のコマンド実行機能で、ユーザ入力がシェルに渡る箇所のサニタイズが不十分だった。攻撃者はここに OS コマンドを注入し、アプライアンス上で **root 権限の任意コード実行** を得る。①で権限を作り②で root を取る、という完全チェーンが成立する。
4. **片方単独でも致命的**: 10523 だけでも管理者掌握＝全設定改変・認証情報窃取が可能。10520 だけでも（何らか管理アクセスを得れば）root RCE。境界アプライアンスゆえ、掌握は内部ネットワークへのピボット起点になる。

一般原理として、(a) 認証は「すべての特権経路」を漏れなく覆う必要があり、代替経路が1つでも抜けると全体が破れること、(b) 管理者前提の機能でも入力検証を省略してはならないこと（多層防御）を示す事例。武器化の具体的手順・PoC は記載しない。

### 検知・暫定緩和・恒久対策

- **検知の着眼点**: 想定外の管理者アカウント新規作成、未知の IP からの管理 API 呼び出し、管理機能経由のプロセス生成（シェル/コマンド実行）、watchTowr PoC に対応する既知 IoC、バックドア・Webシェルの設置痕跡。
- **暫定緩和**: 管理インターフェイスのインターネット露出を遮断し、管理アクセスを VPN/許可 IP に限定。露出していた期間がある場合は侵害前提で調査。
- **恒久対策**: R10.5.2 / R10.6.2 / R10.7.1 へ即時更新。更新後、既存アカウント・設定・ログを精査し、不正アカウントやバックドアを除去。必要に応じ資格情報をローテーション。
- **運用示唆**: 境界アプライアンスは「パッチ＋露出最小化＋侵害想定の事後調査」をセットで実施。CISA の3日パッチ指定は緊急度の高さの表れ。

## 📖 用語解説（このニュースとのつながり）

- **認証バイパス（CWE-288/CWE-306）**: 本人確認を経ずに保護機能を使えてしまう欠陥。本件 10523 の核心で、未認証から管理者作成を可能にする。
- **OS コマンドインジェクション（CWE-78）**: アプリがユーザ入力を OS コマンドの一部として実行してしまう欠陥。連鎖相手 10520 が該当し、root RCE の出口になる。
- **エクスプロイトチェーン**: 複数の脆弱性を順に繋いで最終目的を達する攻撃。本件は「認証バイパス→管理者→root RCE」という教科書的チェーン。
- **KEV（Known Exploited Vulnerabilities）**: CISA が「実環境で悪用が確認された」と認定する一覧。10520 が登録され、パッチ期限が課された＝今まさに狙われている指標。
- **境界アプライアンス**: 社外と社内の境目に置く中継機器。侵害されると内部侵入の足場になるため優先的に守る必要がある。

## 影響範囲 / 推奨対応

Ivanti Sentry を境界に設置している組織が対象。とくに管理面をインターネットに露出している環境は最優先。直ちに固定版（R10.5.2 / R10.6.2 / R10.7.1）へ更新し、(1) 不正管理者アカウントの有無、(2) 想定外のコマンド実行・バックドア、(3) 横展開の痕跡を調査する。露出期間があった場合は侵害を前提とした対応（資格情報ローテーション含む）を行う。
