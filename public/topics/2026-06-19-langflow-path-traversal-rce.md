# Langflow CVE-2026-5027 — 認証なしでファイル書き込み、AI開発基盤がRCEで悪用中

> トピック: 脆弱性ニュース｜出典: The Hacker News / Orca Security / SecurityWeek｜種別: パストラバーサル→未認証RCE

## 一行サマリー

AIワークフロー構築ツール Langflow のファイルアップロード機能にパストラバーサル（CVSS 8.8）があり、認証なしでサーバ上の任意の場所にファイルを書き込み、cronジョブ注入などで遠隔コード実行に至る。パッチ（v1.9.0）公開後も未更新インスタンスが実環境で悪用されているため、即時アップデートと侵害調査が必要。

## 🔰 初学者向け（何が・なぜ問題か）

Langflow は、AI（生成AI）の処理の流れを「部品をつなぐ」感覚でビジュアルに組み立てられる開発ツールです。社内のAIアプリ作りに使われ、サーバ上で動いていることが多いソフトです。

このツールには「ファイルをアップロードする」窓口があります。本来は決められた保管庫にだけファイルを置くはずですが、攻撃者がファイル名に「`../`（一つ上のフォルダへ戻る合言葉）」を仕込むと、保管庫の外、サーバのどこにでもファイルを置けてしまいます。これがパストラバーサル（道をたどって本来行けない場所へ抜ける）という欠陥です。

例えるなら、宅配ボックスにしか入れられないはずの配達員が、合言葉ひとつで家の鍵束や家電の配線盤の場所にまで荷物を置けてしまうようなものです。攻撃者は「定期的に自動実行される設定ファイル（cron）」を上書きする形で、自分のプログラムをサーバに実行させます。

さらに悪いことに、Langflow は初期設定で「ログイン不要で自動的に入れる」状態のため、攻撃者はパスワードすら要りません。リクエスト1回で侵入できます。修正版は出ていますが、更新していないサーバが実際に狙われています。

## 🧠 専門的解説（技術詳細・メカニズム・検知対応）

### 技術的詳細（種別・前提条件）

- **CVE**: CVE-2026-5027 / **CVSS**: 8.8（High）
- **CWE**: CWE-22（Improper Limitation of a Pathname / Path Traversal）→ RCE
- **影響コンポーネント**: `POST /api/v2/files` エンドポイントの `upload_user_file()` 関数
- **根本原因**: multipart フォームデータの `filename` パラメータを正規化・検証せずにファイル保存先パスへ連結。`../` シーケンスでサンドボックス外へエスケープ可能。
- **影響バージョン**: Langflow ≤ 1.8.4 / **修正版**: 1.9.0
- **悪用状況**: VulnCheck が実環境での悪用を確認（ハニーポットでテストファイル投下を観測）。公開PoCあり。Tenable が2026年1〜2月に責任ある開示を試み、2026-03-27 に公表。
- **前提条件**: Langflow が既定の「未認証オートログイン（自動ログイン）」で公開されている場合、認証情報不要。単一リクエストで有効なセッショントークンを取得し悪用可能。

### メカニズム（攻撃が成立する仕組み）

※以下は脆弱性クラスの一般原理に基づく概念説明。武器化手順・PoC・正確なパスは記載しない。

1. **入力検証の欠落（信頼境界の誤り）**: アップロード処理はクライアント由来の `filename` を「信頼できる文字列」として扱い、サニタイズせず保存先パスに連結する。本来、外部入力は信頼境界の外側にあり検証が必須。
2. **パス正規化の不在**: `../` を含む相対パスがそのまま解決され、想定の保管ディレクトリ（サンドボックス）の外へ書き込み先が逸脱する。これが「任意ファイル書き込み」プリミティブを攻撃者に与える。
3. **認証境界の事実上の無効化**: 既定のオートログインにより、未認証の攻撃者がリクエスト1回でセッションを得る。アクセス制御の前提が崩れ、書き込みプリミティブが匿名で行使可能になる。
4. **書き込みから実行への橋渡し**: 任意書き込みプリミティブを、定期実行される設定（cron ジョブ）の注入や既存設定ファイルの上書きに用いることで、サーバ側で攻撃者コードが実行され RCE が成立する。
   - 脆弱性クラスの一般原理: 「任意ファイル書き込み」は単体でも危険だが、自動実行される場所（cron、起動スクリプト、Webルート配下のスクリプト）に到達できると確実に RCE へ昇格する。書き込み先の選択肢を制限できていないことが致命的。

### 検知・暫定緩和・恒久対策

- **検知の着眼点**: `/api/v2/files` への異常な POST（`filename` に `../` やエンコードされたトラバーサル列）、Webアクセスログ上の連続したアップロード試行、cron/設定ファイルの予期せぬ変更タイムスタンプ、Langflow プロセスからの不審な子プロセス生成。
- **暫定緩和**: 即時更新できない場合は、Langflow をインターネットから隔離（VPN/IP制限の背後へ）、認証必須化（オートログイン無効化、`LANGFLOW_AUTO_LOGIN=false` 相当）、WAF でトラバーサルパターンを遮断。
- **恒久対策**: v1.9.0 以降へ更新。公開済みインスタンスは「パッチ＋侵害調査」をセットで実施し、cron・設定ファイル・追加されたユーザ/鍵を点検。

## 📖 用語解説（このニュースとのつながり）

- **Langflow**: 生成AIのフロー（処理連鎖）を視覚的に組むツール。サーバ常駐型のため、穴があるとサーバ全体が危険にさらされる。
- **パストラバーサル（CWE-22）**: `../` 等で本来のディレクトリ外へアクセスする攻撃。本件では「任意の場所にファイルを書ける」起点。
- **任意ファイル書き込みプリミティブ**: 攻撃者が場所を選んでファイルを書ける能力。cron 注入などと組み合わせて RCE に直結する。
- **オートログイン**: ログインなしで利用できる初期設定。本件では認証の壁を実質ゼロにし、未認証悪用を可能にした。
- **cronジョブ注入**: 定期実行されるスケジュールに攻撃コードを仕込む手口。書き込みを実行へ橋渡しする典型経路。

## 影響範囲 / 推奨対応

影響は、インターネットや社内ネットに公開された Langflow ≤ 1.8.4 のインスタンス全般。とくに既定設定（オートログイン有効・直接公開）の環境はリスクが高い。推奨対応は、ただちに v1.9.0 以降へ更新し、オートログインを無効化、外部公開を見直すこと。公開実績のあるインスタンスは既に侵害された前提で、cron・設定ファイル・新規アカウント・鍵・不審プロセスを調査する。
