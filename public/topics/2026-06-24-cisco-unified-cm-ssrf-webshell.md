# Cisco Unified Communications Manager の未認証SSRF（CVE-2026-20230）が悪用中 — WebDialer経由でWebシェル設置・root昇格の恐れ

> トピック: 脆弱性ニュース ｜ 出典: Cisco / BleepingComputer / Help Net Security / SecurityWeek ｜ 種別: 不適切な入力検証（CWE-20）に起因するサーバーサイドリクエストフォージェリ（SSRF, CWE-918）→ 任意ファイル書き込み → root昇格

## 一行サマリー

企業の電話・ユニファイドコミュニケーション基盤であるCisco Unified Communications Manager（Unified CM）とSession Management Edition（SME）に、未認証で悪用できるSSRF脆弱性CVE-2026-20230（CVSS 8.6、Ciscoは"Critical"扱い）が見つかり、6月23日の週末から実環境で悪用が始まった。攻撃者はWebDialerコンポーネントの`file://`処理を悪用して基盤OSへ任意ファイルを書き込み、JSP Webシェルを設置してroot昇格を狙う。修正版14SU6 / 15SU5への更新が急務。

## 🔰 初学者向け（何が・なぜ問題か）

Cisco Unified CMは、社内の電話・ビデオ会議・内線などを束ねる「企業の電話交換機（PBX）のソフト版」です。多くの企業の音声インフラの中心にあり、ここが止まると業務連絡が麻痺し、乗っ取られれば通話の盗聴や内部侵入の足場にされます。

今回の弱点は **SSRF（サーバーサイドリクエストフォージェリ）** と呼ばれるタイプです。たとえるなら、会社の「お使い係（サーバー）」に外部から偽の依頼書を渡し、本来は社外の人が触れない**社内の金庫やファイル棚**へお使い係自身に行かせる手口です。お使い係は社内の信頼された存在なので、外部の人間が直接行けない場所にもアクセスできてしまう——その権限を攻撃者が"代理で"乗っ取るわけです。

さらに今回は、このお使い係に`file://`という特殊な依頼を出すことで、サーバー上の好きな場所に**ファイルを書き込ませる**ことができます。攻撃者はここに「Webシェル」という遠隔操作用の小さなプログラムを置き、後からそれを呼び出してサーバーを自由に操ります。最終的には機器の最高権限（root）への昇格まで狙えるため、認証もパスワードも不要で企業の音声基盤が乗っ取られうる、という深刻な事態です。

## 🧠 専門的解説（技術詳細・メカニズム・検知対応）

### 技術的詳細（種別・前提条件）

- **CVSS**: 8.6（High）だが、Ciscoは昇格可能性を踏まえ **Critical** として扱う。未認証・ネットワーク経由で悪用可能。
- **CWE分類**: CWE-918（SSRF）。根本は特定HTTPリクエストに対する**不適切な入力検証（CWE-20）**。
- **前提条件**: **Cisco WebDialer Web Serviceが有効**であること。攻撃者は管理画面のHTTPエンドポイントへ到達できればよい。
- **影響製品**: Cisco Unified CM および Unified CM SME。**修正版は 14SU6 と 15SU5**。
- **悪用状況**: **実環境で悪用中**。6月23日の週末、単一IPから`file://`ペイロードでデバイス上にファイルを作成する活動が観測。Tor経由の自動スイープでWebシェルを投下し、不正なApache Axisサービスを立ててJSPシェルを書き込む連鎖が報告されている。公開エクスプロイトコードも存在。

### メカニズム（攻撃が成立する仕組み）

以下は概念レベルの説明であり、武器化可能な手順・PoC・正確なペイロード等は含めない。

1. **信頼境界の所在**: Unified CMのWeb管理プレーンは、外部からのHTTPリクエストを受ける。WebDialer（クリック発信機能）はユーザー指定のURLを取り扱うため、「外部入力のURLをサーバー自身がどう解釈するか」が信頼境界となる。
2. **入力検証の不備（SSRFの成立）**: WebDialerがユーザー供給のURL／リクエストを十分に検証せず処理するため、攻撃者はサーバー自身に任意の宛先へリクエストを発行させられる。これがSSRFの本体で、攻撃者は「サーバーの内部ネットワーク権限」を間接的に握る。
3. **スキーム悪用による任意ファイル書き込み**: 宛先に`file://`スキームを与えることで、HTTP取得ではなく**ローカルファイルシステムへの書き込み操作**を誘発できる。攻撃者は基盤OS上の任意パスにファイル（JSP Webシェル等）を作成するプリミティブを獲得する。
4. **Webシェルによる遠隔操作と昇格**: 書き込まれたJSPシェルは、後続のHTTPアクセスで攻撃者の任意コマンドを実行する遠隔操作点になる。観測された攻撃では不正なApache Axisサービスを介してシェルを動かす。ここから設定・サービスの権限を踏み台に **root昇格** を狙う。
5. **未認証で完結する致命性**: 一連が認証前に成立するため、攻撃者は資格情報不要。企業の音声基盤という"内部の中心"に外部から足場を作れる点が極めて危険。

脆弱性クラスとしては、「ユーザー制御URLを検証せずサーバー側で処理するSSRF」が「危険スキーム（file://）による任意書き込み」へ橋渡しされ、最終的にWebシェル経由のコード実行へ至る、SSRF×ファイル書き込みの古典的な合わせ技である。

### 検知・暫定緩和・恒久対策

- **検知の着眼点**: WebDialerエンドポイントへの異常なHTTPリクエスト（特に`file://`等の異常スキームを含む要求）、Webルート配下への**新規JSPファイル**の出現、不正なApache Axisサービスやプロセスの起動、外部・Tor出口IPからの管理画面アクセス。
- **暫定緩和（パッチ前）**: **WebDialer Web Serviceを無効化**（業務影響を確認のうえ）。管理／Webインターフェースのインターネット露出を遮断し、アクセス元を管理セグメントに限定。
- **恒久対策**: **Unified CM / SME を 14SU6 または 15SU5 へアップデート**。露出していた機器はWebシェル設置の有無を点検し、侵害前提でフォレンジック実施。
- **運用示唆**: SSRFは「サーバーの信頼」を逆手に取る攻撃。外部到達可能なコンポーネントでのURL／スキーム検証と、不要機能（WebDialer等）の無効化を恒常原則とする。

## 📖 用語解説（このニュースとのつながり）

- **Unified CM / SME**: Ciscoの企業向け音声・UC基盤（ソフトPBX）。意味＝「会社の電話交換機」。本件＝ここが侵入の足場にされる。
- **SSRF（CWE-918）**: サーバーに代理でリクエストを出させる攻撃。本件＝WebDialerのURL処理が悪用される起点。
- **WebDialer**: クリック発信を担うコンポーネント。本件＝有効だと攻撃の前提が成立する。
- **file:// スキーム**: ローカルファイルを指すURIスキーム。本件＝任意ファイル書き込みを誘発し、Webシェル設置に使われる。
- **Webシェル / JSP**: サーバーに置く遠隔操作用プログラム。本件＝設置後にコマンド実行・root昇格の拠点となる。

## 影響範囲 / 推奨対応

影響を受けるのは、WebDialer Web Serviceを有効化しているUnified CM / SME。特に管理・Webインターフェースをインターネットに露出している組織が高リスク。推奨対応は、(1) 14SU6 / 15SU5への即時アップデート、(2) 当面WebDialerの無効化、(3) Webルートの新規JSP・不審サービスの調査（既に悪用観測あり＝侵害想定）、(4) 管理面のインターネット露出遮断とアクセス元制限。
