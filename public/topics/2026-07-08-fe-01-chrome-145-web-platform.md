# Chrome 145 の Web プラットフォーム新機能：column-wrap・::search-text・Origin API・DBSC

> トピック: Chrome 145 リリース｜出典: Chrome for Developers（New in Chrome 145 / Release Notes）｜種別: メジャーブラウザリリース

## 一行サマリー

Chrome 145 が、マルチカラムレイアウトを段組み折り返しできる `column-wrap` / `column-height`、ページ内検索のハイライトを装飾できる `::search-text` 疑似要素、オリジンを扱う `Origin` API、そしてセッションをデバイスに縛りつける DBSC（Device Bound Session Credentials）などを搭載してロールアウトされた。

## 🔰 初学者向け（何が・なぜ重要か）

ブラウザは数週間おきにバージョンを上げ、その都度「Web で新しくできること」が増えていく。Chrome 145 はその最新版で、CSS の見栄えを整える機能から、ログイン状態を安全に保つセキュリティ機能まで幅広く追加されている。

たとえば `column-wrap` は「新聞のような段組みを、縦にあふれたら横の新しい段へ折り返す」という指定ができる。これまで CSS のマルチカラム（`columns` プロパティ）は縦に伸び続けるしかなく、レイアウトが崩れがちだった。段の高さに上限を決めて折り返せるようになると、カードやメニューを均等に段組みするのがぐっと楽になる。

`::search-text` は、ブラウザの「ページ内検索（Ctrl+F / Cmd+F）」でヒットした文字の背景色や文字色を、サイト側から指定できる仕組み。ダークテーマのサイトだと検索ハイライトの黄色が見えづらいことがあったが、コントラストを自分で調整できるようになる。

セキュリティ面では DBSC が目玉だ。ログインを保持する「クッキー」が盗まれると、他人になりすまされてしまう。DBSC はそのクッキーを「この端末でしか使えない」ように、端末内のハードウェア鍵と結びつける。盗んだだけでは使えないので、セッション乗っ取りが難しくなる。

## 🧠 専門的解説（技術詳細・設計・使いどころ）

### 技術的詳細（何が変わったか）

- **CSS `column-wrap` / `column-height`**: マルチカラムレイアウト仕様 Level 2 由来。段（column）がブロック方向にあふれたときに新しい「行」へ折り返せる。`column-height` で各段の高さ上限を指定する。（Chrome 145 で対応・出典で要確認）
- **`::search-text` 疑似要素**: ページ内検索（find-in-page）のハイライト表示に対する前景色・背景色・text-decoration を CSS で指定可能。`:current` との組み合わせで「現在フォーカス中のヒット」を区別できる。
- **`Origin` API**: オリジン（scheme + host + port）を表す `Origin` オブジェクトを導入。比較・シリアライズ・パースのメソッドを提供し、これまで URL 文字列を手で分解していた処理を標準化する。
- **DBSC（Device Bound Session Credentials）**: 短命クッキー + ハードウェアバックの鍵ペアでセッションを端末にバインド。ブラウザが定期的に秘密鍵の所持を証明してセッションクッキーを更新する。
- **`text-justify` プロパティ**: テキストの均等割り付け方法を制御。`letter-spacing` / `word-spacing` にパーセント値（スペース文字の送り幅基準）が指定可能に。

### 仕組み / 背景（なぜこう設計されたか）

DBSC の背景には「クッキー窃取（cookie theft / session hijacking）」への根本対策という課題がある。従来のセッションクッキーは単なる文字列で、盗めば別端末からでも使えてしまう。多要素認証を突破された後の「セッション盗難」が実害の大きい攻撃経路だった。DBSC は TPM 等のハードウェアに閉じた秘密鍵で「所持証明」を要求するため、鍵をエクスポートできない＝クッキーだけ盗んでも別端末では再利用できない。トレードオフとして、サーバー側に鍵登録・チャレンジ応答のエンドポイント実装が必要になり、導入コストは上がる。

`::search-text` と `Origin` API はいずれも「これまで暗黙にブラウザ任せ／文字列処理任せだったものを、明示的な API・スタイル対象に格上げする」流れの一部だ。アクセシビリティ（検索ハイライトのコントラスト確保）とセキュリティ（オリジン比較の取り違え防止）の両面で、暗黙処理を減らして意図を明示できる。

`column-wrap` は、Flexbox/Grid が普及した後もマルチカラムが「縦に伸び続ける」制約で使いにくかった点を解消する。段の高さを制御して横方向に折り返せると、Grid では表現しづらい「テキストが自然に段へ流れ込む」レイアウトが CSS だけで書ける。

### 実務での使いどころ・移行の勘所

- DBSC はエンタープライズの認証基盤・SSO で効果が大きい。ただし現時点では Chromium 系中心で、他ブラウザ対応やサーバー実装の成熟を待つ判断もあり得る（要確認）。
- `column-wrap` / `::search-text` はプログレッシブエンハンスメント前提で使う。未対応ブラウザでも壊れないフォールバックを先に書き、`@supports` でガードする。
- `Origin` API は URL 文字列を手でパースしている箇所の置き換え候補。CSP やオリジン判定ロジックの取り違えバグを減らせる。

## 📖 用語解説（このトピックとのつながり）

- **マルチカラムレイアウト**: `columns` / `column-count` で段組みを作る CSS 機能。`column-wrap` はその Level 2 拡張で、段の折り返しを可能にする。
- **オリジン（Origin）**: scheme・host・port の三つ組。同一オリジンポリシーの単位。Chrome 145 の `Origin` API はこれを第一級オブジェクトとして扱う。
- **セッションハイジャック**: 盗んだセッションクッキーで他人になりすます攻撃。DBSC が対策対象とする脅威。
- **プログレッシブエンハンスメント**: 基本機能を全ブラウザで担保しつつ、対応ブラウザにだけ追加体験を上乗せする設計手法。新 CSS 機能の導入で必須。
- **Baseline**: 主要ブラウザで安定して使えるかを示す指標。Chrome 単独対応の機能はまだ Baseline ではない。

## 影響範囲 / 推奨アクション

- 認証基盤を持つチームは DBSC を評価対象に加える（サーバー側の鍵管理・チャレンジ応答の設計が前提）。
- ダークテーマサイトは `::search-text` で検索ハイライトのコントラストを見直す。
- マルチカラムを使うレイアウトは `column-wrap` を `@supports` ガード付きで試す。
- URL 文字列を手でパースしている箇所は `Origin` API への移行を検討。
- いずれも Chromium 系先行のため、クロスブラウザ状況は caniuse / MDN で最新を確認する。
