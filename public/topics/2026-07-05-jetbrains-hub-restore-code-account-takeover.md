# JetBrains Hub/IDE 群の重大脆弱性 — 弱い乱数の復旧コードで管理者アカウント乗っ取り（CVE-2026-56141 ほか）

> トピック: 脆弱性ニュース ｜ 出典: GBHackers / CyberPress / Vulnerability-Lookup(CIRCL) ｜ 種別: アカウント乗っ取り・認証バイパス・RCE（CVSS 最大 9.8）

## 一行サマリー

JetBrains の統合ID基盤 Hub が、アカウント復旧コードを**予測可能な弱い乱数**で生成していたため（CVE-2026-56141, CVSS 9.8）、攻撃者が総当たりで任意アカウント（管理者含む）を乗っ取れる。IDE 群にも RCE 系の欠陥が同時修正された。

## 🔰 初学者向け（何が・なぜ問題か）

JetBrains は IntelliJ IDEA や PyCharm などの開発ツールで有名です。今回問題になった **Hub** は、それらのサービスを**まとめてログインさせる「共通の入口（社員証の発行所）」**のような存在。ここが破られると、つながっている課題管理やCI/CD（自動ビルド）まで芋づる式に危険になります。

一番深刻なのは「パスワードを忘れたときの**復旧コード**」の作り方でした。復旧コードは、本来は誰にも当てられないよう**完全にランダム**でなければなりません。ところが Hub は、質の悪い（＝規則性が読める）乱数でコードを作っていました。たとえるなら、金庫の暗証番号を「サイコロ」ではなく「時計の秒針」で決めるようなもの。攻撃者は規則を推測して**片っ端から試す（総当たり）**ことで、正しいコードを当てられてしまいます。相手のユーザー名やメールさえ分かれば、管理者アカウントすら乗っ取り可能でした。

さらに、IntelliJ IDEA などの開発ツール本体にも、細工したプロジェクトを開くと**攻撃者のコードが実行されてしまう**穴（RCE）が見つかり、まとめて修正されています。開発者のPCは「本番サーバに入れる鍵束」を持っていることが多く、そこを取られる影響は大きいです。

## 🧠 専門的解説（技術詳細・メカニズム・検知対応）

### 技術的詳細（種別・前提条件）

対象は JetBrains の**オンプレミス**製品群（Hub, YouTrack, IntelliJ 系 IDE, Kotlin, TeamCity 等）。主要な脆弱性:

- **CVE-2026-56141（Critical, CVSS 9.8, CWE-338）**: Hub のアカウント復旧コードが暗号学的に弱い擬似乱数（PRNG）で生成され、予測・列挙により**任意アカウント乗っ取り**。未認証・リモート。報告者 Ngoc Thuan。
- **CVE-2026-50242（Critical, CWE-306）**: DB直接アクセス等を介した**認証バイパス**で管理制御を取得（Hub/YouTrack 共有）。
- **CVE-2026-56142（CWE-915）**: 既存アカウントへ不正な認証情報を紐付けする**権限昇格**。
- **CVE-2026-49382（RCE）**: IntelliJ IDEA の Copyright プラグインの**テンプレートインジェクション**によるコード実行。
- **CVE-2026-49366（RCE）**: IntelliJ IDEA のファイル名補完経由の**コマンドインジェクション**。
- **修正版**: Hub は **2026.1.13757**、および 2025.x / 2024.x 系の対応ビルド。IDE は各最新セキュリティビルド。
- **悪用状況**: 現時点で広範な実環境悪用の確証は要確認。ただし攻撃前提が低く（未認証・遠隔）、影響が大きいため優先度は高い。

### メカニズム（攻撃が成立する仕組み）

**主軸：CVE-2026-56141（弱いPRNGによる復旧コード）**。以下は型の説明であり、具体的アルゴリズムや推測手順は示さない。

1. **信頼の起点が「秘密のコード」**: パスワード復旧フローは「本人だけが受け取れる復旧コードを提示できたら本人とみなす」という設計。コードの**予測不可能性**がこのフロー全体のセキュリティを支える単一の要。
2. **エントロピー不足**: Hub はこのコードを**暗号学的に安全でない乱数源（CWE-338）**で生成していた。一般用途の乱数（時刻ベースや線形合同法など）は、内部状態やシードが推定でき、出力に規則性が残る。
3. **攻撃者が握るプリミティブ**: 規則性があるため、攻撃者は**候補コード空間を大幅に絞り込み**、総当たり/列挙で有効コードを発見できる。これは「本人確認の突破」という認証プリミティブになる。
4. **権限への橋渡し**: 標的をユーザー名/メールで指定して復旧を実行すれば、そのアカウントを掌握できる。**管理者を標的にすれば管理制御を取得**。
5. **IdP としての増幅**: Hub は複数 JetBrains サービスの共通 IdP のため、1アカウント奪取が**課題管理・CI/CD・ソース管理など横断的アクセス**に波及する（SSO の逆流）。

**副軸：IDE の RCE（CVE-2026-49382 / -49366）**。テンプレートやファイル名補完という「ユーザー入力が動的評価される箇所」に**適切なサニタイズ/サンドボックス化が無い**ことが原因。信頼できないプロジェクトを開く＝攻撃者が制御する入力を評価させる、という**入力→実行の橋渡し**が成立する。開発者端末は本番資格情報の宝庫であり、RCE の実害が大きい。

一般原理として、(a) セキュリティ用途に汎用 PRNG を流用する誤り、(b) SSO/IdP の単一障害点化、(c) IDE がプロジェクト内容を暗黙に信頼して評価する設計、はいずれも繰り返し問題になるアンチパターンである。

### 検知・暫定緩和・恒久対策

- **検知の着眼点**: 短時間に大量発生する復旧コード検証の失敗（列挙の痕跡）、管理者アカウントの想定外ログイン/設定変更、Hub 認証ログの異常なリトライ。IDE 側は未知プロジェクト展開直後の子プロセス生成。
- **暫定緩和（パッチ前）**: Hub/YouTrack 管理面のインターネット公開を停止し、VPN/IP制限の背後へ。復旧フローの一時制限やレート制限強化。管理者アカウントの MFA 必須化と棚卸し。
- **恒久対策**: **Hub 2026.1.13757 以降**および各 IDE の最新セキュリティビルドへ更新。更新後は全セッション失効、資格情報ローテーション。「信頼できないプロジェクトはセーフモードで開く」を徹底。
- **運用示唆**: 開発基盤（IdP＋IDE＋CI/CD）は攻撃者にとって「本番への近道」。開発系こそ公開面最小化と多要素・監査を優先する。

## 📖 用語解説（このニュースとのつながり）

- **Hub（JetBrains Hub）**: 複数 JetBrains サービスの共通ログイン基盤（IdP）。ここを乗っ取られると横断被害。本件の中心。
- **PRNG / CWE-338**: 擬似乱数生成器。暗号用途に弱いものを使うと出力が予測でき、復旧コードが当てられる。
- **アカウント復旧コード**: パスワード再設定用の一時コード。予測不可能性が生命線で、そこが崩れた。
- **RCE（リモートコード実行）**: 遠隔から任意コードを走らせる攻撃。今回は IDE のプラグイン/補完機能経由。
- **SSO/IdP の単一障害点**: 共通ログインは便利だが、1点突破で全サービスに波及するリスクを持つ。
- **オンプレミス**: 自社サーバに設置する運用形態。今回の影響対象で、パッチ適用は利用者側の責任。

## 影響範囲 / 推奨対応

- **影響範囲**: オンプレミスの JetBrains Hub / YouTrack / IntelliJ 系 IDE / TeamCity など。2024〜2026 系の広い版が対象で、比較的新しい版でも未更新なら露出。
- **推奨対応**: (1) Hub 2026.1.13757 以降・各IDE最新ビルドへ更新、(2) 全セッション失効と資格情報ローテーション、(3) 管理面の公開停止とMFA必須化、(4) 復旧フローの異常列挙をログ監査、(5) 信頼できないプロジェクトはセーフモードで開く。
