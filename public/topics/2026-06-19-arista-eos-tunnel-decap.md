# Arista EOS CVE-2026-7473 — トンネルの検証不備でネットワーク分離を突破、KEV登録

> トピック: 脆弱性ニュース｜出典: The Hacker News / Arista Security Advisory 0137 / CISA KEV｜種別: 入力検証不備（トンネルのデカプセル化）

## 一行サマリー

データセンタ向けスイッチOS「Arista EOS」のトンネル復号（デカプセル化）処理に検証不備（CVSS 6.9）があり、宛先IPだけで外側ヘッダを剥がすためプロトコルを偽装した不正トンネル通信を内部へ送り込める。ネットワーク分離（セグメンテーション）を回避され、CISA KEV にも登録済み。恒久パッチ提供予定がなく設定での緩和が中心となる。

## 🔰 初学者向け（何が・なぜ問題か）

大きなネットワークでは、データを「トンネル」という仮想の専用通路に包んで運ぶことがあります（VXLAN や GRE といった技術）。荷物（通信）を二重封筒に入れ、外側の封筒に「この拠点宛て」と書いて運び、到着したら外側の封筒を開けて中身を取り出す——このイメージです。

Arista EOS というネットワーク機器のOSは、外側の封筒を開ける（デカプセル化＝外側ヘッダを剥がす）かどうかを「宛先の住所（宛先IP）」だけで判断していました。本来は「正しい配送方式（決められたトンネルの種類）で届いたか」も確認すべきなのに、それをしていなかったのです。

結果、攻撃者は「住所だけ合っていれば中身は何でも通る」ことを悪用し、本来は内部ネットワークに入れないはずの通信を、内部の配送ルートに紛れ込ませられます。社内の「立入禁止エリアを分ける仕切り（ネットワーク分離）」をすり抜けられてしまうわけです。すでに実環境での悪用が確認されており、しかも恒久パッチの提供予定がないため、設定での対策が重要になります。

## 🧠 専門的解説（技術詳細・メカニズム・検知対応）

### 技術的詳細（種別・前提条件）

- **CVE**: CVE-2026-7473 / **CVSS**: 6.9（Medium、要確認: ソースにより表記差あり）
- **CWE**: CWE-1023（Incomplete Comparison with Missing Factors / 比較に必要な要素が欠落）
- **影響コンポーネント**: EOS フォワーディングパイプラインのトンネル・デカプセル化処理
- **根本原因**: トンネル終端IP宛のパケットに対し外側ヘッダを剥がす判断を「宛先IPのみ」で実施。設定したカプセル化プロトコル（VXLAN/GRE等）と実際のプロトコルが一致するかを検証していない。
- **影響構成**: トンネル・デカプセル設定（VXLAN、decap-groups、GREトンネルインタフェース等）が有効な EOS プラットフォーム。
- **悪用状況**: CISA KEV 登録済み（2026-06-09 追加）。実環境のネットワーク基盤に対する悪用を確認。
- **対応状況**: 恒久パッチ提供予定なしとの報道（要確認）。設定変更・フィルタによる緩和が中心。
- **前提条件**: 攻撃者は終端IP宛のトンネル通信を到達させられること。認証は不要（リモート・未認証）。

### メカニズム（攻撃が成立する仕組み）

※以下は脆弱性クラスの一般原理に基づく概念説明。武器化手順・PoCは記載しない。

1. **判定要素の欠落（不完全な比較）**: デカプセル化を行うか否かの判定が「宛先IP == 設定したトンネル終端IP」だけで行われる。本来必要な「カプセル化プロトコルの一致」という比較要素が欠落している（CWE-1023の核心）。
2. **信頼境界の誤認**: 外側ヘッダの宛先IPは攻撃者が自由に詐称できる外部入力にすぎない。それだけを根拠に「内側に通してよい正規トンネル」と誤認する。
3. **不正トンネル注入のプリミティブ**: 攻撃者は終端IP宛にパケットを送り、設定外のプロトコルで包んだ通信を機器に剥がさせる。これにより「内側フォワーディングパスへ任意の内部向けパケットを注入する」原始操作を得る。
4. **分離の突破へ橋渡し**: 内側に注入された通信は、本来セグメント境界で遮断されるはずのトラフィックとして内部ネットワークへ到達する。結果、ネットワーク分離・マイクロセグメンテーションを回避し、横展開（ラテラルムーブメント）の足がかりになる。
   - 脆弱性クラスの一般原理: 「アクセス可否を複数条件で決めるべき箇所を単一条件で判断する」誤りは、認可・フィルタ・ルーティング判定で繰り返し現れる。比較に必要な因子を一つでも欠くと、境界そのものが意味を失う。

### 検知・暫定緩和・恒久対策

- **検知の着眼点**: トンネル終端IP宛で「設定外プロトコル」のカプセル化パケットの観測、想定外のVXLAN/GREトラフィック、内部セグメントへ突然現れる出所不明のフロー、NetFlow/sFlow 上の異常な内向き通信。
- **暫定緩和**: 終端IP宛のトンネル通信を信頼できる送信元・経路のみに ACL で制限。境界での ingress フィルタリング（外部からのトンネル終端IP宛トラフィック遮断）。不要なデカプセル設定（decap-groups 等）の無効化。Arista Advisory 0137 の推奨設定に従う。
- **恒久対策**: 恒久パッチが提供されない可能性があるため、設定とネットワーク設計（境界フィルタ・到達性制限）での恒久的な緩和を前提に運用設計する。ベンダー情報を継続追跡。

## 📖 用語解説（このニュースとのつながり）

- **EOS（Arista）**: Arista 製スイッチのネットワークOS。データセンタやキャンパス基盤で広く使われ、影響範囲が大きい。
- **デカプセル化（decapsulation）**: トンネルで包んだ通信の外側ヘッダを剥がす処理。本件はこの判定が甘く不正通信を内部へ通した。
- **VXLAN / GRE**: 代表的なトンネル技術。これらの設定が有効な機器が影響対象。
- **CWE-1023 不完全な比較**: 判定に必要な条件の一部が欠けている欠陥。本件では「プロトコル一致」の確認漏れ。
- **ネットワーク分離 / セグメンテーション**: 通信を区切って横展開を防ぐ設計。本件はその仕切りを回避される。
- **KEV**: CISA が「実際に悪用中」と認定した一覧。登録済みのため優先対応対象。

## 影響範囲 / 推奨対応

影響は、VXLAN・GRE・decap-groups などトンネル復号設定を有効にしている Arista EOS 機器。データセンタやキャンパスのコア/分散層が中心で、分離突破は内部侵害の足場になりうる。推奨対応は、Arista Security Advisory 0137 を確認のうえ、トンネル終端IP宛通信を ACL/境界フィルタで制限し、不要なデカプセル設定を無効化すること。恒久パッチが出ない前提で、設計レベルの緩和とログ監視を継続する。
