# Mistral 3 ファミリーと Apache 2.0 全面開放 — 675B MoE「Large 3」が2026年オープンモデル勢力図の欧州代表に

> トピック: Mistral 3 ファミリーの Apache 2.0 開放と2026年夏時点の位置付け｜出典: Mistral AI 公式発表｜種別: オープンウェイトモデルの戦略転換（発表は2025年12月、2026年7月の各種リーダーボードで主力として定着）

## 一行サマリー

Mistral AI が発表した Mistral 3 ファミリー（3B/8B/14B の密モデルと、総パラメータ675B・アクティブ41BのMoE旗艦 Mistral Large 3）はすべて Apache 2.0 で商用制限なく公開されており、256kトークンの長文脈とマルチモーダル対応を備えた「制約のないフロンティア級オープンウェイト」として、2026年7月現在のオープンモデル選定で Qwen・DeepSeek・Llama と並ぶ主要候補に定着している。

## 🔰 初学者向け（何が・なぜ重要か）

AI モデルには大きく2種類あります。OpenAI などの「中身非公開・API 利用のみ」のモデルと、モデルの中身（重み）をダウンロードして自分のサーバーで動かせる「オープンウェイト」モデルです。フランスの Mistral AI は後者の代表格ですが、以前は一部モデルに「研究用のみ」といった制限付きライセンスを使っていました。

Mistral 3 世代の大きな決断は、小型から旗艦まで全部を Apache 2.0 という「商用利用も改変も自由」の緩いライセンスで出したことです。これは「どうぞ会社の製品に組み込んでください、改造も転売も自由です」という宣言で、利用者から見ると法務確認の手間が激減します。

旗艦の Large 3 は「専門家の分業（MoE）」という構造で、総勢675B（6,750億）のパラメータを持ちながら、1回の応答では41Bぶんだけが働きます。巨大な病院に例えると、医師は大勢いるが、1人の患者を診るのは適切な専門医数人だけ——賢さの在庫は大きく、動かすコストは中型並み、という仕組みです。発表は2025年12月ですが、2026年夏の今もオープンモデルの主力として使われ続けており、「欧州製・制限なし・フロンティア級」という選択肢の意味は EU の AI 規制が本格化する今むしろ増しています。

## 🧠 専門的解説（技術詳細・設計・使いどころ）

### 技術的詳細（何が変わったか）

- **ファミリー構成**: 密モデル 3B / 8B / 14B（エッジ・ノート PC・ドローン等を想定）＋ 旗艦 **Mistral Large 3**（sparse MoE、総675B・アクティブ41B）。
- **ライセンス**: base / instruct とも全モデル **Apache 2.0**。商用利用・派生・再配布に制限なし。
- **能力**: 最大256kトークンの長文脈、テキスト＋画像のマルチモーダル推論、堅牢な指示追従。
- **学習基盤**: Large 3 は NVIDIA H200 を3,000基使いスクラッチから学習。
- **提供**: 重み公開に加え、IBM watsonx、DigitalOcean 等の主要クラウドでもホスト提供が拡大。
- 2026年7月時点の各種オープンモデル比較では、Qwen 3 系・DeepSeek R1 系・Llama 4 系と並ぶ常連（順位はタスク依存。要確認）。

### 仕組み / 背景（なぜこう設計されたか）

**MoE（sparse Mixture of Experts）** は、フィードフォワード層を多数の「専門家」に分割し、トークンごとにルーターが少数（Large 3 では総675B中41B相当）だけ活性化する構造。知識容量（総パラメータ）と推論コスト（アクティブパラメータ）を分離できるのが本質で、DeepSeek V3、GPT-4 系（推定）など現代のフロンティア級はほぼこの構造に収斂している。代償は、(1) 全専門家をメモリに載せる必要がありVRAM要件は総パラメータ側に引きずられる、(2) ルーティングの負荷分散が学習・推論の難所になる、こと。自宅 GPU では動かず、「オープンだが動かせるのはクラウド/企業」という階層化を生む。

**Apache 2.0 全面開放**の戦略的意味は3層ある。第一に、Llama 系の独自ライセンス（利用規模条項等）や「オープンだが商用条件付き」の中間ライセンス群に対する差別化。第二に、EU AI Act（本日別記事参照）はオープンソース GPAI に義務軽減を与えており、欧州企業 Mistral にとって「真にオープン」であることは規制上も営業上も資産になる。第三に、モデル自体を無料にしてもホスティング・カスタマイズ（企業向けファインチューニング）で収益化する路線への確信で、これは Red Hat 型のビジネスモデルの AI 版といえる。

### 実務での使いどころ・移行の勘所

- **自社ホスト要件（データ主権・機密性）があるフロンティア級用途**: Large 3 は「API 禁止環境で使える最上位クラス」の候補。ただし推論には H100/H200 級の複数枚構成が前提（具体的な最小構成は量子化次第、要確認）。
- **エッジ・組み込み**: 3B/8B/14B の密モデルは Ollama 等で手元実行が現実的。ライセンス確認不要で製品組み込みできる点が Llama 系に対する明確な利点。
- **EU 圏コンプライアンス**: 欧州顧客向け案件では「欧州製＋Apache 2.0＋AI Act のオープン軽減」の3点セットが選定理由として通りやすい。
- 比較の目安: 総合力とエコシステムなら Qwen 3 系、数理推論特化なら DeepSeek R1 系、超長文脈なら Llama 4 Scout、ライセンスの単純さと欧州要件なら Mistral 3、が2026年夏時点の大づかみな整理。

## 📖 用語解説（このトピックとのつながり）

- **オープンウェイト**: 重みを公開し自前で実行可能なモデル。学習データやコードまで公開する「オープンソース」とは厳密には区別される。
- **Apache 2.0**: 特許条項を含む寛容ライセンス。商用・改変・再配布が自由で、法務審査コストが最小。
- **MoE / アクティブパラメータ**: 応答ごとに実際に計算に使われるパラメータ量。コストはこちら、メモリ要件は総量で決まる。
- **密（dense）モデル**: 全パラメータが常に働く従来型。小型帯では MoE より扱いやすく、Mistral 3 の 3B/8B/14B はこちら。
- **GPAI 義務軽減（AI Act）**: オープンライセンスの汎用モデルに対する EU の義務緩和。Mistral のライセンス戦略と直結する。
