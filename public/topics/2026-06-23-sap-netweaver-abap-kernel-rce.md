# SAP NetWeaver/ABAPカーネルに CVSS 9.8 の未認証メモリ破壊（CVE-2026-27671）— 細工RFCで基幹システム侵害

> トピック: 脆弱性ニュース ｜ 出典: SAP Security Note 3717897 / SecurityWeek / BleepingComputer ｜ 種別: RFCプロトコル検証不備に起因するメモリ破壊（CWE-121系）→未認証RCE

## 一行サマリー

多くの企業の基幹業務（会計・人事・物流）を支えるSAPの土台「ABAPアプリケーションサーバー（NetWeaver）」に、認証なしで細工した **RFC通信** を送るだけでメモリを破壊し、最悪は任意コード実行に至る CVSS 9.8 の脆弱性 CVE-2026-27671 が見つかった。基幹システムの掌握は事業停止や全社的なデータ漏えいに直結するため、SAPカーネルのパッチ適用を最優先で進める必要がある。

## 🔰 初学者向け（何が・なぜ問題か）

**SAP** は、会社の「お金・人・モノ」の流れをまとめて管理する基幹システム（ERP）の代表格です。請求も、給与計算も、在庫管理も、その上で動いています。会社にとっては心臓部のような存在で、ここが止まると業務全体が止まりかねません。

そのSAPの土台部分が **NetWeaver / ABAPアプリケーションサーバー** で、システム同士が会話するための専用の電話回線として **RFC（Remote Function Call）** という仕組みを使っています。RFCは「別のシステムの機能をリモートで呼び出す」ためのもので、SAPの世界では当たり前に使われている内部通信です。

今回の問題は、その電話を受け取る交換機（SAPカーネル）が、**かかってきた電話の内容を正しくチェックせずに処理してしまう**点にあります。攻撃者がログインもせずに、わざと規格外の・つじつまの合わないRFCの呼び出しを送りつけると、交換機の内部メモリが壊れ（メモリ破壊）、運が悪ければ攻撃者のプログラムが動き出してしまいます。

たとえるなら、受付の自動電話システムに、規定外のデタラメな信号を流し込んで内部の配線をショートさせ、あわよくば受付システムそのものを乗っ取る、というイメージです。ここを奪われると、会社の財務データや個人情報の閲覧・改ざん、システム全体の停止につながります。**ログイン不要**で攻撃できる点が特に危険です。

## 🧠 専門的解説（技術詳細・メカニズム・検知対応）

### 技術的詳細（種別・前提条件）

- **CVSS**: 9.8（Critical）。**未認証・ネットワーク経由・低複雑度・ユーザー操作不要**。
- **CWE分類**: RFCプロトコル検証不備に起因するメモリ管理上の論理エラーで、スタックベースのバッファオーバーフロー（CWE-121）条件を引き起こしうる。
- **前提条件**: 攻撃者が脆弱なRFC受信エンドポイント（ABAPアプリケーションサーバーのRFCポート／ゲートウェイ）に**ネットワーク到達できる**こと。SAP GatewayやMessage Serverが不用意に露出している環境はリスクが高い。
- **影響範囲（カーネル）**: SAP Kernel の KRNL64NUC 7.22 / 7.22EXT、KRNL64UC 7.22 / 7.22EXT、および KERNEL 7.22 / 7.53 / 7.54 / 7.77 / 7.89 / 7.93 / 9.16 / 9.18 / 9.19。
- **修正版・対応**: SAP Security Note **3717897** に基づくカーネルパッチ（`dw.sar` 等のホットフィックス、または SAPEXE.SAR / SAPEXEDB.SAR の SP Stack Kernel）を適用。正確な対象SP/パッチレベルは Note で要確認。
- **悪用状況**: 本稿時点で実環境悪用の確認は**要確認**。ただしSAPは攻撃者の主要標的であり、過去のNetWeaver系Criticalは公開後に急速に武器化された実績がある。

### メカニズム（攻撃が成立する仕組み）

以下は概念レベルの説明であり、武器化可能な手順・PoC・正確なオフセット等は含めない。

1. **信頼境界の所在**: ABAPアプリケーションサーバーのRFC受信経路は、外部システムとの連携のために**認証前の段階でプロトコルデータを解析**する必要がある。SAP Gateway/Message Serverがネットワーク露出していると、この認証前パーサが直接の攻撃面になる。
2. **RFCプロトコル検証の不備**: SAPカーネルがRFCのプロトコルフレーム（フィールド長・構造・状態遷移）を解析する際、**入力の整合性検証が不十分**なため、想定外の長さ・順序・型を持つメッセージでメモリ管理上の論理エラーが発生する。これがバッファ境界を越えた書き込み（オーバーフロー）として顕在化する。
3. **メモリ破壊から実行制御へ**: 範囲外書き込みにより、カーネルプロセスのスタック上の制御データ（戻りアドレス等）や内部状態が破壊される。結果として、最小ではプロセスクラッシュ（DoS＝SAP停止）、最大では制御フロー奪取による任意コード実行（RCE）に至る。被害の幅は破壊できるメモリの位置と保護機構の有無に依存する。
4. **未認証で完結する致命性**: 認証前のRFC経路で成立するため、攻撃者は資格情報を持たずに基幹システムへ到達する。SAPは多くの社内システムのハブであり、ここの掌握は横展開の起点になる。

脆弱性クラスとしては、「認証前にプロトコルを解析するコンポーネントの入力検証欠如によるメモリ破壊」という、ネットワークサービス実装で頻出する型に該当する。RFC/Gatewayの露出最小化が構造的な防御線となる。

### 検知・暫定緩和・恒久対策

- **検知の着眼点**: SAP Gateway / Message Server のログにおける異常なRFC接続・プロトコルエラーの急増、`disp+work`（ディスパッチャ/ワークプロセス）の予期しないクラッシュやダンプ生成、RFCポートへの外部スキャン。SAP Enterprise Threat Detection（ETD）やSIEMでカーネルダンプ・gwログを相関監視。
- **暫定緩和（パッチ前）**: SAP Gateway の **`gw/reg_info` / `gw/sec_info`** によるアクセス制御を厳格化し、登録可能なプログラム・接続元を限定。Message Server の内部/外部ポート分離（`ms/server_port` の保護）。RFC関連ポート（33xx/32xx/36xx/48xx等）を境界ファイアウォールでインターネットから遮断し、信頼セグメントに限定。
- **恒久対策**: SAP Note 3717897 に従いカーネルパッチを即時適用。カーネル更新は全アプリケーションサーバーインスタンスに反映し、適用後はクラッシュダンプ・不審RFC接続の事後ハントを実施。
- **運用示唆**: SAPは「インターネット露出してはならない基幹サービス」。RFC/Gateway/Message Serverの露出排除とアクセス制御（reg_info/sec_info）を恒常原則化し、SAP Security Note月次適用を運用に組み込む。

## 📖 用語解説（このニュースとのつながり）

- **SAP / NetWeaver / ABAP**: 企業の基幹業務を担うERPと、その土台プラットフォーム・開発言語。意味＝会社の「心臓部」。本件＝この土台のカーネルが未認証で攻撃される。
- **RFC（Remote Function Call）**: SAPシステム間で機能を遠隔呼び出しする内部通信。意味＝システム同士の「専用電話」。本件＝この受信処理の検証不備が起点。
- **SAP Gateway / Message Server**: RFC接続やインスタンス間通信を仲介するコンポーネント。本件＝露出していると認証前攻撃面になる。
- **メモリ破壊 / スタックBOF（CWE-121）**: メモリの不正な書き換え。本件＝DoSからRCEまで被害が振れる根本原因。
- **SAP Security Note**: SAPが配布する公式の脆弱性修正情報。本件＝Note 3717897 が修正の入口。

## 影響範囲 / 推奨対応

影響を受けるのは、対象カーネルで稼働する **SAP NetWeaver / ABAP Platform Application Server ABAP** を運用する全組織。会計・人事・物流など基幹業務が乗るため、侵害はデータ漏えい・改ざん・事業停止に直結する。推奨対応は、(1) SAP Note 3717897 に基づくカーネルパッチの即時適用（全AppServerインスタンス）、(2) Gateway/Message ServerのアクセスIPと露出の最小化（reg_info/sec_info強化、ポート遮断）、(3) RFCポートの境界遮断と内部セグメント限定、(4) 適用後のクラッシュダンプ・不審RFC接続のハント。
