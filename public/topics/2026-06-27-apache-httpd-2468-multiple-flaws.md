# Apache HTTP Server 2.4.68 公開 — mod_ldap の解放後使用ほか複数脆弱性を修正

> トピック: 脆弱性ニュース ｜ 出典: Apache HTTP Server Project / The Hacker News / JPCERT-CC・IPA ｜ 種別: 解放後使用（UAF, CWE-416）／クロスサイトスクリプティング（CWE-79）／リソース枯渇 DoS（CWE-400）

## 一行サマリー

世界・国内で最も普及する Web サーバ Apache HTTP Server に複数の脆弱性が見つかり、2.4.68 で 2.4.0〜2.4.67 が対象の計13件が修正された。mod_ldap の解放後使用（CVE-2026-29167）、mod_proxy_ftp の XSS（CVE-2026-29170）、mod_http2 のメモリ枯渇 DoS（CVE-2026-49975）などを含む。該当構成のサーバは 2.4.68 への更新が推奨される（CVSS は要確認）。

## 🔰 初学者向け（何が・なぜ問題か）

「Apache HTTP Server（httpd）」は、世界中の Web サイトを動かしている超定番の“サーバの土台”ソフトです。日本国内でも非常に多くのサイトやシステムで使われています。今回、その Apache に複数の弱点が見つかり、新しい版（2.4.68）でまとめて直されました。

Apache は機能を「部品（モジュール）」として組み込んで使います。今回問題があった部品の例：

ひとつは「mod_ldap」という、社内のユーザー名簿（LDAP）と連携してログイン認証する部品。ここに“もう捨てたメモをうっかり後で読み返す”ような欠陥（解放後使用）があり、サーバが不安定になったり悪用の足がかりになり得ます。もうひとつは「mod_http2」。大量の細工リクエストでサーバのメモリを使い果たさせ、サイトを応答不能（サービス停止＝DoS）にできる弱点です。

つまり、放置するとサイトが落とされたり、認証連携の安定性・安全性が損なわれる恐れがあります。これらの部品を使っている管理者は、Apache を新しい版へ更新するのが対策です。

## 🧠 専門的解説（技術詳細・メカニズム・検知対応）

### 技術的詳細（種別・前提条件）

- 種別（主なもの）:
  - CVE-2026-29167: mod_ldap のディレクトリ単位設定における解放後使用（ダングリングポインタ、CWE-416）。
  - CVE-2026-29170: mod_proxy_ftp の FTP ディレクトリ一覧 HTML 生成における出力無害化不備の XSS（CWE-79）。Apache が FTP をプロキシしている場合。
  - CVE-2026-49975: mod_http2 における悪性 HTTP/2 リクエストでのメモリ割り当て枯渇（DoS、CWE-400）。影響 2.4.17〜2.4.67。
- CVSS: 各 要確認（13件中に UAF/XSS/ヒープオーバーフロー/DoS/権限昇格/境界外読み取りを含む）。
- 前提条件: 各脆弱性で異なる。mod_ldap/mod_proxy_ftp/mod_http2 等、該当モジュールを有効化した構成であることが主な条件。DoS は認証不要でネットワーク到達のみが多い。
- 影響: 2.4.0〜2.4.67 のすべて（モジュール構成に依存）。
- 修正版: Apache HTTP Server 2.4.68（2026-06-08 公開、要確認）。
- 悪用状況: 公開時点で広範な実環境悪用の確証は乏しい（要確認）。ただし HTTP/2 系 DoS は再現容易で武器化されやすい。

### メカニズム（攻撃が成立する仕組み）

注: 概念レベルにとどめ、武器化手順・PoC・正確なオフセットは記載しない。

1. **解放後使用（mod_ldap, CVE-2026-29167）**: ディレクトリ単位設定の処理で、解放済みメモリを指すポインタ（ダングリングポインタ）が残存。後続処理がそれを参照すると、解放済み領域の再利用と相まって、クラッシュや（条件次第で）メモリ内容の制御に繋がりうる。信頼境界は「設定再読込・リクエスト処理のライフサイクル」で、オブジェクト寿命管理の不備が核心。
2. **XSS（mod_proxy_ftp, CVE-2026-29170）**: Apache が FTP をプロキシしてディレクトリ一覧を HTML 化する際、サーバ／ファイル名由来の文字列を無害化（エスケープ）せず出力。攻撃者が制御し得る名前にスクリプトを混ぜると、閲覧者ブラウザで実行される。信頼境界は「外部 FTP コンテンツ→生成 HTML→利用者ブラウザ」。
3. **メモリ枯渇 DoS（mod_http2, CVE-2026-49975）**: 悪性 HTTP/2 リクエストにより、サーバが過剰なメモリ割り当てを誘発される。リソース消費の上限制御が不十分なため、少ないコストで大量割り当て＝枯渇させ、応答不能に追い込む。攻撃者プリミティブは「非対称なリソース増幅」。
4. **被害への橋渡し**: UAF はサービス停止〜（高度な連鎖で）コード実行リスク、XSS はセッション窃取・管理画面操作、DoS は可用性喪失へ。いずれも「該当モジュールが有効」かどうかで露出度が大きく変わる。
5. **一般原理**: UAF（寿命管理不備）、出力エスケープ漏れ（XSS）、リソース上限欠如（DoS）はいずれも古典的かつ普遍的な脆弱性クラス。広く使われるミドルウェアでは、単一バグでも影響台数が膨大になる点が脅威の本質。

### 検知・暫定緩和・恒久対策

- 検知の着眼点: httpd の異常なメモリ使用・OOM・ワーカ再起動の急増（DoS 兆候）、mod_proxy_ftp 経由ページでの不審なスクリプト混入、エラーログの segfault/UAF 関連クラッシュ。
- 暫定緩和: 不要なモジュール（mod_proxy_ftp、mod_ldap、mod_http2 等）の無効化、HTTP/2 のリクエスト/同時ストリーム上限やメモリ関連のチューニング、WAF/リバースプロキシでの異常 HTTP/2 トラフィック制限。
- 恒久対策: Apache HTTP Server を 2.4.68 へ更新。更新後、有効モジュール構成を見直し最小化する。
- 運用示唆: Apache は国内システムに広く組み込まれる基盤。アプライアンスや製品同梱の httpd（間接的に内包）も対象になり得るため、ベンダ提供版の更新も追跡する。

## 📖 用語解説（このニュースとのつながり）

- **mod_ldap**: Apache を LDAP（ユーザー名簿）認証と連携させるモジュール。本件の UAF の所在。
- **解放後使用（UAF, CWE-416）**: 解放済みメモリを再び使う欠陥。クラッシュやメモリ制御の起点。
- **mod_http2 / HTTP/2 DoS**: HTTP/2 を扱うモジュール。少ないコストでメモリを枯渇させられる可用性リスク。
- **XSS（CWE-79）**: 出力の無害化漏れで閲覧者のブラウザでスクリプトが走る欠陥。mod_proxy_ftp が該当。
- **モジュール構成**: Apache は機能を部品で足す。有効化している部品次第で、各脆弱性の影響有無が決まる。

## 影響範囲 / 推奨対応

Apache HTTP Server 2.4.0〜2.4.67 を運用する環境が対象（実影響は有効モジュールに依存）。2.4.68 へ更新し、mod_proxy_ftp・mod_ldap・mod_http2 など不要なモジュールは無効化して攻撃面を縮小する。HTTP/2 のリソース上限を点検し DoS 耐性を高める。製品やアプライアンスに同梱された httpd についてもベンダ更新を追跡する。各 CVSS・該当範囲は確定情報を要確認。
