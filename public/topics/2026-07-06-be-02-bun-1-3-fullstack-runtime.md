# Bun 1.3 — Bun.SQL・組み込み Redis・フロント開発サーバーで「全部入り」フルスタックランタイムへ

> トピック: Bun 1.3 のフルスタック化｜出典: Bun 公式ブログ（bun.com）｜種別: メジャー機能追加

## 一行サマリー

Bun 1.3 は、MySQL / MariaDB / PostgreSQL / SQLite を外部依存なしで扱う統一DB API「Bun.SQL」、ioredis 比 7.9 倍を謳う組み込み Redis クライアント、ゼロ設定のフロントエンド開発サーバーを備え、単なる高速な Node 代替から「1つで完結するフルスタックJavaScriptランタイム」へと踏み込んだ。

## 🔰 初学者向け（何が・なぜ重要か）

JavaScript をサーバー側で動かす土台（ランタイム）といえば長らく Node.js でした。Bun はその高速な対抗馬で、起動やインストールが速いのが売りです。1.3 では「速いだけ」から一歩進み、アプリ開発に必要な部品を本体に取り込みました。

たとえばデータベース接続。普通は MySQL 用・PostgreSQL 用と別々のライブラリを入れますが、Bun 1.3 の `Bun.SQL` は同じ書き方で複数のDBに繋げます。追加インストール不要です。よく使うキャッシュ用の Redis も本体に内蔵され、しかもかなり高速だと報告されています。さらに、フロントエンド（画面側）の開発サーバーも設定なしで立ち上がります。

つまり「ランタイム＋DBクライアント＋キャッシュ＋フロント開発環境」が1つにまとまり、道具を寄せ集める手間が減る、というのが 1.3 の意義です。

## 🧠 専門的解説（技術詳細・設計・使いどころ）

### 技術的詳細（何が変わったか）

- **Bun.SQL（統一DB API）**: MySQL / MariaDB / PostgreSQL / SQLite をネイティブ実装で扱う共通API。アダプタ間で一貫した構文を保ちつつ、外部依存なしで動く。PostgreSQL 配列型を扱う `sql.array` ヘルパーなども追加。
- **組み込み Redis クライアント**: 本体内蔵。ioredis 比 7.9 倍の性能を主張（ワークロード依存のため実測は要確認）。
- **ゼロ設定フロントエンド開発サーバー**: フロント資産の開発サーバーを設定なしで起動。
- **性能改善**: Next.js 等でメモリ 10〜30% 削減、macOS ビルド 60% 高速化、Express ベンチ 9% 高速化などを主張。
- リリースは Bun 1.3（2025-10-13）。数値は公式ブログ主張値で、環境差があるため要確認。

### 仕組み / 背景（なぜこう設計されたか）

Bun の性能面の核心は、JavaScript エンジンに V8 ではなく JavaScriptCore（WebKit 系）を採用し、ランタイム本体を Zig で書いて I/O と起動を最適化している点にある。1.3 の「全部入り」路線は、この速いコアの上に「よく使う機能をネイティブ実装で同梱する」戦略だ。DBドライバやRedisクライアントを JS ライブラリ経由ではなくネイティブで持つと、プロトコル処理やバッファリングのオーバーヘッドを削れ、依存ツリーも縮む——これが 7.9 倍・9% といった主張の背景にある。

設計上のトレードオフは明確だ。「1つに集約」は開発体験と起動速度に効く反面、機能ごとに最適化された成熟ライブラリ（例: 特定DB向けの高機能ORM、実戦検証の厚い ioredis）と比べると、エッジケース対応や運用実績の蓄積で見劣りする場面がある。また本体が肥大化し、Node の「小さなコア＋豊富な外部エコシステム」という思想とは逆方向になる。ベンチ値も「同条件・同一負荷」での再現が前提で、無条件の高速化ではない。

### 実務での使いどころ・移行の勘所

新規の小〜中規模アプリ、社内ツール、CLI、エッジ寄りのサービスで「セットアップを最小化したい」ケースに向く。既存 Node 資産の移行は、Node 互換性が高いとはいえネイティブアドオンや一部 API で差が出るため、まず CI で全テストを Bun 上で回して差分を洗うのが定石。DBは `Bun.SQL` で足りる範囲なら依存を減らせるが、複雑なマイグレーション/リレーション管理が要るなら既存ORM併用が現実的。性能主張は自分のワークロードで必ず実測してから採用判断する。

## 📖 用語解説（このトピックとのつながり）

- **ランタイム**: JS を実行する土台。Bun は Node の代替で、1.3 で機能を内包した。
- **Bun.SQL**: 複数RDBを共通APIで扱う内蔵クライアント。外部依存を減らす中核。
- **Redis**: インメモリのキャッシュ/データストア。1.3 で高速クライアントを内蔵。
- **JavaScriptCore / Zig**: Bun の速さを支えるエンジンと実装言語。
- **Node 互換性**: 既存 Node コードが動く度合い。移行時の検証ポイント。

## 影響範囲 / 推奨アクション

- 対象: JavaScript/TypeScript でサーバーやフルスタックを組むチーム。
- 推奨: 新規小〜中規模や社内ツールで導入検討。DB/Redis の依存削減を評価。
- 注意: 既存 Node 移行は全テストを Bun で実行し差分確認。性能主張は自環境で実測。
