# PHP 8.5 — パイプ演算子 `|>`・URI 拡張・clone with で「関数合成」と「不変更新」が書きやすく

> トピック: PHP 8.5（2025-11-20 リリース）｜出典: php.net・Laravel News・stitcher.io｜種別: 言語

## 一行サマリー

PHP 8.5 は、値を関数へ左から右へ流し込むパイプ演算子 `|>`、RFC 3986 / WHATWG に準拠した組み込み URI 拡張、クローン時にプロパティを書き換える `clone(...)` 構文、戻り値の未使用を警告する `#[\NoDiscard]` 属性などを追加した。中間変数だらけの処理や readonly クラスの「with 系」更新が、素直に書けるようになる。

## 🔰 初学者向け（何が・なぜ重要か）

PHP は Web の定番言語。8.5 では「読みやすく、間違えにくく書ける」ための道具がいくつも入った。

目玉のパイプ演算子 `|>` は、「ある値を関数 A に通し、その結果を関数 B に通し…」という流れを、`$result = $x |> a(...) |> b(...);` のように左から右へ書ける。従来は `$tmp1 = a($x); $tmp2 = b($tmp1);` と一時変数を並べるか、`b(a($x))` と内側から外へ読む入れ子になり、読みにくかった。パイプはこれを「工場のベルトコンベア」のように、処理の順番どおりに並べられる。

`clone(...)` は「元のオブジェクトはそのままに、一部だけ違うコピーを作る」書き方。設定オブジェクトなどを安全に少しずつ変えたいときに便利。`#[\NoDiscard]` は「この関数の戻り値、無視しちゃダメだよ」と警告してくれる安全装置だ。

## 🧠 専門的解説（技術詳細・設計・使いどころ）

### 技術的詳細（何が変わったか）

- **リリース**: 2025 年 11 月 20 日（PHP 8.5.0）。
- **パイプ演算子 `|>`**: callable を左から右へ連結し、中間変数なしに値を流す。関数合成的な記述が可能。
- **URI 拡張**: RFC 3986 と WHATWG URL 標準に沿って URL を解析・正規化・操作する組み込み拡張。自前パースの脆弱性・不整合を減らせる。
- **`clone(...)` でのプロパティ更新**: クローンと同時にプロパティを書き換えられる。readonly クラスでの「with-er」パターンが簡潔に。
- **`#[\NoDiscard]` 属性**: 戻り値が使われないと警告し、API の誤用を防ぐ。
- **配列関数**: `array_first()` / `array_last()` が先頭 / 末尾の値を返す（空なら null）。

### 仕組み / 背景（なぜこう設計されたか）

パイプ演算子は関数型言語（Elixir・F# 等）で定着した「データを関数群に順に通す」発想を PHP に取り込んだもの。PHP には第一級 callable 構文（`a(...)`）が既にあり、`|>` はその callable に左辺の値を単一引数として適用する糖衣として設計された。これにより「入れ子で内→外に読む」認知負荷が消え、処理順＝記述順の可読性が得られる。トレードオフは、多引数関数を挟むときは部分適用やラッパが要る点（`|>` は単一値を流す前提）。

`clone(...)` によるプロパティ更新は、`readonly` プロパティの普及で顕在化した「不変オブジェクトを少しだけ変えたコピーが欲しい」需要への回答。readonly は生成後の代入を禁じるため、従来は with 系メソッドを手書きする必要があった。言語レベルでクローン＋更新を一体化することで、不変データ構造を安全・簡潔に扱える。URI 拡張は「URL パースを各自が正規表現で書いて事故る」問題への標準的解答で、仕様準拠のパーサを言語同梱にしてセキュリティと相互運用性を底上げする狙い。

### 実務での使いどころ・移行の勘所

データ整形パイプライン（バリデーション→変換→整形）、設定/値オブジェクトの不変更新、URL を扱う処理に効く。移行の勘所は、(1) 深い入れ子呼び出しや中間変数の多い箇所を `|>` に置き換えて可読性を上げる、(2) 自前 URL パースを URI 拡張へ寄せてセキュリティを担保、(3) readonly 値オブジェクトの with 系メソッドを `clone(...)` に簡素化、(4) 副作用の戻り値を無視しがちな API に `#[\NoDiscard]` を付与。8.5 は概ね後方互換だが、非推奨化や細かな挙動変更はアップグレードガイドで確認を。

## 📖 用語解説（このトピックとのつながり）

- **パイプ演算子 `|>`**: 左辺の値を右辺の callable に流す演算子。処理順＝記述順にできる。
- **第一級 callable（`a(...)`）**: 関数を値として渡す構文。パイプの右辺に使う。
- **readonly / with-er パターン**: 生成後変更不可のプロパティと、その「一部だけ変えたコピー」を作る流儀。`clone(...)` が支援。
- **URI 拡張**: 仕様準拠の URL 解析・正規化を行う組み込み拡張。自前パースの事故を防ぐ。
- **#[\NoDiscard]**: 戻り値の無視を警告する属性。API 誤用対策。

## 影響範囲 / 推奨アクション

- 整形/変換パイプラインや値オブジェクトの記述が簡潔になる。入れ子呼び出しの `|>` 化を検討。
- 自前 URL パースは URI 拡張へ寄せて脆弱性・不整合を削減。
- readonly 値オブジェクトの with 系は `clone(...)` に、副作用 API には `#[\NoDiscard]` を付与。
