# Cisco ISE に CVSS 9.1 の認証後RCE脆弱性 — root奪取と認証基盤の停止リスク

> トピック: 脆弱性ニュース ｜ 出典: Cisco Security Advisory / SecurityWeek / CyberSecurityNews ｜ 種別: リモートコード実行（RCE）・権限昇格・情報漏えい

## 一行サマリー

ネットワーク認証基盤 Cisco ISE（Identity Services Engine）に CVSS 9.1 のRCE脆弱性 CVE-2026-20181 と、未認証で資格情報ハッシュを盗める CVE-2026-20190 が見つかり、悪用されると認証サーバが乗っ取られ社内ネットワークの「入口管理」そのものが崩される。修正版（3.3 Patch 11 / 3.4 Patch 6）へ即時更新が必要。

## 🔰 初学者向け（何が・なぜ問題か）

Cisco ISE は、会社のネットワークに「誰を・どの端末を・どこまで入れるか」を判断する“受付・入館ゲート”のような製品です。社員証をかざすとゲートが開く、あの仕組みのデジタル版だと思ってください。社内のWi-FiやVPN、有線LANに端末がつながるたびに、ISEが「この人は本物か」「この端末は安全か」を判定しています。

今回の問題は、その受付ゲートのコンピュータ自体に侵入できてしまう欠陥が2つ見つかったことです。

ひとつ目（CVE-2026-20181）は、管理者としてログインできる人が細工したリクエストを送ると、受付ゲートのコンピュータの中で好きなプログラムを動かせてしまう、というものです。しかも最初はただの利用者権限でも、そこから最上位の管理権限（root＝建物の全ての鍵を持つ人）まで昇格できます。受付係のフリをして入った人が、いつのまにかビル全体のマスターキーを握る、というイメージです。

ふたつ目（CVE-2026-20190）はもっと手前の段階の問題で、ログインすらしていない外部の人が細工したリクエストを送るだけで、保管されている資格情報（パスワードのハッシュ＝暗号化された合言葉）を覗き見できてしまいます。盗んだハッシュは解読や再利用の足がかりになり、次の攻撃の燃料になります。

なぜ深刻かというと、ISEは「入口の番人」だからです。番人を乗っ取られると、その後ろにある社内システム全体への出入りを攻撃者が自由に許可・拒否できるようになり、被害が一気に組織全体へ広がります。

## 🧠 専門的解説（技術詳細・メカニズム・検知対応）

### 技術的詳細（種別・前提条件）

- **CVE-2026-20181（RCE / 権限昇格）**: CVSS 9.1（Critical）。原因はユーザー入力の不適切な検証（CWE-20: Improper Input Validation の類型）。前提条件は「管理者権限を持つ認証済みアカウント」で、細工したHTTPリクエストを送信することで任意コマンドを実行。初期はユーザーレベル実行でも root へ昇格可能。単一ノード構成では悪用によりDoS（新規エンドポイントが認証不能になる）状態にも至る。
- **CVE-2026-20190（情報漏えい）**: 不適切な認可チェック（CWE-285: Improper Authorization の類型）。**未認証のリモート攻撃者**が細工リクエストでハッシュ化資格情報を含む機微情報へアクセス可能。
- **影響範囲**: Cisco ISE および ISE Passive Identity Connector（ISE-PIC）。構成に依存せず影響を受けるとされる。
- **修正版**: ISE / ISE-PIC 3.3 Patch 11 および 3.4 Patch 6。
- **悪用状況**: 公開時点（2026-06-17）で広範な実環境悪用の確証は「要確認」。ただしISE系は過去に公開後速やかに攻撃対象化された実績があり、優先度は最高。

### メカニズム（攻撃が成立する仕組み）

以下は概念レベルの説明であり、武器化可能な手順・PoC・オフセット等は含めない。

1. **信頼境界の所在**: ISEの管理用Web/APIインターフェースは「認証済み管理者の入力は正当」という暗黙の前提に立つ。CVE-2026-20181はこの前提の検証不備に起因する。
2. **入力検証の欠落（20181）**: 管理操作経由で渡されるパラメータが、OSコマンドやスクリプト文脈へ渡る前に十分にサニタイズ／文脈別エスケープされない。結果、攻撃者は「データのはずの入力」を「命令」として解釈させるプリミティブ（コマンドインジェクション的な任意実行）を握る。
3. **権限の橋渡し**: アプリケーション層で得た実行権限が、特権分離の不備により root 文脈へ橋渡しされる。Web層→OS層→特権昇格という多段の信頼境界越えが、最終的に機器全体の掌握（root RCE）に結実する。
4. **認可欠落の独立経路（20190）**: 別経路として、本来は認証・認可を要求すべきエンドポイントが認可チェックを欠くため、未認証要求で内部資源（資格情報ハッシュ）が読み出せる。これは20181と独立に成立し、攻撃者に「次段の認証情報」を供給する。
5. **被害の連鎖**: 20190で得たハッシュをオフラインクラックや再利用に回し、20181の前提（管理者アクセス）を満たす、という連結も理論上ありうる。情報漏えい→管理者奪取→RCE→認証基盤の完全制御という増幅が成立する。

脆弱性クラスとしては、(a) 特権ヘルパ／管理APIにおける入力検証不備によるコマンドインジェクション、(b) アクセス制御欠落（missing authorization）、という2つの古典的かつ高危険度のクラスに該当する。

### 検知・暫定緩和・恒久対策

- **検知の着眼点**: ISE管理インターフェース（HTTPS/443）への異常なPOST/PUT、管理者アカウントによる想定外の設定変更、ISE上での予期しないプロセス生成・シェル起動、新規ローカルアカウント作成、`root`権限での予期しないコマンド痕跡。GUIログ・CLIの操作監査・syslog外部転送を突き合わせる。20190については、未認証セッションからの機微エンドポイントアクセスの試行ログに注目。
- **暫定緩和（パッチ前）**: ISE管理プレーンへのアクセスを管理用セグメント／許可IPに厳格制限（管理アクセスは決してインターネット露出させない）。管理者アカウントのMFA・最小権限化。不要な管理者アカウントの棚卸し。
- **恒久対策**: 3.3 Patch 11 / 3.4 Patch 6 へ即時更新。更新後は全管理者資格情報のローテーション（20190による漏えい前提）と、漏えいハッシュの再利用監視を実施。
- **運用示唆**: 認証基盤は「侵害されると全社が崩れる単一点」。パッチ適用の緊急度を最上位に置き、管理プレーン分離を恒常的なアーキテクチャ原則とする。

## 📖 用語解説（このニュースとのつながり）

- **Cisco ISE（Identity Services Engine）**: ネットワークアクセス制御（NAC）／認証・認可・アカウンティング（AAA）を担う製品。意味＝「誰が・どの端末で・どこまでつなげるか」を一元管理する装置。本件＝この“入口の番人”自体が乗っ取り対象になった。
- **RCE（Remote Code Execution）**: 攻撃者が遠隔から任意のコードを実行できること。本件＝20181で root 実行に至る最終被害。
- **権限昇格（Privilege Escalation）**: 低い権限から高い権限へ昇格すること。本件＝ユーザーレベル→root。
- **CWE-285 / Improper Authorization（認可不備）**: 本来許可すべきでない操作を許してしまう欠陥。本件＝20190で未認証アクセスを許す原因。
- **資格情報ハッシュ**: パスワードを不可逆変換した値。意味＝そのままでは平文でないが、解読・再利用の足がかりになる。本件＝20190で漏えいする機微情報。
- **DoS（サービス妨害）**: サービスを使用不能にする攻撃。本件＝単一ノード構成で新規エンドポイント認証が停止する副次影響。

## 影響範囲 / 推奨対応

影響を受けるのは Cisco ISE / ISE-PIC を利用する全組織（構成非依存）。社内Wi-Fi・VPN・有線NACの認証をISEに依存している環境では、侵害が認証基盤全体の制御権喪失に直結する。推奨対応は、(1) 3.3 Patch 11 / 3.4 Patch 6 への即時更新、(2) 管理プレーンのネットワーク分離とアクセス制限、(3) 更新後の管理者資格情報ローテーション、(4) 管理インターフェースへの異常アクセス・新規アカウント作成の監視強化。
